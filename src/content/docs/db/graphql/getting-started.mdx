# Getting Started with GraphQL

## Installation

<Steps>
1. **Create a new Node.js project**

   ```bash
   mkdir graphql-api && cd graphql-api
   npm init -y
   npm install @apollo/server graphql
   ```

2. **Configure package.json for ES modules**

   ```json
   {
     "type": "module",
     "scripts": {
       "start": "node index.js",
       "dev": "node --watch index.js"
     }
   }
   ```

3. **Create your first server**

   ```javascript
   // index.js
   import { ApolloServer } from '@apollo/server';
   import { startStandaloneServer } from '@apollo/server/standalone';

   // 1. Define your schema using SDL
   const typeDefs = `#graphql
     type Book {
       id: ID!
       title: String!
       author: String!
       year: Int
     }

     type Query {
       books: [Book!]!
       book(id: ID!): Book
     }

     type Mutation {
       addBook(title: String!, author: String!, year: Int): Book!
     }
   `;

   // 2. In-memory data store
   const books = [
     { id: '1', title: 'The Pragmatic Programmer', author: 'David Thomas', year: 2019 },
     { id: '2', title: 'Clean Code', author: 'Robert Martin', year: 2008 },
   ];

   // 3. Define resolvers (functions that return the data for each field)
   const resolvers = {
     Query: {
       books: () => books,
       book: (_, { id }) => books.find(b => b.id === id),
     },
     Mutation: {
       addBook: (_, { title, author, year }) => {
         const book = { id: String(books.length + 1), title, author, year };
         books.push(book);
         return book;
       },
     },
   };

   // 4. Create and start the server
   const server = new ApolloServer({ typeDefs, resolvers });

   const { url } = await startStandaloneServer(server, {
     listen: { port: 4000 },
   });

   console.log(`🚀 Server ready at: ${url}`);
   ```

4. **Start the server**

   ```bash
   node index.js
   # 🚀 Server ready at: http://localhost:4000/
   ```
</Steps>

## Apollo Sandbox

Navigate to `http://localhost:4000` to open Apollo Sandbox — an in-browser IDE for exploring your API.

```graphql
# Run your first query in the sandbox
query GetAllBooks {
  books {
    id
    title
    author
    year
  }
}

# Query with argument
query GetBook {
  book(id: "1") {
    title
    author
  }
}

# Run a mutation
mutation AddBook {
  addBook(title: "DDIA", author: "Martin Kleppmann", year: 2017) {
    id
    title
  }
}
```

## SDL Type Definitions

The Schema Definition Language (SDL) is GraphQL's type system expressed as a string.

### Scalar Types

```graphql
# Built-in scalar types
scalar Int       # 32-bit signed integer
scalar Float     # Double-precision floating point
scalar String    # UTF-8 string
scalar Boolean   # true or false
scalar ID        # Unique identifier (serialized as String, may be integer)

# Custom scalars (require resolver implementations)
scalar Date      # ISO 8601 date string: "2024-01-15"
scalar DateTime  # ISO 8601 datetime: "2024-01-15T14:30:00Z"
scalar JSON      # Arbitrary JSON value
scalar Upload    # File upload (graphql-upload)
scalar URL       # Valid URL string
scalar Email     # Valid email address
```

### Object Types

```graphql
type User {
  id: ID!
  name: String!
  email: String!
  age: Int
  # Field with arguments
  posts(limit: Int = 10, offset: Int = 0): [Post!]!
  # Computed field (resolved by a resolver function)
  fullName: String!
  # Relationship
  followers: [User!]!
  following: [User!]!
}

type Post {
  id: ID!
  title: String!
  slug: String!
  content: String!
  published: Boolean!
  publishedAt: DateTime
  author: User!
  tags: [String!]!
  # Paginated comments using Relay Connection spec
  commentsConnection(first: Int, after: String): CommentConnection!
}
```

### Non-null and List Modifiers

```graphql
# Modifier combinations and their meanings:
String       # Nullable string
String!      # Non-null string (required)
[String]     # Nullable list of nullable strings
[String!]    # Nullable list of non-null strings
[String]!    # Non-null list of nullable strings
[String!]!   # Non-null list of non-null strings (most common)

# Examples:
type Product {
  id: ID!          # Always present, never null
  name: String!    # Always present
  description: String  # Optional — can be null
  images: [String!]!   # Always a list (can be empty), each item non-null
  tags: [String!]      # Can be null OR a list of non-null strings
}
```

### Enums

```graphql
enum OrderStatus {
  PENDING
  PROCESSING
  SHIPPED
  DELIVERED
  CANCELLED
  REFUNDED
}

enum SortDirection {
  ASC
  DESC
}

# Use in types and arguments
type Order {
  id: ID!
  status: OrderStatus!
  items: [OrderItem!]!
}

type Query {
  orders(
    status: OrderStatus
    sortBy: String
    direction: SortDirection = ASC
  ): [Order!]!
}
```

### Input Types

Input types are used for mutation arguments and complex query filters.

```graphql
# Input types cannot have circular references
# Input types cannot use Object types as fields (only scalars, enums, other inputs)
input CreateUserInput {
  name: String!
  email: String!
  password: String!
  role: UserRole = USER
}

input UpdateUserInput {
  name: String
  email: String
  bio: String
  avatar: URL
}

input UserFilterInput {
  search: String
  role: UserRole
  createdAfter: DateTime
  createdBefore: DateTime
}

input PaginationInput {
  limit: Int = 20
  offset: Int = 0
}

# Use in mutations and queries
type Mutation {
  createUser(input: CreateUserInput!): User!
  updateUser(id: ID!, input: UpdateUserInput!): User!
}

type Query {
  users(filter: UserFilterInput, pagination: PaginationInput): [User!]!
}
```

### Interfaces

```graphql
interface Node {
  id: ID!
}

interface Timestamped {
  createdAt: DateTime!
  updatedAt: DateTime!
}

# Implement interfaces
type User implements Node & Timestamped {
  id: ID!
  name: String!
  email: String!
  createdAt: DateTime!
  updatedAt: DateTime!
}

type Product implements Node & Timestamped {
  id: ID!
  name: String!
  price: Float!
  createdAt: DateTime!
  updatedAt: DateTime!
}

# Query an interface type
type Query {
  node(id: ID!): Node      # Returns any type implementing Node
  nodes(ids: [ID!]!): [Node]!
}
```

### Unions

```graphql
union SearchResult = User | Post | Product | Tag

type Query {
  search(query: String!): [SearchResult!]!
}
```

## Introspection

GraphQL's introspection system allows you to query the schema itself.

```graphql
# Discover all types in the schema
query IntrospectSchema {
  __schema {
    types {
      name
      kind
      description
    }
  }
}

# Get details about a specific type
query IntrospectUser {
  __type(name: "User") {
    name
    fields {
      name
      type {
        name
        kind
        ofType {
          name
          kind
        }
      }
    }
  }
}

# Get all available queries
query GetQueryFields {
  __type(name: "Query") {
    fields {
      name
      description
      args {
        name
        type { name kind }
      }
    }
  }
}
```

:::caution[Disable Introspection in Production]
Introspection reveals your entire schema to anyone who can reach the endpoint. Disable it in production to prevent schema scraping.
```javascript
new ApolloServer({
  typeDefs,
  resolvers,
  introspection: process.env.NODE_ENV !== 'production',
})
```
:::

## GraphQL Over HTTP

The [graphql-over-http spec](https://graphql.github.io/graphql-over-http/) standardizes how GraphQL works over HTTP.

### POST Request (Standard)

```bash
# POST with JSON body
curl -X POST http://localhost:4000/graphql \
  -H "Content-Type: application/json" \
  -d '{
    "query": "query GetBook($id: ID!) { book(id: $id) { title author } }",
    "variables": { "id": "1" },
    "operationName": "GetBook"
  }'

# Response:
# { "data": { "book": { "title": "...", "author": "..." } } }
```

### GET Request (For Caching)

```bash
# GET with query parameters (suitable for CDN caching)
curl "http://localhost:4000/graphql?\
query=%7B%20books%20%7B%20title%20%7D%20%7D\
&operationName=GetBooks"

# URL-decoded query: { books { title } }
```

### Persisted Queries (APQ)

Automatic Persisted Queries reduce bandwidth by sending only the query hash on repeated requests.

```bash
# Step 1: Send hash first (optimistic)
curl -X GET "http://localhost:4000/graphql?\
extensions=%7B%22persistedQuery%22%3A%7B%22version%22%3A1%2C\
%22sha256Hash%22%3A%22abc123...%22%7D%7D"
# Returns PersistedQueryNotFound if not cached

# Step 2: Send full query with hash (registers it)
curl -X POST http://localhost:4000/graphql \
  -H "Content-Type: application/json" \
  -d '{
    "query": "{ books { title } }",
    "extensions": {
      "persistedQuery": { "version": 1, "sha256Hash": "abc123..." }
    }
  }'

# Step 3: Subsequent requests use hash only
# Server returns cached result, GET is cacheable by CDN
```

## WebSockets for Subscriptions

Subscriptions require a persistent connection. The `graphql-ws` library is the modern standard.

```bash
npm install graphql-ws ws @graphql-tools/schema
```

```javascript
// subscriptions-server.js
import { createServer } from 'http';
import { WebSocketServer } from 'ws';
import { useServer } from 'graphql-ws/lib/use/ws';
import { makeExecutableSchema } from '@graphql-tools/schema';
import { PubSub } from 'graphql-subscriptions';

const pubsub = new PubSub();
const MESSAGE_ADDED = 'MESSAGE_ADDED';

const typeDefs = `#graphql
  type Message {
    id: ID!
    text: String!
    user: String!
    createdAt: String!
  }

  type Query {
    messages: [Message!]!
  }

  type Mutation {
    sendMessage(text: String!, user: String!): Message!
  }

  type Subscription {
    messageAdded: Message!
  }
`;

const resolvers = {
  Query: {
    messages: () => [],
  },
  Mutation: {
    sendMessage: (_, { text, user }) => {
      const message = {
        id: String(Date.now()),
        text,
        user,
        createdAt: new Date().toISOString(),
      };
      pubsub.publish(MESSAGE_ADDED, { messageAdded: message });
      return message;
    },
  },
  Subscription: {
    messageAdded: {
      subscribe: () => pubsub.asyncIterator([MESSAGE_ADDED]),
    },
  },
};

const schema = makeExecutableSchema({ typeDefs, resolvers });
const httpServer = createServer();
const wsServer = new WebSocketServer({ server: httpServer, path: '/graphql' });

useServer({ schema }, wsServer);

httpServer.listen(4000, () => {
  console.log('WebSocket server ready at ws://localhost:4000/graphql');
});
```

:::tip[graphql-ws vs subscriptions-transport-ws]
Use `graphql-ws` (newer, maintained) not `subscriptions-transport-ws` (old, deprecated). Apollo Client and most modern clients support both — configure Apollo Client to use `GraphQLWsLink` with the `graphql-ws` package.
:::

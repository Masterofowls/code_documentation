## Minimal Apollo Server

```javascript
// server.js
import { ApolloServer } from '@apollo/server';
import { startStandaloneServer } from '@apollo/server/standalone';

// 1. Define the schema using SDL (Schema Definition Language)
const typeDefs = `#graphql
  type Book {
    id: ID!
    title: String!
    author: String!
    publishedYear: Int
    genre: String
  }

  type Query {
    books: [Book!]!
    book(id: ID!): Book
  }
`;

// 2. Sample data
const books = [
  {
    id: '1',
    title: 'The Pragmatic Programmer',
    author: 'David Thomas',
    publishedYear: 1999,
    genre: 'Technology',
  },
  {
    id: '2',
    title: 'Clean Code',
    author: 'Robert Martin',
    publishedYear: 2008,
    genre: 'Technology',
  },
  {
    id: '3',
    title: 'Design Patterns',
    author: 'Gang of Four',
    publishedYear: 1994,
    genre: 'Technology',
  },
];

// 3. Define resolvers — functions that return data for each field
const resolvers = {
  Query: {
    books: () => books,
    book: (_, { id }) => books.find((b) => b.id === id),
  },
};

// 4. Create and start the server
const server = new ApolloServer({ typeDefs, resolvers });

const { url } = await startStandaloneServer(server, {
  listen: { port: 4000 },
});

console.log(`🚀 Server ready at ${url}`);
// Open http://localhost:4000 for Apollo Sandbox
```

Run it:

```bash
node server.js
# or with live-reload:
npx tsx --watch server.js
```

## Schema Definition Language (SDL)

GraphQL schemas are written in **SDL** (Schema Definition Language). The schema is the contract between client and server.

### Types

```graphql
# Object type — the most common type
type User {
  id: ID! # Non-null ID
  name: String! # Non-null string
  email: String!
  bio: String # Nullable string
  age: Int
  score: Float
  isVerified: Boolean!
  role: UserRole! # Enum
  posts: [Post!]! # Non-null list of non-null Posts
  createdAt: String!
}

# Enum type
enum UserRole {
  ADMIN
  MODERATOR
  USER
  GUEST
}

# Input type (used for mutation arguments)
input CreateUserInput {
  name: String!
  email: String!
  password: String!
  role: UserRole = USER # Default value
}

# Interface
interface Node {
  id: ID!
}

# Types implementing an interface
type Post implements Node {
  id: ID!
  title: String!
  content: String!
  author: User!
  publishedAt: String
}

type Comment implements Node {
  id: ID!
  body: String!
  author: User!
  post: Post!
}
```

### The Query Type

The `Query` type is the entry point for all read operations. Every GraphQL server must define a `Query` type.

```graphql
type Query {
  # Simple field
  me: User

  # With required argument
  user(id: ID!): User

  # With optional arguments
  users(
    limit: Int = 20
    offset: Int = 0
    role: UserRole
    search: String
  ): [User!]!

  # Deeply nested
  post(id: ID!): Post
  posts(authorId: ID, first: Int, after: String): PostConnection!
}
```

### The Mutation Type

```graphql
type Mutation {
  createUser(input: CreateUserInput!): CreateUserPayload!
  updateUser(id: ID!, input: UpdateUserInput!): UpdateUserPayload!
  deleteUser(id: ID!): DeleteUserPayload!
  login(email: String!, password: String!): AuthPayload!
}

type CreateUserPayload {
  user: User
  errors: [UserError!]!
}

type UserError {
  field: String
  message: String!
  code: String!
}
```

### The Subscription Type

```graphql
type Subscription {
  messageAdded(channelId: ID!): Message!
  userTyping(channelId: ID!): TypingIndicator!
  orderStatusChanged(orderId: ID!): Order!
}
```

## Your First Query

With the server running, open **Apollo Sandbox** at `http://localhost:4000`.

```graphql
# Query all books
query GetAllBooks {
  books {
    id
    title
    author
    publishedYear
  }
}

# Query a specific book
query GetBook($bookId: ID!) {
  book(id: $bookId) {
    id
    title
    author
    genre
  }
}
```

Variables panel (for the second query):

```json
{
  "bookId": "1"
}
```

## Introspection

Introspection is a built-in GraphQL feature that lets you query the schema itself.

```graphql
# Get all types in the schema
query IntrospectSchema {
  __schema {
    types {
      name
      kind
      description
    }
  }
}

# Get fields of a specific type
query IntrospectUser {
  __type(name: "User") {
    name
    fields {
      name
      type {
        name
        kind
        ofType {
          name
          kind
        }
      }
      isDeprecated
      deprecationReason
    }
  }
}

# Get available queries
query IntrospectQueryType {
  __schema {
    queryType {
      fields {
        name
        description
        args {
          name
          type {
            name
            kind
          }
        }
      }
    }
  }
}
```

:::caution[Disable Introspection in Production]
Introspection exposes your entire schema to anyone who can reach the endpoint. Disable it in production for security:

```javascript
const server = new ApolloServer({
  typeDefs,
  resolvers,
  introspection: process.env.NODE_ENV !== 'production',
});
```

:::

## Field Arguments and Non-Null/List Modifiers

```graphql
type Query {
  # String  — nullable string
  # String! — non-null string (must return a value, never null)
  # [String] — nullable list of nullable strings
  # [String!] — nullable list of non-null strings
  # [String]! — non-null list of nullable strings
  # [String!]! — non-null list of non-null strings (most common for collections)

  users(first: Int = 20, after: String): [User!]!
  user(id: ID!): User # nullable — returns null if not found (not an error)
}
```

### Non-Null and List Combinations

| Type         | Meaning                               |
| ------------ | ------------------------------------- |
| `String`     | Can be null                           |
| `String!`    | Cannot be null                        |
| `[String]`   | Can be null, may contain nulls        |
| `[String!]`  | Can be null, but items cannot be null |
| `[String]!`  | Cannot be null, items can be null     |
| `[String!]!` | Cannot be null, items cannot be null  |

## GraphQL over HTTP

GraphQL uses a single HTTP endpoint (typically `/graphql`).

### POST request (standard)

```bash
curl -X POST http://localhost:4000/ \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "query GetBook($id: ID!) { book(id: $id) { title author } }",
    "variables": { "id": "1" },
    "operationName": "GetBook"
  }'
```

Response:

```json
{
  "data": {
    "book": {
      "title": "The Pragmatic Programmer",
      "author": "David Thomas"
    }
  }
}
```

### GET request (for queries, enables CDN caching)

```bash
curl "http://localhost:4000/?query={books{title}}&operationName=GetBooks"
```

### Error responses

GraphQL always returns HTTP 200 (even for errors). Errors are in the `errors` array:

```json
{
  "data": null,
  "errors": [
    {
      "message": "Book not found",
      "locations": [{ "line": 2, "column": 3 }],
      "path": ["book"],
      "extensions": {
        "code": "NOT_FOUND"
      }
    }
  ]
}
```

## Persisted Queries

Persisted queries send a hash instead of the full query string, reducing bandwidth and enabling GET-based CDN caching.

<Steps>
1. Client sends the query to the server and stores the hash
2. On subsequent requests, client sends only `{ "extensions": { "persistedQuery": { "version": 1, "sha256Hash": "abc123..." } } }`
3. Server looks up the query by hash
4. If not found, client resends with full query text
</Steps>

```javascript
// Apollo Client automatic persisted queries (APQ)
import { ApolloClient, InMemoryCache, createHttpLink } from '@apollo/client';
import { createPersistedQueryLink } from '@apollo/client/link/persisted-queries';
import { sha256 } from 'crypto-hash';

const persistedLink = createPersistedQueryLink({ sha256 });
const httpLink = createHttpLink({ uri: '/graphql' });

const client = new ApolloClient({
  cache: new InMemoryCache(),
  link: persistedLink.concat(httpLink),
});
```

## Subscriptions with graphql-ws

```bash
npm install graphql-ws ws
```

```javascript
// Server setup
import { ApolloServer } from '@apollo/server';
import { expressMiddleware } from '@apollo/server/express4';
import { ApolloServerPluginDrainHttpServer } from '@apollo/server/plugin/drainHttpServer';
import { makeExecutableSchema } from '@graphql-tools/schema';
import { WebSocketServer } from 'ws';
import { useServer } from 'graphql-ws/lib/use/ws';
import express from 'express';
import { createServer } from 'http';
import { PubSub } from 'graphql-subscriptions';

const pubsub = new PubSub();

const typeDefs = `#graphql
  type Message { id: ID!, content: String!, author: String! }
  type Query { messages: [Message!]! }
  type Mutation { sendMessage(content: String!, author: String!): Message! }
  type Subscription { messageSent: Message! }
`;

const resolvers = {
  Mutation: {
    sendMessage: (_, { content, author }) => {
      const message = { id: String(Date.now()), content, author };
      pubsub.publish('MESSAGE_SENT', { messageSent: message });
      return message;
    },
  },
  Subscription: {
    messageSent: {
      subscribe: () => pubsub.asyncIterator(['MESSAGE_SENT']),
    },
  },
};

const schema = makeExecutableSchema({ typeDefs, resolvers });
const app = express();
const httpServer = createServer(app);

const wsServer = new WebSocketServer({ server: httpServer, path: '/graphql' });
const serverCleanup = useServer({ schema }, wsServer);

const server = new ApolloServer({
  schema,
  plugins: [
    ApolloServerPluginDrainHttpServer({ httpServer }),
    {
      async serverWillStart() {
        return {
          async drainServer() {
            await serverCleanup.dispose();
          },
        };
      },
    },
  ],
});

await server.start();
app.use('/graphql', expressMiddleware(server));
httpServer.listen(4000, () =>
  console.log('🚀 Server ready at http://localhost:4000/graphql'),
);
```

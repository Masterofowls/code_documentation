# GraphQL Queries & Mutations

## Query Syntax

### Basic Query

```graphql
# Simple query — no operation name, no variables (fine for one-offs)
{
  books {
    title
    author
  }
}

# Named query with operation name (required when sending variables or batching)
query GetAllBooks {
  books {
    id
    title
    author
    publishedYear
  }
}
```

### Query with Variables

Variables are declared in the operation signature and passed separately from the query document.

```graphql
# Variables are declared with $ prefix and their type
query GetUser($id: ID!, $withPosts: Boolean = false) {
  user(id: $id) {
    id
    name
    email
    posts @include(if: $withPosts) {
      title
      publishedAt
    }
  }
}
```

Variables object (sent separately):

```json
{
  "id": "user_abc123",
  "withPosts": true
}
```

### Nested Queries

```graphql
query GetOrderDetails($orderId: ID!) {
  order(id: $orderId) {
    id
    status
    total
    createdAt
    customer {
      name
      email
      shippingAddress {
        street
        city
        country
        postalCode
      }
    }
    lineItems {
      quantity
      unitPrice
      subtotal
      product {
        id
        name
        sku
        images(first: 1) {
          url
          altText
        }
      }
    }
    shipment {
      trackingNumber
      carrier
      estimatedDelivery
      events(last: 3) {
        status
        location
        timestamp
      }
    }
  }
}
```

### Aliases

Aliases let you rename fields or run the same query twice with different arguments.

```graphql
query CompareProducts($id1: ID!, $id2: ID!) {
  productA: product(id: $id1) {
    name
    price
    rating
  }
  productB: product(id: $id2) {
    name
    price
    rating
  }
}

# Multiple queries in one request
query Dashboard {
  recentOrders: orders(first: 5, sortBy: CREATED_AT, sortOrder: DESC) {
    id
    total
    status
  }
  pendingOrders: orders(status: PENDING, first: 10) {
    id
    total
    customer {
      name
    }
  }
}
```

### Fragments

Named fragments allow you to reuse field selections across multiple queries.

```graphql
# Define a fragment
fragment UserBasicFields on User {
  id
  name
  email
  avatarUrl
  createdAt
}

fragment PostSummary on Post {
  id
  title
  slug
  publishedAt
  author {
    ...UserBasicFields
  }
}

# Use the fragment
query GetPostsWithAuthors {
  posts(first: 20) {
    ...PostSummary
    category {
      name
    }
    tags
  }
}

query GetProfile($userId: ID!) {
  user(id: $userId) {
    ...UserBasicFields
    bio
    followerCount
    posts(first: 5) {
      ...PostSummary
    }
  }
}
```

### Inline Fragments

Inline fragments handle polymorphic types (unions and interfaces):

```graphql
query SearchEverything($query: String!) {
  search(query: $query) {
    __typename # always include __typename for union types
    ... on Product {
      id
      name
      price
      category {
        name
      }
    }
    ... on Article {
      id
      title
      excerpt
      publishedAt
      author {
        name
      }
    }
    ... on User {
      id
      name
      bio
    }
  }
}
```

### @skip and @include Directives

```graphql
query GetUserProfile(
  $userId: ID!
  $includePrivate: Boolean!
  $skipActivity: Boolean!
) {
  user(id: $userId) {
    name
    publicBio: bio

    # Only include if includePrivate = true
    email @include(if: $includePrivate)
    phone @include(if: $includePrivate)

    # Skip if skipActivity = true
    recentActivity @skip(if: $skipActivity) {
      action
      timestamp
    }
  }
}
```

## Mutations

### Single Mutation

```graphql
mutation CreateUser($input: CreateUserInput!) {
  createUser(input: $input) {
    user {
      id
      name
      email
      role
      createdAt
    }
    errors {
      field
      message
      code
    }
  }
}
```

Variables:

```json
{
  "input": {
    "name": "Alice Smith",
    "email": "alice@example.com",
    "password": "SecurePass123!",
    "role": "USER"
  }
}
```

### Multiple Mutations

Multiple mutation fields execute **sequentially** (not in parallel), unlike queries.

```graphql
mutation SetupAccount(
  $userInput: CreateUserInput!
  $profileInput: CreateProfileInput!
) {
  user: createUser(input: $userInput) {
    user {
      id
    }
    errors {
      message
    }
  }
  profile: createProfile(input: $profileInput) {
    profile {
      id
    }
    errors {
      message
    }
  }
}
```

### Mutation Error Patterns

There are two main patterns for handling errors in mutations:

<Tabs>
  <TabItem label="Union Result Type (recommended)">
    ```graphql
    # Schema
    union CreateUserResult = User | UserAlreadyExistsError | ValidationError

    type UserAlreadyExistsError {
      message: String!
      existingUserId: ID!
    }

    type ValidationError {
      message: String!
      field: String!
    }

    type Mutation {
      createUser(input: CreateUserInput!): CreateUserResult!
    }

    # Query
    mutation CreateUser($input: CreateUserInput!) {
      createUser(input: $input) {
        ... on User {
          id
          name
          email
        }
        ... on UserAlreadyExistsError {
          message
          existingUserId
        }
        ... on ValidationError {
          message
          field
        }
      }
    }
    ```

</TabItem>
  <TabItem label="Payload with errors array">
    ```graphql
    # Schema
    type CreateUserPayload {
      user: User               # null on error
      errors: [UserError!]!   # empty on success
    }

    type UserError {
      field: String            # null for general errors
      message: String!
      code: String!
    }

    # Query — check both user and errors
    mutation CreateUser($input: CreateUserInput!) {
      createUser(input: $input) {
        user {
          id
          name
        }
        errors {
          field
          message
          code
        }
      }
    }
    ```

</TabItem>
</Tabs>

## Pagination Patterns

### Offset/Limit (Simple)

```graphql
query GetProducts($limit: Int!, $offset: Int!) {
  products(limit: $limit, offset: $offset) {
    id
    name
    price
  }
  productCount
}
```

Variables: `{ "limit": 20, "offset": 40 }` for page 3 of 20 items per page.

### Cursor-Based (Relay Connection)

```graphql
query GetProducts($first: Int!, $after: String, $category: String) {
  products(first: $first, after: $after, category: $category) {
    edges {
      cursor
      node {
        id
        name
        price
        category {
          name
        }
      }
    }
    pageInfo {
      hasNextPage
      hasPreviousPage
      startCursor
      endCursor
    }
    totalCount
  }
}
```

```javascript
// Load more / infinite scroll with Apollo Client
const { data, fetchMore } = useQuery(GET_PRODUCTS, {
  variables: { first: 20 },
});

const loadMore = () => {
  fetchMore({
    variables: {
      first: 20,
      after: data.products.pageInfo.endCursor,
    },
    updateQuery: (prev, { fetchMoreResult }) => {
      if (!fetchMoreResult) return prev;
      return {
        products: {
          ...fetchMoreResult.products,
          edges: [...prev.products.edges, ...fetchMoreResult.products.edges],
        },
      };
    },
  });
};
```

## Apollo Client — React Integration

### Setup

```javascript
// apollo-client.js
import { ApolloClient, InMemoryCache, HttpLink, split } from '@apollo/client';
import { GraphQLWsLink } from '@apollo/client/link/subscriptions';
import { createClient } from 'graphql-ws';
import { getMainDefinition } from '@apollo/client/utilities';

const httpLink = new HttpLink({ uri: '/graphql' });

const wsLink = new GraphQLWsLink(
  createClient({
    url: 'ws://localhost:4000/graphql',
    connectionParams: () => ({
      authorization: `Bearer ${localStorage.getItem('token')}`,
    }),
  }),
);

// Route queries/mutations to HTTP, subscriptions to WebSocket
const splitLink = split(
  ({ query }) => {
    const def = getMainDefinition(query);
    return (
      def.kind === 'OperationDefinition' && def.operation === 'subscription'
    );
  },
  wsLink,
  httpLink,
);

export const client = new ApolloClient({
  link: splitLink,
  cache: new InMemoryCache({
    typePolicies: {
      Query: {
        fields: {
          products: {
            // Merge paginated results
            keyArgs: ['category'],
            merge(existing = { edges: [] }, incoming) {
              return {
                ...incoming,
                edges: [...existing.edges, ...incoming.edges],
              };
            },
          },
        },
      },
    },
  }),
});
```

### useQuery Hook

```javascript
import { useQuery, gql } from '@apollo/client';

const GET_USERS = gql`
  query GetUsers($role: UserRole) {
    users(role: $role) {
      id
      name
      email
      role
    }
  }
`;

function UserList({ role }) {
  const { data, loading, error, refetch, networkStatus } = useQuery(GET_USERS, {
    variables: { role },
    fetchPolicy: 'cache-and-network', // use cache but also update from network
    pollInterval: 30000, // refetch every 30s
    skip: !role, // skip if role is null
    onCompleted: (data) => console.log('Users loaded:', data.users.length),
    onError: (error) => console.error('Failed to load users:', error),
  });

  if (loading) return <Spinner />;
  if (error) return <ErrorMessage message={error.message} />;

  return (
    <ul>
      {data.users.map((user) => (
        <li key={user.id}>
          {user.name} ({user.email})
        </li>
      ))}
    </ul>
  );
}
```

### useMutation Hook

```javascript
import { useMutation, gql } from '@apollo/client';

const CREATE_USER = gql`
  mutation CreateUser($input: CreateUserInput!) {
    createUser(input: $input) {
      user {
        id
        name
        email
      }
      errors {
        field
        message
      }
    }
  }
`;

function CreateUserForm() {
  const [createUser, { loading, error }] = useMutation(CREATE_USER, {
    // Refetch the users list after creating a user
    refetchQueries: ['GetUsers'],
    // Or update the cache directly
    update(cache, { data: { createUser } }) {
      if (!createUser.user) return;
      cache.modify({
        fields: {
          users(existingUsers = []) {
            const newUserRef = cache.writeFragment({
              data: createUser.user,
              fragment: gql`
                fragment NewUser on User {
                  id
                  name
                  email
                }
              `,
            });
            return [...existingUsers, newUserRef];
          },
        },
      });
    },
    // Optimistic response
    optimisticResponse: ({ input }) => ({
      createUser: {
        __typename: 'CreateUserPayload',
        user: {
          __typename: 'User',
          id: 'temp-id-' + Date.now(),
          name: input.name,
          email: input.email,
        },
        errors: [],
      },
    }),
  });

  const handleSubmit = async (formData) => {
    const { data } = await createUser({ variables: { input: formData } });
    if (data.createUser.errors.length > 0) {
      // Handle field errors
    }
  };
}
```

### useSubscription Hook

```javascript
import { useSubscription, gql } from '@apollo/client';

const MESSAGE_SUBSCRIPTION = gql`
  subscription OnMessageAdded($channelId: ID!) {
    messageAdded(channelId: $channelId) {
      id
      content
      createdAt
      author {
        id
        name
        avatarUrl
      }
    }
  }
`;

function MessageFeed({ channelId }) {
  const [messages, setMessages] = useState([]);

  useSubscription(MESSAGE_SUBSCRIPTION, {
    variables: { channelId },
    onData: ({ data }) => {
      setMessages((prev) => [...prev, data.data.messageAdded]);
    },
    onError: (err) => console.error('Subscription error:', err),
  });

  return (
    <div>
      {messages.map((m) => (
        <Message key={m.id} {...m} />
      ))}
    </div>
  );
}
```

## Fragment Colocation

Co-locate fragments with the components that use them for better maintainability:

```javascript
// UserCard.jsx
const USER_CARD_FRAGMENT = gql`
  fragment UserCard_user on User {
    id
    name
    email
    avatarUrl
    role
  }
`;

export const UserCard = ({ user }) => (
  <div>
    <img src={user.avatarUrl} alt={user.name} />
    <h2>{user.name}</h2>
    <p>{user.email}</p>
  </div>
);
UserCard.fragments = { user: USER_CARD_FRAGMENT };

// UserList.jsx — uses the colocated fragment
const GET_USERS = gql`
  query GetUsers {
    users {
      ...UserCard_user # use the fragment from UserCard
    }
  }
  ${UserCard.fragments.user}
  # spread the fragment definition
`;
```

## HTTP Caching GraphQL

```javascript
// Use GET requests for cacheable queries
const httpLink = new HttpLink({
  uri: '/graphql',
  useGETForQueries: true, // Apollo Client: use GET for queries
});

// Response caching with persisted queries + CDN
// GET /graphql?extensions={"persistedQuery":{"version":1,"sha256Hash":"abc123..."}}
// Cache-Control: public, max-age=60, s-maxage=300
// Vary: Accept-Encoding

// Apollo Server cache hints
const typeDefs = `#graphql
  type Product @cacheControl(maxAge: 300, scope: PUBLIC) {
    id: ID!
    name: String! @cacheControl(maxAge: 3600)
    price: Float! @cacheControl(maxAge: 30)   # prices change often
  }

  type User @cacheControl(maxAge: 0, scope: PRIVATE) {
    id: ID!
    name: String!
    email: String!
  }
`;
```

:::tip[CDN Caching with GraphQL]
Combine persisted queries + GET requests + `@cacheControl` directives to serve public GraphQL queries from a CDN. This is how large-scale GraphQL APIs (GitHub, Shopify) achieve sub-millisecond response times for catalog data.
:::

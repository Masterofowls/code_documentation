# GraphQL Resolvers

Resolvers are the functions that populate the data for each field in your GraphQL schema. Understanding how they work is essential to building correct, efficient GraphQL APIs.

## Resolver Signature

Every resolver function receives four arguments:

```javascript
const resolvers = {
  TypeName: {
    fieldName: (parent, args, context, info) => {
      // parent  — the return value of the parent resolver
      // args    — the arguments provided to this field
      // context — shared object across all resolvers in a request
      // info    — field metadata (AST, schema, path, return type)
    },
  },
};
```

| Parameter | Also Called   | Description                                                                                                                      |
| --------- | ------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `parent`  | `root`, `obj` | Return value of the parent field resolver. Root resolvers (Query/Mutation) receive the root value (usually `{}` or `undefined`). |
| `args`    | `args`        | Object containing the arguments passed to the field in the query.                                                                |
| `context` | `ctx`         | Shared object available to all resolvers in a single request. Used for auth, data sources, database connections, etc.            |
| `info`    | `info`        | GraphQL execution metadata: field name, return type, schema, path, selection set, etc.                                           |

## Resolver Chain

When a query is executed, resolvers are called depth-first:

```graphql
query {
  user(id: "1") {
    # 1. Query.user resolver called with args={id:"1"}
    name # 2. User.name resolver called with parent=<user object>
    posts {
      # 3. User.posts resolver called with parent=<user object>
      title # 4. Post.title resolver called with parent=<post object>
      author {
        # 5. Post.author resolver called with parent=<post object>
        name # 6. User.name resolver called with parent=<author object>
      }
    }
  }
}
```

```javascript
const resolvers = {
  Query: {
    // parent = {} (root value), args = { id: "1" }
    user: async (_, { id }, { db }) => {
      return db.users.findById(id);
    },
  },

  User: {
    // parent = { id: "1", name: "Alice", ... } (return value from Query.user)
    posts: async (parent, _, { db }) => {
      return db.posts.findByUserId(parent.id);
    },
  },

  Post: {
    // parent = { id: "101", title: "...", authorId: "1", ... }
    author: async (parent, _, { db }) => {
      return db.users.findById(parent.authorId);
    },
  },
};
```

## Default Resolvers

If you don't define a resolver for a field, GraphQL uses the **default resolver**: `(parent) => parent[fieldName]`. This means:

```javascript
// You DON'T need these trivial resolvers:
const resolvers = {
  User: {
    id: (user) => user.id, // ← redundant, same as default
    name: (user) => user.name, // ← redundant
    email: (user) => user.email, // ← redundant
  },
};

// ONLY define resolvers where the logic is non-trivial:
const resolvers = {
  User: {
    // Computed field: not a direct database property
    fullName: (user) => `${user.firstName} ${user.lastName}`,
    // Relationship: requires a database query
    posts: (user, _, { db }) => db.posts.findByUserId(user.id),
  },
};
```

## Context Object

The context is created fresh for each request and shared across all resolvers in that request.

```javascript
// Building context in Apollo Server 4
const server = new ApolloServer({ typeDefs, resolvers });

const { url } = await startStandaloneServer(server, {
  context: async ({ req }) => {
    // Parse auth token
    const token = req.headers.authorization?.replace('Bearer ', '');
    let currentUser = null;

    if (token) {
      try {
        const payload = jwt.verify(token, process.env.JWT_SECRET);
        currentUser = await db.users.findById(payload.userId);
      } catch {
        // Invalid token — leave currentUser as null
      }
    }

    return {
      // Database connection
      db,
      // Authenticated user (or null)
      currentUser,
      // Data loaders (created per-request for per-request caching)
      loaders: createLoaders(db),
      // Request metadata
      requestId: req.headers['x-request-id'] ?? crypto.randomUUID(),
    };
  },
});
```

## The N+1 Problem

The N+1 problem is the most common performance pitfall in GraphQL.

```graphql
query {
  posts {
    # 1 query: SELECT * FROM posts  → returns 100 posts
    title
    author {
      # 100 queries: SELECT * FROM users WHERE id = ?
      name # one per post! = 101 total queries
    }
  }
}
```

```javascript
// ❌ N+1 resolver (naive implementation)
const resolvers = {
  Query: {
    posts: (_, __, { db }) => db.query('SELECT * FROM posts LIMIT 100'),
  },
  Post: {
    author: (post, _, { db }) =>
      // This fires once per post → 100 separate queries!
      db.query('SELECT * FROM users WHERE id = ?', [post.authorId]),
  },
};
```

## DataLoader: Batching and Caching

DataLoader solves N+1 by **batching** multiple loads into a single request and **caching** results within a single request.

```bash
npm install dataloader
```

```javascript
import DataLoader from 'dataloader';

// DataLoader batch function: receives an array of keys, returns array of values
// The returned array MUST be the same length and same order as the keys array
const createUserLoader = (db) =>
  new DataLoader(
    async (userIds) => {
      // One query for ALL user IDs
      const users = await db.query('SELECT * FROM users WHERE id IN (?)', [
        userIds,
      ]);

      // Return in same order as userIds (DataLoader requirement!)
      const userMap = Object.fromEntries(users.map((u) => [u.id, u]));
      return userIds.map((id) => userMap[id] ?? null);
    },
    {
      // Options
      maxBatchSize: 1000, // max IDs per batch query
      cache: true, // cache within request (default: true)
    },
  );

// Create loaders fresh per request (so cache is per-request)
const createLoaders = (db) => ({
  users: createUserLoader(db),
  posts: new DataLoader(async (postIds) => {
    const posts = await db.query('SELECT * FROM posts WHERE id IN (?)', [
      postIds,
    ]);
    const postMap = Object.fromEntries(posts.map((p) => [p.id, p]));
    return postIds.map((id) => postMap[id] ?? null);
  }),
  postsByUser: new DataLoader(async (userIds) => {
    const posts = await db.query(
      'SELECT * FROM posts WHERE author_id IN (?) ORDER BY created_at DESC',
      [userIds],
    );
    // Group posts by user_id
    const grouped = userIds.map((userId) =>
      posts.filter((p) => p.authorId === userId),
    );
    return grouped;
  }),
});

// In resolvers
const resolvers = {
  Post: {
    // ✅ Uses DataLoader — batched into single query
    author: (post, _, { loaders }) => loaders.users.load(post.authorId),
  },
  User: {
    // ✅ Batches all post lookups for multiple users
    posts: (user, _, { loaders }) => loaders.postsByUser.load(user.id),
  },
};

// DataLoader advanced methods
loader.loadMany(['id1', 'id2', 'id3']); // batch load multiple keys
loader.prime('id4', value); // manually prime the cache
loader.clear('id1'); // remove from cache
loader.clearAll(); // clear entire cache
```

## Error Handling

GraphQL errors come in two flavors:

1. **Unexpected errors** — unhandled exceptions (exposed as generic "Internal server error" in production)
2. **Expected errors** — business logic errors (user not found, insufficient permissions)

```javascript
import { GraphQLError } from 'graphql';

const resolvers = {
  Query: {
    user: async (_, { id }, { db, currentUser }) => {
      // Authorization check
      if (!currentUser) {
        throw new GraphQLError('You must be logged in to view users', {
          extensions: {
            code: 'UNAUTHENTICATED',
            http: { status: 401 },
          },
        });
      }

      const user = await db.users.findById(id);

      if (!user) {
        throw new GraphQLError(`User with id "${id}" not found`, {
          extensions: {
            code: 'NOT_FOUND',
            argumentName: 'id',
          },
        });
      }

      if (!currentUser.canViewUser(user)) {
        throw new GraphQLError('You are not authorized to view this user', {
          extensions: {
            code: 'FORBIDDEN',
            http: { status: 403 },
          },
        });
      }

      return user;
    },
  },
};

// Error response:
// {
//   "data": { "user": null },
//   "errors": [{
//     "message": "User with id \"999\" not found",
//     "locations": [{"line": 2, "column": 3}],
//     "path": ["user"],
//     "extensions": { "code": "NOT_FOUND", "argumentName": "id" }
//   }]
// }
```

### Error Formatting

```javascript
// Apollo Server 4: format errors before sending to client
const server = new ApolloServer({
  typeDefs,
  resolvers,
  formatError: (formattedError, error) => {
    // Hide internal error details in production
    if (process.env.NODE_ENV === 'production') {
      if (formattedError.extensions?.code === 'INTERNAL_SERVER_ERROR') {
        // Log the original error
        console.error('Internal error:', error);
        return {
          message: 'Internal server error',
          extensions: { code: 'INTERNAL_SERVER_ERROR' },
        };
      }
    }
    return formattedError;
  },
});
```

## Abstract Type Resolvers

For interfaces and unions, you need `__resolveType` to tell GraphQL which concrete type an object is.

```javascript
const resolvers = {
  // Union resolver
  SearchResult: {
    __resolveType(obj, context, info) {
      // You can add __typename when the objects are created
      if (obj.__typename) return obj.__typename;

      // Or inspect the object's properties
      if (obj.price !== undefined) return 'Product';
      if (obj.body !== undefined) return 'Article';
      if (obj.email !== undefined) return 'User';

      return null; // GraphQL will throw if null
    },
  },

  // Interface resolver
  Node: {
    __resolveType(obj) {
      // Use a type discriminator field
      return obj.__typename ?? obj.type ?? null;
    },
  },
};
```

## Mutation Resolvers

```javascript
const resolvers = {
  Mutation: {
    createUser: async (_, { input }, { db, currentUser }) => {
      // Authorization
      if (!currentUser?.isAdmin) {
        throw new GraphQLError('Only admins can create users', {
          extensions: { code: 'FORBIDDEN' },
        });
      }

      // Input validation (use a validation library like zod)
      const { name, email, password, role } = input;
      if (!email.includes('@')) {
        throw new GraphQLError('Invalid email address', {
          extensions: { code: 'BAD_USER_INPUT', argumentName: 'email' },
        });
      }

      // Check uniqueness
      const existing = await db.users.findByEmail(email);
      if (existing) {
        // Return errors in payload (preferred over throwing)
        return {
          user: null,
          errors: [
            {
              field: 'email',
              message: 'Email already in use',
              code: 'DUPLICATE_EMAIL',
            },
          ],
        };
      }

      // Create user
      const hashedPassword = await bcrypt.hash(password, 12);
      const user = await db.users.create({
        name,
        email,
        password: hashedPassword,
        role,
      });

      return { user, errors: [] };
    },
  },
};
```

## Subscription Resolvers

```javascript
import { PubSub } from 'graphql-subscriptions';

const pubsub = new PubSub();

const MESSAGE_ADDED = 'MESSAGE_ADDED';
const USER_TYPING = 'USER_TYPING';

const resolvers = {
  Mutation: {
    sendMessage: async (_, { content, channelId }, { db, currentUser }) => {
      if (!currentUser)
        throw new GraphQLError('Unauthenticated', {
          extensions: { code: 'UNAUTHENTICATED' },
        });

      const message = await db.messages.create({
        content,
        channelId,
        authorId: currentUser.id,
      });

      // Publish to subscribers
      await pubsub.publish(MESSAGE_ADDED, {
        messageAdded: message,
        channelId, // used for filtering below
      });

      return message;
    },
  },

  Subscription: {
    messageAdded: {
      // subscribe function returns an async iterator
      subscribe: (_, { channelId }, { currentUser }) => {
        if (!currentUser)
          throw new GraphQLError('Unauthenticated', {
            extensions: { code: 'UNAUTHENTICATED' },
          });

        return pubsub.asyncIterator([MESSAGE_ADDED]);
      },

      // Filter: only emit if channelId matches
      resolve: (payload, { channelId }) => {
        if (payload.channelId !== channelId) return null;
        return payload.messageAdded;
      },
    },
  },
};
```

## Resolver Performance Patterns

### Query Complexity Analysis

```bash
npm install graphql-query-complexity
```

```javascript
import {
  createComplexityRule,
  simpleEstimator,
  fieldExtensionsEstimator,
} from 'graphql-query-complexity';

const server = new ApolloServer({
  typeDefs,
  resolvers,
  plugins: [
    {
      requestDidStart: async () => ({
        didResolveOperation({ request, document }) {
          const complexity = getComplexity({
            schema,
            operationName: request.operationName,
            query: document,
            variables: request.variables,
            estimators: [
              fieldExtensionsEstimator(),
              simpleEstimator({ defaultComplexity: 1 }),
            ],
          });

          if (complexity > 100) {
            throw new GraphQLError(
              `Query complexity ${complexity} exceeds maximum of 100`,
              { extensions: { code: 'COMPLEXITY_LIMIT_EXCEEDED', complexity } },
            );
          }
        },
      }),
    },
  ],
});
```

### Field-Level Authorization

```javascript
// Authorization in individual resolvers
const resolvers = {
  User: {
    // Only the user themselves or admins can see email
    email: (user, _, { currentUser }) => {
      if (!currentUser) return null;
      if (currentUser.id === user.id || currentUser.isAdmin) return user.email;
      return null; // return null for unauthorized fields
    },
    // Or throw for stricter enforcement
    ssn: (user, _, { currentUser }) => {
      if (currentUser?.id !== user.id && !currentUser?.isAdmin) {
        throw new GraphQLError('Unauthorized', {
          extensions: { code: 'FORBIDDEN' },
        });
      }
      return user.ssn;
    },
  },
};
```

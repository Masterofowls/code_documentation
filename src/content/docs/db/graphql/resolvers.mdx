# GraphQL Resolvers

Resolvers are the functions that compute the value of each field in your schema. Every field in a GraphQL schema has a corresponding resolver — if you don't write one, GraphQL uses a default property resolver.

## Resolver Signature

Every resolver receives four arguments:

```typescript
type Resolver<TParent, TArgs, TContext, TReturn> = (
  parent: TParent,   // Value returned by the parent field's resolver
  args: TArgs,       // Arguments provided in the query/mutation field
  context: TContext, // Shared object passed to all resolvers in a request
  info: GraphQLResolveInfo  // Query AST, field name, path, schema info
) => TReturn | Promise<TReturn>;
```

```typescript
const resolvers = {
  Query: {
    // parent is root value (usually undefined for root queries)
    // args contains query arguments: { id: "42" }
    // context contains shared state: { db, user, dataSources }
    user: async (parent, args: { id: string }, context: Context) => {
      return context.db.users.findById(args.id);
    },

    posts: async (_parent, args: { filter?: PostFilter; first?: number }, context: Context) => {
      return context.db.posts.findAll(args);
    },
  },

  User: {
    // parent is the User object returned by the parent resolver
    // Resolves the `posts` field on a User
    posts: async (parent: User, args: { first?: number }, context: Context) => {
      return context.db.posts.findByAuthorId(parent.id, args.first ?? 10);
    },

    // Computed field — no DB query needed
    fullName: (parent: User) => `${parent.firstName} ${parent.lastName}`,
  },

  Post: {
    // Resolves the author of a post
    author: async (parent: Post, _args, context: Context) => {
      return context.db.users.findById(parent.authorId);
    },
  },
};
```

## Resolver Chain (Parent Propagation)

GraphQL resolves fields in a tree, where each parent passes its return value to child resolvers:

```
Query.user(id: "1")         → returns User { id: "1", name: "Alice", ... }
  └── User.posts             → parent = User { id: "1" }, returns [Post, Post, ...]
        └── Post.author      → parent = Post { authorId: "2" }, returns User { id: "2" }
              └── User.name  → parent = User { id: "2" }, returns "Bob"
```

## Default Resolvers

If you don't define a resolver for a field, GraphQL uses the **default field resolver**, which simply looks up the property with the same name on the parent object:

```typescript
// This is what GraphQL does automatically for unresolved fields:
const defaultFieldResolver = (source: Record<string, unknown>, args: unknown, context: unknown, info: { fieldName: string }) => {
  return source[info.fieldName];
};

// So you only need to write resolvers for:
// 1. Root fields (Query, Mutation, Subscription)
// 2. Fields that require transformation or computation
// 3. Fields that fetch related data (relationships)
// 4. Fields whose property name differs from the GraphQL field name
```

## Context Object

The context is a per-request object shared across all resolvers. Use it to pass database connections, authenticated user info, and shared services:

```typescript
// src/context.ts
import type { User } from './types';
import { db } from './db';
import { verifyJWT } from './auth';
import { UserDataSource, PostDataSource } from './dataSources';

export interface Context {
  db: typeof db;
  user: User | null;
  dataSources: {
    users: UserDataSource;
    posts: PostDataSource;
  };
  requestId: string;
}

// In Apollo Server:
const server = new ApolloServer({ typeDefs, resolvers });
await startStandaloneServer(server, {
  context: async ({ req }): Promise<Context> => {
    // Parse auth token
    const token = req.headers.authorization?.replace('Bearer ', '');
    let user: User | null = null;
    if (token) {
      try { user = await verifyJWT(token); } catch { /* invalid token */ }
    }

    return {
      db,
      user,
      dataSources: {
        users: new UserDataSource(db),
        posts: new PostDataSource(db),
      },
      requestId: crypto.randomUUID(),
    };
  },
});
```

## The N+1 Problem

The N+1 problem is the most common performance pitfall in GraphQL APIs.

```typescript
// Schema:
// type Post { id: ID!, title: String!, author: User! }
// type Query { posts: [Post!]! }

// Naive resolver — causes N+1 queries
const resolvers = {
  Query: {
    posts: () => db.posts.findAll(), // 1 query → returns [post1, post2, ..., postN]
  },
  Post: {
    // This fires once per post — N additional queries!
    author: (parent: Post) => db.users.findById(parent.authorId),
    // If 100 posts, 101 total queries (1 for posts + 100 for authors)
  },
};
```

## DataLoader

DataLoader solves N+1 by **batching** multiple individual loads into a single batch request, and **caching** within a single request:

```bash
npm install dataloader
```

```typescript
import DataLoader from 'dataloader';

// Create a batch loading function
// Keys: array of authorIds to load
// Returns: array of Users in the SAME ORDER as keys
async function batchLoadUsers(ids: readonly string[]): Promise<(User | Error)[]> {
  const users = await db.users.findManyByIds(ids as string[]);
  // Map results back to key order (critical!)
  const userMap = new Map(users.map(u => [u.id, u]));
  return ids.map(id => userMap.get(id) ?? new Error(`User ${id} not found`));
}

// Create DataLoader (once per request — in context factory)
const userLoader = new DataLoader<string, User>(batchLoadUsers);

// Now the Post.author resolver uses the loader
const resolvers = {
  Post: {
    author: (parent: Post, _args, context: Context) => {
      return context.loaders.users.load(parent.authorId);
      // DataLoader accumulates all .load() calls in a tick
      // Then calls batchLoadUsers([id1, id2, ...idN]) once
    },
  },
};

// Updated context to include loaders
export interface Context {
  db: typeof db;
  user: User | null;
  loaders: {
    users: DataLoader<string, User>;
    posts: DataLoader<string, Post>;
  };
}

// Context factory: create fresh DataLoaders per request (important for cache isolation)
context: async ({ req }) => ({
  db,
  user: await getUserFromRequest(req),
  loaders: {
    users: new DataLoader<string, User>(batchLoadUsers),
    posts: new DataLoader<string, Post>(batchLoadPosts),
  },
}),

// DataLoader methods
await loader.load('id1');         // Load single key
await loader.loadMany(['id1', 'id2']); // Load multiple (returns array, no throw)
loader.clear('id1');              // Remove from cache
loader.clearAll();                // Clear entire cache
loader.prime('id1', userData);    // Pre-populate cache (e.g., from a findAll query)
```

## Error Handling

```typescript
import { GraphQLError } from 'graphql';

const resolvers = {
  Mutation: {
    createUser: async (_parent, args: { input: CreateUserInput }, context: Context) => {
      // Authorization check
      if (!context.user) {
        throw new GraphQLError('You must be logged in to create a user', {
          extensions: {
            code: 'UNAUTHENTICATED',
            http: { status: 401 },
          },
        });
      }

      // Input validation
      if (!args.input.email.includes('@')) {
        throw new GraphQLError('Invalid email address', {
          extensions: {
            code: 'BAD_USER_INPUT',
            field: 'email',
          },
        });
      }

      // Business logic error
      const existing = await context.db.users.findByEmail(args.input.email);
      if (existing) {
        throw new GraphQLError('Email already in use', {
          extensions: {
            code: 'DUPLICATE_EMAIL',
            http: { status: 409 },
          },
        });
      }

      return context.db.users.create(args.input);
    },
  },
};

// Standard error codes (Apollo convention)
// UNAUTHENTICATED — not logged in (401)
// FORBIDDEN — insufficient permissions (403)
// NOT_FOUND — resource not found (404)
// BAD_USER_INPUT — invalid input data (400)
// INTERNAL_SERVER_ERROR — unexpected server error (500)
// PERSISTED_QUERY_NOT_FOUND — APQ miss
```

## Abstract Type Resolvers

For interfaces and unions, you must define `__resolveType` to tell GraphQL which concrete type a value is:

```typescript
const resolvers = {
  // Union resolver
  SearchResult: {
    __resolveType(obj: User | Post | Comment) {
      if ('email' in obj) return 'User';
      if ('title' in obj) return 'Post';
      if ('text' in obj) return 'Comment';
      return null; // Should not happen
    },
  },

  // Interface resolver
  Node: {
    __resolveType(obj: { __typename?: string; id: string }) {
      // Best approach: store __typename on returned objects
      return obj.__typename ?? null;
    },
  },
};

// Tip: Include __typename in your DB result objects
async function findUserById(id: string): Promise<User & { __typename: 'User' }> {
  const user = await db.query('SELECT * FROM users WHERE id = ?', [id]);
  return { ...user, __typename: 'User' };
}
```

## Mutation Resolvers

```typescript
const resolvers = {
  Mutation: {
    createPost: async (
      _parent,
      args: { input: CreatePostInput },
      context: Context
    ) => {
      if (!context.user) throw new GraphQLError('Unauthenticated', {
        extensions: { code: 'UNAUTHENTICATED' }
      });

      // Validate input
      if (args.input.title.trim().length === 0) {
        throw new GraphQLError('Title cannot be empty', {
          extensions: { code: 'BAD_USER_INPUT', field: 'title' }
        });
      }

      const post = await context.db.posts.create({
        ...args.input,
        authorId: context.user.id,
        slug: slugify(args.input.title),
        createdAt: new Date(),
        updatedAt: new Date(),
      });

      // Publish subscription event
      context.pubsub.publish('POST_CREATED', { postCreated: post });

      return post;
    },
  },
};
```

## Subscription Resolvers

```typescript
import { PubSub } from 'graphql-subscriptions';
// For production: use graphql-redis-subscriptions instead
import { RedisPubSub } from 'graphql-redis-subscriptions';

const pubsub = new PubSub();

const resolvers = {
  Mutation: {
    createPost: async (_parent, args, context) => {
      const post = await context.db.posts.create(args.input);
      // Publish event after mutation
      await pubsub.publish('POST_PUBLISHED', { postPublished: post });
      return post;
    },
  },

  Subscription: {
    postPublished: {
      // subscribe: returns an AsyncIterator that yields events
      subscribe: () => pubsub.asyncIterator(['POST_PUBLISHED']),
      // resolve: transform event payload if needed
      resolve: (payload: { postPublished: Post }) => payload.postPublished,
    },

    // Filtered subscription — only send to relevant clients
    commentAdded: {
      subscribe: (_parent, args: { postId: string }) =>
        pubsub.asyncIterator([`COMMENT_ADDED_${args.postId}`]),
    },
  },
};
```

## Resolver Performance Patterns

### Query Complexity Analysis

```typescript
import { fieldExtensionsEstimator, simpleEstimator, createComplexityLimitRule } from 'graphql-query-complexity';

const complexityPlugin: ApolloServerPlugin = {
  async requestDidStart() {
    return {
      async didResolveOperation({ request, document, schema }) {
        const complexity = getComplexity({
          schema,
          operationName: request.operationName,
          query: document,
          variables: request.variables,
          estimators: [
            fieldExtensionsEstimator(),
            simpleEstimator({ defaultComplexity: 1 }),
          ],
        });

        if (complexity > 1000) {
          throw new GraphQLError(
            `Query is too complex: ${complexity}. Maximum allowed: 1000.`,
            { extensions: { code: 'QUERY_TOO_COMPLEX', complexity } }
          );
        }
      },
    };
  },
};
```

### Field-Level Authorization

```typescript
const resolvers = {
  User: {
    // Only return email to the user themselves or admins
    email: (parent: User, _args, context: Context) => {
      if (context.user?.id === parent.id || context.user?.role === 'ADMIN') {
        return parent.email;
      }
      return null; // or throw a FORBIDDEN error
    },

    // Sensitive field — always check
    paymentMethods: (_parent, _args, context: Context) => {
      if (!context.user) throw new GraphQLError('Unauthenticated', {
        extensions: { code: 'UNAUTHENTICATED' }
      });
      return context.db.paymentMethods.findByUserId(context.user.id);
    },
  },
};
```

:::tip[Use a Permissions Library]
For complex authorization rules, use `graphql-shield` (middleware-based permissions) or `@pothos/plugin-authz`. Scattering `if (!context.user)` checks throughout resolvers becomes hard to maintain at scale.
:::

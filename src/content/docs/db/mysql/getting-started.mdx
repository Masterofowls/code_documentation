# Getting Started with MySQL

## Installation

<Tabs>
  <TabItem label="Ubuntu/Debian">
    ```bash
    # Ubuntu 22.04 — install from official MySQL APT repository
    wget https://dev.mysql.com/get/mysql-apt-config_0.8.29-1_all.deb
    sudo dpkg -i mysql-apt-config_0.8.29-1_all.deb
    ```

## Securing the Installation

After installation, run `mysql_secure_installation` to remove insecure defaults:

```bash
sudo mysql_secure_installation
```

This interactive wizard will:
1. Set (or change) the root password and configure password validation policy
2. Remove anonymous users created by default
3. Disallow remote root login (root should only connect from `localhost`)
4. Remove the `test` database
5. Reload privilege tables

:::tip[On Fresh Ubuntu Installs]
Ubuntu's MySQL package may install with `auth_socket` for root, meaning root logs in via the OS user, not a password. Run `sudo mysql` to connect, then change the auth plugin if you need password-based root access:

```sql
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'StrongPassword!1';
FLUSH PRIVILEGES;
```
:::

## The mysql CLI

```bash
# Connect as root locally
mysql -u root -p

# Connect to a specific database
mysql -u root -p myapp

# Connect to a remote server
mysql -h db.example.com -P 3306 -u appuser -p myapp

# Execute a single query without interactive mode
mysql -u root -p myapp -e "SELECT COUNT(*) FROM orders;"

# Execute a SQL file
mysql -u root -p myapp < schema.sql

# Output as tab-separated (useful in scripts)
mysql -u root -p myapp --batch --silent -e "SELECT id, name FROM users;"
```

Useful CLI shortcuts inside the mysql shell:

| Command | Effect |
|---------|--------|
| `\g` or `;` | Execute query |
| `\G` | Execute query, vertical output |
| `\c` | Cancel current input |
| `\h` | Help |
| `\q` or `exit` | Quit |
| `SHOW WARNINGS;` | Show warnings from last statement |
| `\! clear` | Clear screen |
| `SOURCE file.sql` | Execute a SQL file |
| `\P less` | Set pager |

## User Management

### Create a User

```sql
-- Create user with password
CREATE USER 'alice'@'localhost' IDENTIFIED BY 'SecurePass!42';

-- Allow connection from any host
CREATE USER 'bob'@'%' IDENTIFIED BY 'SecurePass!42';

-- Allow connection from a specific subnet
CREATE USER 'api'@'10.0.1.%' IDENTIFIED BY 'ApiPass!99';

-- Create with caching_sha2_password (MySQL 8.0 default auth plugin)
CREATE USER 'charlie'@'%' IDENTIFIED WITH caching_sha2_password BY 'Pass!123';

-- Create with mysql_native_password (for older clients)
CREATE USER 'legacy'@'%' IDENTIFIED WITH mysql_native_password BY 'Pass!123';

-- Create with no password expiry and locked account
CREATE USER 'service'@'localhost' IDENTIFIED BY 'SvcPass!77'
  PASSWORD EXPIRE NEVER
  ACCOUNT UNLOCK;
```

:::caution[Always Restrict Hosts]
Avoid `'user'@'%'` in production unless you have network-level restrictions (VPC, firewall, security groups). Prefer specific hosts or IP ranges to minimize attack surface.
:::

### Authentication Plugins

MySQL 8.0 changed the default auth plugin from `mysql_native_password` to `caching_sha2_password`. This is more secure but can cause issues with older clients:

```sql
-- Check auth plugin for all users
SELECT user, host, plugin FROM mysql.user;

-- Change auth plugin for a user
ALTER USER 'alice'@'localhost' IDENTIFIED WITH mysql_native_password BY 'NewPass!1';

-- Set default auth plugin globally (my.cnf)
-- [mysqld]
-- default_authentication_plugin=mysql_native_password
```

### GRANT — Assigning Privileges

```sql
-- Database-level grants
GRANT ALL PRIVILEGES ON myapp.* TO 'alice'@'localhost';
GRANT SELECT, INSERT, UPDATE, DELETE ON myapp.* TO 'bob'@'%';
GRANT SELECT ON myapp.* TO 'readonly'@'%';

-- Table-level grants
GRANT SELECT, INSERT ON myapp.orders TO 'alice'@'localhost';
GRANT UPDATE ON myapp.products TO 'pricing_service'@'10.0.0.%';

-- Column-level grants
GRANT SELECT (id, name, email) ON myapp.users TO 'limited'@'localhost';
GRANT UPDATE (status) ON myapp.orders TO 'fulfillment'@'%';

-- Grant GRANT OPTION (allows user to grant their privileges to others)
GRANT SELECT ON myapp.* TO 'team_lead'@'%' WITH GRANT OPTION;

-- Global grants (use sparingly)
GRANT PROCESS ON *.* TO 'monitor'@'localhost';   -- can see SHOW PROCESSLIST
GRANT RELOAD ON *.* TO 'admin'@'localhost';       -- can FLUSH

-- Grant a role (MySQL 8.0+)
GRANT 'app_read' TO 'bob'@'%';
```

### REVOKE — Removing Privileges

```sql
-- Revoke specific privilege
REVOKE INSERT ON myapp.* FROM 'bob'@'%';

-- Revoke all privileges on a database
REVOKE ALL PRIVILEGES ON myapp.* FROM 'alice'@'localhost';

-- Revoke grant option only
REVOKE GRANT OPTION ON myapp.* FROM 'team_lead'@'%';

-- View current grants for a user
SHOW GRANTS FOR 'alice'@'localhost';
SHOW GRANTS FOR CURRENT_USER();
```

### FLUSH PRIVILEGES

```sql
-- Required after direct manipulation of grant tables
-- (UPDATE mysql.user, INSERT INTO mysql.db, etc.)
FLUSH PRIVILEGES;

-- NOT required after using GRANT/REVOKE/CREATE USER/ALTER USER
-- Those statements automatically reload the grant tables
```

:::tip[Modern User Management]
If you use `CREATE USER`, `GRANT`, `REVOKE`, and `ALTER USER` statements (rather than directly editing `mysql.user` tables), you **do not need** `FLUSH PRIVILEGES`. MySQL automatically updates the in-memory grant cache.
:::

### Drop and Alter Users

```sql
-- Drop a user
DROP USER 'alice'@'localhost';
DROP USER IF EXISTS 'alice'@'localhost';

-- Rename a user
RENAME USER 'alice'@'localhost' TO 'alice'@'%';

-- Change password
ALTER USER 'alice'@'localhost' IDENTIFIED BY 'NewSecurePass!1';

-- Force password expiry
ALTER USER 'alice'@'localhost' PASSWORD EXPIRE;

-- Lock/unlock an account
ALTER USER 'alice'@'localhost' ACCOUNT LOCK;
ALTER USER 'alice'@'localhost' ACCOUNT UNLOCK;
```

## Database Operations

### Creating Databases

```sql
-- Always specify utf8mb4 and a unicode collation
CREATE DATABASE myapp
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- With IF NOT EXISTS
CREATE DATABASE IF NOT EXISTS myapp
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- Show existing databases
SHOW DATABASES;

-- Select a database to use
USE myapp;

-- Show the current database
SELECT DATABASE();
```

:::caution[Always Use utf8mb4, Never utf8]
MySQL's `utf8` character set is **not real UTF-8** — it only supports up to 3-byte characters and cannot store emoji (4-byte Unicode) like 😀. Always use `utf8mb4` with `utf8mb4_unicode_ci` (for sorting/comparison) or `utf8mb4_0900_ai_ci` (MySQL 8.0 default, faster).

```sql
-- Check current charset of a database
SELECT DEFAULT_CHARACTER_SET_NAME, DEFAULT_COLLATION_NAME
FROM information_schema.SCHEMATA
WHERE SCHEMA_NAME = 'myapp';
```
:::

### Exploring Schema

```sql
-- List all tables in current database
SHOW TABLES;
SHOW TABLES FROM myapp;
SHOW FULL TABLES;                   -- includes table type (BASE TABLE / VIEW)

-- Show table structure
DESCRIBE orders;
DESC orders;
SHOW COLUMNS FROM orders;
SHOW FULL COLUMNS FROM orders;      -- includes collation, privileges, comment

-- Show CREATE TABLE statement
SHOW CREATE TABLE orders\G

-- Show indexes on a table
SHOW INDEX FROM orders\G
SHOW INDEX FROM orders FROM myapp\G

-- Show table status (engine, rows, size, etc.)
SHOW TABLE STATUS LIKE 'orders'\G
SHOW TABLE STATUS FROM myapp\G
```

## MySQL Configuration

The MySQL configuration file is `my.cnf` on Linux/macOS and `my.ini` on Windows.

**Common locations:**
- Linux: `/etc/mysql/my.cnf` or `/etc/my.cnf` or `/etc/mysql/conf.d/*.cnf`
- macOS (Homebrew): `/usr/local/etc/my.cnf` or `/opt/homebrew/etc/my.cnf`
- Windows: `C:\ProgramData\MySQL\MySQL Server 8.0\my.ini`

```ini
[mysqld]
# ── Connection ──────────────────────────────────────────────
max_connections         = 200        # max simultaneous connections
connect_timeout         = 10
wait_timeout            = 28800      # idle connection timeout (seconds)
interactive_timeout     = 28800

# ── Memory ──────────────────────────────────────────────────
innodb_buffer_pool_size = 2G         # 50-75% of RAM for dedicated DB servers
innodb_buffer_pool_instances = 4     # one per GB of buffer pool (up to 8)
innodb_log_file_size    = 256M       # redo log size — larger = better write perf
innodb_flush_log_at_trx_commit = 1  # 1=fully durable, 2=faster, 0=fastest/risky
sort_buffer_size        = 4M
join_buffer_size        = 4M
tmp_table_size          = 64M
max_heap_table_size     = 64M

# ── Slow Query Log ───────────────────────────────────────────
slow_query_log          = ON
slow_query_log_file     = /var/log/mysql/slow.log
long_query_time         = 1          # seconds; queries longer than this are logged
log_queries_not_using_indexes = ON   # log full-table scans

# ── Binary Logging (required for replication/PITR) ──────────
log_bin                 = /var/log/mysql/mysql-bin
binlog_format           = ROW        # ROW is safest; STATEMENT or MIXED also available
binlog_expire_logs_seconds = 604800  # 7 days (MySQL 8.0+)
server_id               = 1          # must be unique in replication topology

# ── Character Set ────────────────────────────────────────────
character-set-server    = utf8mb4
collation-server        = utf8mb4_unicode_ci

# ── InnoDB ───────────────────────────────────────────────────
innodb_file_per_table   = ON         # each table in its own .ibd file
innodb_strict_mode      = ON
```

### Viewing and Setting Variables at Runtime

```sql
-- View a specific variable
SHOW VARIABLES LIKE 'max_connections';
SHOW VARIABLES LIKE 'innodb_buffer%';
SHOW GLOBAL VARIABLES LIKE '%slow%';

-- View system status
SHOW STATUS LIKE 'Threads_connected';
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool%';

-- Set a session variable
SET SESSION sort_buffer_size = 8388608;  -- 8M

-- Set a global variable (persists until restart)
SET GLOBAL max_connections = 300;

-- Persist a variable to my.cnf (MySQL 8.0+)
SET PERSIST max_connections = 300;
SET PERSIST_ONLY innodb_buffer_pool_size = 4294967296;   -- applies on restart only
```

## Backup and Restore

### mysqldump

```bash
# Dump a single database
mysqldump -u root -p myapp > myapp_backup.sql

# Dump with additional safety options (recommended for InnoDB)
mysqldump -u root -p \
  --single-transaction \      # consistent snapshot without locking
  --routines \                 # include stored procedures and functions
  --triggers \                 # include triggers
  --events \                   # include events
  --hex-blob \                 # binary data as hex
  --master-data=2 \            # include binlog position as a comment
  myapp > myapp_full.sql

# Dump specific tables
mysqldump -u root -p myapp orders order_items > orders_backup.sql

# Dump all databases
mysqldump -u root -p --all-databases --single-transaction > all_dbs.sql

# Compress the output
mysqldump -u root -p myapp | gzip > myapp_$(date +%Y%m%d).sql.gz
```

### Restore

```bash
# Restore from a dump
mysql -u root -p myapp < myapp_backup.sql

# Restore from compressed dump
gunzip -c myapp_20240101.sql.gz | mysql -u root -p myapp

# Restore with verbose progress
pv myapp_backup.sql | mysql -u root -p myapp   # pv shows progress bar
```

:::tip[Binary Log for Point-in-Time Recovery]
With `log_bin` enabled, you can recover to any point in time:
```bash
# Apply binary logs after restoring a dump
mysqlbinlog --start-datetime="2024-01-15 10:00:00" \
            --stop-datetime="2024-01-15 14:30:00" \
            /var/log/mysql/mysql-bin.000042 | mysql -u root -p myapp
```
:::

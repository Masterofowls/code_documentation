# Getting Started with MySQL

This guide covers installing MySQL, initial security hardening, user and privilege management,
database creation, and essential configuration for development and production environments.

## Installation

<Tabs>
  <TabItem label="Ubuntu/Debian">
    ```bash
    # Add MySQL APT repository
    wget https://dev.mysql.com/get/mysql-apt-config_0.8.29-1_all.deb
    sudo dpkg -i mysql-apt-config_0.8.29-1_all.deb
    sudo apt update

    # Install MySQL 8.0
    sudo apt install mysql-server

    # Start and enable service
    sudo systemctl start mysql
    sudo systemctl enable mysql

    # Check status
    sudo systemctl status mysql
    ```
  </TabItem>
  <TabItem label="RHEL/CentOS/Rocky">
    ```bash
    # Add MySQL Yum Repository
    sudo rpm -Uvh https://dev.mysql.com/get/mysql84-community-release-el9-1.noarch.rpm

    # Install MySQL
    sudo dnf install mysql-community-server

    # Start and enable
    sudo systemctl start mysqld
    sudo systemctl enable mysqld

    # Get temporary root password (RHEL/CentOS only)
    sudo grep 'temporary password' /var/log/mysqld.log
    ```
  </TabItem>
  <TabItem label="macOS (Homebrew)">

    # Install via Homebrew

## Initial Security Setup

After installation, run the security script to harden your MySQL instance:

```bash
sudo mysql_secure_installation
```

This script will:
1. Set or change the root password
2. Remove anonymous user accounts
3. Disable remote root login
4. Remove the test database
5. Reload privilege tables

:::tip[Validate Password Plugin]
The script may ask about the `validate_password` component which enforces password complexity.
For development, choose LOW (0). For production, use MEDIUM (1) or STRONG (2).
:::

## MySQL CLI

```bash
# Connect as root
mysql -u root -p

# Connect to a specific database
mysql -u appuser -p mydb

# Connect to remote server
mysql -h 192.168.1.100 -P 3306 -u appuser -p mydb

# Execute a query directly
mysql -u root -p -e "SHOW DATABASES;"

# Run a SQL file
mysql -u root -p mydb < schema.sql

# Dump and restore (covered later)
```

**Useful CLI commands inside the MySQL shell:**

```sql
-- Show current user and database
SELECT USER(), DATABASE();

-- Show current status
STATUS;   -- or \s

-- Show all databases
SHOW DATABASES;

-- Switch database
USE mydb;

-- Show tables in current database
SHOW TABLES;

-- Show table structure
DESCRIBE users;
SHOW CREATE TABLE users\G

-- Show warnings from last statement
SHOW WARNINGS;

-- Show errors
SHOW ERRORS;

-- Clear screen
\! clear   -- or system clear

-- Exit
EXIT;   -- or QUIT; or \q
```

## MySQL Workbench

MySQL Workbench is the official GUI for MySQL. It provides:
- Visual schema design (EER diagrams)
- SQL editor with auto-complete and syntax highlighting
- Database connection management
- Data import/export wizards
- Server performance dashboards
- Migration wizard (from SQL Server, PostgreSQL, SQLite)

Download from [dev.mysql.com/downloads/workbench/](https://dev.mysql.com/downloads/workbench/).

:::caution[Workbench vs CLI]
For scripting and automation, always use the CLI or a programmatic driver.
Workbench is excellent for exploration and schema design but is not suitable for
production automation workflows.
:::

## User Management

### Creating Users

```sql
-- Create user with password (MySQL 8.0 default: caching_sha2_password)
CREATE USER 'alice'@'localhost' IDENTIFIED BY 'SecurePass123!';

-- Create user for any host
CREATE USER 'appuser'@'%' IDENTIFIED BY 'AppPass456!';

-- Create user with specific auth plugin
CREATE USER 'legacy_user'@'localhost'
  IDENTIFIED WITH mysql_native_password BY 'LegacyPass789!';

-- Create user with password expiry
CREATE USER 'temp_user'@'localhost'
  IDENTIFIED BY 'TempPass!'
  PASSWORD EXPIRE INTERVAL 90 DAY;

-- Create user with connection limits
CREATE USER 'limited_user'@'%'
  IDENTIFIED BY 'LimitedPass!'
  WITH MAX_QUERIES_PER_HOUR 1000
       MAX_CONNECTIONS_PER_HOUR 100
       MAX_USER_CONNECTIONS 10;

-- View all users
SELECT User, Host, plugin, authentication_string
FROM mysql.user;

-- Show grants for a user
SHOW GRANTS FOR 'alice'@'localhost';
```

### Granting Privileges

```sql
-- Grant all privileges on a database
GRANT ALL PRIVILEGES ON mydb.* TO 'alice'@'localhost';

-- Grant specific privileges on a database
GRANT SELECT, INSERT, UPDATE, DELETE ON mydb.* TO 'appuser'@'%';

-- Grant on a specific table
GRANT SELECT ON mydb.products TO 'readonly_user'@'localhost';

-- Grant on specific columns (column-level privilege)
GRANT SELECT (id, name, email), UPDATE (email)
  ON mydb.users TO 'support_user'@'localhost';

-- Grant with ability to re-grant (GRANT OPTION)
GRANT SELECT ON mydb.* TO 'manager'@'localhost' WITH GRANT OPTION;

-- Grant SUPER privilege (be careful — very powerful)
GRANT SUPER ON *.* TO 'dba'@'localhost';

-- Grant EXECUTE on a stored procedure
GRANT EXECUTE ON PROCEDURE mydb.process_order TO 'appuser'@'%';

-- Grant EXECUTE on all routines in a database
GRANT EXECUTE ON mydb.* TO 'appuser'@'%';

-- Grant CREATE, DROP, ALTER for schema management
GRANT CREATE, DROP, ALTER, INDEX, CREATE ROUTINE, ALTER ROUTINE
  ON mydb.* TO 'developer'@'localhost';

-- Apply privileges immediately (usually automatic in MySQL 8.0)
FLUSH PRIVILEGES;
```

### Revoking Privileges

```sql
-- Revoke specific privilege
REVOKE INSERT ON mydb.* FROM 'appuser'@'%';

-- Revoke all privileges from a database
REVOKE ALL PRIVILEGES ON mydb.* FROM 'alice'@'localhost';

-- Revoke GRANT OPTION
REVOKE GRANT OPTION ON mydb.* FROM 'manager'@'localhost';

-- Drop a user entirely
DROP USER 'temp_user'@'localhost';

-- Rename a user
RENAME USER 'old_name'@'localhost' TO 'new_name'@'localhost';

-- Change password
ALTER USER 'alice'@'localhost' IDENTIFIED BY 'NewPassword123!';
-- Or:
SET PASSWORD FOR 'alice'@'localhost' = 'NewPassword123!';
```

:::tip[FLUSH PRIVILEGES]
In MySQL 8.0+, `GRANT`, `REVOKE`, `CREATE USER`, and `DROP USER` automatically
reload the grant tables. You only need `FLUSH PRIVILEGES` if you modified the
`mysql.user` table directly with `INSERT`/`UPDATE` (which you should avoid).
:::

## Creating Databases

```sql
-- Create database with proper character set
-- ALWAYS use utf8mb4, not utf8 (utf8 in MySQL is actually utf8mb3 — only 3 bytes)
CREATE DATABASE myapp
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- utf8mb4_unicode_ci: accurate Unicode sorting, case-insensitive
-- utf8mb4_general_ci: slightly faster but less accurate sorting
-- utf8mb4_0900_ai_ci: MySQL 8.0 default, best performance + accuracy

-- Create if not exists
CREATE DATABASE IF NOT EXISTS myapp
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- Switch to database
USE myapp;

-- Check current database
SELECT DATABASE();

-- Show database creation info
SHOW CREATE DATABASE myapp;

-- List all databases
SHOW DATABASES;

-- Show databases matching a pattern
SHOW DATABASES LIKE 'my%';

-- Drop a database (DANGEROUS — drops all tables)
DROP DATABASE IF EXISTS old_db;
```

:::caution[utf8 vs utf8mb4]
**Always use `utf8mb4`**. MySQL's `utf8` charset is actually `utf8mb3` — it only supports
3-byte Unicode characters and cannot store emoji (4-byte characters like 😀).
`utf8mb4` is the true UTF-8 encoding and is the MySQL 8.0 default.
:::

## MySQL Configuration File

The MySQL configuration file (`my.cnf` on Linux/macOS, `my.ini` on Windows) controls
server behavior. Locations (checked in order):

- `/etc/my.cnf`
- `/etc/mysql/my.cnf`
- `~/.my.cnf`
- `--defaults-file=path` (explicit)

```ini
# /etc/mysql/mysql.conf.d/mysqld.cnf (Ubuntu)
[mysqld]

# ── General ──────────────────────────────────────────────────
port                    = 3306
bind-address            = 127.0.0.1    # Change to 0.0.0.0 for remote access
max_connections         = 500          # Default 151 — increase for high traffic
connect_timeout         = 10
wait_timeout            = 600          # Close idle connections after 10 min
max_allowed_packet      = 64M          # Max packet size for queries/results

# ── Character Set ─────────────────────────────────────────────
character-set-server    = utf8mb4
collation-server        = utf8mb4_unicode_ci

# ── InnoDB ────────────────────────────────────────────────────
innodb_buffer_pool_size = 2G           # 70-80% of available RAM
innodb_buffer_pool_instances = 4       # 1 per GB of buffer pool
innodb_log_file_size    = 512M         # Larger = better write performance
innodb_flush_log_at_trx_commit = 1     # 1=safe (fsync), 2=faster (risk of 1s data loss)
innodb_flush_method     = O_DIRECT     # Bypass OS cache for InnoDB files

# ── Slow Query Log ────────────────────────────────────────────
slow_query_log          = 1
slow_query_log_file     = /var/log/mysql/mysql-slow.log
long_query_time         = 2            # Log queries taking > 2 seconds
log_queries_not_using_indexes = 1      # Also log full table scans

# ── Binary Log (required for replication and point-in-time recovery) ─
log_bin                 = /var/log/mysql/mysql-bin
binlog_format           = ROW          # ROW is safest and most common
expire_logs_days        = 7            # Auto-delete binary logs after 7 days
server_id               = 1            # Must be unique per server in replication

# ── Query Cache (removed in MySQL 8.0) ───────────────────────
# query_cache_type = 0                 # Disabled (deprecated, removed in 8.0)

# ── Logging ───────────────────────────────────────────────────
general_log             = 0            # Enable only for debugging (very verbose)
general_log_file        = /var/log/mysql/mysql-general.log
error_log               = /var/log/mysql/error.log

[client]
default-character-set   = utf8mb4

[mysql]
default-character-set   = utf8mb4
```

```bash
# Reload configuration (requires restart for most variables)
sudo systemctl restart mysql

# Check a variable at runtime (no restart needed)
SHOW VARIABLES LIKE 'max_connections';
SET GLOBAL max_connections = 300;  -- Some variables can be set at runtime
```

## Backup and Restore

### mysqldump

```bash
# Backup a single database
mysqldump -u root -p myapp > myapp_backup.sql

# Backup with common options for InnoDB
mysqldump -u root -p \
  --single-transaction \      # Consistent snapshot without locking (InnoDB)
  --routines \                # Include stored procedures and functions
  --triggers \                # Include triggers
  --events \                  # Include events
  --hex-blob \                # Safely encode BLOB data
  myapp > myapp_full.sql

# Backup multiple databases
mysqldump -u root -p --databases db1 db2 db3 > multi_db.sql

# Backup ALL databases
mysqldump -u root -p --all-databases > all_databases.sql

# Backup only structure (no data)
mysqldump -u root -p --no-data myapp > schema_only.sql

# Backup only data (no CREATE TABLE statements)
mysqldump -u root -p --no-create-info myapp > data_only.sql

# Compress backup
mysqldump -u root -p myapp | gzip > myapp_$(date +%Y%m%d).sql.gz

# Backup to remote server via SSH
mysqldump -u root -p myapp | ssh user@backup-server "cat > /backups/myapp.sql"
```

### Restoring

```bash
# Restore from SQL file
mysql -u root -p myapp < myapp_backup.sql

# Restore compressed backup
gunzip < myapp_20240101.sql.gz | mysql -u root -p myapp

# Create database first if it doesn't exist
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS myapp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root -p myapp < myapp_backup.sql
```

:::tip[Binary Log (Point-in-Time Recovery)]
Enable binary logging (`log_bin`) to allow point-in-time recovery. After restoring from a
full dump, you can replay binary logs up to a specific timestamp to recover more recent
changes:
```bash
mysqlbinlog --start-datetime="2024-01-15 09:00:00" \
            --stop-datetime="2024-01-15 12:00:00" \
            /var/log/mysql/mysql-bin.000042 | mysql -u root -p
```
:::

# Getting Started with MySQL

This guide walks you through installing MySQL, securing the installation, connecting with the CLI and Workbench, managing users and permissions, and configuring essential server variables.

## Installation

<Tabs>
  <TabItem label="Ubuntu/Debian">

```bash
# Ubuntu 22.04+ / Debian
sudo apt update
sudo apt install -y mysql-server

# Check status
sudo systemctl status mysql

# Enable on boot
sudo systemctl enable mysql

# Start if not running
sudo systemctl start mysql
```

For a specific version, use MySQL's APT repository:

```bash
wget https://dev.mysql.com/get/mysql-apt-config_0.8.29-1_all.deb
sudo dpkg -i mysql-apt-config_0.8.29-1_all.deb
sudo apt update
sudo apt install -y mysql-server
```

  </TabItem>
  <TabItem label="CentOS/RHEL/Fedora">

```bash
# RHEL 9 / CentOS Stream 9
sudo dnf install -y mysql-server

sudo systemctl start mysqld
sudo systemctl enable mysqld

# On first start, get temporary root password
sudo grep 'temporary password' /var/log/mysqld.log
```

For MySQL's YUM repository:

```bash
sudo rpm -Uvh https://dev.mysql.com/get/mysql80-community-release-el9-5.noarch.rpm
sudo dnf install -y mysql-community-server
```

<Steps>
1. Download the **MySQL Installer** from [dev.mysql.com/downloads/installer](https://dev.mysql.com/downloads/installer/).

2. Run the installer and choose **Developer Default** or **Server only**.

3. Follow the Configuration Wizard:
   - Choose **Standalone MySQL Server**.
   - Select **Development Computer** (or adjust `innodb_buffer_pool_size` appropriately).
   - Set root password and optionally create additional accounts.

4. MySQL is installed as a Windows Service (`MySQL80`) and starts automatically.

5. Add `C:\Program Files\MySQL\MySQL Server 8.0\bin` to your `PATH`.
</Steps>

## mysql_secure_installation

Run this immediately after installation to harden your MySQL server:

```bash
sudo mysql_secure_installation
```

It will prompt you to:

1. **Set up VALIDATE PASSWORD component** — enforces password strength.
2. **Set root password** — or confirm the current one.
3. **Remove anonymous users** — **always yes**.
4. **Disallow root login remotely** — **yes** unless you have a specific need.
5. **Remove test database** — **yes** in production.
6. **Reload privilege tables** — **yes**.

:::caution[Security Baseline]
Never skip `mysql_secure_installation` on a production server. Anonymous users and the test database are genuine security risks.
:::

## Connecting with the MySQL CLI

```bash
# Connect as root locally
mysql -u root -p

# Connect to a specific database
mysql -u root -p myapp_db

# Connect to a remote host
mysql -h 192.168.1.100 -P 3306 -u appuser -p myapp_db

# Execute a single query and exit
mysql -u root -p -e "SHOW DATABASES;"

# Execute a SQL file
mysql -u root -p myapp_db < schema.sql

# Output to file
mysql -u root -p -e "SELECT * FROM users" myapp_db > users.csv
```

Useful CLI options:

| Option | Description |
|---|---|
| `-u` | Username |
| `-p` | Prompt for password |
| `-h` | Host (default: localhost) |
| `-P` | Port (default: 3306) |
| `-D` or bare arg | Database to use |
| `-e` | Execute statement and exit |
| `--ssl-mode=REQUIRED` | Require SSL |
| `--batch` / `-B` | Non-interactive output (tab-separated) |
| `--column-names` | Include column headers in batch output |
| `--vertical` / `-E` | Display results vertically (`\G` in session) |
| `-s` | Silent (suppress table borders) |

Inside the mysql CLI, useful commands:

```sql
\h        -- Help
\s        -- Server status
\q        -- Quit
\G        -- Show last result vertically
\e        -- Edit command in $EDITOR
\! ls     -- Run shell command
PAGER less -- Page long output
\nopager  -- Disable pager
```

## MySQL Workbench

MySQL Workbench is the official GUI tool for database design, development, and administration.

<Steps>
1. Download from [dev.mysql.com/downloads/workbench](https://dev.mysql.com/downloads/workbench/).
2. Create a new **Connection** under "MySQL Connections."
3. Fill in hostname, port, username, and optionally save the password in the vault.
4. Use **Database → Connect to Database** or double-click the connection.
5. Explore schemas in the left panel, run queries in the SQL editor.
</Steps>

Key Workbench features:
- **EER Diagram** — visual entity-relationship modelling with forward/reverse engineering.
- **Performance Dashboard** — real-time server metrics.
- **Table Data Import Wizard** — import CSV/JSON directly into tables.
- **Schema Inspector** — detailed metadata about tables, indexes, and triggers.
- **Migration Wizard** — migrate from other RDBMS (SQL Server, PostgreSQL, etc.) to MySQL.

## User Management

### Creating Users

```sql
-- Basic user with password
CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'StrongPassword!1';

-- Allow from any host (less secure)
CREATE USER 'appuser'@'%' IDENTIFIED BY 'StrongPassword!1';

-- Allow from a specific subnet
CREATE USER 'appuser'@'10.0.0.%' IDENTIFIED BY 'StrongPassword!1';

-- MySQL 8.0: specify authentication plugin explicitly
CREATE USER 'appuser'@'%'
  IDENTIFIED WITH caching_sha2_password
  BY 'StrongPassword!1';

-- Require SSL for this user
CREATE USER 'secure_user'@'%'
  IDENTIFIED BY 'StrongPassword!1'
  REQUIRE SSL;

-- With resource limits
CREATE USER 'limited_user'@'%'
  IDENTIFIED BY 'Pass!1'
  WITH MAX_QUERIES_PER_HOUR 1000
       MAX_CONNECTIONS_PER_HOUR 100
       MAX_USER_CONNECTIONS 10;
```

:::tip[Authentication Plugins]
MySQL 8.0 uses `caching_sha2_password` as the default authentication plugin. Older clients that don't support it may fail to connect. Use `mysql_native_password` for compatibility with legacy clients, but prefer `caching_sha2_password` for new setups.
:::

### Granting Privileges

```sql
-- Grant all privileges on a specific database
GRANT ALL PRIVILEGES ON myapp_db.* TO 'appuser'@'%';

-- Grant specific privileges on a database
GRANT SELECT, INSERT, UPDATE, DELETE ON myapp_db.* TO 'readonly_user'@'%';

-- Grant privileges on a specific table
GRANT SELECT ON myapp_db.users TO 'reporting'@'%';

-- Column-level grant
GRANT SELECT (id, email, created_at) ON myapp_db.users TO 'limited'@'%';

-- Allow user to grant their own privileges to others
GRANT SELECT ON myapp_db.* TO 'manager'@'%' WITH GRANT OPTION;

-- Grant a role
GRANT 'read_only_role' TO 'appuser'@'%';

-- Grant SUPER (avoid in production — use more specific privileges)
-- GRANT SUPER ON *.* TO 'admin'@'localhost';

-- Grant PROCESS (needed to see all processes)
GRANT PROCESS ON *.* TO 'monitor'@'%';
```

Common privilege levels:

| Privilege | Scope | Description |
|---|---|---|
| `ALL PRIVILEGES` | Global/DB/Table | Grants everything except GRANT OPTION |
| `SELECT` | Table/Column | Read data |
| `INSERT` | Table/Column | Insert rows |
| `UPDATE` | Table/Column | Update rows |
| `DELETE` | Table | Delete rows |
| `CREATE` | DB/Table | Create databases/tables |
| `DROP` | DB/Table | Drop databases/tables |
| `INDEX` | Table | Create/drop indexes |
| `ALTER` | Table | Alter table structure |
| `CREATE VIEW` | DB | Create views |
| `SHOW VIEW` | DB | See view definitions |
| `CREATE ROUTINE` | DB | Create stored procedures/functions |
| `EXECUTE` | DB | Execute stored routines |
| `TRIGGER` | Table | Create/drop triggers |
| `REFERENCES` | Table | Create foreign keys |
| `REPLICATION SLAVE` | Global | Required for replication |
| `PROCESS` | Global | See all running queries |
| `RELOAD` | Global | Execute FLUSH statements |
| `SUPER` | Global | Various admin operations (avoid) |

### Revoking Privileges

```sql
-- Revoke specific privilege
REVOKE INSERT ON myapp_db.* FROM 'appuser'@'%';

-- Revoke all privileges
REVOKE ALL PRIVILEGES ON myapp_db.* FROM 'appuser'@'%';

-- Revoke GRANT OPTION only
REVOKE GRANT OPTION ON myapp_db.* FROM 'manager'@'%';

-- Drop a user entirely
DROP USER 'olduser'@'%';

-- Show grants for a user
SHOW GRANTS FOR 'appuser'@'%';

-- Show current user's grants
SHOW GRANTS;
```

### FLUSH PRIVILEGES

```sql
-- Reload grant tables from disk (needed after direct edits to mysql.user table)
FLUSH PRIVILEGES;
```

:::tip[When FLUSH Is Needed]
When you use `GRANT`, `REVOKE`, `CREATE USER`, or `DROP USER` statements, MySQL automatically reloads the privilege tables. `FLUSH PRIVILEGES` is only needed if you directly modify the `mysql.user` table with `INSERT`/`UPDATE`/`DELETE`, which you should avoid doing anyway.
:::

## Creating Databases

```sql
-- Always use utf8mb4, never utf8
-- MySQL's 'utf8' is actually UTF-8 with only 3-byte characters (no emoji support!)
-- 'utf8mb4' is true UTF-8 with full 4-byte character support
CREATE DATABASE myapp_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- Case-insensitive but accent-sensitive (better for most apps)
CREATE DATABASE myapp_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_0900_ai_ci;  -- MySQL 8.0 default collation

-- Show all databases
SHOW DATABASES;

-- Select a database to use
USE myapp_db;

-- Show current database
SELECT DATABASE();

-- Drop a database (CAREFUL — irreversible!)
DROP DATABASE IF EXISTS old_db;

-- Show create statement for existing DB
SHOW CREATE DATABASE myapp_db;
```

:::caution[utf8 vs utf8mb4]
**Always use `utf8mb4`**, never `utf8` in MySQL. MySQL's `utf8` charset is a misnomer — it only supports up to 3-byte UTF-8 characters, meaning it cannot store emoji (🎉), some Chinese characters, or other 4-byte Unicode code points. `utf8mb4` is the correct, full UTF-8 implementation.
:::

## Inspecting Schemas

```sql
-- List tables in current database
SHOW TABLES;

-- List tables in another database
SHOW TABLES FROM other_db;

-- Describe a table's structure
DESCRIBE users;
-- or
DESC users;
-- or
SHOW COLUMNS FROM users;

-- Show create statement (DDL) for a table
SHOW CREATE TABLE users\G

-- Show table status (engine, row count estimate, etc.)
SHOW TABLE STATUS LIKE 'users'\G

-- Show indexes on a table
SHOW INDEX FROM users\G

-- Information Schema queries
SELECT TABLE_NAME, TABLE_ROWS, DATA_LENGTH, INDEX_LENGTH
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'myapp_db'
ORDER BY DATA_LENGTH DESC;
```

## MySQL Configuration File

MySQL reads configuration from `my.cnf` (Linux/macOS) or `my.ini` (Windows).

Common locations (read in order):
1. `/etc/my.cnf`
2. `/etc/mysql/my.cnf`
3. `~/.my.cnf`
4. `C:\ProgramData\MySQL\MySQL Server 8.0\my.ini` (Windows)

```ini
[mysqld]
# === BASIC ===
port            = 3306
socket          = /var/run/mysqld/mysqld.sock
datadir         = /var/lib/mysql
pid-file        = /var/run/mysqld/mysqld.pid

# === CHARACTER SET ===
character-set-server  = utf8mb4
collation-server      = utf8mb4_unicode_ci

# === CONNECTIONS ===
max_connections       = 500
max_connect_errors    = 10000
wait_timeout          = 28800
interactive_timeout   = 28800

# === INNODB ===
# Set to 50-80% of available RAM
innodb_buffer_pool_size    = 4G
innodb_buffer_pool_instances = 4      # 1 per GB of buffer pool
innodb_log_file_size       = 512M
innodb_flush_method        = O_DIRECT
innodb_flush_log_at_trx_commit = 1   # 1=fully durable, 2=faster but 1s risk

# === QUERY CACHE (disabled in MySQL 8.0) ===
# query_cache_type = 0  # removed in 8.0

# === SLOW QUERY LOG ===
slow_query_log         = ON
slow_query_log_file    = /var/log/mysql/slow.log
long_query_time        = 1
log_queries_not_using_indexes = ON
min_examined_row_limit = 100

# === BINARY LOG ===
log_bin                = /var/log/mysql/mysql-bin.log
binlog_format          = ROW             # ROW is safest for replication
expire_logs_days       = 7
max_binlog_size        = 100M
sync_binlog            = 1               # Sync after every transaction

# === ERROR LOG ===
log_error              = /var/log/mysql/error.log

# === PERFORMANCE SCHEMA ===
performance_schema     = ON

[mysql]
default-character-set = utf8mb4

[client]
default-character-set = utf8mb4
```

View and set variables at runtime:

```sql
-- Show all variables
SHOW VARIABLES;

-- Show specific variable
SHOW VARIABLES LIKE 'innodb_buffer_pool_size';
SHOW VARIABLES LIKE '%timeout%';

-- Show global status
SHOW GLOBAL STATUS LIKE 'Threads_connected';

-- Change variable at runtime (not all variables are dynamic)
SET GLOBAL long_query_time = 2;
SET GLOBAL max_connections = 300;

-- Session-level change (affects current connection only)
SET SESSION wait_timeout = 600;

-- Persist variable to my.cnf (MySQL 8.0+)
SET PERSIST innodb_buffer_pool_size = 8589934592;  -- 8GB
SET PERSIST_ONLY innodb_log_file_size = 1073741824; -- 1GB (requires restart)
```

## Backup and Restore

### mysqldump

```bash
# Backup a single database
mysqldump -u root -p myapp_db > myapp_db_backup.sql

# Backup with all options recommended for production
mysqldump \
  -u root -p \
  --single-transaction \      # Consistent snapshot for InnoDB
  --routines \                 # Include stored procedures/functions
  --triggers \                 # Include triggers
  --events \                   # Include scheduled events
  --set-gtid-purged=OFF \      # Omit GTID info (use if GTID not configured)
  --compress \                 # Compress data in transit
  myapp_db > myapp_db_backup.sql

# Backup all databases
mysqldump -u root -p --all-databases --single-transaction > all_dbs.sql

# Backup specific tables
mysqldump -u root -p myapp_db users orders > selected_tables.sql

# Backup only structure (no data)
mysqldump -u root -p --no-data myapp_db > schema_only.sql

# Compress on the fly
mysqldump -u root -p myapp_db | gzip > myapp_db_$(date +%Y%m%d).sql.gz
```

### Restoring

```bash
# Restore a database
mysql -u root -p myapp_db < myapp_db_backup.sql

# Create database first if needed
mysql -u root -p -e "CREATE DATABASE myapp_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root -p myapp_db < myapp_db_backup.sql

# Restore compressed backup
gunzip < myapp_db_backup.sql.gz | mysql -u root -p myapp_db

# Restore with progress (using pv)
pv myapp_db_backup.sql | mysql -u root -p myapp_db
```

:::tip[Use --single-transaction for InnoDB]
Always use `--single-transaction` when backing up InnoDB tables with `mysqldump`. This starts a repeatable-read transaction before dumping, ensuring a consistent snapshot without locking your tables. Without this flag, `mysqldump` uses `LOCK TABLES`, blocking writes during the dump.
:::

## Binary Logging

Binary logging records all changes to the database and is essential for point-in-time recovery (PITR) and replication.

```sql
-- Check if binary logging is enabled
SHOW VARIABLES LIKE 'log_bin';

-- List all binary log files
SHOW BINARY LOGS;

-- Show events in a binary log
SHOW BINLOG EVENTS IN 'mysql-bin.000001' LIMIT 20;

-- Get current binary log position (important for replication setup)
SHOW MASTER STATUS;

-- Flush binary logs (creates new log file)
FLUSH BINARY LOGS;

-- Purge binary logs older than 7 days
PURGE BINARY LOGS BEFORE NOW() - INTERVAL 7 DAY;
```

Point-in-time recovery with `mysqlbinlog`:

```bash
# Replay binary log from a specific position
mysqlbinlog --start-position=154 mysql-bin.000001 | mysql -u root -p

# Replay between timestamps
mysqlbinlog \
  --start-datetime="2024-01-15 10:00:00" \
  --stop-datetime="2024-01-15 11:00:00" \
  mysql-bin.000001 | mysql -u root -p
```

# MySQL

MySQL is the world's most popular open-source relational database management system (RDBMS).
It powers countless web applications, from small startups to large-scale enterprises, and is a
core component of the classic LAMP stack (Linux, Apache, MySQL, PHP/Python/Perl).

## History

MySQL was originally developed by Swedish company MySQL AB, founded by Michael Widenius
("Monty"), David Axmark, and Allan Larsson in 1995. The name "MySQL" combines the name of
Monty's daughter, My, with SQL.

| Year | Event |
|------|-------|
| 1995 | MySQL 1.0 released internally |
| 2000 | MySQL goes open source under GPL |
| 2001 | MySQL 3.23 — first widely adopted release |
| 2008 | Sun Microsystems acquires MySQL AB for $1 billion |
| 2010 | Oracle acquires Sun Microsystems, gaining MySQL |
| 2010 | MariaDB forked by original MySQL developers |
| 2013 | MySQL 5.6 — major performance and replication improvements |
| 2015 | MySQL 5.7 — JSON support, virtual columns |
| 2018 | MySQL 8.0 — window functions, CTEs, roles, UTF8MB4 default |

:::tip[Still Open Source]
MySQL Community Edition remains free and open source under the GPL license. Oracle maintains
both Community (free) and Enterprise (commercial) editions.
:::

## MySQL vs MariaDB

When Oracle acquired MySQL, original creator Michael Widenius forked the project into **MariaDB**
(named after his other daughter, Maria). MariaDB was designed as a drop-in replacement.

| Feature | MySQL 8.0 | MariaDB 10.x |
|---------|-----------|--------------|
| License | GPL (Community) | GPL |
| JSON | Native JSON type | Longtext-based (until 10.7) |
| Window functions | Yes (8.0+) | Yes (10.2+) |
| CTEs | Yes (8.0+) | Yes (10.2+) |
| GTID replication | Yes | Yes (different implementation) |
| Authentication | caching_sha2_password default | mysql_native_password default |
| Storage engines | InnoDB, MyISAM, Memory, CSV, Archive | + Aria, ColumnStore, Spider, Connect |
| Roles | Yes (8.0+) | Yes (10.0+) |
| Invisible indexes | Yes (8.0+) | Yes (10.6+) |
| Full-text | InnoDB + MyISAM | InnoDB + MyISAM + Mroonga |
| Backward compat | Breaks some MySQL 5.7 syntax | Stricter 5.7 compat |

:::caution[Compatibility Warning]
MySQL 8.0 changed the default authentication plugin to `caching_sha2_password`. Older clients
may fail to connect. Use `mysql_native_password` if needed for legacy compatibility.
:::

## MySQL vs PostgreSQL

Both are mature, production-ready open-source RDBMS. The choice depends on your use case.

| Feature | MySQL 8.0 | PostgreSQL 16 |
|---------|-----------|---------------|
| ACID compliance | Full (InnoDB) | Full |
| JSON | JSON type + functions | JSONB (binary, indexed) |
| Array types | No | Yes |
| Full-text search | Basic | Advanced (tsvector/tsquery) |
| Window functions | Yes | Yes (more complete) |
| CTEs | Yes | Yes (writable) |
| Materialized views | No (use views + tables) | Yes |
| Partitioning | Yes | Yes (more flexible) |
| Parallel queries | Limited | Excellent |
| Extensions | Limited | Rich ecosystem (PostGIS, TimescaleDB…) |
| Replication | Mature, widely used | Logical + streaming |
| Performance | Excellent for OLTP reads | Better for complex queries |
| Learning curve | Gentler | Steeper but more powerful |
| UPSERT | `ON DUPLICATE KEY UPDATE` | `ON CONFLICT DO UPDATE` |

## Storage Engines

MySQL's pluggable storage engine architecture lets different tables use different engines.
The engine handles how data is stored, indexed, and retrieved.

### InnoDB (Default)

InnoDB is the default and recommended storage engine since MySQL 5.5.

**Key features:**
- Full ACID transactions (BEGIN, COMMIT, ROLLBACK, SAVEPOINT)
- Foreign key constraints with `ON DELETE` / `ON UPDATE` actions
- Row-level locking (reduces contention vs table-level)
- Clustered primary key index (data stored with PK)
- MVCC (Multi-Version Concurrency Control) for consistent reads
- Crash recovery via redo/undo logs
- Full-text indexing (MySQL 5.6+)
- Adaptive hash index for frequently-accessed data
- Buffer pool for caching data and indexes in memory

### MyISAM

The legacy default engine before MySQL 5.5. Still present but largely deprecated.

**Characteristics:**
- No transactions — single write crashes corrupt the table
- No foreign key support
- Table-level locking (major bottleneck under concurrent writes)
- Faster for read-heavy, single-user workloads
- Full-text indexing (historically the only engine with this)
- Smaller storage footprint for static data
- Used by MySQL internal system tables (some)

### Comparison: InnoDB vs MyISAM

| Feature | InnoDB | MyISAM |
|---------|--------|--------|
| Transactions | ✅ | ❌ |
| Foreign keys | ✅ | ❌ |
| Row-level locking | ✅ | ❌ (table-level) |
| Crash recovery | ✅ Automatic | ❌ Manual repair needed |
| Full-text search | ✅ (5.6+) | ✅ |
| MVCC | ✅ | ❌ |
| `COUNT(*)` speed | Slower (counts rows) | Fast (stored metadata) |
| Memory usage | Higher | Lower |
| Recommended for | Production workloads | Read-only, legacy |

### Other Engines

| Engine | Purpose |
|--------|---------|
| MEMORY | All data in RAM; lost on restart. Fast temp tables. |
| CSV | Data stored as CSV files. Easy import/export. |
| ARCHIVE | Compressed, insert-only. Great for logs/auditing. |
| FEDERATED | Query remote MySQL servers as local tables. |
| BLACKHOLE | Accepts writes, stores nothing. Used in replication. |
| NDB (Cluster) | Distributed, high-availability clustering. |

```sql
-- Check available storage engines
SHOW ENGINES;

-- Check which engine a table uses
SHOW TABLE STATUS LIKE 'users';
SELECT ENGINE FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'mydb' AND TABLE_NAME = 'users';

-- Create table with specific engine
CREATE TABLE logs (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  message TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=ARCHIVE;

-- Convert engine
ALTER TABLE legacy_table ENGINE=InnoDB;
```

## MySQL 8.0+ Features

MySQL 8.0 was a major release with many long-awaited features:

### Window Functions

```sql
-- Rank employees by salary within department
SELECT
  name,
  department,
  salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank,
  DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank,
  ROW_NUMBER() OVER (ORDER BY salary DESC) AS overall_row,
  LAG(salary) OVER (PARTITION BY department ORDER BY salary) AS prev_salary,
  LEAD(salary) OVER (PARTITION BY department ORDER BY salary) AS next_salary,
  SUM(salary) OVER (PARTITION BY department) AS dept_total
FROM employees;
```

### Common Table Expressions (CTEs)

```sql
-- Recursive CTE for hierarchical data
WITH RECURSIVE org_chart AS (
  -- Anchor member
  SELECT id, name, manager_id, 0 AS level
  FROM employees
  WHERE manager_id IS NULL

  UNION ALL

  -- Recursive member
  SELECT e.id, e.name, e.manager_id, oc.level + 1
  FROM employees e
  INNER JOIN org_chart oc ON e.manager_id = oc.id
)
SELECT * FROM org_chart ORDER BY level, name;
```

### Roles

```sql
-- Create and assign roles (MySQL 8.0+)
CREATE ROLE 'app_read', 'app_write';
GRANT SELECT ON mydb.* TO 'app_read';
GRANT INSERT, UPDATE, DELETE ON mydb.* TO 'app_write';

CREATE USER 'alice'@'localhost' IDENTIFIED BY 'password';
GRANT 'app_read', 'app_write' TO 'alice'@'localhost';
SET DEFAULT ROLE ALL TO 'alice'@'localhost';
```

### Other 8.0 Highlights

- **Invisible indexes** — disable an index without dropping it to test query plans
- **Descending indexes** — `CREATE INDEX idx ON t (col DESC)` for reverse-sort queries
- **JSON improvements** — `JSON_TABLE()`, `JSON_ARRAYAGG()`, `JSON_OBJECTAGG()`, multi-valued indexes
- **Atomic DDL** — `CREATE TABLE`, `DROP TABLE`, `ALTER TABLE` are now atomic (crash-safe)
- **UTF8MB4 default** — `utf8mb4_0900_ai_ci` is the new default collation
- **`INSTANT` ADD COLUMN** — adding columns without table rebuild in many cases
- **`CHECK` constraints** — enforced (were parsed but ignored in 5.7)
- **Hash join** — optimizer can use hash joins for equi-joins
- **Skip Scan range** — index optimization for non-leftmost prefix queries

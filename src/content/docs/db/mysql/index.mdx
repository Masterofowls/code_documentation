# MySQL

MySQL is the world's most popular open-source **relational database management system (RDBMS)**. It powers some of the largest websites on earth — Wikipedia, Twitter, Facebook, YouTube — and remains a cornerstone of the LAMP stack (Linux, Apache, MySQL, PHP/Python/Perl).

## History

MySQL was originally developed by Swedish company **MySQL AB**, founded by Michael Widenius ("Monty"), David Axmark, and Allan Larsson. Version 1.0 was released internally in 1994, with the first public release arriving in 1995.

| Year | Event |
|------|-------|
| 1995 | MySQL 1.0 — first public release |
| 2001 | MySQL 4.0 — InnoDB integration |
| 2004 | MySQL 5.0 — stored procedures, views, triggers |
| 2008 | **Sun Microsystems acquires MySQL AB** for ~$1 billion |
| 2010 | **Oracle acquires Sun Microsystems**, inheriting MySQL |
| 2013 | MySQL 5.6 — performance improvements, InnoDB enhancements |
| 2016 | MySQL 5.7 — JSON support, generated columns |
| 2018 | **MySQL 8.0** — window functions, CTEs, roles, atomic DDL |
| 2023 | MySQL 8.1+ — innovation track releases |

:::tip[Open Source Licensing]
MySQL is dual-licensed: the **Community Edition** is GPL v2 open source. Commercial licenses are available from Oracle for proprietary products that cannot comply with GPL.
:::

## MySQL vs MariaDB

After Oracle's acquisition, the original MySQL creator Michael Widenius forked MySQL to create **MariaDB** in 2009 (named after his daughter Maria).

| Feature | MySQL 8.0+ | MariaDB 10.x |
|---------|-----------|-------------|
| Owner | Oracle | MariaDB Foundation |
| License | GPL / Commercial | GPL |
| Default storage engine | InnoDB | InnoDB (Aria for internal) |
| JSON | JSON type with functions | JSON type (alias for LONGTEXT) |
| Window functions | ✅ MySQL 8.0+ | ✅ MariaDB 10.2+ |
| CTEs | ✅ MySQL 8.0+ | ✅ MariaDB 10.2+ |
| Temporal tables | ❌ | ✅ |
| Galera Cluster | Via MySQL NDB | Built-in option |
| Oracle compatibility | Limited | Better (CONNECT engine) |
| Wire compatibility | MySQL protocol | MySQL protocol compatible |

:::caution[MariaDB Divergence]
MariaDB and MySQL are increasingly divergent. Features added in MySQL 8.0 (roles, invisible indexes, window functions) may work differently or have different syntax in MariaDB. Always test against your specific version.
:::

## Editions

### Community Edition
The free, open-source version licensed under GPL v2. Suitable for most production workloads. All features covered in this documentation refer to the Community Edition unless otherwise stated.

### MySQL Enterprise Edition
Oracle's commercial offering adds:
- **MySQL Enterprise Monitor** — real-time query analysis
- **MySQL Enterprise Backup** — hot backup with incremental support
- **MySQL Enterprise Security** — PAM authentication, LDAP
- **MySQL Enterprise Firewall** — query whitelisting
- **MySQL Enterprise Audit** — compliance logging
- **MySQL HeatWave** — in-memory analytics engine (cloud)

### MySQL Cluster (NDB)
A separate product providing synchronous replication and shared-nothing clustering for high availability and real-time performance. Uses the NDB storage engine.

## Storage Engines

MySQL's pluggable storage engine architecture is one of its defining features. Different tables in the same database can use different engines.

```sql
-- See available engines
SHOW ENGINES;

-- See which engine a table uses
SHOW TABLE STATUS LIKE 'orders'\G

-- Specify engine at creation
CREATE TABLE events (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255)
) ENGINE=InnoDB;

-- Change engine (rebuilds the table)
ALTER TABLE old_myisam_table ENGINE=InnoDB;
```

### InnoDB vs MyISAM

| Feature | InnoDB | MyISAM |
|---------|--------|--------|
| **Transactions** | ✅ Full ACID | ❌ |
| **Foreign keys** | ✅ | ❌ |
| **Row-level locking** | ✅ | ❌ (table-level) |
| **Crash recovery** | ✅ Automatic (redo log) | ❌ Manual REPAIR TABLE |
| **Full-text search** | ✅ MySQL 5.6+ | ✅ |
| **MVCC** | ✅ | ❌ |
| **Clustered index** | ✅ (PK is clustered) | ❌ (heap table) |
| **Buffer pool** | ✅ Configurable | Key cache only |
| **COUNT(\*)** | Slower (no stored count) | Faster (stored metadata) |
| **Compressed tables** | ✅ ROW_FORMAT=COMPRESSED | ✅ PACK_KEYS |
| **Use case** | OLTP, general purpose | Legacy read-only archives |

:::caution[MyISAM is Legacy]
**Do not use MyISAM for new tables.** InnoDB has been the default engine since MySQL 5.5 (2010). MyISAM lacks transactions and foreign keys, and without crash recovery, a server crash can leave MyISAM tables corrupt. Use InnoDB for everything.
:::

### Other Notable Engines

- **MEMORY** — all data in RAM, HASH indexes supported, lost on restart. Use for temporary lookup tables.
- **ARCHIVE** — compressed, insert-only. Good for log archives.
- **BLACKHOLE** — discards all data, used in replication relay servers.
- **CSV** — stores data as CSV files on disk.
- **FEDERATED** — accesses tables on remote MySQL servers.

## MySQL 8.0+ Key Features

MySQL 8.0 was a major release that brought MySQL closer to SQL standard compliance and added enterprise-grade features.

### Window Functions
```sql
SELECT
  name,
  department,
  salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank,
  SUM(salary) OVER (PARTITION BY department) AS dept_total,
  AVG(salary) OVER () AS company_avg
FROM employees;
```

### Common Table Expressions (CTEs)
```sql
-- Recursive CTE for org chart traversal
WITH RECURSIVE org_tree AS (
  SELECT id, name, manager_id, 0 AS depth
  FROM employees
  WHERE manager_id IS NULL        -- root
  UNION ALL
  SELECT e.id, e.name, e.manager_id, ot.depth + 1
  FROM employees e
  INNER JOIN org_tree ot ON e.manager_id = ot.id
)
SELECT depth, name FROM org_tree ORDER BY depth, name;
```

### Roles
```sql
CREATE ROLE 'app_read', 'app_write';
GRANT SELECT ON myapp.* TO 'app_read';
GRANT INSERT, UPDATE, DELETE ON myapp.* TO 'app_write';
GRANT 'app_read' TO 'alice'@'%';
GRANT 'app_read', 'app_write' TO 'bob'@'%';
SET DEFAULT ROLE 'app_read' TO 'alice'@'%';
```

### Invisible Indexes
```sql
-- Make an index invisible without dropping it (test impact)
ALTER TABLE orders ALTER INDEX idx_status INVISIBLE;
-- Query optimizer ignores invisible indexes
-- Make it visible again
ALTER TABLE orders ALTER INDEX idx_status VISIBLE;
```

### Descending Indexes
```sql
-- MySQL 8.0+ supports true descending index storage
CREATE INDEX idx_created_desc ON logs (created_at DESC);
-- Avoids filesort for ORDER BY created_at DESC
```

### JSON Improvements
```sql
-- Inline path operator (MySQL 5.7.9+)
SELECT data->'$.name' AS name FROM users;        -- returns JSON string
SELECT data->>'$.name' AS name FROM users;       -- unquotes the string

-- JSON_TABLE (MySQL 8.0)
SELECT jt.*
FROM config,
JSON_TABLE(config.options, '$[*]' COLUMNS(
  name VARCHAR(50) PATH '$.name',
  value INT PATH '$.value'
)) AS jt;

-- Multi-valued indexes (MySQL 8.0.17+)
CREATE INDEX idx_tags ON articles ((CAST(tags AS CHAR(50) ARRAY)));
```

### Atomic DDL
In MySQL 8.0, DDL statements are atomic — either fully committed or fully rolled back. In MySQL 5.7, a crash mid-DDL could leave the data dictionary inconsistent.

```sql
-- This is now atomic in MySQL 8.0
DROP TABLE IF EXISTS t1, t2;
-- If t2 doesn't exist, t1 is NOT dropped (or both fail cleanly)
```

## Connectors

<CardGrid>
  <Card title="Node.js: mysql2" icon="seti:javascript">
    The `mysql2` package supports async/await, prepared statements, connection pools, and MySQL 8.0 auth plugins. Prefer it over the older `mysql` package.
    ```
    npm install mysql2
    ```
  </Card>
  <Card title="Python: mysql-connector-python" icon="seti:python">
    Oracle's official Python connector. Also consider `PyMySQL` (pure Python) or `mysqlclient` (C extension, fastest) via SQLAlchemy.
    ```
    pip install mysql-connector-python
    ```
  </Card>
  <Card title="PHP: mysqlnd" icon="seti:php">
    MySQL Native Driver is bundled with PHP 5.3+ and powers both `mysqli` and `PDO_MySQL`. Prefer `PDO` for portability.
  </Card>
  <Card title="Java: JDBC" icon="seti:java">
    Use the MySQL Connector/J JDBC driver. Include `com.mysql:mysql-connector-j` in your Maven/Gradle build. Supports connection pooling via HikariCP.
  </Card>
  <Card title="Go: go-sql-driver/mysql" icon="seti:go">
    The standard Go MySQL driver implements `database/sql`. Supports context cancellation, prepared statements, and connection pooling.
    ```
    go get github.com/go-sql-driver/mysql
    ```
  </Card>
  <Card title="Rust: sqlx / diesel" icon="seti:rust">
    `sqlx` provides async, compile-time checked queries. `diesel` is a type-safe ORM. Both support MySQL through the `mysql` feature flag.
  </Card>
</CardGrid>

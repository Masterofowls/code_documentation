# MySQL

MySQL is an open-source relational database management system (RDBMS) that uses Structured Query Language (SQL) for managing and querying data. It is the backbone of millions of web applications worldwide and remains one of the most widely deployed databases in the industry.

## History

MySQL was originally created by **Michael Widenius** (Monty), **David Axmark**, and **Allan Larsson** in 1994–1995 under the company MySQL AB. The name "MySQL" is a combination of "My" (the name of Monty's daughter) and "SQL."

Key milestones:

- **1995** — First internal release of MySQL.
- **2000** — MySQL AB releases the source code under the GPL license.
- **2001** — MySQL 3.23 becomes widely adopted for LAMP stack development.
- **2008** — **Sun Microsystems** acquires MySQL AB for approximately $1 billion.
- **2010** — **Oracle Corporation** acquires Sun Microsystems, bringing MySQL under Oracle's umbrella.
- **2010** — **MariaDB** is forked from MySQL by Monty Widenius in response to concerns about Oracle's stewardship.
- **2013** — MySQL 5.6 introduces significant performance and replication improvements.
- **2016** — MySQL 5.7 brings JSON support and enhanced InnoDB.
- **2018** — MySQL 8.0 released with window functions, CTEs, and a host of modern SQL features.

:::tip[Check Your Version]
Run `SELECT VERSION();` to see exactly which MySQL version your server is running. Many features discussed here require MySQL 8.0+.
:::

## MySQL vs MariaDB

After Oracle's acquisition, concerns about MySQL's open-source future led to the creation of MariaDB as a drop-in replacement. While they share a common ancestry, they have diverged significantly.

| Feature | MySQL | MariaDB |
|---|---|---|
| Creator | Oracle | MariaDB Foundation |
| Default storage engine | InnoDB | InnoDB |
| JSON support | Native JSON type (8.0) | Dynamic columns + JSON alias |
| Window functions | 8.0+ | 10.2+ |
| CTEs | 8.0+ | 10.2+ |
| Roles | 8.0+ | 10.0.5+ |
| Invisible indexes | 8.0+ | 10.6+ |
| Thread pool | Enterprise only | Included |
| Authentication | caching_sha2_password (8.0 default) | ed25519 |
| License | GPL v2 (Community) | GPL v2 |
| Compatibility | Reference | High (diverging over time) |

:::caution[Compatibility Warning]
Although MariaDB is largely compatible with MySQL syntax, some MySQL 8.0 features (like `caching_sha2_password`, window function syntax differences, and JSON function names) may not behave identically in MariaDB. Always test migrations thoroughly.
:::

## MySQL Editions

### Community Edition (Free)

The Community Edition is the freely available, open-source version licensed under the GPL. It includes:

- InnoDB storage engine with full transactional support.
- Replication (binary log, GTID, Group Replication).
- Partitioning.
- Full-text search.
- JSON support (8.0+).
- Performance Schema.

### Enterprise Edition (Commercial)

Oracle's commercial offering adds:

- **MySQL Enterprise Monitor** — real-time monitoring and advisors.
- **MySQL Enterprise Backup** — hot backups with incremental/compressed support.
- **MySQL Enterprise Security** — external authentication (LDAP, Kerberos, PAM), data masking, audit log.
- **MySQL Enterprise Firewall** — query allow-listing against SQL injection.
- **MySQL Enterprise Thread Pool** — efficient thread management for high-concurrency workloads.
- **MySQL Router** — transparent routing for InnoDB Cluster.

### MySQL HeatWave (Cloud)

Oracle's cloud-native offering with an in-memory query accelerator (HeatWave) for analytics directly within the MySQL database engine, eliminating the need for separate ETL pipelines.

## Storage Engines

MySQL's pluggable storage engine architecture is one of its defining characteristics. The engine handles how data is stored, indexed, and retrieved.

### InnoDB

InnoDB is the **default and recommended** storage engine since MySQL 5.5.

| Feature | Support |
|---|---|
| Transactions | ✅ Full ACID |
| Foreign keys | ✅ Yes |
| Row-level locking | ✅ Yes |
| Crash recovery | ✅ Automatic (redo log) |
| Full-text search | ✅ (5.6+) |
| MVCC | ✅ Yes |
| Clustered primary key | ✅ Yes |
| Buffer pool | ✅ In-memory cache |

### MyISAM

The legacy default before InnoDB took over. Still occasionally encountered in older systems.

| Feature | Support |
|---|---|
| Transactions | ❌ None |
| Foreign keys | ❌ None |
| Row-level locking | ❌ Table-level only |
| Crash recovery | ❌ Manual repair needed |
| Full-text search | ✅ Yes (older versions) |
| MVCC | ❌ None |
| Clustered primary key | ❌ No |
| Compression | ✅ (read-only) |

:::caution[Avoid MyISAM for New Projects]
MyISAM lacks transactions, foreign key constraints, and row-level locking. It is essentially obsolete for new development. Always use InnoDB unless you have a very specific legacy reason.
:::

### Other Storage Engines

| Engine | Use Case |
|---|---|
| `MEMORY` | Temporary in-memory tables; data lost on restart |
| `ARCHIVE` | High-compression, insert-only archival storage |
| `CSV` | Tables stored as comma-separated value files |
| `BLACKHOLE` | Accepts writes but discards them; useful in replication topologies |
| `FEDERATED` | Access tables on remote MySQL servers |
| `NDB` (Cluster) | MySQL Cluster distributed engine |

## MySQL 8.0+ Features

MySQL 8.0 was a landmark release, closing the gap with PostgreSQL on modern SQL features.

### Window Functions

```sql
SELECT
  employee_id,
  department,
  salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank,
  SUM(salary) OVER (PARTITION BY department) AS dept_total,
  AVG(salary) OVER (ORDER BY hire_date ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) AS rolling_avg
FROM employees;
```

### Common Table Expressions (CTEs)

```sql
-- Regular CTE
WITH regional_sales AS (
  SELECT region, SUM(amount) AS total
  FROM orders
  GROUP BY region
),
top_regions AS (
  SELECT region FROM regional_sales WHERE total > 100000
)
SELECT * FROM orders WHERE region IN (SELECT region FROM top_regions);

-- Recursive CTE (org chart)
WITH RECURSIVE org AS (
  SELECT id, name, manager_id, 0 AS depth
  FROM employees WHERE manager_id IS NULL
  UNION ALL
  SELECT e.id, e.name, e.manager_id, o.depth + 1
  FROM employees e
  INNER JOIN org o ON e.manager_id = o.id
)
SELECT * FROM org ORDER BY depth, name;
```

### Roles

```sql
CREATE ROLE 'read_only', 'app_writer';
GRANT SELECT ON mydb.* TO 'read_only';
GRANT SELECT, INSERT, UPDATE, DELETE ON mydb.* TO 'app_writer';
GRANT 'read_only' TO 'reporting_user'@'%';
SET DEFAULT ROLE 'read_only' TO 'reporting_user'@'%';
```

### Invisible Indexes

```sql
-- Make an index invisible (optimizer ignores it but it's maintained)
ALTER TABLE orders ALTER INDEX idx_created_at INVISIBLE;
-- Test query performance without the index
-- Then re-enable
ALTER TABLE orders ALTER INDEX idx_created_at VISIBLE;
```

### Descending Indexes

```sql
-- MySQL 8.0 properly supports descending index scans
CREATE INDEX idx_created_desc ON orders (created_at DESC);
```

### JSON Improvements

```sql
-- JSON_TABLE: unpack JSON arrays into rows
SELECT *
FROM JSON_TABLE(
  '[{"id":1,"name":"Alice"},{"id":2,"name":"Bob"}]',
  '$[*]' COLUMNS (
    id INT PATH '$.id',
    name VARCHAR(50) PATH '$.name'
  )
) AS jt;

-- Multi-valued indexes on JSON arrays (8.0.17+)
CREATE INDEX idx_tags ON products ((CAST(tags->'$[*]' AS UNSIGNED ARRAY)));
```

### Atomic DDL

```sql
-- In MySQL 8.0, DDL statements are atomic (crash-safe)
-- If this fails, the table is not partially created
CREATE TABLE t1 (id INT PRIMARY KEY);
-- The dictionary is stored in InnoDB, not .frm files
```

## MySQL Connectors

<CardGrid>
  <Card title="Node.js — mysql2" icon="seti:db">
    The `mysql2` package is the modern, Promise-based MySQL client for Node.js with full prepared statement support and connection pooling.

    ```bash
    npm install mysql2
    ```

    ```js
    import mysql from 'mysql2/promise';
    const pool = mysql.createPool({ host, user, password, database });
    const [rows] = await pool.query('SELECT * FROM users WHERE id = ?', [1]);
    ```
  </Card>
  <Card title="Python — mysql-connector-python" icon="seti:python">
    Oracle's official Python connector, or use `PyMySQL` for a pure-Python alternative.

    ```bash
    pip install mysql-connector-python
    ```

    ```python
    import mysql.connector
    cnx = mysql.connector.connect(host='localhost', user='root', database='mydb')
    cursor = cnx.cursor(dictionary=True)
    cursor.execute("SELECT * FROM users WHERE id = %s", (1,))
    ```
  </Card>
  <Card title="PHP — mysqlnd" icon="seti:php">
    The MySQL Native Driver (`mysqlnd`) ships with PHP 5.3+ and is the recommended extension, replacing the older `mysql_*` functions. Use PDO or `mysqli` on top of it.

    ```php
    $pdo = new PDO('mysql:host=localhost;dbname=mydb;charset=utf8mb4', $user, $pass, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]);
    $stmt = $pdo->prepare('SELECT * FROM users WHERE id = ?');
    $stmt->execute([1]);
    ```
  </Card>
  <Card title="Java — JDBC (MySQL Connector/J)" icon="seti:java">
    The official JDBC driver for Java applications. Works with plain JDBC, Spring, Hibernate, and more.

    ```xml
    <!-- Maven -->
    <dependency>
      <groupId>com.mysql</groupId>
      <artifactId>mysql-connector-j</artifactId>
      <version>8.3.0</version>
    </dependency>
    ```

    ```java
    Connection conn = DriverManager.getConnection(
        "jdbc:mysql://localhost:3306/mydb?useSSL=false", user, pass);
    PreparedStatement stmt = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
    stmt.setInt(1, 1);
    ResultSet rs = stmt.executeQuery();
    ```
  </Card>
</CardGrid>

This page covers everything you need to get a PostgreSQL installation up and running, from installation to user management, connections, and database operations.

## Installation

<Tabs>
<TabItem label="Ubuntu/Debian">
```bash
# Add PostgreSQL apt repository
sudo sh -c 'echo "deb https://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

sudo apt-get update
sudo apt-get install -y postgresql-16 postgresql-client-16

# Start and enable service

sudo systemctl start postgresql
sudo systemctl enable postgresql

````
</TabItem>
<TabItem label="RHEL/CentOS">
```bash
# Install PostgreSQL repository
sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# Install
sudo dnf install -y postgresql16-server
sudo /usr/pgsql-16/bin/postgresql-16-setup initdb
sudo systemctl start postgresql-16
sudo systemctl enable postgresql-16
````

</TabItem>

## Service Management

```bash
# Linux systemd
sudo systemctl start   postgresql

sudo systemctl restart postgresql
sudo systemctl status  postgresql

# macOS Homebrew
brew services start   postgresql@16
brew services stop    postgresql@16
brew services restart postgresql@16

# Check if running and accepting connections
pg_isready -h localhost -p 5432
````

## Connecting with psql

```bash
# Connect as the postgres superuser (Linux)
sudo -u postgres psql

# Connect to a specific database
psql -U myuser -d mydb

# Connect to a remote host
psql -U myuser -d mydb -h db.example.com -p 5432

# Using a connection URI
psql "postgresql://myuser:mypassword@localhost:5432/mydb"
psql "postgresql://myuser:mypassword@db.example.com:5432/mydb?sslmode=require"

# Using environment variables
export PGHOST=localhost
export PGPORT=5432
export PGUSER=myuser
export PGPASSWORD=mypassword
export PGDATABASE=mydb
psql   # Uses all the above automatically
```

### The `.pgpass` File

Store passwords securely so you don't need to type them or set `PGPASSWORD`:

```
# ~/.pgpass — format: hostname:port:database:username:password
localhost:5432:mydb:myuser:mypassword
db.prod.example.com:5432:*:admin:secretpassword

# Required permissions
chmod 600 ~/.pgpass
```

## User and Role Management

PostgreSQL uses "roles" as a unified concept for users and groups.

```sql
-- Create a login role (user)
CREATE ROLE app_user
  WITH LOGIN
  PASSWORD 'secure_password'
  CREATEDB               -- Can create databases (optional)
  VALID UNTIL '2025-12-31';  -- Password expiry (optional)

-- Create a superuser (full privileges — use sparingly)
CREATE ROLE dba_admin WITH LOGIN PASSWORD 'secret' SUPERUSER;

-- Create a read-only group role (no login)
CREATE ROLE readonly_group;

-- Grant group membership
GRANT readonly_group TO app_user;

-- Alter an existing role
ALTER ROLE app_user PASSWORD 'new_password';
ALTER ROLE app_user VALID UNTIL 'infinity';

-- Drop a role
DROP ROLE IF EXISTS old_user;
```

## GRANT and REVOKE

```sql
-- Database-level permissions
GRANT CONNECT ON DATABASE mydb TO app_user;
GRANT CREATE ON DATABASE mydb TO developer;

-- Schema-level permissions
GRANT USAGE ON SCHEMA public TO app_user;
GRANT CREATE ON SCHEMA public TO developer;

-- Table-level permissions
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly_group;

-- Future tables (apply to tables created after this GRANT)
ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO app_user;

ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT USAGE, SELECT ON SEQUENCES TO app_user;

-- Revoke permissions
REVOKE DELETE ON orders FROM app_user;

-- Grant ownership
ALTER TABLE products OWNER TO app_user;
```

## Creating Databases

```sql
-- Create a database
CREATE DATABASE myapp
  OWNER = app_user
  ENCODING = 'UTF8'
  LC_COLLATE = 'en_US.UTF-8'
  LC_CTYPE = 'en_US.UTF-8'
  TEMPLATE = template0     -- Use template0 for custom encoding/locale
  CONNECTION LIMIT = 100;  -- Max connections to this database

-- List databases
\l
SELECT datname, datcollate, datacl FROM pg_database;

-- Connect
\c myapp

-- Drop database (must not be connected to it)
DROP DATABASE IF EXISTS old_myapp;
```

## pg_hba.conf Authentication

`pg_hba.conf` controls client authentication. Each rule is checked top-to-bottom; the first match wins.

```
# /etc/postgresql/16/main/pg_hba.conf
# TYPE  DATABASE   USER       ADDRESS         METHOD

# Local socket connections
local   all        postgres                   peer      # Linux system user maps to pg user
local   all        all                        md5       # Other local users need password

# IPv4 connections
host    myapp      app_user   127.0.0.1/32    scram-sha-256  # App server, password
host    myapp      all        10.0.0.0/8      scram-sha-256  # Private network
host    all        all        0.0.0.0/0       reject    # Block everything else

# SSL-required for remote superuser
hostssl all        postgres   0.0.0.0/0       cert      # Certificate auth only
```

After editing `pg_hba.conf`, reload without restarting:

```bash
sudo systemctl reload postgresql
# or in psql:
SELECT pg_reload_conf();
```

## Essential psql Commands

```bash
# List all databases
\l

# Connect to a database
\c mydb

# List tables in current schema
\dt

# List tables in all schemas
\dt *.*

# Describe a table (columns, indexes, constraints)
\d users

# Describe with extra info (storage, stats)
\d+ users

# List schemas
\dn

# List users/roles
\du

# List indexes
\di

# Toggle query timing
\timing on

# Toggle expanded display (one column per line)
\x auto

# Execute a SQL file
\i /path/to/migration.sql

# Export query results to CSV
\copy (SELECT * FROM users) TO '/tmp/users.csv' CSV HEADER;

# Import CSV
\copy users FROM '/tmp/users.csv' CSV HEADER;

# Set output format
\pset format csv
\pset format aligned
\pset format json

# Open query in your editor
\e

# Show current connection info
\conninfo

# List all psql commands
\?
```

## pg_dump and pg_restore

```bash
# Dump a single database to a custom format (recommended)
pg_dump -U postgres -d mydb -Fc -f mydb.dump

# Dump to plain SQL
pg_dump -U postgres -d mydb -f mydb.sql

# Dump specific tables
pg_dump -U postgres -d mydb -t users -t orders -Fc -f partial.dump

# Dump schema only (no data)
pg_dump -U postgres -d mydb --schema-only -f schema.sql

# Dump data only
pg_dump -U postgres -d mydb --data-only -Fc -f data.dump

# Parallel dump (faster for large databases)
pg_dump -U postgres -d mydb -Fd -j 4 -f mydb_dir/

# Restore from custom format
pg_restore -U postgres -d newdb -Fc mydb.dump

# Restore with parallel workers
pg_restore -U postgres -d newdb -j 4 mydb.dump

# Restore specific tables
pg_restore -U postgres -d newdb -t users mydb.dump

# Dump all databases
pg_dumpall -U postgres -f all_databases.sql

# Restore all databases
psql -U postgres -f all_databases.sql
```

:::tip[Use custom format for production backups]
The `-Fc` (custom format) is compressed, supports parallel restore with `-j`, and lets you selectively restore tables. Prefer it over plain SQL for all production backups.
:::

## Connection Pooling Preview

In production, always use a connection pooler between your application and PostgreSQL. Direct application connections are expensive (each creates a new OS process).

```bash
# PgBouncer (most common)
# See the Performance page for full PgBouncer configuration
```

PostgreSQL has a default `max_connections = 100`. Each connection uses ~5-10 MB of RAM. At 100 connections, that's 500 MB - 1 GB just for connection overhead. PgBouncer can multiplex thousands of application connections into a small pool of real database connections.

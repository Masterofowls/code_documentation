PostgreSQL (often called "Postgres") is a powerful, open-source object-relational database system with over 35 years of active development. It is known for its reliability, feature richness, and strict adherence to SQL standards.

## What is PostgreSQL?

PostgreSQL began as the POSTGRES project at UC Berkeley in 1986 and became open source in 1994. Today it is maintained by the PostgreSQL Global Development Group — a community of thousands of contributors worldwide. It is released under the PostgreSQL License (similar to MIT/BSD).

PostgreSQL is ACID-compliant, supports complex queries, foreign keys, triggers, views, transactional integrity, and multiversion concurrency control (MVCC). It is used by companies ranging from small startups to Fortune 500 enterprises, and is the database behind Apple, Instagram, Reddit, Twitch, and many others.

```sql
-- Check your PostgreSQL version
SELECT version();
-- PostgreSQL 16.2 on x86_64-pc-linux-gnu, compiled by gcc 12.2.0, 64-bit

SHOW server_version;
-- 16.2
```

## PostgreSQL vs MySQL vs SQL Server

| Feature | PostgreSQL | MySQL / MariaDB | SQL Server |
|---|---|---|---|
| **License** | Open source (PostgreSQL License) | Open source (GPL) / BSL | Commercial (Microsoft) |
| **ACID compliance** | Full | InnoDB: full | Full |
| **Standards compliance** | Excellent (best of class) | Moderate | Good |
| **JSON support** | JSONB (binary, indexed, operators) | JSON (text-based, limited) | JSON functions (no native type) |
| **Array types** | Native array columns | No | No |
| **Full-text search** | Built-in (tsvector/tsquery) | Limited | Full-text index |
| **Window functions** | Full SQL:2003 | 8.0+ full support | Full support |
| **CTEs** | Full (including writable) | 8.0+ | Full |
| **Partitioning** | Declarative (PG10+) | Basic | Full |
| **Logical replication** | Built-in | Built-in | Transactional replication |
| **Geospatial** | PostGIS extension | Basic spatial | Full spatial |
| **Extensions** | Rich ecosystem (pgvector, TimescaleDB…) | Plugins | Limited |
| **Table inheritance** | Yes | No | No |
| **Advisory locks** | Yes | No | Application locks |
| **LISTEN/NOTIFY** | Built-in pub/sub | No | Service Broker |
| **Parallel queries** | Built-in | Limited | Built-in |
| **Vacuum** | Required (MVCC) | No | No |
| **Max DB size** | Unlimited | ~64 TB | 524 PB (Enterprise) |
| **Max table size** | 32 TB | ~64 TB | 524 PB (Enterprise) |
| **Cloud managed** | RDS, Aurora, Cloud SQL, Supabase, Neon | RDS, Aurora, Cloud SQL | Azure SQL, RDS |

## Standout Features

### JSONB — Binary JSON

PostgreSQL's JSONB type stores JSON data in a binary format that can be indexed with GIN indexes — enabling fast queries on arbitrary JSON structure.

```sql
-- Create a table with JSONB column
CREATE TABLE events (
  id         BIGSERIAL PRIMARY KEY,
  type       TEXT NOT NULL,
  payload    JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Query JSON fields
SELECT payload->>'user_id' AS user_id
FROM events
WHERE type = 'purchase'
  AND payload @> '{"status": "completed"}';   -- Containment operator

-- GIN index for fast JSONB queries
CREATE INDEX idx_events_payload ON events USING GIN (payload);
```

### Native Arrays

```sql
CREATE TABLE users (
  id    BIGSERIAL PRIMARY KEY,
  name  TEXT NOT NULL,
  tags  TEXT[]                -- Array of text
);

SELECT * FROM users WHERE 'admin' = ANY(tags);
SELECT * FROM users WHERE tags @> ARRAY['admin', 'editor'];
```

### Full-Text Search

```sql
-- Built-in full-text search without external tools
SELECT title, ts_rank(search_vector, query) AS rank
FROM articles,
     to_tsquery('english', 'postgresql & database') query
WHERE search_vector @@ query
ORDER BY rank DESC;
```

### Table Partitioning

```sql
CREATE TABLE orders (
  id         BIGSERIAL,
  created_at TIMESTAMPTZ NOT NULL,
  total      NUMERIC(12,2)
) PARTITION BY RANGE (created_at);

CREATE TABLE orders_2024 PARTITION OF orders
  FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');
```

### LISTEN / NOTIFY

Built-in pub/sub messaging within the database:

```sql
-- Publisher
NOTIFY user_activity, '{"user_id": 42, "action": "login"}';

-- Subscriber (in another connection)
LISTEN user_activity;
-- Receives notifications asynchronously
```

### Extensions

PostgreSQL has a rich extension ecosystem:

| Extension | Purpose |
|---|---|
| `pgvector` | Vector similarity search (AI/ML embeddings) |
| `PostGIS` | Geospatial data and queries |
| `TimescaleDB` | Time-series data |
| `pg_trgm` | Trigram-based fuzzy text search |
| `uuid-ossp` | UUID generation functions |
| `pg_stat_statements` | Query performance tracking |
| `pg_cron` | Cron-scheduled jobs inside Postgres |
| `hstore` | Key-value pairs |
| `ltree` | Hierarchical/tree data |
| `pg_partman` | Automated partition management |

```sql
-- Enable an extension
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS pgvector;

-- List installed extensions
SELECT name, default_version, installed_version
FROM pg_available_extensions
WHERE installed_version IS NOT NULL;
```

:::tip[Upgrade Path]
PostgreSQL typically releases one major version per year. Major versions are supported for 5 years. Use `pg_upgrade` to upgrade between major versions with minimal downtime.
:::

## Tools

### psql — CLI Client

The official interactive terminal for PostgreSQL:

```bash
# Connect to a database
psql -U postgres -d mydb -h localhost -p 5432

# Use a connection URI
psql "postgres://user:password@host:5432/dbname"

# Execute a single query
psql -U postgres -d mydb -c "SELECT version();"

# Execute a SQL file
psql -U postgres -d mydb -f schema.sql
```

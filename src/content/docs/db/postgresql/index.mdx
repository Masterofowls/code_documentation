PostgreSQL (pronounced "post-GRESS-cue-el", often called "Postgres") is a powerful open-source object-relational database system with over 35 years of active development. It is renowned for its standards compliance, extensibility, and support for complex data types and queries.

## What Makes PostgreSQL Stand Out

PostgreSQL goes far beyond standard SQL. Its unique features make it a first-class choice for both transactional and analytical workloads:

- **JSONB** — Binary JSON storage with full indexing (GIN indexes), operators, and functions
- **Arrays** — Native array columns with element-level operators and `unnest()`
- **Full-text search** — Built-in `tsvector`/`tsquery` with ranking, without Elasticsearch
- **hstore** — Key-value store in a column (predecessor to JSONB)
- **Extensions** — PostGIS (geospatial), pg_trgm (fuzzy search), uuid-ossp, pgcrypto, and hundreds more
- **Table inheritance** — Tables can inherit from parent tables
- **Logical replication** — Subscribe to row-level changes in real-time
- **Table partitioning** — Range, list, and hash partitioning with partition pruning
- **LISTEN/NOTIFY** — Lightweight pub/sub messaging built into the database
- **Advisory locks** — Application-level locking without requiring a dedicated lock table
- **Custom types** — CREATE TYPE for enums, composite types, domain types
- **Write-ahead logging (WAL)** — Foundation for replication and point-in-time recovery

## PostgreSQL vs Other Databases

| Feature                | PostgreSQL                  | MySQL 8              | SQL Server            | SQLite            |
| ---------------------- | --------------------------- | -------------------- | --------------------- | ----------------- |
| **License**            | PostgreSQL (MIT-like)       | GPL/Commercial       | Commercial            | Public Domain     |
| **ACID compliance**    | Full                        | Full (InnoDB)        | Full                  | Full (WAL mode)   |
| **JSON support**       | JSONB (binary, indexed)     | JSON (text, indexed) | JSON (text, computed) | JSON functions    |
| **Full-text search**   | Built-in                    | Built-in             | Full-text indexes     | FTS5 extension    |
| **Arrays**             | Native columns              | None                 | None                  | None              |
| **Window functions**   | Full SQL:2011               | 8.0+ (partial)       | Full                  | 3.25+ (partial)   |
| **CTEs (WITH)**        | Full + materialized hint    | 8.0+                 | Full                  | 3.35+             |
| **Table partitioning** | Declarative (10+)           | Native               | Native                | None              |
| **Replication**        | Physical + Logical          | Physical + Logical   | Physical + Logical    | None (Litestream) |
| **Geospatial (GIS)**   | PostGIS (best-in-class)     | Limited              | Limited               | SpatiaLite        |
| **Stored procedures**  | PL/pgSQL, PL/Python, etc.   | Yes                  | T-SQL                 | None              |
| **Max DB size**        | Unlimited                   | Unlimited            | Unlimited             | 281 TB            |
| **Concurrent writes**  | MVCC (no write locks)       | MVCC                 | Row/page locks        | WAL mode          |
| **Extensions**         | 1000+ on PGXN               | Limited              | Limited               | None              |
| **LISTEN/NOTIFY**      | Built-in                    | None                 | Service Broker        | None              |
| **Community**          | Very active, non-commercial | Oracle-owned         | Microsoft             | SQLite org        |

| Version        | Year | Notable Features                                            |
| -------------- | ---- | ----------------------------------------------------------- |
| PostgreSQL 9.4 | 2014 | JSONB, materialized view refresh CONCURRENTLY               |

| PostgreSQL 14  | 2021 | TOAST compression, range multirange types                   |

:::tip[Current Stable Release]
PostgreSQL 17 is the current major release. PostgreSQL 16 remains supported. Check the [official release policy](https://www.postgresql.org/support/versioning/) — each major version is supported for 5 years.
:::

## psql — The Command Line Client

`psql` is the official PostgreSQL interactive terminal. It is pre-installed with PostgreSQL and essential for administration.

```bash
# Connect to a database
psql -U postgres -d mydb -h localhost -p 5432

# Connect with a URI
psql "postgres://user:password@host:5432/dbname"
psql "postgresql://postgres@localhost/mydb?sslmode=require"

# Run a command and exit
psql -U postgres -c "SELECT version();"

# Execute a SQL file
psql -U postgres -d mydb -f schema.sql

# Connect as the postgres OS user (Linux)
sudo -u postgres psql
```

### Essential psql Meta-Commands

```bash
\l                      # list all databases
\c dbname               # connect to a database
\dt                     # list tables in current schema
\dt schema.*            # list tables in specific schema
\d tablename            # describe a table (columns, indexes, constraints)
\d+ tablename           # verbose describe
\dn                     # list schemas
\du                     # list roles/users
\di                     # list indexes
\dv                     # list views
\dm                     # list materialized views
\df                     # list functions
\dp tablename           # show table privileges
\timing                 # toggle query timing display
\e                      # open query buffer in $EDITOR
\i file.sql             # execute a SQL file
\o output.txt           # redirect query output to file
\x                      # toggle expanded display (vertical output)
\pset format csv        # set output format: aligned, unaligned, csv, json, html
\copy table FROM 'file.csv' CSV HEADER  # client-side copy
\q                      # quit
```

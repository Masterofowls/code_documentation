PostgreSQL (often called "Postgres") is a powerful, open-source object-relational database system with over 35 years of active development. It is known for its reliability, feature richness, and strong standards compliance.

## What is PostgreSQL?

PostgreSQL extends the SQL standard with advanced data types, programmability, and extensibility that no other open-source database matches. It is fully ACID-compliant, supports complex queries, foreign keys, triggers, views, transactional integrity, and multiversion concurrency control (MVCC).

Unlike MySQL (which prioritizes simplicity and speed) or SQLite (which prioritizes embeddability), PostgreSQL prioritizes correctness, completeness, and flexibility. It is the default choice for complex applications, analytics workloads, and systems requiring advanced data modeling.

## History

PostgreSQL evolved from the POSTGRES project at UC Berkeley (1986), designed by Michael Stonebraker. It was first released to the public in 1989. The SQL query language was added in 1994, and the project was renamed PostgreSQL. The first open-source release (v6.0) came in 1996.

## PostgreSQL vs MySQL vs SQL Server

| Feature | PostgreSQL | MySQL | SQL Server |
|---|---|---|---|
| **License** | PostgreSQL (open source) | GPL / Commercial | Proprietary |
| **ACID compliance** | Full | InnoDB: full | Full |
| **JSON support** | JSONB (binary, indexed, fastest) | JSON (good) | JSON (good) |
| **Arrays** | Native type | No | No |
| **Full-text search** | tsvector/tsquery + GIN | FULLTEXT index | CONTAINS/FREETEXT |
| **Window functions** | Complete SQL:2003 | MySQL 8.0+ (complete) | Complete |
| **CTEs** | Full (writeable, recursive) | 8.0+ (recursive) | Full |
| **Table partitioning** | Native (range/list/hash) | Native | Enterprise feature |
| **Logical replication** | Native | Native (binlog) | Replication |
| **Extensions** | Rich ecosystem (PostGIS, TimescaleDB, pgvector...) | Limited | Limited |
| **LISTEN/NOTIFY** | Yes | No | No |
| **Lateral joins** | Yes | Yes (8.0+) | Yes |
| **Parallel queries** | Yes | Limited | Yes |
| **RETURNING** | Yes | No | OUTPUT clause |
| **Max DB size** | Unlimited | Unlimited | Edition-dependent |
| **Concurrency model** | MVCC | MVCC (InnoDB) | Locking + MVCC |

## Standout Features

### JSONB — Binary JSON with Indexing

JSONB stores JSON as a parsed binary format, enabling efficient queries and GIN indexing:

```sql
-- Store structured data with schema flexibility
CREATE TABLE events (
  id      BIGSERIAL PRIMARY KEY,
  type    TEXT NOT NULL,
  payload JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Query nested JSON
SELECT payload->>'user_id' FROM events WHERE type = 'purchase';

-- GIN index makes JSON queries fast
CREATE INDEX ON events USING GIN (payload);
SELECT * FROM events WHERE payload @> '{"status": "completed"}';
```

### Native Array Types

```sql
-- Store multiple values without a junction table
CREATE TABLE articles (
  id   SERIAL PRIMARY KEY,
  tags TEXT[] DEFAULT '{}'
);

SELECT * FROM articles WHERE tags @> ARRAY['postgresql', 'sql'];
```

### Full-Text Search

```sql
CREATE INDEX ON articles USING GIN (to_tsvector('english', title || ' ' || body));

SELECT title, ts_rank(to_tsvector('english', body), query) AS rank
FROM articles, to_tsquery('english', 'postgresql & index') AS query
WHERE to_tsvector('english', body) @@ query
ORDER BY rank DESC;
```

### Extensions

PostgreSQL has a rich extension ecosystem:

| Extension | Purpose |
|---|---|
| `postgis` | Geographic/spatial data |
| `pgvector` | Vector similarity search (AI/ML embeddings) |
| `timescaledb` | Time-series data |
| `pg_trgm` | Trigram similarity and fuzzy search |
| `hstore` | Key-value store |
| `uuid-ossp` | UUID generation functions |
| `pg_stat_statements` | Query performance tracking |
| `pg_cron` | Cron job scheduling |
| `pgcrypto` | Cryptographic functions |
| `citext` | Case-insensitive text type |

```sql
-- Install an extension (requires superuser)
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE EXTENSION IF NOT EXISTS pgvector;

-- List installed extensions
SELECT * FROM pg_extension;
```

### Logical Replication and LISTEN/NOTIFY

```sql
-- Real-time pub/sub between database and application
LISTEN order_updates;
NOTIFY order_updates, '{"order_id": 42, "status": "shipped"}';
```

### Table Partitioning

```sql
CREATE TABLE events (
  id         BIGSERIAL,
  type       TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (created_at);

CREATE TABLE events_2024 PARTITION OF events
  FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');
```

### Advisory Locks

```sql
-- Application-level distributed locks
SELECT pg_advisory_lock(12345);       -- Lock with custom key
-- ... critical section ...
SELECT pg_advisory_unlock(12345);
```

## psql CLI

`psql` is the official PostgreSQL interactive terminal. Learn these commands:

```bash
# Connect
psql -U postgres -d mydb -h localhost -p 5432

# Connection string
psql "postgresql://user:pass@host:5432/dbname"
```

Essential psql meta-commands:

| Command | Description |
|---|---|
| `\l` | List databases |
| `\c dbname` | Connect to database |
| `\dt` | List tables |
| `\dt schema.*` | List tables in schema |
| `\d tablename` | Describe table |
| `\d+ tablename` | Describe table with extra info |
| `\dn` | List schemas |
| `\du` | List users/roles |
| `\di` | List indexes |
| `\timing` | Toggle query timing |
| `\e` | Open query in $EDITOR |
| `\i file.sql` | Execute SQL file |
| `\o file.txt` | Write output to file |
| `\copy` | Copy data to/from file |
| `\x` | Toggle expanded display |
| `\q` | Quit |

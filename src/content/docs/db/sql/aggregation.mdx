Aggregation and window functions are the analytical power tools of SQL. They let you compute summaries across rows while maintaining the ability to access individual row values.

## Aggregate Functions

Aggregate functions operate on a set of rows and return a single value per group.

### COUNT

```sql
-- COUNT(*) counts all rows including NULLs
SELECT COUNT(*) FROM orders;

-- COUNT(column) counts non-NULL values only
SELECT COUNT(cancelled_at) FROM orders;    -- Only counts rows where cancelled_at is not NULL

-- COUNT(DISTINCT column) counts unique non-NULL values
SELECT COUNT(DISTINCT customer_id) FROM orders;

-- Conditional count
SELECT
  COUNT(*)                                        AS total_orders,
  COUNT(CASE WHEN status = 'completed' THEN 1 END) AS completed,
  COUNT(CASE WHEN status = 'cancelled' THEN 1 END) AS cancelled
FROM orders;
```

### SUM and AVG

```sql
SELECT
  SUM(total)                              AS revenue,
  AVG(total)                              AS avg_order_value,
  SUM(total) / COUNT(DISTINCT customer_id) AS revenue_per_customer
FROM orders
WHERE status = 'completed';
```

### MIN and MAX

```sql
SELECT
  MIN(price)         AS cheapest,
  MAX(price)         AS most_expensive,
  MAX(price) - MIN(price) AS price_range
FROM products
WHERE category_id = 5;
```

### String Aggregation

```sql
-- PostgreSQL: STRING_AGG
SELECT
  category_id,
  STRING_AGG(name, ', ' ORDER BY name)    AS product_names,
  STRING_AGG(name, ', ' ORDER BY price DESC) AS by_price
FROM products
GROUP BY category_id;

-- MySQL: GROUP_CONCAT
SELECT category_id, GROUP_CONCAT(name ORDER BY name SEPARATOR ', ')
FROM products
GROUP BY category_id;

-- PostgreSQL: ARRAY_AGG (returns an array)
SELECT customer_id, ARRAY_AGG(id ORDER BY created_at) AS order_ids
FROM orders
GROUP BY customer_id;

-- PostgreSQL: JSON_AGG
SELECT
  customer_id,
  JSON_AGG(
    JSON_BUILD_OBJECT('id', id, 'total', total, 'status', status)
    ORDER BY created_at DESC
  ) AS orders_json
FROM orders
GROUP BY customer_id;
```

## GROUP BY

### Single Column

```sql
SELECT
  status,
  COUNT(*)     AS order_count,
  SUM(total)   AS total_revenue
FROM orders
GROUP BY status
ORDER BY total_revenue DESC;
```

### Multiple Columns

```sql
SELECT
  DATE_TRUNC('month', created_at) AS month,
  status,
  COUNT(*)                         AS count,
  SUM(total)                       AS revenue
FROM orders
GROUP BY DATE_TRUNC('month', created_at), status
ORDER BY month, status;
```

### GROUP BY with Expression

```sql
SELECT
  EXTRACT(YEAR  FROM created_at) AS year,
  EXTRACT(MONTH FROM created_at) AS month,
  COUNT(*) AS orders
FROM orders
GROUP BY
  EXTRACT(YEAR  FROM created_at),
  EXTRACT(MONTH FROM created_at)
ORDER BY year, month;
```

## HAVING

`HAVING` filters groups **after** aggregation. `WHERE` filters rows **before** aggregation.

```sql
-- Only categories with more than 5 active products
SELECT
  c.name,
  COUNT(p.id) AS product_count,
  AVG(p.price) AS avg_price
FROM categories c
INNER JOIN products p ON p.category_id = c.id
WHERE p.status = 'active'        -- Filter rows before grouping
GROUP BY c.id, c.name
HAVING COUNT(p.id) > 5           -- Filter groups after aggregation
   AND AVG(p.price) > 20         -- Multiple HAVING conditions
ORDER BY product_count DESC;

-- Customers with high lifetime value
SELECT
  customer_id,
  COUNT(*)     AS order_count,
  SUM(total)   AS lifetime_value
FROM orders
WHERE status = 'completed'
GROUP BY customer_id
HAVING SUM(total) > 1000
   AND COUNT(*) >= 3
ORDER BY lifetime_value DESC;
```

## FILTER Clause (Conditional Aggregation)

The `FILTER` clause is cleaner than `CASE` expressions inside aggregates:

```sql
SELECT
  COUNT(*)                                  AS total_orders,
  COUNT(*) FILTER (WHERE status = 'completed')   AS completed,
  COUNT(*) FILTER (WHERE status = 'cancelled')   AS cancelled,
  COUNT(*) FILTER (WHERE status = 'pending')     AS pending,
  SUM(total) FILTER (WHERE status = 'completed') AS completed_revenue,
  AVG(total) FILTER (WHERE total > 100)          AS avg_of_large_orders
FROM orders;

-- Equivalent using CASE (more verbose)
SELECT
  COUNT(CASE WHEN status = 'completed' THEN 1 END) AS completed
FROM orders;
```

## ROLLUP, CUBE, GROUPING SETS

These generate subtotals and grand totals efficiently.

### ROLLUP

Hierarchical subtotals — generates all combinations of the columns from right to left, plus a grand total:

```sql
SELECT
  COALESCE(region, 'ALL REGIONS')    AS region,
  COALESCE(country, 'ALL COUNTRIES') AS country,
  SUM(revenue)                        AS total_revenue
FROM sales
GROUP BY ROLLUP (region, country)
ORDER BY region NULLS LAST, country NULLS LAST;
-- Produces: region+country, region total, grand total
```

### CUBE

All possible combinations of subtotals:

```sql
SELECT
  COALESCE(year::text, 'ALL')     AS year,
  COALESCE(quarter::text, 'ALL')  AS quarter,
  COALESCE(product, 'ALL')        AS product,
  SUM(sales)                       AS total
FROM revenue_data
GROUP BY CUBE (year, quarter, product);
-- Produces all 8 combinations (including grand total)
```

### GROUPING SETS

Explicitly specify exactly which groupings you want:

```sql
SELECT region, country, product, SUM(revenue)
FROM sales
GROUP BY GROUPING SETS (
  (region, country),    -- By region and country
  (region),             -- By region only
  (product),            -- By product only
  ()                    -- Grand total
);
```

## Window Functions

Window functions compute values across a set of rows related to the current row, **without collapsing rows into a single group**. This is their key advantage over aggregate functions.

### OVER Clause

```sql
-- Running total without collapsing rows
SELECT
  id,
  created_at,
  total,
  SUM(total) OVER (ORDER BY created_at)   AS running_total,
  AVG(total) OVER ()                       AS overall_avg,     -- Entire result set
  COUNT(*) OVER ()                         AS total_orders
FROM orders
WHERE status = 'completed'
ORDER BY created_at;
```

### PARTITION BY

Divide rows into groups (partitions) — like GROUP BY, but rows are not collapsed:

```sql
SELECT
  customer_id,
  id AS order_id,
  total,
  SUM(total)   OVER (PARTITION BY customer_id) AS customer_total,
  COUNT(*)     OVER (PARTITION BY customer_id) AS customer_order_count,
  AVG(total)   OVER (PARTITION BY customer_id) AS customer_avg_order,
  MAX(total)   OVER (PARTITION BY customer_id) AS customer_max_order
FROM orders
WHERE status = 'completed'
ORDER BY customer_id, created_at;
```

### Frame Specification (ROWS/RANGE BETWEEN)

```sql
SELECT
  created_at::date AS day,
  total,
  -- 7-day rolling average (current row + 6 preceding rows)
  AVG(total) OVER (
    ORDER BY created_at
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
  ) AS rolling_7day_avg,

  -- Running total from the beginning
  SUM(total) OVER (
    ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS running_total,

  -- Cumulative from start of partition
  SUM(total) OVER (
    PARTITION BY DATE_TRUNC('month', created_at)
    ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS monthly_running_total
FROM orders;
```

## Ranking Functions

```sql
SELECT
  name,
  department,
  salary,

  -- ROW_NUMBER: unique sequential number, no ties
  ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS row_num,

  -- RANK: tied values get the same rank; next rank skips (1,1,3)
  RANK()       OVER (PARTITION BY department ORDER BY salary DESC) AS rank,

  -- DENSE_RANK: tied values get same rank; no gaps (1,1,2)
  DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank,

  -- NTILE: divide into N buckets (quartiles = 4)
  NTILE(4)     OVER (ORDER BY salary)                              AS quartile
FROM employees
ORDER BY department, salary DESC;
```

### Top N Per Group Pattern

```sql
-- Top 3 most expensive products per category
SELECT *
FROM (
  SELECT
    p.*,
    ROW_NUMBER() OVER (
      PARTITION BY category_id
      ORDER BY price DESC
    ) AS price_rank
  FROM products
  WHERE status = 'active'
) ranked
WHERE price_rank <= 3
ORDER BY category_id, price DESC;
```

## Offset Functions

Access values from other rows in the window:

```sql
SELECT
  created_at::date AS sale_date,
  revenue,

  -- Previous row's value
  LAG(revenue, 1)  OVER (ORDER BY created_at) AS prev_day_revenue,

  -- Value 7 rows ahead
  LEAD(revenue, 7) OVER (ORDER BY created_at) AS next_week_revenue,

  -- First value in the partition
  FIRST_VALUE(revenue) OVER (
    ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
  ) AS first_day_revenue,

  -- Last value in the partition
  LAST_VALUE(revenue) OVER (
    ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
  ) AS latest_revenue,

  -- Day-over-day growth
  (revenue - LAG(revenue, 1) OVER (ORDER BY created_at))
    / NULLIF(LAG(revenue, 1) OVER (ORDER BY created_at), 0) * 100
  AS pct_change
FROM daily_revenue
ORDER BY sale_date;
```

## Named Windows (WINDOW Clause)

Avoid repeating the same window definition:

```sql
SELECT
  employee_id,
  department_id,
  salary,
  RANK()       OVER dept_window AS dept_rank,
  PERCENT_RANK() OVER dept_window AS dept_percentile,
  NTILE(4)     OVER dept_window AS quartile,
  AVG(salary)  OVER dept_window AS dept_avg
FROM employees

WINDOW dept_window AS (
  PARTITION BY department_id
  ORDER BY salary DESC
)

ORDER BY department_id, dept_rank;
```

:::tip[Window vs GROUP BY]
Use window functions when you need both the detail rows AND aggregate values. Use GROUP BY when you only need the summary. Window functions never reduce the row count.
:::

## Complete Analytical Query Example

```sql
WITH monthly_revenue AS (
  SELECT
    DATE_TRUNC('month', created_at)  AS month,
    SUM(total)                        AS revenue,
    COUNT(*)                          AS order_count,
    COUNT(DISTINCT customer_id)       AS unique_customers
  FROM orders
  WHERE status = 'completed'
  GROUP BY DATE_TRUNC('month', created_at)
),
with_metrics AS (
  SELECT
    month,
    revenue,
    order_count,
    unique_customers,
    LAG(revenue)     OVER (ORDER BY month) AS prev_month_revenue,
    AVG(revenue)     OVER (
      ORDER BY month
      ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS rolling_3m_avg,
    SUM(revenue)     OVER (ORDER BY month) AS ytd_revenue,
    RANK()           OVER (ORDER BY revenue DESC) AS revenue_rank
  FROM monthly_revenue
)
SELECT
  TO_CHAR(month, 'YYYY-MM')   AS month,
  revenue,
  ROUND((revenue - prev_month_revenue) / NULLIF(prev_month_revenue, 0) * 100, 1)
                               AS mom_growth_pct,
  ROUND(rolling_3m_avg, 2)    AS rolling_3m_avg,
  ytd_revenue,
  revenue_rank
FROM with_metrics
ORDER BY month;
```

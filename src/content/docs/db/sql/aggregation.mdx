## Aggregate Functions

Aggregate functions compute a single value from a set of rows. They collapse multiple rows into one result per group.

### Core Aggregates

```sql
-- COUNT
SELECT COUNT(*)          FROM orders;                  -- all rows including NULLs
SELECT COUNT(deleted_at) FROM users;                   -- only non-NULL values
SELECT COUNT(DISTINCT customer_id) FROM orders;        -- unique values

-- SUM and AVG
SELECT
  SUM(total)                    AS total_revenue,
  AVG(total)                    AS avg_order_value,
  AVG(total) FILTER (WHERE status = 'delivered') AS avg_delivered
FROM orders;

-- MIN and MAX
SELECT
  MIN(price)  AS lowest_price,
  MAX(price)  AS highest_price,
  MAX(price) - MIN(price) AS price_range
FROM products
WHERE category_id = 3;

-- String aggregation
SELECT
  category_id,
  STRING_AGG(name, ', ' ORDER BY name) AS product_names,  -- PostgreSQL
  GROUP_CONCAT(name ORDER BY name SEPARATOR ', ')          -- MySQL
FROM products
GROUP BY category_id;

-- Array aggregation (PostgreSQL)
SELECT user_id, ARRAY_AGG(tag ORDER BY tag) AS tags
FROM user_tags GROUP BY user_id;

-- JSON aggregation (PostgreSQL)
SELECT
  category_id,
  JSON_AGG(
    JSON_BUILD_OBJECT('id', id, 'name', name, 'price', price)
    ORDER BY price
  ) AS products
FROM products GROUP BY category_id;
```

## GROUP BY

`GROUP BY` partitions rows into groups and applies aggregate functions per group.

```sql
-- Single column grouping
SELECT status, COUNT(*) AS count
FROM orders
GROUP BY status;

-- Multiple column grouping
SELECT
  DATE_TRUNC('month', created_at) AS month,
  status,
  COUNT(*)                         AS count,
  SUM(total)                       AS revenue
FROM orders
GROUP BY DATE_TRUNC('month', created_at), status
ORDER BY month DESC, revenue DESC;

-- Group by expression
SELECT
  EXTRACT(HOUR FROM created_at) AS hour_of_day,
  COUNT(*)                       AS order_count
FROM orders
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY EXTRACT(HOUR FROM created_at)
ORDER BY hour_of_day;

-- Group by with JOIN
SELECT
  c.country,
  COUNT(DISTINCT c.id)  AS customer_count,
  COUNT(o.id)           AS order_count,
  COALESCE(SUM(o.total), 0) AS total_revenue
FROM customers c
LEFT JOIN orders o ON o.customer_id = c.id
GROUP BY c.country
HAVING COUNT(o.id) > 0     -- only countries with orders
ORDER BY total_revenue DESC;
```

:::tip[GROUP BY Rules]
Every column in the SELECT list that is not an aggregate function **must** appear in the GROUP BY clause (or be functionally dependent on a GROUP BY column). PostgreSQL is strict about this; some other databases are not.
:::

## HAVING — Filtering After Aggregation

`WHERE` filters rows before aggregation. `HAVING` filters groups after aggregation.

```sql
-- Products ordered more than 100 times
SELECT
  p.id,
  p.name,
  COUNT(oi.id) AS order_count,
  SUM(oi.quantity) AS total_units
FROM products p
JOIN order_items oi ON oi.product_id = p.id
GROUP BY p.id, p.name
HAVING COUNT(oi.id) > 100
ORDER BY order_count DESC;

-- Customers with more than 5 orders AND lifetime value > $500
SELECT
  customer_id,
  COUNT(*) AS order_count,
  SUM(total) AS lifetime_value
FROM orders
GROUP BY customer_id
HAVING COUNT(*) > 5
  AND SUM(total) > 500
ORDER BY lifetime_value DESC;

-- WHERE + HAVING together
SELECT
  DATE_TRUNC('month', created_at) AS month,
  COUNT(*) AS orders,
  AVG(total) AS avg_total
FROM orders
WHERE status = 'delivered'                           -- filter rows first
GROUP BY DATE_TRUNC('month', created_at)
HAVING AVG(total) > 100                             -- filter groups after
ORDER BY month;
```

## FILTER Clause (Conditional Aggregation)

The FILTER clause applies a condition within an aggregate — cleaner than CASE WHEN inside aggregates.

```sql
SELECT
  DATE_TRUNC('month', created_at)                          AS month,
  COUNT(*)                                                  AS total_orders,
  COUNT(*) FILTER (WHERE status = 'delivered')              AS delivered,
  COUNT(*) FILTER (WHERE status = 'cancelled')              AS cancelled,
  COUNT(*) FILTER (WHERE status = 'refunded')               AS refunded,
  SUM(total) FILTER (WHERE status = 'delivered')            AS delivered_revenue,
  AVG(total) FILTER (WHERE total > 0 AND status = 'delivered') AS avg_delivered_value
FROM orders
WHERE created_at >= NOW() - INTERVAL '6 months'
GROUP BY DATE_TRUNC('month', created_at)
ORDER BY month;
```

## ROLLUP, CUBE, and GROUPING SETS

### ROLLUP — Hierarchical Totals

ROLLUP adds subtotals and grand totals to GROUP BY:

```sql
SELECT
  COALESCE(category, 'ALL CATEGORIES')   AS category,
  COALESCE(region, 'ALL REGIONS')        AS region,
  SUM(revenue)                           AS total_revenue
FROM sales
GROUP BY ROLLUP (category, region)
ORDER BY GROUPING(category), category, GROUPING(region), region;
-- Produces:
-- Electronics | North | 1000
-- Electronics | South | 800
-- Electronics | NULL  | 1800   <- subtotal for Electronics
-- Clothing    | North | 500
-- Clothing    | South | 300
-- Clothing    | NULL  | 800    <- subtotal for Clothing
-- NULL        | NULL  | 2600   <- grand total
```

### CUBE — All Possible Combinations

CUBE generates subtotals for every possible combination of dimensions:

```sql
SELECT
  COALESCE(category, 'ALL')  AS category,
  COALESCE(region, 'ALL')    AS region,
  COALESCE(channel, 'ALL')   AS channel,
  SUM(revenue)                AS revenue
FROM sales
GROUP BY CUBE (category, region, channel)
ORDER BY GROUPING(category), GROUPING(region), GROUPING(channel);
-- Produces subtotals for every combination:
-- category+region+channel, category+region, category+channel,
-- region+channel, category, region, channel, grand total
```

### GROUPING SETS — Custom Combinations

Specify exactly which groupings you want:

```sql
SELECT
  category,
  region,
  month,
  SUM(revenue) AS revenue
FROM sales
GROUP BY GROUPING SETS (
  (category, region, month),   -- detailed
  (category, region),          -- by category and region
  (category, month),           -- by category and month
  (category),                  -- by category only
  ()                           -- grand total
)
ORDER BY GROUPING(category), GROUPING(region), GROUPING(month);
```

## Window Functions

Window functions compute a value for each row based on a "window" of related rows — unlike GROUP BY, they **do not collapse rows**.

```sql
-- Basic syntax:
function_name(args) OVER (
  [PARTITION BY partition_expression]
  [ORDER BY sort_expression [ASC|DESC] [NULLS FIRST|LAST]]
  [ROWS|RANGE BETWEEN frame_start AND frame_end]
)
```

### Ranking Functions

```sql
SELECT
  id,
  customer_id,
  total,
  -- Unique sequential rank (1, 2, 3, 4...)
  ROW_NUMBER() OVER (ORDER BY total DESC) AS row_num,

  -- Rank with gaps for ties (1, 2, 2, 4...)
  RANK() OVER (ORDER BY total DESC) AS rank_with_gaps,

  -- Dense rank — no gaps (1, 2, 2, 3...)
  DENSE_RANK() OVER (ORDER BY total DESC) AS dense_rank,

  -- Divide into N buckets (quartiles here)
  NTILE(4) OVER (ORDER BY total DESC) AS quartile,

  -- Per-customer rank by order total
  RANK() OVER (PARTITION BY customer_id ORDER BY total DESC) AS customer_rank
FROM orders
WHERE status = 'delivered';
```

### Offset Functions

```sql
SELECT
  id,
  created_at,
  total,

  -- Value from the previous row
  LAG(total) OVER (ORDER BY created_at)              AS prev_total,
  LAG(total, 2) OVER (ORDER BY created_at)           AS total_2_ago,

  -- Value from the next row
  LEAD(total) OVER (ORDER BY created_at)             AS next_total,

  -- Value growth from previous order
  total - LAG(total) OVER (ORDER BY created_at)      AS change_from_prev,

  -- Per-customer: lag within the same customer
  LAG(total) OVER (PARTITION BY customer_id ORDER BY created_at) AS prev_customer_order,

  -- First and last values in the window
  FIRST_VALUE(total) OVER (ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS first_ever,
  LAST_VALUE(total) OVER (ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS last_ever,

  -- Nth value (3rd highest order this customer made)
  NTH_VALUE(total, 3) OVER (
    PARTITION BY customer_id
    ORDER BY total DESC
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
  ) AS third_highest
FROM orders;
```

### Running Totals and Moving Averages

```sql
SELECT
  created_at::date                   AS date,
  total,

  -- Running total (cumulative sum)
  SUM(total) OVER (ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS running_total,

  -- Running total, reset per customer
  SUM(total) OVER (
    PARTITION BY customer_id
    ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS customer_running_total,

  -- 7-day moving average
  AVG(total) OVER (ORDER BY created_at
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS moving_avg_7d,

  -- Percentage of grand total
  total / SUM(total) OVER () * 100  AS pct_of_total,

  -- Percentage of customer total
  total / SUM(total) OVER (PARTITION BY customer_id) * 100 AS pct_of_customer_total
FROM orders
WHERE status = 'delivered'
ORDER BY created_at;
```

### Frame Specification

```sql
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW   -- from start to current row
ROWS BETWEEN 3 PRECEDING AND CURRENT ROW           -- last 4 rows (3 + current)
ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING           -- sliding 3-row window
ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING   -- current row to end
RANGE BETWEEN INTERVAL '7 days' PRECEDING AND CURRENT ROW  -- time-based window
```

### Named Windows (WINDOW Clause)

When using the same window definition multiple times:

```sql
SELECT
  id,
  customer_id,
  total,
  created_at,
  SUM(total)   OVER w AS running_total,
  AVG(total)   OVER w AS running_avg,
  COUNT(*)     OVER w AS orders_so_far,
  MAX(total)   OVER w AS max_so_far,
  RANK()       OVER (w ORDER BY total DESC) AS rank
FROM orders
WINDOW w AS (
  PARTITION BY customer_id
  ORDER BY created_at
  ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
)
ORDER BY customer_id, created_at;
```

## Practical Window Function Patterns

### Top-N Per Group

```sql
-- Top 3 products by revenue per category
WITH ranked AS (
  SELECT
    p.category_id,
    p.name,
    SUM(oi.quantity * oi.unit_price) AS revenue,
    ROW_NUMBER() OVER (
      PARTITION BY p.category_id
      ORDER BY SUM(oi.quantity * oi.unit_price) DESC
    ) AS rn
  FROM products p
  JOIN order_items oi ON oi.product_id = p.id
  GROUP BY p.category_id, p.id, p.name
)
SELECT category_id, name, revenue
FROM ranked
WHERE rn <= 3
ORDER BY category_id, revenue DESC;
```

### Year-Over-Year Comparison

```sql
SELECT
  DATE_TRUNC('month', created_at)  AS month,
  SUM(total)                        AS revenue,
  LAG(SUM(total), 12) OVER (ORDER BY DATE_TRUNC('month', created_at))
                                    AS revenue_same_month_last_year,
  SUM(total) / NULLIF(LAG(SUM(total), 12)
    OVER (ORDER BY DATE_TRUNC('month', created_at)), 0) - 1
                                    AS yoy_growth_pct
FROM orders
WHERE status = 'delivered'
GROUP BY DATE_TRUNC('month', created_at)
ORDER BY month;
```

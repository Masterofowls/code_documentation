SQL (Structured Query Language) is the standard language for interacting with relational databases. Declared in 1974 and standardized in 1986, SQL remains one of the most widely used programming languages in the world — present in virtually every data-driven application.

## The Relational Model

SQL is built on the **relational model**, formalized by E.F. Codd in 1970. The core idea: all data is organized into **relations** (tables), and queries produce new relations.

### Key Concepts

| Concept                      | Description                                                 |
| ---------------------------- | ----------------------------------------------------------- |
| **Table (Relation)**         | A two-dimensional structure of named columns and rows       |
| **Row (Tuple/Record)**       | A single data entry in a table                              |
| **Column (Attribute/Field)** | A named, typed property shared by all rows                  |
| **Primary Key**              | A column (or combination) that uniquely identifies each row |
| **Foreign Key**              | A column referencing the primary key of another table       |
| **Schema**                   | A namespace grouping related tables and objects             |
| **Index**                    | A data structure for fast row lookup by column values       |
| **View**                     | A named query that behaves like a virtual table             |
| **Constraint**               | A rule enforced on data (NOT NULL, UNIQUE, CHECK, FK)       |
| **Transaction**              | A sequence of operations treated as a single atomic unit    |

```sql
-- A simple relational model: users and orders
CREATE TABLE users (
  id         SERIAL PRIMARY KEY,
  email      VARCHAR(255) UNIQUE NOT NULL,
  name       TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE orders (
  id         SERIAL PRIMARY KEY,
  user_id    INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  total      NUMERIC(10, 2) NOT NULL,
  status     TEXT NOT NULL DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Relational query: join users with their orders
SELECT u.name, u.email, COUNT(o.id) AS order_count, SUM(o.total) AS total_spent
FROM users u
LEFT JOIN orders o ON o.user_id = u.id
GROUP BY u.id, u.name, u.email
ORDER BY total_spent DESC NULLS LAST;
```

## SQL Standards

SQL has been standardized by ANSI and ISO through multiple revisions:

| Standard     | Year | Key Additions                                  |
| ------------ | ---- | ---------------------------------------------- |
| **SQL-86**   | 1986 | First ANSI standard                            |
| **SQL-89**   | 1989 | Integrity constraints                          |
| **SQL-92**   | 1992 | Major expansion — JOINs, subqueries, CASE      |
| **SQL:1999** | 1999 | Recursive queries (CTEs), triggers, procedural |
| **SQL:2003** | 2003 | Window functions, XML, sequences, MERGE        |
| **SQL:2006** | 2006 | XQuery integration                             |
| **SQL:2008** | 2008 | TRUNCATE, FETCH FIRST                          |
| **SQL:2011** | 2011 | Temporal tables, PERIOD                        |
| **SQL:2016** | 2016 | JSON support, row pattern matching             |
| **SQL:2019** | 2019 | Multi-dimensional arrays, property graphs      |

:::tip[Portability Reminder]
Core SQL (SELECT, INSERT, UPDATE, DELETE, JOINs, aggregates, CTEs) is highly portable. Advanced features (window functions, JSONB, specific data types) vary significantly between vendors.
:::

## SQL Statement Categories

SQL statements are grouped into four categories:

### DDL — Data Definition Language

Defines and modifies database structure. Changes are auto-committed.

```sql
CREATE TABLE products (id SERIAL PRIMARY KEY, name TEXT);
ALTER TABLE products ADD COLUMN price NUMERIC(10,2);
DROP TABLE IF EXISTS products CASCADE;
CREATE INDEX idx_products_name ON products(name);
CREATE VIEW active_products AS SELECT * FROM products WHERE active = true;
TRUNCATE TABLE products;
```

### DML — Data Manipulation Language

Reads and modifies data. These statements participate in transactions.

```sql
SELECT id, name, price FROM products WHERE price > 10;
INSERT INTO products (name, price) VALUES ('Widget', 9.99);
UPDATE products SET price = 12.99 WHERE id = 1;
DELETE FROM products WHERE id = 1;
MERGE INTO products USING ...;
```

### DCL — Data Control Language

Manages access permissions.

```sql
GRANT SELECT, INSERT ON products TO app_user;
REVOKE DELETE ON products FROM app_user;
GRANT ALL PRIVILEGES ON DATABASE mydb TO admin_user;
```

### TCL — Transaction Control Language

Controls transaction boundaries.

```sql
BEGIN;              -- or START TRANSACTION
SAVEPOINT my_save;
ROLLBACK TO SAVEPOINT my_save;
RELEASE SAVEPOINT my_save;
COMMIT;
ROLLBACK;
```

## ACID Properties

ACID is the set of properties that guarantee database transactions are processed reliably:

### Atomicity

A transaction is all-or-nothing. If any part fails, the entire transaction is rolled back.

```sql
BEGIN;

UPDATE accounts SET balance = balance - 500 WHERE id = 1;  -- debit
UPDATE accounts SET balance = balance + 500 WHERE id = 2;  -- credit

-- If the second UPDATE fails, the first is automatically rolled back
COMMIT;
```

### Consistency

A transaction brings the database from one valid state to another. Constraints, triggers, and rules are enforced.

```sql
-- This will fail atomically — CHECK constraint prevents negative balance
UPDATE accounts SET balance = balance - 10000
WHERE id = 1;  -- fails if balance < 10000 due to CHECK (balance >= 0)
```

### Isolation

Concurrent transactions execute as if they were sequential. Isolation levels control the trade-off between correctness and performance.

```sql
-- Isolation levels (from weakest to strongest):
SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;   -- dirty reads possible
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;     -- PostgreSQL default
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;       -- strongest
```

### Durability

Once committed, data survives system failures. Achieved through write-ahead logging (WAL) and persistent storage.

:::caution[SQLite and ACID]
SQLite is ACID-compliant in WAL mode. In the default rollback journal mode, it still provides ACID guarantees but with lower concurrency. Always enable `PRAGMA journal_mode=WAL` for application use.
:::

## What is SQLite?

SQLite is a C library that implements a self-contained, serverless, zero-configuration, transactional SQL database engine. Unlike PostgreSQL or MySQL, SQLite is not a client-server database — the database engine runs **in the same process** as the application, reading and writing directly to a single file on disk.

SQLite is the most widely deployed database in the world. It ships with:

- Every smartphone (Android and iOS)
- Every web browser (Chrome, Firefox, Safari)
- Python's standard library (`sqlite3` module)
- Every macOS and Windows installation
- Countless embedded systems, IoT devices, and desktop applications

### Key Characteristics

- **Serverless**: No separate server process, no network communication
- **Zero-configuration**: No installation, no setup, no admin required
- **Single-file**: The entire database lives in one `.db` file
- **Self-contained**: No external dependencies beyond the C standard library
- **Cross-platform**: The `.db` file format is identical on all platforms
- **ACID-compliant**: Full transaction support with durability
- **Public domain**: No license required — freely usable in any project

## SQLite vs Server Databases

| Feature               | SQLite                    | PostgreSQL              | MySQL                  |
| --------------------- | ------------------------- | ----------------------- | ---------------------- |
| **Architecture**      | Embedded library          | Client-server           | Client-server          |
| **Setup**             | Zero config               | Install + configure     | Install + configure    |
| **Concurrent writes** | Single writer at a time   | Many concurrent writes  | Many concurrent writes |
| **Concurrent reads**  | Unlimited (WAL mode)      | Unlimited               | Unlimited              |
| **Network access**    | No (single machine)       | Yes                     | Yes                    |
| **Data size**         | Up to 281 TB              | Unlimited               | ~64 TB (InnoDB)        |
| **Row size**          | 1 GB                      | 1.6 TB                  | 65 KB                  |
| **User management**   | OS-level file permissions | Full user/role system   | Full user/role system  |
| **Data types**        | Dynamic typing            | Strict static typing    | Strict static typing   |
| **Full-text search**  | FTS5                      | tsvector/tsquery        | FULLTEXT               |
| **JSON support**      | JSON functions            | JSONB (binary, indexed) | JSON                   |
| **Stored procedures** | No                        | PL/pgSQL + more         | Yes                    |
| **Replication**       | Via Litestream/Turso      | Streaming/logical       | Group/binlog           |
| **Deployment**        | Single binary             | Server infrastructure   | Server infrastructure  |

## When to Use SQLite

:::tip[SQLite is the right choice for:]

- Mobile apps (iOS, Android) — ships with the OS, no backend needed
- Desktop applications — single-file distribution, easy backup (just copy the file)
- Development and testing — no Docker, no credentials, fast spin-up with `:memory:`
- Edge/serverless — runs in Lambda, workers, CLI tools, scripts
- Small-to-medium websites — Django and Flask default to SQLite for good reason
- Embedded devices — IoT, routers, set-top boxes, game consoles
- Application file format — use SQLite as a structured file format instead of CSV/JSON
  :::

:::caution[SQLite is NOT the right choice for:]

- High write concurrency — multiple processes writing to the same database will contend
- Networked databases — SQLite requires the application to be on the same machine as the file
- Data larger than available RAM + disk of one machine — scale-out requires server databases
- Multi-user web applications with heavy write loads — switch to PostgreSQL at this point
  :::

## The SQLite File Format

```bash
# Default extensions (all equivalent)
myapp.db
myapp.sqlite
myapp.sqlite3

# File signature (first 16 bytes)
# "SQLite format 3\000"
hexdump -C myapp.db | head -1
# 00000000  53 51 4c 69 74 65 20 66  6f 72 6d 61 74 20 33 00

# Database info
sqlite3 myapp.db "PRAGMA database_list;"
sqlite3 myapp.db "PRAGMA integrity_check;"
sqlite3 myapp.db "PRAGMA page_count;"
```

The SQLite database file format is extremely stable — files created in 2004 can still be read by the current SQLite library. The format is [officially documented](https://www.sqlite.org/fileformat.html) and is designed to be an archival format.

## SQLite Version

SQLite 3.x has been the stable series since 2004. Key recent milestones:

| Version         | Year | Key Features                                               |
| --------------- | ---- | ---------------------------------------------------------- |
| 3.35 (2021)     | 2021 | `RETURNING` clause, `DROP COLUMN`, math functions          |
| 3.37 (2021)     | 2021 | `STRICT` tables (enforce column types)                     |
| 3.38 (2022)     | 2022 | JSON functions built-in (no longer needs compilation flag) |
| 3.40 (2022)     | 2022 | Improved performance, recovery tools                       |
| 3.42 (2023)     | 2023 | `SQLITE_OPEN_EXRESCODE` error codes                        |
| 3.45 (2024)     | 2024 | JSONB binary format, improved performance                  |
| **3.47** (2024) | 2024 | Current stable — improved performance                      |

Check your SQLite version:

```bash
sqlite3 --version
# 3.47.0 2024-10-21 16:30:22

python3 -c "import sqlite3; print(sqlite3.sqlite_version)"
node -e "const db = require('better-sqlite3')(':memory:'); console.log(db.pragma('user_version', {simple: true}));"
```

## SQLite in Different Languages

### Python

```python
import sqlite3
conn = sqlite3.connect('myapp.db')
```

### Node.js

```javascript
const Database = require('better-sqlite3');
const db = new Database('myapp.db');
```

### Rust

```rust
let conn = rusqlite::Connection::open("myapp.db")?;
```

### Go

```go
db, err := sql.Open("sqlite3", "myapp.db")
```

### Bun (JavaScript runtime)

```javascript
import { Database } from 'bun:sqlite';
const db = new Database('myapp.db');
```

## WAL Mode — The Essential Setting

By default, SQLite uses a rollback journal. **WAL (Write-Ahead Logging) mode dramatically improves performance** for most applications:

```sql
PRAGMA journal_mode=WAL;
-- Returns: wal
```

Benefits of WAL mode:

- **Concurrent reads**: Readers don't block writers; writers don't block readers
- **Faster writes**: Sequential WAL writes vs random journal writes
- **Better crash recovery**: No database corruption if the process crashes mid-write
- **Smaller transaction overhead**: WAL is append-only

```bash
# WAL mode creates two extra files (they're normal, don't delete them)
myapp.db
myapp.db-wal      # Write-ahead log
myapp.db-shm      # Shared memory file
```

# Deploying Astro.js

Astro's flexible output modes (`static`, `server`, `hybrid`) let you deploy to any host — from simple CDNs
serving HTML files to fully dynamic edge/serverless environments.

## Output Modes

```typescript
// astro.config.mjs
export default defineConfig({
  output: 'static',  // Default: pre-render everything at build time
  // output: 'server',  // SSR: render on each request (requires adapter)
  // output: 'hybrid',  // Mix: static by default, opt-in to SSR per page
});
```

| Mode | Description | Requires Adapter |
|---|---|---|
| `static` | All pages pre-rendered at build time | No |
| `server` | All pages rendered on each request | Yes |
| `hybrid` | Static by default, individual pages can opt into SSR | Yes |

## Static Deployment (output: 'static')

Static output generates a `dist/` folder of HTML, CSS, and JS files that can be served by any web host.

### Vercel (Static)

<Steps>

1. **Connect your repository** on [vercel.com](https://vercel.com) via the dashboard.

2. **Configure the build**

   Vercel auto-detects Astro. No configuration needed. Optionally add `vercel.json`:

   ```json
   {
     "buildCommand": "npm run build",
     "outputDirectory": "dist",
     "framework": "astro"
   }
   ```

3. **Deploy**

   Every push to `main` triggers an automatic deployment.

</Steps>

### Netlify (Static)

```bash
# Install Netlify CLI
npm install -g netlify-cli

# Deploy
npm run build
netlify deploy --dir=dist --prod
```

Or add a `netlify.toml`:

```toml
[build]
  command = "npm run build"
  publish = "dist"

[build.environment]
  NODE_VERSION = "20"

[[headers]]
  for = "/_astro/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[redirects]]
  from = "/old-path"
  to = "/new-path"
  status = 301
```

### Cloudflare Pages (Static)

<Steps>

1. **Connect your repository** in the Cloudflare dashboard → Pages → Create a project.

2. **Configure build settings**

   - Build command: `npm run build`
   - Build output directory: `dist`
   - Node.js version: 20

3. **Environment variables** — add in the Cloudflare dashboard under Settings → Environment Variables.

</Steps>

### GitHub Pages

```typescript
// astro.config.mjs
export default defineConfig({
  site: 'https://username.github.io',
  base: '/repo-name',  // Required if deploying to /repo-name, not root
});
```

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Build Astro site
        run: npm run build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## SSR Deployment (output: 'server')

SSR requires an adapter. Each adapter targets a specific runtime environment.

### Vercel (SSR / Edge Functions)

```bash
npx astro add vercel
```

```typescript
// astro.config.mjs
import vercel from '@astrojs/vercel/serverless';

export default defineConfig({
  output: 'server',
  adapter: vercel({
    webAnalytics: { enabled: true },
    imageService: true,
    edgeMiddleware: true,
    functionPerRoute: false,
    maxDuration: 30,
  }),
});
```

Push to your connected repository — Vercel handles the rest.

### Netlify (SSR / Edge Functions)

```bash
npx astro add netlify
```

```typescript
import netlify from '@astrojs/netlify';

export default defineConfig({
  output: 'server',
  adapter: netlify({
    edgeMiddleware: true,
  }),
});
```

### Node.js Standalone Server

```bash
npx astro add node
```

```typescript
import node from '@astrojs/node';

export default defineConfig({
  output: 'server',
  adapter: node({ mode: 'standalone' }),
});
```

```bash
npm run build
node dist/server/entry.mjs
```

Environment variables for the Node.js server:

```bash
HOST=0.0.0.0
PORT=4321
node dist/server/entry.mjs
```

## Docker Deployment

```dockerfile
# Dockerfile
FROM node:20-alpine AS base
WORKDIR /app

# Install dependencies
FROM base AS deps
COPY package.json package-lock.json ./
RUN npm ci

# Build the application
FROM base AS builder
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Production image
FROM node:20-alpine AS runner
WORKDIR /app

ENV HOST=0.0.0.0
ENV PORT=4321
ENV NODE_ENV=production

# Copy built output
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./package.json

EXPOSE 4321
CMD ["node", "dist/server/entry.mjs"]
```

```yaml
# docker-compose.yml
services:
  astro:
    build: .
    ports:
      - "4321:4321"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=${DATABASE_URL}
      - PUBLIC_SITE_URL=https://example.com
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:4321/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

:::tip[Static Docker Alternative]
For static sites, serve the `dist/` folder with nginx instead of Node.js — it's faster and uses fewer resources:

```dockerfile
FROM nginx:alpine
COPY dist/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```
:::

## GitHub Actions — Full CI/CD Pipeline

```yaml
# .github/workflows/ci-deploy.yml
name: CI & Deploy

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci

      - name: Type check
        run: npx astro check

  build-and-deploy:
    needs: check
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci

      - name: Build
        run: npm run build
        env:
          PUBLIC_SITE_URL: ${{ secrets.PUBLIC_SITE_URL }}
          DATABASE_URL: ${{ secrets.DATABASE_URL }}

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

## Environment Variables in Deployment

```bash
# .env.example — commit this, not .env
PUBLIC_SITE_URL=https://example.com
PUBLIC_GA_ID=G-XXXXXXXXXX
DATABASE_URL=postgresql://user:pass@host:5432/db
SECRET_API_KEY=your-secret-key
```

:::caution[Never Commit Secrets]
Never commit `.env` files with real secrets. Use your deployment platform's environment variable UI
(Vercel Dashboard, Netlify Environment Variables, GitHub Actions Secrets) to inject secrets at build time.
:::

## Build Configuration Options

```typescript
// astro.config.mjs
export default defineConfig({
  site: 'https://example.com',

  // Serve under a sub-path (e.g., GitHub Pages at /my-repo/)
  base: '/',

  // Trailing slash behavior
  trailingSlash: 'never',   // 'always' | 'never' | 'ignore'

  // Build output directory (default: './dist')
  outDir: './dist',

  // Public directory (default: './public')
  publicDir: './public',

  // Prefix for generated asset URLs (CDN support)
  build: {
    assets: '_astro',
    assetsPrefix: 'https://cdn.example.com',  // All /_astro/ URLs point to CDN
    inlineStylesheets: 'auto',  // Inline small stylesheets
  },
});
```

## Custom 404 in Deployment Platforms

Most static hosts need to know your custom 404 page location:

<Tabs>
  <TabItem label="Vercel">
    ```json
    // vercel.json
    {
      "routes": [
        { "handle": "filesystem" },
        { "src": "/(.*)", "dest": "/404" }
      ]
    }
    ```
</TabItem>
  <TabItem label="Netlify">
    ```toml
    # netlify.toml
    [[redirects]]
      from = "/*"
      to = "/404"
      status = 404
    ```
</TabItem>
  <TabItem label="Cloudflare Pages">
    Cloudflare Pages automatically serves `/404.html` for unmatched routes — no config needed.
</TabItem>
</Tabs>

## Cache Headers for Static Assets

Astro generates hashed filenames for all assets (e.g., `_astro/main.a1b2c3d4.css`), enabling aggressive
immutable caching:

```toml
# netlify.toml
[[headers]]
  for = "/_astro/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[headers]]
  for = "*.html"
  [headers.values]
    Cache-Control = "public, max-age=0, must-revalidate"
```

```json
// vercel.json
{
  "headers": [
    {
      "source": "/_astro/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    },
    {
      "source": "/(.*).html",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=0, must-revalidate" }
      ]
    }
  ]
}
```

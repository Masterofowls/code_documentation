# Astro.js

Astro is a modern web framework designed for building content-focused websites that are fast by default.
It achieves exceptional performance through its **islands architecture** — a paradigm that renders most
of your UI as static HTML on the server and only hydrates interactive components in the browser.

## What is Astro?

Astro was built on the belief that performance should be the default, not an optimization. Its core philosophy:

- **Server-first** — Render UI on the server, ship less JavaScript
- **Zero JS by default** — No JavaScript is sent to the browser unless you explicitly opt in
- **Content-focused** — Purpose-built for blogs, docs, portfolios, e-commerce, and marketing sites
- **Framework agnostic** — Bring your own UI framework (React, Vue, Svelte, Solid, Preact, Lit)

## Islands Architecture

The **Astro Islands** pattern (also called Component Islands) describes a technique for partial hydration
on otherwise static HTML pages:

```
┌─────────────────────────────────────────────────────┐
│                    Page (Static HTML)                │
│  ┌────────────────┐  ┌──────────────────────────┐   │
│  │  Static Header │  │   Static Article Text     │   │
│  │  (no JS)       │  │   (no JS)                 │   │
│  └────────────────┘  └──────────────────────────┘   │
│                                                      │
│  ┌──────────────────────────────────────────────┐   │
│  │   🏝️  Interactive Search Widget (client:load) │   │
│  │   (hydrated with React, ~15kb)               │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  ┌───────────────┐  ┌──────────────────────────┐    │
│  │  Static Footer │  │  🏝️ Cart Counter          │    │
│  │  (no JS)       │  │  (client:visible, ~3kb)  │    │
│  └───────────────┘  └──────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

Only the "islands" — individual interactive components — are hydrated in the browser. Everything else is
pure HTML with zero JavaScript overhead.

## Zero JavaScript by Default

```astro
---
// This runs only on the server at build time (or on each request in SSR mode)
const posts = await fetch('https://api.example.com/posts').then((r) =>
  r.json(),
);
---

<!-- Pure HTML output — zero JavaScript sent to the browser -->
<ul>
  {
    posts.map((post) => (
      <li>
        <a href={`/blog/${post.slug}`}>{post.title}</a>
      </li>
    ))
  }
</ul>
```

## Server-First Rendering

Astro renders pages on the server by default. This means:

- No client-side data fetching waterfalls
- No loading spinners for initial content
- Full HTML available for search engine crawlers
- Smaller JavaScript bundles since business logic stays on the server

## Astro vs Other Frameworks

| Feature                 | Astro               | Next.js         | Remix           | Gatsby        |
| ----------------------- | ------------------- | --------------- | --------------- | ------------- |
| **Primary Use Case**    | Content sites       | Full-stack apps | Full-stack apps | Static sites  |
| **Default JS**          | Zero                | React bundle    | React bundle    | React bundle  |
| **Rendering**           | Static / SSR        | SSG / SSR / ISR | SSR (loader)    | SSG           |
| **Islands Arch.**       | ✅ Native           | ❌ Manual       | ❌ Manual       | ❌ Manual     |
| **Framework Agnostic**  | ✅ React/Vue/Svelte | ❌ React only   | ❌ React only   | ❌ React only |
| **Build Speed**         | Very Fast           | Fast            | Fast            | Slow          |
| **View Transitions**    | ✅ Native API       | 🔧 Manual       | 🔧 Manual       | ❌            |
| **Content Collections** | ✅ Built-in         | 🔧 External     | 🔧 External     | 🔧 Plugins    |
| **Bundle Size**         | Minimal             | Large           | Medium          | Large         |
| **Learning Curve**      | Low                 | Medium          | Medium          | Medium        |
| **TypeScript**          | ✅ First-class      | ✅ First-class  | ✅ First-class  | ✅ Good       |
| **SSR Adapters**        | ✅ Multiple         | ✅ Built-in     | ✅ Built-in     | ❌ Limited    |

:::tip[When to Choose Astro]
Astro shines for content-heavy sites: blogs, documentation (like this site!), marketing pages, portfolios,
and e-commerce storefronts. For highly interactive applications like dashboards or real-time collaboration
tools, consider Next.js or Remix instead.
:::

## UI Framework Agnostic

Astro's most unique feature is the ability to mix UI frameworks in the same project:

```astro
---
import ReactSearch from './components/Search.tsx'; // React
import VueCounter from './components/Counter.vue'; // Vue
import SvelteForm from './components/Form.svelte'; // Svelte
import SolidChart from './components/Chart.tsx'; // Solid
---

<html>
  <body>
    <!-- Each component is its own hydration island -->
    <ReactSearch client:load />
    <VueCounter client:visible />
    <SvelteForm client:idle />
    <SolidChart client:visible />
  </body>
</html>
```

This means you can:

- Use the best tool for each component's specific needs
- Gradually migrate from one framework to another
- Use battle-tested React components in a mostly static Astro site

## View Transitions API

Astro provides first-class support for the browser's native View Transitions API:

```astro
---
import { ViewTransitions } from 'astro:transitions';
---

<html>
  <head>
    <ViewTransitions />
  </head>
  <body>
    <!-- All page navigations now animate with View Transitions -->
    <slot />
  </body>
</html>
```

```astro
<!-- Customize per-element transitions -->
<h1 transition:name="hero-title" transition:animate="slide">
  My Blog Post Title
</h1>

<img
  src={post.image}
  transition:name={`hero-image-${post.slug}`}
  transition:animate="morph"
/>
```

## Starlight — Documentation Framework

[Starlight](https://starlight.astro.build) is Astro's official documentation framework, which this site
is built with. It provides:

- Sidebar navigation with automatic TOC
- Full-text search (Pagefind)
- Multi-language / i18n support
- Light/dark mode
- Accessible, keyboard-navigable UI
- Built on Astro — zero JS by default

```bash
npm create astro@latest -- --template starlight
```

## Quick Start

```bash
# Create a new Astro project
npm create astro@latest

# Add a UI framework
npx astro add react

# Add Tailwind CSS
npx astro add tailwind

# Start dev server
npm run dev
```

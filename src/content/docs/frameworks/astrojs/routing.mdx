# Routing in Astro.js

Astro uses **file-based routing** — the file structure of your `src/pages/` directory directly determines
the URL structure of your site. No configuration required.

## File-Based Routing

Each file in `src/pages/` becomes a page on your site:

```
src/pages/
├── index.astro          → /
├── about.astro          → /about
├── contact.astro        → /contact
├── blog/
│   ├── index.astro      → /blog
│   └── first-post.astro → /blog/first-post
└── shop/
    ├── index.astro      → /shop
    └── cart.astro       → /shop/cart
```

## Static Routes

The simplest kind of route: a static `.astro` file:

```astro
---
// src/pages/about.astro
import Layout from '../layouts/BaseLayout.astro';

const team = [
  { name: 'Alice', role: 'Founder' },
  { name: 'Bob', role: 'CTO' },
];
---

<Layout title="About Us">
  <h1>About Us</h1>
  <ul>
    {team.map(member => (
      <li>{member.name} — {member.role}</li>
    ))}
  </ul>
</Layout>
```

## Dynamic Routes

Use square bracket notation to create dynamic routes. You must export a `getStaticPaths()` function to
tell Astro which values to generate:

```astro
---
// src/pages/blog/[slug].astro
import Layout from '../../layouts/BlogLayout.astro';
import { getCollection } from 'astro:content';

export async function getStaticPaths() {
  const posts = await getCollection('blog');

  return posts.map(post => ({
    params: { slug: post.slug },
    props: { post },          // Pass data to the page component
  }));
}

// TypeScript types for this page
interface Props {
  post: Awaited<ReturnType<typeof getCollection<'blog'>>>[number];
}

const { post } = Astro.props;
const { Content } = await post.render();
---

<Layout title={post.data.title}>
  <article>
    <h1>{post.data.title}</h1>
    <Content />
  </article>
</Layout>
```

### Dynamic Routes with Numeric IDs

```astro
---
// src/pages/products/[id].astro
export async function getStaticPaths() {
  const products = await fetch('https://api.example.com/products').then(r => r.json());

  return products.map((product: { id: number; name: string; price: number }) => ({
    params: { id: String(product.id) },
    props: { product },
  }));
}

const { product } = Astro.props;
---

<h1>{product.name}</h1>
<p>${product.price}</p>
```

:::tip[params Must Be Strings]
All route parameters are always strings. Convert to numbers or other types in your component using
`parseInt()`, `parseFloat()`, etc.
:::

## Rest Parameters — [...slug].astro

Use rest parameters to match any number of path segments:

```astro
---
// src/pages/docs/[...slug].astro
// Matches: /docs, /docs/intro, /docs/api/reference, etc.

export async function getStaticPaths() {
  return [
    { params: { slug: undefined } },          // Matches /docs
    { params: { slug: 'intro' } },             // Matches /docs/intro
    { params: { slug: 'api/reference' } },     // Matches /docs/api/reference
    { params: { slug: 'api/endpoints/users' } }, // Deeper nesting
  ];
}

const { slug } = Astro.params;
const segments = slug ? slug.split('/') : [];
---

<nav>
  {segments.map((seg, i) => (
    <span>
      {i > 0 && ' / '}
      <a href={'/' + segments.slice(0, i + 1).join('/')}>{seg}</a>
    </span>
  ))}
</nav>
```

## Page Component Properties

Inside any `.astro` page, you have access to the `Astro` global:

```astro
---
// Astro.params — dynamic route parameters
const { slug, id } = Astro.params;

// Astro.props — data passed from getStaticPaths
const { post, category } = Astro.props;

// Astro.url — current request URL (URL object)
const currentPath = Astro.url.pathname;
const searchQuery = Astro.url.searchParams.get('q');

// Astro.site — base URL from astro.config.mjs
const siteUrl = Astro.site?.href;

// Astro.request — full Request object (SSR mode)
const method = Astro.request.method;

// Astro.locals — data from middleware
const user = Astro.locals.user;

// Astro.cookies — get/set cookies (SSR mode)
const sessionId = Astro.cookies.get('session_id')?.value;
---
```

## Redirects

```astro
---
// src/pages/old-path.astro
// Redirect old URL to new one
return Astro.redirect('/new-path', 301);
---
```

Conditional redirect based on auth state:

```astro
---
// src/pages/dashboard.astro
const user = Astro.locals.user;

if (!user) {
  return Astro.redirect('/login?returnTo=/dashboard', 302);
}
---

<h1>Welcome, {user.name}!</h1>
```

## Custom 404 Page

Create `src/pages/404.astro` to customize your not-found page:

```astro
---
// src/pages/404.astro
import Layout from '../layouts/BaseLayout.astro';
---

<Layout title="404 — Page Not Found">
  <div class="error-page">
    <h1>404</h1>
    <p>Oops! The page you're looking for doesn't exist.</p>
    <a href="/">← Go Home</a>
  </div>
</Layout>

<style>
  .error-page {
    text-align: center;
    padding: 4rem 1rem;
  }
  h1 { font-size: 6rem; margin: 0; color: #94a3b8; }
</style>
```

## Endpoint Files — API Routes

Files in `src/pages/` that export `GET`, `POST`, etc. functions become API endpoints:

<Tabs>
  <TabItem label="GET Endpoint">
    ```typescript
    // src/pages/api/users.ts
    import type { APIRoute } from 'astro';

    export const GET: APIRoute = async ({ url }) => {
      const page = Number(url.searchParams.get('page') ?? '1');
      const limit = Number(url.searchParams.get('limit') ?? '20');

      const users = await db.user.findMany({
        skip: (page - 1) * limit,
        take: limit,
      });

      return new Response(JSON.stringify({ data: users, page, limit }), {
        status: 200,
        headers: { 'Content-Type': 'application/json' },
      });
    };
    ```
</TabItem>
  <TabItem label="POST Endpoint">
    ```typescript
    // src/pages/api/users.ts
    import type { APIRoute } from 'astro';

    export const POST: APIRoute = async ({ request }) => {
      const body = await request.json();

      if (!body.email || !body.name) {
        return new Response(
          JSON.stringify({ error: 'email and name are required' }),
          { status: 400, headers: { 'Content-Type': 'application/json' } }
        );
      }

      const user = await db.user.create({ data: body });

      return new Response(JSON.stringify(user), {
        status: 201,
        headers: { 'Content-Type': 'application/json' },
      });
    };
    ```
</TabItem>
  <TabItem label="Dynamic API Route">
    ```typescript
    // src/pages/api/users/[id].ts
    import type { APIRoute } from 'astro';

    export const GET: APIRoute = async ({ params }) => {
      const id = Number(params.id);
      const user = await db.user.findUnique({ where: { id } });

      if (!user) {
        return new Response(JSON.stringify({ error: 'Not found' }), {
          status: 404,
          headers: { 'Content-Type': 'application/json' },
        });
      }

      return Response.json(user);
    };

    export const DELETE: APIRoute = async ({ params }) => {
      await db.user.delete({ where: { id: Number(params.id) } });
      return new Response(null, { status: 204 });
    };
    ```
</TabItem>

## Middleware

Astro middleware runs on every request before page or endpoint handlers. Create it at `src/middleware.ts`:

```typescript
// src/middleware.ts
import { defineMiddleware, sequence } from 'astro:middleware';

// Auth middleware
const auth = defineMiddleware(async ({ locals, cookies, redirect, url }, next) => {
  const sessionId = cookies.get('session_id')?.value;

  if (sessionId) {
    const session = await getSession(sessionId);
    if (session) {
      locals.user = session.user;
    }
  }

  // Protect /dashboard routes
  if (url.pathname.startsWith('/dashboard') && !locals.user) {
    return redirect(`/login?returnTo=${url.pathname}`);
  }

  return next();
});

// Logging middleware
const logger = defineMiddleware(async ({ request }, next) => {
  const start = Date.now();
  const response = await next();
  console.log(`${request.method} ${request.url} — ${Date.now() - start}ms`);
  return response;
});

// Run middleware in sequence
export const onRequest = sequence(logger, auth);
```

### Extending Locals Type

```typescript
// src/env.d.ts
declare namespace App {
  interface Locals {
    user?: {
      id: number;
      email: string;
      role: string;
    };
    sessionId?: string;
  }
}
```

## Routing Priority

When multiple routes could match the same URL, Astro uses this priority order:

1. Static routes — `/about.astro` matches `/about` exactly
2. Dynamic routes with params — `/blog/[slug].astro`
3. Rest routes — `/docs/[...slug].astro`
4. `404.astro` — fallback for unmatched routes

:::caution[Route Conflicts]
`/blog/[slug].astro` and `/blog/my-post.astro` would conflict. Static routes always win over dynamic ones.
Be careful not to create ambiguous route structures.
:::

## Pagination

Astro provides a built-in `paginate()` helper for `getStaticPaths()`:

```astro
---
// src/pages/blog/[...page].astro
import type { GetStaticPaths, Page } from 'astro';
import type { CollectionEntry } from 'astro:content';
import { getCollection } from 'astro:content';
import Layout from '../../layouts/BaseLayout.astro';

export const getStaticPaths: GetStaticPaths = async ({ paginate }) => {
  const posts = await getCollection('blog', ({ data }) => !data.draft);
  const sorted = posts.sort(
    (a, b) => b.data.publishedAt.getTime() - a.data.publishedAt.getTime()
  );

  return paginate(sorted, { pageSize: 10 });
};

interface Props {
  page: Page<CollectionEntry<'blog'>>;
}

const { page } = Astro.props;
---

<Layout title={`Blog — Page ${page.currentPage}`}>
  <h1>Blog</h1>

  <ul>
    {page.data.map(post => (
      <li>
        <a href={`/blog/${post.slug}`}>{post.data.title}</a>
      </li>
    ))}
  </ul>

  <nav class="pagination">
    {page.url.prev && <a href={page.url.prev}>← Previous</a>}
    <span>Page {page.currentPage} of {page.lastPage}</span>
    {page.url.next && <a href={page.url.next}>Next →</a>}
  </nav>
</Layout>
```

The `page` object contains:
- `page.data` — Array of items for the current page
- `page.currentPage` — Current page number (1-indexed)
- `page.lastPage` — Total number of pages
- `page.url.prev` — URL of the previous page or `undefined`
- `page.url.next` — URL of the next page or `undefined`
- `page.total` — Total number of items across all pages
- `page.start` / `page.end` — Item index range for this page

# NestJS

NestJS is a progressive Node.js framework for building efficient, reliable, and scalable server-side applications.
Built with TypeScript and deeply inspired by Angular's architecture, it provides a production-ready foundation for
enterprise-grade APIs, microservices, and real-time applications.

## What is NestJS?

NestJS combines Object-Oriented Programming (OOP), Functional Programming (FP), and Functional Reactive Programming (FRP)
into a cohesive, opinionated structure. It provides an out-of-the-box application architecture that allows developers
and teams to create highly testable, scalable, loosely coupled, and easily maintainable applications.

Under the hood, NestJS makes use of [Express](https://expressjs.com/) by default but can optionally be configured to
use [Fastify](https://fastify.dev/) instead, enabling significant throughput improvements for high-load scenarios.

:::tip[TypeScript First]
NestJS is written entirely in TypeScript and every built-in decorator, interface, and utility is fully typed.
All examples in this documentation use TypeScript. JavaScript is supported but not recommended.
:::

## Angular-Inspired Architecture

NestJS draws heavy inspiration from Angular's design philosophy. If you're familiar with Angular, many concepts will
feel immediately familiar:

- **Decorators** — Define controllers, providers, modules, guards, and more using class decorators
- **Dependency Injection** — A first-class DI container is built directly into the framework
- **Modules** — Organize your application into cohesive, encapsulated blocks of functionality
- **Pipes, Guards, Interceptors, Filters** — Middleware-like constructs for the full request lifecycle

## Decorators-First Approach

NestJS leverages TypeScript decorators extensively. This metadata-driven approach keeps your code declarative,
expressive, and easy to reason about:

```typescript
import { Controller, Get, Post, Body, Param, Delete, Patch } from '@nestjs/common';
import { CreateUserDto } from './dto/create-user.dto';
import { UpdateUserDto } from './dto/update-user.dto';
import { UsersService } from './users.service';

@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

  @Get(':id')
  findOne(@Param('id') id: string) {
    return this.usersService.findOne(+id);
  }

  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return this.usersService.create(createUserDto);
  }

  @Patch(':id')
  update(@Param('id') id: string, @Body() updateUserDto: UpdateUserDto) {
    return this.usersService.update(+id, updateUserDto);
  }

  @Delete(':id')
  remove(@Param('id') id: string) {
    return this.usersService.remove(+id);
  }
}
```

## NestJS vs Other Node.js Frameworks

| Feature | NestJS | Express | Fastify | Hono |
|---|---|---|---|---|
| **Language** | TypeScript (first-class) | JavaScript / TS | JavaScript / TS | TypeScript |
| **Architecture** | Opinionated (Angular-like) | Unopinionated | Unopinionated | Unopinionated |
| **Performance** | High (Fastify adapter) | Medium | Very High | Extremely High |
| **DI Container** | ✅ Built-in | ❌ None | ❌ None | ❌ None |
| **Module System** | ✅ Built-in | ❌ Manual | ❌ Manual | ❌ Manual |
| **Decorators** | ✅ Extensive | ❌ None | ❌ None | ✅ Limited |
| **CLI Tooling** | ✅ Powerful (`nest` CLI) | ❌ None | ❌ None | ❌ None |
| **WebSockets** | ✅ Built-in | 🔧 Plugin | 🔧 Plugin | 🔧 Plugin |
| **Microservices** | ✅ Built-in | ❌ None | ❌ None | ❌ None |
| **GraphQL** | ✅ Built-in | 🔧 Plugin | 🔧 Plugin | 🔧 Plugin |
| **OpenAPI / Swagger** | ✅ `@nestjs/swagger` | 🔧 Plugin | 🔧 Plugin | 🔧 Plugin |
| **Testing Utilities** | ✅ Built-in (`Test.createTestingModule`) | 🔧 Manual | 🔧 Manual | 🔧 Manual |
| **Learning Curve** | Steep | Low | Low | Very Low |
| **Enterprise Adoption** | Excellent | Good | Good | Limited |

:::caution[Learning Curve]
NestJS has a steeper learning curve than Express or Fastify due to its opinionated architecture and reliance on
decorators and DI patterns. The investment pays off considerably for mid-to-large applications that need
maintainability, testability, and team scalability.
:::

## Built-in Features

### Dependency Injection

NestJS ships a powerful IoC (Inversion of Control) container that manages your application's dependency graph
automatically. You declare what you need; NestJS wires it together:

```typescript
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { ConfigService } from '@nestjs/config';
import { User } from './entities/user.entity';

@Injectable()
export class UsersService {
  constructor(
    @InjectRepository(User)
    private readonly usersRepository: Repository<User>,
    private readonly configService: ConfigService,
  ) {}

  findAll(): Promise<User[]> {
    return this.usersRepository.find();
  }
}
```

### Guards

Guards implement `CanActivate` and determine whether a given request should be processed by the route handler.
They are ideal for authentication and authorization logic:

```typescript
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';
import { Observable } from 'rxjs';

@Injectable()
export class RolesGuard implements CanActivate {
  canActivate(
    context: ExecutionContext,
  ): boolean | Promise<boolean> | Observable<boolean> {
    const request = context.switchToHttp().getRequest();
    return validateUserRole(request.user);
  }
}
```

### Interceptors

Interceptors can transform results, extend function behavior, add logging, or completely override a response.
They wrap the route handler execution using RxJS `Observable`:

```typescript
import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const now = Date.now();
    return next
      .handle()
      .pipe(tap(() => console.log(`Request took ${Date.now() - now}ms`)));
  }
}
```

### Pipes

Pipes validate or transform input data before it reaches the route handler:

```typescript
import { PipeTransform, Injectable, ArgumentMetadata, BadRequestException } from '@nestjs/common';

@Injectable()
export class ParsePositiveIntPipe implements PipeTransform<string, number> {
  transform(value: string, metadata: ArgumentMetadata): number {
    const val = parseInt(value, 10);
    if (isNaN(val) || val <= 0) {
      throw new BadRequestException('Value must be a positive integer');
    }
    return val;
  }
}
```

### Exception Filters

Exception filters catch all unhandled exceptions and return a standardized error response:

```typescript
import { ExceptionFilter, Catch, ArgumentsHost, HttpException } from '@nestjs/common';
import { Request, Response } from 'express';

@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();
    const status = exception.getStatus();

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      path: request.url,
      message: exception.message,
    });
  }
}
```

### WebSockets

NestJS provides first-class WebSocket support via Gateways — unified across Socket.io and native WebSockets:

```typescript
import { WebSocketGateway, SubscribeMessage, MessageBody } from '@nestjs/websockets';

@WebSocketGateway(80, { namespace: 'chat' })
export class ChatGateway {
  @SubscribeMessage('message')
  handleMessage(@MessageBody() data: string): string {
    return `Echo: ${data}`;
  }
}
```

### Microservices

Built-in support for microservice patterns over multiple transport layers including TCP, Redis, NATS, Kafka,
RabbitMQ, and gRPC:

```typescript
import { NestFactory } from '@nestjs/core';
import { MicroserviceOptions, Transport } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(
    AppModule,
    {
      transport: Transport.KAFKA,
      options: {
        client: { brokers: ['localhost:9092'] },
        consumer: { groupId: 'my-consumer' },
      },
    },
  );
  await app.listen();
}
bootstrap();
```

### GraphQL

Integrated GraphQL support using both code-first and schema-first approaches via `@nestjs/graphql`:

```typescript
import { Resolver, Query, Mutation, Args, Int } from '@nestjs/graphql';
import { UsersService } from './users.service';
import { User } from './entities/user.entity';
import { CreateUserInput } from './dto/create-user.input';

@Resolver(() => User)
export class UsersResolver {
  constructor(private readonly usersService: UsersService) {}

  @Query(() => [User], { name: 'users' })
  findAll(): Promise<User[]> {
    return this.usersService.findAll();
  }

  @Mutation(() => User)
  createUser(@Args('createUserInput') createUserInput: CreateUserInput): Promise<User> {
    return this.usersService.create(createUserInput);
  }
}
```

## Fastify Adapter Option

NestJS defaults to Express but can be swapped for Fastify with minimal code changes, providing up to 2x
better throughput in high-load scenarios:

```typescript
import { NestFactory } from '@nestjs/core';
import {
  FastifyAdapter,
  NestFastifyApplication,
} from '@nestjs/platform-fastify';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter({ logger: true }),
  );
  // Listen on all interfaces (required for Docker)
  await app.listen(3000, '0.0.0.0');
}
bootstrap();
```

:::tip[Fastify Performance]
Fastify processes up to 2× more requests per second than Express in microbenchmarks. Use it when building
high-throughput APIs. Note that some Express-specific middleware (e.g., `body-parser`) requires Fastify equivalents.
:::

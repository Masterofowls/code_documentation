Tauri is a framework for building desktop and mobile applications using a Rust backend and any web frontend (React, Vue, Svelte, Angular, or vanilla JS). Instead of bundling a full browser engine like Electron, Tauri uses the **operating system's native WebView** — resulting in dramatically smaller bundle sizes and lower memory usage.

## Architecture Overview

```
┌─────────────────────────────────────────────┐
│              Your Application               │
│                                             │
│   ┌─────────────────┐  ┌─────────────────┐  │
│   │   WebView (UI)  │  │  Rust Core      │  │
│   │  React/Vue/etc  │◄─►│  (Backend)      │  │
│   │  HTML/CSS/TS    │  │  Commands       │  │
│   └─────────────────┘  │  Events         │  │
│                        │  Filesystem     │  │
│   System WebView:      │  Native APIs    │  │
│   • WKWebView (macOS)  └─────────────────┘  │
│   • WebView2 (Windows)                      │
│   • WebKitGTK (Linux)                       │
└─────────────────────────────────────────────┘
```

The Rust core handles all privileged operations — filesystem access, HTTP requests, OS APIs — and exposes them to the frontend through a typed **command** and **event** system.

## Tauri 2.0 Features

Tauri 2.0 is the current major version with significant improvements over 1.x:

- **Mobile support** — iOS and Android targets alongside desktop (macOS, Windows, Linux)
- **Capabilities system** — fine-grained permission model replacing the 1.x allowlist
- **Plugin architecture** — official plugins for filesystem, HTTP, shell, notifications, and more
- **Channel API** — efficient streaming from Rust to frontend (replaces polling patterns)
- **Multi-window improvements** — better APIs for managing multiple WebviewWindows
- **IPC improvements** — faster serialization, `invoke` with typed channels
- **Minimum system requirements** — Windows 10 (WebView2), macOS 10.13, major Linux distros

## Tauri vs Electron

| Feature | Tauri 2 | Electron |
|---------|---------|----------|
| **Bundle size** | ~5–15 MB | ~150–200 MB |
| **Memory baseline** | ~10–30 MB | ~100–150 MB |
| **Backend language** | Rust | Node.js (JavaScript) |
| **Renderer** | System WebView | Bundled Chromium |
| **Rendering consistency** | Varies by OS | Consistent (Chromium) |
| **Node.js npm access** | No (Rust plugins) | Full npm ecosystem |
| **Mobile support** | Yes (iOS + Android) | No |
| **Security model** | Capabilities-based | contextIsolation |
| **Binary size** | Very small | Very large |
| **Startup time** | Fast | Moderate |
| **Rust knowledge needed** | Yes (for backend) | No |
| **Maturity** | Growing | Very mature |
| **Learning curve** | Steeper (Rust) | Moderate |
| **Community size** | Large, growing | Very large |
| **GitHub Stars** | ~85k+ | ~115k+ |

:::tip[Choosing Tauri]
Choose Tauri when: bundle size and memory usage are critical, you want OS-native file dialogs/menus, security is paramount, you need mobile support, or your team is comfortable with Rust. Choose Electron when: you need consistent rendering across OS, require the full npm ecosystem, need broad WebRTC/media API support, or prefer JavaScript-only development.
:::

## WebView Foundation (WRY)

Tauri uses [WRY](https://github.com/tauri-apps/wry) — a cross-platform WebView rendering library:

| Platform | WebView Engine | Notes |
|----------|----------------|-------|
| **macOS** | WKWebView | Part of macOS SDK; very capable |
| **Windows** | WebView2 | Chromium-based; installed with Windows 11+ |
| **Linux** | WebKitGTK | GTK-integrated WebKit |
| **iOS** | WKWebView | Same as macOS |
| **Android** | Android WebView | Chromium-based |

:::caution[WebView Rendering Differences]
Because Tauri uses the system WebView, CSS and JavaScript behavior can differ across operating systems. Test your UI on all target platforms. CSS features well-supported in Chrome may be missing in older WebKitGTK on Linux.
:::

## Mobile Support (Tauri 2)

Tauri 2 adds iOS and Android targets using the same Rust core:

```bash
# Add mobile targets
npm run tauri ios init
npm run tauri android init

# Run on iOS simulator
npm run tauri ios dev

# Run on Android emulator
npm run tauri android dev

# Build for iOS
npm run tauri ios build

# Build for Android
npm run tauri android build
```

The same `#[tauri::command]` functions and plugin APIs work across desktop and mobile.

## Security-First Design

Tauri's security model is built around the principle of **least privilege**:

1. **No Node.js in frontend** — the WebView has no system access by default
2. **Capabilities system** — each permission is explicitly declared in `tauri.conf.json`
3. **Command allowlist** — only `#[tauri::command]` functions registered in `generate_handler![]` are callable
4. **Rust type safety** — all command inputs are validated at the type level before your code runs
5. **CSP enforcement** — Content Security Policy is configured in `tauri.conf.json`
6. **No eval by default** — the CSP blocks `eval()` and inline scripts

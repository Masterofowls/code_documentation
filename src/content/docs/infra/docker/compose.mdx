# Docker Compose

Docker Compose is a tool for defining and running multi-container applications. A single `compose.yml` file describes the entire application stack — services, networks, volumes, configs, and secrets.

:::tip[Compose v2 is built into Docker]
Docker Compose v2 (`docker compose`) is a Go rewrite bundled with Docker Desktop and the Docker Compose plugin. The old Python-based `docker-compose` v1 is deprecated. Always use `docker compose` (space, not hyphen).
:::

## compose.yml Structure

```yaml
# compose.yml
name: myapp   # optional — defaults to directory name

services:     # container definitions
  web: ...
  db: ...

networks:     # custom networks
  frontend: ...
  backend: ...

volumes:      # named volumes
  pgdata: ...

configs:      # non-sensitive config files
  nginx_conf: ...

secrets:      # sensitive data
  db_password: ...
```

## Service Definition

A complete service definition with all commonly used fields:

```yaml
services:
  api:
    # ── Image / Build ─────────────────────────────────────────
    image: myapp/api:1.2.3          # use a pre-built image
    build:
      context: ./api                # path to Dockerfile context
      dockerfile: Dockerfile        # optional — defaults to Dockerfile
      args:
        NODE_ENV: production
      target: runtime               # multi-stage build target
      cache_from:
        - myapp/api:cache

    # ── Identity ──────────────────────────────────────────────
    container_name: myapp-api       # override auto-generated name

    # ── Ports ─────────────────────────────────────────────────
    ports:
      - "3000:3000"                 # host:container
      - "127.0.0.1:9229:9229"      # bind to specific host IP

    # ── Environment ───────────────────────────────────────────
    environment:
      NODE_ENV: production
      PORT: 3000
      DB_HOST: postgres
      DB_PORT: 5432
    env_file:
      - .env                        # load from file
      - .env.local                  # later files override earlier

    # ── Volumes ───────────────────────────────────────────────
    volumes:
      - ./src:/app/src              # bind mount for development
      - node_modules:/app/node_modules   # named volume (preserve)
      - /app/dist                   # anonymous volume

    # ── Dependencies ──────────────────────────────────────────
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started

    # ── Restart ───────────────────────────────────────────────
    restart: unless-stopped         # no | always | on-failure | unless-stopped

    # ── Health Check ──────────────────────────────────────────
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

    # ── Networking ────────────────────────────────────────────
    networks:
      - frontend
      - backend

    # ── Command ───────────────────────────────────────────────
    command: ["node", "dist/server.js"]
    # entrypoint: ["/sbin/tini", "--"]

    # ── Resources ─────────────────────────────────────────────
    deploy:
      resources:
        limits:
          cpus: "0.50"
          memory: 512M
        reservations:
          cpus: "0.25"
          memory: 256M

    # ── Profiles ──────────────────────────────────────────────
    profiles:
      - api                         # only start with --profile api

    # ── Logging ───────────────────────────────────────────────
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
```

## Named Volumes vs Bind Mounts

```yaml
volumes:
  # Top-level named volume declaration (Docker-managed)
  pgdata:
    driver: local
  redisdata:
    driver: local
  # External volume (must exist before compose up)
  backup:
    external: true
    name: myapp-backup

services:
  postgres:
    volumes:
      # Named volume — survives container removal
      - pgdata:/var/lib/postgresql/data

  web:
    volumes:
      # Bind mount — mounts host directory into container
      - ./src:/app/src
      # Read-only bind mount
      - ./config/nginx.conf:/etc/nginx/conf.d/default.conf:ro
      # tmpfs — in-memory, not persisted
      - type: tmpfs
        target: /tmp
        tmpfs:
          size: 100m
```

## Networks

```yaml
networks:
  # Default bridge if you omit network definitions
  frontend:
    driver: bridge
  backend:
    driver: bridge
    internal: true      # no external connectivity

services:
  nginx:
    networks:
      - frontend        # public-facing

  api:
    networks:
      - frontend        # reachable from nginx by hostname "api"
      - backend         # can reach database

  postgres:
    networks:
      backend:
        aliases:
          - db          # reachable as "db" or "postgres" in backend network
```

:::tip[Service name = hostname]
Within the same Compose network, services can reach each other using the service name as the hostname. `api` can connect to `postgres:5432` without any additional configuration.
:::

## Override Files

Compose merges multiple files in order. This lets you maintain a base configuration and environment-specific overrides.

```bash
# Default merge order
docker compose up
# reads: compose.yml + compose.override.yml (auto-merged)

# Explicit file list
docker compose -f compose.yml -f compose.prod.yml up -d
```

**compose.yml** (base):
```yaml
services:
  api:
    image: myapp/api:${TAG:-latest}
    environment:
      NODE_ENV: production
  postgres:
    image: postgres:16-alpine
```

**compose.override.yml** (development — auto-loaded):
```yaml
services:
  api:
    build: .                        # build locally in dev
    volumes:
      - ./src:/app/src              # hot reload
    environment:
      NODE_ENV: development
    ports:
      - "9229:9229"                 # Node.js debugger
```

**compose.prod.yml** (production):
```yaml
services:
  api:
    restart: always
    deploy:
      resources:
        limits:
          memory: 512M
    logging:
      driver: awslogs
      options:
        awslogs-group: /myapp/api
```

## Compose CLI Reference

```bash
# Start all services (build if needed, detached)
docker compose up -d

# Start and force rebuild images
docker compose up -d --build

# Start specific services
docker compose up -d api postgres

# Stop and remove containers, networks (keep volumes)
docker compose down

# Also remove volumes (CAUTION: data loss)
docker compose down -v

# Also remove images
docker compose down --rmi all

# List running services
docker compose ps

# View logs
docker compose logs -f

# View logs for specific service
docker compose logs -f api

# Execute command in running service
docker compose exec api sh

# Run a one-off command (creates a new container)
docker compose run --rm api npm run migrate

# Rebuild images without starting
docker compose build

# Pull latest images
docker compose pull

# Show current configuration (after variable substitution)
docker compose config

# Validate the compose file
docker compose config --quiet
```

## Scaling Services

```bash
# Scale the api service to 3 instances
docker compose up -d --scale api=3

# View the scaled containers
docker compose ps
```

:::caution[Port conflicts when scaling]
If your service maps a specific host port (e.g., `"3000:3000"`), scaling it will fail due to port conflicts. Use a range (`"3000-3002:3000"`) or remove the host port and use a load balancer.
:::

## Variable Substitution

```yaml
# compose.yml
services:
  api:
    image: ${REGISTRY:-docker.io}/myapp:${TAG:-latest}
    environment:
      PORT: ${API_PORT:-3000}
```

```bash
# .env (auto-loaded by Compose)
REGISTRY=ghcr.io
TAG=1.2.3
API_PORT=3001
```

## Full Stack Example

A production-like stack: Node.js API + PostgreSQL + Redis + Nginx reverse proxy.

```yaml
# compose.yml
name: myapp

services:
  # ── Nginx Reverse Proxy ───────────────────────────────────
  nginx:
    image: nginx:1.27-alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
    depends_on:
      api:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - frontend
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"

  # ── Node.js API ───────────────────────────────────────────
  api:
    build:
      context: .
      target: runtime
    environment:
      NODE_ENV: production
      PORT: 3000
      DB_HOST: postgres
      DB_PORT: 5432
      DB_NAME: myapp
      DB_USER: app
      REDIS_URL: redis://redis:6379
    env_file:
      - .env.production
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - frontend
      - backend
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 30s
    deploy:
      resources:
        limits:
          cpus: "1.0"
          memory: 512M

  # ── PostgreSQL ────────────────────────────────────────────
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: app
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
    secrets:
      - db_password
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./postgres/init:/docker-entrypoint-initdb.d:ro
    restart: unless-stopped
    networks:
      - backend
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U app -d myapp"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 20s

  # ── Redis ─────────────────────────────────────────────────
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes --requirepass ${REDIS_PASSWORD}
    volumes:
      - redisdata:/data
    restart: unless-stopped
    networks:
      - backend
    healthcheck:
      test: ["CMD", "redis-cli", "--no-auth-warning", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

# ── Named Volumes ─────────────────────────────────────────
volumes:
  pgdata:
  redisdata:

# ── Networks ──────────────────────────────────────────────
networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    internal: true    # no direct internet access from backend

# ── Secrets ───────────────────────────────────────────────
secrets:
  db_password:
    file: ./secrets/db_password.txt
```

**Nginx configuration** (`nginx/nginx.conf`):

```nginx
upstream api {
    server api:3000;
}

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## Profiles

Use profiles to activate optional services (e.g., debug tools, admin UIs):

```yaml
services:
  api:
    image: myapp/api
    # no profile — always starts

  pgadmin:
    image: dpage/pgadmin4
    profiles:
      - debug
    ports:
      - "5050:80"

  mailhog:
    image: mailhog/mailhog
    profiles:
      - debug
    ports:
      - "8025:8025"
```

```bash
# Start core services only
docker compose up -d

# Start core + debug services
docker compose --profile debug up -d
```

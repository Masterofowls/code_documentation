# Getting Started with Docker

This guide walks you through installing Docker, verifying your setup, and building fluency with the essential CLI commands you'll use every day.

---

## Install Docker

<Tabs>
  <TabItem label="macOS">
    Install **Docker Desktop** for macOS (Apple Silicon or Intel):

    ```bash
    # Using Homebrew (recommended)
    brew install --cask docker

    # Then launch Docker Desktop from Applications
    open /Applications/Docker.app
    ```

    Or download the installer directly from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    Verify the installation:

    ```bash
    docker --version
    # Docker version 27.x.x, build xxxxxxx

    docker compose version
    # Docker Compose version v2.x.x
    ```

</TabItem>
  <TabItem label="Windows">
    Install **Docker Desktop** for Windows with WSL2 backend:

    ```powershell
    # Using winget
    winget install Docker.DockerDesktop

    # Or using Chocolatey
    choco install docker-desktop
    ```

    Requirements:
    - Windows 10/11 64-bit
    - WSL2 enabled (`wsl --install`)
    - Virtualization enabled in BIOS

    After installation, Docker Desktop starts automatically. Look for the whale icon in the system tray.

</TabItem>
  <TabItem label="Linux (Ubuntu/Debian)">
    Install **Docker Engine** directly on Linux:

    ```bash
    # Remove old versions
    sudo apt remove docker docker-engine docker.io containerd runc

    # Install prerequisites
    sudo apt update
    sudo apt install -y ca-certificates curl gnupg lsb-release

    # Add Docker's official GPG key
    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
      | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    sudo chmod a+r /etc/apt/keyrings/docker.gpg

    # Add the repository
    echo \
      "deb [arch=$(dpkg --print-architecture) \
      signed-by=/etc/apt/keyrings/docker.gpg] \
      https://download.docker.com/linux/ubuntu \
      $(lsb_release -cs) stable" \
      | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    # Install Docker Engine
    sudo apt update
    sudo apt install -y docker-ce docker-ce-cli containerd.io \
      docker-buildx-plugin docker-compose-plugin

    # Add your user to the docker group (avoid sudo)
    sudo usermod -aG docker $USER
    newgrp docker
    ```

</TabItem>
  <TabItem label="Linux (RHEL/Fedora)">
    ```bash
    # Fedora
    sudo dnf install -y dnf-plugins-core
    sudo dnf config-manager --add-repo \
      https://download.docker.com/linux/fedora/docker-ce.repo
    sudo dnf install -y docker-ce docker-ce-cli containerd.io \
      docker-buildx-plugin docker-compose-plugin

    # Start and enable
    sudo systemctl enable --now docker

    # Rootless (optional, recommended)
    dockerd-rootless-setuptool.sh install
    ```

</TabItem>
</Tabs>

:::tip[Post-Install Verification]
Always run `docker run hello-world` after installing. If you see a success message, both the daemon and registry connectivity are working.
:::

## Your First Container

```bash
docker run hello-world
```

What happens under the hood:

<Steps>
  1. Docker checks if the `hello-world` image exists locally 2. It's not found,
  so Docker pulls it from Docker Hub (`docker pull hello-world`) 3. Docker
  creates a new container from the image 4. The container runs, prints a
  message, and exits 5. The container stops (but remains in `docker ps -a` until
  removed)
</Steps>

## Working with Images

Images are the blueprints for containers. They live in registries and are pulled to your local image cache.

```bash
# Pull an image without running it
docker pull nginx:alpine
docker pull postgres:16
docker pull node:20-alpine

# List all local images
docker images
# or
docker image ls

# REPOSITORY    TAG        IMAGE ID       CREATED        SIZE
# nginx         alpine     abc123def456   2 weeks ago    43.3MB
# postgres      16         def789abc012   3 weeks ago    432MB
# node          20-alpine  ghi345jkl678   4 weeks ago    179MB

# Show image history (layers)
docker history nginx:alpine

# Inspect image metadata
docker image inspect nginx:alpine

# Remove an image
docker rmi nginx:alpine
# or
docker image rm nginx:alpine

# Remove all dangling (untagged) images
docker image prune

# Remove all unused images
docker image prune -a
```

:::caution[Image Tags Matter]
Always pin image versions in production (`postgres:16.2`, not `postgres:latest`). The `latest` tag is just a convention — it doesn't guarantee the newest version and can break builds unexpectedly.
:::

## Running Containers

The `docker run` command creates and starts a container. It has dozens of flags — here are the ones you'll use constantly.

### Essential `docker run` Flags

```bash
# -d  Detached mode (run in background)
docker run -d nginx:alpine

# -p  Publish port (host:container)
docker run -d -p 8080:80 nginx:alpine

# --name  Give the container a name
docker run -d -p 8080:80 --name my-nginx nginx:alpine

# -e  Set environment variables
docker run -d -e POSTGRES_PASSWORD=secret postgres:16

# --rm  Auto-remove container when it exits
docker run --rm alpine echo "hello"

# -v  Mount a volume or bind mount
docker run -d -v mydata:/var/lib/postgresql/data postgres:16
docker run -d -v $(pwd)/html:/usr/share/nginx/html nginx:alpine

# -it  Interactive terminal (attach stdin + pseudo-TTY)
docker run -it ubuntu:22.04 bash

# --env-file  Load env vars from a file
docker run --env-file .env myapp:latest

# --network  Attach to a specific network
docker run --network my-network myapp:latest

# --restart  Restart policy
docker run -d --restart unless-stopped nginx:alpine

# --memory / --cpus  Resource limits
docker run -d --memory 512m --cpus 0.5 myapp:latest
```

### Practical Examples

**Run Nginx and serve a local directory:**

```bash
docker run -d \
  --name webserver \
  -p 8080:80 \
  -v $(pwd)/public:/usr/share/nginx/html:ro \
  --restart unless-stopped \
  nginx:alpine

# Visit http://localhost:8080
```

**Run PostgreSQL with a persistent volume:**

```bash
docker run -d \
  --name postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=app \
  -e POSTGRES_PASSWORD=secret \
  -e POSTGRES_DB=mydb \
  -v postgres_data:/var/lib/postgresql/data \
  --restart unless-stopped \
  postgres:16-alpine
```

**Run Redis:**

```bash
docker run -d \
  --name redis \
  -p 6379:6379 \
  -v redis_data:/data \
  --restart unless-stopped \
  redis:7-alpine redis-server --appendonly yes
```

## Managing Running Containers

```bash
# List running containers
docker ps

# List ALL containers (including stopped)
docker ps -a

# CONTAINER ID   IMAGE          COMMAND    STATUS         PORTS            NAMES
# a1b2c3d4e5f6   nginx:alpine   ...        Up 5 minutes   0.0.0.0:8080->   my-nginx

# Stop a container (sends SIGTERM, waits, then SIGKILL)
docker stop my-nginx

# Start a stopped container
docker start my-nginx

# Restart a container
docker restart my-nginx

# Kill immediately (SIGKILL)
docker kill my-nginx

# Remove a stopped container
docker rm my-nginx

# Force remove a running container
docker rm -f my-nginx

# Remove all stopped containers
docker container prune
```

## Executing Commands in Containers

```bash
# Open an interactive shell in a running container
docker exec -it my-nginx sh       # Alpine-based (no bash)
docker exec -it my-app bash       # Debian/Ubuntu-based

# Run a one-off command
docker exec my-nginx nginx -t     # Test nginx config

# Run as a specific user
docker exec -u root my-nginx sh

# Set environment variables
docker exec -e DEBUG=true my-app node debug.js
```

:::tip[Shell Availability]
Alpine-based images use `sh` (busybox), not `bash`. Debian/Ubuntu images usually have `bash`. If neither works, try `docker exec -it container cat /etc/os-release` to identify the OS.
:::

## Logs

```bash
# View logs
docker logs my-nginx

# Follow (stream) logs
docker logs -f my-nginx

# Last N lines
docker logs --tail 100 my-nginx

# With timestamps
docker logs -t my-nginx

# Since a time (e.g., last 5 minutes)
docker logs --since 5m my-nginx

# Between timestamps
docker logs --since "2024-01-01T00:00:00" --until "2024-01-02T00:00:00" my-nginx
```

## Inspecting Containers

```bash
# Full JSON metadata about a container
docker inspect my-nginx

# Extract a specific field with Go template
docker inspect -f '{{.State.Status}}' my-nginx
docker inspect -f '{{.NetworkSettings.IPAddress}}' my-nginx
docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' my-nginx

# Live resource usage stats
docker stats

# Stats for a specific container (no-stream = one snapshot)
docker stats --no-stream my-nginx

# Top processes inside container
docker top my-nginx
```

## Disk Usage and Cleanup

Docker objects accumulate quickly. Use these commands to understand and reclaim disk space.

```bash
# Show disk usage by Docker objects
docker system df

# TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
# Images          12        3         4.2GB     3.1GB (73%)
# Containers      5         2         120MB     80MB (66%)
# Local Volumes   8         3         2.1GB     1.5GB (71%)
# Build Cache     45        0         890MB     890MB

# Verbose breakdown
docker system df -v

# Remove stopped containers
docker container prune

# Remove unused images
docker image prune

# Remove unused volumes
docker volume prune

# Remove unused networks
docker network prune

# Remove EVERYTHING unused (containers, images, networks, build cache)
docker system prune

# Also remove volumes (⚠️ data loss!)
docker system prune --volumes

# Skip confirmation prompt
docker system prune -af
```

:::caution[Volume Prune is Destructive]
`docker volume prune` and `docker system prune --volumes` permanently delete volume data. Always back up important data before pruning. Named volumes that are not attached to any container are considered "unused."
:::

## Port Mapping

Containers have their own network namespace. To access a service running inside a container, you must **publish** the port.

```bash
# Map host port 8080 to container port 80
docker run -p 8080:80 nginx

# Map all interfaces on host port 5432 to container port 5432
docker run -p 5432:5432 postgres

# Bind only to localhost (not exposed externally)
docker run -p 127.0.0.1:5432:5432 postgres

# Random host port (Docker chooses)
docker run -p 5432 postgres
docker port my-container 5432   # Find the assigned host port

# Map multiple ports
docker run -p 80:80 -p 443:443 nginx
```

## Environment Variables

```bash
# Pass individual variables
docker run -e NODE_ENV=production -e PORT=3000 myapp

# Load from a file
cat .env
# NODE_ENV=production
# DATABASE_URL=postgres://...

docker run --env-file .env myapp

# Inspect env vars in a running container
docker exec my-app env
docker inspect -f '{{range .Config.Env}}{{println .}}{{end}}' my-app
```

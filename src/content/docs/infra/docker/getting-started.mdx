# Getting Started with Docker

This guide covers installing Docker, running your first container, and mastering the essential commands you will use every day.

---

## Installing Docker

<Tabs>
  <TabItem label="macOS">
    Download and install **Docker Desktop for Mac** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    Or install via Homebrew:

    ```bash
    brew install --cask docker
    ```

    After installation, launch Docker Desktop from your Applications folder. The Docker whale icon in the menu bar confirms the daemon is running.
  </TabItem>
  <TabItem label="Windows">
    Download **Docker Desktop for Windows** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    Ensure **WSL 2** backend is enabled (recommended) or Hyper-V is available. Docker Desktop will guide you through the configuration during setup.

    After installation, restart your machine and launch Docker Desktop from the Start menu.
  </TabItem>
  <TabItem label="Linux (Ubuntu/Debian)">
    Install Docker Engine directly — no GUI required:

    ```bash
    # Remove old versions
    sudo apt-get remove -y docker docker-engine docker.io containerd runc

    # Install prerequisites
    sudo apt-get update
    sudo apt-get install -y ca-certificates curl gnupg

    # Add Docker's official GPG key
    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
      | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    sudo chmod a+r /etc/apt/keyrings/docker.gpg

    # Add the repository
    echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
      https://download.docker.com/linux/ubuntu \
      $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
      | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    # Install Docker Engine + Compose plugin
    sudo apt-get update
    sudo apt-get install -y \
      docker-ce \
      docker-ce-cli \
      containerd.io \
      docker-buildx-plugin \
      docker-compose-plugin

    # Allow running docker without sudo (requires logout/login to take effect)
    sudo usermod -aG docker $USER
    newgrp docker
    ```
  </TabItem>
</Tabs>

### Verify Installation

```bash
docker --version
# Docker version 25.0.3, build 4debf41

docker compose version
# Docker Compose version v2.24.5

# Run the hello-world container to confirm everything works
docker run hello-world
```

The `hello-world` container prints a confirmation message and exits. This verifies the daemon is running and can pull from Docker Hub.

## Core Concepts

- **Image** — A read-only, layered blueprint. Think of it as a class definition.
- **Container** — A running instance of an image. Think of it as an object created from that class.
- **Registry** — Where images are stored and distributed. Docker Hub is the default public registry.
- **Layer** — Each Dockerfile instruction creates a layer. Layers are shared and cached for efficiency.

## Pulling and Listing Images

```bash
# Pull a specific version from Docker Hub
docker pull nginx:1.25-alpine

# Pull the latest tag (avoid in production — use a specific version)
docker pull postgres:latest

# List all locally available images
docker images

# List images in quiet mode (IDs only — useful for scripting)
docker images -q

# List with full SHA digests
docker images --digests

# Remove a specific image
docker rmi nginx:1.25-alpine

# Remove all dangling (untagged) images
docker image prune

# Remove all unused images (including tagged but unreferenced)
docker image prune -a
```

## Running Containers

The `docker run` command creates and starts a container from an image in a single step.

```bash
# Basic run — print a message and exit
docker run --rm ubuntu echo "Hello from Ubuntu"

# Detached mode — run in the background
docker run -d nginx:alpine

# Give the container a name
docker run -d --name my-nginx nginx:alpine

# Port mapping: hostPort:containerPort
docker run -d -p 8080:80 --name web nginx:alpine

# Multiple port mappings
docker run -d -p 8080:80 -p 8443:443 nginx:alpine

# Set a single environment variable
docker run -d -e MY_VAR=hello nginx:alpine

# PostgreSQL with multiple environment variables
docker run -d \
  --name postgres \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=secret \
  -e POSTGRES_DB=mydb \
  postgres:16-alpine

# Interactive terminal — useful for exploration and debugging
docker run -it ubuntu bash

# Auto-remove container when it exits
docker run --rm alpine echo "one-off task"

# Bind mount: host-path:container-path
docker run -d -v "$(pwd)/html":/usr/share/nginx/html nginx:alpine

# Named volume
docker run -d -v pgdata:/var/lib/postgresql/data postgres:16

# Run with resource limits
docker run -d --memory="512m" --cpus="0.5" --name limited nginx:alpine
```

:::tip[Flag quick reference]
| Flag | Meaning |
|---|---|
| `-d` | Detach — run in background |
| `-p host:container` | Publish a port |
| `-e KEY=VALUE` | Set environment variable |
| `-v src:dst` | Bind mount or named volume |
| `-it` | Interactive + allocate TTY |
| `--rm` | Auto-remove on exit |
| `--name` | Assign a container name |
| `--memory` | Memory limit |
| `--cpus` | CPU limit |
:::

## Listing and Managing Containers

```bash
# List running containers
docker ps

# List ALL containers including stopped ones
docker ps -a

# List only container IDs (useful for scripting)
docker ps -q

# List all container IDs including stopped
docker ps -aq

# Stop a container gracefully (SIGTERM → SIGKILL after 10s)
docker stop my-nginx

# Stop with custom timeout (seconds before SIGKILL)
docker stop --time 30 my-nginx

# Start a stopped container
docker start my-nginx

# Restart a running or stopped container
docker restart my-nginx

# Remove a stopped container
docker rm my-nginx

# Force-remove a running container (SIGKILL immediately)
docker rm -f my-nginx

# Remove all stopped containers
docker container prune

# Stop all running containers
docker stop $(docker ps -q)
```

## Interacting with Running Containers

```bash
# Execute a single command inside a running container
docker exec my-nginx ls /etc/nginx/conf.d

# Open an interactive shell (use sh for Alpine-based images)
docker exec -it my-nginx sh

# Open bash (if available in the image)
docker exec -it my-postgres bash

# Run as a specific user
docker exec -it --user root my-nginx sh

# Set an environment variable for the exec session
docker exec -it -e DEBUG=true my-app sh

# Copy a file FROM container TO host
docker cp my-nginx:/etc/nginx/nginx.conf ./nginx.conf

# Copy a file FROM host TO container
docker cp ./nginx.conf my-nginx:/etc/nginx/nginx.conf
```

## Logs

```bash
# Show all logs for a container
docker logs my-nginx

# Follow (stream) logs in real time — Ctrl+C to stop
docker logs -f my-nginx

# Show only the last 50 lines
docker logs --tail 50 my-nginx

# Show logs with timestamps
docker logs -t my-nginx

# Combine: follow + last 20 lines + timestamps
docker logs -ft --tail 20 my-nginx

# Show logs since a point in time
docker logs --since 2024-01-01T00:00:00 my-nginx
docker logs --since 1h my-nginx   # Last hour
```

## Inspecting Containers and Images

```bash
# Full JSON metadata for a container
docker inspect my-nginx

# Extract the IP address using a Go template
docker inspect -f '{{.NetworkSettings.IPAddress}}' my-nginx

# Extract the container's restart count
docker inspect -f '{{.RestartCount}}' my-nginx

# Inspect an image
docker inspect nginx:alpine

# Show image build history and layer sizes
docker history nginx:alpine

# Show changes to a container's filesystem since creation
docker diff my-nginx
# C = changed, A = added, D = deleted
```

## Resource Usage and Statistics

```bash
# Live CPU, memory, network I/O, and block I/O stats for all containers
docker stats

# One-shot stats (no streaming) — useful for scripting
docker stats --no-stream

# Stats for a specific container only
docker stats --no-stream my-nginx

# Format output as JSON
docker stats --format json --no-stream

# Show disk usage: images, containers, volumes, build cache
docker system df

# Verbose breakdown by object
docker system df -v
```

## Cleanup Commands

```bash
# Remove stopped containers, unused networks, dangling images, and build cache
docker system prune

# Also include unused images (not just dangling)
docker system prune -a

# Skip confirmation prompt
docker system prune -af

# Remove only unused named volumes (NOT included in system prune by default)
docker volume prune

# Remove everything including volumes
docker system prune -af --volumes
```

:::caution[system prune -a removes everything]
`docker system prune -a` removes ALL locally cached images that are not used by a running container. This includes images you pulled but haven't used recently. It can cause very long re-pull times on the next `docker run`. Use `docker images` first to review what exists.
:::

## Practical Examples

### Running Nginx

```bash
# Serve the current directory on port 8080
docker run -d \
  --name web \
  -p 8080:80 \
  -v "$(pwd)":/usr/share/nginx/html:ro \
  nginx:1.25-alpine

# Verify
curl http://localhost:8080
```

### Running PostgreSQL

```bash
docker run -d \
  --name postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=supersecret \
  -e POSTGRES_DB=myapp \
  -v pgdata:/var/lib/postgresql/data \
  postgres:16-alpine

# Connect with psql inside the container
docker exec -it postgres psql -U admin -d myapp

# Run a SQL command directly
docker exec -it postgres psql -U admin -d myapp -c "SELECT version();"
```

### Running Redis

```bash
docker run -d \
  --name redis \
  -p 6379:6379 \
  redis:7-alpine redis-server --appendonly yes

# Test with redis-cli
docker exec -it redis redis-cli ping
# PONG

# Set and get a key
docker exec -it redis redis-cli set greeting "hello docker"
docker exec -it redis redis-cli get greeting
```

## Port Mapping Deep Dive

Port mapping (publishing) exposes a container port to the host.

```bash
# Map host port 8080 to container port 80
docker run -p 8080:80 nginx

# Bind to loopback only (not accessible from network)
docker run -p 127.0.0.1:8080:80 nginx

# Publish all EXPOSEd ports to random host ports
docker run -P nginx

# Check what ports are mapped
docker port my-nginx
# 80/tcp -> 0.0.0.0:49153

# Map a UDP port
docker run -p 53:53/udp my-dns-server
```

## Environment Variables

Environment variables are the standard 12-factor way to configure containers at runtime.

```bash
# Single variable
docker run -e NODE_ENV=production myapp

# Multiple variables
docker run \
  -e NODE_ENV=production \
  -e PORT=3000 \
  -e LOG_LEVEL=info \
  myapp

# Load from a .env file
docker run --env-file .env myapp

# Read from host shell environment (no = sign)
export API_KEY=abc123
docker run -e API_KEY myapp
```

Example `.env` file:

```bash
# .env — never commit this to version control
NODE_ENV=production
DATABASE_URL=postgres://admin:secret@localhost:5432/myapp
REDIS_URL=redis://localhost:6379
PORT=3000
JWT_SECRET=changethisinproduction
```

:::caution[Environment variables are not secure secrets]
Environment variables are visible via `docker inspect` and to all processes inside the container. For sensitive credentials in production, prefer Docker secrets (Swarm), Kubernetes secrets, or a secrets manager like HashiCorp Vault or AWS Secrets Manager.
:::

# Getting Started with Docker

This guide walks you through installing Docker, verifying your installation, and building confidence with the essential CLI commands you'll use daily.

---

## Installing Docker

<Tabs>
  <TabItem label="macOS">
    Download **Docker Desktop for Mac** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    Or install via Homebrew:

    ```bash
    brew install --cask docker
    ```

    After installation, launch Docker Desktop from your Applications folder. Wait for the whale icon in the menu bar to stop animating — Docker is ready.
</TabItem>
  <TabItem label="Windows">
    Download **Docker Desktop for Windows** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    **Prerequisites:**
    - Windows 10/11 (64-bit, Build 19041+)
    - WSL 2 enabled (recommended) or Hyper-V

    Enable WSL 2:
    ```bash
    wsl --install
    ```

    After installing Docker Desktop, enable WSL 2 backend in Settings → General → "Use the WSL 2 based engine".
</TabItem>
  <TabItem label="Linux (Ubuntu/Debian)">
    ```bash
    # Remove old versions
    sudo apt-get remove docker docker-engine docker.io containerd runc

    # Install prerequisites
    sudo apt-get update
    sudo apt-get install ca-certificates curl gnupg lsb-release

    # Add Docker's official GPG key
    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
      | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    sudo chmod a+r /etc/apt/keyrings/docker.gpg

    # Add repository
    echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
      https://download.docker.com/linux/ubuntu \
      $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
      | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    # Install Docker Engine
    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

    # Allow running docker without sudo
    sudo usermod -aG docker $USER
    newgrp docker
    ```
</TabItem>
</Tabs>

## Your First Container

<Steps>
1. **Run the hello-world container**

   ```bash
   docker run hello-world
   ```

   Docker will:
   - Check if the `hello-world` image exists locally
   - Pull it from Docker Hub if not found
   - Create and start a container
   - Print a message and exit

2. **Run an interactive Ubuntu shell**

   ```bash
   docker run -it ubuntu bash
   ```

   The `-it` flag gives you an interactive terminal. Type `exit` to stop.

3. **Run Nginx in the background**

   ```bash
   docker run -d -p 8080:80 --name my-nginx nginx
   ```

   Open `http://localhost:8080` — you should see the Nginx welcome page.
</Steps>

## Essential Docker Commands

### Pulling and Listing Images

```bash
# Pull an image from Docker Hub
docker pull nginx
docker pull nginx:1.25-alpine       # specific tag
docker pull ubuntu:22.04

# List local images
docker images
docker image ls

# Remove an image
docker rmi nginx
docker image rm nginx:1.25-alpine

# Remove all dangling (untagged) images
docker image prune

# Remove all unused images
docker image prune -a
```

### Running Containers

The `docker run` command is the most important command. It creates and starts a container in one step.

```bash
# Basic run
docker run nginx

# Run in detached (background) mode
docker run -d nginx

# Name the container
docker run -d --name web-server nginx

# Map ports (host:container)
docker run -d -p 8080:80 nginx
docker run -d -p 127.0.0.1:8080:80 nginx   # bind to localhost only

# Set environment variables
docker run -d -e NODE_ENV=production -e PORT=3000 my-node-app

# Mount a volume
docker run -d -v my-data:/app/data nginx          # named volume
docker run -d -v /host/path:/container/path nginx  # bind mount
docker run -d -v $(pwd):/app nginx                 # current dir

# Remove container automatically when it exits
docker run --rm ubuntu echo "hello"

# Run interactively with pseudo-TTY
docker run -it ubuntu bash
docker run -it --rm alpine sh

# Set working directory
docker run -w /app -it node:20 bash

# Limit resources
docker run -d --memory="512m" --cpus="1.0" nginx

# Set restart policy
docker run -d --restart=unless-stopped nginx
```

:::tip[Combining flags]
You can combine short flags: `-d -p 8080:80 -e FOO=bar` is equivalent to `-dp 8080:80 -e FOO=bar`. The `-it` combination is very common for interactive containers.
:::

### Listing and Inspecting Containers

```bash
# List running containers
docker ps

# List ALL containers (including stopped)
docker ps -a

# Show only container IDs
docker ps -q

# Show last N created containers
docker ps -n 5

# Formatted output
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

### Container Lifecycle

```bash
# Stop a container (sends SIGTERM, waits, then SIGKILL)
docker stop my-nginx
docker stop my-nginx --time 30    # 30 second grace period

# Start a stopped container
docker start my-nginx

# Restart a container
docker restart my-nginx

# Kill a container immediately (SIGKILL)
docker kill my-nginx

# Remove a stopped container
docker rm my-nginx

# Force remove a running container
docker rm -f my-nginx

# Remove all stopped containers
docker container prune
```

### Executing Commands in Running Containers

```bash
# Open an interactive shell in a running container
docker exec -it my-nginx bash
docker exec -it my-nginx sh          # for alpine-based images

# Run a single command
docker exec my-nginx nginx -t         # test nginx config
docker exec my-nginx env              # list environment variables

# Run as a specific user
docker exec -it -u root my-nginx bash
docker exec -it -u 1000 my-nginx sh

# Set environment variable for the exec'd command
docker exec -e DEBUG=true my-nginx node debug.js
```

### Viewing Logs

```bash
# View container logs
docker logs my-nginx

# Follow log output (like tail -f)
docker logs -f my-nginx

# Show last 50 lines
docker logs --tail 50 my-nginx

# Show logs with timestamps
docker logs -t my-nginx

# Combine: last 100 lines, follow, with timestamps
docker logs -f -t --tail 100 my-nginx

# Show logs since a timestamp or duration
docker logs --since 2024-01-15T10:00:00 my-nginx
docker logs --since 30m my-nginx
```

### Inspecting Containers and Images

```bash
# Full JSON inspection of a container
docker inspect my-nginx

# Extract a specific field with --format
docker inspect --format '{{.State.Status}}' my-nginx
docker inspect --format '{{.NetworkSettings.IPAddress}}' my-nginx
docker inspect --format '{{json .Mounts}}' my-nginx | jq

# Inspect an image
docker inspect nginx:latest

# Show image history (layers)
docker history nginx:latest
docker history --no-trunc nginx:latest
```

### Monitoring and Stats

```bash
# Live resource usage stats for all containers
docker stats

# Stats for specific containers
docker stats my-nginx my-postgres

# One-shot stats (no streaming)
docker stats --no-stream

# Formatted stats
docker stats --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"
```

## Practical Examples

### Running Nginx

```bash
# Serve a static site from current directory
docker run -d \
  --name nginx \
  -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  nginx:alpine

# Custom nginx config
docker run -d \
  --name nginx \
  -p 80:80 \
  -v $(pwd)/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  nginx:alpine
```

### Running PostgreSQL

```bash
docker run -d \
  --name postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=myuser \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_DB=mydb \
  -v postgres-data:/var/lib/postgresql/data \
  --restart unless-stopped \
  postgres:16-alpine

# Connect with psql
docker exec -it postgres psql -U myuser -d mydb
```

### Running Redis

```bash
docker run -d \
  --name redis \
  -p 6379:6379 \
  -v redis-data:/data \
  --restart unless-stopped \
  redis:7-alpine redis-server --appendonly yes

# Connect with redis-cli
docker exec -it redis redis-cli
```

### Port Mapping Details

```bash
# Map host port 8080 to container port 80
docker run -p 8080:80 nginx

# Map to a specific host interface (localhost only)
docker run -p 127.0.0.1:8080:80 nginx

# Randomly assign host port
docker run -p 80 nginx       # Docker picks the host port
docker port <container>      # Find what port was assigned

# Map multiple ports
docker run -p 80:80 -p 443:443 nginx

# Map UDP port
docker run -p 5353:5353/udp dns-server
```

## System-Level Commands

```bash
# Show disk usage by Docker (images, containers, volumes, build cache)
docker system df

# Verbose disk usage
docker system df -v

# Remove all stopped containers, unused networks,
# dangling images, and build cache
docker system prune

# Also remove all unused images (not just dangling)
docker system prune -a

# Also remove volumes (DANGEROUS — data loss possible)
docker system prune -a --volumes

# Show Docker system information
docker info

# Version information
docker version
```

:::caution[Data Loss Risk]
`docker system prune -a --volumes` will delete **all** data stored in Docker volumes that aren't attached to a running container. Always verify your data is backed up before running this command in production.
:::

## Copying Files

```bash
# Copy a file from host into a running container
docker cp ./config.json my-container:/app/config.json

# Copy a file from container to host
docker cp my-container:/app/output.log ./output.log

# Copy a directory
docker cp ./src my-container:/app/src
```

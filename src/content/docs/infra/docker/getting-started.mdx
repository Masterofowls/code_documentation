# Getting Started with Docker

This guide walks through installing Docker and covers the essential CLI commands you'll use every day.

## Installing Docker

<Tabs>
  <TabItem label="macOS">
    Download Docker Desktop from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop).
    The installer includes Docker Engine, Docker CLI, Docker Compose, and BuildKit.

    ```bash
    # Verify installation
    docker --version
    docker compose version
    ```
  </TabItem>
  <TabItem label="Windows">
    Download Docker Desktop for Windows. Ensure WSL 2 is enabled for best performance.

    ```bash
    # In PowerShell or WSL 2 terminal
    docker --version
    docker compose version
    ```
  </TabItem>
  <TabItem label="Ubuntu / Debian">
    ```bash
    # Add Docker's official GPG key and repository
    sudo apt-get update
    sudo apt-get install ca-certificates curl
    sudo install -m 0755 -d /etc/apt/keyrings
    sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
      -o /etc/apt/keyrings/docker.asc
    sudo chmod a+r /etc/apt/keyrings/docker.asc

    echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
      https://download.docker.com/linux/ubuntu \
      $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
      sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io \
      docker-buildx-plugin docker-compose-plugin

    # Run without sudo (log out and back in after)
    sudo usermod -aG docker $USER
    ```
  </TabItem>
  <TabItem label="RHEL / Fedora">
    ```bash
    sudo dnf -y install dnf-plugins-core
    sudo dnf config-manager --add-repo \
      https://download.docker.com/linux/fedora/docker-ce.repo
    sudo dnf install docker-ce docker-ce-cli containerd.io \
      docker-buildx-plugin docker-compose-plugin
    sudo systemctl enable --now docker
    sudo usermod -aG docker $USER
    ```
  </TabItem>
</Tabs>

## Your First Container

```bash
docker run hello-world
```

Docker pulls the `hello-world` image from Docker Hub, creates a container, runs it, and prints a confirmation message. This verifies the full Docker installation is working.

## Pulling and Listing Images

```bash
# Pull an image from Docker Hub without running it
docker pull nginx:1.27-alpine

# Pull a specific digest (immutable reference)
docker pull nginx@sha256:a5127daff3d6f4606be3100a252419bfa84fd6ee5cd74d0feaca1a5068f97dcf

# List all locally cached images
docker images

# Same command, alternative syntax
docker image ls

# Show image IDs only
docker images -q

# Filter by reference
docker images nginx
```

## Running Containers

### Basic Run

```bash
# Run a container and remove it when it exits
docker run --rm nginx:alpine

# Run in detached (background) mode
docker run -d nginx:alpine

# Run with a human-readable name
docker run -d --name my-nginx nginx:alpine

# Run interactively with a pseudo-TTY (great for shells)
docker run -it ubuntu:24.04 bash
```

### Port Mapping (`-p`)

```bash
# Map host port 8080 to container port 80
docker run -d -p 8080:80 --name web nginx:alpine

# Map a specific host IP
docker run -d -p 127.0.0.1:8080:80 nginx:alpine

# Map multiple ports
docker run -d -p 8080:80 -p 8443:443 nginx:alpine

# Let Docker assign a random host port
docker run -d -p 80 nginx:alpine
docker port web   # see the assigned port
```

Now visit `http://localhost:8080` to see the Nginx welcome page.

### Environment Variables (`-e`)

```bash
# Single variable
docker run -d -e POSTGRES_PASSWORD=secret postgres:16-alpine

# Multiple variables
docker run -d \
  -e POSTGRES_DB=myapp \
  -e POSTGRES_USER=app \
  -e POSTGRES_PASSWORD=secret \
  --name postgres \
  -p 5432:5432 \
  postgres:16-alpine

# Load from a file
docker run -d --env-file ./.env postgres:16-alpine
```

### Volume Mounts (`-v`)

```bash
# Named volume (Docker-managed)
docker run -d -v pgdata:/var/lib/postgresql/data postgres:16-alpine

# Bind mount (host directory)
docker run -d -v $(pwd)/html:/usr/share/nginx/html:ro nginx:alpine

# tmpfs (in-memory, not persisted)
docker run -d --tmpfs /tmp:noexec,nosuid,size=100m nginx:alpine
```

## Common Run Examples

### Nginx

```bash
docker run -d \
  --name nginx \
  -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  --restart unless-stopped \
  nginx:1.27-alpine
```

### PostgreSQL

```bash
docker run -d \
  --name postgres \
  -e POSTGRES_DB=myapp \
  -e POSTGRES_USER=app \
  -e POSTGRES_PASSWORD=supersecret \
  -v pgdata:/var/lib/postgresql/data \
  -p 5432:5432 \
  --restart unless-stopped \
  postgres:16-alpine
```

### Redis

```bash
docker run -d \
  --name redis \
  -p 6379:6379 \
  -v redisdata:/data \
  --restart unless-stopped \
  redis:7-alpine redis-server --appendonly yes
```

## Listing and Inspecting Containers

```bash
# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# Show only container IDs
docker ps -q

# Format output
docker ps --format "table {{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"
```

## Executing Commands in Running Containers

```bash
# Open an interactive shell in a running container
docker exec -it my-nginx sh

# Run a one-off command
docker exec my-nginx nginx -t

# Run as a specific user
docker exec -u root -it my-nginx sh

# Set environment variables for the exec session
docker exec -e DEBUG=true -it my-app sh
```

:::tip[Alpine images use `sh`, not `bash`]
Alpine-based images do not include `bash` by default. Use `sh` or `ash`. Install bash with `apk add bash` if needed.
:::

## Viewing Logs

```bash
# Print all logs
docker logs my-nginx

# Follow (tail -f equivalent)
docker logs -f my-nginx

# Show last 50 lines
docker logs --tail 50 my-nginx

# Show logs with timestamps
docker logs -t my-nginx

# Combine: follow with timestamps since 1 hour ago
docker logs -f -t --since 1h my-nginx
```

## Stopping, Starting, and Removing Containers

```bash
# Gracefully stop (sends SIGTERM, waits 10s, then SIGKILL)
docker stop my-nginx

# Stop with custom timeout (seconds)
docker stop -t 30 my-nginx

# Start a stopped container
docker start my-nginx

# Restart a running container
docker restart my-nginx

# Kill immediately (SIGKILL)
docker kill my-nginx

# Remove a stopped container
docker rm my-nginx

# Remove a running container (force)
docker rm -f my-nginx

# Remove all stopped containers
docker container prune

# Stop and remove in one step
docker rm -f $(docker ps -aq)
```

## Inspecting Containers

```bash
# Full JSON metadata
docker inspect my-nginx

# Extract specific fields with Go template
docker inspect -f '{{.NetworkSettings.IPAddress}}' my-nginx
docker inspect -f '{{.State.Status}}' my-nginx
docker inspect -f '{{.HostConfig.RestartPolicy.Name}}' my-nginx

# Inspect an image
docker inspect nginx:alpine
```

## Resource Usage

```bash
# Live resource stats (CPU, memory, network, disk I/O)
docker stats

# Stats for specific containers
docker stats my-nginx postgres redis

# Single snapshot (no live stream)
docker stats --no-stream

# Custom format
docker stats --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"
```

## Disk Usage

```bash
# Show Docker disk usage summary
docker system df

# Verbose — show individual items
docker system df -v

# Example output:
# TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
# Images          12        3         2.1GB     1.4GB (66%)
# Containers      4         2         128MB     0B (0%)
# Local Volumes   6         2         890MB     450MB (50%)
# Build Cache     23        0         310MB     310MB
```

## Cleaning Up

```bash
# Remove stopped containers, unused networks, dangling images, build cache
docker system prune

# Also remove unused volumes (CAUTION: data loss possible)
docker system prune --volumes

# Remove all unused images (not just dangling)
docker system prune -a

# Remove dangling images only
docker image prune

# Remove all unused images
docker image prune -a

# Remove unused volumes
docker volume prune

# Remove unused networks
docker network prune
```

:::caution[`system prune --volumes` removes data]
Always verify volume contents before running `docker system prune --volumes`. This permanently deletes any volumes not currently mounted by a container.
:::

## Copying Files

```bash
# Copy from container to host
docker cp my-nginx:/etc/nginx/nginx.conf ./nginx.conf

# Copy from host to container
docker cp ./nginx.conf my-nginx:/etc/nginx/nginx.conf
```

## Building Images

```bash
# Build from Dockerfile in current directory
docker build -t myapp:1.0 .

# Specify Dockerfile path
docker build -f docker/Dockerfile.prod -t myapp:prod .

# Pass build arguments
docker build --build-arg NODE_ENV=production -t myapp:prod .

# No cache
docker build --no-cache -t myapp:latest .
```

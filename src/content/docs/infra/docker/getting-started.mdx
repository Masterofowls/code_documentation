# Getting Started with Docker

This guide walks you through installing Docker, running your first container, and learning the CLI commands you'll use every day.

## Installing Docker

<Tabs>
  <TabItem label="macOS">
    Download and install **Docker Desktop for Mac** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    Alternatively, install via Homebrew:

    ```bash
    brew install --cask docker
    ```

    Open Docker Desktop from your Applications folder. The Docker whale icon in the menu bar confirms the daemon is running.
  </TabItem>
  <TabItem label="Windows">
    Download **Docker Desktop for Windows** from [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/).

    **Prerequisites:**
    - Windows 10/11 64-bit with WSL 2 backend (recommended) or Hyper-V
    - Enable WSL 2: `wsl --install` in an elevated PowerShell terminal

    After installation, Docker Desktop starts automatically and shows a tutorial on first launch.
  </TabItem>
  <TabItem label="Linux">
    Install Docker Engine directly on Linux (no VM layer required):

    ```bash
    # Remove old versions
    sudo apt-get remove docker docker-engine docker.io containerd runc

    # Add Docker's official GPG key and repository
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
      sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

    echo "deb [arch=$(dpkg --print-architecture) \
      signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
      https://download.docker.com/linux/ubuntu \
      $(lsb_release -cs) stable" | \
      sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    # Install
    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

    # Add your user to the docker group (avoids sudo)
    sudo usermod -aG docker $USER
    newgrp docker
    ```
  </TabItem>
</Tabs>

## Verify the Installation

```bash
docker --version
# Docker version 27.x.x, build xxxxxxx

docker info
# Shows daemon status, number of containers, images, storage driver, etc.
```

## Your First Container

```bash
docker run hello-world
```

Docker will:
1. Look for the `hello-world` image locally — not found
2. Pull it from Docker Hub (`docker.io/library/hello-world:latest`)
3. Create a container from the image
4. Run the container, which prints a confirmation message and exits

:::tip[What Just Happened?]
The `hello-world` image contains a single executable that prints a message and exits. The container lifecycle completes in under a second. This is the simplest proof that Docker is working correctly.
:::

## Pulling Images

```bash
# Pull an image without running it
docker pull nginx:1.27-alpine

# Pull a specific digest (immutable reference)
docker pull nginx@sha256:a5127daff3d...

# Pull from a non-Hub registry
docker pull ghcr.io/myorg/myapp:v1.2.0
```

## Listing Images

```bash
# List all local images
docker images

# Same as above (long form)
docker image ls

# Show all images including intermediates
docker image ls -a

# Filter by repository name
docker images nginx

# Show only image IDs
docker images -q

# Show image sizes sorted
docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"
```

## Running Containers

The `docker run` command is the workhorse of the Docker CLI. Here are the most important flags:

```bash
# Detached mode — runs in background, returns container ID
docker run -d nginx:alpine

# Name the container for easier reference
docker run -d --name my-nginx nginx:alpine

# Publish a port: host-port:container-port
docker run -d -p 8080:80 --name web nginx:alpine

# Set environment variables
docker run -d -e POSTGRES_PASSWORD=secret -e POSTGRES_DB=mydb postgres:16-alpine

# Remove container automatically when it stops
docker run --rm alpine echo "hello"

# Interactive terminal — useful for debugging
docker run -it ubuntu:24.04 bash

# Mount a volume: host-path:container-path
docker run -d -v /host/data:/var/lib/postgresql/data postgres:16-alpine

# Named volume
docker run -d -v pgdata:/var/lib/postgresql/data postgres:16-alpine

# Read-only bind mount
docker run -d -v $(pwd)/config:/etc/nginx/conf.d:ro nginx:alpine

# Set resource limits
docker run -d --memory="512m" --cpus="0.5" nginx:alpine

# Set restart policy
docker run -d --restart unless-stopped nginx:alpine

# Pass multiple environment variables from a file
docker run -d --env-file .env myapp:latest
```

## Listing Containers

```bash
# Running containers only
docker ps

# All containers (including stopped)
docker ps -a

# Show only container IDs
docker ps -q

# Custom format
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# Filter by status
docker ps -f status=exited
docker ps -f name=web
```

## Interacting with Running Containers

```bash
# Execute a command in a running container
docker exec my-nginx ls /etc/nginx

# Open an interactive shell in a running container
docker exec -it my-nginx sh

# Run as a specific user
docker exec -it --user root my-nginx bash

# Set environment variables for the exec session
docker exec -it -e DEBUG=1 my-nginx bash
```

## Viewing Logs

```bash
# Print logs and exit
docker logs my-nginx

# Follow log output (like tail -f)
docker logs -f my-nginx

# Show last 50 lines
docker logs --tail 50 my-nginx

# Show logs with timestamps
docker logs -t my-nginx

# Since a specific time
docker logs --since 2024-01-15T10:00:00 my-nginx

# Combine: follow last 20 lines with timestamps
docker logs -f --tail 20 -t my-nginx
```

## Managing Container Lifecycle

```bash
# Stop a container gracefully (SIGTERM, then SIGKILL after 10s)
docker stop my-nginx

# Stop with custom timeout
docker stop --time 30 my-nginx

# Start a stopped container
docker start my-nginx

# Restart a container
docker restart my-nginx

# Kill immediately (SIGKILL)
docker kill my-nginx

# Remove a stopped container
docker rm my-nginx

# Force remove a running container
docker rm -f my-nginx

# Remove all stopped containers
docker container prune

# Stop and remove in one step (only works if stopped)
docker rm $(docker ps -aq -f status=exited)
```

## Inspecting Containers and Images

```bash
# Full JSON metadata of a container
docker inspect my-nginx

# Extract a specific field with Go template
docker inspect my-nginx --format '{{ .NetworkSettings.IPAddress }}'
docker inspect my-nginx --format '{{ .HostConfig.RestartPolicy.Name }}'

# Inspect an image
docker inspect nginx:alpine

# Show image layers and history
docker history nginx:alpine

# Show real-time resource usage
docker stats

# Stats for a specific container
docker stats my-nginx

# Stats snapshot (no stream)
docker stats --no-stream
```

## Disk Usage

```bash
# Show Docker disk usage summary
docker system df

# Verbose breakdown by object
docker system df -v
```

## Cleaning Up

```bash
# Remove all stopped containers, unused networks, dangling images, and build cache
docker system prune

# Also remove unused volumes (be careful!)
docker system prune --volumes

# Remove all unused images, not just dangling
docker system prune -a

# Remove dangling images only
docker image prune

# Remove all stopped containers
docker container prune

# Remove unused volumes
docker volume prune

# Remove unused networks
docker network prune
```

:::caution[Prune is Destructive]
`docker system prune -a --volumes` will remove everything not currently used by a running container. Do not run this on a production host or you may lose important data and images.
:::

## Practical Examples

### Running Nginx

```bash
# Serve static files from current directory
docker run -d \
  --name web \
  -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  --restart unless-stopped \
  nginx:1.27-alpine

# Test it
curl http://localhost:8080
```

### Running PostgreSQL

```bash
docker run -d \
  --name postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=supersecret \
  -e POSTGRES_DB=myapp \
  -v pgdata:/var/lib/postgresql/data \
  --restart unless-stopped \
  postgres:16-alpine

# Connect with psql client inside the container
docker exec -it postgres psql -U admin -d myapp
```

### Running Redis

```bash
docker run -d \
  --name redis \
  -p 6379:6379 \
  -v redisdata:/data \
  --restart unless-stopped \
  redis:7-alpine redis-server --appendonly yes

# Connect with redis-cli
docker exec -it redis redis-cli ping
# PONG
```

## Port Mapping

Docker port mapping (`-p`) connects a host port to a container port:

```
-p [host-ip:]host-port:container-port[/protocol]
```

```bash
# Basic mapping — listen on all interfaces
docker run -p 8080:80 nginx

# Bind to localhost only (more secure)
docker run -p 127.0.0.1:8080:80 nginx

# UDP port
docker run -p 5353:53/udp my-dns-server

# Map multiple ports
docker run -p 80:80 -p 443:443 nginx

# Let Docker choose a random host port
docker run -p 80 nginx
docker port container-name  # Show the assigned port
```

:::tip[localhost vs 0.0.0.0]
Binding to `127.0.0.1:8080:80` means only processes on the host machine can reach the container. Binding to `0.0.0.0:8080:80` (the default) means any network interface can reach it, including external traffic. Prefer `127.0.0.1` for development databases.
:::

## Environment Variables

Environment variables are the standard way to configure containers at runtime:

```bash
# Pass individual variables
docker run -e DATABASE_URL=postgres://... -e NODE_ENV=production myapp

# Load from a .env file
docker run --env-file ./production.env myapp
```

A `.env` file format:

```bash
NODE_ENV=production
DATABASE_URL=postgres://user:pass@db:5432/myapp
REDIS_URL=redis://redis:6379
PORT=3000
```

:::caution[Secrets in Environment Variables]
Environment variables are visible in `docker inspect` output and in `/proc/<pid>/environ` on the host. For sensitive credentials in production, prefer Docker secrets or a secrets manager. See the [Production guide](/infra/docker/production) for details.
:::

## Copying Files

```bash
# Copy a file from host to container
docker cp ./config.json my-container:/app/config.json

# Copy a file from container to host
docker cp my-container:/app/logs/app.log ./app.log

# Copy a directory
docker cp ./dist/ my-container:/app/dist/
```

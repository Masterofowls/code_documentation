# Docker in Production

Running Docker in production requires a different mindset from development. This guide covers the hardening, observability, and reliability practices that separate production-ready containers from development ones.

## Production Dockerfile Best Practices

```dockerfile
# syntax=docker/dockerfile:1

# ── 1. Pin base image to digest (not just tag) ─────────────────
FROM node:20.11.1-alpine3.19@sha256:a1b2c3...
# Falls back to tag pinning if digest not feasible:
FROM node:20.11.1-alpine3.19

# ── 2. Set build metadata ──────────────────────────────────────
LABEL org.opencontainers.image.source="https://github.com/org/repo"
LABEL org.opencontainers.image.revision="${GIT_SHA}"
LABEL org.opencontainers.image.version="${VERSION}"

# ── 3. Non-root user ───────────────────────────────────────────
RUN addgroup --system --gid 1001 appgroup && \
    adduser  --system --uid 1001 --ingroup appgroup --no-create-home appuser

WORKDIR /app

# ── 4. Install deps with reproducible lockfile ─────────────────
COPY package.json package-lock.json ./
RUN npm ci --only=production && npm cache clean --force

# ── 5. Copy application files with correct ownership ──────────
COPY --chown=appuser:appgroup . .

# ── 6. Switch to non-root ──────────────────────────────────────
USER appuser

# ── 7. No secrets in ENV / COPY ────────────────────────────────
# Use runtime env vars, Docker secrets, or secret mounts

# ── 8. Health check ────────────────────────────────────────────
HEALTHCHECK --interval=30s \
            --timeout=10s \
            --start-period=60s \
            --retries=3 \
  CMD wget -qO- http://localhost:3000/health || exit 1

# ── 9. Signal handling (tini as PID 1) ─────────────────────────
RUN apk add --no-cache tini
ENTRYPOINT ["/sbin/tini", "--"]

# ── 10. Minimal, explicit startup command ──────────────────────
CMD ["node", "dist/server.js"]

# ── 11. Correct stop signal ────────────────────────────────────
STOPSIGNAL SIGTERM
```

## Resource Limits

Without resource limits, a runaway container can starve other containers and destabilise the host.

```bash
# Memory limit (container OOM-killed if exceeded)
docker run -d --memory="512m" myapp

# Soft memory limit (container can exceed this but prefers to stay below)
docker run -d --memory="512m" --memory-reservation="256m" myapp

# CPU limit (1.5 = 1.5 CPU cores)
docker run -d --cpus="1.5" myapp

# CPU shares (relative weight — default 1024)
docker run -d --cpu-shares=512 myapp       # half the weight of default

# PID limit (prevent fork bombs)
docker run -d --pids-limit=100 myapp

# Swap (set equal to memory to disable swap)
docker run -d --memory="512m" --memory-swap="512m" myapp

# OOM kill score adjustment
docker run -d --oom-score-adj=200 myapp   # more likely to be killed under pressure

# Combined limits
docker run -d \
  --memory="512m" \
  --memory-reservation="256m" \
  --cpus="1.0" \
  --pids-limit=200 \
  --restart unless-stopped \
  myapp:latest
```

```yaml
# Compose with resource limits
services:
  app:
    image: myapp:latest
    deploy:
      resources:
        limits:
          cpus: '1.0'
          memory: 512M
          pids: 200
        reservations:
          cpus: '0.25'
          memory: 128M
```

:::tip[Memory OOM Kills]
When a container exceeds its memory limit, the Linux kernel OOM killer terminates the container process. Set up proper alerting on OOM kills — they're often silent. Check with `docker inspect --format '{{.State.OOMKilled}}'` or `docker stats`.
:::

## Logging Drivers

```bash
# Default: json-file (stored on host disk)
docker run -d \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=5 \
  myapp

# Syslog
docker run -d \
  --log-driver syslog \
  --log-opt syslog-address=tcp://syslog-server:514 \
  --log-opt tag="myapp-{{.Name}}" \
  myapp

# Fluentd / Fluent Bit
docker run -d \
  --log-driver fluentd \
  --log-opt fluentd-address=localhost:24224 \
  --log-opt tag="docker.{{.Name}}" \
  myapp

# AWS CloudWatch
docker run -d \
  --log-driver=awslogs \
  --log-opt awslogs-region=us-east-1 \
  --log-opt awslogs-group=/myapp/prod \
  --log-opt awslogs-stream=app-container \
  --log-opt awslogs-create-group=true \
  myapp

# Splunk
docker run -d \
  --log-driver=splunk \
  --log-opt splunk-token=YOUR-TOKEN \
  --log-opt splunk-url=https://splunk.example.com:8088 \
  myapp

# none — disable logging (for high-throughput services that self-log)
docker run -d --log-driver none myapp
```

## Restart Policies

```bash
# No restart (default)
docker run --restart no myapp

# Always restart (even on explicit docker stop — starts on daemon restart)
docker run --restart always myapp

# Unless explicitly stopped (best for long-running services)
docker run --restart unless-stopped myapp

# On failure only, max 3 retries
docker run --restart on-failure:3 myapp

# Production recommendation
docker run -d \
  --restart unless-stopped \
  --name myapp \
  myapp:latest
```

```yaml
# Compose restart policies
services:
  app:
    restart: unless-stopped # recommended for services

  worker:
    restart: on-failure # restart on crash, not on explicit stop

  migration:
    restart: 'no' # run once and stop
```

## Health Checks

### Dockerfile HEALTHCHECK

```dockerfile
# HTTP endpoint health check
HEALTHCHECK --interval=30s \
            --timeout=10s \
            --start-period=60s \
            --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1

# Using wget (smaller footprint than curl)
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1

# TCP check (no HTTP required)
HEALTHCHECK --interval=15s --timeout=5s \
  CMD nc -z localhost 5432 || exit 1

# Native application health check (no external tools)
HEALTHCHECK --interval=30s --timeout=10s \
  CMD node -e "require('http').get('http://localhost:3000/health', r => process.exit(r.statusCode === 200 ? 0 : 1)).on('error', () => process.exit(1))"

# Disable inherited health check
HEALTHCHECK NONE
```

### Compose Health Check Override

```yaml
services:
  app:
    healthcheck:
      test: ['CMD', 'wget', '-qO-', 'http://localhost:3000/health']
      interval: 30s
      timeout: 10s
      start_period: 60s
      retries: 3

  postgres:
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}']
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

  redis:
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 3s
      retries: 5
```

```bash
# Check container health status
docker inspect --format '{{.State.Health.Status}}' myapp
# starting | healthy | unhealthy

# View health check history
docker inspect --format '{{json .State.Health}}' myapp | python3 -m json.tool

# Wait for healthy status
until [ "$(docker inspect --format '{{.State.Health.Status}}' myapp)" = "healthy" ]; do
  echo "Waiting for container to be healthy..."
  sleep 5
done
```

## Docker Secrets

### Compose Secrets

```yaml
# compose.yml
services:
  app:
    image: myapp:latest
    secrets:
      - db_password
      - api_key
    environment:
      DB_PASSWORD_FILE: /run/secrets/db_password
      API_KEY_FILE: /run/secrets/api_key

  db:
    image: postgres:16
    secrets:
      - db_password
    environment:
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password

secrets:
  db_password:
    file: ./secrets/db_password.txt # file on host

  api_key:
    environment: API_KEY # from environment variable
```

```bash
# Create secrets directory (never commit to git)
mkdir -p secrets
echo "supersecretpassword" > secrets/db_password.txt
chmod 600 secrets/db_password.txt
echo "secrets/" >> .gitignore

# Start
docker compose up -d

# In the container, secrets are available at /run/secrets/
docker exec app cat /run/secrets/db_password
```

### Reading Secrets in Application Code

```javascript
// Node.js — read secret from file
const fs = require('fs');

function getSecret(secretName) {
  const secretFile = process.env[`${secretName}_FILE`];
  if (secretFile) {
    return fs.readFileSync(secretFile, 'utf8').trim();
  }
  return process.env[secretName]; // fallback to env var
}

const dbPassword = getSecret('DB_PASSWORD');
```

```python
# Python
import os

def get_secret(name: str) -> str:
    file_var = os.environ.get(f"{name}_FILE")
    if file_var:
        with open(file_var) as f:
            return f.read().strip()
    return os.environ[name]

db_password = get_secret("DB_PASSWORD")
```

## Container Security Hardening

```bash
# Drop all Linux capabilities, add back only what's needed
docker run -d \
  --cap-drop ALL \
  --cap-add NET_BIND_SERVICE \
  --cap-add CHOWN \
  myapp

# Read-only root filesystem
docker run -d \
  --read-only \
  --tmpfs /tmp:rw,noexec,nosuid,size=64m \
  --tmpfs /var/cache:rw,noexec \
  myapp

# Prevent privilege escalation
docker run -d \
  --security-opt no-new-privileges:true \
  myapp

# Custom seccomp profile
docker run -d \
  --security-opt seccomp=/path/to/custom-seccomp.json \
  myapp

# AppArmor profile
docker run -d \
  --security-opt apparmor=docker-default \
  myapp

# Combine all hardening options
docker run -d \
  --name myapp \
  --cap-drop ALL \
  --cap-add NET_BIND_SERVICE \
  --read-only \
  --tmpfs /tmp \
  --security-opt no-new-privileges:true \
  --security-opt seccomp=/etc/docker/seccomp-default.json \
  --user 1001:1001 \
  --memory="512m" \
  --cpus="1.0" \
  --pids-limit=200 \
  --restart unless-stopped \
  myapp:latest
```

```yaml
# Compose security options
services:
  app:
    cap_drop:
      - ALL
    cap_add:
      - NET_BIND_SERVICE
    read_only: true
    tmpfs:
      - /tmp:rw,noexec,nosuid,size=65536k
    security_opt:
      - no-new-privileges:true
      - seccomp:/etc/docker/seccomp-default.json
    user: '1001:1001'
```

### Linux Capabilities Reference

| Capability          | Purpose                       | Needed For                 |
| ------------------- | ----------------------------- | -------------------------- |
| `NET_BIND_SERVICE`  | Bind to ports < 1024          | Web servers on port 80/443 |
| `CHOWN`             | Change file ownership         | App setup scripts          |
| `DAC_OVERRIDE`      | Bypass file permission checks | Rarely needed              |
| `SETUID` / `SETGID` | Change user/group ID          | Privilege dropping         |
| `SYS_PTRACE`        | Process tracing               | Debuggers, profilers       |
| `NET_ADMIN`         | Network configuration         | VPN, custom routing        |

:::caution[Read-Only Filesystem]
When using `--read-only`, your application must not write to the root filesystem. Use `--tmpfs` for directories that need temporary writes (`/tmp`, `/var/run`, `/var/cache`). Any writes to other paths will fail with "Read-only file system".
:::

## Graceful Shutdown

Containers receive `SIGTERM` when stopped. Your application must handle it properly to avoid data loss or dropped connections.

```bash
# Docker stop — sends SIGTERM, waits (default 10s), then SIGKILL
docker stop myapp

# Custom grace period (seconds)
docker stop --time 30 myapp

# Compose stop grace period
```

```yaml
services:
  app:
    stop_grace_period: 30s # how long before SIGKILL
    stop_signal: SIGTERM
```

### Node.js Graceful Shutdown

```javascript
const server = app.listen(3000);

function gracefulShutdown(signal) {
  console.log(`Received ${signal}, shutting down gracefully`);

  server.close(async () => {
    console.log('HTTP server closed');
    await db.disconnect();
    await redis.quit();
    console.log('All connections closed. Exiting.');
    process.exit(0);
  });

  // Force exit after grace period
  setTimeout(() => {
    console.error('Graceful shutdown timed out, forcing exit');
    process.exit(1);
  }, 25000);
}

process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
process.on('SIGINT', () => gracefulShutdown('SIGINT'));
```

### Python Graceful Shutdown (FastAPI/uvicorn)

```python
import signal
import sys
import asyncio

async def shutdown(loop, signal_type=None):
    if signal_type:
        print(f"Received exit signal {signal_type.name}")

    tasks = [t for t in asyncio.all_tasks() if t is not asyncio.current_task()]
    for task in tasks:
        task.cancel()

    await asyncio.gather(*tasks, return_exceptions=True)
    loop.stop()

loop = asyncio.get_event_loop()
for sig in (signal.SIGTERM, signal.SIGINT):
    loop.add_signal_handler(sig, lambda s=sig: asyncio.create_task(shutdown(loop, s)))
```

## Production Checklist

```bash
# Before deploying to production, verify:

# 1. Image is pinned to a specific tag or digest
docker inspect myapp:latest | grep -i "repodigest"

# 2. No root process
docker inspect --format '{{.Config.User}}' myapp:latest

# 3. Health check defined
docker inspect --format '{{.Config.Healthcheck}}' myapp:latest

# 4. Resource limits set
docker inspect --format '{{.HostConfig.Memory}}' mycontainer

# 5. Restart policy set
docker inspect --format '{{.HostConfig.RestartPolicy.Name}}' mycontainer

# 6. No capabilities beyond necessary
docker inspect --format '{{.HostConfig.CapAdd}}' mycontainer

# 7. Logging configured
docker inspect --format '{{.HostConfig.LogConfig}}' mycontainer

# 8. No secrets in env vars (or image layers)
docker history myapp:latest | grep -i "password\|secret\|token\|key"
```

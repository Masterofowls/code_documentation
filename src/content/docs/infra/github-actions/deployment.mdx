## GitHub Environments

Environments provide named deployment targets with protection rules, secrets, and approval gates.

### Creating Environments

Navigate to `Repository Settings > Environments > New environment`. You can create:

- **development** — auto-deploy on every push
- **staging** — auto-deploy from main branch
- **production** — requires manual approval + deployment timer

### Environment Protection Rules

```yaml
# Environment: production settings
# Protection rules (configured in GitHub UI):
# ✓ Required reviewers: @team/platform-leads (min 1)
# ✓ Wait timer: 5 minutes (allows cancellation window)
# ✓ Restrict deployments to: refs/heads/main, refs/tags/v*
# ✓ Allow administrators to bypass

# Environment-specific secrets (only accessible when deploying to this env):
# PROD_DATABASE_URL
# PROD_API_KEY
# PROD_K8S_KUBECONFIG
```

### Using Environments in Workflows

```yaml
jobs:
  deploy-production:
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://app.example.com # shown in deployment panel
    steps:
      - uses: actions/checkout@v4
      - run: ./deploy.sh
        env:
          # Secrets from the 'production' environment
          DATABASE_URL: ${{ secrets.PROD_DATABASE_URL }}
          API_KEY: ${{ secrets.PROD_API_KEY }}
```

## Standard Deployment Pipeline Pattern

A production-grade pipeline: build once, promote through environments.

```yaml
name: Deploy Pipeline

on:
  push:
    branches: [main]
  workflow_dispatch:
    inputs:
      skip-staging:
        type: boolean
        default: false

concurrency:
  group: deploy
  cancel-in-progress: false # queue deployments, never cancel

permissions:
  id-token: write
  contents: read
  deployments: write

jobs:
  build:
    name: Build & Test
    runs-on: ubuntu-latest
    timeout-minutes: 20
    outputs:
      image-tag: ${{ steps.meta.outputs.version }}
      image-digest: ${{ steps.build.outputs.digest }}
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci
      - run: npm run lint && npm run test && npm run build

      - name: Set image metadata
        id: meta
        run: echo "version=${{ github.sha }}" >> $GITHUB_OUTPUT

      - uses: actions/upload-artifact@v4
        with:
          name: dist-${{ github.sha }}
          path: dist/

  deploy-staging:
    name: Deploy → Staging
    needs: build
    runs-on: ubuntu-latest
    if: ${{ !inputs.skip-staging }}
    timeout-minutes: 15
    environment:
      name: staging
      url: https://staging.example.com
    steps:
      - uses: actions/checkout@v4

      - uses: actions/download-artifact@v4
        with:
          name: dist-${{ github.sha }}
          path: dist/

      - name: Deploy to staging
        run: ./scripts/deploy.sh staging
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
          DEPLOY_KEY: ${{ secrets.DEPLOY_KEY }}

      - name: Run smoke tests
        run: npm run test:smoke
        env:
          BASE_URL: https://staging.example.com

  deploy-production:
    name: Deploy → Production
    needs: [build, deploy-staging]
    runs-on: ubuntu-latest
    timeout-minutes: 20
    environment:
      name: production
      url: https://app.example.com
    steps:
      - uses: actions/checkout@v4

      - uses: actions/download-artifact@v4
        with:
          name: dist-${{ github.sha }}
          path: dist/

      - name: Deploy to production
        run: ./scripts/deploy.sh production
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
          DEPLOY_KEY: ${{ secrets.DEPLOY_KEY }}

      - name: Verify deployment
        run: |
          sleep 30
          curl --retry 5 --retry-delay 10 \
            -f https://app.example.com/health || exit 1

      - name: Write deployment summary
        run: |
          echo "## 🚀 Production Deployment" >> $GITHUB_STEP_SUMMARY
          echo "- **Version**: ${{ needs.build.outputs.image-tag }}" >> $GITHUB_STEP_SUMMARY
          echo "- **URL**: https://app.example.com" >> $GITHUB_STEP_SUMMARY
          echo "- **Deployed by**: ${{ github.actor }}" >> $GITHUB_STEP_SUMMARY
```

:::tip[Build Once, Deploy Everywhere]
Always build your artifact (Docker image, static files, binary) in a single job, upload it as an artifact or Docker image, and then download/pull it in each deployment job. Never rebuild in deployment jobs — you risk deploying a different artifact than what was tested.
:::

## Deploy to Vercel

<Tabs>
<TabItem label="Vercel CLI (Recommended)">

```yaml
jobs:
  deploy-vercel:
    runs-on: ubuntu-latest
    environment:
      name: production
      url: ${{ steps.deploy.outputs.url }}
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Vercel CLI
        run: npm install --global vercel@latest

      - name: Pull Vercel Environment
        run: vercel pull --yes --environment=production --token=${{ secrets.VERCEL_TOKEN }}

      - name: Build Project
        run: vercel build --prod --token=${{ secrets.VERCEL_TOKEN }}

      - name: Deploy to Vercel
        id: deploy
        run: |
          URL=$(vercel deploy --prebuilt --prod --token=${{ secrets.VERCEL_TOKEN }})
          echo "url=$URL" >> $GITHUB_OUTPUT
```

## Deploy to AWS

### S3 + CloudFront (Static Sites)

```yaml
jobs:
  deploy-s3:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: read
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build

      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_ROLE_ARN }}
          aws-region: us-east-1

      - name: Sync to S3
        run: |
          # Upload immutable assets with long cache
          aws s3 sync dist/_next/static s3://${{ vars.S3_BUCKET }}/_next/static \
            --cache-control "public, max-age=31536000, immutable" \
            --delete

          # Upload HTML with short cache
          aws s3 sync dist s3://${{ vars.S3_BUCKET }} \
            --cache-control "public, max-age=0, must-revalidate" \
            --delete \
            --exclude "_next/static/*"

      - name: Invalidate CloudFront
        run: |
          aws cloudfront create-invalidation \
            --distribution-id ${{ vars.CF_DISTRIBUTION_ID }} \
            --paths "/*"
```

### ECS (Container Deployments)

```yaml
jobs:
  deploy-ecs:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_ROLE_ARN }}
          aws-region: us-east-1

      - uses: aws-actions/amazon-ecr-login@v2
        id: login-ecr

      - name: Build and push to ECR
        id: build
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/my-app:$IMAGE_TAG .
          docker push $ECR_REGISTRY/my-app:$IMAGE_TAG
          echo "image=$ECR_REGISTRY/my-app:$IMAGE_TAG" >> $GITHUB_OUTPUT

      - name: Download task definition
        run: |
          aws ecs describe-task-definition \
            --task-definition my-app \
            --query taskDefinition > task-def.json

      - name: Update ECS task definition with new image
        id: task-def
        uses: aws-actions/amazon-ecs-render-task-definition@v1
        with:
          task-definition: task-def.json
          container-name: my-app
          image: ${{ steps.build.outputs.image }}

      - name: Deploy to ECS
        uses: aws-actions/amazon-ecs-deploy-task-definition@v2
        with:
          task-definition: ${{ steps.task-def.outputs.task-definition }}
          service: my-app-service
          cluster: production
          wait-for-service-stability: true
```

### Lambda Deployment

```yaml
jobs:
  deploy-lambda:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build

      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_ROLE_ARN }}
          aws-region: us-east-1

      - name: Package Lambda
        run: |
          cd dist
          zip -r ../function.zip .
          cd ..

      - name: Update Lambda function
        run: |
          aws lambda update-function-code \
            --function-name my-function \
            --zip-file fileb://function.zip \
            --publish

          aws lambda wait function-updated \
            --function-name my-function
```

## Deploy to GCP Cloud Run

```yaml
jobs:
  deploy-cloudrun:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.GCP_WORKLOAD_IDENTITY_PROVIDER }}
          service_account: ${{ vars.GCP_SERVICE_ACCOUNT }}

      - uses: google-github-actions/setup-gcloud@v2

      - name: Build and push to Artifact Registry
        run: |
          IMAGE="us-central1-docker.pkg.dev/${{ vars.GCP_PROJECT }}/my-app/api:${{ github.sha }}"
          gcloud builds submit --tag "$IMAGE" .
          echo "IMAGE=$IMAGE" >> $GITHUB_ENV

      - uses: google-github-actions/deploy-cloudrun@v2
        with:
          service: my-api
          region: us-central1
          image: ${{ env.IMAGE }}
          flags: '--min-instances=1 --max-instances=10 --memory=512Mi'
```

## Deploy to Kubernetes

```yaml
jobs:
  deploy-k8s:
    runs-on: ubuntu-latest
    environment: production
    steps:
      - uses: actions/checkout@v4

      - name: Configure kubectl
        run: |
          echo "${{ secrets.KUBECONFIG }}" | base64 -d > kubeconfig.yaml
          echo "KUBECONFIG=$PWD/kubeconfig.yaml" >> $GITHUB_ENV

      - name: Update image in manifests
        run: |
          sed -i "s|IMAGE_TAG|${{ github.sha }}|g" k8s/deployment.yaml

      - name: Apply manifests
        run: |
          kubectl apply -f k8s/
          kubectl rollout status deployment/my-app -n production --timeout=300s

      - name: Verify deployment
        run: |
          kubectl get pods -n production -l app=my-app
          kubectl rollout history deployment/my-app -n production
```

## Release Automation

### `softprops/action-gh-release` (Recommended)

```yaml
name: Release

on:
  push:
    tags: ['v*']

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0 # needed for changelog generation

      - run: npm ci && npm run build

      - name: Create release archives
        run: |
          tar -czf app-linux-amd64.tar.gz -C dist .
          zip -r app-windows-amd64.zip dist/

      - uses: softprops/action-gh-release@v2
        with:
          generate_release_notes: true # auto-generate from PR titles
          draft: false
          prerelease: ${{ contains(github.ref_name, '-rc') || contains(github.ref_name, '-beta') }}
          files: |
            app-linux-amd64.tar.gz
            app-windows-amd64.zip
          body: |
            ## Installation
            Download the appropriate binary for your platform.

            ## Changelog
            See [CHANGELOG.md](CHANGELOG.md) for details.
```

### `release-please` (Automated Versioning)

```yaml
# .github/workflows/release-please.yml
name: Release Please

on:
  push:
    branches: [main]

permissions:
  contents: write
  pull-requests: write

jobs:
  release-please:
    runs-on: ubuntu-latest
    outputs:
      release_created: ${{ steps.release.outputs.release_created }}
      tag_name: ${{ steps.release.outputs.tag_name }}
    steps:
      - uses: googleapis/release-please-action@v4
        id: release
        with:
          release-type: node # reads version from package.json
          package-name: my-app

  deploy-on-release:
    needs: release-please
    if: ${{ needs.release-please.outputs.release_created }}
    uses: ./.github/workflows/deploy.yml
    with:
      version: ${{ needs.release-please.outputs.tag_name }}
    secrets: inherit
```

:::caution[release-please Commit Convention]
`release-please` requires [Conventional Commits](https://www.conventionalcommits.org/) format: `feat:` bumps minor, `fix:` bumps patch, `feat!:` or `BREAKING CHANGE:` bumps major. Without this convention, no release PR will be generated.
:::

## Blue-Green Deployments

```yaml
jobs:
  blue-green:
    runs-on: ubuntu-latest
    environment: production
    steps:
      - uses: actions/checkout@v4

      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_ROLE_ARN }}
          aws-region: us-east-1

      - name: Determine current color and target
        id: color
        run: |
          CURRENT=$(aws ssm get-parameter \
            --name /myapp/active-color \
            --query 'Parameter.Value' \
            --output text)
          TARGET=$([ "$CURRENT" = "blue" ] && echo "green" || echo "blue")
          echo "current=$CURRENT" >> $GITHUB_OUTPUT
          echo "target=$TARGET" >> $GITHUB_OUTPUT

      - name: Deploy to ${{ steps.color.outputs.target }}
        run: |
          aws ecs update-service \
            --cluster myapp-${{ steps.color.outputs.target }} \
            --service myapp \
            --force-new-deployment
          aws ecs wait services-stable \
            --cluster myapp-${{ steps.color.outputs.target }} \
            --services myapp

      - name: Run smoke tests against new environment
        run: |
          curl -f https://${{ steps.color.outputs.target }}.internal.example.com/health

      - name: Flip traffic to ${{ steps.color.outputs.target }}
        run: |
          aws elbv2 modify-rule \
            --rule-arn ${{ vars.ALB_RULE_ARN }} \
            --actions "[{\"Type\":\"forward\",\"TargetGroupArn\":\"${{ vars[format('{0}_TG_ARN', steps.color.outputs.target)] }}\"}]"

          aws ssm put-parameter \
            --name /myapp/active-color \
            --value ${{ steps.color.outputs.target }} \
            --overwrite
```

Jobs are the execution units of a workflow. Each job runs on its own fresh runner. This page covers the complete job API, service containers, inter-job outputs, caching, and artifact management.

## Job Structure

A job is defined by its ID key under `jobs:`. Here is the full set of job-level configuration options:

```yaml
jobs:
  deploy:
    name: Deploy to Production       # Display name (optional)
    runs-on: ubuntu-latest            # Runner label
    needs: [build, test]              # Job dependencies
    if: github.ref == 'refs/heads/main'
    environment:
      name: production
      url: https://myapp.com
    timeout-minutes: 30               # Default: 360 (6 hours)
    continue-on-error: false
    concurrency:
      group: deploy-production
      cancel-in-progress: false
    permissions:
      contents: read
      id-token: write
    outputs:
      deployed-url: ${{ steps.deploy.outputs.url }}
    strategy:
      matrix:
        region: [us-east-1, eu-west-1]
    env:
      DEPLOY_ENV: production
    steps:
      - uses: actions/checkout@v4
```

## Job Outputs

Pass data between jobs using outputs. A step writes to `$GITHUB_OUTPUT`; the job exposes the value; a downstream job reads it via `needs.<job-id>.outputs.<name>`.

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    outputs:
      image-tag: ${{ steps.meta.outputs.image-tag }}
      build-time: ${{ steps.build.outputs.timestamp }}

    steps:
      - name: Generate image tag
        id: meta
        run: |
          TAG="sha-${GITHUB_SHA::8}"
          echo "image-tag=$TAG" >> $GITHUB_OUTPUT

      - name: Record build time
        id: build
        run: |
          echo "timestamp=$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> $GITHUB_OUTPUT

  deploy:
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy image
        run: |
          echo "Deploying ${{ needs.build.outputs.image-tag }}"
          echo "Built at ${{ needs.build.outputs.build-time }}"
```

:::tip[`$GITHUB_OUTPUT` replaces `set-output`]
The old `echo "::set-output name=key::value"` workflow command is deprecated. Always use `echo "key=value" >> $GITHUB_OUTPUT` (appending to the file at `$GITHUB_OUTPUT`).
:::

## Service Containers

Service containers are Docker containers that run alongside your job — useful for databases, caches, and other external dependencies in integration tests.

### PostgreSQL Service

```yaml
jobs:
  integration-test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: testuser
          POSTGRES_PASSWORD: testpass
          POSTGRES_DB: testdb
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4

      - name: Run migrations
        env:
          DATABASE_URL: postgresql://testuser:testpass@localhost:5432/testdb
        run: npm run db:migrate

      - name: Run integration tests
        env:
          DATABASE_URL: postgresql://testuser:testpass@localhost:5432/testdb
        run: npm run test:integration
```

### Redis and PostgreSQL Together

```yaml
services:
  postgres:
    image: postgres:16
    env:
      POSTGRES_PASSWORD: secret
    options: >-
      --health-cmd pg_isready
      --health-interval 10s
      --health-timeout 5s
      --health-retries 5
    ports:
      - 5432:5432

  redis:
    image: redis:7-alpine
    options: >-
      --health-cmd "redis-cli ping"
      --health-interval 10s
      --health-timeout 5s
      --health-retries 5
    ports:
      - 6379:6379
```

:::caution[Port mapping requirement]
When running on `ubuntu-latest` (not inside a container), you must map ports with `ports:` to access the service via `localhost`. The service container is on Docker's internal network by default.
:::

## Container Jobs

Run your entire job inside a Docker container instead of on the bare runner:

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    container:
      image: node:20-alpine
      env:
        NODE_ENV: test
      options: --cpus 2
      volumes:
        - /tmp:/tmp
      credentials:
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}

    services:
      db:
        image: postgres:16
        env:
          POSTGRES_PASSWORD: secret
        # Note: use service name as hostname, not localhost!

    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - name: Connect to DB
        env:
          DATABASE_URL: postgresql://postgres:secret@db:5432/postgres
        run: npm test
```

When using container jobs, service containers are accessible by their **service name** as the hostname (e.g., `db`, `redis`), not `localhost`.

## Caching with `actions/cache@v4`

Caching stores and restores arbitrary paths between runs, reducing build times significantly.

```yaml
- name: Cache node_modules
  id: cache-npm
  uses: actions/cache@v4
  with:
    path: ~/.npm
    key: npm-${{ runner.os }}-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      npm-${{ runner.os }}-

- name: Install (skip if cache hit)
  if: steps.cache-npm.outputs.cache-hit != 'true'
  run: npm ci
```

### Cache Keys and Restore Keys

The `key` must be an exact match for a cache hit. `restore-keys` are fallback prefix patterns:

```yaml
key: build-${{ runner.os }}-${{ hashFiles('**/yarn.lock') }}-${{ github.sha }}
restore-keys: |
  build-${{ runner.os }}-${{ hashFiles('**/yarn.lock') }}-
  build-${{ runner.os }}-
  build-
```

If the exact `key` is found → **cache hit** (restored, not saved again).
If only `restore-keys` match → **partial hit** (restored, new cache saved with exact key at job end).

## Artifacts

Artifacts are files uploaded during a workflow run. They persist after the job finishes and can be downloaded from the GitHub UI or shared between jobs.

### Upload Artifacts

```yaml
- name: Upload test results
  if: always()   # Upload even if tests fail
  uses: actions/upload-artifact@v4
  with:
    name: test-results-${{ matrix.node }}
    path: |
      coverage/
      test-results.xml
    retention-days: 30       # Default: 90, max: 90
    if-no-files-found: error # error | warn | ignore
    overwrite: false
```

### Download Artifacts

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: npm run build
      - uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/

  deploy:
    runs-on: ubuntu-latest
    needs: build
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist/      # Download to this path

      - run: ls dist/

  # Download ALL artifacts from this workflow run
  report:
    runs-on: ubuntu-latest
    needs: [build]
    steps:
      - uses: actions/download-artifact@v4
        with:
          path: all-artifacts/   # No 'name' = download all
```

## `actions/github-script`

Run JavaScript that has full access to the GitHub API via the `@octokit/rest` client:

```yaml
- name: Comment on PR
  uses: actions/github-script@v7
  with:
    script: |
      const { data: comments } = await github.rest.issues.listComments({
        owner: context.repo.owner,
        repo: context.repo.repo,
        issue_number: context.issue.number,
      });

      const botComment = comments.find(c =>
        c.user.type === 'Bot' && c.body.includes('## Test Results')
      );

      const body = `## Test Results\n✅ All tests passed at ${new Date().toISOString()}`;

      if (botComment) {
        await github.rest.issues.updateComment({
          owner: context.repo.owner,
          repo: context.repo.repo,
          comment_id: botComment.id,
          body,
        });
      } else {
        await github.rest.issues.createComment({
          owner: context.repo.owner,
          repo: context.repo.repo,
          issue_number: context.issue.number,
          body,
        });
      }
```

## Job Summary (`$GITHUB_STEP_SUMMARY`)

Write Markdown to `$GITHUB_STEP_SUMMARY` and it appears in the workflow run summary page:

```yaml
- name: Generate test summary
  if: always()
  run: |
    echo "## Test Results" >> $GITHUB_STEP_SUMMARY
    echo "" >> $GITHUB_STEP_SUMMARY
    echo "| Status | Count |" >> $GITHUB_STEP_SUMMARY
    echo "|--------|-------|" >> $GITHUB_STEP_SUMMARY
    echo "| ✅ Passed | $(grep -c 'PASS' test-output.txt || echo 0) |" >> $GITHUB_STEP_SUMMARY
    echo "| ❌ Failed | $(grep -c 'FAIL' test-output.txt || echo 0) |" >> $GITHUB_STEP_SUMMARY
```

## Masking Sensitive Values

Use `add-mask` to prevent a value from appearing in logs:

```yaml
- name: Set dynamic secret
  run: |
    DYNAMIC_TOKEN=$(./get-token.sh)
    echo "::add-mask::$DYNAMIC_TOKEN"
    echo "TOKEN=$DYNAMIC_TOKEN" >> $GITHUB_ENV
```

Any value masked with `add-mask` is replaced with `***` in all subsequent log output.

## Environment Variables: `$GITHUB_ENV` and `$GITHUB_PATH`

### Persist env vars across steps (`$GITHUB_ENV`)

```yaml
- name: Set environment variables for next steps
  run: |
    echo "APP_VERSION=$(node -p "require('./package.json').version")" >> $GITHUB_ENV
    echo "BUILD_DATE=$(date -u +%Y%m%d)" >> $GITHUB_ENV

- name: Use persisted env
  run: echo "Building version $APP_VERSION on $BUILD_DATE"
```

### Add to PATH (`$GITHUB_PATH`)

```yaml
- name: Add custom tools to PATH
  run: echo "/usr/local/my-tools/bin" >> $GITHUB_PATH

- name: Use custom tool
  run: my-tool --version
```

## Complete Job with All Features

```yaml
jobs:
  full-example:
    name: Full Integration Test
    runs-on: ubuntu-latest
    timeout-minutes: 45
    outputs:
      test-coverage: ${{ steps.coverage.outputs.pct }}

    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_PASSWORD: secret
          POSTGRES_DB: testdb
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-retries 5
        ports:
          - 5432:5432
      redis:
        image: redis:7-alpine
        ports:
          - 6379:6379

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - uses: actions/cache@v4
        with:
          path: ~/.cache/playwright
          key: playwright-${{ runner.os }}-${{ hashFiles('**/package-lock.json') }}

      - run: npm ci

      - name: Run migrations
        env:
          DATABASE_URL: postgresql://postgres:secret@localhost:5432/testdb
        run: npm run db:migrate

      - name: Run integration tests
        id: test
        env:
          DATABASE_URL: postgresql://postgres:secret@localhost:5432/testdb
          REDIS_URL: redis://localhost:6379
        run: npm run test:integration -- --coverage

      - name: Parse coverage
        id: coverage
        if: always()
        run: |
          PCT=$(node -p "require('./coverage/coverage-summary.json').total.lines.pct")
          echo "pct=$PCT" >> $GITHUB_OUTPUT
          echo "## Coverage: $PCT%" >> $GITHUB_STEP_SUMMARY

      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: test-artifacts
          path: |
            coverage/
            test-results/
          retention-days: 14
```

Security is non-negotiable in CI/CD pipelines. This page covers every mechanism for managing sensitive values — from repository secrets to OIDC federated identity — along with the full reference for environment variables, contexts, and expression functions.

## Secrets

### Repository Secrets

Stored at Settings → Secrets and variables → Actions → Repository secrets. Available to all workflows in the repository.

```yaml
steps:
  - name: Deploy
    env:
      API_KEY: ${{ secrets.API_KEY }}
      DB_PASSWORD: ${{ secrets.DB_PASSWORD }}
    run: ./deploy.sh
```

### Environment Secrets

Scoped to a specific GitHub Environment (staging, production). Only accessible when a job targets that environment:

```yaml
jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: production   # activates production environment secrets
    steps:
      - name: Deploy
        env:
          PROD_API_KEY: ${{ secrets.PROD_API_KEY }}  # environment secret
        run: ./deploy-prod.sh
```

### Organization Secrets

Managed at the organization level and shared across repositories. Set at Settings → Secrets → Actions in the org.

```yaml
# Org secrets are referenced the same way
env:
  SHARED_REGISTRY_TOKEN: ${{ secrets.SHARED_REGISTRY_TOKEN }}
```

### Dependabot Secrets

Dependabot can't access regular repository secrets. Set Dependabot-specific secrets at Settings → Secrets → Dependabot:

```yaml
# These are available only to Dependabot-triggered workflows
env:
  NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

:::caution[Secret Limits]
Secrets are limited to 48 KB. For larger values (certificates, SSH keys), use a secrets manager (AWS Secrets Manager, HashiCorp Vault) and retrieve them at runtime using OIDC.
:::

## `GITHUB_TOKEN`

Automatically available in every workflow — no setup required. It's a short-lived token with scoped permissions to the current repository.

```yaml
- name: Create release
  uses: actions/github-script@v7
  with:
    github-token: ${{ secrets.GITHUB_TOKEN }}
    script: |
      await github.rest.repos.createRelease({
        owner: context.repo.owner,
        repo: context.repo.repo,
        tag_name: 'v1.0.0',
        name: 'Release v1.0.0',
        generate_release_notes: true,
      });
```

Default token permissions can be configured at the repo or org level. Always declare `permissions:` explicitly:

```yaml
permissions:
  contents: write        # needed to create releases
  pull-requests: write   # needed to comment on PRs
  packages: write        # needed to push to GHCR
```

## OIDC — Cloud Authentication Without Long-Lived Secrets

OpenID Connect (OIDC) lets GitHub Actions authenticate to cloud providers by exchanging a short-lived JWT token — no stored cloud credentials needed.

### AWS Authentication with OIDC

<Steps>
1. Create an IAM OIDC identity provider in AWS pointing to `token.actions.githubusercontent.com`
2. Create an IAM role with a trust policy scoped to your repository
3. Use `aws-actions/configure-aws-credentials` in your workflow
</Steps>

```yaml
permissions:
  id-token: write    # required to request the OIDC token
  contents: read

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::123456789012:role/GitHubActionsRole
          aws-region: us-east-1
          role-session-name: GitHubActions-${{ github.run_id }}

      - name: Deploy to S3
        run: aws s3 sync dist/ s3://my-bucket/
```

Trust policy for the IAM role (scope to a specific repo and branch):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Federated": "arn:aws:iam::123456789012:oidc-provider/token.actions.githubusercontent.com"
      },
      "Action": "sts:AssumeRoleWithWebIdentity",
      "Condition": {
        "StringEquals": {
          "token.actions.githubusercontent.com:aud": "sts.amazonaws.com",
          "token.actions.githubusercontent.com:sub": "repo:myorg/myrepo:ref:refs/heads/main"
        }
      }
    }
  ]
}
```

### GCP Authentication with OIDC

```yaml
- name: Authenticate to GCP
  uses: google-github-actions/auth@v2
  with:
    workload_identity_provider: 'projects/123456789/locations/global/workloadIdentityPools/my-pool/providers/my-provider'
    service_account: 'github-actions@my-project.iam.gserviceaccount.com'

- name: Deploy to Cloud Run
  uses: google-github-actions/deploy-cloudrun@v2
  with:
    service: my-service
    region: us-central1
    image: gcr.io/my-project/my-image:latest
```

### Azure Authentication with OIDC

```yaml
- name: Authenticate to Azure
  uses: azure/login@v2
  with:
    client-id: ${{ secrets.AZURE_CLIENT_ID }}
    tenant-id: ${{ secrets.AZURE_TENANT_ID }}
    subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}
```

:::tip[OIDC vs Stored Credentials]
OIDC is dramatically more secure than storing access keys as secrets. The token is valid for only the duration of the job, scoped to the specific workflow run, and automatically rotated. Prefer OIDC for all cloud provider authentication.
:::

## Default Environment Variables

GitHub automatically provides these environment variables in every workflow run:

| Variable | Description | Example |
|---|---|---|
| `GITHUB_SHA` | Full commit SHA triggering the workflow | `d6fde92930d...` |
| `GITHUB_REF` | Branch or tag ref | `refs/heads/main` |
| `GITHUB_HEAD_REF` | Source branch of a PR | `feature/my-feature` |
| `GITHUB_BASE_REF` | Target branch of a PR | `main` |
| `GITHUB_ACTOR` | Username that triggered the run | `octocat` |
| `GITHUB_REPOSITORY` | Owner and repository name | `myorg/myrepo` |
| `GITHUB_WORKSPACE` | Path to checked-out repository | `/home/runner/work/myrepo/myrepo` |
| `GITHUB_RUN_ID` | Unique numeric ID for this run | `1658821493` |
| `GITHUB_RUN_NUMBER` | Sequential run number for this workflow | `42` |
| `GITHUB_WORKFLOW` | Name of the workflow | `CI` |
| `GITHUB_JOB` | ID of the current job | `build` |
| `GITHUB_EVENT_NAME` | Name of the triggering event | `push` |
| `RUNNER_OS` | OS of the runner | `Linux` |
| `RUNNER_ARCH` | Architecture of the runner | `X64` |
| `RUNNER_TEMP` | Path to a temp directory | `/home/runner/work/_temp` |
| `CI` | Always `true` in Actions | `true` |

```yaml
- name: Print context info
  run: |
    echo "Repo: $GITHUB_REPOSITORY"
    echo "Branch: $GITHUB_REF_NAME"
    echo "SHA: ${GITHUB_SHA:0:7}"
    echo "Run #: $GITHUB_RUN_NUMBER"
    echo "Actor: $GITHUB_ACTOR"
```

## `$GITHUB_ENV` — Persist Variables Across Steps

```yaml
- name: Compute and persist values
  run: |
    APP_VERSION=$(node -p "require('./package.json').version")
    BUILD_DATE=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
    IMAGE_TAG="myapp:$APP_VERSION-${GITHUB_SHA:0:7}"

    echo "APP_VERSION=$APP_VERSION" >> $GITHUB_ENV
    echo "BUILD_DATE=$BUILD_DATE" >> $GITHUB_ENV
    echo "IMAGE_TAG=$IMAGE_TAG" >> $GITHUB_ENV

- name: Use persisted variables
  run: |
    echo "Version: $APP_VERSION"
    echo "Built at: $BUILD_DATE"
    docker tag myapp:latest $IMAGE_TAG
```

## `$GITHUB_PATH` — Add to PATH

```yaml
- name: Install custom tool
  run: |
    mkdir -p $HOME/.local/bin
    curl -L https://example.com/mytool -o $HOME/.local/bin/mytool
    chmod +x $HOME/.local/bin/mytool
    echo "$HOME/.local/bin" >> $GITHUB_PATH

- name: Use the tool
  run: mytool --version   # now available in PATH
```

## Context Objects

Contexts are collections of runtime information available via `${{ context.property }}` syntax.

### `github` Context

```yaml
${{ github.sha }}                    # current commit SHA
${{ github.ref }}                    # refs/heads/main
${{ github.ref_name }}               # main (short name)
${{ github.ref_type }}               # branch | tag
${{ github.event_name }}             # push | pull_request | etc.
${{ github.event.head_commit.message }} # commit message
${{ github.repository }}             # owner/repo
${{ github.actor }}                  # triggering username
${{ github.run_id }}                 # unique run ID
${{ github.run_number }}             # sequential run number
${{ github.workflow }}               # workflow name
${{ github.job }}                    # current job ID
${{ github.server_url }}             # https://github.com
${{ github.api_url }}                # https://api.github.com
```

### `env` Context

```yaml
env:
  MY_VAR: hello

steps:
  - run: echo ${{ env.MY_VAR }}
```

### `vars` Context — Non-Secret Variables

Configuration variables (not secrets) stored in Settings → Variables:

```yaml
steps:
  - run: |
      echo "App name: ${{ vars.APP_NAME }}"
      echo "Region: ${{ vars.AWS_REGION }}"
```

### `needs` Context

```yaml
jobs:
  build:
    outputs:
      version: ${{ steps.ver.outputs.version }}

  deploy:
    needs: build
    steps:
      - run: echo "Deploying ${{ needs.build.outputs.version }}"
        # needs.build.result == 'success' | 'failure' | 'cancelled' | 'skipped'
```

### `job` Context

```yaml
- run: |
    echo "Job status: ${{ job.status }}"  # success | failure | cancelled
    echo "Container ID: ${{ job.container.id }}"
```

### `steps` Context

```yaml
- name: Check previous step
  if: steps.my-step.outcome == 'failure'
  run: echo "Step failed!"
```

### `matrix` Context

```yaml
strategy:
  matrix:
    node: [18, 20]
steps:
  - run: echo "Node version: ${{ matrix.node }}"
```

## Expression Syntax

Expressions are evaluated at runtime: `${{ expression }}`

### Operators

```yaml
# Comparison
${{ github.ref == 'refs/heads/main' }}
${{ github.run_number > 100 }}
${{ github.actor != 'dependabot[bot]' }}

# Logical
${{ github.event_name == 'push' && github.ref == 'refs/heads/main' }}
${{ github.event_name == 'push' || github.event_name == 'workflow_dispatch' }}
${{ !contains(github.event.head_commit.message, '[skip ci]') }}
```

### Built-in Functions

```yaml
# String functions
${{ contains('hello world', 'hello') }}    # true
${{ startsWith(github.ref, 'refs/tags/') }} # true for tags
${{ endsWith(github.ref, '/main') }}

# Format
${{ format('Hello {0}!', github.actor) }}
${{ format('{0}/{1}:{2}', github.repository, 'main', github.sha) }}

# Array/JSON
${{ join(matrix.os, ', ') }}
${{ toJSON(github.event) }}
${{ fromJSON('{"key":"value"}').key }}

# File hashing (for cache keys)
${{ hashFiles('**/package-lock.json') }}
${{ hashFiles('requirements*.txt', 'setup.py') }}

# Status check functions (for if: conditions)
${{ success() }}    # all previous steps succeeded
${{ failure() }}    # any previous step failed
${{ always() }}     # always runs (even on cancel)
${{ cancelled() }}  # workflow was cancelled
```

### Conditional Step Execution Patterns

```yaml
# Run only on main branch pushes
- if: github.ref == 'refs/heads/main' && github.event_name == 'push'

# Run on failure to notify
- if: failure()
  name: Notify on failure
  run: ./notify.sh

# Always clean up
- if: always()
  name: Cleanup
  run: ./cleanup.sh

# Skip on forks
- if: github.repository == 'myorg/myrepo'
  name: Deploy (only on origin)

# Run when PR is not a draft
- if: github.event.pull_request.draft == false

# Run when specific label is present
- if: contains(github.event.pull_request.labels.*.name, 'run-e2e')
```

## Complete Secrets & Auth Example

```yaml
name: Secure Deploy

on:
  push:
    branches: [main]

permissions:
  id-token: write
  contents: read

env:
  APP_NAME: ${{ vars.APP_NAME }}
  AWS_REGION: ${{ vars.AWS_REGION }}

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: production

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials (OIDC)
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}

      - name: Get secrets from AWS Secrets Manager
        run: |
          SECRET=$(aws secretsmanager get-secret-value \
            --secret-id myapp/production \
            --query SecretString \
            --output text)
          DB_PASS=$(echo $SECRET | jq -r .db_password)
          echo "::add-mask::$DB_PASS"
          echo "DB_PASSWORD=$DB_PASS" >> $GITHUB_ENV

      - name: Deploy
        env:
          DEPLOY_API_KEY: ${{ secrets.DEPLOY_API_KEY }}
        run: |
          echo "Deploying $APP_NAME to production"
          ./deploy.sh --env production

      - name: Write summary
        if: always()
        run: |
          echo "## Deployment Summary" >> $GITHUB_STEP_SUMMARY
          echo "- **App**: $APP_NAME" >> $GITHUB_STEP_SUMMARY
          echo "- **SHA**: ${GITHUB_SHA:0:7}" >> $GITHUB_STEP_SUMMARY
          echo "- **Status**: ${{ job.status }}" >> $GITHUB_STEP_SUMMARY
```

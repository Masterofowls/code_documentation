# Getting Started with Kubernetes

This guide covers installing `kubectl`, choosing a local cluster, and learning the essential commands for daily Kubernetes work.

## Installing kubectl

<Tabs>
  <TabItem label="macOS">
    ```bash
    # Homebrew
    brew install kubectl

    # Or download directly
    curl -LO "https://dl.k8s.io/release/$(curl -L -s \
      https://dl.k8s.io/release/stable.txt)/bin/darwin/arm64/kubectl"
    chmod +x kubectl
    sudo mv kubectl /usr/local/bin/

    # Verify
    kubectl version --client
    ```
  </TabItem>
  <TabItem label="Linux">
    ```bash
    # Download latest stable
    curl -LO "https://dl.k8s.io/release/$(curl -L -s \
      https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
    chmod +x kubectl
    sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

    # Via package manager (Ubuntu/Debian)
    sudo apt-get install -y kubectl

    # Verify
    kubectl version --client
    ```
  </TabItem>
  <TabItem label="Windows">
    ```powershell
    # Chocolatey
    choco install kubernetes-cli

    # Winget
    winget install Kubernetes.kubectl

    # Verify
    kubectl version --client
    ```
  </TabItem>
</Tabs>

## Local Cluster Options

<Tabs>
  <TabItem label="minikube">
    Full-featured local Kubernetes cluster. Supports multiple container runtimes and add-ons.

    ```bash
    # Install
    brew install minikube       # macOS
    # or https://minikube.sigs.k8s.io/docs/start/

    # Start cluster (uses Docker driver by default)
    minikube start

    # Start with specific resources
    minikube start --cpus 4 --memory 8g

    # Enable useful add-ons
    minikube addons enable ingress
    minikube addons enable metrics-server
    minikube addons enable dashboard

    # Open dashboard
    minikube dashboard

    # Expose service URL
    minikube service myapp --url

    # Stop / delete
    minikube stop
    minikube delete
    ```
  </TabItem>
  <TabItem label="kind">
    Kubernetes-IN-Docker. Creates a cluster using Docker containers as nodes. Fast and great for CI.

    ```bash
    # Install
    brew install kind

    # Create a single-node cluster
    kind create cluster --name dev

    # Create a multi-node cluster
    cat <<EOF | kind create cluster --name dev --config=-
    kind: Cluster
    apiVersion: kind.x-k8s.io/v1alpha4
    nodes:
      - role: control-plane
      - role: worker
      - role: worker
    EOF

    # Load local image into kind cluster
    kind load docker-image myapp:latest --name dev

    # Delete cluster
    kind delete cluster --name dev
    ```
  </TabItem>
  <TabItem label="k3d">
    k3s (lightweight K8s) running in Docker. Very fast to spin up.

    ```bash
    # Install
    brew install k3d

    # Create a cluster
    k3d cluster create dev

    # Create with load balancer and port mapping
    k3d cluster create dev \
      --port "80:80@loadbalancer" \
      --port "443:443@loadbalancer" \
      --agents 2

    # Import local image
    k3d image import myapp:latest --cluster dev

    # Delete
    k3d cluster delete dev
    ```
  </TabItem>
  <TabItem label="Docker Desktop">
    Docker Desktop (macOS and Windows) includes a single-node Kubernetes cluster.

    1. Docker Desktop → Settings → Kubernetes
    2. Check **Enable Kubernetes**
    3. Click **Apply & Restart**

    ```bash
    # Switch context to Docker Desktop K8s
    kubectl config use-context docker-desktop
    ```
  </TabItem>
</Tabs>

## kubeconfig and Contexts

`kubectl` uses `~/.kube/config` to store cluster connection details.

```bash
# View current config
kubectl config view

# List all contexts
kubectl config get-contexts

# Show current context
kubectl config current-context

# Switch context
kubectl config use-context my-cluster

# Set default namespace for current context
kubectl config set-context --current --namespace myapp

# Rename a context
kubectl config rename-context old-name new-name

# Merge multiple kubeconfig files
export KUBECONFIG=~/.kube/config:~/.kube/config-staging
kubectl config view --flatten > ~/.kube/config-merged
```

:::tip[kubectx + kubens]
Install `kubectx` and `kubens` for fast context and namespace switching:
```bash
brew install kubectx
kubectx                    # list contexts
kubectx my-cluster         # switch context
kubens                     # list namespaces
kubens myapp               # switch namespace
```
:::

## Essential kubectl Commands

### Get Resources

```bash
# Get pods in current namespace
kubectl get pods

# Get pods in all namespaces
kubectl get pods -A
kubectl get pods --all-namespaces

# Get pods in a specific namespace
kubectl get pods -n kube-system

# Wide output (shows node, IP, etc.)
kubectl get pods -o wide

# Watch mode (refresh on changes)
kubectl get pods -w

# YAML output
kubectl get pod my-pod -o yaml

# JSON output
kubectl get pod my-pod -o json

# Custom columns
kubectl get pods \
  -o custom-columns='NAME:.metadata.name,STATUS:.status.phase,IP:.status.podIP'

# Get all resource types
kubectl get all
kubectl get all -n myapp

# Get specific resources
kubectl get deployments
kubectl get services
kubectl get ingresses
kubectl get configmaps
kubectl get secrets
kubectl get pvc
kubectl get nodes
```

### Describe Resources

```bash
# Detailed information (events, conditions, etc.)
kubectl describe pod my-pod
kubectl describe node worker-1
kubectl describe deployment my-app
kubectl describe service my-service
```

### Apply and Delete

```bash
# Apply a manifest (create or update)
kubectl apply -f deployment.yaml

# Apply all YAML files in a directory
kubectl apply -f ./k8s/

# Apply from a URL
kubectl apply -f https://example.com/manifest.yaml

# Delete resources
kubectl delete -f deployment.yaml
kubectl delete pod my-pod
kubectl delete deployment my-app
kubectl delete namespace myapp     # deletes EVERYTHING in namespace

# Delete with label selector
kubectl delete pods -l app=myapp
```

### Dry Run and Diff

```bash
# Dry run — validate without applying
kubectl apply -f deployment.yaml --dry-run=client
kubectl apply -f deployment.yaml --dry-run=server

# Diff — show what would change
kubectl diff -f deployment.yaml
```

### Logs

```bash
# View pod logs
kubectl logs my-pod

# Follow logs
kubectl logs -f my-pod

# Logs from a specific container in a multi-container pod
kubectl logs my-pod -c sidecar

# Previous container logs (after crash)
kubectl logs my-pod --previous

# Last 100 lines
kubectl logs --tail=100 my-pod

# Logs since 1 hour ago
kubectl logs --since=1h my-pod

# Logs for all pods with a label
kubectl logs -l app=myapp --all-containers
```

### Exec into Pods

```bash
# Interactive shell
kubectl exec -it my-pod -- bash
kubectl exec -it my-pod -- sh   # for Alpine-based images

# Run a command
kubectl exec my-pod -- env
kubectl exec my-pod -- ls /app

# Exec into a specific container
kubectl exec -it my-pod -c main-container -- bash
```

### Port Forwarding

```bash
# Forward local port 8080 to pod port 3000
kubectl port-forward pod/my-pod 8080:3000

# Forward to a service
kubectl port-forward service/my-service 8080:80

# Forward to a deployment (picks any ready pod)
kubectl port-forward deployment/my-app 8080:3000

# Bind to all interfaces (accessible from other machines)
kubectl port-forward --address 0.0.0.0 service/my-service 8080:80
```

### Rollout Management

```bash
# Check rollout status
kubectl rollout status deployment/my-app

# View rollout history
kubectl rollout history deployment/my-app

# View specific revision
kubectl rollout history deployment/my-app --revision=3

# Rollback to previous version
kubectl rollout undo deployment/my-app

# Rollback to specific revision
kubectl rollout undo deployment/my-app --to-revision=2

# Pause a rollout
kubectl rollout pause deployment/my-app

# Resume a rollout
kubectl rollout resume deployment/my-app

# Restart all pods (triggers rolling update)
kubectl rollout restart deployment/my-app
```

### Scale

```bash
# Scale manually
kubectl scale deployment/my-app --replicas=5

# Scale to zero (useful for stopping without deleting)
kubectl scale deployment/my-app --replicas=0
```

### Resource Usage

```bash
# Pod CPU/memory (requires metrics-server)
kubectl top pods
kubectl top pods -A

# Node CPU/memory
kubectl top nodes

# Sort by CPU
kubectl top pods --sort-by=cpu
```

## Namespaces

Namespaces are virtual clusters within a Kubernetes cluster. Use them to isolate teams, environments, or applications.

```bash
# List namespaces
kubectl get namespaces
kubectl get ns

# Create a namespace
kubectl create namespace myapp
# Or declaratively:
kubectl apply -f - <<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: myapp
  labels:
    env: production
EOF

# Set default namespace for current session
kubectl config set-context --current --namespace=myapp

# Delete a namespace (removes ALL resources inside)
kubectl delete namespace myapp
```

## Deploying Your First Application

### Imperative (Quick)

```bash
# Create a deployment
kubectl create deployment nginx --image=nginx:1.27-alpine --replicas=2

# Expose it as a service
kubectl expose deployment nginx --port=80 --type=ClusterIP

# Access via port forward
kubectl port-forward service/nginx 8080:80
```

### Declarative (Recommended)

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx
  namespace: default
  labels:
    app: nginx
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.27-alpine
          ports:
            - containerPort: 80
          resources:
            requests:
              cpu: 100m
              memory: 64Mi
            limits:
              cpu: 200m
              memory: 128Mi
---
apiVersion: v1
kind: Service
metadata:
  name: nginx
spec:
  selector:
    app: nginx
  ports:
    - port: 80
      targetPort: 80
```

```bash
kubectl apply -f deployment.yaml
kubectl get pods
kubectl port-forward service/nginx 8080:80
```

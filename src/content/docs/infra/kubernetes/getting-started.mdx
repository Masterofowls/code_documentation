# Getting Started with Kubernetes

This guide covers installing `kubectl`, choosing a local cluster tool, and mastering the commands you'll use every day when working with Kubernetes.

---

## Installing kubectl

`kubectl` is the Kubernetes command-line tool. It communicates with the cluster API server.

<Tabs>
  <TabItem label="macOS">
    ```bash
    # Homebrew
    brew install kubectl

    # Verify
    kubectl version --client
    ```
  </TabItem>
  <TabItem label="Linux">
    ```bash
    # Download latest stable release
    curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

    # Install
    sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

    # Verify
    kubectl version --client
    ```
  </TabItem>
  <TabItem label="Windows">
    ```powershell
    # Winget
    winget install Kubernetes.kubectl

    # Or with chocolatey
    choco install kubernetes-cli

    # Verify
    kubectl version --client
    ```
  </TabItem>
</Tabs>

## Local Cluster Options

| Tool | Description | Best For |
|---|---|---|
| **minikube** | Full K8s in a VM or container | General learning, addons ecosystem |
| **kind** | K8s inside Docker containers | CI, multi-node testing |
| **k3d** | k3s inside Docker | Lightweight, fast startup |
| **Docker Desktop** | Built-in K8s toggle | macOS/Windows users already using Docker Desktop |
| **k3s** | Lightweight K8s for Linux | Edge, Raspberry Pi, minimal resource use |

<Tabs>
  <TabItem label="minikube">
    ```bash
    # Install
    brew install minikube        # macOS
    # Linux: curl/install from GitHub

    # Start a cluster
    minikube start
    minikube start --cpus 4 --memory 8192 --kubernetes-version v1.29.0

    # Useful commands
    minikube status
    minikube dashboard          # Open K8s dashboard in browser
    minikube addons list
    minikube addons enable ingress
    minikube addons enable metrics-server
    minikube tunnel              # Expose LoadBalancer services locally
    minikube stop
    minikube delete
    ```
  </TabItem>
  <TabItem label="kind">
    ```bash
    # Install
    brew install kind

    # Create a single-node cluster
    kind create cluster

    # Create a multi-node cluster
    cat > kind-config.yaml << 'EOF'
    kind: Cluster
    apiVersion: kind.x-k8s.io/v1alpha4
    nodes:
      - role: control-plane
      - role: worker
      - role: worker
    EOF
    kind create cluster --config kind-config.yaml --name my-cluster

    # Load a local Docker image into kind
    kind load docker-image my-app:latest

    # List clusters
    kind get clusters

    # Delete cluster
    kind delete cluster --name my-cluster
    ```
  </TabItem>
  <TabItem label="k3d">
    ```bash
    # Install
    brew install k3d

    # Create a cluster
    k3d cluster create mycluster

    # Multi-server + agent cluster
    k3d cluster create mycluster \
      --servers 1 \
      --agents 3 \
      --port "80:80@loadbalancer" \
      --port "443:443@loadbalancer"

    # Import local image
    k3d image import my-app:latest --cluster mycluster

    # List clusters
    k3d cluster list

    # Delete
    k3d cluster delete mycluster
    ```
  </TabItem>
</Tabs>

## kubectl Configuration

### kubeconfig

kubectl reads cluster configuration from `~/.kube/config` (or `$KUBECONFIG` env var). This file contains cluster endpoints, credentials, and contexts.

```bash
# View your kubeconfig
kubectl config view

# View raw kubeconfig (including credentials)
kubectl config view --raw

# Use a different kubeconfig file
kubectl --kubeconfig /path/to/config get pods
KUBECONFIG=/path/to/config kubectl get pods

# Merge multiple kubeconfigs
KUBECONFIG=~/.kube/config:~/.kube/cluster2 kubectl config view --merge --flatten > ~/.kube/merged-config
```

### Contexts

A context binds a cluster, user, and namespace together.

```bash
# List all contexts
kubectl config get-contexts

# Show current context
kubectl config current-context

# Switch to a different context
kubectl config use-context my-cluster
kubectl config use-context docker-desktop

# Rename a context
kubectl config rename-context old-name new-name

# Delete a context
kubectl config delete-context my-old-cluster

# Create/update a context
kubectl config set-context my-context \
  --cluster=my-cluster \
  --user=my-user \
  --namespace=production
```

:::tip[kubectx and kubens]
Install [kubectx and kubens](https://github.com/ahmetb/kubectx) for faster context and namespace switching:

```bash
brew install kubectx

kubectx              # List contexts
kubectx my-cluster   # Switch context
kubens production    # Switch namespace
kubens -             # Switch to previous namespace
```
:::

## Essential kubectl Commands

### Getting Resources

```bash
# Basic get
kubectl get pods
kubectl get deployments
kubectl get services
kubectl get nodes
kubectl get namespaces

# Short aliases work
kubectl get po
kubectl get deploy
kubectl get svc
kubectl get ns
kubectl get pvc
kubectl get cm               # configmaps
kubectl get secrets

# Get all resource types
kubectl get all

# Get with namespace
kubectl get pods -n kube-system
kubectl get pods --all-namespaces    # or -A

# Wide output (shows node, IP, etc.)
kubectl get pods -o wide

# JSON/YAML output
kubectl get pod my-pod -o json
kubectl get pod my-pod -o yaml

# Custom columns
kubectl get pods -o custom-columns='NAME:.metadata.name,STATUS:.status.phase,IP:.status.podIP'

# Watch for changes
kubectl get pods -w
kubectl get pods --watch
```

### Describing Resources

```bash
# Detailed description including events
kubectl describe pod my-pod
kubectl describe deployment my-deploy
kubectl describe node my-node
kubectl describe service my-svc

# Events are at the bottom — most useful for debugging crashes
kubectl describe pod crashing-pod | tail -30
```

### Applying and Deleting

```bash
# Apply a manifest (create or update — declarative)
kubectl apply -f deployment.yaml
kubectl apply -f ./manifests/          # Apply all files in directory
kubectl apply -f ./manifests/ -R       # Recursive
kubectl apply -f https://example.com/manifest.yaml

# Delete resources
kubectl delete -f deployment.yaml
kubectl delete pod my-pod
kubectl delete deployment my-deploy
kubectl delete service my-svc

# Delete all pods in a namespace (Deployment will recreate them)
kubectl delete pods --all -n my-namespace

# Force delete (skip graceful termination)
kubectl delete pod my-pod --grace-period=0 --force
```

### Logs

```bash
# Get logs from a pod
kubectl logs my-pod

# Follow logs in real time
kubectl logs -f my-pod

# Last 100 lines
kubectl logs --tail=100 my-pod

# With timestamps
kubectl logs -t my-pod

# From a specific container in a multi-container pod
kubectl logs my-pod -c my-container

# From a previous container instance (after crash)
kubectl logs my-pod --previous

# From all pods matching a selector
kubectl logs -l app=my-app --all-containers=true
kubectl logs -l app=my-app -f          # Follow all matched pods
```

### Exec and Port-Forward

```bash
# Execute command in a running container
kubectl exec my-pod -- ls /app
kubectl exec my-pod -c my-container -- env

# Interactive shell
kubectl exec -it my-pod -- bash
kubectl exec -it my-pod -- sh            # For alpine-based images

# Port forwarding (local:pod)
kubectl port-forward pod/my-pod 8080:3000
kubectl port-forward deployment/my-deploy 8080:3000
kubectl port-forward service/my-svc 8080:3000

# Bind to all interfaces
kubectl port-forward --address 0.0.0.0 service/my-svc 8080:3000
```

## Namespaces

Namespaces provide virtual clusters within a physical cluster.

```bash
# List namespaces
kubectl get namespaces
kubectl get ns

# Create namespace
kubectl create namespace my-namespace
kubectl apply -f - <<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: my-namespace
  labels:
    env: staging
EOF

# Delete namespace (deletes ALL resources in it!)
kubectl delete namespace my-namespace

# Set default namespace for current context
kubectl config set-context --current --namespace=my-namespace

# Always specify namespace explicitly in production
kubectl get pods -n production
kubectl apply -f manifest.yaml -n production
```

## Deploying Your First Application

<Steps>
1. **Imperative deployment (quick, not recommended for production)**

   ```bash
   kubectl create deployment hello \
     --image=nginx:alpine \
     --replicas=2

   kubectl expose deployment hello \
     --type=LoadBalancer \
     --port=80
   ```

2. **Declarative deployment (recommended)**

   ```yaml
   # hello-app.yaml
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: hello
     namespace: default
   spec:
     replicas: 2
     selector:
       matchLabels:
         app: hello
     template:
       metadata:
         labels:
           app: hello
       spec:
         containers:
           - name: nginx
             image: nginx:alpine
             ports:
               - containerPort: 80
   ---
   apiVersion: v1
   kind: Service
   metadata:
     name: hello
   spec:
     selector:
       app: hello
     type: LoadBalancer
     ports:
       - port: 80
         targetPort: 80
   ```

   ```bash
   kubectl apply -f hello-app.yaml
   kubectl get pods -w                # Watch pods start
   kubectl get service hello          # Check external IP
   ```

3. **Access the application**

   ```bash
   # On cloud (wait for EXTERNAL-IP)
   kubectl get service hello

   # On minikube
   minikube service hello --url

   # Port-forward for local access
   kubectl port-forward service/hello 8080:80
   # Open http://localhost:8080
   ```
</Steps>

## Dry Run and Diff

```bash
# Preview what would be created (client-side)
kubectl apply -f manifest.yaml --dry-run=client

# Validate against the server without applying (server-side dry run)
kubectl apply -f manifest.yaml --dry-run=server

# Diff current live resources vs manifest
kubectl diff -f manifest.yaml
```

## Useful Utilities

```bash
# Explain any resource/field
kubectl explain pod
kubectl explain pod.spec
kubectl explain pod.spec.containers
kubectl explain pod.spec.containers.resources.limits

# List all available resource types
kubectl api-resources
kubectl api-resources --namespaced=true    # Only namespaced resources
kubectl api-resources -o wide              # Show verbs and API groups

# Get API versions
kubectl api-versions

# Rollout management
kubectl rollout status deployment/my-app
kubectl rollout history deployment/my-app
kubectl rollout undo deployment/my-app
kubectl rollout undo deployment/my-app --to-revision=2
kubectl rollout pause deployment/my-app
kubectl rollout resume deployment/my-app

# Scale
kubectl scale deployment my-app --replicas=5

# Resource usage (requires metrics-server)
kubectl top nodes
kubectl top pods
kubectl top pods --containers
kubectl top pods -n kube-system
```

## Helpful Aliases

```bash
# Add to ~/.bashrc or ~/.zshrc
alias k='kubectl'
alias kgp='kubectl get pods'
alias kgd='kubectl get deployments'
alias kgs='kubectl get services'
alias kaf='kubectl apply -f'
alias kdel='kubectl delete'
alias klog='kubectl logs -f'
alias kex='kubectl exec -it'
alias kpf='kubectl port-forward'
alias kdesc='kubectl describe'

# Shell completion
source <(kubectl completion bash)     # bash
source <(kubectl completion zsh)      # zsh

# With alias
complete -F __start_kubectl k
```

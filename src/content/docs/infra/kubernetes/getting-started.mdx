# Getting Started with Kubernetes

This guide walks you through installing kubectl, choosing a local Kubernetes cluster, and learning the essential commands for daily Kubernetes work.

## Installing kubectl

<Tabs>
  <TabItem label="macOS">
    ```bash
    # Homebrew (recommended)
    brew install kubectl

    # Or download directly
    curl -LO "https://dl.k8s.io/release/$(curl -s https://dl.k8s.io/release/stable.txt)/bin/darwin/amd64/kubectl"
    chmod +x kubectl
    sudo mv kubectl /usr/local/bin/

    # Verify
    kubectl version --client
    ```

</TabItem>
  <TabItem label="Linux">
    ```bash
    # Download latest stable
    curl -LO "https://dl.k8s.io/release/$(curl -sL https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
    chmod +x kubectl
    sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

    # Or via package manager (Debian/Ubuntu)
    sudo apt-get install -y apt-transport-https ca-certificates curl
    curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.30/deb/Release.key \
      | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
    echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /' \
      | sudo tee /etc/apt/sources.list.d/kubernetes.list
    sudo apt-get update && sudo apt-get install -y kubectl

    kubectl version --client
    ```

</TabItem>
  <TabItem label="Windows">
    ```powershell
    # winget
    winget install Kubernetes.kubectl

    # Chocolatey
    choco install kubernetes-cli

    # Or scoop
    scoop install kubectl

    # Verify
    kubectl version --client
    ```

</TabItem>
</Tabs>

## Local Cluster Options

<Tabs>
  <TabItem label="minikube">
    The original local Kubernetes. Runs a single-node cluster in a VM or container.

    ```bash
    # Install (macOS)
    brew install minikube

    # Start cluster (uses Docker by default)
    minikube start
    minikube start --driver=docker
    minikube start --cpus=4 --memory=8192 --disk-size=50g

    # With specific Kubernetes version
    minikube start --kubernetes-version=v1.30.0

    # Enable addons
    minikube addons enable ingress
    minikube addons enable metrics-server
    minikube addons enable dashboard

    # Open dashboard
    minikube dashboard

    # Tunnel for LoadBalancer services
    minikube tunnel

    # Stop / delete
    minikube stop
    minikube delete
    ```

</TabItem>
  <TabItem label="kind">
    Kubernetes IN Docker — runs cluster nodes as Docker containers. Fast to create/destroy, great for CI.

    ```bash
    # Install
    brew install kind

    # Create cluster (single node)
    kind create cluster

    # Create with custom name
    kind create cluster --name dev

    # Multi-node cluster
    cat > kind-config.yaml <<EOF
    kind: Cluster
    apiVersion: kind.x-k8s.io/v1alpha4
    nodes:
      - role: control-plane
      - role: worker
      - role: worker
    EOF
    kind create cluster --config kind-config.yaml

    # Load local Docker image into kind
    kind load docker-image myapp:latest

    # List clusters
    kind get clusters

    # Delete cluster
    kind delete cluster
    kind delete cluster --name dev
    ```

</TabItem>
  <TabItem label="k3d">
    k3s (lightweight K8s) running in Docker. Even lighter than kind, great for local dev.

    ```bash
    # Install
    brew install k3d

    # Create cluster
    k3d cluster create dev

    # With load balancer and port mapping
    k3d cluster create dev \
      --port "8080:80@loadbalancer" \
      --port "8443:443@loadbalancer"

    # Multi-node
    k3d cluster create dev --agents 2

    # Import local image
    k3d image import myapp:latest -c dev

    # List clusters
    k3d cluster list

    # Delete cluster
    k3d cluster delete dev
    ```

</TabItem>
  <TabItem label="Docker Desktop">
    Docker Desktop includes a built-in single-node Kubernetes cluster.

    ```
    Settings → Kubernetes → Enable Kubernetes → Apply & Restart
    ```

    ```bash
    # Switch to Docker Desktop context
    kubectl config use-context docker-desktop

    # Verify
    kubectl cluster-info
    ```

    :::tip[Easiest for Beginners]
    Docker Desktop Kubernetes is the easiest way to get started if you already use Docker Desktop. No additional installation required.
    :::

</TabItem>
</Tabs>

## kubectl Configuration (kubeconfig)

kubectl uses a `~/.kube/config` file to store cluster connection information.

```bash
# View current kubeconfig
kubectl config view

# View raw kubeconfig (including secrets)
kubectl config view --raw

# List all contexts (cluster + user + namespace combinations)
kubectl config get-contexts

# Switch context
kubectl config use-context my-cluster
kubectl config use-context docker-desktop
kubectl config use-context minikube

# Rename a context
kubectl config rename-context old-name new-name

# Show current context
kubectl config current-context

# Set default namespace for current context
kubectl config set-context --current --namespace=myapp

# Merge kubeconfig files
KUBECONFIG=~/.kube/config:~/downloads/new-cluster.yaml \
  kubectl config view --merge --flatten > ~/.kube/config-merged
mv ~/.kube/config-merged ~/.kube/config
```

### Multiple Kubeconfig Files

```bash
# Set KUBECONFIG env var to load multiple files
export KUBECONFIG=~/.kube/config:~/.kube/dev-cluster:~/.kube/prod-cluster

# Add to ~/.bashrc or ~/.zshrc
echo 'export KUBECONFIG=$(ls ~/.kube/configs/* | tr "\n" ":")' >> ~/.zshrc
```

## Essential kubectl Commands

### Getting Resources

```bash
# Get resources (short names work: po, svc, deploy, rs, ns, no, cm, secret, pvc)
kubectl get pods
kubectl get po
kubectl get pods -n kube-system
kubectl get pods --all-namespaces
kubectl get pods -A             # short for --all-namespaces

# Wide output (shows node, IP)
kubectl get pods -o wide

# YAML output
kubectl get pod mypod -o yaml

# JSON output
kubectl get pod mypod -o json

# Custom columns
kubectl get pods -o custom-columns="NAME:.metadata.name,STATUS:.status.phase,NODE:.spec.nodeName"

# Watch (live updates)
kubectl get pods -w
kubectl get pods --watch

# Sort output
kubectl get pods --sort-by=.metadata.creationTimestamp
kubectl get pods --sort-by=.status.startTime

# Filter by label
kubectl get pods -l app=myapp
kubectl get pods -l 'app=myapp,env=prod'
kubectl get pods -l 'env in (prod,staging)'

# Get everything in a namespace
kubectl get all -n myapp

# Get all resource types
kubectl get all --all-namespaces
```

### Describing Resources

```bash
# Detailed info including events
kubectl describe pod mypod
kubectl describe deployment myapp
kubectl describe service myapp
kubectl describe node node-01
kubectl describe ingress myapp

# Events are at the bottom — most useful for debugging
kubectl describe pod mypod | tail -30
```

### Apply and Delete

```bash
# Apply (create or update)
kubectl apply -f manifest.yaml
kubectl apply -f ./k8s/             # apply all YAML files in directory
kubectl apply -f https://raw.githubusercontent.com/org/repo/main/k8s/deploy.yaml

# Dry run (validate without applying)
kubectl apply -f manifest.yaml --dry-run=client
kubectl apply -f manifest.yaml --dry-run=server   # more thorough — checks against API

# Show diff before applying
kubectl diff -f manifest.yaml

# Delete resources
kubectl delete -f manifest.yaml
kubectl delete pod mypod
kubectl delete deployment myapp
kubectl delete pod mypod --grace-period=0 --force   # immediate deletion

# Delete all pods matching label
kubectl delete pods -l app=myapp
```

### Logs

```bash
# Container logs
kubectl logs mypod
kubectl logs mypod -c mycontainer    # specific container in multi-container pod
kubectl logs -f mypod                # follow (stream)
kubectl logs --tail=100 mypod        # last 100 lines
kubectl logs --since=1h mypod        # logs from last hour
kubectl logs --since-time="2024-01-15T10:00:00Z" mypod

# Logs for all pods of a deployment
kubectl logs -l app=myapp --all-containers

# Previous container logs (after a crash)
kubectl logs mypod --previous
kubectl logs mypod -p
```

### Exec and Port-Forward

```bash
# Open a shell in a pod
kubectl exec -it mypod -- sh
kubectl exec -it mypod -- bash
kubectl exec -it mypod -c mycontainer -- sh

# Run a one-off command
kubectl exec mypod -- ls /app
kubectl exec mypod -- env | grep DATABASE

# Port forward pod to local
kubectl port-forward pod/mypod 8080:3000

# Port forward service to local
kubectl port-forward svc/myapp 8080:80

# Port forward deployment (picks a random pod)
kubectl port-forward deployment/myapp 8080:3000

# Background port-forward
kubectl port-forward svc/myapp 8080:80 &
```

### Rollout Management

```bash
# Check rollout status
kubectl rollout status deployment/myapp

# View rollout history
kubectl rollout history deployment/myapp
kubectl rollout history deployment/myapp --revision=3

# Undo last rollout
kubectl rollout undo deployment/myapp

# Undo to specific revision
kubectl rollout undo deployment/myapp --to-revision=2

# Pause a rollout (during canary testing)
kubectl rollout pause deployment/myapp

# Resume a paused rollout
kubectl rollout resume deployment/myapp

# Restart all pods in a deployment (rolling)
kubectl rollout restart deployment/myapp
```

### Scaling

```bash
# Scale a deployment
kubectl scale deployment myapp --replicas=5
kubectl scale deployment myapp --replicas=0   # scale to zero

# Scale multiple deployments
kubectl scale deployment frontend backend --replicas=3

# Conditional scale (only if current count matches)
kubectl scale deployment myapp --current-replicas=3 --replicas=5
```

### Resource Usage (requires metrics-server)

```bash
# Node resource usage
kubectl top nodes
kubectl top node node-01

# Pod resource usage
kubectl top pods
kubectl top pods -n myapp
kubectl top pods --sort-by=cpu
kubectl top pods --sort-by=memory
```

## Namespaces

```bash
# List namespaces
kubectl get namespaces
kubectl get ns

# Default namespaces
# default — where resources go when no namespace is specified
# kube-system — K8s system components
# kube-public — publicly readable
# kube-node-lease — node heartbeat leases

# Create namespace
kubectl create namespace myapp
# Or declaratively:
kubectl apply -f - <<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: myapp
  labels:
    app.kubernetes.io/part-of: myplatform
EOF

# Run commands in a namespace
kubectl get pods -n myapp
kubectl apply -f manifest.yaml -n myapp

# Set default namespace for current context
kubectl config set-context --current --namespace=myapp

# Delete namespace (and ALL resources within it)
kubectl delete namespace myapp
```

## Imperative vs Declarative

<Tabs>
  <TabItem label="Imperative (Quick)">
    Good for ad-hoc operations, troubleshooting, and getting started quickly.

    ```bash
    # Create deployment
    kubectl create deployment myapp --image=nginx:alpine --replicas=3

    # Expose deployment as a service
    kubectl expose deployment myapp --port=80 --target-port=80 --type=ClusterIP

    # Create a ConfigMap
    kubectl create configmap my-config --from-literal=KEY=value --from-file=config.properties

    # Create a Secret
    kubectl create secret generic my-secret --from-literal=password=mysecret

    # Run a temporary pod
    kubectl run tmp-shell --image=busybox --rm -it --restart=Never -- sh

    # Quick debug pod
    kubectl run debug --image=nicolaka/netshoot --rm -it --restart=Never -- bash
    ```

</TabItem>
  <TabItem label="Declarative (Production)">
    Preferred for anything that should be version-controlled and repeatable.

    ```yaml
    # deployment.yaml
    apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: myapp
      namespace: production
      labels:
        app: myapp
    spec:
      replicas: 3
      selector:
        matchLabels:
          app: myapp
      template:
        metadata:
          labels:
            app: myapp
        spec:
          containers:
            - name: myapp
              image: myapp:v1.2.3
              ports:
                - containerPort: 3000
    ```

    ```bash
    kubectl apply -f deployment.yaml
    kubectl apply -f ./k8s/          # all files in directory
    ```

</TabItem>
</Tabs>

## Useful Tools and Aliases

```bash
# Aliases — add to ~/.bashrc or ~/.zshrc
alias k='kubectl'
alias kg='kubectl get'
alias kd='kubectl describe'
alias kl='kubectl logs'
alias ke='kubectl exec -it'
alias kaf='kubectl apply -f'
alias kdelf='kubectl delete -f'
alias kns='kubectl config set-context --current --namespace'

# kubectx / kubens — fast context and namespace switching
brew install kubectx

kubectx                    # list contexts
kubectx prod               # switch to prod context
kubens myapp               # switch to myapp namespace

# k9s — terminal UI for Kubernetes
brew install k9s
k9s                        # opens TUI; navigate with arrows, : for commands

# stern — multi-pod log tailing
brew install stern
stern myapp                # tail logs for all pods matching "myapp"
stern -n myapp .           # all pods in namespace
```

## kubectl explain

```bash
# Explain any resource type or field
kubectl explain pod
kubectl explain pod.spec
kubectl explain pod.spec.containers
kubectl explain pod.spec.containers.resources
kubectl explain deployment.spec.strategy

# List all API resources
kubectl api-resources

# List resources with API versions
kubectl api-resources -o wide

# Get specific API version info
kubectl api-versions
```

:::tip[Generate YAML from Imperative Commands]
Use `--dry-run=client -o yaml` to generate YAML templates from imperative commands:

```bash
kubectl create deployment myapp --image=nginx --dry-run=client -o yaml > deployment.yaml
kubectl expose deployment myapp --port=80 --dry-run=client -o yaml > service.yaml
kubectl create configmap myconfig --from-literal=key=val --dry-run=client -o yaml
```

This is a fast way to get a correctly structured YAML file to customise.
:::

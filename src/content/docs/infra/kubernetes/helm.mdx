# Helm

Helm is the package manager for Kubernetes. It bundles Kubernetes manifests into reusable **charts**, manages releases with versioning and rollback, and provides a powerful templating engine for configuration.

## Why Helm?

Without Helm, deploying an application requires managing 5–15 YAML files with repeated values (app name, image tag, namespace). Helm:

- **Packages** all those files into a single chart
- **Templates** values so one chart deploys to dev, staging, and prod
- **Tracks releases** — knows what's deployed and can roll back
- **Manages dependencies** — pulls in charts for PostgreSQL, Redis, etc.
- **Shares** via registries (Artifact Hub, OCI registries)

## Installing Helm

```bash
# macOS
brew install helm

# Linux
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# Windows
winget install Helm.Helm
# or
choco install kubernetes-helm

# Verify
helm version
```

## Chart Structure

```
mychart/
├── Chart.yaml              # Chart metadata (name, version, description)
├── values.yaml             # Default configuration values
├── values.schema.json      # Optional JSON schema to validate values
├── charts/                 # Dependency charts (downloaded here)
├── crds/                   # Custom Resource Definitions (installed first)
├── templates/              # Kubernetes manifest templates
│   ├── _helpers.tpl        # Template helpers and partials
│   ├── deployment.yaml
│   ├── service.yaml
│   ├── ingress.yaml
│   ├── configmap.yaml
│   ├── secret.yaml
│   ├── serviceaccount.yaml
│   ├── hpa.yaml
│   ├── NOTES.txt           # Shown to user after helm install
│   └── tests/
│       └── test-connection.yaml
└── .helmignore             # Files to exclude from chart archive
```

### Chart.yaml

```yaml
apiVersion: v2 # Helm 3 chart format (v2)
name: myapp
version: 1.2.3 # Chart version (SemVer)
appVersion: '2.5.1' # Application version (informational)
description: My application Helm chart
type: application # application | library

# Optional metadata
keywords:
  - nodejs
  - web
  - api
home: https://github.com/org/myapp
sources:
  - https://github.com/org/myapp
maintainers:
  - name: Jane Smith
    email: jane@example.com
    url: https://example.com
icon: https://example.com/icon.png
annotations:
  category: WebApp

# Dependencies
dependencies:
  - name: postgresql
    version: '13.2.x'
    repository: https://charts.bitnami.com/bitnami
    condition: postgresql.enabled # only install if values.postgresql.enabled=true
  - name: redis
    version: '18.x.x'
    repository: https://charts.bitnami.com/bitnami
    condition: redis.enabled
```

## Template Syntax

### Built-in Objects

```yaml
# .Release — information about the current release
{{ .Release.Name }}           # release name
{{ .Release.Namespace }}      # target namespace
{{ .Release.IsInstall }}      # true on first install
{{ .Release.IsUpgrade }}      # true on upgrade
{{ .Release.Revision }}       # revision number (1 on install)

# .Chart — Chart.yaml contents
{{ .Chart.Name }}
{{ .Chart.Version }}
{{ .Chart.AppVersion }}

# .Values — merged values (default + overrides)
{{ .Values.image.repository }}
{{ .Values.replicaCount }}

# .Files — access files in the chart
{{ .Files.Get "config/app.conf" }}

# .Capabilities — K8s API capabilities
{{ .Capabilities.KubeVersion.Major }}
{{ .Capabilities.APIVersions.Has "networking.k8s.io/v1/Ingress" }}
```

### Template Functions and Pipelines

```yaml
# Quote strings (important for values that might be numbers)
image: {{ .Values.image.repository }}:{{ .Values.image.tag | quote }}

# Default value
tag: {{ .Values.image.tag | default .Chart.AppVersion | quote }}

# Required value (fail if not set)
password: {{ required "A password is required!" .Values.auth.password | b64enc }}

# Base64 encoding
password: {{ .Values.auth.password | b64enc }}

# Upper/lower case
name: {{ .Values.name | upper }}

# Truncate and trim
name: {{ .Release.Name | trunc 63 | trimSuffix "-" }}

# Replace
name: {{ .Values.name | replace "." "-" }}

# toString
port: {{ .Values.service.port | toString }}

# Type check with kindIs
{{- if kindIs "string" .Values.image.tag }}

# nindent — add indentation with leading newline (essential for embedded YAML)
annotations:
  {{- toYaml .Values.podAnnotations | nindent 4 }}

# indent — add indentation without leading newline
  {{- toYaml .Values.podAnnotations | indent 4 }}

# include — render a named template and return as string (usable in pipelines)
name: {{ include "myapp.fullname" . }}
labels:
  {{- include "myapp.labels" . | nindent 4 }}
```

### Control Flow

```yaml
# if / else if / else
{{- if .Values.ingress.enabled }}
# ... ingress manifest
{{- else if .Values.service.type "LoadBalancer" }}
# ... alternative
{{- else }}
# ... fallback
{{- end }}

# with — changes context to a value (skips block if nil/empty)
{{- with .Values.nodeSelector }}
nodeSelector:
  {{- toYaml . | nindent 2 }}
{{- end }}

# range — iterate over lists
{{- range .Values.ingress.hosts }}
- host: {{ .host }}
  http:
    paths:
    {{- range .paths }}
      - path: {{ .path }}
        pathType: {{ .pathType }}
    {{- end }}
{{- end }}

# range with index
{{- range $i, $val := .Values.envVars }}
- name: VAR_{{ $i }}
  value: {{ $val }}
{{- end }}

# range over map
{{- range $key, $val := .Values.env }}
- name: {{ $key }}
  value: {{ $val | quote }}
{{- end }}
```

### Whitespace Control

```yaml
# {{-  removes whitespace BEFORE the action
# -}}  removes whitespace AFTER the action

# Without whitespace control — blank lines appear
labels:
  app: myapp
  {{ if .Values.extra }}
  extra: true
  {{ end }}

# With whitespace control — clean output
labels:
  app: myapp
  {{- if .Values.extra }}
  extra: "true"
  {{- end }}
```

## \_helpers.tpl

Define reusable template snippets:

```yaml
{{/*
Expand the name of the chart.
*/}}
{{- define "myapp.name" -}}
{{- default .Chart.Name .Values.nameOverride | trunc 63 | trimSuffix "-" }}
{{- end }}

{{/*
Create a default fully qualified app name.
*/}}
{{- define "myapp.fullname" -}}
{{- if .Values.fullnameOverride }}
{{- .Values.fullnameOverride | trunc 63 | trimSuffix "-" }}
{{- else }}
{{- $name := default .Chart.Name .Values.nameOverride }}
{{- if contains $name .Release.Name }}
{{- .Release.Name | trunc 63 | trimSuffix "-" }}
{{- else }}
{{- printf "%s-%s" .Release.Name $name | trunc 63 | trimSuffix "-" }}
{{- end }}
{{- end }}
{{- end }}

{{/*
Common labels — applied to all resources
*/}}
{{- define "myapp.labels" -}}
helm.sh/chart: {{ include "myapp.chart" . }}
{{ include "myapp.selectorLabels" . }}
{{- if .Chart.AppVersion }}
app.kubernetes.io/version: {{ .Chart.AppVersion | quote }}
{{- end }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end }}

{{/*
Selector labels
*/}}
{{- define "myapp.selectorLabels" -}}
app.kubernetes.io/name: {{ include "myapp.name" . }}
app.kubernetes.io/instance: {{ .Release.Name }}
{{- end }}

{{/*
Service account name
*/}}
{{- define "myapp.serviceAccountName" -}}
{{- if .Values.serviceAccount.create }}
{{- default (include "myapp.fullname" .) .Values.serviceAccount.name }}
{{- else }}
{{- default "default" .Values.serviceAccount.name }}
{{- end }}
{{- end }}
```

## Helm CLI Reference

### Install and Upgrade

```bash
# Install a chart
helm install myrelease ./mychart
helm install myrelease ./mychart -n mynamespace --create-namespace

# Install with value overrides
helm install myrelease ./mychart \
  --set image.tag=v2.0.0 \
  --set replicaCount=3

# Install with custom values file
helm install myrelease ./mychart -f values-prod.yaml

# Install with multiple values files (later files override earlier)
helm install myrelease ./mychart \
  -f values.yaml \
  -f values-prod.yaml \
  --set image.tag=v2.0.0

# Dry run (template rendering + validation, no apply)
helm install myrelease ./mychart --dry-run

# Template only (outputs YAML, no cluster connection)
helm template myrelease ./mychart -f values-prod.yaml

# Upgrade existing release
helm upgrade myrelease ./mychart -f values-prod.yaml

# Upgrade or install if not exists
helm upgrade --install myrelease ./mychart -f values-prod.yaml

# Wait for pods to be ready before marking upgrade complete
helm upgrade myrelease ./mychart --wait --timeout 5m

# Automatically roll back on failure
helm upgrade myrelease ./mychart --atomic --timeout 5m
```

### List, Status, History

```bash
# List all releases
helm list
helm list -A              # all namespaces
helm list -n production

# Release status and info
helm status myrelease
helm status myrelease -n production

# Release history (revisions)
helm history myrelease
helm history myrelease -n production

# Show current values
helm get values myrelease
helm get values myrelease --all    # includes defaults

# Show rendered manifests
helm get manifest myrelease

# Show release notes
helm get notes myrelease
```

### Rollback and Uninstall

```bash
# Roll back to previous revision
helm rollback myrelease

# Roll back to specific revision
helm rollback myrelease 3

# Uninstall release
helm uninstall myrelease
helm uninstall myrelease -n production

# Uninstall but keep history
helm uninstall myrelease --keep-history
```

### Repositories

```bash
# Add a repository
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo add cert-manager https://charts.jetstack.io
helm repo add kedacore https://kedacore.github.io/charts

# Update repo index
helm repo update

# List repositories
helm repo list

# Search for charts
helm search repo nginx
helm search repo bitnami/postgresql --versions

# Remove a repo
helm repo remove bitnami
```

## Chart Dependencies

```bash
# After adding dependencies to Chart.yaml
helm dependency update ./mychart

# This creates:
# charts/postgresql-13.2.3.tgz
# charts/redis-18.0.1.tgz
# Chart.lock                     # pinned versions

# Install chart with dependencies
helm install myrelease ./mychart

# Disable a dependency at install time
helm install myrelease ./mychart --set postgresql.enabled=false
```

## Artifact Hub

[artifacthub.io](https://artifacthub.io) is the CNCF hub for finding public Helm charts.

```bash
# Search Artifact Hub via CLI
helm search hub nginx
helm search hub postgresql

# Install a chart from Artifact Hub
helm repo add bitnami https://charts.bitnami.com/bitnami
helm install my-postgres bitnami/postgresql \
  --set auth.postgresPassword=mysecret \
  --set primary.persistence.size=20Gi
```

## OCI Registries for Charts

Modern Helm (3.8+) supports storing charts in OCI-compliant container registries.

```bash
# Login to OCI registry
helm registry login ghcr.io --username myuser --password "$GITHUB_TOKEN"

# Push a chart to OCI registry
helm package ./mychart
helm push mychart-1.2.3.tgz oci://ghcr.io/myorg/charts

# Install from OCI registry
helm install myrelease oci://ghcr.io/myorg/charts/mychart --version 1.2.3

# Pull chart without installing
helm pull oci://ghcr.io/myorg/charts/mychart --version 1.2.3
```

## Helmfile — Multi-Chart Deployments

Helmfile manages multiple Helm releases declaratively.

```bash
# Install Helmfile
brew install helmfile

# Apply all releases
helmfile apply

# Sync (detect drift and reconcile)
helmfile sync

# Diff before applying
helmfile diff

# Destroy all releases
helmfile destroy
```

```yaml
# helmfile.yaml
repositories:
  - name: bitnami
    url: https://charts.bitnami.com/bitnami
  - name: ingress-nginx
    url: https://kubernetes.github.io/ingress-nginx
  - name: cert-manager
    url: https://charts.jetstack.io

helmDefaults:
  wait: true
  timeout: 300
  atomic: true

releases:
  - name: cert-manager
    namespace: cert-manager
    createNamespace: true
    chart: cert-manager/cert-manager
    version: v1.14.0
    set:
      - name: installCRDs
        value: true

  - name: ingress-nginx
    namespace: ingress-nginx
    createNamespace: true
    chart: ingress-nginx/ingress-nginx
    version: 4.9.0
    values:
      - charts/ingress-nginx/values.yaml

  - name: postgresql
    namespace: data
    chart: bitnami/postgresql
    version: 13.2.x
    values:
      - charts/postgresql/values.yaml
    secrets:
      - charts/postgresql/secrets.yaml # Helm secrets (encrypted)

  - name: myapp
    namespace: production
    chart: ./charts/myapp
    values:
      - charts/myapp/values.yaml
      - charts/myapp/values-{{ .Environment.Name }}.yaml # environment-specific
    needs:
      - postgresql
      - ingress-nginx
```

## Helm Secrets Plugin

Encrypt values files with SOPS for safe GitOps storage.

```bash
# Install
helm plugin install https://github.com/jkroepke/helm-secrets --version v4.5.1

# Install SOPS
brew install sops

# Create SOPS config (.sops.yaml)
cat > .sops.yaml <<EOF
creation_rules:
  - path_regex: .*secrets.*
    kms: arn:aws:kms:us-east-1:123456:key/my-key-id
EOF

# Encrypt a values file
sops -e charts/myapp/secrets.yaml > charts/myapp/secrets.enc.yaml

# Use encrypted values with helm secrets
helm secrets install myrelease ./mychart \
  -f values.yaml \
  -f secrets.enc.yaml
```

## Chart Testing

```bash
# Lint the chart (syntax and best-practice check)
helm lint ./mychart
helm lint ./mychart -f values-prod.yaml

# Render templates locally
helm template myrelease ./mychart -f values-prod.yaml

# Render and pipe to kubectl dry-run
helm template myrelease ./mychart | kubectl apply --dry-run=server -f -

# Run chart tests (requires pods defined in templates/tests/)
helm test myrelease
helm test myrelease --logs    # show test pod logs

# Install with test validation
helm install myrelease ./mychart && helm test myrelease
```

```yaml
# templates/tests/test-connection.yaml
apiVersion: v1
kind: Pod
metadata:
  name: {{ include "myapp.fullname" . }}-test-connection
  annotations:
    helm.sh/hook: test
    helm.sh/hook-delete-policy: before-hook-creation,hook-succeeded
spec:
  restartPolicy: Never
  containers:
    - name: wget
      image: busybox
      command: ['wget', '--spider', '-q',
                '{{ include "myapp.fullname" . }}:{{ .Values.service.port }}/health']
```

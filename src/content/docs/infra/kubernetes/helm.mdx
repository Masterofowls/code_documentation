# Helm

Helm is the package manager for Kubernetes. It packages, configures, and deploys applications using **charts** — reusable, versioned collections of Kubernetes manifests with parameterised templating.

## Why Helm?

Without Helm, managing a multi-resource Kubernetes application means maintaining many YAML files, manually updating image tags and replica counts across files, and inventing your own upgrade/rollback strategy.

Helm solves this with:
- **Charts** — versioned packages of related K8s resources
- **Values** — parameterisation of any field in any manifest
- **Releases** — named instances of a chart with tracked history
- **Rollback** — atomic rollback to any previous revision

## Installation

```bash
# macOS
brew install helm

# Linux
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# Verify
helm version
```

## Chart Structure

```
mychart/
├── Chart.yaml              # Chart metadata
├── values.yaml             # Default values
├── values-prod.yaml        # Environment override (not standard, by convention)
├── .helmignore             # Files to exclude from chart package
├── templates/              # Kubernetes manifest templates
│   ├── _helpers.tpl        # Reusable template helpers (not rendered directly)
│   ├── deployment.yaml
│   ├── service.yaml
│   ├── ingress.yaml
│   ├── configmap.yaml
│   ├── serviceaccount.yaml
│   ├── hpa.yaml
│   └── NOTES.txt           # Post-install instructions (shown after helm install)
├── charts/                 # Chart dependencies (downloaded by helm dep update)
└── crds/                   # Custom Resource Definitions (applied first)
```

### Chart.yaml

```yaml
apiVersion: v2            # v2 for Helm 3
name: myapp
description: My application Helm chart
type: application         # application | library
version: 0.1.0            # chart version (semver)
appVersion: "1.2.3"       # app version (informational)
icon: https://example.com/icon.png
keywords:
  - nodejs
  - api
home: https://github.com/myorg/myapp
sources:
  - https://github.com/myorg/myapp
maintainers:
  - name: Team Platform
    email: platform@example.com
dependencies:
  - name: postgresql
    version: "12.5.6"
    repository: https://charts.bitnami.com/bitnami
    condition: postgresql.enabled
  - name: redis
    version: "17.11.0"
    repository: https://charts.bitnami.com/bitnami
    condition: redis.enabled
```

## Template Syntax

Helm templates use Go templating (`{{ ... }}`).

### Built-in Objects

```yaml
# .Values — values from values.yaml and --set flags
image: {{ .Values.image.repository }}:{{ .Values.image.tag | default .Chart.AppVersion }}

# .Release — information about the current release
name: {{ .Release.Name }}-api
namespace: {{ .Release.Namespace }}
labels:
  app.kubernetes.io/managed-by: {{ .Release.Service }}

# .Chart — Chart.yaml contents
version: {{ .Chart.Version }}
appVersion: {{ .Chart.AppVersion }}

# .Capabilities — cluster capabilities
{{- if .Capabilities.APIVersions.Has "autoscaling/v2" }}

# .Files — access to chart files (non-template)
config: {{ .Files.Get "config/app.conf" | b64enc }}
```

### Control Structures

```yaml
# if / else
{{- if .Values.ingress.enabled }}
# ... ingress resources
{{- else }}
# no ingress
{{- end }}

# Comparison
{{- if gt (int .Values.replicaCount) 1 }}
strategy:
  type: RollingUpdate
{{- end }}

# range — iterate over a list
{{- range .Values.ingress.hosts }}
  - host: {{ .host | quote }}
{{- end }}

# range over map
env:
  {{- range $key, $val := .Values.env }}
  - name: {{ $key }}
    value: {{ $val | quote }}
  {{- end }}

# with — change scope
{{- with .Values.ingress.annotations }}
annotations:
  {{- toYaml . | nindent 4 }}
{{- end }}
```

### Functions

```yaml
# quote — wrap in double quotes
name: {{ .Values.name | quote }}

# upper / lower
env: {{ .Values.config.nodeEnv | upper }}

# default — use fallback if empty
tag: {{ .Values.image.tag | default .Chart.AppVersion | quote }}

# required — fail with message if empty
password: {{ required "You must set .Values.db.password" .Values.db.password }}

# toYaml + nindent — serialise a map/list with indentation
resources:
  {{- toYaml .Values.resources | nindent 2 }}

# b64enc / b64dec
data:
  config: {{ .Values.config | toJson | b64enc }}

# trunc + trimSuffix — enforce name length limits
{{- define "myapp.fullname" -}}
{{- printf "%s-%s" .Release.Name .Chart.Name | trunc 63 | trimSuffix "-" }}
{{- end }}

# sha256sum — force restart on config change
checksum/config: {{ include (print $.Template.BasePath "/configmap.yaml") . | sha256sum }}
```

### _helpers.tpl

Define reusable named templates in `_helpers.tpl`:

```
{{/*
Expand the name of the chart.
*/}}
{{- define "myapp.name" -}}
{{- default .Chart.Name .Values.nameOverride | trunc 63 | trimSuffix "-" }}
{{- end }}

{{/*
Create a default fully qualified app name.
*/}}
{{- define "myapp.fullname" -}}
{{- if .Values.fullnameOverride }}
{{- .Values.fullnameOverride | trunc 63 | trimSuffix "-" }}
{{- else }}
{{- $name := default .Chart.Name .Values.nameOverride }}
{{- printf "%s-%s" .Release.Name $name | trunc 63 | trimSuffix "-" }}
{{- end }}
{{- end }}

{{/*
Common labels
*/}}
{{- define "myapp.labels" -}}
helm.sh/chart: {{ include "myapp.chart" . }}
{{ include "myapp.selectorLabels" . }}
app.kubernetes.io/version: {{ .Chart.AppVersion | quote }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end }}

{{/*
Selector labels
*/}}
{{- define "myapp.selectorLabels" -}}
app.kubernetes.io/name: {{ include "myapp.name" . }}
app.kubernetes.io/instance: {{ .Release.Name }}
{{- end }}
```

Use in templates:

```yaml
metadata:
  name: {{ include "myapp.fullname" . }}
  labels:
    {{- include "myapp.labels" . | nindent 4 }}
```

## Helm CLI Commands

### Install and Upgrade

```bash
# Add a Helm repository
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update

# Install a chart
helm install myapp ./mychart
helm install myapp ./mychart --namespace myapp --create-namespace

# Install with value overrides
helm install myapp ./mychart \
  --set replicaCount=3 \
  --set image.tag=1.3.0

# Install with a values file
helm install myapp ./mychart -f values-prod.yaml

# Install from a public chart
helm install postgres bitnami/postgresql --version 12.5.6

# Upgrade an existing release
helm upgrade myapp ./mychart -f values-prod.yaml

# Install or upgrade (idempotent)
helm upgrade --install myapp ./mychart \
  --namespace myapp \
  --create-namespace \
  -f values-prod.yaml

# Wait for all resources to be ready
helm upgrade --install myapp ./mychart --wait --timeout 5m
```

### Inspect and List

```bash
# List all releases in current namespace
helm list
helm list -n myapp
helm list -A   # all namespaces

# Get release status
helm status myapp -n myapp

# View release history
helm history myapp -n myapp

# Show computed values
helm get values myapp -n myapp

# Show all values (including defaults)
helm get values myapp -n myapp --all

# Show rendered manifests
helm get manifest myapp -n myapp
```

### Rollback and Uninstall

```bash
# Rollback to previous revision
helm rollback myapp -n myapp

# Rollback to specific revision
helm rollback myapp 3 -n myapp

# Uninstall (removes all resources)
helm uninstall myapp -n myapp

# Keep history after uninstall
helm uninstall myapp --keep-history -n myapp
```

## Dry Run and Diff

```bash
# Render templates without applying
helm install myapp ./mychart --dry-run --debug

# Server-side dry run (validates against cluster)
helm install myapp ./mychart --dry-run=server

# Diff between current and new (requires helm-diff plugin)
helm plugin install https://github.com/databus23/helm-diff
helm diff upgrade myapp ./mychart -f values-prod.yaml
```

## Chart Dependencies

```bash
# Download dependencies listed in Chart.yaml
helm dependency update ./mychart

# List dependencies
helm dependency list ./mychart

# Build (package downloaded deps into charts/)
helm dependency build ./mychart
```

## Testing Charts

```bash
# Lint — check for errors and best practices
helm lint ./mychart
helm lint ./mychart -f values-prod.yaml

# Template render (dry run to stdout)
helm template myapp ./mychart -f values-prod.yaml

# Run chart tests (requires test pod definitions)
helm test myapp -n myapp
```

Test pod in `templates/tests/`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: {{ include "myapp.fullname" . }}-test
  annotations:
    helm.sh/hook: test
    helm.sh/hook-delete-policy: hook-succeeded
spec:
  restartPolicy: Never
  containers:
    - name: test
      image: curlimages/curl:latest
      command:
        - sh
        - -c
        - curl -f http://{{ include "myapp.fullname" . }}/health
```

## Helmfile

[Helmfile](https://helmfile.readthedocs.io) manages multiple Helm releases declaratively.

```yaml
# helmfile.yaml
repositories:
  - name: bitnami
    url: https://charts.bitnami.com/bitnami
  - name: ingress-nginx
    url: https://kubernetes.github.io/ingress-nginx

environments:
  production:
    values:
      - environments/production/values.yaml
  staging:
    values:
      - environments/staging/values.yaml

releases:
  - name: ingress-nginx
    namespace: ingress-nginx
    chart: ingress-nginx/ingress-nginx
    version: 4.7.0
    createNamespace: true

  - name: cert-manager
    namespace: cert-manager
    chart: jetstack/cert-manager
    version: v1.13.0
    set:
      - name: crds.enabled
        value: "true"

  - name: myapp
    namespace: myapp
    chart: ./charts/myapp
    values:
      - values/myapp.yaml
      - values/myapp.{{ .Environment.Name }}.yaml
    needs:
      - ingress-nginx/ingress-nginx
      - cert-manager/cert-manager
```

```bash
# Install
helmfile sync

# Diff
helmfile diff

# Template render
helmfile template

# Target environment
helmfile -e production sync

# Destroy all releases
helmfile destroy
```

## Helm Secrets Plugin

```bash
# Install
helm plugin install https://github.com/jkroepke/helm-secrets

# Encrypt a values file with age or GPG
helm secrets encrypt values-secrets.yaml > values-secrets.enc.yaml

# Install with encrypted values
helm secrets upgrade --install myapp ./mychart -f values-secrets.enc.yaml
```

## OCI Registries for Charts

Helm 3.8+ supports storing charts in OCI registries (like container images).

```bash
# Push chart to GHCR
helm package ./mychart
helm push mychart-0.1.0.tgz oci://ghcr.io/myorg/charts

# Install from OCI registry
helm install myapp oci://ghcr.io/myorg/charts/mychart --version 0.1.0

# Login to OCI registry
echo "$GITHUB_TOKEN" | helm registry login ghcr.io -u myuser --password-stdin
```

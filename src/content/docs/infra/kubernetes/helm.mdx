# Helm

Helm is the package manager for Kubernetes. It packages Kubernetes manifests into **charts** — reusable, configurable, versioned bundles that simplify deployment and management of complex applications.

---

## Why Helm?

Without Helm, deploying a complex application means managing dozens of individual YAML files with `kubectl apply`. Helm solves:

- **Packaging** — Bundle all manifests for an application together
- **Configuration** — Parameterize manifests with a `values.yaml` file
- **Versioning** — Track application releases and roll back when needed
- **Dependency management** — Bundle dependent charts (e.g., PostgreSQL alongside your app)
- **Reuse** — Community charts on Artifact Hub for common software

## Chart Structure

```
mychart/
├── Chart.yaml              # Chart metadata (name, version, description)
├── values.yaml             # Default configuration values
├── values.schema.json      # Optional: JSON Schema for values validation
├── README.md               # Documentation
├── LICENSE                 # License file
├── .helmignore             # Files to exclude from chart package
├── charts/                 # Dependent charts (subcharts)
│   └── postgresql-13.2.0.tgz
├── crds/                   # Custom Resource Definitions (installed first)
│   └── myapp-crd.yaml
└── templates/              # Kubernetes manifest templates
    ├── _helpers.tpl        # Template helper functions (not rendered directly)
    ├── deployment.yaml
    ├── service.yaml
    ├── ingress.yaml
    ├── configmap.yaml
    ├── secret.yaml
    ├── serviceaccount.yaml
    ├── hpa.yaml
    └── NOTES.txt           # Shown to user after helm install
```

### Chart.yaml

```yaml
apiVersion: v2                    # Helm 3 chart format
name: my-app
description: A production-ready web application
type: application                 # application | library
version: 1.2.3                    # Chart version (SemVer)
appVersion: "2.5.1"               # Application version (informational)
keywords:
  - web
  - api
home: https://example.com
sources:
  - https://github.com/myorg/my-app
maintainers:
  - name: Jane Smith
    email: jane@example.com
dependencies:
  - name: postgresql
    version: "13.2.0"
    repository: https://charts.bitnami.com/bitnami
    condition: postgresql.enabled   # Only install if postgresql.enabled=true
    alias: db                       # Use as 'db' in values.yaml
  - name: redis
    version: "18.x.x"
    repository: https://charts.bitnami.com/bitnami
    condition: redis.enabled
annotations:
  category: Application
```

## Helm CLI Commands

### Installing and Managing Releases

```bash
# Install a chart
helm install my-release ./mychart
helm install my-release ./mychart --namespace production --create-namespace

# Install from a remote repository
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update
helm install my-postgres bitnami/postgresql

# Install with custom values
helm install my-app ./mychart \
  --set image.tag=1.2.3 \
  --set replicaCount=3 \
  --set ingress.enabled=true

# Install with a values file
helm install my-app ./mychart -f values.production.yaml

# Combine multiple values files (later files override earlier)
helm install my-app ./mychart \
  -f values.yaml \
  -f values.production.yaml \
  -f values.secrets.yaml

# Upgrade a release
helm upgrade my-app ./mychart --set image.tag=1.2.4

# Install or upgrade (atomic)
helm upgrade --install my-app ./mychart \
  --namespace production \
  --create-namespace \
  --atomic \           # Roll back if upgrade fails
  --timeout 5m \
  -f values.production.yaml

# Rollback to previous release
helm rollback my-app
helm rollback my-app 2             # Roll back to revision 2

# Uninstall
helm uninstall my-app
helm uninstall my-app --keep-history  # Keep release history

# List releases
helm list
helm list --all-namespaces
helm list -n production

# Status of a release
helm status my-app

# Release history
helm history my-app
```

## Template Syntax

Helm uses Go templates with the [Sprig](https://masterminds.github.io/sprig/) function library.

### Built-in Objects

```yaml
# .Values — values from values.yaml (or --set/--values flags)
image: {{ .Values.image.repository }}:{{ .Values.image.tag | default .Chart.AppVersion }}

# .Release — release metadata
name: {{ .Release.Name }}-deployment
namespace: {{ .Release.Namespace }}
isUpgrade: {{ .Release.IsUpgrade }}

# .Chart — Chart.yaml metadata
app.kubernetes.io/version: {{ .Chart.AppVersion | quote }}

# .Files — access non-template files in chart directory
config: {{ .Files.Get "files/config.json" | b64enc }}

# .Capabilities — cluster capabilities
{{- if .Capabilities.APIVersions.Has "networking.k8s.io/v1/Ingress" }}
```

### Control Flow

```yaml
# if / else / end
{{- if .Values.ingress.enabled }}
apiVersion: networking.k8s.io/v1
kind: Ingress
{{- end }}

# if / else
{{- if eq .Values.service.type "LoadBalancer" }}
  type: LoadBalancer
{{- else if eq .Values.service.type "NodePort" }}
  type: NodePort
{{- else }}
  type: ClusterIP
{{- end }}

# range (loop)
{{- range .Values.ingress.hosts }}
- host: {{ .host | quote }}
  http:
    paths:
    {{- range .paths }}
    - path: {{ .path }}
      pathType: {{ .pathType }}
    {{- end }}
{{- end }}

# with (change scope)
{{- with .Values.resources }}
resources:
  {{- toYaml . | nindent 10 }}
{{- end }}
```

### Functions

```yaml
# quote — add quotes
image: {{ .Values.image.tag | quote }}

# default — use fallback if empty
tag: {{ .Values.image.tag | default .Chart.AppVersion }}

# required — error if value not set
password: {{ required "A database password is required!" .Values.db.password }}

# toYaml + nindent — convert value to YAML with indentation
resources:
  {{- toYaml .Values.resources | nindent 2 }}

# indent vs nindent (nindent adds leading newline)
labels:
  {{- include "my-app.labels" . | nindent 4 }}

# tpl — render a value as a template
{{- tpl .Values.configMap.data . | nindent 4 }}

# b64enc / b64dec
data:
  password: {{ .Values.password | b64enc }}

# sha256sum — for annotations to force pod restarts on config change
annotations:
  checksum/config: {{ include (print $.Template.BasePath "/configmap.yaml") . | sha256sum }}
```

### `_helpers.tpl` — Reusable Templates

```yaml
{{/*
Expand the name of the chart.
*/}}
{{- define "my-app.name" -}}
{{- default .Chart.Name .Values.nameOverride | trunc 63 | trimSuffix "-" }}
{{- end }}

{{/*
Create a default fully qualified app name.
*/}}
{{- define "my-app.fullname" -}}
{{- if .Values.fullnameOverride }}
{{- .Values.fullnameOverride | trunc 63 | trimSuffix "-" }}
{{- else }}
{{- $name := default .Chart.Name .Values.nameOverride }}
{{- printf "%s-%s" .Release.Name $name | trunc 63 | trimSuffix "-" }}
{{- end }}
{{- end }}

{{/*
Common labels
*/}}
{{- define "my-app.labels" -}}
helm.sh/chart: {{ include "my-app.chart" . }}
{{ include "my-app.selectorLabels" . }}
app.kubernetes.io/version: {{ .Chart.AppVersion | quote }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end }}

{{/*
Selector labels
*/}}
{{- define "my-app.selectorLabels" -}}
app.kubernetes.io/name: {{ include "my-app.name" . }}
app.kubernetes.io/instance: {{ .Release.Name }}
{{- end }}
```

Using helpers in templates:

```yaml
# deployment.yaml
metadata:
  name: {{ include "my-app.fullname" . }}
  labels:
    {{- include "my-app.labels" . | nindent 4 }}
spec:
  selector:
    matchLabels:
      {{- include "my-app.selectorLabels" . | nindent 6 }}
```

```bash

helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo add grafana https://grafana.github.io/helm-charts

# Update all repos
helm repo update

# Search for charts
helm search repo postgresql
helm search hub nginx               # Search Artifact Hub

# Show chart values
helm show values bitnami/postgresql
helm show chart bitnami/postgresql
helm show readme bitnami/postgresql
```

## Helmfile

[Helmfile](https://github.com/helmfile/helmfile) manages multiple Helm releases declaratively in a single file.

```yaml
# helmfile.yaml
environments:
  default:
    values:
      - environments/default.yaml
  production:
    values:
      - environments/production.yaml

repositories:
  - name: bitnami
    url: https://charts.bitnami.com/bitnami
  - name: ingress-nginx
    url: https://kubernetes.github.io/ingress-nginx

releases:
  - name: ingress-nginx
    namespace: ingress-nginx
    chart: ingress-nginx/ingress-nginx
    version: "4.9.0"
    values:
      - controller:
          replicaCount: 2

  - name: postgres
    namespace: data
    chart: bitnami/postgresql
    version: "13.2.0"
    values:
      - auth:
          username: myapp
          password: {{ requiredEnv "POSTGRES_PASSWORD" }}
          database: myapp

  - name: my-app
    namespace: production
    chart: ./charts/my-app
    version: "1.2.3"
    values:
      - values/common.yaml
      - values/{{ .Environment.Name }}.yaml
    needs:
      - data/postgres            # Deploy postgres first
```

```bash
# Apply all releases
helmfile apply

# Apply only specific releases
helmfile apply --selector name=my-app

# Diff planned changes
helmfile diff

# Sync all releases
helmfile sync

# Destroy all releases
helmfile destroy
```

## Helm Secrets Plugin

```bash
# Install
helm plugin install https://github.com/jkroepke/helm-secrets

# Encrypt a values file with SOPS + AWS KMS
sops --encrypt --kms arn:aws:kms:us-east-1:123456789:key/my-key \
  values.secrets.yaml > values.secrets.enc.yaml

# Install with encrypted values
helm secrets install my-app ./mychart \
  -f values.yaml \
  -f values.secrets.enc.yaml
```

## OCI Registries for Charts

Helm 3.8+ supports storing charts in OCI registries (Docker Hub, GHCR, ECR, etc.).

```bash
# Login to OCI registry
helm registry login ghcr.io --username myuser --password $GITHUB_TOKEN

# Push chart to OCI registry
helm package ./mychart
helm push mychart-1.2.3.tgz oci://ghcr.io/myorg/charts

# Install from OCI registry
helm install my-app oci://ghcr.io/myorg/charts/mychart --version 1.2.3

# ECR
aws ecr create-repository --repository-name helm-charts/my-app
aws ecr get-login-password --region us-east-1 \
  | helm registry login --username AWS --password-stdin \
    123456789.dkr.ecr.us-east-1.amazonaws.com
helm push mychart-1.2.3.tgz oci://123456789.dkr.ecr.us-east-1.amazonaws.com/helm-charts
```

## Testing Charts

```bash
# Lint — check for issues
helm lint ./mychart
helm lint ./mychart -f values.production.yaml

# Dry-run / template rendering
helm template my-app ./mychart > rendered.yaml
helm template my-app ./mychart -f values.production.yaml | kubectl apply --dry-run=server -f -

# Install and run tests
helm install my-app ./mychart
helm test my-app

# Test with a dedicated test pod
```

```yaml
# templates/tests/test-connection.yaml
apiVersion: v1
kind: Pod
metadata:
  name: "{{ include "my-app.fullname" . }}-test"
  annotations:
    "helm.sh/hook": test
    "helm.sh/hook-delete-policy": before-hook-creation,hook-succeeded
spec:
  restartPolicy: Never
  containers:
    - name: wget
      image: busybox:1.35
      command:
        - wget
        - --spider
        - "http://{{ include "my-app.fullname" . }}:{{ .Values.service.port }}/health"
```

:::tip[helm template for debugging]
`helm template` renders all templates to stdout without connecting to Kubernetes. Use it to verify your templates render correctly before installing, or pipe through `kubectl diff` to preview changes:

```bash
helm template my-app ./mychart -f values.prod.yaml | kubectl diff -f -
```
:::

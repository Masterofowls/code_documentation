# Kubernetes

Kubernetes (K8s) is an open-source container orchestration platform that automates deployment, scaling, and management of containerised applications. Originally developed at Google (based on internal system Borg), it was donated to the CNCF in 2014 and is now the industry standard for running containers at scale.

## The Container Orchestration Problem

Running a single container on a single host is straightforward. Running hundreds of containers across dozens of hosts requires answers to hard questions:

- Which host should run this container? (scheduling)
- What happens if a host dies? (self-healing)
- How do containers find each other? (service discovery)
- How do we roll out updates without downtime? (rolling updates)
- How do we scale services under load? (autoscaling)
- How do we manage configuration and secrets? (config management)
- How do we route external traffic? (ingress)

Kubernetes answers all of these questions with a declarative, API-driven control plane.

### Control Plane Components

**kube-apiserver**
The front door to Kubernetes. All API requests (from `kubectl`, other components, external clients) flow through the API server. It validates and persists objects to etcd.

**etcd**
A distributed key-value store that holds the entire cluster state. Every Kubernetes object is stored in etcd. Treat it as the single source of truth вҖ” if you lose etcd without a backup, you lose your cluster.

**kube-scheduler**
Watches for newly created Pods with no assigned node and selects a node for them to run on, considering resource requirements, affinity rules, taints/tolerations, and data locality.

**kube-controller-manager**
Runs a collection of controllers: Deployment controller, ReplicaSet controller, Node controller, Job controller, etc. Each controller watches cluster state and moves it toward the desired state.

**cloud-controller-manager**
Integrates with cloud provider APIs (AWS, GCP, Azure) to provision load balancers, persistent disks, and node lifecycle management.

### Data Plane Components

**kubelet**
An agent running on each worker node. It receives Pod specifications from the API server and ensures containers are running and healthy. Reports node and pod status back to the API server.

**kube-proxy**
Implements Kubernetes Services by maintaining iptables or IPVS rules on each node to forward traffic to the correct Pod endpoints.

**Container Runtime**
The software that runs containers: `containerd` (default), `CRI-O`, or Docker (deprecated as of K8s 1.24). Must implement the Container Runtime Interface (CRI).

## K8s vs Docker Swarm vs Nomad

| Feature | Kubernetes | Docker Swarm | HashiCorp Nomad |
|---|---|---|---|
| Complexity | High | Low | Medium |
| Learning curve | Steep | Gentle | Moderate |
| Ecosystem | Enormous (CNCF) | Limited | Growing |
| Auto-scaling | HPA, VPA, KEDA | Manual | Yes (enterprise) |
| Multi-tenancy | Namespaces + RBAC | Limited | Namespaces |
| Storage | Rich CSI ecosystem | Plugins | CSI + plugins |
| Community | Massive | Shrinking | Active |
| Best for | Large scale, cloud-native | Small teams, simple deployments | Multi-workload (containers + VMs + batch) |

:::tip[K8s is the industry default]
If you're running containers in production and expect to grow, invest in learning Kubernetes. The ecosystem вҖ” Helm, ArgoCD, cert-manager, Prometheus, Istio вҖ” is unmatched.
:::

## Core Concepts Overview

| Object | Purpose |
|---|---|
| **Pod** | Smallest deployable unit вҖ” one or more containers sharing network/storage |
| **Deployment** | Declares desired state for stateless workloads; manages ReplicaSets |
| **StatefulSet** | Like Deployment but with stable network identity and ordered updates |
| **DaemonSet** | Runs one Pod per node (logging agents, monitoring, CNI plugins) |
| **Job / CronJob** | Run-to-completion workloads and scheduled tasks |
| **Service** | Stable virtual IP and DNS name that load-balances traffic to Pods |
| **Ingress** | HTTP/HTTPS routing rules from external traffic to Services |
| **ConfigMap** | Non-sensitive configuration data as key-value pairs |
| **Secret** | Sensitive data (passwords, tokens, certs) вҖ” base64 encoded at rest |
| **PersistentVolumeClaim** | Request for storage from the cluster |
| **Namespace** | Virtual cluster for multi-tenancy and resource isolation |
| **ServiceAccount** | Identity for Pods to authenticate with the API server |
| **HorizontalPodAutoscaler** | Automatically scales Deployment replicas based on metrics |

# Kubernetes

Kubernetes (K8s) is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications. Originally developed at Google, it was donated to the Cloud Native Computing Foundation (CNCF) in 2014 and has become the de facto standard for running containers at scale.

---

## The Container Orchestration Problem

Running a single container on a single host is straightforward. But production applications require:

- **High availability** — survive node failures without downtime
- **Horizontal scaling** — add/remove replicas based on load
- **Rolling updates** — deploy new versions without downtime
- **Service discovery** — containers find each other dynamically
- **Load balancing** — distribute traffic across healthy replicas
- **Self-healing** — replace failed containers automatically
- **Resource management** — pack containers efficiently across nodes
- **Secret management** — inject credentials securely at runtime
- **Storage orchestration** — attach persistent volumes as needed

Kubernetes solves all of these problems declaratively — you describe the **desired state**, and Kubernetes continuously works to make reality match that declaration.

Kubernetes clusters are divided into a **control plane** and one or more **worker nodes** (the data plane).

### Control Plane Components

**kube-apiserver**
The front door to the cluster. All communication — from kubectl, controllers, and nodes — goes through the API server. It validates and processes REST requests, then persists state to etcd.

**etcd**
A distributed, consistent key-value store that holds the entire cluster state. etcd is the source of truth. If etcd data is lost without a backup, the cluster state is lost.

**kube-scheduler**
Watches for newly created Pods that have no assigned node and selects the best node based on resource requirements, affinity rules, taints/tolerations, and available capacity.

**kube-controller-manager**
Runs control loops (controllers) that watch the cluster state and take action to drive it toward the desired state. Includes: Node controller, Deployment controller, ReplicaSet controller, Service controller, and many others.

### Data Plane Components

**kubelet**
The node agent. It receives Pod specifications from the API server and ensures the described containers are running and healthy via the container runtime.

**kube-proxy**
Maintains network rules (iptables/IPVS) on each node to implement Kubernetes Services. Routes traffic to correct Pod endpoints.

**Container Runtime**
Executes containers. Kubernetes supports any CRI-compatible runtime: containerd (most common), CRI-O, or others.

## Kubernetes vs Other Orchestrators

| Feature | Kubernetes | Docker Swarm | HashiCorp Nomad |
|---|---|---|---|
| **Maturity** | Very mature (CNCF graduated) | Mature but declining | Active, growing |
| **Complexity** | High | Low | Medium |
| **Workload types** | Containers, VMs (KubeVirt) | Containers only | Containers, VMs, bare metal |
| **Auto-scaling** | HPA, VPA, KEDA, Cluster Autoscaler | Manual only | HPA (Enterprise) |
| **Networking** | CNI plugins (Calico, Cilium, etc.) | Built-in overlay | Built-in |
| **Storage** | CSI drivers — extensive | Limited volume support | CSI support |
| **Learning curve** | Steep | Gentle | Moderate |
| **Ecosystem** | Enormous (CNCF landscape) | Limited | Growing |
| **Cloud-native** | First-class | Limited | Good |

## Key Concepts Overview

**Pod**
The smallest deployable unit. A Pod wraps one or more containers that share network namespace and storage. Containers in a Pod communicate via localhost.

**Deployment**
Manages a ReplicaSet to maintain a desired number of identical Pod replicas. Handles rolling updates and rollbacks.

**Service**
A stable virtual IP and DNS name that load-balances traffic to a set of Pods. Pods come and go; Services provide consistent endpoints.

**Ingress**
Routes external HTTP/HTTPS traffic to Services based on hostname and path rules. Requires an Ingress controller (nginx, traefik, AWS ALB).

**ConfigMap**
Stores non-sensitive configuration data as key-value pairs. Injected into Pods as environment variables or volume-mounted files.

**Secret**
Stores sensitive data (passwords, tokens, certificates). Similar to ConfigMap but base64-encoded and handled with more care (RBAC restrictions, encryption at rest).

**PersistentVolumeClaim (PVC)**
A request for persistent storage. Kubernetes binds it to a PersistentVolume (or dynamically provisions one via a StorageClass).

**Namespace**
A virtual cluster within a cluster. Provides scope for names and a boundary for RBAC policies and resource quotas.

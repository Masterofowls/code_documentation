# Services & Ingress

A **Service** gives a stable endpoint to a dynamic set of Pods. An **Ingress** exposes HTTP/HTTPS routes from outside the cluster to services. Together they handle all traffic routing needs.

## Why Services Exist

Pods have ephemeral IPs that change when they restart or reschedule. Services provide a **stable virtual IP (ClusterIP)** and **DNS name** that always routes to healthy pods matching a label selector.

```
Client → Service (stable ClusterIP:Port) → Pod 1
                                         → Pod 2
                                         → Pod 3
```

## Service Types

### ClusterIP (Default)

Only accessible within the cluster. Used for internal service-to-service communication.

```yaml
apiVersion: v1
kind: Service
metadata:
  name: myapp
  namespace: production
  labels:
    app: myapp
spec:
  type: ClusterIP # default — internal only
  selector:
    app: myapp # routes to pods with this label
  ports:
    - name: http
      port: 80 # port the Service listens on
      targetPort: 3000 # port on the Pod
      protocol: TCP
    - name: metrics
      port: 9090
      targetPort: 9090
```

### NodePort

Exposes the service on each Node's IP at a static port (30000–32767). Accessible from outside using `NodeIP:NodePort`.

```yaml
apiVersion: v1
kind: Service
metadata:
  name: myapp-nodeport
spec:
  type: NodePort
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 3000
      nodePort: 30080 # optional — auto-assigned if omitted (30000-32767)
```

```bash
# Access via any node IP
curl http://<NODE_IP>:30080

# Get node IPs
kubectl get nodes -o wide
```

### LoadBalancer

Provisions a cloud load balancer (AWS ALB/NLB, GCP LB, Azure LB). Routes external traffic to pods.

```yaml
apiVersion: v1
kind: Service
metadata:
  name: myapp-lb
  annotations:
    # AWS-specific
    service.beta.kubernetes.io/aws-load-balancer-type: nlb
    service.beta.kubernetes.io/aws-load-balancer-internal: 'false'
    # GKE-specific
    cloud.google.com/load-balancer-type: External
spec:
  type: LoadBalancer
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 3000
  loadBalancerSourceRanges: # restrict access to these CIDRs
    - '10.0.0.0/8'
    - '203.0.113.0/24'
  externalTrafficPolicy:
    Local # Local | Cluster
    # Local: preserves client IP, may be uneven
    # Cluster: even distribution, loses client IP
```

```bash
# Get the external IP (may take a few minutes to provision)
kubectl get svc myapp-lb
kubectl get svc myapp-lb -o jsonpath='{.status.loadBalancer.ingress[0].hostname}'
```

### ExternalName

Maps a Service to an external DNS name. No proxying — just DNS CNAME.

```yaml
apiVersion: v1
kind: Service
metadata:
  name: external-db
spec:
  type: ExternalName
  externalName: my-database.us-east-1.rds.amazonaws.com
# Pods can now reach the RDS instance at "external-db"
# external-db.namespace.svc.cluster.local → my-database...rds.amazonaws.com
```

### Headless Service

No ClusterIP — DNS returns Pod IPs directly. Used by StatefulSets for stable per-pod DNS.

```yaml
apiVersion: v1
kind: Service
metadata:
  name: postgres-headless
spec:
  clusterIP: None # makes it headless
  selector:
    app: postgres
  ports:
    - port: 5432
      targetPort: 5432
```

```bash
# DNS lookup returns individual pod IPs
# postgres-0.postgres-headless.namespace.svc.cluster.local
# postgres-1.postgres-headless.namespace.svc.cluster.local
```

## Service DNS

Kubernetes runs a cluster DNS server (CoreDNS) that automatically creates DNS records for every Service.

```
# Full DNS name format
<service-name>.<namespace>.svc.cluster.local

# Within the same namespace — short name works
curl http://myapp
curl http://myapp:80

# Cross-namespace
curl http://myapp.production
curl http://myapp.production.svc.cluster.local

# StatefulSet pod DNS (headless service required)
# postgres-0.postgres-headless.production.svc.cluster.local
```

```bash
# Debug DNS from inside a pod
kubectl exec -it debug-pod -- nslookup myapp
kubectl exec -it debug-pod -- nslookup myapp.production.svc.cluster.local

# Test connectivity
kubectl exec -it debug-pod -- curl http://myapp:80/health
```

## Endpoints and EndpointSlices

Services discover backing pods through Endpoints (and modern EndpointSlices):

```bash
# View endpoints for a service
kubectl get endpoints myapp
kubectl get endpointslices -l kubernetes.io/service-name=myapp

# If endpoints are empty — your selector doesn't match any pod labels
kubectl get pods --show-labels | grep app=myapp
```

## Ingress

An Ingress defines HTTP(S) routing rules and requires an **Ingress Controller** (nginx, Traefik, AWS ALB, etc.) to be installed.

### Basic Ingress

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp-ingress
  namespace: production
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/use-regex: 'true'
spec:
  ingressClassName: nginx # which IngressClass to use

  # ── TLS ───────────────────────────────────────────────────────
  tls:
    - hosts:
        - myapp.example.com
        - www.myapp.example.com
      secretName: myapp-tls # TLS secret (created by cert-manager)

  # ── Rules ─────────────────────────────────────────────────────
  rules:
    - host: myapp.example.com
      http:
        paths:
          - path: /api
            pathType: Prefix
            backend:
              service:
                name: api-service
                port:
                  name: http
          - path: /
            pathType: Prefix
            backend:
              service:
                name: frontend-service
                port:
                  number: 80

    - host: admin.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: admin-service
                port:
                  number: 80
```

### Path Types

| Type                     | Behaviour                                                |
| ------------------------ | -------------------------------------------------------- |
| `Prefix`                 | Matches path prefix; `/foo` matches `/foo`, `/foo/bar`   |
| `Exact`                  | Exact path match only; `/foo` matches only `/foo`        |
| `ImplementationSpecific` | Behaviour defined by IngressClass (controller-dependent) |

### Default Backend

```yaml
spec:
  defaultBackend: # handles requests that don't match any rule
    service:
      name: default-service
      port:
        number: 80
```

### IngressClass

```yaml
apiVersion: networking.k8s.io/v1
kind: IngressClass
metadata:
  name: nginx
  annotations:
    ingressclass.kubernetes.io/is-default-class: 'true' # make default
spec:
  controller: k8s.io/ingress-nginx
```

## Nginx Ingress Controller — Common Annotations

```yaml
annotations:
  # SSL redirect
  nginx.ingress.kubernetes.io/ssl-redirect: 'true'
  nginx.ingress.kubernetes.io/force-ssl-redirect: 'true'

  # CORS
  nginx.ingress.kubernetes.io/enable-cors: 'true'
  nginx.ingress.kubernetes.io/cors-allow-origin: 'https://myapp.com'

  # Rate limiting
  nginx.ingress.kubernetes.io/limit-rps: '10'
  nginx.ingress.kubernetes.io/limit-burst-multiplier: '3'

  # Request size
  nginx.ingress.kubernetes.io/proxy-body-size: '50m'

  # Timeouts
  nginx.ingress.kubernetes.io/proxy-connect-timeout: '60'
  nginx.ingress.kubernetes.io/proxy-read-timeout: '60'
  nginx.ingress.kubernetes.io/proxy-send-timeout: '60'

  # Sticky sessions
  nginx.ingress.kubernetes.io/affinity: 'cookie'
  nginx.ingress.kubernetes.io/session-cookie-name: 'INGRESSCOOKIE'

  # WebSocket support
  nginx.ingress.kubernetes.io/proxy-http-version: '1.1'

  # Auth
  nginx.ingress.kubernetes.io/auth-type: basic
  nginx.ingress.kubernetes.io/auth-secret: basic-auth
  nginx.ingress.kubernetes.io/auth-realm: 'Authentication Required'

  # Custom headers
  nginx.ingress.kubernetes.io/configuration-snippet: |
    more_set_headers "X-Frame-Options: DENY";
    more_set_headers "X-Content-Type-Options: nosniff";
```

## TLS with cert-manager

cert-manager automates TLS certificate provisioning from Let's Encrypt or other CAs.

```bash
# Install cert-manager
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.14.0/cert-manager.yaml

# Verify installation
kubectl get pods -n cert-manager
```

```yaml
# ClusterIssuer for Let's Encrypt (HTTP-01 challenge)
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: admin@example.com
    privateKeySecretRef:
      name: letsencrypt-prod-key
    solvers:
      - http01:
          ingress:
            ingressClassName: nginx
```

```yaml
# Ingress with cert-manager annotation — automatically provisions certificate
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp-ingress
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
spec:
  ingressClassName: nginx
  tls:
    - hosts:
        - myapp.example.com
      secretName: myapp-tls # cert-manager creates this Secret
  rules:
    - host: myapp.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: myapp
                port:
                  number: 80
```

```bash
# Check certificate status
kubectl get certificate -n production
kubectl describe certificate myapp-tls -n production

# Check cert-manager logs
kubectl logs -n cert-manager -l app=cert-manager
```

## Gateway API (Modern Alternative to Ingress)

The Gateway API is the next generation of Kubernetes traffic management, replacing Ingress with more expressive resources.

```yaml
# GatewayClass — defines the controller
apiVersion: gateway.networking.k8s.io/v1
kind: GatewayClass
metadata:
  name: nginx
spec:
  controllerName: k8s.nginx.org/nginx-gateway-controller
---
# Gateway — the actual load balancer / listener
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: main-gateway
  namespace: production
spec:
  gatewayClassName: nginx
  listeners:
    - name: https
      protocol: HTTPS
      port: 443
      tls:
        certificateRefs:
          - name: myapp-tls
---
# HTTPRoute — routing rules (replaces Ingress rules)
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: myapp-route
  namespace: production
spec:
  parentRefs:
    - name: main-gateway
  hostnames:
    - myapp.example.com
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /api
      backendRefs:
        - name: api-service
          port: 80
          weight: 100
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: frontend-service
          port: 80
```

## External DNS

ExternalDNS automatically creates DNS records in Route53, Cloud DNS, Azure DNS, etc. when LoadBalancer/Ingress resources are created.

```yaml
# Ingress with external-dns annotation
metadata:
  annotations:
    external-dns.alpha.kubernetes.io/hostname: myapp.example.com
    external-dns.alpha.kubernetes.io/ttl: '60'
```

:::tip[Service vs Ingress — When to Use Which]

- **ClusterIP** — internal service-to-service communication (always use this between microservices)
- **LoadBalancer** — directly expose a TCP/UDP service (databases with external access, non-HTTP services)
- **Ingress** — HTTP/HTTPS routing for web apps and APIs (one load balancer for many services)
- **Gateway API** — new projects where the controller supports it; more flexible than Ingress
  :::

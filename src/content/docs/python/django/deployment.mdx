Deploying Django to production requires disabling debug mode, configuring a production-grade web server, securing environment variables, serving static files efficiently, and ensuring HTTPS. This guide covers every step.

---

## Pre-Deployment Checklist

Run this command first — Django checks your settings for common security issues:

```bash
python manage.py check --deploy
```

Expected output (aim for zero issues):

```
System check identified no issues (0 silenced).
```

Common warnings and how to fix them:

| Warning                          | Fix                                  |
| -------------------------------- | ------------------------------------ |
| `SECURITY WARNING: DEBUG = True` | Set `DEBUG = False` in production    |
| `ALLOWED_HOSTS not set`          | Set `ALLOWED_HOSTS = ["myapp.com"]`  |
| `SECRET_KEY too simple/short`    | Generate a new 50+ char random key   |
| `SECURE_SSL_REDIRECT not set`    | Set `SECURE_SSL_REDIRECT = True`     |
| `SESSION_COOKIE_SECURE not set`  | Set `SESSION_COOKIE_SECURE = True`   |
| `CSRF_COOKIE_SECURE not set`     | Set `CSRF_COOKIE_SECURE = True`      |
| `SECURE_HSTS_SECONDS not set`    | Set `SECURE_HSTS_SECONDS = 31536000` |

```python
# mysite/settings/production.py
from .base import *

        default=config("DATABASE_URL"),
```

## Static Files with WhiteNoise

WhiteNoise lets Django serve static files efficiently without a separate web server (great for PaaS platforms).

```bash
pip install whitenoise
```

```python
# settings.py
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "whitenoise.middleware.WhiteNoiseMiddleware",  # ← Must be second
    ...
]

STATICFILES_STORAGE = "whitenoise.storage.CompressedManifestStaticFilesStorage"
# Compresses and fingerprints static files for optimal caching
```

```bash
# Collect static files before deploying
python manage.py collectstatic --no-input
```

WhiteNoise features:

- **Compression**: serves gzip/brotli compressed static files
- **Cache headers**: sets far-future `Cache-Control: max-age=31536000` on versioned files
- **Content hashing**: filenames include content hash (e.g., `app.abc123.css`)

## Gunicorn — WSGI Server

```bash
pip install gunicorn
```

```bash
# Basic usage
gunicorn mysite.wsgi:application

# With options (recommended for production)
gunicorn mysite.wsgi:application \
  --bind 0.0.0.0:8000 \
  --workers 4 \
  --worker-class gthread \
  --threads 2 \
  --timeout 120 \
  --keepalive 5 \
  --max-requests 1000 \
  --max-requests-jitter 50 \
  --access-logfile - \
  --error-logfile - \
  --log-level info
```

**Worker formula**: `(2 × CPU cores) + 1`

```bash
# 4-core machine: 2*4+1 = 9 workers
gunicorn mysite.wsgi:application --workers 9
```

Create a `gunicorn.conf.py` for reusable config:

```python
# gunicorn.conf.py
import multiprocessing

bind = "0.0.0.0:8000"
workers = multiprocessing.cpu_count() * 2 + 1
worker_class = "gthread"
threads = 2
timeout = 120
keepalive = 5
max_requests = 1000
max_requests_jitter = 50
accesslog = "-"     # stdout
errorlog = "-"      # stderr
loglevel = "info"
preload_app = True  # Load app code before forking workers (saves memory)
```

```bash
gunicorn -c gunicorn.conf.py mysite.wsgi:application
```

## Nginx Configuration

Nginx sits in front of gunicorn, handles SSL termination, static files, and proxying.

```nginx
# /etc/nginx/sites-available/myapp
upstream django {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name myapp.com www.myapp.com;

    # Redirect all HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name myapp.com www.myapp.com;

    # SSL certificates (use Certbot/Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/myapp.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/myapp.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers off;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256';
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options DENY;
    add_header Referrer-Policy "strict-origin-when-cross-origin";

    # Serve static files directly (bypasses Python entirely)
    location /static/ {
        alias /var/www/myapp/staticfiles/;
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # Serve media files
    location /media/ {
        alias /var/www/myapp/media/;
        expires 1d;
        add_header Cache-Control "public";
    }

    # Pass all other requests to gunicorn
    location / {
        proxy_pass http://django;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;

        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;

        # Buffer settings
        proxy_buffer_size 128k;
        proxy_buffers 4 256k;

        # File upload size
        client_max_body_size 25M;
    }
}
```

```bash
# Enable the site and reload nginx
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
sudo nginx -t        # Test config
sudo systemctl reload nginx
```

## Dockerfile

```dockerfile
# Dockerfile
FROM python:3.12-slim

# Set environment variables
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PIP_NO_CACHE_DIR=1 \
    PIP_DISABLE_PIP_VERSION_CHECK=1

# Create non-root user
RUN groupadd --gid 1000 appuser && \
    useradd --uid 1000 --gid appuser --shell /bin/bash --create-home appuser

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq-dev \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY --chown=appuser:appuser . .

# Switch to non-root user
USER appuser

# Collect static files
RUN python manage.py collectstatic --no-input --settings=mysite.settings.production

EXPOSE 8000

# Run gunicorn
CMD ["gunicorn", "mysite.wsgi:application", \
     "--bind", "0.0.0.0:8000", \
     "--workers", "4", \
     "--worker-class", "gthread", \
     "--threads", "2", \
     "--timeout", "120"]
```

:::tip[Multi-Stage Build]
For even smaller images, use a multi-stage Dockerfile with a builder stage for compiling dependencies and a slim runtime stage.
:::

## Docker Compose (with PostgreSQL and Redis)

```yaml
# docker-compose.yml
version: '3.9'

services:
  db:
    image: postgres:16-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: myappdb
      POSTGRES_USER: myappuser
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U myappuser -d myappdb']
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  web:
    build: .
    command: >
      sh -c "python manage.py migrate &&
             python manage.py collectstatic --no-input &&
             gunicorn mysite.wsgi:application --bind 0.0.0.0:8000 --workers 4"
    volumes:
      - static_files:/app/staticfiles
      - media_files:/app/media
    ports:
      - '8000:8000'
    env_file:
      - .env
    environment:
      DATABASE_URL: postgres://myappuser:${DB_PASSWORD}@db:5432/myappdb
      REDIS_URL: redis://redis:6379/1
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - static_files:/app/staticfiles:ro
      - media_files:/app/media:ro
    ports:
      - '80:80'
      - '443:443'
    depends_on:
      - web
    restart: unless-stopped

  celery:
    build: .
    command: celery -A mysite worker -l info -c 4
    env_file:
      - .env
    environment:
      DATABASE_URL: postgres://myappuser:${DB_PASSWORD}@db:5432/myappdb
      REDIS_URL: redis://redis:6379/0
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
  static_files:
  media_files:
```

```bash
# Start all services
docker compose up -d

# View logs
docker compose logs -f web

# Run management commands
docker compose exec web python manage.py createsuperuser
docker compose exec web python manage.py migrate

# Stop everything
docker compose down

# Stop and remove volumes (destroys data)
docker compose down -v
```

## Database Migrations in Production

:::caution[Always Back Up First]
Before running migrations on production, take a database snapshot or pg_dump backup.
:::

```bash
# 1. Backup (PostgreSQL)
pg_dump -U myappuser myappdb > backup_$(date +%Y%m%d_%H%M%S).sql

# 2. Check what migrations will run
python manage.py showmigrations --list | grep '\[ \]'

# 3. Preview the SQL
python manage.py sqlmigrate blog 0005

# 4. Run migrations
python manage.py migrate

# 5. Rollback if needed
python manage.py migrate blog 0004
```

For zero-downtime migrations (additive only):

1. **Expand**: Add new columns as nullable / with defaults
2. **Deploy**: Deploy new code that works with both old and new schema
3. **Migrate**: Run the migration (adding nullable columns is non-blocking)
4. **Contract**: Deploy code that drops the old columns, then run that migration

## Post-Deployment Verification

```bash
# 1. Check manage.py --deploy passes
python manage.py check --deploy

# 2. Verify the health endpoint
curl https://myapp.com/health/
# {"status": "ok", "db": "connected", "cache": "connected"}

# 3. Check SSL grade
# Visit: https://www.ssllabs.com/ssltest/analyze.html?d=myapp.com

# 4. Security headers
curl -I https://myapp.com/
# HTTP/2 200
# strict-transport-security: max-age=31536000; includeSubDomains; preload
# x-content-type-options: nosniff
# x-frame-options: DENY

# 5. Static files are fingerprinted
curl -I https://myapp.com/static/css/app.abc123.css
# cache-control: public, max-age=31536000, immutable

# 6. Admin accessible
curl -L https://myapp.com/admin/
# Should return the login page (not a 500 error)
```

FastAPI's dependency injection system is one of its most powerful features. The `Depends()` function allows you to declare shared logic — database connections, authentication, configuration, rate limiting — that is injected into route handlers automatically, with support for async, teardown, sub-dependencies, and testing overrides.

## The `Depends()` Basics

A dependency is simply a callable (function or class) that FastAPI calls before your route handler, injecting its return value as a parameter:

```python
from fastapi import FastAPI, Depends

app = FastAPI()


def get_query(q: str | None = None, limit: int = 10, offset: int = 0):
    """Reusable query parameter extraction — no duplication across routes."""
    return {"q": q, "limit": min(limit, 100), "offset": offset}


@app.get("/items")
async def list_items(filters: dict = Depends(get_query)):
    return {"filters": filters, "items": []}


@app.get("/products")
async def list_products(filters: dict = Depends(get_query)):
    return {"filters": filters, "products": []}
```

```bash
curl "http://localhost:8000/items?q=widget&limit=5&offset=10"
```

```json
{ "filters": { "q": "widget", "limit": 5, "offset": 10 }, "items": [] }
```

:::tip[Dependencies are Cached per Request]
Within a single request, FastAPI calls each dependency **once** and caches the result. If multiple routes or sub-dependencies need the same value (e.g., the current user), the dependency function runs once and its result is reused — even if `Depends(get_current_user)` appears five times in the same request's call chain.
:::

## Dependencies with `yield` (Teardown Logic)

Use `yield` to run cleanup code after the response is sent — ideal for database sessions:

```python
from sqlalchemy.ext.asyncio import AsyncSession
from app.db.session import AsyncSessionLocal


async def get_db() -> AsyncGenerator[AsyncSession, None]:
    """
    Provides a database session for the duration of a single request.
    The session is committed on success, rolled back on any exception,
    and always closed in the finally block.
    """
    async with AsyncSessionLocal() as session:
        try:
            yield session          # Route handler receives this session
            await session.commit() # Commit after handler returns successfully
        except Exception:
            await session.rollback()
            raise
        # AsyncSessionLocal context manager calls session.close() automatically
```

Use in a route:

```python
from fastapi import FastAPI, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from app.db.models.user import User

app = FastAPI()


@app.get("/users/{user_id}")
async def get_user(user_id: int, db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(User).where(User.id == user_id))
    user = result.scalar_one_or_none()
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user
```

## Sub-Dependencies

Dependencies can depend on other dependencies, forming a tree:

```python
from fastapi import FastAPI, Depends, Header, HTTPException
from app.db.session import AsyncSessionLocal
from app.db.models.user import User
from app.core.security import decode_access_token


async def get_token_payload(
    authorization: str = Header(..., description="Bearer <token>"),
) -> dict:
    """Step 1: Extract and decode the JWT from the Authorization header."""
    if not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Invalid authorization header format")
    token = authorization.removeprefix("Bearer ")
    payload = decode_access_token(token)  # Raises 401 if invalid/expired
    return payload


async def get_current_user(
    payload: dict = Depends(get_token_payload),    # Sub-dependency
    db: AsyncSession = Depends(get_db),            # Another sub-dependency
) -> User:
    """Step 2: Load the User record from the database using the token's subject."""
    user_id = payload.get("sub")
    if not user_id:
        raise HTTPException(status_code=401, detail="Invalid token payload")

    result = await db.execute(select(User).where(User.id == int(user_id)))
    user = result.scalar_one_or_none()
    if not user:
        raise HTTPException(status_code=401, detail="User not found")
    return user


async def get_current_active_user(
    current_user: User = Depends(get_current_user),  # Sub-dependency
) -> User:
    """Step 3: Verify the user account is not disabled."""
    if not current_user.is_active:
        raise HTTPException(status_code=403, detail="Account is disabled")
    return current_user
```

Now routes only need to declare the final dependency:

```python
@app.get("/me")
async def read_current_user(user: User = Depends(get_current_active_user)):
    return {
        "id": user.id,
        "username": user.username,
        "email": user.email,
    }
```

```bash
curl http://localhost:8000/me \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```json
{ "id": 42, "username": "alice", "email": "alice@example.com" }
```

```python
from fastapi.security import APIKeyHeader

API_KEY_HEADER = APIKeyHeader(name="X-API-Key", auto_error=True)

    "prod-key-abc123": {"name": "Service A", "rate_limit": 1000},
```

```python
from fastapi import FastAPI, Depends, Query
from dataclasses import dataclass


@dataclass
class PaginationParams:
    page: int
    page_size: int
    offset: int
    limit: int


def pagination(
    page: int = Query(default=1, ge=1, description="Page number (1-indexed)"),
    page_size: int = Query(default=20, ge=1, le=100, description="Items per page"),
) -> PaginationParams:
    offset = (page - 1) * page_size
    return PaginationParams(
        page=page,
        page_size=page_size,
        offset=offset,
        limit=page_size,
    )


@app.get("/items")
async def list_items(
    pag: PaginationParams = Depends(pagination),
    db: AsyncSession = Depends(get_db),
):
    result = await db.execute(
        select(Item).offset(pag.offset).limit(pag.limit)
    )
    items = result.scalars().all()
    return {
        "page": pag.page,
        "page_size": pag.page_size,
        "items": items,
    }
```

## Class-Based Dependencies

For stateful dependencies, use a class with `__call__`:

```python
from fastapi import FastAPI, Depends, HTTPException


class RateLimiter:
    """
    Simple in-memory rate limiter.
    In production, back this with Redis for multi-instance support.
    """

    def __init__(self, max_calls: int, window_seconds: int = 60):
        self.max_calls = max_calls
        self.window_seconds = window_seconds
        self._counts: dict[str, tuple[int, float]] = {}

    async def __call__(self, request: Request) -> None:
        import time
        client_ip = request.client.host
        now = time.time()
        count, window_start = self._counts.get(client_ip, (0, now))

        if now - window_start > self.window_seconds:
            # Reset window
            count, window_start = 0, now

        count += 1
        self._counts[client_ip] = (count, window_start)

        if count > self.max_calls:
            remaining_window = int(self.window_seconds - (now - window_start))
            raise HTTPException(
                status_code=429,
                detail=f"Rate limit exceeded. Retry after {remaining_window}s.",
                headers={"Retry-After": str(remaining_window)},
            )


# Instantiate with different limits for different route groups
strict_limiter = RateLimiter(max_calls=5, window_seconds=60)
relaxed_limiter = RateLimiter(max_calls=100, window_seconds=60)


@app.post("/auth/login", dependencies=[Depends(strict_limiter)])
async def login():
    return {"token": "..."}


@app.get("/items", dependencies=[Depends(relaxed_limiter)])
async def list_items():
    return {"items": []}
```

## Router-Level and Global Dependencies

Apply dependencies to all routes in a router or the entire application:

```python
from fastapi import FastAPI, APIRouter, Depends

# ── Router-level ─────────────────────────────────────────────────────────────
admin_router = APIRouter(
    prefix="/admin",
    tags=["admin"],
    dependencies=[Depends(get_current_active_user), Depends(require_admin_role)],
)

@admin_router.get("/stats")
async def admin_stats():
    # No need to declare Depends here — already applied at router level
    return {"active_users": 1234}


# ── App-level (global) ────────────────────────────────────────────────────────
app = FastAPI(
    dependencies=[Depends(verify_api_key)]  # Every single route requires a valid API key
)
```

:::caution[Global Dependency Performance]
Global dependencies run on **every request**, including `/docs`, `/openapi.json`, and `/health`. Exclude the health endpoint from authentication by declaring it before registering global-dependency routers, or implement a skip-list in the dependency itself.
:::

## Dependency Overrides for Testing

Override any dependency in tests without modifying application code:

```python
# tests/conftest.py
import pytest
from httpx import AsyncClient, ASGITransport
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

from main import app
from app.dependencies.db import get_db
from app.db.base import Base
from app.dependencies.auth import get_current_active_user
from app.db.models.user import User

TEST_DATABASE_URL = "sqlite+aiosqlite:///:memory:"


@pytest.fixture(scope="session")
def event_loop_policy():
    import asyncio
    return asyncio.DefaultEventLoopPolicy()


@pytest.fixture(scope="function")
async def test_db():
    """Provide a fresh in-memory SQLite database for each test."""
    engine = create_async_engine(TEST_DATABASE_URL, echo=False)
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)

    TestSession = async_sessionmaker(engine, expire_on_commit=False)

    async def override_get_db():
        async with TestSession() as session:
            yield session

    app.dependency_overrides[get_db] = override_get_db
    yield
    app.dependency_overrides.pop(get_db, None)
    await engine.dispose()


@pytest.fixture
def mock_user() -> User:
    return User(id=1, username="testuser", email="test@example.com", is_active=True)


@pytest.fixture
async def authenticated_client(test_db, mock_user):
    """HTTP client with authentication bypassed."""
    app.dependency_overrides[get_current_active_user] = lambda: mock_user

    async with AsyncClient(transport=ASGITransport(app=app), base_url="http://test") as ac:
        yield ac

    app.dependency_overrides.pop(get_current_active_user, None)
```

Using the fixtures in tests:

```python
# tests/test_items.py
import pytest


@pytest.mark.asyncio
async def test_create_item(authenticated_client):
    response = await authenticated_client.post(
        "/items/",
        json={"name": "Test Widget", "price": 9.99},
    )
    assert response.status_code == 201
    data = response.json()
    assert data["name"] == "Test Widget"
    assert data["price"] == 9.99


@pytest.mark.asyncio
async def test_get_item_not_found(authenticated_client):
    response = await authenticated_client.get("/items/99999")
    assert response.status_code == 404
    assert response.json()["detail"] == "Item with id=99999 was not found."
```

## OAuth2PasswordBearer as a Dependency

FastAPI provides built-in OAuth2 security scheme helpers that double as dependencies:

```python
from fastapi import FastAPI, Depends
from fastapi.security import OAuth2PasswordBearer

# Tells Swagger UI which URL to post credentials to for a token
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")


async def get_current_user(token: str = Depends(oauth2_scheme)) -> User:
    """
    oauth2_scheme extracts the Bearer token from the Authorization header.
    It also adds a security requirement to the OpenAPI schema, enabling
    the 'Authorize' button in Swagger UI.
    """
    payload = decode_access_token(token)
    user = await load_user_from_payload(payload)
    return user


@app.get("/me")
async def read_me(user: User = Depends(get_current_user)):
    return {"id": user.id, "username": user.username}
```

## Dependency Lifecycle Summary

```
Request arrives
    │
    ├─► Global app dependencies run
    │       │
    │       ├─► Router dependencies run
    │       │       │
    │       │       └─► Route dependencies run
    │       │               │
    │       │               ├─► Sub-dependencies (cached per request)
    │       │               │
    │       │               └─► Route handler called
    │       │                       │
    │       │               ◄── Response prepared
    │       │
    │       ├─► yield dependencies: code after yield runs (teardown)
    │       │   (in reverse declaration order)
    │       │
Response sent to client
```

FastAPI is a modern, fast (high-performance) web framework for building APIs with Python 3.10+ based on standard Python type hints. Created by Sebastián Ramírez in 2018, it has rapidly become one of the most widely adopted Python frameworks for API development, used in production by Microsoft, Netflix, Uber, and hundreds of enterprises worldwide.

## What is FastAPI?

FastAPI is built on top of two foundational libraries:

- **[Starlette](https://www.starlette.io/)** — A lightweight ASGI framework/toolkit for the web and networking layer
- **[Pydantic v2](https://docs.pydantic.dev/)** — Data validation, serialization, and settings management via Python type annotations

This combination delivers a framework that is simultaneously easy to use, high-performance, and production-ready — with automatic interactive API documentation generated directly from your code.

## Hello World API

A complete FastAPI application in under 20 lines:

```python
# main.py
from fastapi import FastAPI

app = FastAPI(
    title="My First API",
    description="A simple example API built with FastAPI",
    version="1.0.0",
)


@app.get("/")
async def root():
    """Return a welcome message."""
    return {"message": "Hello, World!"}


@app.get("/items/{item_id}")
async def read_item(item_id: int, q: str | None = None):
    """Fetch a single item by integer ID with an optional search query."""
    return {"item_id": item_id, "query": q}
```

Start the development server with hot-reload:

```bash
uvicorn main:app --reload
```

```bash
# Test the root endpoint
curl http://localhost:8000/
```

```json
{ "message": "Hello, World!" }
```

```bash
# Path parameter + optional query parameter
curl "http://localhost:8000/items/42?q=sneakers"
```

```json

```

```bash
# Wrong type → automatic 422 validation error
curl http://localhost:8000/items/not-a-number
```

```json
{
  "detail": [
    {
      "type": "int_parsing",
      "loc": ["path", "item_id"],
      "msg": "Input should be a valid integer, unable to parse string as an integer",
      "input": "not-a-number",
      "url": "https://errors.pydantic.dev/2.6/v/int_parsing"
    }
  ]
}
```

```bash
# Missing optional query — returns null
curl http://localhost:8000/items/1
```

```json
{ "item_id": 1, "query": null }
```

:::tip[Zero Validation Code]
The 422 error above was generated with **zero validation code**. The `int` type hint on `item_id` is the only instruction FastAPI needs to reject non-integer input with a structured, field-level error response.
:::

## How FastAPI Works Under the Hood

When the application starts:

<Steps>
  1. FastAPI introspects all route handler signatures using Python's `inspect`
  module 2. It extracts parameter names, type annotations, and default values 3.
  A complete OpenAPI 3.0 JSON schema is built and cached 4. Pydantic models are
  compiled (Pydantic v2 uses Rust-backed validators for speed) 5. Swagger UI and
  ReDoc are mounted at `/docs` and `/redoc` 6. The JSON schema is served at
  `/openapi.json`
</Steps>

On every incoming request:

<Steps>
  1. Starlette routes the ASGI request to the correct path operation handler 2.
  FastAPI extracts path parameters, query parameters, headers, cookies, and body
  3. All values are validated and coerced via Pydantic — 422 returned on any
  failure 4. The dependency tree (`Depends()`) is resolved in topological order
  5. Your `async def` (or sync) handler is called with fully-typed Python
  objects 6. The return value is serialized to JSON via Pydantic's
  `model_dump()` / `jsonable_encoder()` 7. An HTTP response is returned with the
  correct `Content-Type`, status code, and headers
</Steps>

## Typical Production Stack

FastAPI works with the full Python ecosystem. A representative production setup:

```python
# pyproject.toml [project.dependencies]
dependencies = [

    "pydantic-settings>=2.2.0",

    "alembic>=1.13.0",           # Schema migrations
    # Auth
    "python-jose[cryptography]>=3.3.0",
    "passlib[bcrypt]>=1.7.4",
    # Utilities
    "python-multipart>=0.0.9",   # Form data + file uploads
    "httpx>=0.27.0",             # Async HTTP client + TestClient
    "slowapi>=0.1.9",            # Rate limiting
    "redis[asyncio]>=5.0.0",     # Caching and sessions
]
```

:::caution[WSGI vs ASGI Servers]
FastAPI is an **ASGI application**. It **cannot** run on traditional WSGI servers like plain Gunicorn. You must use an ASGI server: **Uvicorn** (development + production) or **Hypercorn**. In production, use Gunicorn strictly as a **process manager** with Uvicorn workers: `gunicorn -k uvicorn.workers.UvicornWorker main:app`. See the [Deployment guide](/python/fastapi/deployment) for the full production configuration.
:::

## Core Concepts at a Glance

### Type Hint-Driven Everything

FastAPI's core philosophy is that **Python type hints are the single source of truth** for your API's contract. The same annotation that tells your editor what type a variable holds also:

- Validates incoming HTTP data (Pydantic)
- Generates the OpenAPI schema (FastAPI)
- Produces structured error messages (Pydantic)
- Powers IDE autocompletion in your route handlers

```python
# This single annotation does four jobs simultaneously:
async def create_user(user_id: int):  # 1. Editor knows it's int
    #                  ^^^^^^^^^^      2. FastAPI validates the path param
    #                                  3. 422 returned if not an integer
    #                                  4. OpenAPI schema shows type: integer
    ...
```

### Async First

FastAPI supports both synchronous and asynchronous handlers. Use `async def` for I/O-bound work and `def` for CPU-bound or blocking library calls:

```python
import asyncio
import httpx

# async def: non-blocking I/O — preferred for DB queries, HTTP calls
@app.get("/async-data")
async def async_handler():
    async with httpx.AsyncClient() as client:
        response = await client.get("https://api.example.com/data")
    return response.json()


# def: use for CPU-bound work or blocking libraries (Pillow, NumPy, etc.)
# FastAPI runs sync handlers in a threadpool automatically
@app.post("/process-image")
def process_image(file: UploadFile):
    # PIL is blocking — but FastAPI handles this safely
    from PIL import Image
    img = Image.open(file.file)
    return {"width": img.width, "height": img.height}
```

:::tip[Mixing async and sync]
FastAPI intelligently runs `def` route functions in a separate thread pool to avoid blocking the event loop. You do **not** need to wrap every library call in `asyncio.to_thread()` — just declare the function as `def` instead of `async def` and FastAPI handles the rest.
:::

### Automatic OpenAPI Schema

Every docstring becomes API documentation, every type hint becomes a schema property, and every `Field()` description appears in the generated spec:

```python

class ProductCreate(BaseModel):
    """Request body for creating a new product."""

    name: str = Field(
        min_length=2,
        max_length=200,
        description="Human-readable product name",
        examples=["Wireless Mechanical Keyboard"],
    )
    price: float = Field(gt=0, description="Price in USD, must be positive")
    sku: str = Field(pattern=r"^[A-Z]{2}-\d{6}$", description="SKU in format XX-000000")
```

:::caution[Pydantic v1 vs v2]
FastAPI 0.100+ ships with **Pydantic v2** by default — a complete rewrite with a Rust core that is 5–50× faster than v1. If you are migrating from an older project using Pydantic v1, review the [Pydantic v1 → v2 migration guide](https://docs.pydantic.dev/latest/migration/) before upgrading. Key changes: `__fields__` → `model_fields`, `.dict()` → `.model_dump()`, `.json()` → `.model_dump_json()`.
:::

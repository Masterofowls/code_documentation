This guide walks you from a bare Python environment to a well-structured Flask project with multiple configuration profiles, environment variable management, and a scalable application factory. By the end you'll have a production-ready project skeleton.

## Prerequisites

- Python 3.10 or newer (`python --version`)
- `pip` or `uv` package manager
- A virtual environment (strongly recommended)

## Installation

<Steps>
1. Create and activate a virtual environment — choose your platform:

**Windows:** `python -m venv .venv && .venv\Scripts\activate`

**macOS / Linux:** `python -m venv .venv && source .venv/bin/activate`

**uv:** `uv venv && source .venv/bin/activate`

2. Install Flask

   ```bash
   pip install flask
   ```

   Verify the installation:

   ```bash
   flask --version
   ```

   ```
   Python 3.12.3
   Flask 3.1.0
   Werkzeug 3.1.3
   ```

3. Install common extensions upfront

   ```bash
   pip install flask-sqlalchemy flask-migrate flask-login flask-wtf python-decouple
   ```

4. Freeze your dependencies

   ```bash
   pip freeze > requirements.txt
   ```

</Steps>

## Your First Flask App

The absolute minimum Flask application is five lines:

```python
# app.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
    return 'Hello, Flask!'
```

## Debug Mode

Debug mode provides two key features: the **interactive debugger** in the browser when an exception occurs, and **automatic reloading** when source files change.

```bash
# Method 1: CLI flag (Flask 2.3+)
flask run --debug

# Method 2: Environment variable
export FLASK_DEBUG=1
flask run

# Method 3: In code (development only)
app.run(debug=True)
```

```
 * Debug mode: on
 * WARNING: This is a development server. Do not use it in a production deployment.
 * Running on http://127.0.0.1:5000
 * Restarting with stat
 * Debugger is active!
 * Debugger PIN: 123-456-789
```

:::caution[Never Enable Debug Mode in Production]
The Werkzeug debugger allows arbitrary Python execution in the browser. With `DEBUG=True` in production, anyone who can trigger an exception can run code on your server. Always set `DEBUG=False` before deploying.
:::

## Project Structure

A flat `app.py` works for demos. Real applications need a consistent structure. The recommended layout for a medium-to-large Flask project uses the **Application Factory** pattern with **Blueprints**:

```
my_flask_app/
├── app/
│   ├── __init__.py          ← Application factory (create_app)
│   ├── config.py            ← Configuration classes
│   ├── extensions.py        ← Extension instances (db, login_manager, etc.)
│   ├── models/
│   │   ├── __init__.py
│   │   └── user.py
│   ├── auth/
│   │   ├── __init__.py      ← Blueprint definition
│   │   ├── routes.py
│   │   └── forms.py
│   ├── main/
│   │   ├── __init__.py
│   │   └── routes.py
│   ├── api/
│   │   ├── __init__.py
│   │   └── routes.py
│   ├── templates/
│   │   ├── base.html
│   │   ├── auth/
│   │   │   ├── login.html
│   │   │   └── register.html
│   │   └── main/
│   │       └── index.html
│   └── static/
│       ├── css/
│       ├── js/
│       └── img/
├── migrations/              ← Flask-Migrate files
├── tests/
│   ├── conftest.py
│   ├── test_auth.py
│   └── test_api.py
├── .env                     ← Local secrets (never commit)
├── .env.example             ← Template (commit this)
├── .gitignore
├── config.py                ← Alias or re-export
├── requirements.txt
├── requirements-dev.txt
└── wsgi.py                  ← Production entry point
```

## Configuration Classes

Flask supports multiple configuration approaches. Using Python classes gives you type-safe config with inheritance:

```python
# app/config.py
import os
from decouple import config as env_config


class Config:
    """Base configuration shared by all environments."""

    # Security
    SECRET_KEY: str = env_config('SECRET_KEY', default='change-me-in-production')
    WTF_CSRF_ENABLED: bool = True

    # Database
    SQLALCHEMY_TRACK_MODIFICATIONS: bool = False
    SQLALCHEMY_DATABASE_URI: str = env_config(
        'DATABASE_URL',
        default='sqlite:///app.db'
    )

    # Mail
    MAIL_SERVER: str = env_config('MAIL_SERVER', default='localhost')
    MAIL_PORT: int = env_config('MAIL_PORT', default=25, cast=int)
    MAIL_USE_TLS: bool = env_config('MAIL_USE_TLS', default=False, cast=bool)

    # Pagination
    POSTS_PER_PAGE: int = 20


class DevelopmentConfig(Config):
    """Configuration for local development."""

    DEBUG: bool = True
    SQLALCHEMY_DATABASE_URI: str = 'sqlite:///dev.db'
    SQLALCHEMY_ECHO: bool = True  # Log all SQL queries


class TestingConfig(Config):
    """Configuration for automated tests."""

    TESTING: bool = True
    DEBUG: bool = True
    SQLALCHEMY_DATABASE_URI: str = 'sqlite:///:memory:'
    WTF_CSRF_ENABLED: bool = False  # Disable CSRF for test forms
    SERVER_NAME: str = 'localhost.localdomain'


class ProductionConfig(Config):
    """Configuration for production deployment."""

    DEBUG: bool = False
    TESTING: bool = False

    # Require these to be set in the environment — fail loudly if missing
    @classmethod
    def init_app(cls, app):
        Config.init_app(app)
        assert os.environ.get('SECRET_KEY'), 'SECRET_KEY environment variable not set!'
        assert os.environ.get('DATABASE_URL'), 'DATABASE_URL environment variable not set!'


# Registry mapping string names to config classes
config = {
    'development': DevelopmentConfig,
    'testing': TestingConfig,
    'production': ProductionConfig,
    'default': DevelopmentConfig,
}
```

## Environment Variables with python-decouple

`python-decouple` reads from `.env` files and environment variables, with type casting and validation:

```bash
pip install python-decouple
```

Create a `.env` file (never commit this):

```ini
# .env
SECRET_KEY=your-very-secret-key-here-use-openssl-rand-hex-32
DATABASE_URL=postgresql://user:password@localhost:5432/myapp
FLASK_ENV=development
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USE_TLS=True
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=your-app-password
```

Create `.env.example` (commit this as a template):

```ini
# .env.example
SECRET_KEY=replace-with-random-secret-key
DATABASE_URL=sqlite:///app.db
FLASK_ENV=development
MAIL_SERVER=localhost
MAIL_PORT=25
MAIL_USE_TLS=False
MAIL_USERNAME=
MAIL_PASSWORD=
```

Use in config:

```python
from decouple import config, Csv

SECRET_KEY = config('SECRET_KEY')
DATABASE_URL = config('DATABASE_URL', default='sqlite:///app.db')
DEBUG = config('DEBUG', default=False, cast=bool)
ALLOWED_HOSTS = config('ALLOWED_HOSTS', default='localhost', cast=Csv())
```

## Extension Instances

Declare extension instances **once** in a dedicated module. This avoids circular imports:

```python
# app/extensions.py
from flask_sqlalchemy import SQLAlchemy
from flask_migrate import Migrate
from flask_login import LoginManager
from flask_wtf.csrf import CSRFProtect
from flask_mail import Mail
from flask_caching import Cache

db = SQLAlchemy()
migrate = Migrate()
login_manager = LoginManager()
csrf = CSRFProtect()
mail = Mail()
cache = Cache()

# Configure LoginManager defaults
login_manager.login_view = 'auth.login'
login_manager.login_message = 'Please log in to access this page.'
login_manager.login_message_category = 'info'
```

## The Application Factory

The factory pattern solves several problems: testability (each test can create a fresh app), multiple instances, and deferred extension initialization:

```python
# app/__init__.py
from flask import Flask

from app.config import config
from app.extensions import db, migrate, login_manager, csrf, mail, cache


def create_app(config_name: str = 'default') -> Flask:
    """Application factory — creates and configures the Flask app."""

    app = Flask(__name__)

    # ── Load configuration ────────────────────────────────────────────────────
    app.config.from_object(config[config_name])
    config[config_name].init_app(app)

    # ── Initialize extensions ─────────────────────────────────────────────────
    db.init_app(app)
    migrate.init_app(app, db)
    login_manager.init_app(app)
    csrf.init_app(app)
    mail.init_app(app)
    cache.init_app(app)

    # ── Register blueprints ───────────────────────────────────────────────────
    from app.main import main as main_blueprint
    app.register_blueprint(main_blueprint)

    from app.auth import auth as auth_blueprint
    app.register_blueprint(auth_blueprint, url_prefix='/auth')

    from app.api import api as api_blueprint
    app.register_blueprint(api_blueprint, url_prefix='/api/v1')

    # ── Shell context processor ───────────────────────────────────────────────
    @app.shell_context_processor
    def make_shell_context():
        from app.models.user import User
        return {'db': db, 'User': User}

    return app
```

## Production Entry Point

```python
# wsgi.py
import os
from app import create_app

app = create_app(os.environ.get('FLASK_ENV', 'production'))

if __name__ == '__main__':
    app.run()
```

## Running with Different Configs

```bash
# Development
export FLASK_ENV=development
flask run --debug

# Testing (used by pytest)
export FLASK_ENV=testing
pytest

# Production (via Gunicorn)
export FLASK_ENV=production
gunicorn wsgi:app
```

## Testing the Factory

```python
# tests/conftest.py
import pytest
from app import create_app
from app.extensions import db as _db


@pytest.fixture(scope='session')
def app():
    """Create test application."""
    app = create_app('testing')
    with app.app_context():
        _db.create_all()
        yield app
        _db.drop_all()


@pytest.fixture()
def client(app):
    """Create test client."""
    return app.test_client()


@pytest.fixture()
def runner(app):
    """Create CLI test runner."""
    return app.test_cli_runner()
```

```python
# tests/test_main.py
def test_index_route(client):
    response = client.get('/')
    assert response.status_code == 200

def test_health_endpoint(client):
    response = client.get('/health')
    assert response.status_code == 200
    assert response.get_json() == {'status': 'ok'}
```

## Complete Bootstrap Checklist

<Steps>
  1. Create and activate virtual environment 2. `pip install flask
  flask-sqlalchemy flask-migrate flask-login flask-wtf python-decouple gunicorn`
  3. Create project directory structure with `app/`, `tests/`, `migrations/` 4.
  Write `app/config.py` with `DevelopmentConfig`, `TestingConfig`,
  `ProductionConfig` 5. Write `app/extensions.py` with all extension instances
  6. Write `app/__init__.py` with `create_app()` factory 7. Create `.env` with
  secrets and `.env.example` as a committed template 8. Add `.env` and `*.db` to
  `.gitignore` 9. Write `wsgi.py` as the production entry point 10. Write
  `tests/conftest.py` with app and client fixtures 11. Run `flask db init` to
  initialize migrations 12. Run `flask run --debug` to verify everything starts
  correctly
</Steps>

:::tip[Generate a Secure SECRET_KEY]
Never use a guessable secret key. Generate a cryptographically secure one with:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

Copy the output into your `.env` file.
:::

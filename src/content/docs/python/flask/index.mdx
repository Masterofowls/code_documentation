Flask is a lightweight WSGI web application framework written in Python. Built on top of **Werkzeug** (WSGI toolkit) and **Jinja2** (template engine), it gives you the essential building blocks for web development without imposing any particular project structure or forcing specific libraries. Its "micro" philosophy keeps the core minimal while letting extensions fill every gap.

## What Is WSGI?

WSGI (Web Server Gateway Interface) is the Python standard defined in [PEP 3333](https://peps.python.org/pep-3333/) that specifies how web servers communicate with Python web applications. Every Flask app is a WSGI application at its core — a callable that receives an environment dict and a `start_response` callback.

```python
# What WSGI looks like under the hood
def raw_wsgi_app(environ, start_response):
    status = '200 OK'
    headers = [('Content-Type', 'text/plain')]
    start_response(status, headers)
    return [b'Hello from raw WSGI!']

# Flask wraps all of this complexity for you
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello from Flask!'
```

Flask's `Flask` class is itself a valid WSGI application. Any WSGI-compatible server — Gunicorn, uWSGI, Waitress, or the built-in development server — can run it without modification.

## The Micro-Framework Philosophy

The word "micro" does **not** mean your application has to fit in one file, nor that Flask is limited in capability. It means:

- **Minimal core** — Flask ships only what's universally needed (routing, request/response handling, templating, sessions).
- **No forced decisions** — Flask doesn't prescribe a database, ORM, form library, or authentication system.
- **Extensible by design** — Every missing feature can be added via a Flask extension that integrates cleanly with the application context.

This philosophy makes Flask ideal for developers who want control, and for teams with existing tool preferences.

## Extension Ecosystem

Flask's power multiplies through its extension ecosystem. Each extension follows Flask's design patterns and integrates with the application context:

| Category         | Extension            | Purpose                                 |
| ---------------- | -------------------- | --------------------------------------- |
| Database ORM     | Flask-SQLAlchemy     | SQLAlchemy integration with app context |
| Migrations       | Flask-Migrate        | Alembic-powered schema migrations       |
| Authentication   | Flask-Login          | User session management, `current_user` |
| Password hashing | Flask-Bcrypt         | Bcrypt hashing wrapper                  |
| Forms & CSRF     | Flask-WTF            | WTForms with CSRF protection            |
| REST APIs        | Flask-RESTX          | REST API scaffold with Swagger UI       |
| Caching          | Flask-Caching        | Redis, Memcached, simple cache backends |
| Email            | Flask-Mail           | SMTP email sending                      |
| Rate limiting    | Flask-Limiter        | Per-route request rate limiting         |
| Admin UI         | Flask-Admin          | Auto-generated admin interface          |
| Serialization    | Flask-Marshmallow    | Object serialization/deserialization    |
| JWT tokens       | Flask-JWT-Extended   | JWT access/refresh token authentication |
| CORS             | Flask-CORS           | Cross-Origin Resource Sharing headers   |
| Celery tasks     | (direct integration) | Async task queue support                |

:::tip[Installing Extensions]
All official Flask extensions follow the naming convention `Flask-Xxx` on PyPI and are imported as `flask_xxx` in Python. Install them alongside Flask: `pip install flask flask-sqlalchemy flask-login`.
:::

## Flask vs Django vs FastAPI

Choosing the right framework depends heavily on project scope and team preferences:

| Feature             | Flask                           | Django                         | FastAPI                     |
| ------------------- | ------------------------------- | ------------------------------ | --------------------------- |
| **Type**            | Micro-framework                 | Full-stack framework           | Async micro-framework       |
| **Learning curve**  | Low                             | Medium–High                    | Medium                      |
| **Built-in ORM**    | ❌ (Flask-SQLAlchemy)           | ✅ Django ORM                  | ❌ (SQLAlchemy/Tortoise)    |
| **Built-in Admin**  | ❌ (Flask-Admin)                | ✅ Django Admin                | ❌                          |
| **Built-in Auth**   | ❌ (Flask-Login)                | ✅ django.contrib.auth         | ❌                          |
| **Auto API Docs**   | ❌ (Flask-RESTX)                | ❌ (DRF)                       | ✅ OpenAPI/Swagger built-in |
| **Async Support**   | Partial (Flask 2.0+)            | Partial (Django 3.1+)          | Native (core feature)       |
| **Performance**     | Good                            | Good                           | Excellent                   |
| **Flexibility**     | Very High                       | Low–Medium                     | High                        |
| **Community Size**  | Large                           | Very Large                     | Rapidly growing             |
| **Template Engine** | Jinja2                          | Django Templates               | Jinja2 (optional)           |
| **Best For**        | APIs, microservices, prototypes | Content sites, full-stack apps | High-perf APIs, ML serving  |

## Your First Flask Application

<Steps>
1. Install Flask

```bash
pip install flask
```

2. Create `app.py`

   ```python
   from flask import Flask

   app = Flask(__name__)

   @app.route('/')
   def index():
       return '<h1>Hello, World!</h1>'

   @app.route('/greet/<name>')
   def greet(name: str):
       return f'<h1>Hello, {name}!</h1>'

   if __name__ == '__main__':
       app.run(debug=True)
   ```

3. Run the development server

   ```bash
   flask run
   ```

   ```
    * Debug mode: off
    * Running on http://127.0.0.1:5000
    * Press CTRL+C to quit
   ```

4. Test your routes

   ```bash
   curl http://127.0.0.1:5000/
   # <h1>Hello, World!</h1>

   curl http://127.0.0.1:5000/greet/Alice
   # <h1>Hello, Alice!</h1>
   ```

</Steps>

:::tip[Flask 2.0+ Async Support]
Flask 2.0 introduced first-class support for `async def` view functions. Install `flask[async]` to pull in `asgiref` and run async views on the development server without switching to a full ASGI framework.
:::

## Core Concepts at a Glance

```python
from flask import Flask, request, jsonify, render_template, redirect, url_for, abort

app = Flask(__name__)

# ── Routing ─────────────────────────────────────────────────────────────────
@app.route('/users/<int:user_id>', methods=['GET'])
def get_user(user_id: int):
    if user_id <= 0:
        abort(400)
    return jsonify({'id': user_id, 'name': 'Alice'})

# ── Accepting JSON ───────────────────────────────────────────────────────────
@app.route('/users', methods=['POST'])
def create_user():
    data = request.get_json(force=True)
    # ... save to database ...
    return jsonify(data), 201

# ── Template rendering ───────────────────────────────────────────────────────
@app.route('/dashboard')
def dashboard():
    users = [{'name': 'Alice'}, {'name': 'Bob'}]
    return render_template('dashboard.html', title='Dashboard', users=users)

# ── Redirect ─────────────────────────────────────────────────────────────────
@app.route('/home')
def home():
    return redirect(url_for('index'))

# ── Custom error handler ─────────────────────────────────────────────────────
@app.errorhandler(404)
def not_found(error):
    return jsonify({'error': 'Not found'}), 404
```

## Request/Response Lifecycle

Understanding how Flask processes a request helps you hook into the right place:

```
Browser / API Client
        │
        ▼
  WSGI Server (Gunicorn)
        │
        ▼
  Flask Application
        │
        ├─► before_request() hooks   ← authentication, logging
        │
        ├─► URL Router
        │         │
        │         ▼
        │   View Function            ← your code runs here
        │
        ├─► after_request() hooks    ← add headers, compress response
        │
        ├─► teardown_request() hooks ← close DB connections
        │
        ▼
  HTTP Response → Client
```

## When to Choose Flask

**✅ Choose Flask when you need to:**

- Build REST APIs or JSON microservices
- Prototype quickly and iterate fast
- Integrate with an existing Python codebase
- Have full control over every library choice
- Build a small-to-medium web application
- Serve as the backend for a React/Vue/Svelte SPA

**⚠️ Consider alternatives when:**

- You need a batteries-included admin interface from day one → **Django**
- You're building a high-performance async API with automatic docs → **FastAPI**
- Your team is more comfortable with conventions over configuration → **Django**

:::caution[Structure Matters at Scale]
Flask imposes no structure on your application. A flat `app.py` works for toy projects, but medium-to-large applications require deliberate organization using **Blueprints** and the **Application Factory** pattern. Plan your architecture before you need to refactor it.
:::

## Flask Version History

| Version    | Highlights                                                         |
| ---------- | ------------------------------------------------------------------ |
| 0.1 (2010) | Initial release by Armin Ronacher                                  |
| 1.0 (2018) | CLI support, environment variables, JSON improvements              |
| 2.0 (2021) | Async/await support, nested Blueprints, `flask shell` improvements |
| 2.1 (2022) | Performance improvements, deprecation cleanups                     |
| 2.2 (2022) | Async view functions stabilized, Blueprint nesting                 |
| 2.3 (2023) | `FLASK_ENV` deprecated in favour of `--debug` flag                 |
| 3.0 (2023) | Python 3.8+ required, cleaned up deprecated APIs                   |
| 3.1 (2024) | Improved async support, typing improvements                        |

Flask's minimalist design has kept it relevant for over a decade. Whether you're building your first API endpoint or architecting a production microservice, Flask provides a stable, well-documented foundation that scales with your team's needs.

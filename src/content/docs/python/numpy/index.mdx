NumPy (Numerical Python) is the cornerstone of scientific computing in Python. It provides a powerful
N-dimensional array object, broadcasting capabilities, and a rich library of mathematical functions
that operate on entire arrays without writing explicit loops.

## What is NumPy?

At its core, NumPy introduces the **ndarray** — a fixed-type, contiguous-memory, N-dimensional array
that is orders of magnitude faster and more memory-efficient than Python's built-in lists.

NumPy was created by Travis Oliphant in 2005, merging the earlier Numeric and Numarray projects. It
has since become the de-facto standard for numerical data in Python, with virtually every data-science
library depending on it directly or indirectly.

## ndarray vs Python List

The difference between a NumPy array and a Python list is not merely syntactic — it is architectural.

### Memory Layout

A Python `list` stores an array of **pointers** to arbitrary Python objects scattered throughout the
heap. Each object carries type metadata, a reference count, and the value itself. This indirection
adds overhead on every access.

A NumPy `ndarray` stores raw numeric values in a **single, contiguous block of memory**. There is no
per-element Python overhead; the entire array is described by a small metadata struct (shape, strides,
dtype pointer).

```
Python list of integers
┌───┬───┬───┬───┬───┐
│ * │ * │ * │ * │ * │  ← array of pointers
└─┬─┴─┬─┴─┬─┴─┬─┴─┬─┘
  ↓   ↓   ↓   ↓   ↓
 [1] [2] [3] [4] [5]  ← separate PyObject allocs (28 bytes each)

NumPy int64 array
┌──────────────────────────────────────┐
│ 1 │ 2 │ 3 │ 4 │ 5                   │  ← 5 × 8 = 40 bytes, contiguous
└──────────────────────────────────────┘
```

### Speed Comparison

```python
import numpy as np
import time

size = 10_000_000

# Python list multiplication
py_list = list(range(size))
start = time.perf_counter()
result = [x * 2 for x in py_list]
py_time = time.perf_counter() - start
# ~0.85 seconds

# NumPy array multiplication
np_arr = np.arange(size)
start = time.perf_counter()
result = np_arr * 2
np_time = time.perf_counter() - start
# ~0.012 seconds  (~70× faster)

print(f"Python list: {py_time:.3f}s")
print(f"NumPy array: {np_time:.4f}s")
print(f"Speedup:     {py_time / np_time:.1f}×")
# Python list: 0.850s
# NumPy array: 0.012s
# Speedup:     70.8×
```

### Memory Comparison

```python
import numpy as np
import sys

n = 1_000_000

# Python list of ints
py_list = list(range(n))
py_bytes = sys.getsizeof(py_list) + sum(sys.getsizeof(x) for x in py_list[:10]) * n // 10
# ~28 MB (approximate)

# NumPy int64 array
np_arr = np.arange(n, dtype=np.int64)
np_bytes = np_arr.nbytes
# 8,000,000 bytes = ~7.6 MB

print(f"Python list:  ~{py_bytes / 1e6:.0f} MB")  # ~28 MB
print(f"NumPy int64:  {np_bytes / 1e6:.1f} MB")   # 8.0 MB
```

:::tip[Rule of Thumb]
For homogeneous numerical data of more than a few hundred elements, always prefer NumPy arrays over
Python lists. The performance gap grows with array size.
:::

## C-Contiguous Memory and Strides

NumPy arrays can be stored in **C order** (row-major, default) or **Fortran order** (column-major).
The `strides` attribute describes how many bytes to step in each dimension to reach the next element.

```python
import numpy as np

a = np.array([[1, 2, 3],
              [4, 5, 6]], dtype=np.int32)

print(a.strides)   # (12, 4) — step 12 bytes to next row, 4 bytes to next column
print(a.flags)
# C_CONTIGUOUS    : True
# F_CONTIGUOUS    : False
# OWNDATA         : True
# WRITEABLE       : True
# ALIGNED         : True
# WRITEBACKIFCOPY : False

# Fortran-order array
b = np.asfortranarray(a)
print(b.strides)   # (4, 8) — step 4 bytes to next row, 8 bytes to next column
print(b.flags['C_CONTIGUOUS'])   # False
print(b.flags['F_CONTIGUOUS'])   # True
```

Understanding strides matters for performance: operations traversing memory sequentially (cache-
friendly) are significantly faster than those that skip large chunks of memory.

## The NumPy Ecosystem

NumPy is not an island — it is the shared language that connects the entire scientific Python stack.

```
                         ┌─────────────────────────────────┐
                         │           Applications           │
                         │  Jupyter · Matplotlib · Seaborn │
                         └────────────────┬────────────────┘
                                          │
          ┌───────────────┬───────────────┼───────────────┬───────────────┐
          │               │               │               │               │
     ┌────┴────┐     ┌────┴────┐    ┌────┴────┐    ┌────┴────┐    ┌────┴────┐
     │  Pandas │     │  SciPy  │    │Scikit-  │    │TensorFlow│   │ PyTorch │
     │(DataFrm)│     │(science)│    │ learn   │    │  (deep  │   │ (deep   │
     └────┬────┘     └────┬────┘    └────┬────┘    │learning)│   │learning)│
          │               │               │         └────┬────┘   └────┬────┘
          └───────────────┴───────────────┴──────────────┴─────────────┘
                                          │
                               ┌──────────┴──────────┐
                               │        NumPy         │
                               │  ndarray · ufuncs   │
                               │  linalg · fft · rng │
                               └─────────────────────┘
```

| Library | Relationship to NumPy |
|---|---|
| **Pandas** | `DataFrame` and `Series` are backed by NumPy arrays |
| **SciPy** | Extends NumPy with optimization, integration, signal processing |
| **Matplotlib** | Accepts NumPy arrays directly for plotting |
| **Scikit-learn** | All estimators consume and return NumPy arrays |
| **TensorFlow** | `tf.Tensor` interoperates via `__array__` protocol |
| **PyTorch** | `torch.Tensor` ↔ NumPy array zero-copy via `numpy()` / `from_numpy()` |
| **Numba** | JIT-compiles NumPy-style code to LLVM machine code |
| **CuPy** | Drop-in NumPy replacement that runs on NVIDIA GPUs |

## The dtype System

Every NumPy array has a **dtype** (data type) that describes the type and size of each element. All
elements in an array share the same dtype — this homogeneity is what enables contiguous memory storage.

```python
import numpy as np

# NumPy infers dtype from input
a = np.array([1, 2, 3])
print(a.dtype)        # int64  (platform default)

b = np.array([1.0, 2.0, 3.0])
print(b.dtype)        # float64

c = np.array([True, False, True])
print(c.dtype)        # bool

d = np.array(['hello', 'world'])
print(d.dtype)        # <U5  (Unicode string, max 5 chars)

# Explicit dtype specification
e = np.array([1, 2, 3], dtype=np.float32)
print(e.dtype)        # float32

f = np.zeros(5, dtype=np.complex128)
print(f.dtype)        # complex128
```

### dtype Categories

| Category | dtypes | Notes |
|---|---|---|
| Signed integers | `int8`, `int16`, `int32`, `int64` | Default: `int64` on 64-bit |
| Unsigned integers | `uint8`, `uint16`, `uint32`, `uint64` | Images often use `uint8` |
| Floating point | `float16`, `float32`, `float64`, `float128` | Default: `float64` |
| Complex | `complex64`, `complex128` | Two floats per element |
| Boolean | `bool` | 1 byte per element |
| Unicode string | `<U{n}` | Fixed-width, `n` chars max |
| Bytes string | `|S{n}` | Fixed-width, `n` bytes max |
| Python object | `object` | Pointer to arbitrary object |

:::caution[Overflow is Silent]
Unlike Python integers, NumPy integers can silently overflow. A `np.int8` storing `127 + 1` wraps
to `-128` without raising an error. Always choose an appropriate dtype for your value range.
:::

## Quick-Reference Cheat Sheet

```python
import numpy as np

# ── Creation ──────────────────────────────────────────────
np.array([1, 2, 3])                   # array([1, 2, 3])
np.zeros((3, 4))                      # 3×4 array of 0.0
np.ones((2, 3), dtype=int)            # 2×3 array of 1
np.arange(0, 10, 2)                   # array([0, 2, 4, 6, 8])
np.linspace(0, 1, 5)                  # array([0., 0.25, 0.5, 0.75, 1.])
np.eye(3)                             # 3×3 identity matrix
np.random.default_rng(0).random((2,3))# 2×3 uniform [0,1)

# ── Attributes ────────────────────────────────────────────
a = np.arange(12).reshape(3, 4)
a.shape    # (3, 4)
a.ndim     # 2
a.dtype    # int64
a.size     # 12
a.nbytes   # 96

# ── Math ──────────────────────────────────────────────────
a + b; a - b; a * b; a / b; a ** 2
np.sqrt(a); np.exp(a); np.log(a)
np.sum(a, axis=0); np.mean(a, axis=1)

# ── Indexing ──────────────────────────────────────────────
a[0]          # first row
a[:, 1]       # second column
a[a > 5]      # boolean mask
a[[0, 2]]     # fancy: rows 0 and 2

# ── Reshape / Stack ───────────────────────────────────────
a.reshape(2, 6)
a.flatten()
np.vstack([a, b]); np.hstack([a, b])

# ── Linear Algebra ────────────────────────────────────────
A @ B                 # matrix multiply
np.linalg.inv(A)
np.linalg.eig(A)
np.linalg.solve(A, b)
```

:::tip[Version Check]
Always check your NumPy version when following tutorials — some APIs changed significantly in
NumPy 1.17 (new random generator), 1.20 (type annotations), and 2.0 (breaking changes to defaults).

```python
import numpy as np
print(np.__version__)  # e.g., 2.1.0
```
:::

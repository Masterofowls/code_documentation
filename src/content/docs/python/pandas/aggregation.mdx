Aggregation transforms detailed row-level data into summary statistics. Pandas provides a rich set of tools — from simple `groupby()` sums to complex rolling window computations — all following the split-apply-combine pattern.

## GroupBy Basics

The `groupby()` method implements the **split-apply-combine** pattern:

1. **Split** the DataFrame into groups based on a key
2. **Apply** an aggregation function to each group
3. **Combine** the results into a new DataFrame

```python
# Total revenue by region
print(sales.groupby("region")["revenue"].sum())
```

```text
region
East     36700
North    42600
South    37200
Name: revenue, dtype: int64
```

```python
# Multiple aggregations on one column
print(sales.groupby("region")["revenue"].agg(["sum", "mean", "min", "max", "count"]))
```

```text

region
East  36700  12233.333333   9700  15800      3
North 42600  10650.000000   8500  12000      4
South 37200  12400.000000   9800  14300      3
```

```python
# Groupby multiple keys
print(sales.groupby(["region", "product"])["revenue"].sum())
```

## agg() with Multiple Functions

```python
# Apply multiple functions to multiple columns
result = sales.groupby("region").agg(
    total_revenue=("revenue", "sum"),
    avg_revenue=("revenue", "mean"),
    max_revenue=("revenue", "max"),
    total_units=("units", "sum"),
    avg_discount=("discount", "mean"),
)
print(result.round(3))
```

## Named Aggregations

Named aggregations using `pd.NamedAgg` give clean, flat output column names.

```python
result = sales.groupby("rep").agg(
    total_revenue  = pd.NamedAgg(column="revenue",  aggfunc="sum"),
    avg_revenue    = pd.NamedAgg(column="revenue",  aggfunc="mean"),
    deal_count     = pd.NamedAgg(column="revenue",  aggfunc="count"),
    total_units    = pd.NamedAgg(column="units",    aggfunc="sum"),
    avg_discount   = pd.NamedAgg(column="discount", aggfunc="mean"),
    best_deal      = pd.NamedAgg(column="revenue",  aggfunc="max"),
)
print(result.round(2))
```

```text
       total_revenue  avg_revenue  deal_count  total_units  avg_discount  best_deal
rep
Alice          42600        10650           4          426          0.07      12000
Bob            37200        12400           3          372          0.09      14300
Carol          36700        12233           3          367          0.07      15800
```

:::tip[Prefer Named Aggregations]
Named aggregations produce flat, readable column names and avoid the multi-level column index that `{"col": ["f1", "f2"]}` creates. They are also self-documenting — the output column name describes what was computed.
:::

## transform() — Broadcast Group Statistics Back

`transform()` returns a Series the same length as the original DataFrame, with the group statistic broadcast to each row. This is ideal for adding group-level features.

```python
# Add region average revenue as a new column (same shape as original)
sales["region_avg_revenue"] = sales.groupby("region")["revenue"].transform("mean")
sales["vs_region_avg"]      = (sales["revenue"] - sales["region_avg_revenue"]).round(0)

print(sales[["rep", "region", "revenue", "region_avg_revenue", "vs_region_avg"]])
```

## filter() — Keep or Remove Entire Groups

`filter()` keeps or drops entire groups based on a function that returns `True` or `False` for each group.

```python
# Keep only regions where total revenue exceeds 38,000
high_rev_regions = sales.groupby("region").filter(
    lambda g: g["revenue"].sum() > 38000
)
print(high_rev_regions[["region", "rep", "revenue"]])
```

```text
  region    rep  revenue
0  North  Alice    12000
1  North  Alice     8500
```

```python
# Keep only reps with at least 3 deals
active_reps = sales.groupby("rep").filter(lambda g: len(g) >= 3)
print(active_reps[["rep", "revenue"]].sort_values("rep"))
```

## pivot_table()

`pivot_table()` is a powerful summarization tool combining groupby and reshape in one step.

```python
pt = pd.pivot_table(
    sales,
    values="revenue",
    index="region",
    columns="product",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="Total",
)
print(pt)
```

```text

region

North  34100    8500   42600
South  24100   13100   37200
```

```python
# Multiple value columns
```

## crosstab()

`crosstab()` counts the frequency of combinations of two categorical variables.

## Rolling Windows

Rolling window calculations apply a function over a sliding window of N rows.

```python
# Create a daily time series
daily = pd.DataFrame({
    "date":    pd.date_range("2024-01-01", periods=10),
    "revenue": [12000, 8500, 9800, 11200, 14300, 10500, 9700, 13100, 15800, 11600],
}).set_index("date")

print(daily)
```

```text
            revenue
date
2024-01-01    12000
2024-01-02     8500
2024-01-03     9800
2024-01-04    11200
2024-01-05    14300
2024-01-06    10500
2024-01-07     9700
2024-01-08    13100
2024-01-09    15800
2024-01-10    11600
```

```python
# 3-day rolling mean
daily["rolling_3d_mean"] = daily["revenue"].rolling(window=3).mean().round(0)
# 3-day rolling sum
daily["rolling_3d_sum"]  = daily["revenue"].rolling(window=3).sum()
# 3-day rolling std
daily["rolling_3d_std"]  = daily["revenue"].rolling(window=3).std().round(0)
```

## Multi-Level GroupBy

```python
# Group by two keys and apply multiple aggregations
multi = sales.groupby(["region", "product"]).agg(
    total_rev  = ("revenue",  "sum"),
    avg_rev    = ("revenue",  "mean"),
    deals      = ("revenue",  "count"),
    top_rep    = ("rep",      lambda x: x.mode()[0]),
)
print(multi.round(0))
```

```text
                 total_rev   avg_rev  deals  top_rep
region product
East   A             15800  15800.0      1    Carol
       B             20900  10450.0      2    Carol
North  A             34100  11367.0      3    Alice
       B              8500   8500.0      1    Alice
South  A             24100  12050.0      2      Bob
       B             13100  13100.0      1      Bob
```

```python
# Access a specific group from the multi-level result
print(multi.loc["North"])
```

```text

product

B             8500   8500.0      1    Alice
```

:::caution[GroupBy Performance Tips]
For large DataFrames: convert groupby key columns to `category` dtype before grouping (`df["col"] = df["col"].astype("category")`). This can reduce memory use by 50–90% and speed up `groupby()` operations significantly. Also avoid `apply()` with custom lambda functions on large groups — use built-in agg functions whenever possible, as they run in optimized Cython.
:::

## Indexing with .loc and .iloc

`.loc` is **label-based**: you refer to row and column labels. `.iloc` is **position-based**: you use integer positions starting at 0.

| Feature                 | `.loc`                              | `.iloc`                     |
| ----------------------- | ----------------------------------- | --------------------------- |
| Basis                   | Labels (index values, column names) | Integer positions (0-based) |
| Row slice end           | **Inclusive**                       | **Exclusive**               |
| Works with custom index | Yes                                 | Yes (ignores labels)        |
| Missing label           | Raises `KeyError`                   | Raises `IndexError`         |
| Use case                | When labels are meaningful          | When position matters       |
| Boolean array           | Supported                           | Supported                   |

## Scalar Access: .at and .iat

For single-cell access, `.at` (label) and `.iat` (integer position) are faster than `.loc`/`.iloc`.

```python
# .at — label-based scalar
print(df.at[3, "salary"])
```

```text
68000
```

```python
# .iat — position-based scalar
print(df.iat[3, 2])    # row 3, column 2 (salary)
```

```text
68000
```

```python
# Modify a single cell with .at
df.at[0, "rating"] = 4.6
print(df.loc[0])
```

```text
name          Alice
department    Engineering
salary            95000
years                 4
rating              4.6
remote             True
Name: 0, dtype: object
```

:::tip[Performance]
Use `.at` / `.iat` inside loops or when you need to read/write one cell at a time. They skip the overhead of `__getitem__` slicing and are typically 2–5× faster than `.loc` / `.iloc` for scalar access.
:::

## Boolean Indexing

Filter rows by evaluating conditions against column values.

```python
# Single condition: salary above 80,000
high_earners = df[df["salary"] > 80000]
print(high_earners)
```


```python
# AND condition (both must be True)
senior_remote = df[(df["years"] >= 4) & (df["remote"] == True)]
```

```python
    elif val >= 80000:

        return "Junior"
```

## Concatenating DataFrames

```python
q1 = pd.DataFrame({"month": ["Jan","Feb","Mar"], "revenue": [45200, 39800, 52100]})
q2 = pd.DataFrame({"month": ["Apr","May","Jun"], "revenue": [61400, 58900, 67200]})

# Stack vertically (axis=0)
full = pd.concat([q1, q2], ignore_index=True)
print(full)
```

```text
  month  revenue
0   Jan    45200
1   Feb    39800
2   Mar    52100
3   Apr    61400
4   May    58900
5   Jun    67200
```

```python
# Stack horizontally (axis=1)
names = pd.DataFrame({"first": ["Alice","Bob"], "last": ["Smith","Jones"]})
scores = pd.DataFrame({"math": [90, 85], "english": [88, 92]})
combined = pd.concat([names, scores], axis=1)
print(combined)
```

```text
   first   last  math  english
0  Alice  Smith    90       88
1    Bob  Jones    85       92
```

:::caution[Use pd.concat Instead of append]
`DataFrame.append()` was removed in Pandas 2.0. Always use `pd.concat([df1, df2])` to combine DataFrames. For building a DataFrame iteratively in a loop, collect records in a list first and call `pd.concat()` or `pd.DataFrame()` once at the end — never concatenate inside a loop.
:::

## Reshaping: melt

`melt()` converts wide-format data (multiple value columns) to long format (one row per observation).

```python
wide = pd.DataFrame({
    "employee": ["Alice", "Bob", "Carol"],
    "Q1_sales": [12000, 9500, 15000],
    "Q2_sales": [13500, 10200, 14800],
    "Q3_sales": [11800, 11000, 16200],
})
print("Wide format:")
print(wide)
```

```text
Wide format:
  employee  Q1_sales  Q2_sales  Q3_sales
0    Alice     12000     13500     11800
1      Bob      9500     10200     11000
2    Carol     15000     14800     16200
```

```python
long = wide.melt(
    id_vars="employee",
    value_vars=["Q1_sales", "Q2_sales", "Q3_sales"],
    var_name="quarter",

)
long["quarter"] = long["quarter"].str.replace("_sales", "")
print(long.sort_values(["employee", "quarter"]).reset_index(drop=True))
```

## Reshaping: stack and unstack

```python
# stack() — moves columns to rows (wide → long)
df_stacked = pivot.stack()
print(df_stacked.head(6))
```

```text
region  product
East    A          110
        B           75
North   A          120
        B           85
South   A           95
        B          140
dtype: int64
```

```python
# unstack() — moves row level to columns (long → wide)
df_unstacked = df_stacked.unstack(level="product")
print(df_unstacked)
```

```text
         Q1       Q2
product   A    B    A    B
region
East    110   75  125   80
North   120   85  135   92
South    95  140   88  155
```

:::tip[stack vs melt]
Both `stack()` and `melt()` convert wide to long format, but they differ in context. `stack()` works on MultiIndex DataFrames and pivots column levels into row levels. `melt()` is more general and explicit — it names the variable and value columns, making it better for messy real-world data. Prefer `melt()` for most use cases.
:::

**Pandas** is the foundational Python library for data manipulation and analysis. Built on top of NumPy, it provides fast, flexible, and expressive data structures designed to make working with structured (tabular), time series, and heterogeneous data both easy and intuitive. From a 10-row CSV to a 10-million-row production dataset, Pandas is the first tool most data scientists reach for.

:::tip[Widely Adopted Standard]
Pandas exceeds 100 million monthly downloads on PyPI. Its DataFrame API has become the de facto standard for tabular data in Python — recognized by Scikit-learn, Seaborn, Plotly, Polars, and virtually every other data library.
:::

## What Is Pandas?

Pandas was created by Wes McKinney in 2008 at AQR Capital Management to solve a pragmatic problem: performing fast, flexible data analysis in Python without writing thousands of lines of boilerplate. The name derives from **Pan**el **Da**ta — an econometrics term for multi-dimensional structured data sets.

Today, Pandas is maintained by an active open-source community under the **NumFOCUS** umbrella. The **Pandas 2.0** release (April 2023) introduced Copy-on-Write semantics, optional Apache Arrow-backed dtypes, and significant performance improvements.

## Core Data Structures

Pandas provides two primary data structures that handle the vast majority of use cases:

### Series — 1D Labeled Array

A **Series** is a one-dimensional array with an associated label array called the **index**. Think of it as a single column of a spreadsheet, or a Python `dict` that also supports vectorized operations.

```python
import pandas as pd

# Basic Series — integer index auto-generated
prices = pd.Series([10.99, 25.50, 8.75, 44.00, 19.99])
print(prices)
```

```text
0    10.99
1    25.50
2     8.75
3    44.00
4    19.99
dtype: float64
```

```python
# With a custom string index
prices = pd.Series(
    [10.99, 25.50, 8.75, 44.00, 19.99],
    index=["apple", "mango", "banana", "durian", "grape"],
    name="price_usd",
)
print(prices)
print(f"\nMost expensive: {prices.idxmax()} at ${prices.max():.2f}")
print(f"Average price:  ${prices.mean():.2f}")
```

```text
apple     10.99
mango     25.50
banana     8.75
durian    44.00
grape     19.99
Name: price_usd, dtype: float64

Most expensive: durian at $44.00
Average price:  $22.05
```

```python
# From a dictionary — keys become the index
monthly_revenue = pd.Series({
    "Jan": 45200,
    "Feb": 39800,
    "Mar": 52100,
    "Apr": 61400,
    "May": 58900,
}, name="revenue_usd")
print(monthly_revenue)
print(f"\nTotal Q1: ${monthly_revenue[:3].sum():,}")
```

```text
Jan    45200
Feb    39800
Mar    52100
Apr    61400
May    58900
Name: revenue_usd, dtype: int64

Total Q1: $137,100
```

### DataFrame — 2D Labeled Table

A **DataFrame** is a two-dimensional labeled data structure with columns of potentially different types. It is analogous to a spreadsheet, a SQL table, or a dict of Series objects sharing the same index.

```python
employees = pd.DataFrame({
    "name":       ["Alice", "Bob", "Carol", "David", "Eve"],
    "department": ["Engineering", "Marketing", "Engineering", "HR", "Marketing"],
    "salary":     [95000, 72000, 105000, 68000, 78000],
    "years":      [4, 7, 2, 9, 5],
    "remote":     [True, False, True, False, True],
})
print(employees)
```

```text
    name   department  salary  years  remote
0  Alice  Engineering   95000      4    True
1    Bob    Marketing   72000      7   False
2  Carol  Engineering  105000      2    True
3  David           HR   68000      9   False
4    Eve    Marketing   78000      5    True
```

```python

print(f"Memory:  {employees.memory_usage(deep=True).sum()} bytes\n")
print(employees.dtypes)
```

```text

name          object

salary         int64

```

## The Pandas Ecosystem

<CardGrid>
  <Card title="NumPy" icon="seti:python">
    **Underlying array engine.** Pandas DataFrames are backed by NumPy arrays.
    NumPy ufuncs (`np.log`, `np.sqrt`, `np.where`) apply directly to Series and
    DataFrames. Use `df.to_numpy()` to extract the underlying array.
  </Card>
  <Card title="Matplotlib" icon="seti:image">
    **Default plotting backend.** `df.plot()` and `series.plot()` delegate to
    Matplotlib. Access the `Figure` and `Axes` objects for fine-grained control
    over titles, legends, tick marks, and layout.
  </Card>
  <Card title="Seaborn" icon="seti:image">
    **Statistical visualization.** Accepts Pandas DataFrames natively.
    `sns.heatmap(df.corr())`, `sns.pairplot(df)`, `sns.boxplot(data=df, x="cat",
    y="val")` — all work with DataFrame column names automatically.
  </Card>
  <Card title="Scikit-learn" icon="seti:python">
    **Machine learning pipeline.** `Pipeline` and `ColumnTransformer` accept
    DataFrames. `set_output(transform="pandas")` preserves column names through
    `fit_transform`. Feature importance results map back to column names.
  </Card>
  <Card title="SQLAlchemy" icon="seti:db">
    **Database connectivity.** `pd.read_sql(query, engine)` and `df.to_sql(name,
    engine)` use SQLAlchemy connections for any relational database —
    PostgreSQL, MySQL, SQLite, MS SQL Server, Oracle.
  </Card>
  <Card title="Arrow / Parquet" icon="document">
    **High-performance storage.** Apache Arrow enables zero-copy sharing between
    Pandas, Polars, DuckDB, and Spark. Parquet columnar format offers 10–50×
    compression vs CSV and dramatically faster reads for analytical queries.
  </Card>
</CardGrid>

:::tip[Arrow-Backed Pandas 2.x]
Use `pd.read_csv("file.csv", dtype_backend="pyarrow")` to get Arrow-backed types. These use less memory, support nullable integers natively (no `float64` promotion for missing ints), and interoperate zero-copy with Polars and DuckDB.
:::

## When to Use Pandas vs Alternatives

| Scenario                                  | Recommended          | Reason                                                         |
| ----------------------------------------- | -------------------- | -------------------------------------------------------------- |
| Exploratory analysis, fewer than 5M rows  | **Pandas**           | Richest API, best documentation, easiest debugging             |
| SQL queries on existing database          | **SQL / SQLAlchemy** | Database handles optimization, no data movement                |
| Tabular data 5M–500M rows, single machine | **Polars**           | Rust core, lazy evaluation, ~10–30× faster than Pandas         |
| More than 500M rows or multi-machine      | **Dask / Spark**     | Distributed execution across a cluster                         |
| In-process SQL analytics on local data    | **DuckDB**           | Columnar OLAP in-process, excellent SQL support                |
| Time series resampling and windowing      | **Pandas**           | Best-in-class `resample()`, `rolling()`, timezone support      |
| Pure numerical array work                 | **NumPy**            | Less overhead when labels and DataFrame features aren't needed |
| Real-time streaming data                  | **Kafka / Faust**    | Batch tool — not designed for streaming                        |

<Tabs>
  <TabItem label="Pandas vs Polars">

```python
# Pandas — eager, Python-first
import pandas as pd

result = (
    pd.read_csv("sales.csv")
    .query("region == 'North'")
    .groupby("product")["revenue"]
    .agg(["sum", "mean", "count"])
    .reset_index()
    .sort_values("sum", ascending=False)
)
```

```python
# Polars — lazy evaluation, Rust core, much faster for large data
import polars as pl

result = (
    pl.scan_csv("sales.csv")            # lazy — reads metadata only
    .filter(pl.col("region") == "North")
    .group_by("product")
    .agg([
        pl.col("revenue").sum().alias("sum"),
        pl.col("revenue").mean().alias("mean"),
        pl.col("revenue").count().alias("count"),
    ])
    .sort("sum", descending=True)
    .collect()                          # execute the full query plan
)
```

</TabItem>
  <TabItem label="Pandas vs DuckDB">

```python
# DuckDB can query Pandas DataFrames using SQL
import duckdb
import pandas as pd

df = pd.read_parquet("orders.parquet")

# SQL on a Pandas DataFrame in-process — no database server required
result = duckdb.sql("""
    SELECT
        region,
        product_category,
        SUM(revenue)  AS total_revenue,
        AVG(revenue)  AS avg_order_value,
        COUNT(*)      AS order_count
    FROM df
    WHERE order_date >= '2024-01-01'
    GROUP BY 1, 2
    ORDER BY total_revenue DESC
""").df()   # .df() converts the result back to a Pandas DataFrame
```

**Use SQL when:** data lives in a database, you need complex multi-table joins, or the query planner provides better performance.

:::caution[Memory Limits]
Pandas loads the entire dataset into RAM immediately. A 5 GB CSV file can require 15–25 GB of RAM after parsing, due to Python object overhead for string columns. Use `read_csv(..., chunksize=N)` for iterative processing, convert to Parquet for repeat analysis, or switch to Polars for large datasets. Full strategies are covered in the [IO guide](/python/pandas/io).
:::

## Key Capabilities

- **Flexible indexing** — label (`loc`), position (`iloc`), and boolean selection
- **Missing data handling** — first-class `NaN`/`NA` with `dropna()`, `fillna()`, `interpolate()`
- **Data alignment** — arithmetic operations automatically align on shared index labels
- **Group operations** — powerful `groupby()` with `agg()`, `transform()`, `filter()`
- **Reshaping** — `pivot_table()`, `melt()`, `stack()`, `unstack()`, `crosstab()`
- **Time series** — `resample()`, `rolling()`, `shift()`, `dt` accessor, timezone handling
- **String operations** — vectorized `str` accessor for regex, splitting, and extraction
- **IO tools** — CSV, Excel, JSON, Parquet, Feather, HDF5, SQL, HTML, Clipboard
- **Merging** — database-style `merge()`, `join()`, and `concat()` with all join types
- **Visualization** — `.plot()` API with line, bar, scatter, histogram, boxplot, and more

## Quick Method Reference

```python
import pandas as pd

# ── Creation ──────────────────────────────────────────────────────────────────
s  = pd.Series([1, 2, 3], index=["a", "b", "c"], name="values")
df = pd.DataFrame({"a": [1, 2], "b": [3, 4]})
df = pd.read_csv("data.csv")
df = pd.read_excel("data.xlsx", sheet_name="Sheet1")
df = pd.read_parquet("data.parquet")

# ── Exploration ───────────────────────────────────────────────────────────────
df.head(10)               # First N rows (default 5)
df.tail(5)                # Last N rows
df.sample(10)             # Random N rows
df.info()                 # Schema, dtypes, non-null counts, memory
df.describe()             # Count/mean/std/min/quartiles/max for numerics
df.describe(include="all")# Include categorical columns
df.shape                  # (n_rows, n_cols)
df.dtypes                 # Series of dtype per column
df.columns                # Index of column names
df.index                  # Row index
df.isnull().sum()         # Missing values per column
df.nunique()              # Distinct values per column

# ── Selection ─────────────────────────────────────────────────────────────────
df["col"]                          # Column → Series
df[["col1", "col2"]]               # Multiple columns → DataFrame
df.loc[0]                          # Row by label → Series
df.loc[0, "col"]                   # Scalar by label
df.iloc[0]                         # Row by position → Series
df.iloc[0, 1]                      # Scalar by position
df[df["col"] > 10]                 # Boolean filter
df.query("col > 10 and col2 == 'A'")  # SQL-like syntax

# ── Transformation ────────────────────────────────────────────────────────────
df["new_col"] = df["a"] + df["b"]
df = df.drop(columns=["col"])
df = df.rename(columns={"old": "new"})
df["col"] = df["col"].astype(float)
df["col"] = df["col"].apply(lambda x: x * 2)
df = df.sort_values("col", ascending=False)
df = df.reset_index(drop=True)

# ── Aggregation ───────────────────────────────────────────────────────────────
df.groupby("grp")["val"].mean()
df.groupby("grp").agg({"val": ["mean", "sum", "count"]})
df.pivot_table(values="v", index="r", columns="c", aggfunc="sum")
df["val"].rolling(7).mean()
df.groupby("grp")["val"].transform("mean")

# ── Cleaning ──────────────────────────────────────────────────────────────────
df.dropna(subset=["col"])
df.fillna({"col": 0, "other": "Unknown"})
df.drop_duplicates(subset=["id"])
df["str_col"].str.strip().str.lower()
df["date_col"] = pd.to_datetime(df["date_col"])

# ── Output ────────────────────────────────────────────────────────────────────
df.to_csv("output.csv", index=False)
df.to_excel("output.xlsx", index=False)
df.to_parquet("output.parquet", compression="snappy")
df.to_json("output.json", orient="records", indent=2)
```

:::tip[Version Note]
These docs target **Pandas 2.x**. Key differences from 1.x: Copy-on-Write is default (prevents accidental mutation of views), `DataFrame.append()` is removed (use `pd.concat()`), `applymap()` is renamed to `map()`, and nullable dtypes (`Int64`, `Float64`, `StringDtype`) are recommended over object columns. Check your version with `pd.__version__`.
:::

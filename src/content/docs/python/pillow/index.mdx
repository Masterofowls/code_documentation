Pillow is the most widely used Python imaging library, providing a simple yet powerful interface for opening, manipulating, and saving many different image file formats. It is a maintained, actively developed **fork of the original Python Imaging Library (PIL)**, which ceased development in 2011.

Whether you need to generate thumbnails for a web app, apply watermarks to thousands of photos, or convert images between formats in a batch pipeline, Pillow is the go-to solution for Python developers.

---

## What Is Pillow?

Pillow extends the original PIL API while adding Python 3 support, improved format handling, security fixes, and a growing set of modern features. The package is installed as `Pillow` but imported as `PIL` for backwards compatibility with code written against the original library.

```python
# Install name vs import name — this is intentional and expected
# pip install Pillow
from PIL import Image

img = Image.open("photo.jpg")
print(img.format, img.size, img.mode)
# Output: JPEG (1920, 1080) RGB
```

:::tip[PIL vs Pillow Import]
Always `pip install Pillow` (capital P), but always `from PIL import ...` in code. The two names coexist for historical reasons and this is the correct usage.
:::

## Pillow vs OpenCV vs scikit-image

Choosing the right library depends on your task. Here is how the three main Python imaging libraries compare:

| Feature                          | **Pillow**                     | **OpenCV**                  | **scikit-image**          |
| -------------------------------- | ------------------------------ | --------------------------- | ------------------------- |
| **Primary focus**                | Image I/O & manipulation       | Computer vision & real-time | Scientific image analysis |
| **API style**                    | Pythonic, object-oriented      | C-style, verbose            | NumPy-centric             |
| **Install size**                 | ~3 MB                          | ~35 MB                      | ~15 MB                    |
| **Speed**                        | Good (C extensions)            | Excellent (SIMD, GPU)       | Good (NumPy)              |
| **Format support**               | Excellent (50+ formats)        | Good (common formats)       | Limited (via PIL)         |
| **Drawing tools**                | Full (shapes, text, fonts)     | Full                        | Minimal                   |
| **Machine learning integration** | Via NumPy bridge               | Native                      | Native                    |
| **Color space**                  | RGB by default                 | BGR by default              | RGB by default            |
| **Animated GIF**                 | Read & write                   | Read only                   | Read only                 |
| **PDF support**                  | Basic write                    | None                        | None                      |
| **Learning curve**               | Low                            | Medium–High                 | Medium                    |
| **Best for**                     | Web apps, format work, drawing | CV pipelines, video         | Research, analysis        |

:::tip[Combining Libraries]
Pillow and the other libraries are complementary. A common pattern is to open and preprocess files with Pillow, then convert to a NumPy array for OpenCV or scikit-image processing, and finally save the result back using Pillow.
:::

## Quick Start: Open, Display, Save

The three most fundamental operations in Pillow:

```python
from PIL import Image

# Open an image from disk — works with JPEG, PNG, GIF, WEBP, BMP, TIFF, and 50+ more
img = Image.open("landscape.jpg")

# Display the image using the OS default viewer (Preview on macOS, Photos on Windows)
img.show()

# Save to a different format — Pillow infers the output format from the file extension
img.save("landscape.png")          # Converts JPEG → PNG automatically
img.save("landscape_thumb.jpg", quality=85, optimize=True)

print(f"Format : {img.format}")    # JPEG
print(f"Size   : {img.size}")      # (1920, 1080)  — (width, height) in pixels
print(f"Mode   : {img.mode}")      # RGB
```

:::caution[img.show() Is for Debugging Only]
`Image.show()` saves a temporary file and opens the OS viewer. It is a convenience method for interactive development, not suitable for automated scripts or servers. Use `img.save()` or pass the image bytes to your application layer instead.
:::

## Image Modes

The **mode** of an image defines the type and depth of each pixel. Choosing the right mode matters for memory usage, operations, and output quality.

| Mode      | Description                   | Channels | Bits/pixel | Typical Use                   |
| --------- | ----------------------------- | -------- | ---------- | ----------------------------- |
| `"1"`     | 1-bit pixels, black and white | 1        | 1          | Binary masks, fax images      |
| `"L"`     | 8-bit grayscale               | 1        | 8          | Grayscale photos, alpha masks |
| `"P"`     | 8-bit palette-mapped          | 1        | 8          | GIF images, indexed color     |
| `"RGB"`   | True color                    | 3        | 24         | Standard JPEG, most photos    |
| `"RGBA"`  | True color with alpha         | 4        | 32         | PNG with transparency         |
| `"CMYK"`  | Cyan-Magenta-Yellow-Key       | 4        | 32         | Print-ready files             |
| `"YCbCr"` | Luminance + chrominance       | 3        | 24         | Video frames, JPEG internal   |
| `"LAB"`   | L*a*b\* color space           | 3        | 24         | Perceptual color operations   |
| `"HSV"`   | Hue-Saturation-Value          | 3        | 24         | Color-based filtering         |
| `"I"`     | 32-bit signed integer pixels  | 1        | 32         | Medical imaging, depth maps   |
| `"F"`     | 32-bit floating point pixels  | 1        | 32         | HDR data, scientific images   |

```python
from PIL import Image

img = Image.open("photo.jpg")          # Opens as RGB

gray = img.convert("L")               # Convert to grayscale
rgba = img.convert("RGBA")            # Add alpha channel
cmyk = img.convert("CMYK")            # Convert for print

print(gray.mode)   # L
print(rgba.mode)   # RGBA
print(cmyk.mode)   # CMYK

# Grayscale image — each pixel is a single integer 0–255 (0=black, 255=white)
# RGBA image — each pixel is a tuple (R, G, B, A) where A=0 is fully transparent
# CMYK image — each pixel is a tuple (C, M, Y, K) with values 0–255
```

## Pillow Submodules at a Glance

Pillow is more than just `Image`. Here are the key modules you will use:

| Module              | Purpose                                                      |
| ------------------- | ------------------------------------------------------------ |
| `PIL.Image`         | Core image class — open, save, convert, resize, crop, paste  |
| `PIL.ImageDraw`     | 2D drawing: shapes, lines, text on images                    |
| `PIL.ImageFont`     | Load TrueType and bitmap fonts for text rendering            |
| `PIL.ImageFilter`   | Pre-built convolution filters (blur, sharpen, edge detect)   |
| `PIL.ImageEnhance`  | Adjust color, contrast, brightness, sharpness                |
| `PIL.ImageOps`      | High-level operations: flip, invert, fit, pad, equalize      |
| `PIL.ImageColor`    | Parse color strings (`"red"`, `"#ff0000"`, `"rgb(255,0,0)"`) |
| `PIL.ImageChops`    | Channel arithmetic: add, subtract, multiply, difference      |
| `PIL.ImageStat`     | Compute statistics: mean, median, stddev per channel         |
| `PIL.ImageSequence` | Iterate over frames in animated GIFs and multi-frame TIFFs   |
| `PIL.ExifTags`      | Decode numeric EXIF tag IDs to human-readable names          |

## A Taste of What's Possible

The following complete script demonstrates several Pillow capabilities together — opening a photo, creating a thumbnail, adding a text watermark, and saving both outputs:

```python
from PIL import Image, ImageDraw, ImageFont

def process_photo(input_path: str, output_dir: str) -> None:
    """Open a photo, create a watermarked thumbnail, and save both outputs."""
    with Image.open(input_path) as img:
        # Ensure RGB so JPEG save works without errors
        rgb = img.convert("RGB")

        # --- Thumbnail (preserves aspect ratio, fits within 400x400) ---
        thumb = rgb.copy()
        thumb.thumbnail((400, 400))
        thumb.save(f"{output_dir}/thumb.jpg", quality=80, optimize=True)
        # Output: a 400×N or Nx400 JPEG thumbnail next to the original

        # --- Watermark ---
        draw = ImageDraw.Draw(rgb)
        font = ImageFont.load_default(size=36)
        text = "© My Studio 2024"

        # Draw semi-transparent white text in the bottom-right corner
        w, h = rgb.size
        bbox = draw.textbbox((0, 0), text, font=font)
        text_w = bbox[2] - bbox[0]
        text_h = bbox[3] - bbox[1]
        draw.text(
            (w - text_w - 20, h - text_h - 20),
            text,
            fill=(255, 255, 255, 180),
            font=font,
        )
        rgb.save(f"{output_dir}/watermarked.jpg", quality=90)
        # Output: full-size JPEG with white watermark text in bottom-right corner

process_photo("portrait.jpg", "output")
```

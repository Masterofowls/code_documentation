Pygame's drawing system is built around two concepts: **Surfaces** (pixel grids you draw onto) and **draw functions** (the tools that produce shapes and images on those surfaces). This guide covers all of them.

## The `pygame.draw` Module

All draw functions share the same signature pattern:

```python
pygame.draw.shape(surface, color, geometry, width=0)
```

- **`surface`** — the Surface to draw onto (usually `screen`)
- **`color`** — `(R, G, B)` or `(R, G, B, A)` tuple
- **`geometry`** — points, rects, or radii depending on the shape
- **`width`** — line thickness; `0` means filled (default for most shapes)

All draw functions return a `pygame.Rect` describing the bounding area that was modified (useful for dirty-rect optimization).

## Rectangles

```python
# Filled rectangle: draw.rect(surface, color, rect, width=0, border_radius=0)
pygame.draw.rect(screen, (200, 50, 50), (100, 100, 200, 100))

# Outline only (width > 0)
pygame.draw.rect(screen, (255, 255, 255), (100, 100, 200, 100), width=3)

# Rounded corners (Pygame 2+)
pygame.draw.rect(screen, (50, 150, 255), (350, 100, 200, 100), border_radius=20)

# Different radius per corner
pygame.draw.rect(
    screen, (100, 220, 100),
    pygame.Rect(100, 250, 200, 100),
    border_top_left_radius=30,
    border_bottom_right_radius=30,
)

# Using a pygame.Rect object
button_rect = pygame.Rect(50, 50, 160, 50)
pygame.draw.rect(screen, (80, 80, 200), button_rect)
```

:::tip[Rect Objects]
Pass either a `(x, y, w, h)` tuple or a `pygame.Rect` object — both work. Using `pygame.Rect` gives you access to helper attributes like `.center`, `.midleft`, and `.inflate()`.
:::

## Circles

```python
# Filled circle: draw.circle(surface, color, center, radius, width=0)
pygame.draw.circle(screen, (255, 100, 0), (400, 300), 60)

# Outline circle
pygame.draw.circle(screen, (255, 255, 0), (400, 300), 60, width=4)

# Partial circle (draw_top_right, draw_top_left, draw_bottom_right, draw_bottom_left)
# Only available in Pygame CE / Pygame 2.0+
pygame.draw.circle(
    screen, (200, 50, 200), (200, 400), 80,
    draw_top_right=True, draw_bottom_right=True
)
```

## Ellipses

```python
# Filled ellipse inside bounding rect
pygame.draw.ellipse(screen, (0, 200, 200), (200, 150, 200, 100))

# Outline ellipse
pygame.draw.ellipse(screen, (255, 200, 0), (200, 150, 200, 100), width=3)
```

## Lines

```python
# Single line: draw.line(surface, color, start_pos, end_pos, width=1)
pygame.draw.line(screen, (255, 255, 255), (0, 0), (800, 600), 2)

# Connected polyline: draw.lines(surface, color, closed, points, width=1)
points = [(100, 300), (200, 200), (300, 300), (400, 200), (500, 300)]
pygame.draw.lines(screen, (100, 255, 100), closed=False, points=points, width=3)

# Closed polygon outline via lines
pygame.draw.lines(screen, (255, 100, 100), closed=True, points=points, width=2)

# Anti-aliased line (smoother, 1px only)
pygame.draw.aaline(screen, (255, 255, 255), (0, 0), (800, 600))

# Anti-aliased polyline
pygame.draw.aalines(screen, (200, 200, 255), closed=False, points=points)
```

:::caution[Anti-aliased Lines Are Always 1px]
`draw.aaline` and `draw.aalines` do not accept a `width` parameter. For thick anti-aliased lines, render onto a large surface and scale it down, or use a third-party library like `pygame_gui`.
:::

## Arcs

```python
import math

# draw.arc(surface, color, rect, start_angle, stop_angle, width=1)
# Angles are in radians, measured counter-clockwise from the right
pygame.draw.arc(
    screen,
    (255, 128, 0),
    pygame.Rect(300, 200, 200, 200),
    0,                  # 0 radians = 3 o'clock
    math.pi,            # π radians = 9 o'clock (upper half-circle)
    width=5,
)

# Full progress bar arc
def draw_progress_arc(surf, rect, progress, color, bg_color, width=8):
    """Draw a circular progress indicator (0.0 – 1.0)."""
    pygame.draw.arc(surf, bg_color, rect, 0, 2 * math.pi, width)
    if progress > 0:
        pygame.draw.arc(surf, color, rect, math.pi / 2,
                        math.pi / 2 + 2 * math.pi * progress, width)
```

## Surfaces

A `Surface` is an off-screen canvas. Use them for:
- Pre-rendering complex scenes once per level (not every frame)
- Applying transparency to a group of objects
- Double-buffering sub-components

```python
# Create a blank surface (no alpha channel)
tile = pygame.Surface((64, 64))
tile.fill((80, 80, 80))
pygame.draw.rect(tile, (60, 60, 60), (0, 0, 64, 64), 1)

# Create a surface WITH per-pixel alpha
badge = pygame.Surface((100, 100), pygame.SRCALPHA)
badge.fill((0, 0, 0, 0))  # Fully transparent
pygame.draw.circle(badge, (255, 100, 0, 200), (50, 50), 50)  # Semi-transparent

# Blit onto screen
screen.blit(tile,  (0, 0))
screen.blit(badge, (200, 200))
```

## Surface Performance: `convert()` and `convert_alpha()`

Always call `.convert()` or `.convert_alpha()` after loading or creating a surface that will be blitted frequently:

```python
# Convert to display's pixel format (opaque images)
background = pygame.image.load("bg.png").convert()

# Convert with alpha transparency preserved
player_img = pygame.image.load("player.png").convert_alpha()

# For manually created surfaces with SRCALPHA — already optimal, no convert needed
overlay = pygame.Surface((800, 600), pygame.SRCALPHA)
```

:::tip[Why This Matters]
Blitting a surface whose pixel format differs from the screen requires real-time conversion per pixel. `.convert()` pre-converts once, making every subsequent `blit()` call 2–5× faster.
:::

## Transparency

Pygame offers two transparency models:

<Tabs>
  <TabItem label="Surface alpha (uniform)">
```python
# Set the entire surface's opacity (0=invisible, 255=opaque)
overlay = pygame.Surface((800, 600))
overlay.fill((0, 0, 0))
overlay.set_alpha(180)   # ~70% opaque black overlay
screen.blit(overlay, (0, 0))
```
</TabItem>
  <TabItem label="Per-pixel alpha (SRCALPHA)">
```python
# Each pixel can have its own alpha value
surf = pygame.Surface((200, 200), pygame.SRCALPHA)
surf.fill((0, 0, 0, 0))  # Start fully transparent

# Draw a circle that fades toward its edge
for r in range(80, 0, -1):
    alpha = int(255 * (r / 80))
    pygame.draw.circle(surf, (255, 80, 0, alpha), (100, 100), r)

screen.blit(surf, (300, 200))
```
</TabItem>
  <TabItem label="Color key (chroma key)">
```python
# Treat one color as transparent (useful for legacy BMP sprites)
image = pygame.image.load("sprite.bmp").convert()
image.set_colorkey((255, 0, 255))  # Magenta = transparent
screen.blit(image, (100, 100))
```
</TabItem>

## Loading and Transforming Images

```python
# Load
img = pygame.image.load("ship.png").convert_alpha()

# Scale to exact size
scaled = pygame.transform.scale(img, (128, 128))

# Scale by factor (Pygame CE / 2.1.3+)
bigger = pygame.transform.scale_by(img, 2.0)

# Smooth scale (bilinear filtering — slower but better quality)
smooth = pygame.transform.smoothscale(img, (128, 128))

# Rotate (degrees counter-clockwise; creates new surface, may change size)
rotated = pygame.transform.rotate(img, 45)

# Flip horizontally / vertically
flipped_h = pygame.transform.flip(img, flip_x=True,  flip_y=False)
flipped_v = pygame.transform.flip(img, flip_x=False, flip_y=True)

# Chop (crop a rectangle from the surface)
cropped = pygame.transform.chop(img, pygame.Rect(0, 0, 32, 32))

# Get the rect centered on screen for correct placement after rotate
rect = rotated.get_rect(center=(400, 300))
screen.blit(rotated, rect)
```

:::caution[Rotating Expands the Surface]
`pygame.transform.rotate()` returns a **new, larger** surface to avoid clipping corners. Always use `.get_rect(center=original_center)` on the result to re-center the image.
:::

## Pixel Arrays

For per-pixel manipulation (rare, but sometimes needed):

```python
# Lock the surface for direct pixel access
pixel_array = pygame.PixelArray(screen)

# Set pixel at (x, y) — color must be mapped to surface format
pixel_array[100, 200] = screen.map_rgb(255, 0, 0)

# Replace all red pixels with blue
pixel_array.replace((255, 0, 0), (0, 0, 255))

# Must delete (unlock) before blitting
del pixel_array
```

:::caution[Performance Warning]
Per-pixel operations are extremely slow in Python. Prefer `pygame.draw` functions, surfaces, or NumPy arrays (`pygame.surfarray`) for bulk pixel work.
:::

## Complete Drawing Demo

```python
import pygame
import math

pygame.init()
screen = pygame.display.set_mode((800, 600))
pygame.display.set_caption("Drawing Demo")
clock  = pygame.time.Clock()
font   = pygame.font.SysFont(None, 28)

angle  = 0.0

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

    angle = (angle + 1) % 360

    screen.fill((20, 20, 40))

    # Background grid
    for x in range(0, 801, 80):
        pygame.draw.line(screen, (40, 40, 60), (x, 0), (x, 600), 1)
    for y in range(0, 601, 60):
        pygame.draw.line(screen, (40, 40, 60), (0, y), (800, y), 1)

    # Shapes
    pygame.draw.rect(   screen, (200, 80,  80),  (50,  50,  120, 80))
    pygame.draw.rect(   screen, (255, 255, 255),  (50,  50,  120, 80), 2)
    pygame.draw.circle( screen, (80,  180, 255),  (280, 90),  50)
    pygame.draw.ellipse(screen, (200, 150, 255),  (380, 50,  150, 80))
    pygame.draw.polygon(screen, (255, 220, 0), [
        (600, 50), (630, 120), (700, 120),
        (645, 160), (665, 230), (600, 190),
        (535, 230), (555, 160), (500, 120),
        (570, 120),
    ])

    # Spinning arc
    rad = math.radians(angle)
    pygame.draw.arc(
        screen, (0, 255, 180),
        pygame.Rect(330, 250, 140, 140),
        rad, rad + math.pi * 1.5, 6,
    )

    # Anti-aliased lines
    for i in range(5):
        pygame.draw.aaline(
            screen, (150, 150, 255),
            (0,   600 - i * 120),
            (800, i * 120),
        )

    # Text labels
    labels = [
        ((110, 145), "rect"),
        ((280, 155), "circle"),
        ((455, 145), "ellipse"),
        ((600, 245), "polygon"),
        ((400, 400), "arcs + aalines"),
    ]
    for pos, label in labels:
        surf = font.render(label, True, (180, 180, 180))
        screen.blit(surf, surf.get_rect(center=pos))

    pygame.display.flip()
    clock.tick(60)

pygame.quit()
```

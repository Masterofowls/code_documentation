## Installation

<Steps>

1. **Create a virtual environment**

   ```bash
   python -m venv .venv
   ```

2. **Activate it**

   ```bash
   # Windows (PowerShell)
   .venv\Scripts\Activate.ps1

   # macOS / Linux
   source .venv/bin/activate
   ```

3. **Install Pygame CE**

   ```bash
   pip install pygame-ce
   ```

4. **Verify the install**

   ```bash
   python -c "import pygame; print(pygame.version.ver)"
   # 2.x.x
   ```

</Steps>

:::tip[Pygame CE vs pygame]
`pygame-ce` (Community Edition) is the actively maintained fork with more frequent releases and bug fixes. It is a drop-in replacement — all imports remain `import pygame`.
:::

## Initializing Pygame

Before calling any Pygame function you must call `pygame.init()`. This bootstraps all sub-modules (display, mixer, font, etc.).

```python
import pygame

pygame.init()

# You can also initialize specific modules individually:
pygame.display.init()
pygame.mixer.init()
pygame.font.init()

# Check which modules failed to init (returns (success_count, fail_count)):
success, fails = pygame.init()
print(f"Pygame initialized: {success} modules OK, {fails} failed")
```

:::caution[Always Call pygame.quit()]
Always call `pygame.quit()` before your program exits. This releases SDL resources and prevents OS-level warnings or hangs on some platforms.
:::

## Creating the Display Window

```python
import pygame

pygame.init()

# Set the window size in pixels (width, height)
SCREEN_WIDTH  = 800
SCREEN_HEIGHT = 600

screen = pygame.display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))

# Optional flags for display mode:
# pygame.FULLSCREEN   — true fullscreen
# pygame.RESIZABLE    — allow window resize
# pygame.NOFRAME      — borderless window
# pygame.SCALED        — auto-scale to desktop (CE only)
fullscreen_screen = pygame.display.set_mode(
    (SCREEN_WIDTH, SCREEN_HEIGHT),
    pygame.FULLSCREEN
)

# Set the window title and icon
pygame.display.set_caption("My Game")

icon = pygame.image.load("icon.png")
pygame.display.set_icon(icon)
```

## The Game Loop Skeleton

Every Pygame game follows the same top-level structure:

```python
import pygame

pygame.init()

screen = pygame.display.set_mode((800, 600))
pygame.display.set_caption("Game Loop Demo")
clock = pygame.time.Clock()
FPS = 60

running = True
while running:
    # ── 1. EVENT HANDLING ──────────────────────────────────────
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

    # ── 2. UPDATE GAME STATE ───────────────────────────────────
    # Move objects, check collisions, update score, etc.

    # ── 3. DRAW EVERYTHING ─────────────────────────────────────
    screen.fill((20, 20, 20))          # Clear screen each frame
    # ... draw your game objects here ...
    pygame.display.flip()              # Present the drawn frame

    # ── 4. CAP FRAME RATE ──────────────────────────────────────
    clock.tick(FPS)                    # Sleep to maintain 60 FPS

pygame.quit()
```

### Why We Clear the Screen Every Frame

Pygame does **not** clear the screen automatically. If you don't call `screen.fill()` before drawing, previous frames remain visible — producing a trail effect (sometimes useful, but usually not).

## Clock and FPS

`pygame.time.Clock` is your frame-rate governor.

```python
clock = pygame.time.Clock()

# Inside the game loop:
dt = clock.tick(60)   # Cap at 60 FPS; returns milliseconds since last tick
dt_seconds = dt / 1000.0  # Convert to seconds for physics calculations

# Busy-loop variant (more precise, uses more CPU):
dt = clock.tick_busy_loop(60)

# Display current FPS in window title:
pygame.display.set_caption(f"FPS: {clock.get_fps():.0f}")
```

:::tip[Delta Time]
Multiply all movement by `dt` (delta time in seconds) so your game runs at the same speed on fast and slow machines. See the [Game Loop](/python/pygame/game-loop) guide for details.
:::

## The Coordinate System

Pygame places the origin `(0, 0)` at the **top-left** corner of the window. X increases to the right; Y increases **downward**.

```
Screen (800 × 600)

(0,0)──────────────────(800,0)
  │                        │
  │    (400, 300) center    │
  │                        │
(0,600)────────────────(800,600)
```

```python
# Common reference points for an 800×600 window:
top_left     = (0, 0)
top_right    = (800, 0)
bottom_left  = (0, 600)
bottom_right = (800, 600)
center       = (400, 300)
```

## Colors as Tuples

Colors in Pygame are `(R, G, B)` tuples, each component 0–255. You can also use `(R, G, B, A)` for transparency where A=255 is fully opaque.

```python
# Named color constants (define your own palette)
BLACK   = (0,   0,   0)
WHITE   = (255, 255, 255)
RED     = (255, 0,   0)
GREEN   = (0,   200, 0)
BLUE    = (0,   100, 255)
YELLOW  = (255, 220, 0)
ORANGE  = (255, 140, 0)
PURPLE  = (150, 0,   200)
GRAY    = (128, 128, 128)
DARK    = (30,  30,  30)

# Pygame also accepts pygame.Color objects:
color = pygame.Color(255, 128, 0)       # orange
color = pygame.Color("#FF8000")         # from hex string
color = pygame.Color("orange")          # from name
```

## Loading Images

```python
# Load an image from disk (returns a Surface)
image = pygame.image.load("player.png")

# Convert to display format for faster blitting (no alpha)
image = pygame.image.load("background.png").convert()

# Convert with per-pixel alpha (for PNG with transparency)
image = pygame.image.load("sprite.png").convert_alpha()

# Scale to a specific size
image = pygame.transform.scale(image, (64, 64))

# Draw the image onto the screen at (x, y)
screen.blit(image, (100, 200))
```

:::caution[Always Use .convert() or .convert_alpha()]
Calling `.convert()` or `.convert_alpha()` after loading converts the image to the display's pixel format. This can make blitting **2–3× faster**. Call it once after loading, before entering the game loop.
:::

## `pygame.locals` Constants

`pygame.locals` exports all Pygame constants (key codes, event types, flags) into the current namespace. You can import them for cleaner code:

```python
from pygame.locals import (
    QUIT, KEYDOWN, KEYUP, MOUSEBUTTONDOWN,
    K_LEFT, K_RIGHT, K_UP, K_DOWN, K_SPACE, K_ESCAPE,
    FULLSCREEN, RESIZABLE,
)

# Now you can write:
if event.type == QUIT:
    running = False
# Instead of:
if event.type == pygame.QUIT:
    running = False
```

## Complete Example: Bouncing Ball

Here is a complete, self-contained Pygame program featuring a ball that bounces off all four walls:

```python
import pygame
import sys

# ── Constants ────────────────────────────────────────────────
SCREEN_WIDTH  = 800
SCREEN_HEIGHT = 600
FPS           = 60

BG_COLOR      = (20, 20, 40)
BALL_COLOR    = (220, 80,  80)
TEXT_COLOR    = (200, 200, 200)

BALL_RADIUS   = 20
BALL_START_X  = SCREEN_WIDTH  // 2
BALL_START_Y  = SCREEN_HEIGHT // 2
BALL_SPEED_X  = 5
BALL_SPEED_Y  = 4

# ── Initialization ───────────────────────────────────────────
pygame.init()
screen = pygame.display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))
pygame.display.set_caption("Bouncing Ball")
clock  = pygame.time.Clock()
font   = pygame.font.SysFont("monospace", 18)

# ── Ball State ───────────────────────────────────────────────
ball_x  = float(BALL_START_X)
ball_y  = float(BALL_START_Y)
speed_x = float(BALL_SPEED_X)
speed_y = float(BALL_SPEED_Y)
bounces = 0

# ── Game Loop ────────────────────────────────────────────────
running = True
while running:

    # 1. Events
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
        if event.type == pygame.KEYDOWN:
            if event.key == pygame.K_ESCAPE:
                running = False
            if event.key == pygame.K_SPACE:
                # Reset ball to center
                ball_x, ball_y = BALL_START_X, BALL_START_Y
                bounces = 0

    # 2. Update
    ball_x += speed_x
    ball_y += speed_y

    # Bounce off left / right walls
    if ball_x - BALL_RADIUS <= 0:
        ball_x   = BALL_RADIUS
        speed_x  = abs(speed_x)
        bounces += 1
    elif ball_x + BALL_RADIUS >= SCREEN_WIDTH:
        ball_x   = SCREEN_WIDTH - BALL_RADIUS
        speed_x  = -abs(speed_x)
        bounces += 1

    # Bounce off top / bottom walls
    if ball_y - BALL_RADIUS <= 0:
        ball_y   = BALL_RADIUS
        speed_y  = abs(speed_y)
        bounces += 1
    elif ball_y + BALL_RADIUS >= SCREEN_HEIGHT:
        ball_y   = SCREEN_HEIGHT - BALL_RADIUS
        speed_y  = -abs(speed_y)
        bounces += 1

    # 3. Draw
    screen.fill(BG_COLOR)

    # Draw ball
    pygame.draw.circle(
        screen,
        BALL_COLOR,
        (int(ball_x), int(ball_y)),
        BALL_RADIUS
    )

    # Draw HUD
    fps_text     = font.render(f"FPS: {clock.get_fps():.0f}", True, TEXT_COLOR)
    bounce_text  = font.render(f"Bounces: {bounces}",        True, TEXT_COLOR)
    hint_text    = font.render("SPACE = reset  ESC = quit",  True, TEXT_COLOR)

    screen.blit(fps_text,    (10, 10))
    screen.blit(bounce_text, (10, 32))
    screen.blit(hint_text,   (10, SCREEN_HEIGHT - 28))

    pygame.display.flip()

    # 4. Cap FPS
    clock.tick(FPS)

pygame.quit()
sys.exit()
```

### What This Demonstrates

- `pygame.init()` / `pygame.quit()` lifecycle
- `display.set_mode()` and `display.set_caption()`
- The four-part game loop: events → update → draw → tick
- `pygame.draw.circle()` for rendering
- `pygame.font.SysFont()` and `font.render()` for HUD text
- `screen.blit()` to composite surfaces
- Boundary collision via simple arithmetic
- `clock.tick(FPS)` for frame-rate control
- Keyboard input via `event.key`

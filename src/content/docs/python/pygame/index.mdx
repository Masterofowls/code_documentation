Pygame is a cross-platform set of Python modules designed for writing video games. Built on top of the [SDL](https://www.libsdl.org/) (Simple DirectMedia Layer) library, it provides access to graphics, sound, input devices, and timing — everything you need to build 2D games and interactive applications without a heavy game engine.

:::tip[Pygame CE]
**Pygame Community Edition (Pygame CE)** is the actively maintained fork of the original Pygame project. It includes bug fixes, performance improvements, and new features. For new projects, `pip install pygame-ce` is recommended over the legacy `pygame` package.
:::

## What You Can Build

Pygame is well-suited for a wide range of 2D projects:

- **Arcade games** — shooters, platformers, top-down adventures
- **Puzzle games** — tile-matching, physics puzzles, word games
- **Board and card games** — chess, solitaire, tabletop simulators
- **Simulations** — cellular automata, particle systems, agent-based models
- **Prototypes** — rapid game-concept validation before moving to a larger engine
- **Educational tools** — interactive visualizations, math demos, CS teaching aids
- **Game jams** — Pygame is a popular choice for Ludum Dare and similar events

## Library Comparison

Choosing the right library depends on your goals. Here's how Pygame stacks up against its main alternatives:

| Feature | Pygame / Pygame CE | Arcade | Pyglet | PySDL2 |
|---|---|---|---|---|
| **Maturity** | Very mature (2000+) | Modern (2016+) | Mature (2007+) | Low-level wrapper |
| **Ease of use** | Beginner-friendly | Very beginner-friendly | Moderate | Advanced |
| **Performance** | SDL2 (software + HW) | OpenGL (GPU-accelerated) | OpenGL (GPU-accelerated) | SDL2 raw |
| **Sprite system** | Built-in | Built-in + physics | Manual | None |
| **Sound** | SDL2_mixer | FFmpeg-based | OpenAL | SDL2_mixer |
| **3D support** | None | None | Basic OpenGL | None |
| **Active dev** | ✅ (CE fork) | ✅ | ✅ | Limited |
| **Best for** | General 2D games | Learning, clean API | OpenGL flexibility | C-level control |

:::caution[Performance Ceiling]
Pygame uses software rendering by default. For games with thousands of moving sprites or advanced visual effects, consider **Arcade** (OpenGL) or a full engine like **Godot** with Python bindings.
:::

## Key Modules

Pygame is organized into focused modules. You only import what you need.

| Module | Purpose |
|---|---|
| `pygame.display` | Create and manage the game window |
| `pygame.draw` | Draw shapes (rect, circle, line, polygon, arc) |
| `pygame.event` | Poll and handle user input events |
| `pygame.font` | Render text using system or custom fonts |
| `pygame.image` | Load and save image files (PNG, JPEG, BMP, etc.) |
| `pygame.mixer` | Play background music and sound effects |
| `pygame.sprite` | Sprite and group classes with built-in collision |
| `pygame.time` | Frame rate control and timing utilities |
| `pygame.transform` | Scale, rotate, flip, and zoom surfaces |
| `pygame.key` | Keyboard state polling |
| `pygame.mouse` | Mouse position and button state |
| `pygame.joystick` | Gamepad and joystick input |
| `pygame.mask` | Pixel-perfect collision masks |
| `pygame.math` | 2D/3D vector math |
| `pygame.rect` | Rectangle operations (collision, positioning) |
| `pygame.surface` | Off-screen rendering surfaces |

## Minimal Window Example

The following is the smallest complete Pygame program — a window that stays open until the user closes it:

```python
import pygame

# --- Initialization ---
pygame.init()

SCREEN_WIDTH = 800
SCREEN_HEIGHT = 600
FPS = 60

screen = pygame.display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))
pygame.display.set_caption("My Pygame Window")
clock = pygame.time.Clock()

# --- Game Loop ---
running = True
while running:
    # 1. Handle events
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

    # 2. Update game state
    # (nothing to update yet)

    # 3. Draw
    screen.fill((30, 30, 30))  # Dark background

    # 4. Flip display and tick clock
    pygame.display.flip()
    clock.tick(FPS)

# --- Cleanup ---
pygame.quit()
```

Every Pygame program follows this same four-step loop: **handle events → update state → draw → present**.

## Core Concepts at a Glance

### Surfaces

Everything in Pygame is drawn onto a **Surface** — a rectangular grid of pixels. The main window is a Surface. Images are Surfaces. Text is rendered onto a Surface. You can blit (copy) one Surface onto another.

```python
# Create a 200x100 surface filled with red
surf = pygame.Surface((200, 100))
surf.fill((255, 0, 0))

# Blit it onto the screen at position (50, 50)
screen.blit(surf, (50, 50))
```

### Rects

A `pygame.Rect` stores a position and size. It's used everywhere — for collision detection, positioning sprites, clipping, and more.

```python
rect = pygame.Rect(100, 200, 64, 64)  # x, y, width, height
print(rect.center)    # (132, 232)
print(rect.topleft)   # (100, 200)
print(rect.collidepoint(110, 210))  # True
```

### The Coordinate System

Pygame uses a **top-left origin** system. `(0, 0)` is the top-left corner of the window. X increases to the right; Y increases **downward**.

```
(0,0) ──────────────► x
  │
  │
  │
  ▼
  y
```

### Colors

Colors are RGB tuples: `(red, green, blue)` where each channel is 0–255. Alpha (opacity) is added as a fourth value: `(r, g, b, a)`.

```python
BLACK   = (0,   0,   0)
WHITE   = (255, 255, 255)
RED     = (255, 0,   0)
GREEN   = (0,   255, 0)
BLUE    = (0,   0,   255)
YELLOW  = (255, 255, 0)
CYAN    = (0,   255, 255)
MAGENTA = (255, 0,   255)

# Semi-transparent green (requires SRCALPHA surface)
SEMI_GREEN = (0, 255, 0, 128)
```

PyQt6 is a set of Python bindings for the [Qt](https://www.qt.io/) application framework — one of the most powerful, mature, and cross-platform GUI toolkits in existence. With PyQt6 you can ship polished, native-looking desktop applications on Windows, macOS, and Linux from a single Python codebase.

## What is Qt?

Qt (pronounced _cute_) is a C++ framework originally created by Trolltech in 1991 and now maintained by The Qt Company. It powers everything from embedded touchscreens and automotive infotainment systems to full desktop IDEs (Qt Creator itself) and scientific tools. The framework provides:

- A **widget toolkit** with hundreds of ready-made UI controls.
- A **signal/slot** event system that decouples components cleanly.
- Modules for **networking, databases, multimedia, 3D graphics, WebEngine**, and more.
- A declarative UI language called **QML** for fluid, animated interfaces.
- **Qt Designer** — a drag-and-drop WYSIWYG UI builder.

PyQt6 wraps the entire Qt 6 C++ API in Python, giving you high-level productivity with near-native performance.

## Key Features of PyQt6

### Cross-Platform Native Look

Qt renders using the native platform APIs on each OS, so your app automatically matches the system theme on Windows, macOS, and Linux.

### Rich Widget Library

Hundreds of widgets out of the box — buttons, tables, trees, calendars, web views, charts, and more — all customisable with Qt Style Sheets (QSS), which is a CSS-like syntax.

### Model/View Architecture

PyQt6 includes a full **Model/View/Delegate** framework for displaying structured data (lists, tables, trees) without duplicating data between your logic and your UI.

### Signals & Slots

The signal/slot system is a type-safe, loosely-coupled way for objects to communicate. Any widget can emit a signal; any slot (method or lambda) can receive it — even across threads.

### Qt Designer Integration

Design your UI visually in Qt Designer, save it as a `.ui` file, and load it directly in Python with `uic.loadUi()` — no code generation required.

### Multithreading

`QThread` and `QThreadPool` let you run expensive work off the GUI thread while keeping the UI responsive, communicating results back via signals.

## Hello World — Your First PyQt6 Window

Every PyQt6 program follows the same three-step skeleton:

1. Create a `QApplication` (manages the event loop).
2. Create and show one or more windows.
3. Call `app.exec()` to start the event loop.

```python
import sys
from PyQt6.QtWidgets import QApplication, QMainWindow, QLabel
from PyQt6.QtCore import Qt


class HelloWindow(QMainWindow):
    """A minimal main window that says hello."""

    def __init__(self) -> None:
        super().__init__()
        self.setWindowTitle("Hello, PyQt6!")
        self.setMinimumSize(400, 200)

        # Central widget — every QMainWindow needs one
        label = QLabel("Hello, World! 👋", self)
        label.setAlignment(Qt.AlignmentFlag.AlignCenter)
        label.setStyleSheet("font-size: 24px; color: #2563eb;")
        self.setCentralWidget(label)


def main() -> None:
    app = QApplication(sys.argv)
    app.setApplicationName("HelloPyQt6")
    app.setOrganizationName("MyOrg")

    window = HelloWindow()
    window.show()

    sys.exit(app.exec())


if __name__ == "__main__":
    main()
```

Run it:

```bash
python hello.py
```

You should see a centred, blue "Hello, World!" message in a resizable window.

:::tip[Always pass sys.argv]
`QApplication(sys.argv)` lets Qt parse its own command-line flags (like `--style fusion`). Pass it even if you don't use CLI arguments yourself.
:::

## Quick Reference

```python
# Imports you'll use constantly
from PyQt6.QtWidgets import (
    QApplication, QMainWindow, QWidget,
    QPushButton, QLabel, QLineEdit, QTextEdit,
    QVBoxLayout, QHBoxLayout, QGridLayout,
    QMessageBox, QFileDialog,
)
from PyQt6.QtCore import Qt, pyqtSignal, QThread, QTimer
from PyQt6.QtGui import QIcon, QFont, QColor, QPalette
```

:::tip[Enums changed in PyQt6]
PyQt6 uses **fully-qualified enums**. Where PyQt5 wrote `Qt.AlignCenter`, PyQt6 requires `Qt.AlignmentFlag.AlignCenter`. The same pattern applies to all enums — check Qt docs if you see `AttributeError` on an enum value.
:::

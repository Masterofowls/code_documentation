Python is a dynamically typed language — you don't declare types, but every value has one. Understanding the built-in types is the foundation of writing clear, correct Python. This guide covers all core types with practical examples, common pitfalls, and modern idioms.

## Numbers

Python has three numeric types: `int`, `float`, and `complex`.

### Integers and Floats

```python
# Integers — unlimited precision
population = 8_100_000_000   # underscores for readability
negative = -42
hexadecimal = 0xFF           # 255
binary = 0b1010              # 10
octal = 0o17                 # 15

# Floats — 64-bit IEEE 754 doubles
pi = 3.14159
tiny = 1.5e-10
big = 6.022e23

print(type(population))      # <class 'int'>
print(type(pi))              # <class 'float'>
```

### Arithmetic Operators

```python
a, b = 17, 5

print(a + b)    # 22  — addition
print(a - b)    # 12  — subtraction
print(a * b)    # 85  — multiplication
print(a / b)    # 3.4 — true division (always float)
print(a // b)   # 3   — floor division (integer result)
print(a % b)    # 2   — modulo (remainder)
print(a ** b)   # 1419857 — exponentiation
```

:::caution[Division always returns float]
`/` always returns a `float` in Python 3, even when dividing two integers. Use `//` when you need an integer quotient.
```python
print(10 / 2)    # 5.0  — NOT 5
print(10 // 2)   # 5
```
:::

### Complex Numbers

```python
z1 = 3 + 4j
z2 = complex(1, -2)

print(z1.real)          # 3.0
print(z1.imag)          # 4.0
print(abs(z1))          # 5.0  — magnitude (Pythagorean theorem)
print(z1 + z2)          # (4+2j)
print(z1 * z2)          # (11-2j)
```

### Numeric Conversions

```python
print(int(3.9))         # 3    — truncates, does NOT round
print(int('42'))        # 42
print(int('FF', 16))    # 255  — parse hex string
print(float('3.14'))    # 3.14
print(float(7))         # 7.0
print(round(3.14159, 2)) # 3.14
```

:::tip[Use round() not int() for rounding]
`int()` truncates toward zero. Use `round()` when you want standard rounding behaviour.
```python
int(2.9)    # 2  — probably not what you want
round(2.9)  # 3  — correct
```
:::

### The `math` Module

```python
import math

print(math.floor(3.9))       # 3
print(math.ceil(3.1))        # 4
print(math.sqrt(144))        # 12.0
print(math.log(math.e))      # 1.0
print(math.log10(1000))      # 3.0
print(math.pi)               # 3.141592653589793
print(math.inf)              # inf
print(math.isfinite(math.inf)) # False
print(math.gcd(48, 18))      # 6
```

## Strings

Strings are immutable sequences of Unicode characters.

### Creation

```python
single = 'Hello, World!'
double = "She said, \"hi\""
escaped = 'it\'s a string'

# Triple quotes for multiline strings
poem = """Roses are red,
Violets are blue,
Python is great,
And so are you."""

# Raw strings — backslashes are literal (useful for file paths and regex)
path = r'C:\Users\dan\Documents\file.txt'
pattern = r'\d+\.\d+'       # regex pattern, backslashes not interpreted

print(path)     # C:\Users\dan\Documents\file.txt
print(len(poem))  # 58
```

### F-strings (Recommended for Formatting)

F-strings (formatted string literals) are the modern, readable way to embed expressions in strings.

```python
name = 'Alice'
age = 30
score = 98.765

# Basic interpolation
print(f'Hello, {name}!')                    # Hello, Alice!

# Expressions inside braces
print(f'In 5 years, {name} will be {age + 5}.')  # In 5 years, Alice will be 35.

# Format specifiers
print(f'Score: {score:.2f}')               # Score: 98.77
print(f'Score: {score:>10.2f}')            # Score:      98.77  (right-aligned, width 10)
print(f'Hex: {255:#010x}')                 # Hex: 0x000000ff

# Debugging with = (Python 3.8+)
x = 42
print(f'{x = }')                           # x = 42

# Calling methods inside f-strings
items = ['apple', 'banana', 'cherry']
print(f'Items: {", ".join(items)}')        # Items: apple, banana, cherry
```

### String Slicing

Strings support `[start:stop:step]` slicing (stop is exclusive).

```python
text = 'Hello, Python!'

print(text[0])        # H      — first character
print(text[-1])       # !      — last character
print(text[0:5])      # Hello  — characters 0-4
print(text[7:])       # Python! — from index 7 to end
print(text[:5])       # Hello  — from start to index 4
print(text[::2])      # Hlo yhn — every other character
print(text[0:5:2])    # Hlo    — slice with step
print(text[::-1])     # !nohtyP ,olleH — reversed
```

### Common String Methods

```python
s = '  Hello, World!  '

# Case methods
print(s.upper())              # '  HELLO, WORLD!  '
print(s.lower())              # '  hello, world!  '
print(s.title())              # '  Hello, World!  '
print(s.swapcase())           # '  hELLO, wORLD!  '

# Whitespace
print(s.strip())              # 'Hello, World!'
print(s.lstrip())             # 'Hello, World!  '
print(s.rstrip())             # '  Hello, World!'

# Search and test
print('World' in s)           # True
print(s.find('World'))        # 9   (index, or -1 if not found)
print(s.index('World'))       # 9   (raises ValueError if not found)
print(s.count('l'))           # 3
print(s.strip().startswith('Hello'))   # True
print(s.strip().endswith('!'))         # True

# Modification (returns new string — strings are immutable)
print(s.replace('World', 'Python'))   # '  Hello, Python!  '
print(s.strip().split(', '))          # ['Hello', 'World!']

# Joining
words = ['Python', 'is', 'awesome']
print(' '.join(words))        # Python is awesome
print('-'.join(words))        # Python-is-awesome

# Padding
print('42'.zfill(5))          # 00042
print('hi'.center(10, '-'))   # ----hi----
print('hi'.ljust(10, '.'))    # hi........
```

:::tip[Prefer `in` over `find()` for existence checks]
When you only need to know whether a substring exists, `in` is more readable than checking `find() != -1`.
```python
# Preferred
if 'error' in log_line:
    ...

# Avoid
if log_line.find('error') != -1:
    ...
```
:::

### Bytes vs Strings

```python
# str is Unicode text; bytes is raw binary data
text = 'café'
encoded = text.encode('utf-8')     # str → bytes
decoded = encoded.decode('utf-8')  # bytes → str

print(type(text))           # <class 'str'>
print(type(encoded))        # <class 'bytes'>
print(encoded)              # b'caf\xc3\xa9'
print(len(text))            # 4  (characters)
print(len(encoded))         # 5  (bytes — é is 2 bytes in UTF-8)
print(decoded)              # café
```

## Booleans

`bool` is a subclass of `int`. `True` equals `1` and `False` equals `0`.

```python
print(True + True)     # 2
print(True * 5)        # 5
print(False + 1)       # 1
print(type(True))      # <class 'bool'>
```

### Truthy and Falsy Values

Every Python object has a boolean value. These evaluate to `False`:

```python
falsy_values = [
    False, None, 0, 0.0, 0j,   # false primitives
    '', [], (), {}, set(),       # empty containers
]

for val in falsy_values:
    print(f'{repr(val):10} → {bool(val)}')

# Output:
# False      → False
# None       → False
# 0          → False
# 0.0        → False
# 0j         → False
# ''         → False
# []         → False
# ()         → False
# {}         → False
# set()      → False
```

Everything else is truthy — non-zero numbers, non-empty strings, non-empty containers.

### Comparison and Logical Operators

```python
x = 10

# Comparison operators
print(x > 5)           # True
print(x == 10)         # True
print(x != 20)         # True
print(5 < x < 20)      # True  — chained comparisons

# Logical operators
print(True and False)  # False
print(True or False)   # True
print(not True)        # False
```

### Short-Circuit Evaluation

`and` and `or` don't return `True`/`False` — they return one of their operands.

```python
# `and` returns the first falsy value, or the last value if all are truthy
print(0 and 'hello')       # 0       — stops at 0 (falsy)
print(1 and 'hello')       # 'hello' — returns last value
print('' and 'fallback')   # ''      — stops at '' (falsy)

# `or` returns the first truthy value, or the last value if all are falsy
print(0 or 'default')      # 'default'
print('value' or 'default') # 'value'
print(None or [] or 'last') # 'last'

# Practical use: default values
user_name = ''
display = user_name or 'Anonymous'
print(display)             # Anonymous
```

:::tip[Use `x if condition else y` for clarity in assignments]
Short-circuit evaluation works, but a ternary expression is often more readable when assigning defaults.
```python
# Readable
name = user_input if user_input else 'Anonymous'

# Also fine for simple defaults
name = user_input or 'Anonymous'
```
:::

## Lists

Lists are mutable, ordered sequences that can hold items of any type.

### Creation and Indexing

```python
fruits = ['apple', 'banana', 'cherry']
mixed = [1, 'two', 3.0, True, None]
nested = [[1, 2], [3, 4], [5, 6]]
empty = []

# Indexing
print(fruits[0])       # apple
print(fruits[-1])      # cherry

# Slicing
print(fruits[0:2])     # ['apple', 'banana']
print(fruits[::-1])    # ['cherry', 'banana', 'apple']  — reversed copy
```

### Mutation Methods

```python
numbers = [3, 1, 4, 1, 5, 9, 2, 6]

# Adding items
numbers.append(7)            # add to end
numbers.insert(0, 0)         # insert at index 0
numbers.extend([8, 9, 10])   # add multiple items

# Removing items
numbers.remove(1)            # removes FIRST occurrence of 1
popped = numbers.pop()       # removes and returns last item
popped_at = numbers.pop(0)   # removes and returns item at index 0

# Searching
print(numbers.index(9))      # index of first 9
print(numbers.count(1))      # count occurrences

# Ordering
numbers.sort()               # in-place ascending sort
numbers.sort(reverse=True)   # in-place descending sort
numbers.reverse()            # reverse in place

# sorted() returns a new list; the original is unchanged
words = ['banana', 'apple', 'cherry']
sorted_words = sorted(words, key=str.lower)
print(sorted_words)          # ['apple', 'banana', 'cherry']
print(words)                 # ['banana', 'apple', 'cherry']  — unchanged

# Other
numbers.clear()              # remove all items
copy = numbers.copy()        # shallow copy
```

:::caution[`.sort()` vs `sorted()` — know which you need]
`list.sort()` modifies the list in place and returns `None`. Assigning its return value is a common bug:
```python
numbers = [3, 1, 2]
result = numbers.sort()   # result is None!
result = sorted(numbers)  # result is [1, 2, 3]  ✓
```
:::

### List Comprehensions

List comprehensions are concise and fast — prefer them over `for`+`append` loops.

```python
# Basic form: [expression for item in iterable]
squares = [x ** 2 for x in range(10)]
print(squares)
# [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

# With condition: [expression for item in iterable if condition]
even_squares = [x ** 2 for x in range(10) if x % 2 == 0]
print(even_squares)
# [0, 4, 16, 36, 64]

# Nested comprehension — flatten a 2D list
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flat = [num for row in matrix for num in row]
print(flat)
# [1, 2, 3, 4, 5, 6, 7, 8, 9]

# Transformation with function call
words = ['hello', 'world', 'python']
upper_words = [w.upper() for w in words]
print(upper_words)
# ['HELLO', 'WORLD', 'PYTHON']
```

### Nested Lists

```python
grid = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9],
]

# Access element at row 1, column 2
print(grid[1][2])      # 6

# Iterate rows and columns
for row in grid:
    print(row)
# [1, 2, 3]
# [4, 5, 6]
# [7, 8, 9]
```

:::caution[Avoid multiplying mutable objects to create 2D lists]
```python
# WRONG — all rows are the same list object
bad = [[0] * 3] * 3
bad[0][0] = 99
print(bad)   # [[99, 0, 0], [99, 0, 0], [99, 0, 0]]  — oops!

# CORRECT — use a comprehension
good = [[0] * 3 for _ in range(3)]
good[0][0] = 99
print(good)  # [[99, 0, 0], [0, 0, 0], [0, 0, 0]]  ✓
```
:::

## Tuples

Tuples are immutable ordered sequences. Use them for data that shouldn't change.

### Creation and Access

```python
point = (3, 7)
rgb = (255, 128, 0)
single = (42,)          # trailing comma required for single-element tuple
also_tuple = 1, 2, 3    # parentheses optional — this is a tuple

print(point[0])         # 3
print(len(rgb))         # 3
print(type(single))     # <class 'tuple'>

# Tuples support all read-only sequence operations
print(2 in rgb)         # False
print(rgb.count(255))   # 1
print(rgb.index(128))   # 1
```

### Unpacking

```python
x, y = (3, 7)
print(x, y)              # 3 7

# Swap variables without a temp
a, b = 10, 20
a, b = b, a
print(a, b)              # 20 10

# Extended unpacking with *
first, *middle, last = [1, 2, 3, 4, 5]
print(first)             # 1
print(middle)            # [2, 3, 4]
print(last)              # 5

# Unpack in for loops
coords = [(0, 0), (1, 2), (3, 4)]
for x, y in coords:
    print(f'x={x}, y={y}')
# x=0, y=0
# x=1, y=2
# x=3, y=4
```

### Named Tuples

`namedtuple` from `collections` gives tuple fields human-readable names without the overhead of a full class.

```python
from collections import namedtuple

Point = namedtuple('Point', ['x', 'y'])
Color = namedtuple('Color', 'red green blue')   # space-separated string also works

p = Point(x=10, y=20)
c = Color(255, 128, 0)

print(p.x, p.y)          # 10 20
print(p[0], p[1])        # 10 20  — still works as a regular tuple
print(c.red)             # 255

# Named tuples are immutable
try:
    p.x = 99
except AttributeError as e:
    print(e)             # can't set attribute

# Useful _replace() for modified copies
p2 = p._replace(x=99)
print(p2)                # Point(x=99, y=20)
print(p)                 # Point(x=10, y=20)  — original unchanged
```

:::tip[Tuples vs Lists — the rule of thumb]
Use a **tuple** when the collection represents a single logical record (coordinates, RGB values, database row) or when immutability is important for correctness. Use a **list** when the collection may grow or change, or when items are conceptually the same kind of thing.
:::

## Sets

Sets are unordered collections of unique, hashable values.

### Creation and Basic Operations

```python
fruits = {'apple', 'banana', 'cherry', 'banana'}   # duplicates removed
print(fruits)            # {'apple', 'banana', 'cherry'}  (order not guaranteed)
print(len(fruits))       # 3

# Creating from an iterable
letters = set('mississippi')
print(letters)           # {'m', 'i', 's', 'p'}

empty_set = set()        # NOT {} — that creates a dict!
```

### Mutation

```python
tags = {'python', 'coding'}

tags.add('tutorial')            # add one element
tags.update(['tips', 'coding']) # add multiple; duplicates ignored
print(tags)                     # {'python', 'coding', 'tutorial', 'tips'}

tags.remove('tips')             # raises KeyError if not found
tags.discard('missing')         # silently does nothing if not found
popped = tags.pop()             # removes and returns an arbitrary element

print('python' in tags)         # True  — O(1) membership test
```

### Set Operations

```python
a = {1, 2, 3, 4, 5}
b = {4, 5, 6, 7, 8}

# Union — all elements from either set
print(a | b)               # {1, 2, 3, 4, 5, 6, 7, 8}
print(a.union(b))          # same

# Intersection — elements in both sets
print(a & b)               # {4, 5}
print(a.intersection(b))   # same

# Difference — in a but not b
print(a - b)               # {1, 2, 3}
print(a.difference(b))     # same

# Symmetric difference — in either but not both
print(a ^ b)               # {1, 2, 3, 6, 7, 8}

# Subset / superset checks
print({1, 2}.issubset(a))  # True
print(a.issuperset({1, 2})) # True
print(a.isdisjoint({6, 7})) # True — no common elements
```

### Set Comprehensions

```python
# Unique vowels from a sentence
sentence = 'the quick brown fox jumps over the lazy dog'
vowels = {ch for ch in sentence if ch in 'aeiou'}
print(vowels)              # {'a', 'e', 'i', 'o', 'u'}

# Unique word lengths
words = ['hi', 'hello', 'hey', 'howdy', 'world']
lengths = {len(w) for w in words}
print(lengths)             # {2, 5}
```

## Dictionaries

Dictionaries map unique, hashable keys to values. As of Python 3.7+, insertion order is preserved.

### Creation

```python
# Literal syntax
person = {
    'name': 'Alice',
    'age': 30,
    'city': 'Paris',
}

# dict() constructor
config = dict(host='localhost', port=5432, debug=False)

# From list of key-value pairs
pairs = [('a', 1), ('b', 2), ('c', 3)]
mapping = dict(pairs)

empty = {}
```

### CRUD Operations

```python
inventory = {'apple': 50, 'banana': 30}

# Read
print(inventory['apple'])         # 50
print(inventory.get('mango'))     # None   — no KeyError
print(inventory.get('mango', 0))  # 0      — with default value

# Create / Update
inventory['cherry'] = 100         # create
inventory['apple'] = 55           # update

# Delete
del inventory['banana']
removed = inventory.pop('cherry')           # removes and returns value
removed_or = inventory.pop('missing', -1)   # returns -1 if key absent

# Membership test (checks keys)
print('apple' in inventory)       # True
print('banana' in inventory)      # False
```

:::caution[Always use `.get()` when a key might be absent]
Accessing a missing key with `[]` raises `KeyError`. Use `.get()` for safe access.
```python
data = {'name': 'Alice'}

# This raises KeyError:
# print(data['age'])

# Safe alternatives:
print(data.get('age'))          # None
print(data.get('age', 0))       # 0
```
:::

### Iterating

```python
scores = {'Alice': 95, 'Bob': 87, 'Carol': 92}

# Keys (default iteration)
for name in scores:
    print(name)

# Values
for score in scores.values():
    print(score)

# Key-value pairs — most common
for name, score in scores.items():
    print(f'{name}: {score}')

# Useful views
print(list(scores.keys()))     # ['Alice', 'Bob', 'Carol']
print(list(scores.values()))   # [95, 87, 92]
print(list(scores.items()))    # [('Alice', 95), ('Bob', 87), ('Carol', 92)]
```

### Dict Comprehensions

```python
# Invert a dictionary
original = {'a': 1, 'b': 2, 'c': 3}
inverted = {v: k for k, v in original.items()}
print(inverted)           # {1: 'a', 2: 'b', 3: 'c'}

# Filter entries
high_scores = {name: s for name, s in scores.items() if s >= 90}
print(high_scores)        # {'Alice': 95, 'Carol': 92}

# Transform values
grade_map = {name: ('Pass' if s >= 60 else 'Fail') for name, s in scores.items()}
```

### Merging Dictionaries

```python
defaults = {'color': 'blue', 'size': 'medium', 'weight': 10}
overrides = {'color': 'red', 'size': 'large'}

# Python 3.9+ — merge operator (right side wins on conflict)
merged = defaults | overrides
print(merged)
# {'color': 'red', 'size': 'large', 'weight': 10}

# Update in place
defaults |= overrides

# Earlier Python — unpack into a new dict
merged = {**defaults, **overrides}
```

### `setdefault` and `defaultdict`

```python
# setdefault — insert key with default if absent, then return value
word_counts = {}
for word in 'the cat sat on the mat'.split():
    word_counts.setdefault(word, 0)
    word_counts[word] += 1

# defaultdict — cleaner approach
from collections import defaultdict

word_counts = defaultdict(int)
for word in 'the cat sat on the mat'.split():
    word_counts[word] += 1

print(dict(word_counts))
# {'the': 2, 'cat': 1, 'sat': 1, 'on': 1, 'mat': 1}
```

## None

`None` is Python's null type — it represents the intentional absence of a value. There is exactly one `None` object.

```python
result = None
print(result)           # None
print(type(result))     # <class 'NoneType'>

# Functions without a return statement implicitly return None
def greet(name):
    print(f'Hello, {name}!')

x = greet('Alice')      # Hello, Alice!
print(x)                # None
```

### Checking for None

```python
value = None

# CORRECT — use `is` for identity comparison
if value is None:
    print('no value')

if value is not None:
    print('has a value')

# AVOID — == works but is misleading
# Custom classes can override == to return True for None comparison
if value == None:        # works, but poor style
    ...
```

:::tip[`is None` vs `== None`]
Always use `is None` and `is not None`. The `is` operator checks object identity (there is only one `None` in the interpreter), while `==` calls `__eq__`, which could theoretically be overridden by a custom class to produce unexpected results.
:::

```python
# None as a sentinel default argument
def connect(host, port=None):
    if port is None:
        port = 5432      # apply real default inside the function
    return f'{host}:{port}'

print(connect('localhost'))        # localhost:5432
print(connect('localhost', 3306))  # localhost:3306
```

## Type Checking

### `type()` and `isinstance()`

```python
x = 42
print(type(x))                    # <class 'int'>
print(type(x) is int)             # True
print(type(x).__name__)           # 'int'

# isinstance() is preferred — respects inheritance
print(isinstance(x, int))        # True
print(isinstance(x, (int, float))) # True — checks against a tuple of types
print(isinstance(True, int))      # True — bool is a subclass of int!
```

:::tip[Prefer `isinstance()` over `type() ==`]
`isinstance()` handles inheritance correctly. `type(x) is int` returns `False` for subclasses like `bool`, whereas `isinstance(True, int)` returns `True` as expected.
:::

### Type Hints

Type hints make code easier to read and enable static analysis with tools like `mypy` and `pyright`. They are not enforced at runtime.

<Tabs>
<TabItem label="Basic hints">
```python
# Variable annotations
name: str = 'Alice'
age: int = 30
height: float = 5.9
active: bool = True

# Function annotations
def greet(name: str, times: int = 1) -> str:
    return (f'Hello, {name}! ' * times).strip()

def nothing() -> None:
    print('no return value')
```
</TabItem>
<TabItem label="Collection hints">
```python
from typing import Optional, Union

# Modern syntax (Python 3.9+)
def process(items: list[int]) -> dict[str, int]:
    return {'sum': sum(items), 'count': len(items)}

# Nested types
matrix: list[list[float]] = [[1.0, 2.0], [3.0, 4.0]]
registry: dict[str, list[str]] = {'admins': ['alice', 'bob']}

# Optional — value OR None
def find_user(user_id: int) -> Optional[str]:  # same as str | None
    ...

# Union — multiple possible types (Python 3.10+ uses X | Y syntax)
def stringify(value: Union[int, float, str]) -> str:
    return str(value)

# Python 3.10+ shorthand
def stringify_new(value: int | float | str) -> str:
    return str(value)
```
</TabItem>
<TabItem label="Advanced hints">
```python
from typing import TypeVar, Generic, Callable, Any
from collections.abc import Sequence, Mapping, Iterator

# TypeVar for generic functions
T = TypeVar('T')

def first(items: Sequence[T]) -> T:
    return items[0]

# Callable type
Handler = Callable[[str, int], bool]

def apply(handler: Handler, message: str, code: int) -> bool:
    return handler(message, code)

# Any — escape hatch (use sparingly)
def debug_print(value: Any) -> None:
    print(repr(value))
```

## Type Conversion

Python provides built-in functions to convert between types.

```python
# To int
print(int('42'))           # 42
print(int('0b1010', 2))    # 10  — binary string
print(int('0xFF', 16))     # 255 — hex string
print(int(3.99))           # 3   — truncates
print(int(True))           # 1
print(int(False))          # 0

# To float
print(float('3.14'))       # 3.14
print(float('inf'))        # inf
print(float(7))            # 7.0
print(float(True))         # 1.0

# To string
print(str(42))             # '42'
print(str(3.14))           # '3.14'
print(str(True))           # 'True'
print(str(None))           # 'None'
print(str([1, 2, 3]))      # '[1, 2, 3]'

# To bool
print(bool(0))             # False
print(bool(1))             # True
print(bool(''))            # False
print(bool('hello'))       # True
print(bool([]))            # False
print(bool([0]))           # True  — list has one item, even if it's 0

# To list
print(list('abc'))         # ['a', 'b', 'c']
print(list((1, 2, 3)))     # [1, 2, 3]
print(list({4, 5, 6}))     # [4, 5, 6]  (order not guaranteed)
print(list({'a': 1, 'b': 2}))  # ['a', 'b']  — keys only

# To tuple
print(tuple([1, 2, 3]))    # (1, 2, 3)
print(tuple('hello'))      # ('h', 'e', 'l', 'l', 'o')

# To set
print(set([1, 2, 2, 3, 3, 3]))  # {1, 2, 3}  — deduplication

# To dict
print(dict([('a', 1), ('b', 2)]))  # {'a': 1, 'b': 2}
print(dict(zip('abc', [1, 2, 3]))) # {'a': 1, 'b': 2, 'c': 3}
```

:::caution[Not all conversions are safe]
Some conversions raise exceptions for invalid input. Handle them with `try/except`:
```python
def safe_int(value: str, default: int = 0) -> int:
    try:
        return int(value)
    except (ValueError, TypeError):
        return default

print(safe_int('42'))       # 42
print(safe_int('abc'))      # 0
print(safe_int('3.14'))     # 0  — int() can't parse float strings directly
print(safe_int(None))       # 0
```
:::

## Quick Reference

| Type | Mutable | Ordered | Unique values | Syntax |
|------|---------|---------|---------------|--------|
| `int` | No | — | — | `42` |
| `float` | No | — | — | `3.14` |
| `complex` | No | — | — | `3+4j` |
| `str` | No | Yes | No | `'hello'` |
| `bool` | No | — | — | `True` / `False` |
| `list` | **Yes** | Yes | No | `[1, 2, 3]` |
| `tuple` | No | Yes | No | `(1, 2, 3)` |
| `set` | **Yes** | No | **Yes** | `{1, 2, 3}` |
| `frozenset` | No | No | **Yes** | `frozenset({1, 2})` |
| `dict` | **Yes** | Yes (3.7+) | Keys only | `{'a': 1}` |
| `NoneType` | No | — | — | `None` |

:::tip[Memory tip — mutable vs immutable]
Immutable types (`int`, `float`, `str`, `tuple`, `frozenset`) can be used as dictionary keys and set members. Mutable types (`list`, `dict`, `set`) cannot because their hash value could change.
```python
# This works
d = {(1, 2): 'a point'}

# This raises TypeError: unhashable type: 'list'
d = {[1, 2]: 'a point'}
```
:::

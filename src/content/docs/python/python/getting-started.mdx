## Installing Python

<Tabs>
  <TabItem label="Windows">
    <Steps>
      1. Visit [python.org/downloads](https://www.python.org/downloads/) and download the latest Python 3 installer.
      2. Run the installer. **Check "Add Python to PATH"** before clicking Install.
      3. Verify the installation:
         ```powershell
         python --version
         # Python 3.13.x
         pip --version
         # pip 24.x
         ```
</Steps>
</TabItem>
  <TabItem label="macOS">
    Using [Homebrew](https://brew.sh) (recommended):
    ```bash
    brew install python
    python3 --version
    # Python 3.13.x
    ```
    Or use [pyenv](https://github.com/pyenv/pyenv) for version management:
    ```bash
    brew install pyenv
    pyenv install 3.13
    pyenv global 3.13
    ```
</TabItem>
  <TabItem label="Linux (Ubuntu/Debian)">
    ```bash
    sudo apt update
    sudo apt install python3 python3-pip python3-venv
    python3 --version
    # Python 3.13.x
    ```
</TabItem>
</Tabs>

## Virtual Environments

Always use a virtual environment to isolate project dependencies. This prevents version conflicts between projects.

```bash
# Create a virtual environment named .venv
python -m venv .venv

# Activate it
# macOS / Linux:
source .venv/bin/activate

# Windows (PowerShell):
.\.venv\Scripts\Activate.ps1

# Verify — your prompt will show (.venv)
python --version
which python   # or 'where python' on Windows
```

<Aside type="tip">
Add `.venv/` to your `.gitignore`. Never commit virtual environment directories.
</Aside>

### Installing packages

```bash
# Install a package
pip install requests

# Install from a requirements file
pip install -r requirements.txt

# Save current dependencies
pip freeze > requirements.txt
```

## Your First Python Program

Create a file called `hello.py`:

```python title="hello.py"
# My first Python program
def greet(name: str) -> str:
    """Return a personalised greeting."""
    return f'Hello, {name}!'


if __name__ == '__main__':
    message = greet('World')
    print(message)
```

Run it:

```bash
python hello.py
```

```
Hello, World!
```

## The Python REPL

Python ships with an interactive shell — great for quick experiments:

```bash
python
```

```python
>>> 2 + 2
4
>>> name = 'Python'
>>> f'I love {name}!'
'I love Python!'
>>> [x ** 2 for x in range(5)]
[0, 1, 4, 9, 16]
>>> exit()
```

<Aside type="tip">
Install **IPython** (`pip install ipython`) for a much richer REPL experience with syntax highlighting, auto-completion, and `%magic` commands.
</Aside>

## Recommended Editor Setup

**VS Code** with the [Python extension](https://marketplace.visualstudio.com/items?itemName=ms-python.python) is the most popular choice.

Install useful extensions:
- `ms-python.python` — IntelliSense, linting, debugging
- `ms-python.black-formatter` — Black code formatter
- `ms-python.pylint` — Linting

Create `.vscode/settings.json` in your project:

```json title=".vscode/settings.json"
{
  "python.defaultInterpreterPath": "${workspaceFolder}/.venv/bin/python",
  "editor.formatOnSave": true,
  "[python]": {
    "editor.defaultFormatter": "ms-python.black-formatter"
  }
}
```

## Project Structure

A well-organised Python project looks like this:

```
my_project/
├── .venv/              # Virtual environment (gitignored)
├── src/
│   └── my_project/
│       ├── __init__.py
│       └── main.py
├── tests/
│   └── test_main.py
├── .gitignore
├── pyproject.toml      # Project metadata and dependencies
└── README.md
```

### pyproject.toml (modern standard)

```toml title="pyproject.toml"
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "my-project"
version = "0.1.0"
description = "My Python project"
requires-python = ">=3.11"
dependencies = [
    "requests>=2.31",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
    "black>=24.0",
    "ruff>=0.1",
]
```

## Running Python Scripts

```bash
# Run a script
python src/my_project/main.py

# Run a module (preferred for packages)
python -m my_project.main

# Run with verbose output
python -v script.py

# Check syntax without running
python -m py_compile script.py
```

## Common Pitfalls for Beginners

:::caution[IndentationError]
Python uses **indentation** (spaces, not braces) to define code blocks. Always use 4 spaces — never mix tabs and spaces.
```python
# ✅ Correct
def add(a, b):
    return a + b   # 4 spaces

# ❌ Wrong — mixing tabs and spaces causes IndentationError
def add(a, b):
	return a + b   # tab character
```
:::

:::caution[Python 2 vs Python 3]
On some systems, `python` runs Python 2. Always check with `python --version`. Use `python3` explicitly if needed.
:::

:::tip[Use uv for Speed]
[uv](https://docs.astral.sh/uv/) is a blazing-fast Python package manager written in Rust. It replaces `pip` and `venv`:
```bash
pip install uv
uv venv && uv pip install requests
```
:::

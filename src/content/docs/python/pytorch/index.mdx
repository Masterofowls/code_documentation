PyTorch is an open-source machine learning framework developed by Meta AI Research. Built on dynamic computation graphs and deep Python integration, it has become the dominant framework for AI research and is rapidly expanding in production deployments.

## What is PyTorch?

PyTorch provides two core capabilities:

- **N-dimensional tensor computation** (like NumPy) with strong GPU acceleration
- **Automatic differentiation** for building and training deep neural networks

Its define-by-run (eager execution) model lets you write models in regular Python — standard control flow, debugging tools, and third-party libraries all work natively without any special compilation step.

```python
import torch
import torch.nn as nn

# A simple two-layer network — pure Python, no graph compilation required
class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 128)
        self.fc2 = nn.Linear(128, 10)
        self.relu = nn.ReLU()

    def forward(self, x):
        x = self.relu(self.fc1(x))
        return self.fc2(x)

model = SimpleNet()
x = torch.randn(32, 784)       # batch_size=32, features=784
output = model(x)               # shape: (32, 10)
print(output.shape)             # torch.Size([32, 10])
```

## Key Features

| Feature | Description |
|---------|-------------|
| **Dynamic graphs** | Computation graph built at runtime — full Python control flow |
| **Autograd** | Automatic differentiation through any differentiable operation |
| **GPU acceleration** | Seamless CUDA / ROCm / MPS support via `.to(device)` |
| **TorchScript** | JIT compilation for portable production deployment |
| **ONNX export** | Interoperability with ONNX Runtime, TensorRT, and other runtimes |
| **torch.compile** | Graph-level optimization (PyTorch 2.0+) for 1.5–3× speedups |
| **Distributed** | `torch.distributed`, FSDP, and DDP for multi-GPU / multi-node |

## Dynamic Computation Graphs

PyTorch uses a **define-by-run** paradigm: the computation graph is constructed fresh on each forward pass. This means:

- **Debugging is trivial** — use `print()`, `pdb`, or your IDE debugger anywhere
- **Variable-length inputs** work natively (RNNs, graph neural networks)
- **Conditional computation** uses regular Python `if` / `for` / `while`

```python
import torch

def dynamic_network(x, use_dropout=True):
    # Graph structure changes based on runtime flag
    h = torch.relu(x @ torch.randn(4, 4))
    if use_dropout and h.shape[0] > 1:   # native Python conditional
        mask = (torch.rand_like(h) > 0.5).float()
        h = h * mask * 2.0
    return h.sum()

x = torch.randn(3, 4, requires_grad=True)
loss = dynamic_network(x, use_dropout=True)
loss.backward()
print(x.grad.shape)   # torch.Size([3, 4])
```

:::tip[PyTorch 2.0 torch.compile]
Wrap any model with `model = torch.compile(model)` to get optimized execution via the Triton compiler backend. You keep eager-mode debugging during development and opt into compiled performance for training runs — no code changes required.
:::

## PyTorch vs TensorFlow vs JAX

| | **PyTorch** | **TensorFlow 2** | **JAX** |
|---|---|---|---|
| **Execution** | Eager (dynamic) | Eager + `tf.function` | Eager + `jit` |
| **Primary use** | Research & production | Production & mobile | Research & HPC |
| **API style** | Pythonic, OOP | Keras + low-level | Functional, NumPy-like |
| **Debugging** | Native Python tools | Improved since TF1 | XLA traces opaque |
| **Deployment** | TorchScript, ONNX, ExecuTorch | SavedModel, TFLite, TF.js | JAX export, ONNX |
| **Research share (2024)** | ~75% of papers | ~15% | ~10% |
| **Mobile / Edge** | PyTorch Mobile, ExecuTorch | TFLite, TF.js | Limited |
| **Distributed** | `torch.distributed`, FSDP | `tf.distribute` | `pjit`, `xmap` |
| **TPU support** | Via XLA (`torch_xla`) | Native | Native |
| **Custom hardware** | Moderate | Good | Excellent |

:::caution[Choosing the Right Framework]
Framework selection depends on deployment target and team expertise. PyTorch dominates academic research. TensorFlow/Keras has mature mobile tooling. JAX excels at custom hardware, TPUs, and functional transformations (vmap, grad, jit composition).
:::

## CPU vs GPU Acceleration

PyTorch runs on multiple hardware backends:

| Backend | Hardware | Activation |
|---------|----------|------------|
| **CPU** | Any processor | Default |
| **CUDA** | NVIDIA GPUs | `torch.cuda.is_available()` |
| **MPS** | Apple Silicon (M1/M2/M3) | `torch.backends.mps.is_available()` |
| **ROCm** | AMD GPUs | Custom build |
| **XPU** | Intel GPUs (PyTorch 2.4+) | `torch.xpu.is_available()` |

```python
import torch

# Hardware detection
print(torch.cuda.is_available())         # True on NVIDIA systems
print(torch.backends.mps.is_available()) # True on Apple Silicon
print(torch.cuda.device_count())         # Number of CUDA GPUs
if torch.cuda.is_available():
    print(torch.cuda.get_device_name(0)) # e.g., 'NVIDIA RTX 4090'

# Best-practice device selection (works everywhere)
device = (
    'cuda'  if torch.cuda.is_available()  else
    'mps'   if torch.backends.mps.is_available() else
    'cpu'
)
print(f'Using: {device}')   # Using: cuda

# Move tensors and models to the selected device
x     = torch.randn(1000, 1000).to(device)
model = SimpleNet().to(device)
```

### GPU Speedup Benchmark

```python
import torch, time

size = 4096
a = torch.randn(size, size)
b = torch.randn(size, size)

# CPU
t0 = time.perf_counter()
_ = torch.mm(a, b)
print(f'CPU matmul {size}×{size}: {time.perf_counter() - t0:.3f}s')  # ~0.800s

# GPU (CUDA)
if torch.cuda.is_available():
    ag, bg = a.cuda(), b.cuda()
    torch.cuda.synchronize()  # flush any pending GPU work
    t0 = time.perf_counter()
    _ = torch.mm(ag, bg)
    torch.cuda.synchronize()
    print(f'GPU matmul {size}×{size}: {time.perf_counter() - t0:.4f}s')  # ~0.004s
```

## Quick Start: Linear Regression

A complete end-to-end example illustrating every core PyTorch concept:

```python
import torch
import torch.nn as nn
import torch.optim as optim
import matplotlib.pyplot as plt

# ── 1. Synthetic data: y = 3x - 2 + noise ──────────────────────────────────
torch.manual_seed(42)
X = torch.linspace(-4, 4, 120).unsqueeze(1)   # shape: (120, 1)
y = 3 * X - 2 + torch.randn(120, 1) * 0.6     # shape: (120, 1)

# ── 2. Model ────────────────────────────────────────────────────────────────
model = nn.Linear(1, 1)   # learns weight W and bias b: ŷ = Wx + b

# ── 3. Loss and optimizer ───────────────────────────────────────────────────
criterion = nn.MSELoss()
optimizer = optim.Adam(model.parameters(), lr=0.1)

# ── 4. Training loop ────────────────────────────────────────────────────────
losses = []
for epoch in range(300):
    predictions = model(X)                  # forward pass  shape: (120, 1)
    loss        = criterion(predictions, y) # MSE loss      scalar

    optimizer.zero_grad()   # clear previous gradients
    loss.backward()         # compute gradients via autograd
    optimizer.step()        # update W and b

    losses.append(loss.item())
    if (epoch + 1) % 75 == 0:
        print(f'Epoch {epoch+1:3d} | Loss: {loss.item():.4f}')

# ── 5. Results ──────────────────────────────────────────────────────────────
W = model.weight.item()   # ≈ 3.0
b = model.bias.item()     # ≈ -2.0
print(f'Learned: ŷ = {W:.3f}x + ({b:.3f})')
# Learned: ŷ = 2.998x + (-1.997)
```

Expected output:
```
Epoch  75 | Loss: 0.4127
Epoch 150 | Loss: 0.3622
Epoch 225 | Loss: 0.3601
Epoch 300 | Loss: 0.3601
Learned: ŷ = 2.998x + (-1.997)
```

:::tip[Five Core Concepts in One Example]
This example demonstrates the complete PyTorch workflow: tensor creation → model definition (`nn.Module`) → loss function → optimizer → forward pass → `backward()` → `optimizer.step()`. Every real PyTorch project follows this exact pattern, scaled up.
:::

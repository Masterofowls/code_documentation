Next.js App Router fundamentally changes how data fetching works. Because Server Components are `async` by default, you can `await` data directly in your components — no `useEffect`, no `useState`, no loading state management required (unless you want it).

## Fetching in Server Components

The simplest form of data fetching in Next.js:

```tsx
// app/blog/page.tsx — async Server Component
interface Post {
  id: number;
  title: string;
  excerpt: string;
  slug: string;
}

export default async function BlogPage() {
  // Direct async/await — no useEffect needed
  const posts = await fetch('https://api.example.com/posts').then((res) => {
    if (!res.ok) throw new Error('Failed to fetch posts');
    return res.json() as Promise<Post[]>;
  });

  return (
    <div>
      <h1>Blog</h1>
      {posts.map((post) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.excerpt}</p>
        </article>
      ))}
    </div>
  );
}
```

## Caching and Revalidation

Next.js extends the native `fetch` API with caching options:

```tsx
// Force cache — cache indefinitely (SSG-like behaviour)
// This is the default when using generateStaticParams
const data = await fetch('https://api.example.com/config', {
  cache: 'force-cache',
});

// No cache — always fetch fresh data (SSR-like behaviour)
const data = await fetch('https://api.example.com/live-prices', {
  cache: 'no-store',
});

// Time-based revalidation (ISR) — revalidate every 60 seconds
const posts = await fetch('https://api.example.com/posts', {
  next: { revalidate: 60 },
});

// Tag-based revalidation — revalidate when the tag is invalidated
const post = await fetch(`https://api.example.com/posts/${id}`, {
  next: { tags: [`post-${id}`, 'posts'] },
});
```

### On-Demand Revalidation

Invalidate cached data programmatically via Server Actions or Route Handlers:

```tsx
// app/actions/revalidate.ts
'use server';

import { revalidateTag, revalidatePath } from 'next/cache';

// Revalidate all data tagged with 'posts'
export async function revalidatePosts() {
  revalidateTag('posts');
}

// Revalidate a specific page
export async function revalidateBlogPage(slug: string) {
  revalidatePath(`/blog/${slug}`);
  revalidatePath('/blog'); // Also revalidate the listing page
}
```

:::tip[Choosing the Right Cache Strategy]

- **`force-cache`**: Product listings, marketing content, documentation — infrequently changing
- **`no-store`**: Stock prices, live scores, user-specific data — real-time
- **`revalidate: N`**: Blog posts, news feeds — changes occasionally but not in real-time
- **Tag-based**: CMS-driven content — revalidate on CMS publish events
  :::

## React cache() for Deduplication

The `cache()` function memoises the return value of a function for the duration of a single server request. This prevents the same data from being fetched multiple times across different components in the same render:

```tsx
// lib/data.ts
import { cache } from 'react';
import { db } from './db';

// This function will only execute once per request, even if called
// from multiple server components in the same render tree
export const getUser = cache(async (userId: string) => {
  console.log(`Fetching user ${userId}`); // Only logs once per request
  return db.user.findUnique({ where: { id: userId } });
});

export const getPost = cache(async (postId: string) => {
  return db.post.findUnique({
    where: { id: postId },
    include: { author: true, tags: true },
  });
});
```

```tsx
// app/posts/[id]/page.tsx — calls getPost
// app/posts/[id]/layout.tsx — also calls getPost
// Both use the same cached result — no duplicate DB query
```

## Parallel Data Fetching

Fetch independent data in parallel using `Promise.all` to avoid waterfall requests:

```tsx
// app/dashboard/page.tsx
import { getUserProfile } from '@/lib/user';
import { getRecentOrders } from '@/lib/orders';
import { getNotifications } from '@/lib/notifications';
import { getAnalyticsSummary } from '@/lib/analytics';

export default async function DashboardPage() {
  // ❌ Sequential — slow (sum of all latencies)
  // const profile = await getUserProfile();
  // const orders = await getRecentOrders();
  // const notifications = await getNotifications();

  // ✅ Parallel — fast (max of all latencies)
  const [profile, orders, notifications, analytics] = await Promise.all([
    getUserProfile(),
    getRecentOrders(),
    getNotifications(),
    getAnalyticsSummary(),
  ]);

  return (
    <div>
      <ProfileCard profile={profile} />
      <OrdersList orders={orders} />
      <NotificationBell count={notifications.unread} />
      <AnalyticsWidget data={analytics} />
    </div>
  );
}
```

## Streaming with Suspense

Wrap slow components in `<Suspense>` to stream them separately, showing immediate UI while they load:

```tsx
// app/dashboard/page.tsx
import { Suspense } from 'react';
import { ProfileCard } from '@/components/ProfileCard';
import { OrdersList } from '@/components/OrdersList';
import { AnalyticsSkeleton, OrdersSkeleton } from '@/components/skeletons';
import { Analytics } from '@/components/Analytics'; // Slow component

export default function DashboardPage() {
  return (
    <div className="grid grid-cols-2 gap-6">
      {/* Fast — renders immediately */}
      <ProfileCard />

      {/* Slow — streams in when ready */}
      <Suspense fallback={<AnalyticsSkeleton />}>
        <Analytics /> {/* This component fetches data internally */}
      </Suspense>

      {/* Also slow — streams independently */}
      <Suspense fallback={<OrdersSkeleton />}>
        <OrdersList />
      </Suspense>
    </div>
  );
}
```

```tsx
// components/Analytics.tsx — fetches its own data
export async function Analytics() {
  // This fetch happens independently and doesn't block other components
  const data = await fetch('https://api.example.com/analytics/summary', {
    next: { revalidate: 300 },
  }).then((r) => r.json());

  return (
    <div className="rounded-lg border p-6">
      <h2>Analytics</h2>
      <p>Visitors: {data.visitors}</p>
      <p>Revenue: ${data.revenue}</p>
    </div>
  );
}
```

## unstable_cache

Cache the result of a function (like `cache()`) but **across requests** (unlike `cache()` which is per-request):

```tsx
import { unstable_cache } from 'next/cache';

// Cached across all requests for 1 hour, tagged for on-demand revalidation
export const getCachedProducts = unstable_cache(
  async (category: string) => {
    return db.product.findMany({ where: { category } });
  },
  ['products'], // Cache key prefix
  {
    revalidate: 3600, // 1 hour
    tags: ['products'],
  },
);

// Usage in a Server Component
const products = await getCachedProducts('electronics');
```

## Server Actions

Server Actions let you run server-side code directly from form submissions or event handlers in Client Components.

```tsx
// app/posts/new/actions.ts
'use server';

import { z } from 'zod';
import { revalidatePath } from 'next/cache';
import { redirect } from 'next/navigation';
import { db } from '@/lib/db';
import { auth } from '@/lib/auth';

const CreatePostSchema = z.object({
  title: z.string().min(3).max(100),
  body: z.string().min(10),
  published: z.coerce.boolean(),
});

export async function createPost(prevState: unknown, formData: FormData) {
  const session = await auth();
  if (!session) {
    return { error: 'You must be logged in to create a post.' };
  }

  const parsed = CreatePostSchema.safeParse({
    title: formData.get('title'),
    body: formData.get('body'),
    published: formData.get('published'),
  });

  if (!parsed.success) {
    return {
      error: 'Invalid form data.',
      fieldErrors: parsed.error.flatten().fieldErrors,
    };
  }

  const post = await db.post.create({
    data: {
      ...parsed.data,
      authorId: session.user.id,
    },
  });

  revalidatePath('/blog');
  redirect(`/blog/${post.slug}`);
}
```

```tsx
// app/posts/new/page.tsx — using useFormState and useFormStatus
'use client';

import { useFormState, useFormStatus } from 'react-dom';
import { createPost } from './actions';

function SubmitButton() {
  const { pending } = useFormStatus();
  return (
    <button type="submit" disabled={pending}>
      {pending ? 'Creating...' : 'Create Post'}
    </button>
  );
}

export default function NewPostPage() {
  const [state, formAction] = useFormState(createPost, null);

  return (
    <form action={formAction} className="space-y-4">
      {state?.error && <p className="text-red-500">{state.error}</p>}
      <div>
        <label htmlFor="title">Title</label>
        <input
          id="title"
          name="title"
          type="text"
          className="w-full border rounded p-2"
        />
        {state?.fieldErrors?.title && (
          <p className="text-red-500 text-sm">{state.fieldErrors.title[0]}</p>
        )}
      </div>
      <div>
        <label htmlFor="body">Content</label>
        <textarea
          id="body"
          name="body"
          rows={8}
          className="w-full border rounded p-2"
        />
      </div>
      <label className="flex items-center gap-2">
        <input type="checkbox" name="published" />
        Publish immediately
      </label>
      <SubmitButton />
    </form>
  );
}
```

:::tip[Server Actions Without Forms]
Server Actions can also be called directly from event handlers in Client Components — they're not limited to `<form action>`:

```tsx
'use client';
import { deletePost } from './actions';

export function DeleteButton({ postId }: { postId: string }) {
  return <button onClick={() => deletePost(postId)}>Delete</button>;
}
```

:::

## Client-Side Fetching with SWR

For data that needs to be fetched client-side (user-specific, real-time):

```bash
npm install swr
```

```tsx
'use client';

import useSWR from 'swr';
import useSWRMutation from 'swr/mutation';

const fetcher = (url: string) => fetch(url).then((r) => r.json());

export function UserProfile() {
  const {
    data: user,
    error,
    isLoading,
    mutate,
  } = useSWR('/api/user/me', fetcher, {
    revalidateOnFocus: true,
    revalidateOnReconnect: true,
    refreshInterval: 0,
  });

  const { trigger: updateProfile, isMutating } = useSWRMutation(
    '/api/user/me',
    async (url: string, { arg }: { arg: { name: string } }) => {
      const res = await fetch(url, {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(arg),
      });
      return res.json();
    },
  );

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error loading profile</div>;

  return (
    <div>
      <p>{user.name}</p>
      <button
        onClick={() => updateProfile({ name: 'New Name' })}
        disabled={isMutating}
      >
        {isMutating ? 'Saving...' : 'Update Name'}
      </button>
    </div>
  );
}
```

## ISR (Incremental Static Regeneration)

Generate static pages that automatically regenerate after a specified time:

```tsx
// app/products/[id]/page.tsx
export const revalidate = 3600; // Revalidate this page every hour (route segment config)

export async function generateStaticParams() {
  const products = await getTopProducts(); // Pre-render top 100 products
  return products.map((p) => ({ id: p.id }));
}

export default async function ProductPage({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const product = await getProduct(id);

  return <div>{product.name}</div>;
}
```

## Data Fetching Summary

| Strategy                             | When to Use                 | How                                            |
| ------------------------------------ | --------------------------- | ---------------------------------------------- |
| **Server Component fetch (cached)**  | Static content, ISR         | `fetch(url, { next: { revalidate: N } })`      |
| **Server Component fetch (dynamic)** | User-specific, real-time    | `fetch(url, { cache: 'no-store' })`            |
| **React cache()**                    | Deduplicate in one request  | `const fn = cache(async () => {...})`          |
| **unstable_cache**                   | Cross-request caching       | `const fn = unstable_cache(async () => {...})` |
| **Server Actions**                   | Form submissions, mutations | `'use server'` directive                       |
| **SWR**                              | Client-side, real-time      | `useSWR(key, fetcher)`                         |
| **TanStack Query**                   | Complex client-side state   | `useQuery({ queryKey, queryFn })`              |

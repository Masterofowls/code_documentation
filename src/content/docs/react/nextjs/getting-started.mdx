This guide walks you through creating a new Next.js project from scratch, understanding the generated folder structure, configuring TypeScript path aliases, and setting up your development toolchain.

## Prerequisites

| Tool | Version | Notes |
|---|---|---|
| **Node.js** | 18.17+ | LTS recommended |
| **npm / yarn / pnpm** | Latest | pnpm is fastest for monorepos |
| **VS Code** | Latest | Install the [Next.js extension pack](https://marketplace.visualstudio.com/items?itemName=nextjs.vscode-next-js) |

## create-next-app

The official scaffolding tool generates a fully configured Next.js project with a single command:

```bash
npx create-next-app@latest my-app \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir \
  --import-alias "@/*"
```

Or run it interactively (it will prompt for each option):

```bash
npx create-next-app@latest my-app
```

The prompts:
1. **TypeScript** — Yes (strongly recommended)
2. **ESLint** — Yes
3. **Tailwind CSS** — Yes/No (optional)
4. **`src/` directory** — Yes (cleaner separation of app code from config)
5. **App Router** — Yes (use the modern router)
6. **Customize import alias** — Yes, use `@/*`

```bash
cd my-app
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to see your app.

:::tip[pnpm for Faster Installs]
Use `pnpm create next-app@latest` for significantly faster installs and smaller `node_modules` via hard-linking. Run `npm install -g pnpm` first.
:::

## Folder Structure

```
my-app/
├── src/
│   ├── app/                          # App Router root
│   │   ├── (marketing)/              # Route group (no URL segment)
│   │   │   ├── page.tsx              # / (home)
│   │   │   └── about/page.tsx        # /about
│   │   ├── (dashboard)/
│   │   │   ├── layout.tsx            # Dashboard layout (auth-gated)
│   │   │   ├── dashboard/page.tsx    # /dashboard
│   │   │   └── settings/page.tsx     # /settings
│   │   ├── api/                      # Route Handlers
│   │   │   └── users/route.ts        # /api/users
│   │   ├── layout.tsx                # Root layout (html, body)
│   │   ├── loading.tsx               # Root loading UI
│   │   ├── error.tsx                 # Root error boundary
│   │   ├── not-found.tsx             # 404 page
│   │   └── globals.css               # Global styles
│   ├── components/
│   │   ├── ui/                       # Low-level primitives (Button, Input, Card)
│   │   ├── layout/                   # Header, Footer, Sidebar
│   │   └── features/                 # Feature-specific components
│   ├── lib/
│   │   ├── db.ts                     # Database client (Prisma, Drizzle)
│   │   ├── auth.ts                   # Auth configuration (NextAuth, Clerk)
│   │   └── utils.ts                  # Utility functions
│   ├── hooks/                        # Custom React hooks
│   ├── types/                        # TypeScript type definitions
│   └── middleware.ts                 # Next.js middleware
├── public/                           # Static files served as-is
│   ├── images/
│   └── fonts/
├── .env.local                        # Local secrets (git-ignored)
├── .env.example                      # Template for env vars (committed)
├── next.config.ts                    # Next.js configuration
├── tailwind.config.ts                # Tailwind configuration
├── tsconfig.json                     # TypeScript configuration
├── eslint.config.mjs                 # ESLint flat config
└── package.json
```

## next.config.ts

The main Next.js configuration file. In Next.js 15, it uses `.ts` by default:

```typescript
// next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  // Enable React strict mode (recommended — catches issues early):
  reactStrictMode: true,

  // Configure which domains can serve images via next/image:
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.unsplash.com',
        port: '',
        pathname: '/photo-**',
      },
      {
        protocol: 'https',
        hostname: '**.cloudinary.com',
      },
    ],
    formats: ['image/avif', 'image/webp'],
    deviceSizes: [640, 750, 828, 1080, 1200, 1920],
    imageSizes: [16, 32, 48, 64, 96, 128, 256, 384],
  },

  // Expose public environment variables:
  env: {
    APP_VERSION: process.env.npm_package_version ?? '0.0.0',
  },

  // Experimental features (Next.js 15):
  experimental: {
    ppr: 'incremental',    // Partial Pre-rendering
    typedRoutes: true,     // Type-safe Link hrefs
    serverActions: {
      allowedOrigins: ['localhost:3000', 'myapp.com'],
      bodySizeLimit: '2mb',
    },
  },

  // Custom headers:
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          { key: 'X-Frame-Options', value: 'DENY' },
          { key: 'X-Content-Type-Options', value: 'nosniff' },
          { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
        ],
      },
    ];
  },

  // Custom redirects:
  async redirects() {
    return [
      {
        source: '/old-page',
        destination: '/new-page',
        permanent: true,  // 308 (was 301 in Pages Router)
      },
    ];
  },

  // Output mode for deployment:
  // output: 'standalone', // for Docker
  // output: 'export',     // for static hosting
};

export default nextConfig;
```

## Path Aliases

Path aliases are configured in `tsconfig.json` under `paths`. Next.js and Turbopack automatically pick them up — no extra Babel config needed.

```typescript
// Without alias (brittle relative path):
import { Button } from '../../../components/ui/Button';
import { db } from '../../../../lib/db';

// With @/* alias (always works regardless of nesting depth):
import { Button } from '@/components/ui/Button';
import { db } from '@/lib/db';
```

## ESLint Configuration

Next.js 15 uses the new **flat config** format (`eslint.config.mjs`):

```javascript
// eslint.config.mjs
import { dirname } from 'path';
import { fileURLToPath } from 'url';
import { FlatCompat } from '@eslint/eslintrc';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

const compat = new FlatCompat({ baseDirectory: __dirname });

export default [
  ...compat.extends('next/core-web-vitals', 'next/typescript'),
  {
    rules: {
      'no-unused-vars': 'error',
      'no-console': ['warn', { allow: ['warn', 'error'] }],
      '@typescript-eslint/no-explicit-any': 'error',
      '@typescript-eslint/consistent-type-imports': 'error',
      'react-hooks/exhaustive-deps': 'error',
      'import/order': [
        'warn',
        {
          groups: ['builtin', 'external', 'internal', 'parent', 'sibling', 'index'],
          'newlines-between': 'always',
          alphabetize: { order: 'asc' },
        },
      ],
    },
  },
];
```

Run the linter:

```bash
npm run lint            # check
npm run lint -- --fix   # auto-fix
```

## Prettier

```bash
npm install --save-dev prettier eslint-config-prettier
```

```javascript
// prettier.config.js
/** @type {import("prettier").Config} */
export default {
  semi: true,
  singleQuote: true,
  trailingComma: 'all',
  printWidth: 80,
  tabWidth: 2,
  arrowParens: 'always',
  endOfLine: 'lf',
  plugins: ['prettier-plugin-tailwindcss'], // if using Tailwind
};
```

Add to `package.json`:

```json
{
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "next start",
    "lint": "next lint",
    "format": "prettier --write .",
    "format:check": "prettier --check .",
    "type-check": "tsc --noEmit"
  }
}
```

:::tip[Pre-commit Hook]
Add Husky + lint-staged to run `eslint --fix` and `prettier --write` on staged files before every commit:
```bash
npm install --save-dev husky lint-staged
npx husky init
echo "npx lint-staged" > .husky/pre-commit
```
:::

## Development Server

```bash
# Standard dev server:
npm run dev

# With Turbopack (default in Next.js 15, faster cold start):
npm run dev --turbopack

# Specify a custom port:
npm run dev -- --port 3001

# Expose to local network (for testing on phone):
npm run dev -- --hostname 0.0.0.0
```

## Environment Variables

```bash
# .env.local (never commit — personal secrets)
DATABASE_URL=postgresql://localhost:5432/myapp_dev
NEXTAUTH_SECRET=super-secret-development-key

# .env.development (safe to commit — dev defaults)
NEXT_PUBLIC_API_URL=http://localhost:3000/api

# .env.production (safe to commit — production defaults)
NEXT_PUBLIC_API_URL=https://myapp.com/api
```

```typescript
// Server-only (accessible in Server Components, API routes, Server Actions):
const dbUrl = process.env.DATABASE_URL!;

// Public (accessible everywhere, compiled into client bundle):
const apiUrl = process.env.NEXT_PUBLIC_API_URL!;

// Validate all env vars at startup:
// src/env.ts
import { z } from 'zod';

const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  NEXTAUTH_SECRET: z.string().min(32),
  NEXT_PUBLIC_API_URL: z.string().url(),
});

export const env = envSchema.parse(process.env);
```

:::caution[NEXT_PUBLIC_ Variables Are Public]
Any variable prefixed with `NEXT_PUBLIC_` is inlined into the JavaScript bundle during build and visible to anyone who inspects the bundle. Never put API keys, passwords, or secrets in `NEXT_PUBLIC_` variables.
:::

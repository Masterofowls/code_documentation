This guide walks you through creating a production-ready Next.js project from scratch, covering project structure, TypeScript configuration, path aliases, linting, and formatting.

## Creating a New Project

The official `create-next-app` CLI scaffolds a complete Next.js project with all recommended tooling:

```bash
npx create-next-app@latest my-app
```

The interactive prompt will ask:

```
What is your project named? › my-app
Would you like to use TypeScript? › Yes
Would you like to use ESLint? › Yes
Would you like to use Tailwind CSS? › Yes
Would you like your code inside a `src/` directory? › No
Would you like to use App Router? (recommended) › Yes
Would you like to use Turbopack for next dev? › Yes
Would you like to customize the import alias (@/* by default)? › No
```

:::tip[Always Use TypeScript]
TypeScript is enabled by default in `create-next-app` and is strongly recommended. It provides type safety for Server Components, API routes, `params`, `searchParams`, and all Next.js-specific types.
:::

After creation:

```bash
cd my-app
npm run dev
```

Your app is now running at [http://localhost:3000](http://localhost:3000).

## next.config.ts

The Next.js configuration file controls the framework's behaviour:

```typescript
// next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  // Enable React strict mode for better development warnings
  reactStrictMode: true,

  // Configure allowed image domains for next/image
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.unsplash.com',
      },
      {
        protocol: 'https',
        hostname: '**.cloudinary.com',
        pathname: '/my-cloud/**',
      },
    ],
    // Image formats to generate (AVIF is smaller but slower to generate)
    formats: ['image/avif', 'image/webp'],
  },

  // Environment variables (accessible on the server only)
  // Do NOT use for secrets — use process.env directly
  env: {
    APP_VERSION: process.env.npm_package_version,
  },

  // Experimental features
  experimental: {
    // Partial Prerendering (PPR) — Next.js 14+
    ppr: true,
    // Type-safe Link hrefs
    typedRoutes: true,
  },

  // Custom redirects
  async redirects() {
    return [
      {
        source: '/old-blog/:slug',
        destination: '/blog/:slug',
        permanent: true,
      },
    ];
  },

  // Custom headers
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          { key: 'X-Frame-Options', value: 'DENY' },
          { key: 'X-Content-Type-Options', value: 'nosniff' },
          { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
        ],
      },
    ];
  },
};

export default nextConfig;
```

## Path Aliases

The `@/*` alias (configured by default) allows absolute imports:

```tsx
// Without path aliases (fragile — breaks on folder moves)
import { Button } from '../../../components/ui/Button';
import { formatDate } from '../../lib/utils';

// With path aliases (clean and resilient)
import { Button } from '@/components/ui/Button';
import { formatDate } from '@/lib/utils';
```

## ESLint Configuration

Next.js uses ESLint's flat config format (ESLint 9+):

```javascript
// eslint.config.mjs
import { dirname } from 'path';
import { fileURLToPath } from 'url';
import { FlatCompat } from '@eslint/eslintrc';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

const compat = new FlatCompat({ baseDirectory: __dirname });

const eslintConfig = [
  ...compat.extends('next/core-web-vitals', 'next/typescript'),
  {
    rules: {
      // Enforce consistent imports
      'import/order': ['error', { 'newlines-between': 'always' }],
      // Prevent unused variables
      '@typescript-eslint/no-unused-vars': [
        'error',
        { argsIgnorePattern: '^_' },
      ],
      // Prefer const assertions
      'prefer-const': 'error',
      // No explicit any
      '@typescript-eslint/no-explicit-any': 'error',
      // React hooks rules
      'react-hooks/exhaustive-deps': 'error',
    },
  },
];

export default eslintConfig;
```

Run ESLint:

```bash
# Check
npm run lint

# Auto-fix
npx next lint --fix
```

## Development Server

```bash
# Standard development server
npm run dev

# With Turbopack (faster, experimental)
npm run dev -- --turbopack
# or configure in package.json: "dev": "next dev --turbopack"

# On a specific port
npm run dev -- --port 3001

# With HTTPS (useful for testing service workers)
npm run dev -- --experimental-https
```

:::tip[Turbopack]
Turbopack (`--turbopack`) is Next.js's new Rust-based bundler, replacing webpack for development. It offers significantly faster Hot Module Replacement (HMR) — especially noticeable in large codebases. It became stable for development in Next.js 15.
:::

## Environment Variables

Next.js has a layered environment variable system:

```bash
# .env.local        — local overrides (gitignored, never committed)
# .env.development  — development environment
# .env.production   — production environment
# .env              — defaults for all environments

# NEXT_PUBLIC_ prefix makes variables available in the browser
NEXT_PUBLIC_API_URL=https://api.example.com
NEXT_PUBLIC_APP_NAME=MyApp

# No prefix — server-only (never sent to the browser)
DATABASE_URL=postgresql://...
JWT_SECRET=super-secret-key
STRIPE_SECRET_KEY=sk_live_...
```

```tsx
// Server component — access all env vars
export default async function Page() {
  const data = await fetch(process.env.DATABASE_URL!);
  // ...
}

// Client component — only NEXT_PUBLIC_ vars
('use client');
export function Banner() {
  return <div>Powered by {process.env.NEXT_PUBLIC_APP_NAME}</div>;
}
```

:::caution[Never Expose Secrets to the Browser]
Any environment variable **without** the `NEXT_PUBLIC_` prefix is server-only and will never appear in client-side bundles. Double-check that API keys, database URLs, and secrets do not have the `NEXT_PUBLIC_` prefix.
:::

## Quick Start Verification

After running `npm run dev`, confirm everything is working:

```bash
# Check types
npm run type-check

# Check linting
npm run lint

# Check formatting
npm run format:check

# Run a production build locally
npm run build && npm run start
```

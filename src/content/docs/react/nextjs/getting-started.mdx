This guide covers everything you need to go from zero to a running Next.js application with TypeScript, proper tooling, and a sensible project structure.

## Creating a New Project

```bash
npx create-next-app@latest my-app
```

The interactive CLI will ask:
- TypeScript? → **Yes**
- ESLint? → **Yes**
- Tailwind CSS? → Yes (optional, recommended)
- `src/` directory? → **Yes** (recommended for organization)
- App Router? → **Yes** (recommended for new projects)
- Customize import alias? → **Yes** → `@/*`

Or pass flags directly to skip prompts:

```bash
npx create-next-app@latest my-app \
  --typescript \
  --eslint \
  --tailwind \
  --src-dir \
  --app \
  --import-alias "@/*"
```

## next.config.ts

```typescript
// next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  // Enable React strict mode
  reactStrictMode: true,

  // Image optimization domains
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.unsplash.com',
        pathname: '/**',
      },
      {
        protocol: 'https',
        hostname: '**.cloudinary.com',
      },
    ],
    formats: ['image/avif', 'image/webp'],
    minimumCacheTTL: 60,
  },

  // Expose env vars to the browser (prefix NEXT_PUBLIC_ automatically works)
  env: {
    APP_VERSION: process.env.npm_package_version ?? '0.0.0',
  },

  // HTTP headers
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          { key: 'X-Content-Type-Options', value: 'nosniff' },
          { key: 'X-Frame-Options', value: 'DENY' },
          { key: 'X-XSS-Protection', value: '1; mode=block' },
          { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
        ],
      },
      {
        source: '/api/(.*)',
        headers: [
          { key: 'Cache-Control', value: 'no-store' },
        ],
      },
    ];
  },

  // Redirects
  async redirects() {
    return [
      {
        source: '/old-blog/:slug',
        destination: '/blog/:slug',
        permanent: true,
      },
    ];
  },

  // Rewrites (proxy requests without redirect)
  async rewrites() {
    return [
      {
        source: '/api/legacy/:path*',
        destination: `${process.env.LEGACY_API_URL}/:path*`,
      },
    ];
  },

  // Experimental features
  experimental: {
    // Partial Prerendering (PPR) — Next.js 14+
    ppr: true,
    // React compiler (Next.js 15+)
    reactCompiler: true,
  },

  // Build output mode
  // output: 'standalone', // for Docker deployments
  // output: 'export',     // for static HTML export

  // Transpile specific node_modules
  transpilePackages: ['@company/ui-kit'],

  // Bundle specific packages for server-side code
  serverExternalPackages: ['sharp', 'bcrypt'],
};

export default nextConfig;
```

## Path Aliases

With the `paths` config above, you can use clean imports:

```typescript
// Instead of:
import { Button } from '../../../components/ui/Button';
import { db } from '../../../../lib/db';

// Use:
import { Button } from '@components/ui/Button';
import { db } from '@lib/db';
import { useUser } from '@hooks/useUser';
```

## ESLint Configuration

```javascript
// eslint.config.mjs
import { dirname } from 'path';
import { fileURLToPath } from 'url';
import { FlatCompat } from '@eslint/eslintrc';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

const compat = new FlatCompat({ baseDirectory: __dirname });

export default [
  ...compat.extends(
    'next/core-web-vitals',
    'next/typescript',
    'plugin:@typescript-eslint/recommended-type-checked',
  ),
  {
    rules: {
      // TypeScript
      '@typescript-eslint/no-unused-vars': 'error',
      '@typescript-eslint/no-explicit-any': 'error',
      '@typescript-eslint/consistent-type-imports': ['error', { prefer: 'type-imports' }],

      // React
      'react/self-closing-comp': 'warn',
      'react/jsx-sort-props': ['warn', { callbacksLast: true, shorthandFirst: true }],

      // Imports
      'no-duplicate-imports': 'error',

      // Next.js specific
      '@next/next/no-html-link-for-pages': 'error',
    },
  },
  {
    ignores: ['.next/', 'node_modules/', 'public/'],
  },
];
```

## Environment Variables

```bash
# .env.local — local development only, never committed
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_STRIPE_PUBLIC_KEY=pk_test_...
DATABASE_URL=postgresql://localhost:5432/myapp_dev
JWT_SECRET=your-local-dev-secret-32-chars-min
NEXTAUTH_SECRET=your-nextauth-secret

# .env.example — committed, documents required variables (no real values)
NEXT_PUBLIC_APP_URL=
NEXT_PUBLIC_STRIPE_PUBLIC_KEY=
DATABASE_URL=
JWT_SECRET=
NEXTAUTH_SECRET=
```

```typescript
// src/lib/env.ts — validate env vars at startup (recommended)
import { z } from 'zod';

const envSchema = z.object({
  NODE_ENV: z.enum(['development', 'test', 'production']),
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  NEXT_PUBLIC_APP_URL: z.string().url(),
  NEXT_PUBLIC_STRIPE_PUBLIC_KEY: z.string().startsWith('pk_'),
});

// Throws at startup if any required env var is missing
export const env = envSchema.parse(process.env);
```

## Development Server

```bash
# Start development server (http://localhost:3000)
npm run dev

# With Turbopack (significantly faster, experimental)
npm run dev -- --turbo

# Custom port
npm run dev -- -p 3001

# Custom hostname (for network access)
npm run dev -- -H 0.0.0.0
```

Key development URLs:
- `http://localhost:3000` — Your app
- `http://localhost:3000/_next/static/` — Static assets (auto-served)

:::tip[Turbopack]
Turbopack (Next.js 14+ `--turbo` flag) can be 10x faster than webpack for large projects. It is stable for development in Next.js 15. Production builds still use webpack (Turbopack production support is in progress).
:::

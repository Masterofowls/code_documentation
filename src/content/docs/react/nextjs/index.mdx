Next.js is a React framework built by Vercel that provides the structure, features, and optimisations needed to build production-grade web applications. It handles routing, rendering, data fetching, image optimisation, and more — out of the box.

## What Is Next.js?

Next.js extends React with capabilities that are difficult to implement manually:

- **File-system routing** — Create routes by adding files to the `app/` directory
- **Server-side rendering (SSR)** — Render pages on the server for SEO and fast initial loads
- **Static site generation (SSG)** — Pre-render pages at build time for maximum performance
- **API routes** — Build backend API endpoints in the same repository
- **Image optimisation** — Automatic WebP/AVIF conversion, lazy loading, and responsive sizing
- **Font optimisation** — Zero-layout-shift font loading
- **Full-stack** — Write database queries and server logic alongside your React components

```tsx
// app/page.tsx — a simple Next.js page
export default async function HomePage() {
  // Fetch data directly in the component — no useEffect, no loading state
  const posts = await fetch('https://api.example.com/posts').then((r) =>
    r.json(),
  );

  return (
    <main>
      <h1>Latest Posts</h1>
      <ul>
        {posts.map((post: { id: number; title: string }) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </main>
  );
}
```

## App Router vs Pages Router

Next.js has two routing paradigms. The **App Router** (introduced in Next.js 13, stable since Next.js 13.4) is the current standard:

| Feature                     | App Router (`app/`)         | Pages Router (`pages/`)                |
| --------------------------- | --------------------------- | -------------------------------------- |
| **React Server Components** | ✅ Default                  | ❌ Client components only              |
| **Streaming**               | ✅ Built-in                 | ❌ Not supported                       |
| **Nested layouts**          | ✅ File-based               | ❌ Manual composition                  |
| **Server Actions**          | ✅ Supported                | ❌ Not supported                       |
| **Metadata API**            | ✅ `generateMetadata`       | ❌ `next/head`                         |
| **Data fetching**           | `async` components, `fetch` | `getServerSideProps`, `getStaticProps` |
| **Status**                  | ✅ Current standard         | ⚠️ Maintained, not extended            |

:::tip[Which Should You Use?]
For new projects, always use the **App Router**. It offers better performance through React Server Components and streaming, and it's where all new Next.js features are being built. The Pages Router remains supported and stable for existing projects.
:::

## Rendering Strategies

Next.js supports all React rendering strategies, and you can mix them within a single application:

```
Decision: How should this page be rendered?
│
├── Is the content user-specific or real-time?
│   └── YES → Server-Side Rendering (SSR) — fetch('...', { cache: 'no-store' })
│
├── Does the content change occasionally? (e.g., blog, docs)
│   └── YES → Incremental Static Regeneration (ISR) — fetch('...', { next: { revalidate: 60 } })
│
├── Does the content never change? (e.g., marketing, legal)
│   └── YES → Static Site Generation (SSG) — fetch('...', { cache: 'force-cache' })
│
└── Does the content need browser APIs or user interaction?
    └── YES → Client-Side Rendering (CSR) — 'use client' + useEffect
```

## Next.js vs Remix vs Astro

| Feature             | Next.js 15                    | Remix                       | Astro                         |
| ------------------- | ----------------------------- | --------------------------- | ----------------------------- |
| **Primary Use**     | Full-stack React apps         | Full-stack React apps       | Content-first sites           |
| **Rendering**       | SSR, SSG, ISR, CSR            | SSR (first-class), CSR      | SSG (default), SSR            |
| **Routing**         | File-based (app/)             | File-based (routes/)        | File-based (pages/)           |
| **Data Fetching**   | RSC `async`, Server Actions   | `loader` / `action`         | Frontmatter + API             |
| **React Version**   | React 18+, RSC                | React 18+, RSC              | Partial (islands)             |
| **Bundle Size**     | Moderate                      | Small                       | Very small (no JS by default) |
| **Forms**           | Server Actions                | `<Form>` + `action`         | Partial hydration             |
| **Database Access** | In Server Components          | In loaders                  | In `.astro` files             |
| **Image Opt.**      | ✅ `next/image`               | Manual                      | ✅ `@astrojs/image`           |
| **Edge Runtime**    | ✅ Vercel Edge                | ✅ Cloudflare Workers       | ✅ Multiple                   |
| **Deployment**      | Vercel (optimal), self-hosted | Vercel, Cloudflare, Netlify | Any static host               |
| **Maturity**        | ⭐⭐⭐⭐⭐                    | ⭐⭐⭐⭐                    | ⭐⭐⭐⭐                      |

## Full-Stack Capabilities

Next.js blurs the line between frontend and backend:

```tsx
// app/dashboard/page.tsx — directly query your database
import { db } from '@/lib/db';
import { auth } from '@/lib/auth';
import { redirect } from 'next/navigation';

export default async function DashboardPage() {
  const session = await auth();

  if (!session) {
    redirect('/login');
  }

  // Direct database access — no API layer required
  const stats = await db.query.userStats.findFirst({
    where: (table, { eq }) => eq(table.userId, session.user.id),
  });

  return (
    <div>
      <h1>Welcome, {session.user.name}</h1>
      <p>Total orders: {stats?.orderCount ?? 0}</p>
    </div>
  );
}
```

## Vercel Hosting

Next.js is developed by Vercel and has first-class integration:

- **Zero-config deployments** — Connect your GitHub repo and every push deploys automatically
- **Preview deployments** — Every PR gets its own preview URL
- **Edge Network** — Global CDN with automatic asset caching
- **Analytics** — Core Web Vitals and audience analytics
- **Edge Functions** — Run middleware and API routes at the network edge
- **ISR** — Incremental Static Regeneration with instant purging via the API

```bash
# Deploy to Vercel
npm install -g vercel
vercel
```

Next.js is a React framework built by Vercel that provides everything you need to build production-grade web applications: server-side rendering, static generation, file-based routing, API routes, image optimization, and more — with zero configuration.

## What is Next.js?

Where plain React gives you a UI library, Next.js gives you a complete application framework. It handles:

- **Routing** — File-based, no router configuration needed
- **Rendering** — SSR, SSG, ISR, and CSR in a single project
- **Data Fetching** — Server Components fetch data directly, no API roundtrip
- **Full-Stack** — API routes and Server Actions live alongside your UI
- **Optimization** — Images, fonts, scripts, and bundles are optimized automatically
- **TypeScript** — First-class support with zero configuration

```tsx
// This is a complete Next.js page — a Server Component that fetches data directly
// app/posts/page.tsx
async function getPosts() {
  const res = await fetch('https://api.example.com/posts', {
    next: { revalidate: 3600 }, // ISR: revalidate every hour
  });
  return res.json();
}

export default async function PostsPage() {
  const posts = await getPosts();

  return (
    <main>
      <h1>Latest Posts</h1>
      {posts.map((post: { id: string; title: string }) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
        </article>
      ))}
    </main>
  );
}
```

## App Router vs Pages Router

Next.js 13 introduced the **App Router** — a new routing and rendering paradigm built on React Server Components. The older **Pages Router** still works and is fully supported.

| Feature | App Router | Pages Router |
|---|---|---|
| **React Server Components** | Yes (default) | No |
| **Nested layouts** | Yes (layout.tsx) | Limited (via \_app.tsx) |
| **Streaming** | Yes (Suspense) | No |
| **Server Actions** | Yes | No |
| **Data fetching** | In component (`async/await`) | `getServerSideProps`, `getStaticProps` |
| **Loading UI** | `loading.tsx` | Manual |
| **Error boundaries** | `error.tsx` | Manual |
| **Metadata API** | `generateMetadata()` | `next/head` |
| **Stability** | Stable (Next.js 13.4+) | Stable |
| **Learning curve** | Higher | Lower |

:::tip[Which to Use?]
For all new projects, use the **App Router**. It has a steeper learning curve but is more powerful — async Server Components eliminate entire categories of loading state complexity. The Pages Router is excellent if you're maintaining an existing project or your team needs a gentler start.
:::

## Rendering Strategies

Next.js supports multiple rendering strategies, often mixed in a single application:

### SSR — Server-Side Rendering

HTML is generated on the server for every request. Always fresh data.

```tsx
// Any async Server Component with no-store cache = SSR
export default async function LiveDashboard() {
  const data = await fetch('/api/metrics', { cache: 'no-store' });
  const metrics = await data.json();
  return <Dashboard metrics={metrics} />;
}
```

**Use for:** Personalized pages, real-time data, pages that must always show current data.

### SSG — Static Site Generation

HTML is generated at build time and served from a CDN.

```tsx
// force-cache (default) = SSG at build time
export default async function BlogPost({ params }: { params: { slug: string } }) {
  const post = await fetch(`/api/posts/${params.slug}`).then(r => r.json());
  return <Article post={post} />;
}

// Tell Next.js which slugs to pre-render at build time
export async function generateStaticParams() {
  const posts = await fetch('/api/posts').then(r => r.json());
  return posts.map((p: { slug: string }) => ({ slug: p.slug }));
}
```

**Use for:** Blog posts, documentation, marketing pages, product listings.

### ISR — Incremental Static Regeneration

Pages are statically generated but can be revalidated on a schedule or on-demand.

```tsx
// Revalidate every 60 seconds
export default async function ProductPage({ params }: { params: { id: string } }) {
  const product = await fetch(`/api/products/${params.id}`, {
    next: { revalidate: 60 },
  }).then(r => r.json());
  return <Product product={product} />;
}
```

**Use for:** E-commerce products, news articles, pages that change occasionally.

### CSR — Client-Side Rendering

Data is fetched in the browser after the initial page load. Use for interactive dashboards or user-specific content.

```tsx
'use client';
import useSWR from 'swr';

export function UserProfile() {
  const { data, error, isLoading } = useSWR('/api/me', fetch);
  if (isLoading) return <Spinner />;
  if (error) return <Error />;
  return <Profile user={data} />;
}
```

**Use for:** Authenticated user dashboards, real-time interactive data, widgets.

### Decision Tree

```
Is the content personalized to the user?
├── Yes → SSR (no-store) or CSR
└── No → Does it change frequently?
    ├── More than once per hour → SSR or ISR (short revalidate)
    ├── Hourly / Daily → ISR (revalidate: 3600 or 86400)
    └── Rarely / Never → SSG (generateStaticParams)
```

## Next.js vs Remix vs Astro

| Feature | Next.js 15 | Remix v2 | Astro v4 |
|---|---|---|---|
| **Focus** | Full-stack React | Full-stack React | Content-first (any framework) |
| **Rendering** | SSR/SSG/ISR/RSC | SSR-first | SSG-first, optional SSR |
| **Data fetching** | Server Components, Server Actions | Loaders/Actions | Frontmatter, Content Collections |
| **Routing** | File-based (App Router) | File-based | File-based |
| **Bundle size** | Moderate | Moderate | Minimal (Islands) |
| **Full-stack APIs** | Route Handlers, Server Actions | Remix Loaders/Actions | Endpoints |
| **Image optimization** | Built-in (next/image) | Third-party | Built-in (@astrojs/image) |
| **Edge support** | Yes (Edge Runtime) | Yes | Yes |
| **Best for** | Web apps, e-commerce, SaaS | Data-heavy apps, progressive enhancement | Docs, blogs, marketing sites |
| **Hosting** | Vercel (optimal), any Node.js | Fly.io, Cloudflare, Node.js | Vercel, Netlify, any static host |

## Full-Stack Capabilities

Next.js is a genuine full-stack framework. You can:

```typescript
// app/api/users/route.ts — a REST API endpoint
import { NextRequest, NextResponse } from 'next/server';
import { db } from '@/lib/db';

export async function GET(request: NextRequest) {
  const users = await db.user.findMany();
  return NextResponse.json(users);
}

export async function POST(request: NextRequest) {
  const body = await request.json();
  const user = await db.user.create({ data: body });
  return NextResponse.json(user, { status: 201 });
}
```

```typescript
// app/actions/auth.ts — a Server Action (form submission without an API)
'use server';
import { redirect } from 'next/navigation';
import { signIn } from '@/lib/auth';

export async function loginAction(formData: FormData) {
  const email = formData.get('email') as string;
  const password = formData.get('password') as string;
  await signIn(email, password);
  redirect('/dashboard');
}
```

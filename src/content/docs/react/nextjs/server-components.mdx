React Server Components (RSC) are one of the most significant shifts in React's architecture. Understanding them deeply is essential for building performant Next.js applications.

## What Are React Server Components?

Server Components are React components that render **exclusively on the server**. Their HTML output is streamed to the client — no JavaScript is sent for the component itself. This means:

- **Zero bundle impact** — Server Component code never reaches the browser
- **Direct backend access** — Read databases, filesystems, and call internal services directly
- **No hydration** — No client-side interactivity setup needed
- **Automatic security** — Sensitive logic and API keys stay on the server

In Next.js App Router, **all components are Server Components by default**. You opt into client-side rendering with the `'use client'` directive.

```tsx
// app/page.tsx — Server Component by default
// This entire file runs on the server. The user never downloads this code.
import { db } from '@/lib/db';
import { auth } from '@/lib/auth';

export default async function HomePage() {
  // Direct database access — this is secure because it only runs on the server
  const session = await auth();
  const recentPosts = await db.post.findMany({
    take: 5,
    orderBy: { createdAt: 'desc' },
    include: { author: { select: { name: true, avatar: true } } },
  });

  return (
    <main>
      <h1>Welcome{session ? `, ${session.user.name}` : ''}!</h1>
      <PostList posts={recentPosts} />
    </main>
  );
}
```

## The Server/Client Boundary

The `'use client'` directive marks a **boundary** in your component tree. Everything below that boundary becomes a Client Component — but it doesn't mean the component only runs on the client. Client Components still render on the server for the initial HTML (SSR), then hydrate on the client.

```
Server Components (default):
├── app/layout.tsx          ← Server
│   └── <Header />          ← Server
│       └── <NavLinks />    ← Server (reads from DB)
│           └── <NavItem /> ← Server
└── app/page.tsx            ← Server
    └── <PostList />        ← Server
        └── <PostCard />    ← Server
            └── <LikeButton /> ← 'use client' — boundary here
```

Once you add `'use client'` to a component, **all its children are also Client Components** (unless imported from a separate Server Component file).

## When to Use Client Components

Add `'use client'` only when you genuinely need client-side capabilities:

| Need             | Use Case                                 | Directive                  |
| ---------------- | ---------------------------------------- | -------------------------- |
| Interactivity    | Buttons, forms, dropdowns                | `'use client'`             |
| Browser APIs     | localStorage, geolocation, WebSocket     | `'use client'`             |
| React hooks      | `useState`, `useEffect`, `useReducer`    | `'use client'`             |
| Event listeners  | `onClick`, `onChange`, `onSubmit`        | `'use client'`             |
| Real-time data   | WebSocket subscriptions, polling         | `'use client'`             |
| Third-party libs | Most UI libraries (Radix, Framer Motion) | `'use client'` (wrap them) |

```tsx
// components/LikeButton.tsx — needs useState, so it's a Client Component
'use client';

import { useState } from 'react';

interface LikeButtonProps {
  initialCount: number;
  postId: string;
}

export function LikeButton({ initialCount, postId }: LikeButtonProps) {
  const [count, setCount] = useState(initialCount);
  const [liked, setLiked] = useState(false);

  const handleLike = async () => {
    setLiked((prev) => !prev);
    setCount((prev) => (liked ? prev - 1 : prev + 1));

    // Server Action to persist the like
    await toggleLike(postId);
  };

  return (
    <button
      onClick={handleLike}
      className={`flex items-center gap-1 ${liked ? 'text-red-500' : 'text-gray-500'}`}
    >
      <span>{liked ? '♥' : '♡'}</span>
      <span>{count}</span>
    </button>
  );
}
```

:::tip[Push Client Boundaries Down]
The further down the component tree the `'use client'` boundary is, the more of your app runs as Server Components. Start every component as a Server Component, and only add `'use client'` when needed.
:::

## Composition Patterns

### Passing Server Components as Children to Client Components

This is the key pattern for keeping expensive components on the server while wrapping them in interactive Client Components:

```tsx
// components/Accordion.tsx — Client Component (needs state for open/closed)
'use client';

import { useState } from 'react';

interface AccordionProps {
  title: string;
  children: React.ReactNode; // Can receive Server Components as children!
}

export function Accordion({ title, children }: AccordionProps) {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div className="border rounded-lg">
      <button
        onClick={() => setIsOpen((prev) => !prev)}
        className="w-full p-4 text-left font-semibold flex justify-between"
      >
        {title}
        <span>{isOpen ? '▲' : '▼'}</span>
      </button>
      {isOpen && <div className="p-4">{children}</div>}
    </div>
  );
}
```

```tsx
// app/faq/page.tsx — Server Component that uses the Client Accordion
// The content inside each accordion is a Server Component
import { Accordion } from '@/components/Accordion';
import { getFAQItems } from '@/lib/content';

export default async function FAQPage() {
  const items = await getFAQItems(); // Server-only database call

  return (
    <div className="space-y-2">
      {items.map((item) => (
        <Accordion key={item.id} title={item.question}>
          {/* This content component runs on the server — inside a Client Component */}
          <p className="text-gray-600">{item.answer}</p>
        </Accordion>
      ))}
    </div>
  );
}
```

### Server Components as Props

You can also pass Server Components as other props besides `children`:

```tsx
// components/Modal.tsx — Client Component
'use client';

interface ModalProps {
  isOpen: boolean;
  onClose: () => void;
  title: React.ReactNode; // Can be a Server Component
  content: React.ReactNode; // Can be a Server Component
  footer?: React.ReactNode; // Can be a Server Component
}

export function Modal({ isOpen, onClose, title, content, footer }: ModalProps) {
  if (!isOpen) return null;
  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center">
      <div className="bg-white rounded-xl p-6 max-w-lg w-full">
        <div className="flex justify-between items-center mb-4">
          {title}
          <button onClick={onClose}>✕</button>
        </div>
        <div>{content}</div>
        {footer && <div className="mt-4 pt-4 border-t">{footer}</div>}
      </div>
    </div>
  );
}
```

## Async Server Components

Server Components can be `async` — enabling direct database and API calls:

```tsx
// components/UserDashboard.tsx — async Server Component
import { db } from '@/lib/db';
import { auth } from '@/lib/auth';
import { redirect } from 'next/navigation';

export async function UserDashboard() {
  const session = await auth();

  if (!session) {
    redirect('/login');
  }

  // Multiple parallel queries
  const [orders, wishlist, recommendations] = await Promise.all([
    db.order.findMany({
      where: { userId: session.user.id },
      orderBy: { createdAt: 'desc' },
      take: 5,
    }),
    db.wishlist.findMany({
      where: { userId: session.user.id },
      include: { product: true },
    }),
    getPersonalisedRecommendations(session.user.id),
  ]);

  return (
    <div className="grid grid-cols-1 md:grid-cols-3 gap-6">
      <RecentOrders orders={orders} />
      <Wishlist items={wishlist} />
      <Recommendations items={recommendations} />
    </div>
  );
}
```

## Accessing Headers and Cookies

Use `next/headers` functions inside Server Components:

```tsx
// app/dashboard/page.tsx
import { headers, cookies } from 'next/headers';

export default async function DashboardPage() {
  const headersList = await headers();
  const cookieStore = await cookies();

  // Read headers
  const userAgent = headersList.get('user-agent');
  const ip = headersList.get('x-forwarded-for') ?? '127.0.0.1';

  // Read cookies
  const theme = cookieStore.get('theme')?.value ?? 'light';
  const locale = cookieStore.get('locale')?.value ?? 'en';

  return (
    <div className={theme === 'dark' ? 'dark' : ''}>
      <p>Locale: {locale}</p>
    </div>
  );
}
```

```tsx
// Setting cookies in a Server Action
'use server';

import { cookies } from 'next/headers';

export async function setTheme(theme: 'light' | 'dark') {
  const cookieStore = await cookies();
  cookieStore.set('theme', theme, {
    maxAge: 60 * 60 * 24 * 365, // 1 year
    path: '/',
    httpOnly: false, // Accessible to JavaScript
    sameSite: 'lax',
  });
}
```

## Draft Mode

Draft Mode allows previewing unpublished CMS content:

```tsx
// app/api/draft/route.ts — Enable draft mode
import { draftMode } from 'next/headers';
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest) {
  const { searchParams } = request.nextUrl;
  const secret = searchParams.get('secret');

  if (secret !== process.env.DRAFT_SECRET) {
    return NextResponse.json({ error: 'Invalid token' }, { status: 401 });
  }

  const draft = await draftMode();
  draft.enable();

  return NextResponse.redirect(
    new URL(searchParams.get('slug') ?? '/', request.url),
  );
}
```

```tsx
// app/blog/[slug]/page.tsx — Use draft mode
import { draftMode } from 'next/headers';

export default async function PostPage({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  const { isEnabled } = await draftMode();

  // Fetch from draft or published source based on mode
  const post = await getPost(slug, { preview: isEnabled });

  return (
    <article>
      {isEnabled && (
        <div className="bg-yellow-100 p-2 text-sm">
          Draft Preview Mode Active
        </div>
      )}
      <h1>{post.title}</h1>
      <div>{post.content}</div>
    </article>
  );
}
```

## Context in App Router

React Context requires `'use client'` — you cannot use `createContext` in Server Components. Wrap context providers in a Client Component:

```tsx
// components/providers/ThemeProvider.tsx
'use client';

import { createContext, useContext, useState } from 'react';

const ThemeContext = createContext<{
  theme: 'light' | 'dark';
  toggle: () => void;
} | null>(null);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<'light' | 'dark'>('light');
  return (
    <ThemeContext.Provider
      value={{
        theme,
        toggle: () => setTheme((t) => (t === 'light' ? 'dark' : 'light')),
      }}
    >
      {children}
    </ThemeContext.Provider>
  );
}

export const useTheme = () => {
  const context = useContext(ThemeContext);
  if (!context) throw new Error('useTheme must be used within ThemeProvider');
  return context;
};
```

```tsx
// app/layout.tsx — Server Component that uses the Client Provider
import { ThemeProvider } from '@/components/providers/ThemeProvider';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        {/* ThemeProvider is a Client Component, but children can be Server Components */}
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  );
}
```

## Wrapping Third-Party Libraries

Many third-party libraries (Radix UI, Framer Motion, etc.) use client-only APIs. Wrap them in a thin Client Component:

```tsx
// components/ui/Tooltip.tsx — wraps @radix-ui/react-tooltip
'use client';

import * as RadixTooltip from '@radix-ui/react-tooltip';

// Re-export as a named group for convenience
export const Tooltip = RadixTooltip.Root;
export const TooltipTrigger = RadixTooltip.Trigger;
export const TooltipContent = RadixTooltip.Content;
export const TooltipProvider = RadixTooltip.Provider;
```

```tsx
// app/layout.tsx — use the wrapped version in the Server Component layout
import { TooltipProvider } from '@/components/ui/Tooltip';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <TooltipProvider delayDuration={200}>{children}</TooltipProvider>
      </body>
    </html>
  );
}
```

:::caution[Common RSC Mistake]
Don't import a library directly in a Server Component if the library uses browser-only APIs at the module level (e.g., accessing `window` or `document`). This will cause build errors. Always wrap such libraries in a `'use client'` boundary first.
:::

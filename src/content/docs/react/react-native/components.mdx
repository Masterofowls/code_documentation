React Native ships with a set of **core components** that map directly to native UI primitives on each platform. Unlike HTML, these components are not rendered in a web view — they are genuine iOS `UIView` / `UILabel` and Android `View` / `TextView` elements.

## View

`View` is the fundamental building block for layout — equivalent to a `<div>` in web development. Every `View` accepts style, accessibility, and event props.

```tsx
import { View, StyleSheet } from 'react-native';

// Basic container
<View style={styles.card}>
  <View style={styles.row}>
    {/* children */}
  </View>
</View>

const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff',
    borderRadius: 16,
    padding: 16,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.08,
    shadowRadius: 8,
    elevation: 4, // Android shadow
  },
  row: {
    flexDirection: 'row',
    alignItems: 'center',
    gap: 12,
  },
});
```

:::tip[View Flattening]
The New Architecture (Fabric) can "flatten" certain View nodes that only exist for styling, removing unnecessary nodes from the native view hierarchy and improving rendering performance.
:::

## Text

`Text` renders a string of text. All text **must** be wrapped in a `<Text>` component — bare strings inside a `View` will throw a runtime error.

```tsx
import { Text, StyleSheet } from 'react-native';

// numberOfLines: truncate to N lines
// ellipsizeMode: 'tail' | 'head' | 'middle' | 'clip'
<Text
  style={styles.body}
  numberOfLines={2}
  ellipsizeMode="tail"
  selectable
  onPress={() => console.log('pressed')}
>
  This is a long string that will be truncated after two lines with an ellipsis
  at the end. Tap to handle a press event.
</Text>

// Inline styling via nested Text:
<Text style={styles.base}>
  Hello, {' '}
  <Text style={styles.bold}>React Native</Text>
  {' '}world!
</Text>

const styles = StyleSheet.create({
  body: { fontSize: 16, lineHeight: 24, color: '#4a5568' },
  base: { fontSize: 16, color: '#1a202c' },
  bold: { fontWeight: '700', color: '#4f46e5' },
});
```

## TextInput

`TextInput` is the only way to accept user keyboard input. It exposes numerous props for controlling behavior.

```tsx
import React, { useState, useRef } from 'react';
import { TextInput, View, StyleSheet, TextInput as RNTextInput } from 'react-native';

function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const passwordRef = useRef<RNTextInput>(null);

  return (
    <View style={styles.form}>
      <TextInput
        style={styles.input}
        placeholder="Email address"
        placeholderTextColor="#a0aec0"
        value={email}
        onChangeText={setEmail}           // fires on every keystroke
        onBlur={() => console.log('left email')}
        onFocus={() => console.log('entered email')}
        keyboardType="email-address"       // numeric-pad | phone-pad | url | etc.
        returnKeyType="next"               // done | go | search | send
        autoCapitalize="none"
        autoCorrect={false}
        textContentType="emailAddress"     // iOS autofill hint
        autoComplete="email"              // Android autofill hint
        onSubmitEditing={() => passwordRef.current?.focus()}
      />

      <TextInput
        ref={passwordRef}
        style={styles.input}
        placeholder="Password"
        placeholderTextColor="#a0aec0"
        value={password}
        onChangeText={setPassword}
        secureTextEntry                    // toggles masking
        returnKeyType="done"
        textContentType="password"
        autoComplete="current-password"
        onSubmitEditing={() => console.log('submit login')}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  form: { gap: 12, paddingHorizontal: 16 },
  input: {
    height: 52,
    borderWidth: 1.5,
    borderColor: '#e2e8f0',
    borderRadius: 10,
    paddingHorizontal: 14,
    fontSize: 16,
    color: '#1a202c',
    backgroundColor: '#f8fafc',
  },
});
```

### Controlled vs Uncontrolled Inputs

:::caution[Always Use Controlled Inputs]
React Native TextInput works best as a **controlled component** — always provide `value` and `onChangeText` together. Uncontrolled inputs can cause subtle bugs with Fast Refresh and form reset logic.
:::

## Image

`Image` renders a bitmap from a local file, network URL, or base64 data URI.

```tsx
import { Image, StyleSheet } from 'react-native';

// Local asset (bundled at build time)
<Image
  source={require('../assets/logo.png')}
  style={styles.logo}
  resizeMode="contain"   // cover | contain | stretch | repeat | center
/>

// Remote URL (always specify width/height for network images)
<Image
  source={{
    uri: 'https://example.com/avatar.jpg',
    headers: { Authorization: 'Bearer token' },
    cache: 'force-cache', // iOS only
  }}
  style={styles.avatar}
  resizeMode="cover"
  onLoad={() => console.log('image loaded')}
  onError={({ nativeEvent }) => console.error(nativeEvent.error)}
  defaultSource={require('../assets/placeholder.png')}
/>

const styles = StyleSheet.create({
  logo: { width: 120, height: 40, resizeMode: 'contain' },
  avatar: { width: 56, height: 56, borderRadius: 28 },
});
```

:::tip[Use expo-image for Production]
For better performance, caching, blurhash placeholders, and animated GIF support, prefer [`expo-image`](https://docs.expo.dev/versions/latest/sdk/image/) over the built-in `Image` component.
:::

## ScrollView

`ScrollView` renders all children at once and makes them scrollable. Best for **short, finite content** (settings screens, forms). For long lists, use `FlatList`.

```tsx
import { ScrollView, StyleSheet } from 'react-native';

<ScrollView
  style={styles.scroll}
  contentContainerStyle={styles.content}
  showsVerticalScrollIndicator={false}
  bounces                                 // iOS rubber-band effect (default true)
  overScrollMode="never"                 // Android: never | always | auto
  keyboardShouldPersistTaps="handled"    // dismiss keyboard on tap outside input
  onScroll={({ nativeEvent }) => {
    const { contentOffset } = nativeEvent;
    if (contentOffset.y > 100) { /* show back-to-top button */ }
  }}
  scrollEventThrottle={16}               // how often onScroll fires (ms)
>
  {/* children rendered all at once */}
</ScrollView>

const styles = StyleSheet.create({
  scroll: { flex: 1 },
  content: { padding: 16, paddingBottom: 40 },
});
```

## FlatList

`FlatList` is the performant, lazy-rendering list component. It only renders items visible on screen, recycling cells as the user scrolls.

```tsx
import { FlatList, View, Text, StyleSheet, ActivityIndicator } from 'react-native';

interface Product {
  id: string;
  name: string;
  price: number;
}

interface ProductListProps {
  products: Product[];
  onEndReached: () => void;
  isLoadingMore: boolean;
  onRefresh: () => void;
  isRefreshing: boolean;
}

function ProductList({
  products,
  onEndReached,
  isLoadingMore,
  onRefresh,
  isRefreshing,
}: ProductListProps) {
  const renderItem = ({ item }: { item: Product }) => (
    <View style={styles.item}>
      <Text style={styles.name}>{item.name}</Text>
      <Text style={styles.price}>${item.price.toFixed(2)}</Text>
    </View>
  );

  const ListHeader = () => (
    <Text style={styles.header}>Products ({products.length})</Text>
  );

  const ListFooter = () =>
    isLoadingMore ? (
      <ActivityIndicator style={{ margin: 16 }} color="#4f46e5" />
    ) : null;

  const ListEmpty = () => (
    <View style={styles.empty}>
      <Text style={styles.emptyText}>No products found</Text>
    </View>
  );

  return (
    <FlatList<Product>
      data={products}
      renderItem={renderItem}
      keyExtractor={(item) => item.id}     // must return a unique string
      ListHeaderComponent={ListHeader}
      ListFooterComponent={ListFooter}
      ListEmptyComponent={ListEmpty}
      onEndReached={onEndReached}          // fires when ~10% from bottom
      onEndReachedThreshold={0.1}          // 0.0–1.0 (ratio of visible content)
      initialNumToRender={10}              // items rendered on first paint
      maxToRenderPerBatch={10}             // items rendered per JS frame
      windowSize={5}                       // render window = 5 * screen height
      removeClippedSubviews                // unmount offscreen items (Android)
      onRefresh={onRefresh}
      refreshing={isRefreshing}
      showsVerticalScrollIndicator={false}
      contentContainerStyle={styles.listContent}
      ItemSeparatorComponent={() => <View style={styles.separator} />}
    />
  );
}

const styles = StyleSheet.create({
  item: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    paddingVertical: 14,
    paddingHorizontal: 16,
  },
  name: { fontSize: 16, color: '#1a202c', flex: 1 },
  price: { fontSize: 16, fontWeight: '700', color: '#4f46e5' },
  header: { fontSize: 20, fontWeight: '700', padding: 16, color: '#1a202c' },
  empty: { flex: 1, alignItems: 'center', paddingTop: 60 },
  emptyText: { fontSize: 16, color: '#a0aec0' },
  separator: { height: 1, backgroundColor: '#f0f4f8', marginHorizontal: 16 },
  listContent: { flexGrow: 1 },
});
```

## SectionList

`SectionList` is like `FlatList` but renders grouped data with section headers.

```tsx
import { SectionList, Text, View, StyleSheet } from 'react-native';

const sections = [
  { title: 'Fruits', data: ['Apple', 'Banana', 'Cherry'] },
  { title: 'Vegetables', data: ['Carrot', 'Broccoli', 'Spinach'] },
];

<SectionList
  sections={sections}
  keyExtractor={(item, index) => `${item}-${index}`}
  renderItem={({ item }) => (
    <View style={styles.item}>
      <Text>{item}</Text>
    </View>
  )}
  renderSectionHeader={({ section }) => (
    <View style={styles.sectionHeader}>
      <Text style={styles.sectionTitle}>{section.title}</Text>
    </View>
  )}
  stickySectionHeadersEnabled
/>
```

## TouchableOpacity vs Pressable

`Pressable` is the modern, flexible tap component replacing `TouchableOpacity`, `TouchableHighlight`, and `TouchableWithoutFeedback`.

```tsx
import { Pressable, TouchableOpacity, Text, StyleSheet } from 'react-native';

// Legacy — still works but less flexible:
<TouchableOpacity
  activeOpacity={0.7}
  onPress={() => console.log('press')}
  disabled={isLoading}
>
  <Text>Legacy Button</Text>
</TouchableOpacity>

// Modern Pressable — supports style as a function of press state:
<Pressable
  onPress={() => console.log('pressed')}
  onLongPress={() => console.log('long pressed')}
  onPressIn={() => console.log('finger down')}
  onPressOut={() => console.log('finger up')}
  disabled={isLoading}
  delayLongPress={500}        // ms before onLongPress fires
  hitSlop={8}                 // expand tap target (number or {top, left, bottom, right})
  style={({ pressed, hovered }) => [
    styles.button,
    pressed && styles.buttonPressed,
    hovered && styles.buttonHovered, // iPad/pointer devices
  ]}
>
  {({ pressed }) => (
    <Text style={[styles.label, pressed && styles.labelPressed]}>
      {pressed ? 'Releasing...' : 'Press Me'}
    </Text>
  )}
</Pressable>

const styles = StyleSheet.create({
  button: {
    backgroundColor: '#4f46e5',
    borderRadius: 12,
    padding: 16,
    alignItems: 'center',
  },
  buttonPressed: {
    backgroundColor: '#4338ca',
    transform: [{ scale: 0.97 }],
  },
  buttonHovered: { backgroundColor: '#6366f1' },
  label: { color: '#fff', fontWeight: '700', fontSize: 16 },
  labelPressed: { opacity: 0.8 },
});
```

## Modal

`Modal` overlays content on top of the existing screen, blocking interaction behind it.

```tsx
import React, { useState } from 'react';
import { Modal, View, Text, Pressable, StyleSheet } from 'react-native';

function ConfirmModal({
  visible,
  title,
  message,
  onConfirm,
  onCancel,
}: {
  visible: boolean;
  title: string;
  message: string;
  onConfirm: () => void;
  onCancel: () => void;
}) {
  return (
    <Modal
      visible={visible}
      transparent                          // background shows through
      animationType="fade"                 // slide | fade | none
      onRequestClose={onCancel}           // required on Android (back button)
      statusBarTranslucent                 // modal covers status bar on Android
    >
      <View style={styles.backdrop}>
        <View style={styles.dialog}>
          <Text style={styles.title}>{title}</Text>
          <Text style={styles.message}>{message}</Text>
          <View style={styles.actions}>
            <Pressable style={[styles.btn, styles.cancel]} onPress={onCancel}>
              <Text style={styles.cancelText}>Cancel</Text>
            </Pressable>
            <Pressable style={[styles.btn, styles.confirm]} onPress={onConfirm}>
              <Text style={styles.confirmText}>Confirm</Text>
            </Pressable>
          </View>
        </View>
      </View>
    </Modal>
  );
}

const styles = StyleSheet.create({
  backdrop: {
    flex: 1,
    backgroundColor: 'rgba(0,0,0,0.5)',
    justifyContent: 'center',
    alignItems: 'center',
    padding: 24,
  },
  dialog: {
    backgroundColor: '#fff',
    borderRadius: 20,
    padding: 24,
    width: '100%',
    maxWidth: 400,
    gap: 12,
  },
  title: { fontSize: 20, fontWeight: '700', color: '#1a202c' },
  message: { fontSize: 16, color: '#4a5568', lineHeight: 24 },
  actions: { flexDirection: 'row', gap: 12, marginTop: 8 },
  btn: { flex: 1, borderRadius: 10, paddingVertical: 14, alignItems: 'center' },
  cancel: { backgroundColor: '#f0f4f8' },
  confirm: { backgroundColor: '#ef4444' },
  cancelText: { color: '#4a5568', fontWeight: '600' },
  confirmText: { color: '#fff', fontWeight: '700' },
});
```

## ActivityIndicator & Switch

```tsx
import { ActivityIndicator, Switch, View, Text } from 'react-native';
import { useState } from 'react';

// Loading spinner
<ActivityIndicator
  size="large"          // small | large | number (Android)
  color="#4f46e5"
  animating             // toggle visibility without unmounting
/>

// Toggle switch
function SettingToggle() {
  const [enabled, setEnabled] = useState(false);
  return (
    <View style={{ flexDirection: 'row', alignItems: 'center', gap: 12 }}>
      <Text>Enable notifications</Text>
      <Switch
        value={enabled}
        onValueChange={setEnabled}
        trackColor={{ false: '#cbd5e0', true: '#c7d2fe' }}
        thumbColor={enabled ? '#4f46e5' : '#e2e8f0'}
        ios_backgroundColor="#cbd5e0"
      />
    </View>
  );
}
```

## Platform-Specific Components

### SafeAreaView

On modern iOS devices with notches and home indicators, `SafeAreaView` automatically adds padding so content doesn't hide behind system UI.

```tsx
import { SafeAreaView, StyleSheet } from 'react-native';
// OR the community library (more control):
import { SafeAreaView, useSafeAreaInsets } from 'react-native-safe-area-context';

// Component approach:
<SafeAreaView style={{ flex: 1 }}>
  {/* content */}
</SafeAreaView>

// Hook approach for granular control:
function MyComponent() {
  const insets = useSafeAreaInsets();
  return (
    <View style={{ paddingTop: insets.top, paddingBottom: insets.bottom }}>
      {/* content */}
    </View>
  );
}
```

### KeyboardAvoidingView

Adjusts layout when the software keyboard appears so form inputs remain visible.

```tsx
import { KeyboardAvoidingView, Platform, ScrollView } from 'react-native';

<KeyboardAvoidingView
  style={{ flex: 1 }}
  behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
  keyboardVerticalOffset={Platform.OS === 'ios' ? 88 : 0}
>
  <ScrollView keyboardShouldPersistTaps="handled">
    {/* form inputs */}
  </ScrollView>
</KeyboardAvoidingView>
```

### StatusBar

```tsx
import { StatusBar } from 'expo-status-bar'; // Expo (preferred)
```

## RefreshControl

Add pull-to-refresh to any `ScrollView` or `FlatList`:

```tsx
import { ScrollView, RefreshControl } from 'react-native';
import { useState, useCallback } from 'react';

function RefreshableList() {
  const [refreshing, setRefreshing] = useState(false);

  const onRefresh = useCallback(async () => {
    setRefreshing(true);
    try {
      await fetchData(); // your data reload logic
    } finally {
      setRefreshing(false);
    }
  }, []);

  return (
    <ScrollView
      refreshControl={
        <RefreshControl
          refreshing={refreshing}
          onRefresh={onRefresh}
          tintColor="#4f46e5"          // iOS spinner color
          colors={['#4f46e5']}         // Android spinner color(s)
          progressBackgroundColor="#fff"
        />
      }
    >
      {/* list items */}
    </ScrollView>
  );
}
```

:::tip[Pair RefreshControl with React Query]
When using TanStack Query, combine `onRefresh` with `refetch()` from `useQuery` for automatic cache invalidation and re-fetching.
:::

## Component Composition Best Practices

```tsx
// Prefer composition over configuration mega-props:
interface CardProps {
  children: React.ReactNode;
  style?: ViewStyle;
  onPress?: () => void;
}

function Card({ children, style, onPress }: CardProps) {
  const Container = onPress ? Pressable : View;
  return (
    <Container
      style={[styles.card, style]}
      {...(onPress ? { onPress } : {})}
    >
      {children}
    </Container>
  );
}

// Usage:
<Card onPress={() => navigate('Detail')}>
  <Text style={styles.title}>Product Name</Text>
  <Text style={styles.price}>$29.99</Text>
</Card>
```

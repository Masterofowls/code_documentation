React Native provides a set of built-in components that map directly to native iOS and Android UI elements. This page covers every core component with real-world usage patterns.

## View

`View` is the most fundamental layout component — equivalent to a `<div>` in web development. It supports flexbox layout, styling, touch handling, and accessibility.

```tsx
import { View, StyleSheet } from 'react-native';

export default function Card() {
  return (
    <View style={styles.card}>
      <View style={styles.header} />
      <View style={styles.body} />
    </View>
  );
}

const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
    shadowRadius: 4,
    elevation: 3,
  },
  header: { height: 40, backgroundColor: '#f0f0f0', borderRadius: 8 },
  body: { marginTop: 12, height: 80, backgroundColor: '#f8f8f8', borderRadius: 8 },
});
```

:::tip[View vs Fragment]
Use `View` when you need layout, styling, or touch handling. Use React `<>` fragments when you just need to group elements without adding a DOM node.
:::

## Text

`Text` renders text content. All text **must** be inside a `<Text>` component — you cannot render raw strings directly in a `View`.

```tsx
import { Text, StyleSheet } from 'react-native';

export default function Typography() {
  return (
    <>
      {/* numberOfLines truncates text to N lines */}
      <Text numberOfLines={2} ellipsizeMode="tail" style={styles.body}>
        This is a long paragraph that will be truncated after two lines of text
        when it overflows the available space.
      </Text>

      {/* Nested Text inherits styles */}
      <Text style={styles.body}>
        Normal text with <Text style={styles.bold}>bold</Text> and{' '}
        <Text style={styles.link} onPress={() => {}}>
          a tappable link
        </Text>
      </Text>
    </>
  );
}

const styles = StyleSheet.create({
  body: { fontSize: 16, lineHeight: 24, color: '#333' },
  bold: { fontWeight: 'bold' },
  link: { color: '#007AFF', textDecorationLine: 'underline' },
});
```

**Key props:**

| Prop | Type | Description |
|---|---|---|
| `numberOfLines` | `number` | Max lines before truncation |
| `ellipsizeMode` | `'head' \| 'middle' \| 'tail' \| 'clip'` | Where to truncate |
| `selectable` | `boolean` | Allow text selection |
| `onPress` | `function` | Make text tappable |
| `adjustsFontSizeToFit` | `boolean` | Shrink font to fit container |

## TextInput

`TextInput` is the component for user text entry. It maps to `UITextField`/`UITextView` on iOS and `EditText` on Android.

```tsx
import { useState } from 'react';
import { TextInput, View, Text, StyleSheet } from 'react-native';

export default function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [focused, setFocused] = useState<string | null>(null);

  return (
    <View style={styles.form}>
      <TextInput
        style={[styles.input, focused === 'email' && styles.inputFocused]}
        value={email}
        onChangeText={setEmail}
        onFocus={() => setFocused('email')}
        onBlur={() => setFocused(null)}
        placeholder="Email address"
        placeholderTextColor="#999"
        keyboardType="email-address"
        autoCapitalize="none"
        autoCorrect={false}
        returnKeyType="next"
        textContentType="emailAddress"
      />

      <TextInput
        style={[styles.input, focused === 'password' && styles.inputFocused]}
        value={password}
        onChangeText={setPassword}
        onFocus={() => setFocused('password')}
        onBlur={() => setFocused(null)}
        placeholder="Password"
        placeholderTextColor="#999"
        secureTextEntry
        returnKeyType="done"
        textContentType="password"
      />
    </View>
  );
}

const styles = StyleSheet.create({
  form: { gap: 12 },
  input: {
    borderWidth: 1,
    borderColor: '#ddd',
    borderRadius: 8,
    padding: 12,
    fontSize: 16,
  },
  inputFocused: { borderColor: '#007AFF' },
});
```

**Key props:**

| Prop | Description |
|---|---|
| `keyboardType` | `'default' \| 'email-address' \| 'numeric' \| 'phone-pad' \| 'decimal-pad'` |
| `returnKeyType` | `'done' \| 'next' \| 'search' \| 'send' \| 'go'` |
| `secureTextEntry` | Hides input (for passwords) |
| `multiline` | Allows multiple lines |
| `textContentType` | iOS autofill hint (`'emailAddress'`, `'password'`, etc.) |
| `autoCapitalize` | `'none' \| 'sentences' \| 'words' \| 'characters'` |

## Image

`Image` displays a static image from a local asset or a remote URL.

```tsx
import { Image, StyleSheet } from 'react-native';

export default function Avatar() {
  return (
    <>
      {/* Local asset */}
      <Image
        source={require('./assets/avatar.png')}
        style={styles.avatar}
        resizeMode="cover"
      />

      {/* Remote URL */}
      <Image
        source={{ uri: 'https://example.com/photo.jpg' }}
        style={styles.hero}
        resizeMode="contain"
        defaultSource={require('./assets/placeholder.png')}
      />
    </>
  );
}

const styles = StyleSheet.create({
  avatar: { width: 56, height: 56, borderRadius: 28 },
  hero: { width: '100%', height: 200 },
});
```

**`resizeMode` values:** `cover`, `contain`, `stretch`, `repeat`, `center`

:::tip[Use expo-image for production]
The built-in `Image` component lacks caching, blurhash placeholders, and transition animations. Use `expo-image` (`import { Image } from 'expo-image'`) in production apps for better performance.
:::

## ScrollView

`ScrollView` renders all its children immediately (no virtualization). Use it for short content; use `FlatList` for long lists.

```tsx
import { ScrollView, View, Text, StyleSheet, RefreshControl } from 'react-native';
import { useState, useCallback } from 'react';

export default function ProfileScreen() {
  const [refreshing, setRefreshing] = useState(false);

  const onRefresh = useCallback(async () => {
    setRefreshing(true);
    await fetchLatestData();
    setRefreshing(false);
  }, []);

  return (
    <ScrollView
      style={styles.scroll}
      contentContainerStyle={styles.content}
      showsVerticalScrollIndicator={false}
      refreshControl={
        <RefreshControl refreshing={refreshing} onRefresh={onRefresh} />
      }
    >
      <View style={styles.header} />
      <Text style={styles.name}>Jane Doe</Text>
      {/* More content... */}
    </ScrollView>
  );
}

const styles = StyleSheet.create({
  scroll: { flex: 1 },
  content: { padding: 16, paddingBottom: 40 },
  header: { height: 200, backgroundColor: '#f0f0f0', borderRadius: 12 },
  name: { fontSize: 24, fontWeight: 'bold', marginTop: 12 },
});
```

## FlatList

`FlatList` is a performant scrollable list with virtualization — only visible items are rendered in memory.

```tsx
import { FlatList, View, Text, StyleSheet, ActivityIndicator } from 'react-native';

interface Post {
  id: string;
  title: string;
  excerpt: string;
}

interface FeedProps {
  posts: Post[];
  onLoadMore: () => void;
  isLoading: boolean;
}

export default function Feed({ posts, onLoadMore, isLoading }: FeedProps) {
  return (
    <FlatList
      data={posts}
      keyExtractor={(item) => item.id}
      renderItem={({ item }) => (
        <View style={styles.card}>
          <Text style={styles.title}>{item.title}</Text>
          <Text style={styles.excerpt} numberOfLines={3}>
            {item.excerpt}
          </Text>
        </View>
      )}
      ListHeaderComponent={
        <Text style={styles.heading}>Latest Posts</Text>
      }
      ListFooterComponent={
        isLoading ? <ActivityIndicator style={styles.loader} /> : null
      }
      ListEmptyComponent={
        <Text style={styles.empty}>No posts yet. Check back soon!</Text>
      }
      onEndReached={onLoadMore}
      onEndReachedThreshold={0.5}
      initialNumToRender={10}
      maxToRenderPerBatch={10}
      windowSize={5}
      showsVerticalScrollIndicator={false}
      contentContainerStyle={styles.list}
    />
  );
}

const styles = StyleSheet.create({
  list: { padding: 16, paddingBottom: 40 },
  heading: { fontSize: 20, fontWeight: 'bold', marginBottom: 12 },
  card: { backgroundColor: '#fff', borderRadius: 12, padding: 16, marginBottom: 12 },
  title: { fontSize: 16, fontWeight: '600' },
  excerpt: { marginTop: 4, color: '#666', fontSize: 14 },
  loader: { marginVertical: 20 },
  empty: { textAlign: 'center', color: '#999', marginTop: 40 },
});
```

**Performance props:**

| Prop | Description |
|---|---|
| `initialNumToRender` | Items to render on first mount (default: 10) |
| `maxToRenderPerBatch` | Items rendered per scroll event |
| `windowSize` | Virtual render window size (default: 21 = 10 above + 10 below viewport) |
| `removeClippedSubviews` | Detach offscreen views (may cause blank flashes on fast scroll) |
| `getItemLayout` | Provide fixed item dimensions to skip measurement |

:::tip[Use FlashList for better performance]
Shopify's [FlashList](https://shopify.github.io/flash-list/) is a drop-in FlatList replacement with 5–10x better performance. Install: `npx expo install @shopify/flash-list`
:::

## SectionList

`SectionList` renders a grouped list with section headers — like iOS Contacts.

```tsx
import { SectionList, Text, View, StyleSheet } from 'react-native';

const DATA = [
  {
    title: 'Today',
    data: ['Task A', 'Task B', 'Task C'],
  },
  {
    title: 'Yesterday',
    data: ['Task D', 'Task E'],
  },
];

export default function TaskList() {
  return (
    <SectionList
      sections={DATA}
      keyExtractor={(item, index) => item + index}
      renderItem={({ item }) => (
        <View style={styles.item}>
          <Text>{item}</Text>
        </View>
      )}
      renderSectionHeader={({ section }) => (
        <Text style={styles.header}>{section.title}</Text>
      )}
      stickySectionHeadersEnabled
    />
  );
}

const styles = StyleSheet.create({
  header: { fontSize: 13, fontWeight: '600', color: '#666', backgroundColor: '#f5f5f5', padding: 8 },
  item: { padding: 16, backgroundColor: '#fff', borderBottomWidth: StyleSheet.hairlineWidth, borderBottomColor: '#eee' },
});
```

## TouchableOpacity vs Pressable

`Pressable` is the modern replacement for `TouchableOpacity`. It offers more control over press states.

```tsx
import { TouchableOpacity, Pressable, Text, StyleSheet } from 'react-native';

// Legacy — still works fine for simple cases
function OldButton({ label, onPress }: { label: string; onPress: () => void }) {
  return (
    <TouchableOpacity style={styles.button} onPress={onPress} activeOpacity={0.7}>
      <Text style={styles.label}>{label}</Text>
    </TouchableOpacity>
  );
}

// Modern — dynamic styles based on press state
function NewButton({ label, onPress }: { label: string; onPress: () => void }) {
  return (
    <Pressable
      style={({ pressed }) => [
        styles.button,
        pressed && styles.buttonPressed,
      ]}
      onPress={onPress}
      onLongPress={() => console.log('Long pressed!')}
      delayLongPress={500}
      disabled={false}
      hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}
    >
      {({ pressed }) => (
        <Text style={[styles.label, pressed && styles.labelPressed]}>
          {label}
        </Text>
      )}
    </Pressable>
  );
}

const styles = StyleSheet.create({
  button: { backgroundColor: '#007AFF', borderRadius: 8, padding: 14, alignItems: 'center' },
  buttonPressed: { backgroundColor: '#0056CC', transform: [{ scale: 0.98 }] },
  label: { color: '#fff', fontSize: 16, fontWeight: '600' },
  labelPressed: { opacity: 0.8 },
});
```

## Modal

`Modal` overlays content on top of the current screen.

```tsx
import { Modal, View, Text, Pressable, StyleSheet } from 'react-native';

interface ConfirmModalProps {
  visible: boolean;
  title: string;
  message: string;
  onConfirm: () => void;
  onCancel: () => void;
}

export default function ConfirmModal({
  visible, title, message, onConfirm, onCancel,
}: ConfirmModalProps) {
  return (
    <Modal
      visible={visible}
      transparent
      animationType="fade"
      onRequestClose={onCancel}
    >
      <View style={styles.overlay}>
        <View style={styles.dialog}>
          <Text style={styles.title}>{title}</Text>
          <Text style={styles.message}>{message}</Text>
          <View style={styles.actions}>
            <Pressable style={styles.cancelBtn} onPress={onCancel}>
              <Text style={styles.cancelText}>Cancel</Text>
            </Pressable>
            <Pressable style={styles.confirmBtn} onPress={onConfirm}>
              <Text style={styles.confirmText}>Confirm</Text>
            </Pressable>
          </View>
        </View>
      </View>
    </Modal>
  );
}

const styles = StyleSheet.create({
  overlay: { flex: 1, backgroundColor: 'rgba(0,0,0,0.5)', justifyContent: 'center', alignItems: 'center' },
  dialog: { backgroundColor: '#fff', borderRadius: 14, padding: 20, width: '80%' },
  title: { fontSize: 17, fontWeight: '600', textAlign: 'center' },
  message: { fontSize: 14, color: '#666', textAlign: 'center', marginTop: 8 },
  actions: { flexDirection: 'row', marginTop: 20, gap: 8 },
  cancelBtn: { flex: 1, padding: 12, borderRadius: 8, borderWidth: 1, borderColor: '#ddd', alignItems: 'center' },
  confirmBtn: { flex: 1, padding: 12, borderRadius: 8, backgroundColor: '#007AFF', alignItems: 'center' },
  cancelText: { color: '#333', fontWeight: '500' },
  confirmText: { color: '#fff', fontWeight: '600' },
});
```

## Switch

import { Switch, View, Text, StyleSheet } from 'react-native';
import { useState } from 'react';

export default function SettingsRow() {

## Platform-Specific Components

### SafeAreaView

Prevents content from being obscured by the notch, status bar, or home indicator.

```tsx
import { SafeAreaView, StyleSheet } from 'react-native';
// Or from react-native-safe-area-context (recommended):
import { SafeAreaView } from 'react-native-safe-area-context';

export default function Screen({ children }: { children: React.ReactNode }) {
  return (
    <SafeAreaView style={styles.container} edges={['top', 'bottom']}>
      {children}
    </SafeAreaView>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#fff' },
});
```

### KeyboardAvoidingView

Automatically adjusts layout when the keyboard appears.

```tsx
import { KeyboardAvoidingView, Platform, StyleSheet } from 'react-native';

export default function FormScreen({ children }: { children: React.ReactNode }) {
  return (
    <KeyboardAvoidingView
      style={styles.container}
      behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
      keyboardVerticalOffset={Platform.OS === 'ios' ? 64 : 0}
    >
      {children}
    </KeyboardAvoidingView>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1 },
});
```

:::caution[KeyboardAvoidingView quirks]
`KeyboardAvoidingView` with `behavior="padding"` works best on iOS. On Android, set `android:windowSoftInputMode="adjustResize"` in `AndroidManifest.xml` and use `behavior="height"` or skip the component entirely.
:::

Deploying a React Native app involves building native binaries for iOS and Android, signing them with the correct certificates, and submitting them to the respective app stores. **EAS (Expo Application Services) Build** streamlines this process significantly.

## EAS Build Overview

EAS Build is a cloud build service that compiles your app on Expo's infrastructure. You don't need a Mac for iOS builds — EAS handles everything in the cloud.

<Steps>
1. **Install EAS CLI**

```bash
npm install -g eas-cli
```

2. **Log in to Expo**

   ```bash
   eas login
   ```

3. **Configure EAS in your project**

   ```bash
   eas build:configure
   ```

   This creates `eas.json` in your project root.

4. **Build for both platforms**

   ```bash
   # Development build (installs as a standalone app, includes dev menu)
   eas build --profile development --platform all

   # Production build
   eas build --profile production --platform all
   ```

   </Steps>

## eas.json Configuration

The `eas.json` file defines your build profiles:

```json
{
  "cli": {
    "version": ">= 10.0.0",
    "appVersionSource": "remote"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal",
      "ios": {
        "resourceClass": "m-medium"
      },
      "android": {
        "buildType": "apk",
        "gradleCommand": ":app:assembleDebug"
      },
      "env": {
        "APP_ENV": "development",
        "API_URL": "https://dev-api.example.com"
      }
    },
    "preview": {
      "distribution": "internal",
      "channel": "preview",
      "ios": {
        "simulator": false
      },
      "android": {
        "buildType": "apk"
      },
      "env": {
        "APP_ENV": "staging",
        "API_URL": "https://staging-api.example.com"
      }
    },
    "production": {
      "autoIncrement": true,
      "channel": "production",
      "ios": {
        "resourceClass": "m-medium"
      },
      "android": {
        "buildType": "app-bundle"
      },
      "env": {
        "APP_ENV": "production",
        "API_URL": "https://api.example.com"
      }
    }
  },
  "submit": {
    "production": {
      "ios": {
        "ascAppId": "YOUR_APP_STORE_CONNECT_APP_ID",
        "appleId": "your@apple.com",
        "ascApiKeyIssuerId": "YOUR_ISSUER_ID",
        "ascApiKeyId": "YOUR_KEY_ID"
      },
      "android": {
        "serviceAccountKeyPath": "./google-service-account.json",
        "track": "internal"
      }
    }
  }
}
```

:::tip[Build Profiles Explained]

- **development**: Creates a debug build with the Expo Dev Client for testing with hot reload
- **preview**: Creates a release build distributed internally (TestFlight / internal track) for QA
- **production**: Creates a production-signed build ready for app store submission
  :::

## app.json / app.config.js

Configure your app's metadata, permissions, and native settings:

```javascript
// app.config.js — dynamic configuration (preferred over app.json)
export default ({ config }) => ({
  ...config,
  name: 'My Awesome App',
  slug: 'my-awesome-app',
  version: '1.2.0',
  orientation: 'portrait',
  icon: './assets/icon.png',
  userInterfaceStyle: 'automatic',
  splash: {
    image: './assets/splash.png',
    resizeMode: 'contain',
    backgroundColor: '#6366f1',
  },
  updates: {
    fallbackToCacheTimeout: 0,
    url: 'https://u.expo.dev/YOUR_PROJECT_ID',
  },
  assetBundlePatterns: ['**/*'],
  ios: {
    supportsTablet: true,
    bundleIdentifier: 'com.example.myapp',
    buildNumber: '1',
    infoPlist: {
      NSCameraUsageDescription: 'This app uses the camera to scan QR codes.',
      NSLocationWhenInUseUsageDescription:
        'This app uses your location to show nearby results.',
      NSPhotoLibraryUsageDescription:
        'This app accesses your photos to let you share images.',
    },
    config: {
      usesNonExemptEncryption: false,
    },
  },
  android: {
    adaptiveIcon: {
      foregroundImage: './assets/adaptive-icon.png',
      backgroundColor: '#6366f1',
    },
    package: 'com.example.myapp',
    versionCode: 10,
    permissions: [
      'android.permission.CAMERA',
      'android.permission.ACCESS_FINE_LOCATION',
      'android.permission.READ_MEDIA_IMAGES',
    ],
    googleServicesFile: './google-services.json',
  },
  plugins: [
    'expo-router',
    'expo-secure-store',
    [
      'expo-notifications',
      {
        icon: './assets/notification-icon.png',
        color: '#6366f1',
      },
    ],
    [
      'expo-camera',
      {
        cameraPermission: 'Allow $(PRODUCT_NAME) to access your camera.',
      },
    ],
  ],
  extra: {
    eas: {
      projectId: 'YOUR_EAS_PROJECT_ID',
    },
  },
});
```

## iOS Provisioning and Certificates

EAS can manage all iOS credentials automatically.

<Tabs>
  <TabItem label="Automatic (Recommended)">
    EAS creates and manages your certificates and provisioning profiles:

    ```bash
    # EAS handles credentials automatically
    eas build --profile production --platform ios
    # When prompted: "How would you like to manage your certificates?"
    # Select: "Expo Go (Managed by Expo)"
    ```

</TabItem>
  <TabItem label="Manual">
    If you need manual control:

    <Steps>
    1. Go to [developer.apple.com](https://developer.apple.com) → Certificates, Identifiers & Profiles
    2. Create an **App ID** with your bundle identifier
    3. Create a **Distribution Certificate** (iOS Distribution)
    4. Create a **Provisioning Profile** (App Store distribution)
    5. Download and configure in EAS:
       ```bash
       eas credentials
       ```

</Steps>
</TabItem>
</Tabs>

:::caution[Apple Developer Account Required]
iOS distribution requires an Apple Developer Program membership ($99/year). You also need to create your app in App Store Connect before submitting.
:::

## Android Keystore

Android apps must be signed with a keystore. EAS manages this automatically:

```bash
# EAS creates and stores your keystore securely
eas build --profile production --platform android
# When prompted: "How would you like to manage your keystore?"
# Select: "Generate new keystore (managed by Expo)"
```

:::caution[Never Lose Your Keystore]
Once you publish to the Play Store, you must always sign updates with the **same keystore**. If you lose it, you cannot publish updates to your existing app listing. EAS stores your keystore securely in their infrastructure — download a backup immediately after your first build.
:::

To download your credentials backup:

```bash
eas credentials --platform android
# Navigate to: Keystore → Download
```

## OTA Updates with EAS Update

Over-the-Air (OTA) updates let you push JavaScript and asset changes without going through the app stores — perfect for bug fixes and small feature updates.

```bash
npx expo install expo-updates
```

```bash
# Configure updates
eas update:configure

# Publish an update to the production channel
eas update --branch production --message "Fix login crash"

# Publish to staging
eas update --branch preview --message "New profile screen"
```

```javascript
// app.config.js — updates configuration
export default {
  updates: {
    url: 'https://u.expo.dev/YOUR_PROJECT_ID',
    enabled: true,
    fallbackToCacheTimeout: 0,
    checkAutomatically: 'ON_LOAD',
    requestHeaders: {
      'expo-channel-name':
        process.env.APP_ENV === 'production' ? 'production' : 'preview',
    },
  },
};
```

```tsx
// Programmatically check for updates
import * as Updates from 'expo-updates';

export async function checkForUpdate() {
  try {
    const update = await Updates.checkForUpdateAsync();
    if (update.isAvailable) {
      await Updates.fetchUpdateAsync();
      await Updates.reloadAsync();
    }
  } catch (error) {
    console.error('Update check failed:', error);
  }
}
```

:::tip[OTA Update Limitations]
OTA updates can only change your JavaScript bundle and assets. They **cannot** change native code, add new native modules, or modify `app.json` configuration. For changes requiring native rebuilds, you still need a full app store submission.
:::

## Environment Variables with EAS

```bash
# Set secrets in EAS (not committed to source control)
eas secret:create --scope project --name API_KEY --value "your-secret-api-key"
eas secret:create --scope project --name SENTRY_DSN --value "https://..."

# List all secrets
eas secret:list
```

Access environment variables in your app:

```tsx
// app.config.js — expose env vars to the app
export default {
  extra: {
    apiUrl: process.env.API_URL ?? 'https://api.example.com',
    sentryDsn: process.env.SENTRY_DSN,
    appEnv: process.env.APP_ENV ?? 'development',
  },
};

// In your app code
import Constants from 'expo-constants';
const API_URL = Constants.expoConfig?.extra?.apiUrl as string;
```

## App Store Submission Checklist

Before submitting to the Apple App Store:

- [ ] App version and build number incremented in `app.config.js`
- [ ] App icons provided at all required sizes (EAS handles this from your 1024×1024 source)
- [ ] Splash screen configured with correct background colour
- [ ] Privacy usage descriptions added to `ios.infoPlist` for all requested permissions
- [ ] App tested on latest iOS version and at least one older version (iOS 16+)
- [ ] App tested on iPhone and iPad if `supportsTablet: true`
- [ ] App Store Connect listing created (name, description, keywords, screenshots)
- [ ] Screenshots for 6.7" (iPhone 15 Pro Max) and 12.9" iPad sizes prepared
- [ ] Privacy policy URL added to App Store Connect
- [ ] Age rating questionnaire completed
- [ ] Export compliance question answered in `app.config.js`
- [ ] TestFlight beta testing completed
- [ ] `usesNonExemptEncryption: false` set if app doesn't use custom encryption

```bash
# Submit to App Store from EAS
eas submit --platform ios --profile production
```

## Play Store Submission Checklist

Before submitting to the Google Play Store:

- [ ] `versionCode` incremented (must be higher than previous submission)
- [ ] `version` updated in `app.config.js`
- [ ] App Bundle (`.aab`) built with `buildType: 'app-bundle'` in `eas.json`
- [ ] Adaptive icon provided (foreground + background)
- [ ] App tested on Android 8.0+ (API 26+)
- [ ] Google Play Console listing created
- [ ] Screenshots for phone (16:9) and 7-inch tablet prepared
- [ ] Feature graphic (1024×500) prepared
- [ ] Privacy policy URL added
- [ ] Content rating questionnaire completed
- [ ] App tested on low-end Android device
- [ ] Target API level meets Google's requirements (currently API 34+)
- [ ] `google-services.json` added if using Firebase

```bash
# Submit to Play Store from EAS
eas submit --platform android --profile production
```

## CI/CD with GitHub Actions

Automate builds and submissions:

```yaml
# .github/workflows/eas-build.yml
name: EAS Build & Submit

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    name: EAS Production Build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Setup Expo
        uses: expo/expo-github-action@v8
        with:
          expo-version: latest
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Build on EAS
        run: eas build --platform all --profile production --non-interactive

      - name: Submit to stores
        run: eas submit --platform all --profile production --non-interactive
        env:
          EXPO_APPLE_APP_SPECIFIC_PASSWORD: ${{ secrets.APPLE_APP_SPECIFIC_PASSWORD }}
```

## Fastlane Alternative

For teams preferring Fastlane (especially those already using it for CI/CD):

```ruby
# fastlane/Fastfile
default_platform(:ios)

platform :ios do
  desc "Push a new release build to the App Store"
  lane :release do
    increment_build_number(xcodeproj: "ios/MyApp.xcodeproj")
    build_app(
      workspace: "ios/MyApp.xcworkspace",
      scheme: "MyApp",
      export_method: "app-store"
    )
    upload_to_app_store(
      skip_metadata: false,
      skip_screenshots: false,
      submit_for_review: false
    )
  end
end

platform :android do
  desc "Push a new release build to the Play Store"
  lane :release do
    gradle(
      task: "bundle",
      build_type: "Release",
      project_dir: "android/"
    )
    upload_to_play_store(
      track: "internal",
      aab: "android/app/build/outputs/bundle/release/app-release.aab"
    )
  end
end
```

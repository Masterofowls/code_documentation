This guide walks you through every step to get a React Native app running on iOS and Android — from installing tools to seeing your first change appear on a physical device.

## Prerequisites

Before you begin, ensure the following tools are installed:

- **Node.js** 18+ (LTS recommended) — [nodejs.org](https://nodejs.org)
- **Git** — for version control
- **VS Code** — recommended editor with the React Native Tools extension
- **Watchman** (macOS only) — `brew install watchman`

For iOS development you need **Xcode** (macOS only, App Store) with Command Line Tools.
For Android development you need **Android Studio** with the Android SDK.

## Option A: Expo CLI (Recommended)

Expo is the fastest path to a working React Native app. No native toolchains required to start.

<Steps>

1. **Install the Expo CLI globally (optional) or use npx**

   ```bash
   npm install -g expo-cli
   # or just use npx (no install required):
   npx create-expo-app@latest
   ```

2. **Create a new project**

   ```bash
   npx create-expo-app@latest MyApp --template blank-typescript
   cd MyApp
   ```

   Available templates:
   - `blank` — minimal JavaScript starter
   - `blank-typescript` — minimal TypeScript starter ✅ (recommended)
   - `tabs` — bottom tab navigation starter
   - `bare-minimum` — bare workflow with TypeScript

3. **Start the development server**

   ```bash
   npx expo start
   ```

   You'll see a QR code and options to open on iOS/Android.

4. **Run on a simulator or device**

   Press `i` to open iOS Simulator, `a` for Android Emulator, or scan the QR code with **Expo Go** on your physical device.

</Steps>

:::tip[Expo Go vs Dev Client]
**Expo Go** (available on the App Store / Play Store) lets you test your app instantly without a build. It works great for apps that only use Expo SDK modules. For custom native modules, use `expo-dev-client` which creates a custom development build.
:::

## Option B: React Native CLI (Bare Workflow)

Use this when you need full control over the native project or are integrating into an existing native app.

<Steps>

1. **Install system dependencies**

   <Tabs>
     <TabItem label="macOS (iOS + Android)">
     ```bash
     # Install Homebrew first: https://brew.sh
     brew install node watchman

     # Install Ruby (for CocoaPods)
     brew install ruby

     # Install CocoaPods
     sudo gem install cocoapods

     # Install Android Studio from https://developer.android.com/studio
     # Then add to ~/.zshrc or ~/.bash_profile:
     export ANDROID_HOME=$HOME/Library/Android/sdk
     export PATH=$PATH:$ANDROID_HOME/emulator
     export PATH=$PATH:$ANDROID_HOME/platform-tools
     ```
     </TabItem>
     <TabItem label="Windows (Android only)">
     ```bash
     # Install Node.js from https://nodejs.org
     # Install Android Studio from https://developer.android.com/studio
     # Set ANDROID_HOME environment variable in System Properties

     # Add to PATH:
     # %LOCALAPPDATA%\Android\Sdk\platform-tools
     # %LOCALAPPDATA%\Android\Sdk\emulator
     ```
     </TabItem>
   </Tabs>

2. **Create the project**

   ```bash
   npx @react-native-community/cli@latest init MyApp --template react-native-template-typescript
   cd MyApp
   ```

3. **Install iOS dependencies (macOS only)**

   ```bash
   cd ios && pod install && cd ..
   ```

4. **Run the app**

   ```bash
   # iOS
   npx react-native run-ios

   # Android (with emulator or device connected)
   npx react-native run-android
   ```

</Steps>

:::caution[iOS requires macOS]
Building for iOS always requires a Mac with Xcode. Windows and Linux users can only build for Android locally. Use EAS Build (cloud) to build iOS apps from non-Mac machines.
:::

│   │   └── useAuth.ts

## Metro Bundler

Metro is React Native's JavaScript bundler. It:
- Resolves imports and bundles your JS into a single (or split) bundle
- Transforms JSX and TypeScript on-the-fly
- Enables Hot Refresh / Fast Refresh during development

### Custom Metro Config

```javascript
// metro.config.js
const { getDefaultConfig, mergeConfig } = require('@react-native/metro-config');

const defaultConfig = getDefaultConfig(__dirname);

const config = {
  transformer: {
    babelTransformerPath: require.resolve('react-native-svg-transformer'),
  },
  resolver: {
    assetExts: defaultConfig.resolver.assetExts.filter((ext) => ext !== 'svg'),
    sourceExts: [...defaultConfig.resolver.sourceExts, 'svg'],
  },
};

module.exports = mergeConfig(defaultConfig, config);
```

### Clearing the Metro Cache

```bash
# Expo
npx expo start --clear

# React Native CLI
npx react-native start --reset-cache
```

## TypeScript Configuration

The recommended `tsconfig.json` for React Native with strict mode:

```json
{
  "extends": "@tsconfig/react-native/tsconfig.json",
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@screens/*": ["src/screens/*"],
      "@hooks/*": ["src/hooks/*"]
    }
  }
}
```

Install the base config:

```bash
npm install -D @tsconfig/react-native
```

## Running on iOS Simulator

```bash
# List available simulators
xcrun simctl list devices

# Run on a specific simulator
npx expo run:ios --simulator "iPhone 15 Pro"

# React Native CLI
npx react-native run-ios --simulator "iPhone 15 Pro"
```

## Running on Android Emulator

<Steps>

1. Open Android Studio → **Virtual Device Manager** → Create a device (e.g., Pixel 8 Pro with API 34)
2. Start the emulator from Android Studio or the command line:
   ```bash
   emulator -avd Pixel_8_Pro_API_34
   ```
3. Run your app:
   ```bash
   npx expo run:android
   # or
   npx react-native run-android
   ```

</Steps>

## Physical Device Setup

<Tabs>
  <TabItem label="iOS">
  <Steps>
  1. Connect your iPhone via USB
  2. Trust the computer on your iPhone (popup on first connect)
  3. In Xcode, add your Apple ID under **Settings → Accounts**
  4. Select your device in the Xcode target dropdown
  5. For Expo: `npx expo run:ios --device`
  </Steps>
  </TabItem>
  <TabItem label="Android">
  <Steps>
  1. Enable **Developer Options** on your Android device (tap Build Number 7 times in Settings → About)
  2. Enable **USB Debugging**
  3. Connect via USB and authorize the connection
  4. Verify connection: `adb devices`
  5. Run: `npx expo run:android --device` or `npx react-native run-android`
  </Steps>
  </TabItem>
</Tabs>

## Hot Reload vs Fast Refresh

**Fast Refresh** (the modern system) preserves component state across edits and automatically recovers from syntax errors. It replaced the old Hot Reload system.

- **Fast Refresh**: Edit a component → only that component re-renders, state preserved ✅
- **Full reload**: Shake device → **Reload** (or `r` in Metro terminal) — resets all state
- **Fast Refresh toggle**: Shake device → **Enable/Disable Fast Refresh**

```tsx
// Fast Refresh works automatically — just save your file
// To force a full reload programmatically:
import { DevSettings } from 'react-native';

DevSettings.reload();
```

## Expo Go App

**Expo Go** is the fastest way to test an Expo app on a real device without building a native binary.

1. Install Expo Go from the [App Store](https://apps.apple.com/app/expo-go/id982107779) or [Play Store](https://play.google.com/store/apps/details?id=host.exp.exponent)
2. Start your dev server: `npx expo start`
3. Scan the QR code with your camera (iOS) or the Expo Go app (Android)

:::caution[Expo Go Limitations]
Expo Go only supports modules included in the Expo SDK. If you install a third-party native module (e.g., `react-native-vision-camera`), you must use `expo-dev-client` or `eas build --profile development` to test it.
:::

## EAS Build Introduction

EAS (Expo Application Services) Build is a cloud build service for React Native apps.

```bash
# Install EAS CLI
npm install -g eas-cli

# Log in to Expo
eas login

# Initialize EAS in your project
eas build:configure
```

This creates an `eas.json` file with build profiles:

```json
{
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal"
    },
    "production": {}
  }
}
```

Trigger a build:

```bash
# iOS development build
eas build --profile development --platform ios

# Android production build
eas build --profile production --platform android
```

## Common Setup Issues

| Problem | Solution |
|---|---|
| `ANDROID_HOME not set` | Add `export ANDROID_HOME=$HOME/Library/Android/sdk` to your shell profile |
| `CocoaPods not found` | Run `sudo gem install cocoapods` |
| Metro stuck on startup | Run `npx expo start --clear` |
| `adb: command not found` | Add Android SDK `platform-tools` to your PATH |
| iOS build fails with code signing | Add your Apple ID in Xcode → Settings → Accounts |
| Gradle build fails | Run `cd android && ./gradlew clean && cd ..` |

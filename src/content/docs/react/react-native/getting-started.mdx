This guide walks you through setting up a React Native project from scratch — whether you prefer the streamlined Expo experience or the fully manual React Native CLI setup.

## Prerequisites

Before you begin, ensure the following tools are installed:

| Tool | Version | Purpose |
|---|---|---|
| **Node.js** | 20 LTS+ | JavaScript runtime |
| **npm / yarn / bun** | Latest | Package manager |
| **Git** | Any | Version control |
| **VS Code** | Latest | Recommended editor |

For iOS development you also need **macOS with Xcode 15+**. Android development works on any OS with **Android Studio** and the Android SDK.

:::tip[Install Node with a Version Manager]
Use [nvm](https://github.com/nvm-sh/nvm) (macOS/Linux) or [fnm](https://github.com/Schniz/fnm) (cross-platform) to manage multiple Node versions. React Native projects often have a preferred Node version in `.nvmrc` or `.node-version`.
:::

## Quick Start with Expo

Expo is the recommended starting point. It handles native build tooling so you can focus on writing React code.

<Steps>

1. **Create a new Expo app**

   ```bash
   npx create-expo-app@latest my-app --template blank-typescript
   cd my-app
   ```

   For a more opinionated setup with Expo Router (file-based navigation):

   ```bash
   npx create-expo-app@latest my-app --template tabs
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Start the development server**

   ```bash
   npx expo start
   ```

   This launches **Metro Bundler** and displays a QR code in your terminal.

4. **Run on a device or simulator**

   - Press `i` to open the iOS Simulator (macOS only)
   - Press `a` to open an Android Emulator
   - Scan the QR code with the **Expo Go** app on your physical device

</Steps>

:::caution[Expo Go Limitations]
Expo Go only supports the Expo SDK's built-in native modules. If your project requires a custom native module (e.g., Stripe, Bluetooth, or a company SDK), you'll need a **Development Build** via EAS Build or the bare workflow.
:::

## Expo Go App

The **Expo Go** app (available on the [App Store](https://apps.apple.com/app/expo-go/id982107779) and [Google Play](https://play.google.com/store/apps/details?id=host.exp.exponent)) lets you instantly preview your Expo app on a physical device without going through a native build process.

```bash
# Start with tunnel mode for devices on different networks
npx expo start --tunnel
```

When you save a file, Expo Go automatically reloads your app via **Fast Refresh**.

## Bare React Native CLI Setup

For projects that need full native control from day one:

<Steps>

1. **Install React Native CLI**

   ```bash
   npm install -g react-native
   # Or use npx (recommended — no global install needed):
   npx @react-native-community/cli@latest init MyApp --template react-native-template-typescript
   ```

2. **Set up iOS dependencies**

   ```bash
   cd MyApp/ios
   bundle install        # Install Ruby gems (CocoaPods manager)
   bundle exec pod install
   cd ..
   ```

3. **Set up Android**

   Open `android/` in Android Studio to let it sync Gradle and download the SDK. Alternatively:

   ```bash
   # Ensure ANDROID_HOME is set in your shell profile
   export ANDROID_HOME=$HOME/Library/Android/sdk
   export PATH=$PATH:$ANDROID_HOME/emulator
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```

4. **Run the app**

   ```bash
   # iOS (macOS only)
   npx react-native run-ios

   # Android (any OS with emulator running)
   npx react-native run-android
   ```

</Steps>

## Metro Bundler

Metro is the JavaScript bundler for React Native, developed by Meta. It handles:

- Transforming TypeScript and JSX to JavaScript
- Module resolution (including Node.js-style `require` / ES module `import`)
- Generating source maps
- Hot Module Replacement (Fast Refresh)

### Customizing Metro

The `metro.config.js` file lets you extend Metro's behavior:

```javascript
// metro.config.js
const { getDefaultConfig, mergeConfig } = require('@react-native/metro-config');

const defaultConfig = getDefaultConfig(__dirname);

/** @type {import('metro-config').MetroConfig} */
const config = {
  resolver: {
    // Add SVG to the list of asset extensions
    assetExts: [...defaultConfig.resolver.assetExts.filter((e) => e !== 'svg')],
    sourceExts: [...defaultConfig.resolver.sourceExts, 'svg'],
  },
  transformer: {
    babelTransformerPath: require.resolve('react-native-svg-transformer'),
  },
};

module.exports = mergeConfig(defaultConfig, config);
```

## TypeScript Configuration

The recommended `tsconfig.json` for React Native:

```json
{
  "extends": "@react-native/typescript-config/tsconfig.json",
  "compilerOptions": {
    "strict": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@screens/*": ["src/screens/*"],
      "@hooks/*": ["src/hooks/*"],
      "@utils/*": ["src/utils/*"]
    }
  },
  "include": ["src", "app", "__tests__"],
  "exclude": ["node_modules", "babel.config.js", "metro.config.js"]
}
```

Then configure path aliases in `babel.config.js`:

```javascript
module.exports = {
  presets: ['module:@react-native/babel-preset'],
  plugins: [
    [
      'module-resolver',
      {
        root: ['./src'],
        extensions: ['.ios.js', '.android.js', '.js', '.ts', '.tsx', '.json'],
        alias: {
          '@': './src',
          '@components': './src/components',
          '@screens': './src/screens',
          '@hooks': './src/hooks',
        },
      },
    ],
    'react-native-reanimated/plugin', // Always last if using Reanimated
  ],
};
```

## Running on Physical Devices

### iOS Physical Device

<Steps>

1. Connect your iPhone via USB
2. Open Xcode → select your device from the device list
3. Set your Apple Developer account in **Signing & Capabilities**
4. Trust the certificate on your iPhone: **Settings → General → VPN & Device Management**
5. Run `npx react-native run-ios --device "iPhone name"` or press ▶ in Xcode

</Steps>

### Android Physical Device

<Steps>

1. Enable **Developer Options**: **Settings → About Phone → tap Build Number 7×**
2. Enable **USB Debugging** in Developer Options
3. Connect via USB and authorize the connection on your phone
4. Verify the device is detected: `adb devices`
5. Run `npx react-native run-android`

</Steps>

:::tip[Wireless Debugging (Android 11+)]
Enable **Wireless Debugging** in Developer Options to pair via QR code or pairing code — no USB cable required after initial pairing.
:::

## Hot Reload vs Fast Refresh

| Feature | Fast Refresh | Full Reload |
|---|---|---|
| **Trigger** | File save | Manual (`r` key or shake device) |
| **State preservation** | Yes (hooks state preserved) | No (app restarts) |
| **Speed** | Near-instant | 2–10 seconds |
| **When it fails** | On syntax errors or class components | — |

Fast Refresh is enabled by default. It intelligently re-runs only the changed module and its dependents, preserving React component state when possible.

```tsx
// Fast Refresh preserves this counter value between edits:
function Counter() {
  const [count, setCount] = React.useState(0); // Preserved on save!
  return <Pressable onPress={() => setCount(c => c + 1)}>
    <Text>{count}</Text>
  </Pressable>;
}
```

:::caution[State is Lost on Full Reload]
Fast Refresh won't preserve state if you add or remove hooks, change a class component, or introduce a runtime error. In those cases it falls back to a full reload.
:::

## EAS Build Introduction

**Expo Application Services (EAS) Build** is a cloud build service for creating iOS `.ipa` and Android `.apk`/`.aab` files without Xcode or Android Studio locally.

```bash
# Install EAS CLI
npm install -g eas-cli

# Log in to your Expo account
eas login

# Configure the project
eas build:configure

# Build for both platforms
eas build --platform all
```

The `eas.json` configuration file defines build profiles:

```json
{
  "cli": { "version": ">= 7.0.0" },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal",
      "android": { "buildType": "apk" }
    },
    "production": {
      "autoIncrement": true
    }
  },
  "submit": {
    "production": {}
  }
}
```

## Creating Your First Screen

Here's a complete, production-ready "Home" screen with TypeScript:

```tsx
import React, { useState } from 'react';
import {
  View,
  Text,
  TextInput,
  Pressable,
  StyleSheet,
  SafeAreaView,
  KeyboardAvoidingView,
  Platform,
  Alert,
} from 'react-native';

interface HomeScreenProps {
  onNavigate?: (screen: string) => void;
}

export default function HomeScreen({ onNavigate }: HomeScreenProps) {
  const [name, setName] = useState('');

  const handleGreet = () => {
    if (!name.trim()) {
      Alert.alert('Oops!', 'Please enter your name first.');
      return;
    }
    Alert.alert('Hello!', `Welcome to React Native, ${name}! 👋`);
  };

  return (
    <SafeAreaView style={styles.safe}>
      <KeyboardAvoidingView
        style={styles.container}
        behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
      >
        <Text style={styles.title}>React Native Starter</Text>
        <Text style={styles.subtitle}>Your cross-platform journey begins here</Text>

        <TextInput
          style={styles.input}
          placeholder="Enter your name"
          value={name}
          onChangeText={setName}
          returnKeyType="done"
          onSubmitEditing={handleGreet}
          autoCapitalize="words"
        />

        <Pressable
          style={({ pressed }) => [
            styles.button,
            pressed && styles.buttonPressed,
          ]}
          onPress={handleGreet}
        >
          <Text style={styles.buttonText}>Say Hello</Text>
        </Pressable>
      </KeyboardAvoidingView>
    </SafeAreaView>
  );
}

const styles = StyleSheet.create({
  safe: { flex: 1, backgroundColor: '#fff' },
  container: {
    flex: 1,
    paddingHorizontal: 24,
    justifyContent: 'center',
    gap: 16,
  },
  title: { fontSize: 32, fontWeight: '800', color: '#1a202c', textAlign: 'center' },
  subtitle: { fontSize: 16, color: '#718096', textAlign: 'center', marginBottom: 8 },
  input: {
    borderWidth: 1.5,
    borderColor: '#cbd5e0',
    borderRadius: 12,
    paddingHorizontal: 16,
    paddingVertical: 14,
    fontSize: 16,
    color: '#1a202c',
    backgroundColor: '#f7fafc',
  },
  button: {
    backgroundColor: '#4f46e5',
    borderRadius: 12,
    paddingVertical: 16,
    alignItems: 'center',
  },
  buttonPressed: { backgroundColor: '#4338ca', transform: [{ scale: 0.98 }] },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '700' },
});
```

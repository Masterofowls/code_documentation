This guide walks you through everything needed to create and run your first React Native application — from environment setup to running on real devices.

## Prerequisites

Before you start, ensure the following are installed:

- **Node.js** 18 or later (LTS recommended) — [nodejs.org](https://nodejs.org)
- **Watchman** (macOS only) — `brew install watchman`
- **Git**
- A code editor (VS Code with the React Native Tools extension recommended)

For iOS development you additionally need:
- **Xcode** 14+ (macOS only, from the Mac App Store)
- **CocoaPods** — `sudo gem install cocoapods`

For Android development:
- **Android Studio** with Android SDK, HAXM, and an emulator configured
- `ANDROID_HOME` environment variable set

## Expo CLI Quickstart

The fastest way to get started is with **Expo**. You don't need to configure Xcode or Android Studio to run your app during development — Expo Go handles that for you.

<Steps>

1. **Create a new project**

   ```bash
   npx create-expo-app@latest MyApp --template blank-typescript
   cd MyApp
   ```

   The TypeScript template is strongly recommended. It gives you full type safety from day one.

2. **Start the development server**

   ```bash
   npx expo start
   ```

   This launches **Metro** (the JavaScript bundler) and displays a QR code in your terminal.

3. **Run the app**

   - **On a physical device:** Install [Expo Go](https://expo.dev/client) from the App Store or Play Store, then scan the QR code.
   - **iOS Simulator:** Press `i` in the terminal (requires Xcode).
   - **Android Emulator:** Press `a` in the terminal (requires Android Studio).
   - **Web:** Press `w` (experimental).

4. **Make your first change**

   Open `app/(tabs)/index.tsx` and edit the text. The app updates instantly via **Fast Refresh**.

</Steps>

:::tip[Expo Go for Quick Prototyping]
Expo Go lets you test your app on a real device without any build step. It's perfect for development, but note it can only run apps that use Expo SDK libraries. For custom native modules, you'll need a development build.
:::

## Bare React Native CLI Setup

If you need full control over native code from day one, use the React Native CLI.

:::caution[macOS Required for iOS]
Building for iOS requires a Mac running macOS. If you're on Windows or Linux, you can only target Android unless you use a cloud build service like EAS Build.
:::

<Steps>

1. **Install dependencies**

   ```bash
   # Install the React Native CLI
   npm install -g @react-native-community/cli

   # Create a new project with TypeScript template
   npx @react-native-community/cli@latest init MyApp --template react-native-template-typescript
   cd MyApp
   ```

2. **Install iOS dependencies**

   ```bash
   cd ios && bundle install && bundle exec pod install && cd ..
   ```

3. **Start Metro bundler**

   ```bash
   npx react-native start
   ```

4. **Run the app**

   ```bash
   # In a new terminal tab:
   npx react-native run-ios

   # Or for Android:
   npx react-native run-android
   ```

</Steps>

## Metro Bundler

**Metro** is React Native's JavaScript bundler — the equivalent of webpack for the mobile world. It handles:

- **Module resolution** — resolves `import` statements and `require()` calls
- **Transformation** — transpiles TypeScript and JSX via Babel
- **Bundling** — packages everything into a single JS bundle
- **Hot reloading** — watches for file changes and pushes updates

```javascript
// metro.config.js — customize Metro
const { getDefaultConfig } = require('expo/metro-config');

const config = getDefaultConfig(__dirname);

// Add SVG support
config.transformer.babelTransformerPath = require.resolve('react-native-svg-transformer');
config.resolver.assetExts = config.resolver.assetExts.filter(ext => ext !== 'svg');
config.resolver.sourceExts = [...config.resolver.sourceExts, 'svg'];

// Add custom resolver aliases
config.resolver.alias = {
  '@components': './src/components',
  '@screens': './src/screens',
  '@hooks': './src/hooks',
};

module.exports = config;
```

:::tip[Metro Cache]
If you're seeing unexpected behavior, try clearing the Metro cache: `npx expo start --clear` or `npx react-native start --reset-cache`.
:::

## Running on iOS Simulator

```bash
# List available simulators
xcrun simctl list devices available

# Run on a specific device
npx react-native run-ios --simulator="iPhone 15 Pro"

# Expo equivalent
npx expo run:ios --device "iPhone 15 Pro"
```

Useful simulator keyboard shortcuts:
- `Cmd + R` — Reload
- `Cmd + D` — Open developer menu
- `Cmd + Shift + H` — Home button
- `Cmd + K` — Toggle software keyboard

## Running on Android Emulator

<Steps>

1. **Create an AVD** (Android Virtual Device) in Android Studio via Tools → Device Manager.

2. **Start the emulator** from Android Studio or via the command line:

   ```bash
   # List available AVDs
   emulator -list-avds

   # Start an AVD
   emulator -avd Pixel_7_API_34
   ```

3. **Run your app**

   ```bash
   npx react-native run-android
   # or
   npx expo run:android
   ```

</Steps>

Useful emulator shortcuts:
- `Ctrl + M` — Open developer menu (Windows/Linux) or `Cmd + M` (macOS)
- `Ctrl + R` — Reload (in dev menu)
- `Ctrl + Shift + F` — Toggle fast refresh (in dev menu)


1. On your Android device, go to **Settings → About Phone**.
2. Tap **Build Number** 7 times to enable Developer Mode.
3. Go to **Settings → Developer Options** and enable **USB Debugging**.
4. Connect via USB.
5. Verify connection:

   ```bash
   adb devices
   # Should show your device
   ```

6. Run:

   ```bash
   npx react-native run-android
   ```

## Hot Reload vs Fast Refresh

| Feature | Hot Reload (old) | Fast Refresh (current) |
|---|---|---|
| Preserves state | No | Yes (for function components) |
| Error recovery | Manual reload needed | Automatic after fixing error |
| Class components | Supported | Limited support |
| Hooks | Sometimes broken | Fully supported |
| Speed | Fast | Faster |

**Fast Refresh** is enabled by default in all modern React Native versions. It intelligently re-renders only the components that changed while preserving their state.

```typescript
// Fast Refresh preserves state in this component:
export default function Counter() {
  const [count, setCount] = React.useState(0); // preserved on refresh
  return (
    <View>
      <Text>{count}</Text>
      <Button title="+" onPress={() => setCount(c => c + 1)} />
    </View>
  );
}
```

:::caution[When Fast Refresh Resets State]
Fast Refresh resets state if: (1) the component is not the default export, (2) you add/remove hooks, (3) the file has a non-component export, or (4) you use class components. In these cases, it falls back to a full reload.
:::

## EAS Build Introduction

**Expo Application Services (EAS) Build** is a cloud build service that compiles your native app in the cloud — you don't need Xcode or Android Studio on your machine for production builds.

```bash
# Install EAS CLI
npm install -g eas-cli

# Log in to Expo account
eas login

# Configure your project
eas build:configure
```

This creates `eas.json`:

```json
{
  "cli": {
    "version": ">= 5.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal"
    },
    "production": {
      "autoIncrement": true
    }
  },
  "submit": {
    "production": {}
  }
}
```

```bash
# Build for iOS (production)
eas build --platform ios --profile production

# Build for Android
eas build --platform android --profile production

# Build both simultaneously
eas build --platform all --profile production
```

See the [Deployment guide](/react/react-native/deployment) for full EAS setup including code signing.

## Developer Menu

The in-app developer menu provides essential debugging tools:

| Option | Description |
|---|---|
| Reload | Full JS bundle reload |
| Fast Refresh | Toggle auto-reload on save |
| Open Debugger | Chrome DevTools JS debugger |
| Open React DevTools | Component inspector |
| Performance Monitor | FPS and JS/UI thread metrics |
| Element Inspector | Tap any element to inspect its styles |

Access it by shaking the device, pressing `Cmd+D` (iOS Simulator), `Ctrl+M` (Android Emulator), or `d` in the Metro terminal.

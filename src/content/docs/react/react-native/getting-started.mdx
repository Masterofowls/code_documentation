This guide walks you through everything needed to create and run a React Native app — from installing tools to seeing your app on a real device.

## Prerequisites

Before you start, make sure you have the following installed:

- **Node.js** 18 LTS or later — [nodejs.org](https://nodejs.org)
- **Git** — [git-scm.com](https://git-scm.com)
- **VS Code** (recommended) with the React Native Tools extension
- A package manager: **npm**, **yarn**, or **bun**

For physical device testing via USB:

- **iOS**: Xcode (Mac only) + Apple Developer account
- **Android**: Android Studio with SDK Tools

## Option A: Expo CLI (Recommended)

Expo is the fastest way to start a new React Native project. It handles the native build tooling and provides a rich SDK of pre-built modules.

<Steps>
1. **Create a new Expo app**

```bash
npx create-expo-app@latest my-app
cd my-app
```

To use the TypeScript template explicitly:

```bash
npx create-expo-app@latest my-app --template expo-template-blank-typescript
```

2. **Start the development server**

   ```bash
   npx expo start
   ```

   This launches Metro Bundler and displays a QR code in your terminal.

3. **Open on a device or simulator**
   - **iOS Simulator** (Mac only): Press `i` in the terminal
   - **Android Emulator**: Press `a` in the terminal
   - **Physical device**: Scan the QR code with the Expo Go app
     </Steps>

:::tip[Expo Go for Instant Testing]
Download **Expo Go** from the App Store or Google Play. Scan the QR code from `npx expo start` and your app loads instantly — no build step required. This works for the vast majority of Expo SDK modules.
:::

## Option B: React Native CLI (Bare)

Use the React Native CLI when you need full control over the native layer from day one.

:::caution[System Requirements]
The bare CLI requires platform-specific tooling:

- **iOS**: Xcode 15+, CocoaPods, Xcode Command Line Tools
- **Android**: Java Development Kit (JDK 17), Android Studio, Android SDK

See the [official environment setup guide](https://reactnative.dev/docs/environment-setup) for detailed OS-specific instructions.
:::

<Steps>
1. **Install dependencies**

```bash
# macOS — install Ruby and CocoaPods
brew install ruby
sudo gem install cocoapods

# Install Android Studio and set ANDROID_HOME
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools
```

2. **Create the project**

   ```bash
   npx @react-native-community/cli@latest init MyApp --template react-native-template-typescript
   cd MyApp
   ```

3. **Install iOS pods**

   ```bash
   cd ios && pod install && cd ..
   ```

4. **Run the app**

   ```bash
   # iOS
   npx react-native run-ios

   # Android (with an emulator running or device connected)
   npx react-native run-android
   ```

</Steps>

## Understanding Metro Bundler

Metro is React Native's JavaScript bundler. Unlike webpack (used on the web), Metro is optimised for mobile:

- **Incremental bundling** — Only rebundles changed files
- **Tree shaking** — Dead code elimination
- **Module resolution** — Resolves platform-specific files (`.ios.js`, `.android.js`)
- **Hot Module Replacement** — Updates modules without a full reload

```javascript
// metro.config.js — basic configuration
const { getDefaultConfig, mergeConfig } = require('@react-native/metro-config');

/** @type {import('metro-config').MetroConfig} */
const config = {
  resolver: {
    // Add additional file extensions
    sourceExts: ['js', 'jsx', 'ts', 'tsx', 'json', 'svg'],
  },
  transformer: {
    // Enable inline requires for lazy loading
    inlineRequires: true,
  },
};

module.exports = mergeConfig(getDefaultConfig(__dirname), config);
```

## Running on iOS Simulator

<Steps>
1. **Open Xcode** and install the iOS simulator runtime (Xcode → Settings → Platforms)

2. **List available simulators**

   ```bash
   xcrun simctl list devices
   ```

3. **Start on a specific simulator**

   ```bash
   # Expo
   npx expo start --ios --simulator "iPhone 16 Pro"

   # React Native CLI
   npx react-native run-ios --simulator "iPhone 16 Pro"
   ```

</Steps>

## Running on Android Emulator

<Steps>
1. **Open Android Studio** → Device Manager → Create Virtual Device

2. **Select a device** (e.g., Pixel 8 Pro) and choose a system image (API 35 recommended)

3. **Start the emulator** from Android Studio or via command line:

   ```bash
   # List available AVDs
   emulator -list-avds

   # Start an emulator
   emulator -avd Pixel_8_Pro_API_35
   ```

4. **Run your app**

   ```bash
   # Expo
   npx expo start --android

   # React Native CLI
   npx react-native run-android
   ```

</Steps>

## Hot Reload vs Fast Refresh

| Feature             | Hot Reload (Legacy)  | Fast Refresh (Current)       |
| ------------------- | -------------------- | ---------------------------- |
| **Availability**    | React Native < 0.61  | React Native 0.61+ (default) |
| **Component state** | Lost on reload       | Preserved when possible      |
| **Error handling**  | Full reload on error | Inline error overlay         |
| **Hook support**    | Limited              | Full                         |
| **Speed**           | Slower               | Faster                       |

Fast Refresh is always active during development. If a file doesn't hot-reload cleanly, press `r` in the Metro terminal to do a full reload.

```tsx
// Fast Refresh preserves state for components like this:
import React, { useState } from 'react';
import { Text, Button, View } from 'react-native';

export default function Counter() {
  // This count is preserved across fast refreshes
  const [count, setCount] = useState(0);

  return (
    <View>
      <Text>{count}</Text>
      <Button title="Increment" onPress={() => setCount((c) => c + 1)} />
    </View>
  );
}
```

:::tip[Force Full Reload]
If Fast Refresh produces unexpected behaviour after significant component restructuring, press `r` in the Expo/Metro terminal or shake the device and tap "Reload" to perform a clean reload.
:::

## TypeScript Setup

All modern React Native and Expo templates default to TypeScript. If you need to add TypeScript to an existing project:

```bash
npm install --save-dev typescript @types/react @types/react-native
```

```json
// tsconfig.json — recommended strict configuration
{
  "extends": "expo/tsconfig.base",
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@screens/*": ["src/screens/*"],
      "@hooks/*": ["src/hooks/*"]
    }
  },
  "include": ["**/*.ts", "**/*.tsx", ".expo/types/**/*.d.ts", "expo-env.d.ts"]
}
```

## EAS Build Introduction

Expo Application Services (EAS) Build is the recommended way to create production-ready iOS and Android builds from your Expo or bare React Native project — without needing Xcode or Android Studio locally.

```bash
# Install EAS CLI
npm install -g eas-cli

# Log in to Expo
eas login

# Configure EAS in your project
eas build:configure
```

This creates an `eas.json` file with build profiles:

```json
// eas.json — generated by eas build:configure
{
  "cli": {
    "version": ">= 10.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal"
    },
    "production": {}
  },
  "submit": {
    "production": {}
  }
}
```

```bash
# Create a development build for iOS
eas build --profile development --platform ios

# Create a production build for Android
eas build --profile production --platform android
```

See the [Deployment guide](/react/react-native/deployment) for a complete EAS workflow.

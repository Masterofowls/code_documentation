React Native is an open-source framework developed by Meta that lets you build native mobile applications for iOS and Android using React and JavaScript (or TypeScript). Unlike hybrid frameworks that render inside a WebView, React Native maps your JavaScript components to **real native UI elements**, giving users the performance and feel of a truly native app.

Released publicly in 2015, React Native has grown into one of the most widely-adopted mobile development frameworks, powering apps like Facebook, Instagram, Shopify, Discord, and Microsoft Office.

## What Is React Native?

At its core, React Native bridges the JavaScript world and the native platform. You write components in JSX, just like in a React web app, and the framework takes care of translating those components into native views:

- `&lt;View&gt;` → `UIView` on iOS / `android.view.View` on Android
- `&lt;Text&gt;` → `UILabel` on iOS / `TextView` on Android
- `&lt;Image&gt;` → `UIImageView` on iOS / `ImageView` on Android

This means your app doesn't just _look_ native — it _is_ native.

```tsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.heading}>Hello, React Native!</Text>
      <Text style={styles.body}>
        This text renders as a native UILabel on iOS and a TextView on Android.
      </Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f0f4ff',
  },
  heading: {
    fontSize: 24,
    fontWeight: '700',
    color: '#1a1a2e',
    marginBottom: 8,
  },
  body: {
    fontSize: 16,
    color: '#555',
    textAlign: 'center',
    paddingHorizontal: 24,
  },
});
```

:::tip[One Codebase, Two Platforms]
A single React Native codebase targets both iOS and Android. Platform-specific logic can be handled with `Platform.OS`, platform-specific file extensions (`.ios.tsx` / `.android.tsx`), or conditional rendering — all without maintaining two separate projects.
:::

## React Native vs Expo vs Bare Workflow

When starting a React Native project, you'll encounter three main approaches:

| Approach             | Description                                                          | Best For                                                  |
| -------------------- | -------------------------------------------------------------------- | --------------------------------------------------------- |
| **Expo Managed**     | Fully managed environment; Expo handles native build tooling         | Rapid prototyping, apps without custom native code        |
| **Expo Bare**        | Expo SDK + full access to native directories (`ios/` and `android/`) | Most production apps; best of both worlds                 |
| **React Native CLI** | Pure React Native; no Expo layer                                     | Maximum native control; large teams with native engineers |

### Expo Managed Workflow

Expo's managed workflow is the fastest way to get started. You don't need Xcode or Android Studio installed on day one — you can preview your app in **Expo Go** on a physical device.

```bash
npx create-expo-app@latest my-app --template
cd my-app
npx expo start
```

:::caution[Managed Workflow Limitations]
The Expo managed workflow restricts you to modules available in the Expo SDK. If you need a custom native module (e.g., Bluetooth LE, USB HID, proprietary SDKs), you'll need to eject to the bare workflow or use a development build.
:::

### Bare Workflow

The bare workflow gives you a standard React Native project with Expo SDK support layered on top. You have full access to `ios/` and `android/` directories.

```bash
npx create-expo-app@latest my-app --template bare-minimum
```

### React Native CLI (Pure)

```bash
npx @react-native-community/cli@latest init MyApp --template react-native-template-typescript
```

## The New Architecture

React Native's legacy architecture relied on an asynchronous **bridge** to communicate between JavaScript and native code. Every call crossed this bridge in JSON-serialised form, causing bottlenecks during complex animations or high-frequency updates.

The **New Architecture** — stable since React Native 0.74 — replaces the bridge with three modern systems:

### JSI (JavaScript Interface)

JSI is a C++ layer that allows JavaScript to hold direct references to native objects. Instead of serialising data to JSON and passing it over the bridge, JS can call native functions **synchronously** with zero serialisation overhead.

```tsx
// With JSI, this native call is synchronous — no bridge latency
const result = nativeModule.computeExpensiveValue(input);
```

### Fabric (New Renderer)

Fabric is the new React Native renderer. It reimplements the rendering pipeline in C++ and brings:

- **Synchronous layout** — measure and layout on the JS thread without waiting for the main thread
- **Concurrent mode support** — React 18 features like `useTransition` and `useDeferredValue` work natively
- **Better interoperability** with host platform views

### TurboModules

TurboModules replace the old Native Modules system. They are lazy-loaded (only initialised when first called) and use JSI for direct communication, eliminating the overhead of the legacy bridge for native API access.

```typescript
// TurboModule interface — defined in TypeScript, implemented natively
import type { TurboModule } from 'react-native';
import { TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  readonly getDeviceName: () => string;
  readonly getSystemVersion: () => string;
}

export default TurboModuleRegistry.getEnforcing<Spec>('DeviceInfo');
```

:::tip[Enabling the New Architecture]
New projects created with `create-expo-app` or the React Native CLI default to the New Architecture. For existing projects, set `newArchEnabled=true` in `android/gradle.properties` and enable it in `ios/Podfile`.
:::

## React Native vs Flutter vs Ionic

| Feature            | React Native               | Flutter                    | Ionic                    |
| ------------------ | -------------------------- | -------------------------- | ------------------------ |
| **Language**       | JavaScript / TypeScript    | Dart                       | JavaScript / TypeScript  |
| **UI Rendering**   | Native platform widgets    | Custom Skia canvas         | WebView (Capacitor)      |
| **Performance**    | Near-native                | Near-native                | Lower (WebView overhead) |
| **Code Sharing**   | iOS + Android              | iOS, Android, Web, Desktop | iOS, Android, Web        |
| **Web support**    | React Native Web (partial) | Excellent (stable)         | Native (same codebase)   |
| **Ecosystem**      | Very large (npm)           | Growing (pub.dev)          | Very large (npm)         |
| **Hot Reload**     | Fast Refresh               | Hot Reload                 | Live Reload              |
| **Company**        | Meta (open-source)         | Google (open-source)       | Ionic Team               |
| **Native APIs**    | Via modules (JSI)          | Via platform channels      | Via Capacitor plugins    |
| **Learning Curve** | Low (if you know React)    | Medium (learn Dart)        | Low (web dev skills)     |
| **Hiring Pool**    | Very large                 | Growing                    | Large                    |

:::tip[Choose React Native If...]
You already have a React web team and want to share knowledge, component logic, and potentially even code between web and mobile. The React model — components, hooks, state — transfers directly.
:::

## Essential Community Libraries

The React Native ecosystem is powered by a vibrant community. These are the libraries you'll reach for in almost every project:

### Navigation

- **react-navigation** — The de-facto standard for navigation. Stack, Tab, Drawer navigators, deep linking, and more.
- **expo-router** — File-based routing for Expo apps (similar to Next.js App Router).

### Animation & Gestures

- **react-native-reanimated** — Runs animations entirely on the UI thread for 60/120fps smoothness.
- **react-native-gesture-handler** — Native-driven gesture recognition (pan, pinch, tap, swipe).
- **Lottie** (`lottie-react-native`) — Render After Effects animations as vectors.

### State Management

- **Zustand** — Lightweight, hook-based global state; works great in RN.
- **Jotai** — Atomic state management.
- **Redux Toolkit** — For complex state with strong DevTools support.

### Data & Storage

- **TanStack Query** (`@tanstack/react-query`) — Async state, caching, background refetch.
- **MMKV** (`react-native-mmkv`) — Blazing-fast key-value storage backed by the C++ MMKV library.
- **WatermelonDB** — High-performance reactive database for React Native.
- **expo-sqlite** — SQLite database, now with async/sync API.

### UI Components

- **React Native Paper** — Material Design 3 components.
- **NativeBase / GlueStack UI** — Accessible, themeable component library.
- **FlashList** (`@shopify/flash-list`) — High-performance FlatList replacement.
- **React Native Elements** — Cross-platform UI toolkit.

### Utilities

- **axios** — HTTP client with interceptors and transform support.
- **dayjs** / **date-fns** — Lightweight date libraries.
- **react-native-mmkv** — Fastest key-value storage for React Native.
- **expo-constants** — Access app configuration at runtime.

## React Native in Production

React Native is trusted by some of the world's largest apps:

- **Meta** (Facebook, Instagram) — Original creators; used at massive scale.
- **Shopify** — Entire mobile commerce platform rebuilt in React Native.
- **Microsoft** (Office, Teams mobile) — Heavy usage across the Microsoft 365 suite.
- **Discord** — React Native powers the iOS and Android mobile apps.
- **Coinbase** — Crypto trading app for millions of users.
- **Walmart** — Retail mobile app serving US customers.

The framework's maturity, combined with the New Architecture's performance improvements, makes React Native a compelling choice for teams who want to ship high-quality mobile apps without maintaining separate iOS and Android codebases.

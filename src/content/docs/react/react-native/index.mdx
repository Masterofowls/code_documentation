React Native is an open-source framework developed by Meta that allows you to build native mobile applications for iOS and Android using JavaScript and React. Unlike hybrid frameworks that render web views, React Native compiles to actual native UI components, delivering performance and user experience that matches platform conventions.

## What is React Native?

React Native bridges the gap between web development and native mobile development. You write components in JavaScript/TypeScript using familiar React patterns, and the framework maps those components to native platform equivalents — `View` becomes `UIView` on iOS and `android.view.View` on Android.

```tsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

export default function HelloWorld() {
  return (
    <View style={styles.container}>
      <Text style={styles.heading}>Hello, React Native!</Text>
      <Text style={styles.body}>
        This renders as native UILabel on iOS and TextView on Android.
      </Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f5f5f5',
  },
  heading: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 8,
  },
  body: {
    fontSize: 16,
    color: '#666',
    textAlign: 'center',
    paddingHorizontal: 24,
  },
});
```

:::tip[Why React Native?]
If your team already knows React, React Native has a minimal learning curve. You share business logic, API calls, state management patterns, and even some UI components across web and mobile.
:::

## React Native vs Expo vs Bare Workflow

One of the first decisions when starting a React Native project is choosing your toolchain. The three main options each have distinct trade-offs.

### Expo Managed Workflow

Expo is a set of tools and services built on top of React Native. The **managed workflow** abstracts away the native layer entirely — you never touch Xcode or Android Studio. Expo provides a curated SDK of pre-built native modules for camera, location, notifications, and more.

**Best for:** New projects, teams without native mobile experience, rapid prototyping, and apps that fit within the Expo SDK.

```bash
# Create a new Expo managed project
npx create-expo-app@latest MyApp --template blank-typescript

# Start development server
cd MyApp && npx expo start
```

### Expo Bare Workflow

The **bare workflow** gives you full access to native code (Xcode and Android Studio) while still using Expo modules and the EAS build service. You can `npx expo install expo-camera` and also write your own native modules.

**Best for:** Apps that need custom native code but want to use Expo tooling.

### React Native CLI (Bare React Native)

The official React Native CLI creates a project with full native code exposed from day one. No Expo layer. Complete control over `android/` and `ios/` directories.

**Best for:** Teams with native mobile engineers, apps with complex native integrations, brownfield apps (adding RN to an existing native app).

```bash
# Create a bare React Native project
npx @react-native-community/cli@latest init MyApp --template react-native-template-typescript

# Run on iOS
npx react-native run-ios

# Run on Android
npx react-native run-android
```

:::caution[Expo Managed Limitations]
The managed workflow restricts which native APIs you can access to those in the Expo SDK. If you need a native library that isn't supported, you'll need to eject to bare workflow. Evaluate your requirements before committing.
:::

## The New Architecture

React Native's **New Architecture** (stable since RN 0.73) replaces the old bridge with three major improvements that dramatically improve performance.

### JSI — JavaScript Interface

The old architecture used an asynchronous JSON-serialized bridge to communicate between JavaScript and native code. Every native call meant serializing data to JSON, passing it across a bridge, and deserializing on the other side.

**JSI** (JavaScript Interface) replaces this with a C++ layer that allows JavaScript to hold direct references to native objects. This enables **synchronous** native calls and eliminates serialization overhead.

```typescript
// Old architecture — async bridge call (simplified)
NativeModules.MyModule.doSomething('data'); // async, serialized

// New architecture — JSI allows synchronous direct calls
// Native objects are C++ host objects accessible directly from JS
const nativeObject = global.__nativeModule;
nativeObject.doSomethingSync('data'); // synchronous!
```

### Fabric — New Rendering System

**Fabric** is the new rendering engine. In the old architecture, UI updates were processed asynchronously on a separate thread. Fabric enables:

- **Synchronous rendering** on demand
- **Concurrent mode** support (React 18 features like `useTransition` and `Suspense`)
- **Reduced thread hops** — layout and rendering can happen closer together
- **Better integration** with native views

### TurboModules

**TurboModules** replace Native Modules. Key improvements:

- **Lazy loading** — modules are only loaded when first accessed, improving startup time
- **Type safety** — automatically generated type bindings from a CodegenSpec
- **Synchronous access** via JSI
- **Faster initialization** since unused modules never load

```typescript
// TurboModule spec definition (TypeScript)
import type { TurboModule } from 'react-native';
import { TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  multiply(a: number, b: number): Promise<number>;
  getString(): string; // synchronous via JSI
}

export default TurboModuleRegistry.getEnforcing<Spec>('Calculator');
```

:::tip[Enabling New Architecture]
For Expo projects, set `"newArchEnabled": true` in your `app.json`. For bare React Native, set `newArchEnabled=true` in `android/gradle.properties` and enable it in your iOS Podfile.
:::

## React Native vs Flutter vs Ionic

| Feature | React Native | Flutter | Ionic |
|---|---|---|---|
| **Language** | JavaScript / TypeScript | Dart | JavaScript / TypeScript |
| **UI Rendering** | Native components | Custom canvas (Skia) | Web views (DOM) |
| **Performance** | Near-native | Near-native | Lower (DOM overhead) |
| **Web sharing** | React Native Web | Limited | Full (it's web) |
| **Learning curve** | Low (if you know React) | Medium (new language) | Low (web skills) |
| **Ecosystem** | Very large (npm) | Growing | Large (npm) |
| **Hot Reload** | Fast Refresh | Hot Reload | Live Reload |
| **Company** | Meta (Open Source) | Google (Open Source) | Ionic team |
| **Native feel** | Excellent | Good (custom widgets) | Varies |
| **3D / Games** | Limited | Good (Flame engine) | Limited |
| **Desktop** | Limited (React Native macOS/Windows) | Excellent | Web-based |
| **Hiring pool** | Large (React devs) | Growing | Large (web devs) |

### When to Choose React Native

- Your team knows React and JavaScript/TypeScript
- You want to share code with a React web app
- You need access to a massive npm ecosystem
- You want native-feeling UI that respects platform conventions

### When to Choose Flutter

- You want pixel-perfect custom UI that looks identical on both platforms
- Your team is comfortable learning Dart
- You need excellent desktop and web support in the same codebase

### When to Choose Ionic

- Your existing product is a web app and you want to wrap it in a native shell
- Performance is not a primary concern
- You want maximum code sharing with a web project

## Community Libraries

The React Native ecosystem is rich with high-quality community libraries. These are the essential ones used in production apps:

### Navigation

```bash
npx expo install @react-navigation/native @react-navigation/native-stack
npx expo install react-native-screens react-native-safe-area-context
```

**React Navigation** is the de facto standard for navigation. It provides stack, tab, drawer, and material navigators with deep linking support.

### Animations

```bash
npx expo install react-native-reanimated react-native-gesture-handler
```

**Reanimated 3** enables smooth 60/120fps animations that run on the UI thread (not JS thread), preventing jank during heavy JS work. Pair with **Gesture Handler** for declarative gesture recognition.

### State Management

```bash
npm install zustand
npx expo install @tanstack/react-query
```

**Zustand** for lightweight global state. **TanStack Query** for server state management with caching, background refetching, and offline support.

### Storage

```bash
npx expo install expo-secure-store
# For MMKV (bare RN):
npm install react-native-mmkv
```

**MMKV** is an ultra-fast key-value storage (10x faster than AsyncStorage) built on WeChat's open-source library. **SecureStore** for sensitive data using iOS Keychain / Android Keystore.

### Networking & Data

```bash
npm install axios
npx expo install expo-sqlite
```

**Axios** for HTTP requests with interceptors. **expo-sqlite** for local relational data with SQLite.

### UI Component Libraries

| Library | Description |
|---|---|
| `@shopify/restyle` | Type-safe, theme-based styling |
| `react-native-paper` | Material Design components |
| `@gluestack-ui` | Accessible, unstyled component primitives |
| `tamagui` | High-performance cross-platform UI kit |
| `nativewind` | Tailwind CSS utility classes for RN |

### Developer Tools

```bash
npx expo install expo-dev-client
npm install --save-dev flipper
```

**Expo Dev Client** adds a custom development launcher with Expo modules. **Flipper** is a desktop debugging platform for React Native with network inspection, layout hierarchy, and performance profiling.

React Native is an open-source framework created by Meta that lets you build native mobile applications for iOS and Android using JavaScript and React. Unlike hybrid frameworks that render web views, React Native compiles your JavaScript logic into **truly native UI components**, giving your users a genuine platform experience.

## What is React Native?

At its core, React Native bridges the gap between JavaScript and native platform APIs. You write components using familiar React patterns — JSX, hooks, and props — and the framework maps those components to their native counterparts: a `<View>` becomes a `UIView` on iOS and an `android.view.View` on Android.

```tsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.heading}>Hello, React Native!</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
    backgroundColor: '#f0f4f8',
  },
  heading: {
    fontSize: 24,
    fontWeight: '700',
    color: '#1a202c',
  },
});
```

:::tip[Why React Native?]
If your team already knows React, React Native has the smallest learning curve of any cross-platform mobile framework. You reuse component patterns, hooks, state management, and most third-party libraries across web and mobile.
:::

## React Native vs Expo vs Bare Workflow

The "React Native ecosystem" actually refers to three distinct setup approaches:

| Feature | Expo Managed | Expo Bare | React Native CLI |
|---|---|---|---|
| **Setup complexity** | Minimal | Moderate | High |
| **Native code access** | Via plugins | Full | Full |
| **Custom native modules** | Limited | Yes | Yes |
| **EAS Build / OTA** | First-class | First-class | Manual setup |
| **App size** | Larger (SDK) | Lean | Lean |
| **Recommended for** | New projects | Most projects | Enterprise/advanced |

**Expo Managed Workflow** is the recommended starting point for most teams. The Expo SDK provides pre-built native modules for camera, location, notifications, and dozens of other APIs without touching Xcode or Android Studio. When you need a native module that Expo doesn't support, you can **eject** to the bare workflow.

**Expo Bare Workflow** gives you full control over the native `ios/` and `android/` directories while still using Expo tools like EAS Build and Expo Router.

**React Native CLI** (bare, no Expo) is for teams that need maximum control, often for integrating into existing native apps or requiring obscure native dependencies.

:::caution[Avoid Ejecting Prematurely]
Once you eject from Expo Managed to Bare, you take on the responsibility of maintaining native build configs. Only eject when you genuinely need a native module that Config Plugins cannot satisfy.
:::

## New Architecture: JSI, Fabric, and TurboModules

React Native's **New Architecture** (enabled by default in RN 0.74+) replaces the old asynchronous "Bridge" with three major improvements:

### JavaScript Interface (JSI)

JSI is a C++ layer that replaces the old JSON-serialized bridge. Instead of asynchronously passing messages through a single thread, JSI allows JavaScript to hold **direct references to native objects** and call native functions **synchronously**.

```typescript
// Old Bridge (conceptual): async, JSON-serialized
NativeModules.MyModule.doSomething(data); // fire-and-forget

// New JSI (conceptual): synchronous C++ reference
const nativeRef = global.MyNativeObject;
nativeRef.compute(42); // direct call, no serialization overhead
```

### Fabric (New Renderer)

Fabric is React Native's new UI rendering system. It replaces the old shadow thread with a **synchronous, multi-threaded renderer** built in C++. Key benefits:

- UI updates can happen synchronously, eliminating jank caused by async bridge latency
- Concurrent React features (Suspense, transitions) work natively
- The renderer shares code between platforms

### TurboModules

TurboModules replace the old NativeModules system with **lazy-loaded, strongly-typed native modules**. In the old system, every native module was initialized at startup even if never used. TurboModules load on demand and expose typed interfaces via **CodeGen**, which generates type-safe bindings at build time.

```typescript
// TurboModule interface definition (spec file)
import type { TurboModule } from 'react-native';
import { TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  readonly getDeviceId: () => string;
  readonly vibrate: (duration: number) => void;
}

export default TurboModuleRegistry.getEnforcing<Spec>('MyNativeModule');
```

## Cross-Platform Architecture Overview

```
┌─────────────────────────────────────────┐
│           JavaScript / TypeScript        │
│         (React Components + Logic)       │
└────────────────┬────────────────────────┘
                 │ JSI (C++)
┌────────────────▼────────────────────────┐
│              Fabric Renderer             │
│         (C++ Shadow Tree / Yoga)         │
└──────────┬──────────────────┬───────────┘
           │                  │
┌──────────▼───────┐ ┌────────▼───────────┐
│   iOS Native UI  │ │  Android Native UI  │
│  (UIKit/SwiftUI) │ │  (View/Compose)     │
└──────────────────┘ └────────────────────┘
```

## React Native vs Flutter vs Ionic

| Criterion | React Native | Flutter | Ionic |
|---|---|---|---|
| **Language** | JavaScript / TypeScript | Dart | JavaScript / TypeScript |
| **UI rendering** | Native components | Custom Skia canvas | WebView / Web Components |
| **Performance** | Near-native | Near-native | Web-level |
| **Bundle size** | ~5–15 MB | ~10–20 MB | ~3–8 MB |
| **Hot reload** | Fast Refresh | Hot Reload | Live Reload |
| **Ecosystem** | Massive (npm) | Growing (pub.dev) | Web npm |
| **Web support** | React Native Web | Flutter Web | First-class |
| **TV / Watch** | Limited support | No | No |
| **Learning curve** | Low (React devs) | Medium (new lang) | Low (web devs) |
| **Backed by** | Meta | Google | Ionic/Capacitor |

:::tip[When to Choose React Native]
Choose React Native when your team is already proficient in React/TypeScript, when you need to share code between web and mobile, or when your project requires rapid iteration with over-the-air updates.
:::

## Community Libraries

The React Native ecosystem is rich with high-quality community libraries. Here are the most widely used:

### Navigation
- **[React Navigation](https://reactnavigation.org/)** — The de-facto navigation library. Supports Stack, Tab, Drawer, and Material navigators with deep linking and authentication flows.
- **[Expo Router](https://expo.github.io/router/)** — File-based routing for Expo apps, similar to Next.js App Router.

### Animations
- **[React Native Reanimated](https://docs.swmansion.com/react-native-reanimated/)** — High-performance animations running entirely on the UI thread using Worklets. Essential for gesture-driven UI.
- **[React Native Skia](https://shopify.github.io/react-native-skia/)** — Skia-powered 2D graphics for charts, custom shapes, and visual effects.
- **[Lottie](https://github.com/lottie-react-native/lottie-react-native)** — Play Adobe After Effects animations exported as JSON.

### Gestures
- **[React Native Gesture Handler](https://docs.swmansion.com/react-native-gesture-handler/)** — Declarative API for swipes, pans, pinches, and taps. Pairs with Reanimated.

### Storage
- **[MMKV](https://github.com/mrousavy/react-native-mmkv)** — Blazing-fast key-value storage (10× faster than AsyncStorage) backed by WeChat's MMKV library.
- **[WatermelonDB](https://nozbe.github.io/WatermelonDB/)** — High-performance reactive database for complex relational data.

### Networking & State
- **[TanStack Query](https://tanstack.com/query)** — Server state management with caching, refetching, and mutations.
- **[Zustand](https://zustand-demo.pmnd.rs/)** — Minimal, unopinionated global state.
- **[Axios](https://axios-http.com/)** — HTTP client with interceptors and cancellation.

### UI Component Libraries
- **[React Native Paper](https://reactnativepaper.com/)** — Material Design 3 components.
- **[NativeBase / GlueStack UI](https://gluestack.io/)** — Accessible, themeable component library.
- **[Tamagui](https://tamagui.dev/)** — Universal UI with static extraction for performance.

### Utilities
- **[React Native MMKV](https://github.com/mrousavy/react-native-mmkv)** — Fast sync storage.
- **[React Native Fast Image](https://github.com/DylanVann/react-native-fast-image)** — Image caching and performance.
- **[React Native SVG](https://github.com/software-mansion/react-native-svg)** — SVG rendering.
- **[Expo Image](https://docs.expo.dev/versions/latest/sdk/image/)** — Modern image component with blurhash placeholders.

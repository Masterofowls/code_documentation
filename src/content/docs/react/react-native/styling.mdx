React Native uses a subset of CSS powered by Facebook's **Yoga** layout engine. Styles are defined as JavaScript objects, compiled to native style values, and applied directly to native views — there is no CSS cascade, specificity, or stylesheet files.

## StyleSheet.create

While you can pass raw objects as the `style` prop, `StyleSheet.create` is strongly recommended:

```tsx
import { View, Text, StyleSheet } from 'react-native';

// ✅ Recommended — validated and optimized at startup:
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f8fafc',
    padding: 16,
  },
  title: {
    fontSize: 24,
    fontWeight: '700',
    color: '#1a202c',
    marginBottom: 8,
  },
});

// Usage:
<View style={styles.container}>
  <Text style={styles.title}>Hello</Text>
</View>

// Combine styles with an array (later styles override earlier ones):
<View style={[styles.container, styles.centered, { marginTop: 20 }]}>
  {/* ... */}
</View>
```

**Why `StyleSheet.create`?**
1. **Validation** — property names and value types are validated at startup, not at runtime
2. **Optimization** — styles are registered by ID and sent to the native side once, reducing bridge traffic in the old architecture
3. **TypeScript** — provides full autocompletion and type checking for all style properties
4. **Performance** — repeated style lookups hit an ID-keyed map rather than re-creating objects

:::tip[StyleSheet.flatten]
Use `StyleSheet.flatten(style)` to merge an array of style objects into a single object — useful when passing styles as props and needing to read computed values.
:::

## Flexbox Layout

React Native's layout system is Flexbox, but with important differences from web CSS:

| Property | Web default | React Native default |
|---|---|---|
| `flexDirection` | `row` | **`column`** |
| `alignContent` | `stretch` | `flex-start` |
| `flexShrink` | `1` | **`0`** |
| `position` | `static` | `relative` |

```tsx
import { View, StyleSheet } from 'react-native';

// Column layout (default):
<View style={styles.column}>
  <View style={styles.box} />
  <View style={styles.box} />
  <View style={styles.box} />
</View>

// Row layout:
<View style={styles.row}>
  <View style={[styles.box, { flex: 1 }]} />  {/* takes 1 part */}
  <View style={[styles.box, { flex: 2 }]} />  {/* takes 2 parts */}
  <View style={[styles.box, { flex: 1 }]} />  {/* takes 1 part */}
</View>

const styles = StyleSheet.create({
  column: {
    flex: 1,
    flexDirection: 'column',         // default
    justifyContent: 'flex-start',    // main axis: start | center | flex-end | space-between | space-around
    alignItems: 'stretch',           // cross axis: stretch | flex-start | center | flex-end
    gap: 12,                         // spacing between children (RN 0.71+)
  },
  row: {
    flexDirection: 'row',
    alignItems: 'center',
    justifyContent: 'space-between',
    paddingHorizontal: 16,
  },
  box: {
    height: 50,
    backgroundColor: '#4f46e5',
    borderRadius: 8,
  },
});
```

### Flex Cheat Sheet

```tsx
// flex: 1 — take all available space:
<View style={{ flex: 1 }} />

// flexBasis — initial size before grow/shrink:
<View style={{ flexBasis: 200, flexShrink: 1 }} />

// alignSelf — override parent's alignItems for one child:
<View style={{ alignSelf: 'flex-end' }} />

// Wrap items onto new lines:
<View style={{ flexDirection: 'row', flexWrap: 'wrap', gap: 8 }}>
  {tags.map(tag => <Tag key={tag} label={tag} />)}
</View>
```

## Dimensions

### Static Dimensions

```tsx
import { Dimensions } from 'react-native';

const { width: SCREEN_WIDTH, height: SCREEN_HEIGHT } = Dimensions.get('screen');
const { width: WINDOW_WIDTH, height: WINDOW_HEIGHT } = Dimensions.get('window');

// 'screen' includes status bar; 'window' is the usable area (slightly smaller on Android)

const styles = StyleSheet.create({
  halfWidth: { width: SCREEN_WIDTH / 2 },
  heroImage: { width: SCREEN_WIDTH, height: SCREEN_WIDTH * 0.6 }, // 60% aspect ratio
});
```

:::caution[Static Dimensions Are Stale After Rotation]
`Dimensions.get()` returns the value at call time. If the device rotates, the value won't update. Use the `useWindowDimensions` hook instead.
:::

### useWindowDimensions Hook

```tsx
import { useWindowDimensions, View, StyleSheet } from 'react-native';

function ResponsiveGrid() {
  const { width, height, fontScale, scale } = useWindowDimensions();

  const columns = width > 600 ? 3 : 2; // tablet vs phone
  const itemWidth = (width - 32 - (columns - 1) * 12) / columns;

  return (
    <View style={styles.grid}>
      {items.map(item => (
        <View key={item.id} style={{ width: itemWidth }}>
          {/* ... */}
        </View>
      ))}
    </View>
  );
}
```

## PixelRatio

Different devices have different **pixel densities** (DPR). React Native uses **logical pixels** (dp on Android, pt on iOS), but you sometimes need physical pixels for crisp rendering:

```tsx
import { PixelRatio } from 'react-native';

// Get device pixel ratio (1, 1.5, 2, 3, 3.5...):
const dpr = PixelRatio.get(); // e.g., 3.0 on iPhone 14

// Convert dp to physical pixels:
const pixelSize = PixelRatio.getPixelSizeForLayoutSize(1); // 3 on 3× device

// Round a dp value to the nearest physical pixel boundary (prevents blurry borders):
const hairlineWidth = StyleSheet.hairlineWidth; // 0.333... on 3× device
const borderWidth = PixelRatio.roundToNearestPixel(0.5); // 0.333 on 3×

// Select a font size that respects user's display size setting:
function normalize(size: number) {
  const scale = PixelRatio.getFontScale(); // respects user's accessibility font scale
  return Math.round(PixelRatio.roundToNearestPixel(size * scale));
}
```

## Platform-Specific Styles

### Platform.OS and Platform.select

```tsx
import { Platform, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,
    // Platform.select picks the value for the current platform:
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.1,
        shadowRadius: 8,
      },
      android: {
        elevation: 4,
      },
      default: {}, // web / other
    }),
  },
  text: {
    // Inline conditional:
    fontFamily: Platform.OS === 'ios' ? 'San Francisco' : 'Roboto',
    fontSize: Platform.OS === 'ios' ? 16 : 15,
  },
});

// Platform.Version for version-specific behavior:
const isAndroid12Plus = Platform.OS === 'android' && Platform.Version >= 31;
```

### Platform-Specific Files

React Native's Metro bundler automatically picks the right file based on the platform extension:

```
components/
  Button.tsx           ← fallback (web / other)
  Button.ios.tsx       ← iOS only
  Button.android.tsx   ← Android only
```

```tsx
// Button.ios.tsx — uses native iOS styling
import { Pressable, Text, StyleSheet } from 'react-native';
export function Button({ label, onPress }: ButtonProps) {
  return (
    <Pressable style={styles.button} onPress={onPress}>
      <Text style={styles.label}>{label}</Text>
    </Pressable>
  );
}
const styles = StyleSheet.create({
  button: { backgroundColor: '#007AFF', borderRadius: 10, padding: 14 },
  label: { color: '#fff', fontSize: 17, fontWeight: '600', textAlign: 'center' },
});
```

## Shadows

iOS and Android have fundamentally different shadow APIs:

```tsx
const shadowStyles = StyleSheet.create({
  // iOS — four separate shadow properties:
  iosShadow: {
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 4 },  // x, y displacement
    shadowOpacity: 0.12,                     // 0–1
    shadowRadius: 12,                        // blur radius in dp
    backgroundColor: '#fff',               // required for shadow to render!
  },

  // Android — single elevation value (material shadow):
  androidShadow: {
    elevation: 6,
    backgroundColor: '#fff',               // required for elevation to render!
  },

  // Cross-platform:
  shadow: {
    backgroundColor: '#fff',
    ...Platform.select({
      ios: {
        shadowColor: '#64748b',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.15,
        shadowRadius: 8,
      },
      android: { elevation: 4 },
    }),
  },
});
```

:::tip[Shadow Requires Background Color]
Both iOS shadows and Android elevation require `backgroundColor` to be set on the same View. Without it, the shadow/elevation will not appear.
:::

## Absolute Positioning

```tsx
const styles = StyleSheet.create({
  // Parent must have position: 'relative' (the default):
  container: {
    position: 'relative',
    width: 200,
    height: 200,
    backgroundColor: '#e2e8f0',
  },

  // Child is positioned relative to parent:
  badge: {
    position: 'absolute',
    top: -6,
    right: -6,
    width: 20,
    height: 20,
    borderRadius: 10,
    backgroundColor: '#ef4444',
    justifyContent: 'center',
    alignItems: 'center',
  },

  // Overlay filling parent:
  overlay: {
    position: 'absolute',
    top: 0,
    right: 0,
    bottom: 0,
    left: 0,
    backgroundColor: 'rgba(0,0,0,0.4)',
  },
});
```

## Theme with Context

A complete theming system using Context + MMKV for persistence:

```tsx
// src/theme/ThemeProvider.tsx
import React, { createContext, useContext, useState, useEffect, useMemo } from 'react';
import { useColorScheme } from 'react-native';
import { storage } from '../storage/mmkv';

const PALETTE = {
  indigo: { 50: '#eef2ff', 500: '#6366f1', 600: '#4f46e5', 700: '#4338ca' },
  slate: { 50: '#f8fafc', 100: '#f1f5f9', 200: '#e2e8f0', 500: '#64748b', 900: '#0f172a' },
};

function makeTheme(mode: 'light' | 'dark') {
  const isDark = mode === 'dark';
  return {
    mode,
    isDark,
    colors: {
      background: isDark ? PALETTE.slate[900] : PALETTE.slate[50],
      surface: isDark ? '#1e293b' : '#ffffff',
      primary: PALETTE.indigo[500],
      primaryDark: PALETTE.indigo[700],
      text: isDark ? '#f1f5f9' : PALETTE.slate[900],
      textSecondary: isDark ? '#94a3b8' : PALETTE.slate[500],
      border: isDark ? '#334155' : PALETTE.slate[200],
    },
    spacing: { xs: 4, sm: 8, md: 16, lg: 24, xl: 32 },
    borderRadius: { sm: 8, md: 12, lg: 16, xl: 24, full: 9999 },
    typography: {
      h1: { fontSize: 32, fontWeight: '800' as const, lineHeight: 40 },
      h2: { fontSize: 24, fontWeight: '700' as const, lineHeight: 32 },
      body: { fontSize: 16, fontWeight: '400' as const, lineHeight: 24 },
      caption: { fontSize: 13, fontWeight: '400' as const, lineHeight: 18 },
    },
  };
}

type Theme = ReturnType<typeof makeTheme>;
const ThemeContext = createContext<{ theme: Theme; toggleTheme: () => void } | undefined>(undefined);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const systemColorScheme = useColorScheme();
  const [mode, setMode] = useState<'light' | 'dark'>(() => {
    const saved = storage.getString('theme') as 'light' | 'dark' | undefined;
    return saved ?? systemColorScheme ?? 'light';
  });

  const theme = useMemo(() => makeTheme(mode), [mode]);

  const toggleTheme = () => {
    const next = mode === 'light' ? 'dark' : 'light';
    setMode(next);
    storage.set('theme', next);
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useTheme must be used inside ThemeProvider');
  return ctx;
}

// Usage:
function Button({ label, onPress }: { label: string; onPress: () => void }) {
  const { theme } = useTheme();
  return (
    <Pressable
      onPress={onPress}
      style={{
        backgroundColor: theme.colors.primary,
        borderRadius: theme.borderRadius.md,
        padding: theme.spacing.md,
      }}
    >
      <Text style={{ color: '#fff', ...theme.typography.body, fontWeight: '700' }}>
        {label}
      </Text>
    </Pressable>
  );
}
```

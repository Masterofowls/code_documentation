React Native uses a subset of CSS properties expressed as JavaScript objects. All layout is done with Flexbox, and there is no CSS cascade or inheritance (except for Text inside Text).

## StyleSheet.create

Always use `StyleSheet.create` to define styles. It validates property names in development and maps style IDs for performance — native code receives a number ID, not the full style object.

```tsx
import { View, Text, StyleSheet } from 'react-native';

export default function Card({ title, subtitle }: { title: string; subtitle: string }) {
  return (
    <View style={styles.card}>
      <Text style={styles.title}>{title}</Text>
      <Text style={styles.subtitle}>{subtitle}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,
    marginBottom: 12,
  },
  title: {
    fontSize: 17,
    fontWeight: '600',
    color: '#1C1C1E',
    lineHeight: 22,
  },
  subtitle: {
    fontSize: 14,
    color: '#8E8E93',
    marginTop: 4,
  },
});
```

:::tip[Why StyleSheet.create over plain objects]
When you pass a plain object as a style prop (e.g., `style={{ padding: 16 }}`), React Native creates a new object reference on every render, forcing layout recalculation. `StyleSheet.create` gives the style an integer ID that the native engine caches.
:::

### Composing Styles

You can compose multiple styles using an array. Later styles override earlier ones:

```tsx
<View style={[styles.base, styles.active, disabled && styles.disabled]} />

// Dynamic inline override (use sparingly)
<View style={[styles.card, { backgroundColor: color }]} />
```

## Flexbox Layout

React Native's layout model is Flexbox — but with important differences from web CSS:

| Property | Web default | React Native default |
|---|---|---|
| `flexDirection` | `row` | `column` |
| `alignContent` | `stretch` | `flex-start` |
| `flexShrink` | `1` | `0` |
| `position` | `static` | `relative` |

```tsx
import { View, Text, StyleSheet } from 'react-native';

// Common layout patterns
export function LayoutExamples() {
  return (
    <View style={styles.screen}>
      {/* Row layout with space-between */}
      <View style={styles.row}>
        <Text>Left</Text>
        <Text>Right</Text>
      </View>

      {/* Centered content */}
      <View style={styles.centered}>
        <Text>Centered</Text>
      </View>

      {/* Grid: 2 columns */}
      <View style={styles.grid}>
        <View style={styles.gridItem} />
        <View style={styles.gridItem} />
        <View style={styles.gridItem} />
        <View style={styles.gridItem} />
      </View>

      {/* Fill remaining space */}
      <View style={styles.fill}>
        <Text>This fills the rest</Text>
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  screen: { flex: 1, padding: 16 },

  row: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
  },

  centered: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
  },

  grid: {
    flexDirection: 'row',
    flexWrap: 'wrap',
    gap: 8,
  },

  gridItem: {
    width: '48%',
    aspectRatio: 1,
    backgroundColor: '#F2F2F7',
    borderRadius: 8,
  },

  fill: { flex: 1, backgroundColor: '#E8F5E9' },
});
```

### Key Flexbox Properties

```tsx
const styles = StyleSheet.create({
  // justifyContent: main axis alignment
  // 'flex-start' | 'flex-end' | 'center' | 'space-between' | 'space-around' | 'space-evenly'

  // alignItems: cross axis alignment
  // 'flex-start' | 'flex-end' | 'center' | 'stretch' | 'baseline'

  // flex: shorthand for how a child grows/shrinks
  // flex: 1 — takes all available space
  // flex: 2 — takes twice as much as flex: 1 siblings

  // gap (React Native 0.71+)
  container: { flexDirection: 'row', gap: 8 },
  // rowGap and columnGap also supported
});
```

## Dimensions API

Get the device screen size. Note: this returns the screen dimensions, which may include areas behind status bars.

```tsx
import { Dimensions, StyleSheet } from 'react-native';

const { width, height } = Dimensions.get('window'); // usable window
// Dimensions.get('screen') — full screen including status bar

const CARD_WIDTH = (width - 16 * 3) / 2; // 2 columns with 16pt padding/gap

const styles = StyleSheet.create({
  card: { width: CARD_WIDTH, height: CARD_WIDTH * 1.2 },
  hero: { width, height: height * 0.4 },
});
```

### useWindowDimensions (Preferred)

Unlike `Dimensions.get`, `useWindowDimensions` is a hook that re-renders on orientation change — making it the correct choice for responsive layouts.

```tsx
import { useWindowDimensions, View, StyleSheet } from 'react-native';

export function ResponsiveCard() {
  const { width, height, fontScale } = useWindowDimensions();
  const isLandscape = width > height;
  const cardWidth = isLandscape ? (width - 48) / 3 : (width - 32) / 2;

  return (
    <View style={{ width: cardWidth, aspectRatio: 1 }}>
      {/* content */}
    </View>
  );
}
```

## PixelRatio

React Native uses logical points, not physical pixels. `PixelRatio` converts between them.

```tsx
import { PixelRatio, StyleSheet } from 'react-native';

// Device pixel ratio (e.g., 3 on iPhone 15 Pro)
const ratio = PixelRatio.get();

// 1 logical pt = ratio physical pixels
// 1pt border that looks consistent on all screens:
const hairline = StyleSheet.hairlineWidth; // already 1/PixelRatio

// Rounding pixel values to avoid blurry rendering:
const roundedWidth = PixelRatio.roundToNearestPixel(100.4);
```

## Platform-Specific Styles

### Platform.OS

```tsx
import { Platform, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  container: {
    paddingTop: Platform.OS === 'ios' ? 0 : 24,
    // iOS uses safe-area for status bar, Android needs explicit padding
  },
  shadow: {
    // iOS shadow
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
    shadowRadius: 4,
    // Android shadow
    elevation: 3,
  },
});
```

### Platform.select

```tsx
import { Platform, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  button: {
    ...Platform.select({
      ios: {
        backgroundColor: '#007AFF',
        borderRadius: 10,
        paddingVertical: 14,
      },
      android: {
        backgroundColor: '#1976D2',
        borderRadius: 4,
        paddingVertical: 12,
        elevation: 2,
      },
      default: {
        backgroundColor: '#007AFF',
        borderRadius: 8,
        paddingVertical: 12,
      },
    }),
  },
});
```

### Platform-Specific File Extensions

```
Button.tsx          ← shared logic
Button.ios.tsx      ← iOS-only implementation
Button.android.tsx  ← Android-only implementation
```

React Native's bundler automatically picks the correct file:

```tsx
// This import resolves to Button.ios.tsx on iOS, Button.android.tsx on Android
import Button from './Button';
```

## Shadow Styles

iOS and Android use completely different shadow systems:

```tsx
const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff', // required for iOS shadow to show
    borderRadius: 12,

    // iOS shadow (does NOT work on Android)
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 4 },
    shadowOpacity: 0.12,
    shadowRadius: 8,

    // Android shadow (does NOT work on iOS)
    elevation: 4,

    // To cover both:
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 4 },
        shadowOpacity: 0.12,
        shadowRadius: 8,
      },
      android: { elevation: 4 },
    }),
  },
});
```

:::caution[Overflow and shadows]
On iOS, `overflow: 'hidden'` clips shadows. If you need both a shadow and clipped content (e.g., rounded image card), wrap in two `View`s: outer view has the shadow, inner view has `overflow: 'hidden'`.
:::

## Absolute Positioning

```tsx
const styles = StyleSheet.create({
  // Parent must be position: 'relative' (default)
  container: { position: 'relative', width: 200, height: 200 },

  // Badge in top-right corner
  badge: {
    position: 'absolute',
    top: -4,
    right: -4,
    width: 20,
    height: 20,
    borderRadius: 10,
    backgroundColor: '#FF3B30',
    justifyContent: 'center',
    alignItems: 'center',
  },

  // Full-screen overlay
  overlay: {
    position: 'absolute',
    top: 0,
    left: 0,
    right: 0,
    bottom: 0,
    backgroundColor: 'rgba(0,0,0,0.5)',
  },
});
```

## Themed Styles with Context

Pattern for theme-aware StyleSheet:

```typescript
// src/hooks/useStyles.ts
import { StyleSheet } from 'react-native';
import { useMemo } from 'react';
import { useTheme } from '../context/ThemeContext';

export function useThemedStyles<T extends StyleSheet.NamedStyles<T>>(
  factory: (colors: ReturnType<typeof useTheme>['colors']) => T,
): T {
  const { colors } = useTheme();
  return useMemo(() => StyleSheet.create(factory(colors)), [colors, factory]);
}
```

```tsx
import { useThemedStyles } from '../hooks/useStyles';

export function ProfileCard() {
  const styles = useThemedStyles((colors) => ({
    card: {
      backgroundColor: colors.surface,
      borderRadius: 12,
      padding: 16,
    },
    name: {
      fontSize: 17,
      fontWeight: '600',
      color: colors.text,
    },
    bio: {
      fontSize: 14,
      color: colors.textSecondary,
      marginTop: 4,
    },
  }));

  return (
    <View style={styles.card}>
      <Text style={styles.name}>Alice Smith</Text>
      <Text style={styles.bio}>iOS & Android developer</Text>
    </View>
  );
}
```

## Safe Area Insets

Use `react-native-safe-area-context` for precise safe area padding:

```bash
npx expo install react-native-safe-area-context
```

```tsx
// Wrap your app in SafeAreaProvider
import { SafeAreaProvider } from 'react-native-safe-area-context';

export default function App() {
  return (
    <SafeAreaProvider>
      <NavigationContainer>
        <RootNavigator />
      </NavigationContainer>
    </SafeAreaProvider>
  );
}

// Use the hook for custom components
import { useSafeAreaInsets } from 'react-native-safe-area-context';

export function FloatingButton() {
  const insets = useSafeAreaInsets();

  return (
    <View
      style={{
        position: 'absolute',
        bottom: insets.bottom + 16,
        right: 16,
      }}
    >
      {/* FAB content */}
    </View>
  );
}
```

## Typography Best Practices

```tsx
const styles = StyleSheet.create({
  // System fonts (recommended for native feel)
  body: {
    fontFamily: undefined, // System font: SF Pro on iOS, Roboto on Android
    fontSize: 16,
    lineHeight: 24,
  },

  // Custom font (must be loaded first with expo-font)
  heading: {
    fontFamily: 'Inter-Bold',
    fontSize: 24,
    letterSpacing: -0.5,
  },

  // Dynamic type support (iOS)
  dynamicText: {
    fontSize: 16,
    allowFontScaling: true,
    maxFontSizeMultiplier: 1.5,
  },
});
```

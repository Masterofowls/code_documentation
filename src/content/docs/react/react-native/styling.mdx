React Native uses a JavaScript-based styling system inspired by CSS but adapted for native mobile. Styles are objects — not stylesheets — applied via the `style` prop.

## StyleSheet.create

Always use `StyleSheet.create` over plain objects. It provides:
- **Validation** — catches invalid style properties in development
- **Performance** — styles are registered and referenced by integer IDs, avoiding object creation on each render
- **DevTools** — better display in React Native Debugger

```tsx
import { View, Text, StyleSheet } from 'react-native';

// ✅ Good — validated, optimized, typed
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    padding: 16,
  },
  title: {
    fontSize: 24,
    fontWeight: '700',
    color: '#111',
    marginBottom: 8,
  },
});

// ❌ Avoid — no validation, new object created on every render
const inlineStyles = {
  container: { flex: 1, backgroundColor: '#fff' },
};
```

### Composing Styles

```tsx
// Array syntax for composing multiple styles
<View style={[styles.base, isActive && styles.active, { marginTop: 8 }]}>
  <Text>Content</Text>
</View>

// StyleSheet.flatten to merge into one object
const merged = StyleSheet.flatten([styles.base, styles.active]);

// StyleSheet.absoluteFillObject — shorthand for position:absolute, top/left/right/bottom:0
<View style={StyleSheet.absoluteFillObject} />
```

## Flexbox in React Native

React Native uses Flexbox for layout, but with some **key differences from CSS**:

| Property | Web Default | React Native Default |
|---|---|---|
| `flexDirection` | `row` | **`column`** |
| `alignContent` | `stretch` | `flex-start` |
| `flexShrink` | `1` | **`0`** |
| Units | px, %, rem, etc. | **Density-independent pixels (dp)** |
| `position` | `static` | **`relative`** |

```tsx
import { View, Text, StyleSheet } from 'react-native';

export function FlexExamples() {
  return (
    // Default: column direction, items stack vertically
    <View style={styles.columnContainer}>

      {/* Row: items lay out horizontally */}
      <View style={styles.rowContainer}>
        <View style={[styles.box, { backgroundColor: '#FF6B6B' }]} />
        <View style={[styles.box, { backgroundColor: '#4ECDC4' }]} />
        <View style={[styles.box, { backgroundColor: '#45B7D1' }]} />
      </View>

      {/* justify-content: space-between */}
      <View style={styles.spaceBetween}>
        <Text>Left</Text>
        <Text>Center</Text>
        <Text>Right</Text>
      </View>

      {/* flex: 1 takes remaining space */}
      <View style={{ flexDirection: 'row', height: 100 }}>
        <View style={{ width: 80, backgroundColor: '#eee' }} />
        <View style={{ flex: 1, backgroundColor: '#ddd' }} />
        <View style={{ width: 80, backgroundColor: '#eee' }} />
      </View>

    </View>
  );
}

const styles = StyleSheet.create({
  columnContainer: {
    flex: 1,
    padding: 16,
  },
  rowContainer: {
    flexDirection: 'row',
    alignItems: 'center',
    gap: 8,       // React Native 0.71+ supports gap
    marginBottom: 16,
  },
  box: {
    width: 60,
    height: 60,
    borderRadius: 8,
  },
  spaceBetween: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    padding: 12,
    backgroundColor: '#f5f5f5',
    borderRadius: 8,
  },
});
```

:::tip[gap Support]
`gap`, `rowGap`, and `columnGap` are supported in React Native 0.71+. This is often cleaner than adding margins to individual children.
:::

## Dimensions

```tsx
import { Dimensions, View, Text } from 'react-native';

// Get screen dimensions once (snapshot at call time)
const { width: SCREEN_WIDTH, height: SCREEN_HEIGHT } = Dimensions.get('screen');
// 'window' is the app usable area (excluding status bar on Android):
const { width: WINDOW_WIDTH, height: WINDOW_HEIGHT } = Dimensions.get('window');

// Use for responsive sizing
const styles = StyleSheet.create({
  fullWidthCard: {
    width: SCREEN_WIDTH - 32, // 16px padding each side
    height: SCREEN_WIDTH * 0.6, // aspect ratio
  },
  halfScreen: {
    width: SCREEN_WIDTH / 2,
    height: SCREEN_HEIGHT / 3,
  },
});
```

### useWindowDimensions Hook

Unlike `Dimensions.get()`, this hook **re-renders on orientation change** and on foldable device state changes:

```tsx
import { useWindowDimensions, View, Text, StyleSheet } from 'react-native';

export function ResponsiveGrid() {
  const { width, height, fontScale } = useWindowDimensions();
  const isLandscape = width > height;
  const numColumns = isLandscape ? 3 : 2;

  return (
    <View style={styles.container}>
      <Text style={{ fontSize: 14 * fontScale }}>
        {isLandscape ? 'Landscape' : 'Portrait'}: {Math.round(width)} × {Math.round(height)}
      </Text>
      {/* Responsive grid */}
      <View style={[styles.grid, { flexDirection: 'row', flexWrap: 'wrap' }]}>
        {items.map(item => (
          <View
            key={item.id}
            style={{
              width: (width - 48) / numColumns, // 16px padding + gaps
              marginBottom: 12,
            }}
          >
            <Text>{item.name}</Text>
          </View>
        ))}
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 16 },
  grid: { gap: 8 },
});
```

## PixelRatio

React Native uses density-independent pixels (dp). `PixelRatio` lets you work with physical pixels when needed.

```typescript
import { PixelRatio } from 'react-native';

// Device pixel ratio (e.g., 3 for iPhone 14)
const ratio = PixelRatio.get(); // 1, 1.5, 2, 3, 3.5, etc.

// Font scale factor (user's accessibility setting)
const fontScale = PixelRatio.getFontScale();

// Convert dp to physical pixels
const physicalPixels = PixelRatio.getPixelSizeForLayoutSize(16); // 16dp → 48px on 3x device

// Round to nearest physical pixel (for crisp 1px borders)
const hairline = StyleSheet.hairlineWidth; // 0.333... on 3x device

const styles = StyleSheet.create({
  divider: {
    height: StyleSheet.hairlineWidth,    // 1 physical pixel
    backgroundColor: '#ddd',
  },
  sharpBorder: {
    borderWidth: PixelRatio.roundToNearestPixel(0.5),  // precise half-pixel
    borderColor: '#ccc',
  },
});
```

## Platform-Specific Styles

### Platform.OS

```tsx
import { Platform, StyleSheet, View } from 'react-native';

const styles = StyleSheet.create({
  header: {
    paddingTop: Platform.OS === 'ios' ? 0 : 8, // Status bar on Android
    backgroundColor: Platform.OS === 'ios' ? '#fff' : '#f5f5f5',
    // TypeScript conditional
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 1 },
        shadowOpacity: 0.1,
        shadowRadius: 4,
      },
      android: {
        elevation: 4,
      },
    }),
  },
  input: {
    height: Platform.OS === 'ios' ? 44 : 48,
    paddingHorizontal: 12,
    borderRadius: Platform.OS === 'ios' ? 10 : 4,
  },
});

// Platform.Version for version-specific code
if (Platform.OS === 'android' && Platform.Version >= 31) {
  // Android 12+ specific features
}
```

### Platform-Specific Files

React Native automatically resolves platform-specific files:

```
Button.ios.tsx      # Used on iOS
Button.android.tsx  # Used on Android
Button.tsx          # Fallback for both (or used on web)
```

```tsx
// Button.ios.tsx
export function Button({ title, onPress }: ButtonProps) {
  return (
    <Pressable onPress={onPress} style={styles.iosButton}>
      <Text style={styles.iosText}>{title}</Text>
    </Pressable>
  );
}

// Button.android.tsx
export function Button({ title, onPress }: ButtonProps) {
  return (
    <Pressable
      onPress={onPress}
      style={styles.androidButton}
      android_ripple={{ color: 'rgba(0,0,0,0.2)' }}
    >
      <Text style={styles.androidText}>{title}</Text>
    </Pressable>
  );
}
```

## Shadows

Shadows work differently on iOS and Android:

```tsx
const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff',
    borderRadius: 12,
    padding: 16,

    // iOS shadow (4 separate properties)
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 4 },
    shadowOpacity: 0.12,
    shadowRadius: 8,

    // Android shadow (single elevation value)
    elevation: 6,
  },

  // Cross-platform shadow helper
  elevatedSurface: {
    backgroundColor: '#fff',
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.08,
        shadowRadius: 6,
      },
      android: {
        elevation: 4,
      },
    }),
  },
});
```

:::caution[Android Shadow Color]
On Android, `shadowColor` only affects the shadow color in React Native 0.72+ (New Architecture). Below that, Android shadows are always gray regardless of `shadowColor`. For colored shadows on older RN, use a community library.
:::

## Absolute Positioning

```tsx
const styles = StyleSheet.create({
  container: {
    position: 'relative',   // required for absolute children to position against
    width: 200,
    height: 200,
  },

  // Cover parent completely
  overlay: {
    ...StyleSheet.absoluteFillObject, // position:absolute, top:0, left:0, right:0, bottom:0
    backgroundColor: 'rgba(0,0,0,0.5)',
    zIndex: 1,
  },

  // Badge in top-right corner
  badge: {
    position: 'absolute',
    top: -4,
    right: -4,
    backgroundColor: '#FF3B30',
    borderRadius: 10,
    width: 20,
    height: 20,
    justifyContent: 'center',
    alignItems: 'center',
    zIndex: 1,
  },
});
```

```javascript
// tailwind.config.js
module.exports = {
  content: ['./app/**/*.{ts,tsx}', './components/**/*.{ts,tsx}'],
```

```tsx
// components/Card.tsx
```

## Safe Area Insets

```tsx
import { useSafeAreaInsets } from 'react-native-safe-area-context';

export function BottomBar() {
  const insets = useSafeAreaInsets();

  return (
    <View
      style={{
        paddingBottom: insets.bottom + 8,  // Extend below home indicator
        paddingTop: 8,
        paddingHorizontal: 16,
        backgroundColor: '#fff',
        borderTopWidth: StyleSheet.hairlineWidth,
        borderTopColor: '#e5e5ea',
      }}
    >
      {/* Bottom bar content */}
    </View>
  );
}

export function Screen() {
  const insets = useSafeAreaInsets();

  return (
    <View
      style={{
        flex: 1,
        paddingTop: insets.top,
        paddingLeft: insets.left,
        paddingRight: insets.right,
        // Don't add paddingBottom here — let the content scroll under home indicator
      }}
    >
      {/* Screen content */}
    </View>
  );
}
```

:::tip[Safe Area Insets vs SafeAreaView]
Use `useSafeAreaInsets()` when you need precise control over which edges get padding. Use `<SafeAreaView edges={['top']}>` for simpler cases.
:::

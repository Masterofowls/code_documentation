This guide walks you through creating a new React 18 application with Vite, understanding the project structure, mastering JSX, and writing your first components with conditional and list rendering.

## Creating a Project with Vite

Vite is the recommended build tool for new React projects. It starts in milliseconds, supports Hot Module Replacement (HMR), and produces highly optimised production bundles.

<Steps>

1. **Scaffold the project**

   ```bash
   npm create vite@latest my-app -- --template react-ts
   cd my-app
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Start the dev server**

   ```bash
   npm run dev
   ```

   **Console output:**
   ```
   VITE v5.x.x  ready in 312 ms

   ➜  Local:   http://localhost:5173/
   ➜  Network: use --host to expose
   ➜  press h + enter to show help
   ```

4. **Build for production**

   ```bash
   npm run build
   ```

   **Console output:**
   ```
   vite v5.x.x building for production...
   ✓ 34 modules transformed.
   dist/index.html                  0.46 kB
   dist/assets/index-Bx2kDuOK.css  1.39 kB
   dist/assets/index-CyL_SKPK.js   143.36 kB
   ✓ built in 1.45s
   ```

</Steps>

:::tip[Why Vite over Create React App?]
Create React App (CRA) is officially unmaintained. Vite is 10–100× faster in dev mode, uses native ES modules for instant HMR, and produces smaller production bundles via Rollup. For new projects always choose Vite (or a meta-framework like Next.js / Remix).
:::

## Project Structure

```bash
my-app/
├── public/               # Static assets served as-is
│   └── vite.svg
├── src/
│   ├── assets/           # Imported assets (images, fonts)
│   ├── components/       # Shared/reusable components
│   ├── hooks/            # Custom hooks
│   ├── pages/            # Route-level components
│   ├── types/            # Shared TypeScript types
│   ├── App.tsx           # Root component
│   ├── App.css           # Root styles
│   ├── main.tsx          # Entry point — createRoot lives here
│   └── vite-env.d.ts     # Vite type declarations
├── index.html            # HTML entry point
├── package.json
├── tsconfig.json
├── tsconfig.node.json
└── vite.config.ts
```

### Entry Point: main.tsx

```tsx
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import App from './App.tsx';
import './index.css';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>,
);
```

## JSX Rules

JSX is a syntax extension that looks like HTML but compiles to `React.createElement` calls. Understanding its rules is fundamental.

### Rule 1 — Return a Single Root Element

```tsx
// ❌ Error: JSX expressions must have one parent element
function BadComponent() {
  return (
    <h1>Title</h1>
    <p>Paragraph</p>
  );
}

// ✅ Wrap in a div
function OkComponent() {
  return (
    <div>
      <h1>Title</h1>
      <p>Paragraph</p>
    </div>
  );
}

// ✅ Better: use a Fragment to avoid extra DOM nodes
function GoodComponent() {
  return (
    <>
      <h1>Title</h1>
      <p>Paragraph</p>
    </>
  );
}
```

### Rule 2 — className, not class

```tsx
// ❌ HTML attribute
<div class="container">...</div>

// ✅ JSX attribute
<div className="container">...</div>
```

### Rule 3 — htmlFor, not for

```tsx
// ❌ HTML attribute
<label for="email">Email</label>

// ✅ JSX attribute
<label htmlFor="email">Email</label>
<input id="email" type="email" />
```

### Rule 4 — Self-close Empty Tags

```tsx
// ❌ Unclosed tag is invalid JSX
<input type="text">
<br>
<img src="logo.png">

// ✅ Self-close all tags that have no children
<input type="text" />
<br />
<img src="logo.png" alt="Logo" />
```

### Rule 5 — JavaScript Expressions in `{}`

```tsx
const user = { name: 'Alice', score: 42 };
const isAdmin = true;

function UserCard() {
  return (
    <div>
      {/* String interpolation */}
      <h2>{user.name}</h2>

      {/* Arithmetic */}
      <p>Score doubled: {user.score * 2}</p>

      {/* Ternary */}
      <span>{isAdmin ? 'Admin' : 'User'}</span>

      {/* Function calls */}
      <time>{new Date().toLocaleDateString()}</time>

      {/* Array methods */}
      <ul>
        {['a', 'b', 'c'].map(letter => (
          <li key={letter}>{letter.toUpperCase()}</li>
        ))}
      </ul>
    </div>
  );
}
```

**Rendered output:**
```
Alice
Score doubled: 84
Admin
1/15/2025

• A
• B
• C
```

:::caution[Curly Braces Hold Expressions, Not Statements]
You cannot put `if`, `for`, or `switch` statements directly inside JSX `{}`. Use ternary expressions, `&&` short-circuit, or extract logic to variables outside the return.
:::

## Fragments

Fragments let you group elements without adding extra DOM nodes:

```tsx
import { Fragment } from 'react';

// Short syntax (most common)
function Table() {
  return (
    <table>
      <tbody>
        {rows.map(row => (
          // Key can only be passed to <Fragment>, not <>
          <Fragment key={row.id}>
            <tr>
              <td>{row.name}</td>
              <td>{row.value}</td>
            </tr>
            {row.hasNote && (
              <tr>
                <td colSpan={2}>{row.note}</td>
              </tr>
            )}
          </Fragment>
        ))}
      </tbody>
    </table>
  );
}
```

## StrictMode

`StrictMode` is a development-only tool that helps you find bugs early:

```tsx
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>,
);
```

<Aside type="note" title="What StrictMode Does">
- Double-invokes render functions and reducers to detect side effects in the wrong place
- Double-invokes `useEffect` setup + cleanup in development to verify cleanup is correct
- Warns about deprecated APIs
- Has no effect in production — zero performance cost in prod builds
</Aside>

## Rendering to the DOM

```tsx
import { createRoot } from 'react-dom/client';

// Find the mount point in index.html
const container = document.getElementById('root');

if (!container) {
  throw new Error('Root element not found — check your index.html');
}

// Create a React root and render
const root = createRoot(container);
root.render(<App />);

// Unmount when needed (e.g., micro-frontend teardown)
// root.unmount();
```

## Conditional Rendering

```tsx
function Notification({ message }: { message: string | null }) {
  return (
    <div>
      <h1>Dashboard</h1>

      {message && (
        <div className="alert">
          {message}
        </div>

    </div>
  );
}
```

:::caution[Avoid Falsy Number Rendering]
`{count && <Component />}` renders `0` when `count` is 0. Use `{count > 0 && <Component />}` or a ternary instead.

## List Rendering

```tsx
interface Todo {
  id: number;
  text: string;
  completed: boolean;
}

const todos: Todo[] = [
  { id: 1, text: 'Learn React', completed: true },
  { id: 2, text: 'Build an app', completed: false },
  { id: 3, text: 'Ship to production', completed: false },
];

function TodoList() {
  return (
    <ul>
      {todos.map(todo => (
        <li
          key={todo.id}
          style={{ textDecoration: todo.completed ? 'line-through' : 'none' }}
        >
          {todo.text}
        </li>
      ))}
    </ul>
  );
}
```

**Rendered output:**
```
• ~~Learn React~~
• Build an app
• Ship to production
```

:::tip[The key Prop Rule]
Always use a stable, unique identifier (like a database ID) as the `key` prop when rendering lists. Never use array index as key unless the list is static and never reordered — index keys cause subtle bugs with component state and animations.
:::

### Filtering and Transforming Lists

```tsx
function ActiveTodos({ todos }: { todos: Todo[] }) {
  const active = todos.filter(t => !t.completed);

  if (active.length === 0) {
    return <p>All done! 🎉</p>;
  }

  return (
    <ul>
      {active.map(todo => (
        <li key={todo.id}>{todo.text}</li>
      ))}
    </ul>
  );
}
```

## CSS Modules

CSS Modules scope styles to a single component, preventing class name collisions:

```bash
src/
└── components/
    ├── Button.tsx
    └── Button.module.css
```

```tsx
/* Button.module.css */
.button {
  padding: 8px 16px;
  border-radius: 4px;
  font-weight: 600;
  cursor: pointer;
}

.primary {
  background: #0070f3;
  color: white;
  border: none;
}

.secondary {
  background: white;
  color: #0070f3;
  border: 2px solid #0070f3;
}
```

```tsx
// Button.tsx
import styles from './Button.module.css';

interface ButtonProps {
  variant?: 'primary' | 'secondary';
  children: React.ReactNode;
  onClick?: () => void;
}

export function Button({ variant = 'primary', children, onClick }: ButtonProps) {
  return (
    <button
      className={`${styles.button} ${styles[variant]}`}
      onClick={onClick}
    >
      {children}
    </button>
  );
}
```

The compiler transforms `.button` to a unique class like `Button_button__xK2jP`, ensuring zero collisions.

## Environment Variables

Vite exposes environment variables through `import.meta.env`:

```bash
# .env.local (never commit this)
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=My React App
VITE_FEATURE_FLAGS=dark-mode,beta-dashboard
```

```tsx
// Any component or module
const apiUrl = import.meta.env.VITE_API_URL;
const appTitle = import.meta.env.VITE_APP_TITLE;

// Built-in Vite variables
const isDev = import.meta.env.DEV;        // boolean
const isProd = import.meta.env.PROD;      // boolean
const mode = import.meta.env.MODE;        // 'development' | 'production'
const base = import.meta.env.BASE_URL;    // base URL from vite.config.ts
```

:::caution[Prefix Variables with VITE_]
Only variables prefixed with `VITE_` are exposed to your client-side code. Other variables in `.env` are available only on the server side (Node.js) and are NOT bundled — keeping secrets safe from the browser.
:::

### Type-safe Environment Variables

```tsx
// src/vite-env.d.ts (extend the default)
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_URL: string;
  readonly VITE_APP_TITLE: string;
  readonly VITE_FEATURE_FLAGS?: string;
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```

### Validate at Startup

```tsx
// src/config.ts
function requireEnv(key: keyof ImportMetaEnv): string {
  const value = import.meta.env[key];
  if (!value) {
    throw new Error(`Missing required environment variable: ${key}`);
  }
  return value;
}

export const config = {
  apiUrl: requireEnv('VITE_API_URL'),
  appTitle: import.meta.env.VITE_APP_TITLE ?? 'My App',
} as const;
```

## Vite Config Reference

```tsx
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
      },
    },
  },
  build: {
    sourcemap: true,
    rollupOptions: {
      output: {
        manualChunks: {
          react: ['react', 'react-dom'],
          router: ['react-router-dom'],
        },
      },
    },
  },
});
```

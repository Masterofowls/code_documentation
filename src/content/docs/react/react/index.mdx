React is a declarative, component-based JavaScript library for building user interfaces. Maintained by Meta and a large open-source community, React has become the most widely adopted UI library in the JavaScript ecosystem. React 18 introduced a new concurrent rendering engine, automatic batching, and first-class support for streaming server-side rendering.

## What is React?

React lets you compose complex UIs from small, isolated pieces of code called **components**. React manages when and how these components render, enabling highly interactive and performant web applications.

:::tip[React's Core Philosophy]
Think in components. Every piece of UI — a button, a sidebar, an entire page — is a component. Components are just JavaScript functions that return markup.
:::

### The Component Model

```tsx
// A minimal React component
function Greeting({ name }: { name: string }) {
  return <h1>Hello, {name}!</h1>;
}

// Usage
<Greeting name="World" />
```

**Rendered output:**
```
Hello, World!
```

React components follow these principles:

- **Declarative** — you describe *what* the UI should look like, not *how* to manipulate the DOM
- **Composable** — large UIs are built by composing many small components
- **Unidirectional data flow** — data flows down from parent to child via props
- **Reactive** — the UI automatically updates when state or props change

## The Virtual DOM

React maintains an in-memory representation of the real DOM called the **Virtual DOM**. When state changes, React:

<Steps>
1. Re-renders the affected component in the Virtual DOM
2. Performs a **diffing** algorithm to compare old and new Virtual DOM trees
3. Computes the minimal set of real DOM mutations needed
4. Applies only those changes to the actual DOM (**reconciliation**)
</Steps>

This approach avoids expensive direct DOM manipulation and batches updates efficiently.

:::caution[Virtual DOM is an Implementation Detail]
The Virtual DOM is not the source of React's performance. Poorly written React can still be slow. The real benefit is the **declarative programming model** — the Virtual DOM is what makes the abstraction possible.
:::

## React 18 Concurrent Rendering

React 18's most significant change is the introduction of the **concurrent renderer**, which allows React to prepare multiple versions of the UI simultaneously, pause and resume work, and prioritize urgent updates.

### Automatic Batching

Before React 18, state updates inside timeouts, promises, or native event handlers were not batched:

```tsx
// React 17 — caused 2 separate renders
setTimeout(() => {
  setCount(c => c + 1); // render 1
  setFlag(f => !f);     // render 2
}, 1000);

// React 18 — automatically batched into 1 render
setTimeout(() => {
  setCount(c => c + 1);
  setFlag(f => !f);
  // single render!
}, 1000);
```

### Transitions

The `useTransition` hook lets you mark non-urgent state updates so React can keep the UI responsive:

```tsx
import { useState, useTransition } from 'react';

function SearchPage() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState<string[]>([]);
  const [isPending, startTransition] = useTransition();

  const handleSearch = (value: string) => {
    setQuery(value); // urgent: update input immediately

    startTransition(() => {
      // non-urgent: heavy computation deferred
      setResults(performExpensiveSearch(value));
    });
  };

  return (
    <div>
      <input value={query} onChange={e => handleSearch(e.target.value)} />
      {isPending ? <p>Loading results…</p> : <ResultsList results={results} />}
    </div>
  );
}
```

### Suspense for Data Fetching

React 18 extends `Suspense` beyond code splitting to data fetching, enabling declarative loading states:

```tsx
import { Suspense } from 'react';
import { UserProfile } from './UserProfile';

function App() {
  return (
    <Suspense fallback={<p>Loading profile…</p>}>
      <UserProfile userId="42" />
    </Suspense>
  );
}
```

### Concurrent Features Summary

| Feature | Hook / API | Purpose |
|---|---|---|
| Concurrent rendering | Enabled by `createRoot` | Interruptible rendering |
| Automatic batching | Built-in | Fewer renders |
| Transitions | `useTransition` | Deprioritise slow updates |
| Deferred values | `useDeferredValue` | Debounce-like derived state |
| Streaming SSR | `renderToPipeableStream` | Faster time to first byte |
| Selective hydration | `hydrateRoot` | Prioritise interactive parts |

## Your First React 18 App

```tsx
import { createRoot } from 'react-dom/client';
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
      <button onClick={() => setCount(c => c - 1)}>Decrement</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

const root = createRoot(document.getElementById('root')!);
root.render(<Counter />);
```

**Rendered output:**
```
Count: 0
[ Increment ] [ Decrement ] [ Reset ]
```

:::tip[React 18 Requires createRoot]
Always use `createRoot` instead of the legacy `ReactDOM.render`. The concurrent renderer is only enabled with `createRoot`. Using the legacy API opts you out of all React 18 improvements.
:::

## When to Choose React

React is an excellent choice when:

- Your team already knows JavaScript and TypeScript deeply
- You need the largest possible ecosystem of third-party libraries
- You're building a complex, long-lived SPA or full-stack app with Next.js / Remix
- You require fine-grained control over rendering behavior
- You want concurrent features for complex data-heavy UIs

:::caution[React is a Library, Not a Framework]
React only handles the view layer. You'll need to choose solutions for routing (React Router, TanStack Router), data fetching (React Query, SWR), forms (React Hook Form), and state (Zustand, Jotai, Redux Toolkit). This is a feature — not a bug — but it requires decision-making up front.
:::

## React 18 Upgrade Path

```bash
npm install react@18 react-dom@18
npm install --save-dev @types/react@18 @types/react-dom@18
```

Update your entry point:

```tsx
// Before (React 17)
import ReactDOM from 'react-dom';
ReactDOM.render(<App />, document.getElementById('root'));

// After (React 18)
import { createRoot } from 'react-dom/client';
const root = createRoot(document.getElementById('root')!);
root.render(<App />);
```

## Key Concepts at a Glance

<Aside type="note" title="The React Mental Model">
1. **UI = f(state)** — Your UI is a pure function of your application state
2. **State changes trigger re-renders** — React calls your component function again
3. **Props flow down** — Parent components pass data to children
4. **Events flow up** — Children communicate back via callback props
5. **Side effects live in `useEffect`** — Data fetching, subscriptions, DOM mutations
</Aside>

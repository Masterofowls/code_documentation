# Bash

Bash (Bourne Again SHell) is the most widely used Unix shell, serving as both an interactive command interpreter and a powerful scripting language. It is the default shell on most Linux distributions and was the default on macOS until Catalina (which switched to zsh).

## What is Bash?

Bash is a command language interpreter that executes commands read from standard input or from a file. It implements the POSIX shell standard and extends it with many features for interactive use and scripting.

Key capabilities:
- **Command execution** — run programs, pipelines, and redirections
- **Variables and data types** — scalars, arrays, associative arrays
- **Control flow** — if/else, case, for, while, until
- **Functions** — reusable code blocks with local variables
- **Job control** — background processes, foreground, stop/resume
- **History** — command history with search
- **Tab completion** — programmable completion for commands and arguments
- **Scripting** — full automation with error handling and signals

## Bash vs sh vs zsh vs fish

| Feature | sh (POSIX) | Bash | zsh | fish |
|---------|-----------|------|-----|------|
| Standard | POSIX.1 | Superset of POSIX | POSIX + extensions | Non-POSIX |
| Arrays | No | Indexed arrays | Indexed + assoc | Lists only |
| Associative arrays | No | Yes (4.0+) | Yes | No (use fish vars) |
| `[[` test | No | Yes | Yes | No |
| Here-strings | No | Yes (`&lt;&lt;&lt;`) | Yes | No |
| Process substitution | No | Yes `&lt;()` `>()` | Yes | No |
| Brace expansion | No | Yes | Yes | Yes |
| Globbing | Basic | Extended | Extended+zle | Extended |
| Auto-cd | No | No | Yes | Yes |
| Syntax highlighting | No | No | (plugin) | Built-in |
| Web integration | No | No | No | Built-in |
| Startup file | .profile | .bashrc/.bash_profile | .zshrc | config.fish |
| Default on | Many unices | Most Linux | macOS 10.15+ | User choice |

:::tip[Use #!/usr/bin/env bash for Scripts]
Use `#!/usr/bin/env bash` instead of `#!/bin/bash` in scripts. The `env` version finds Bash in the PATH, making scripts portable to systems where Bash is not at `/bin/bash` (macOS Homebrew installs to `/usr/local/bin/bash` or `/opt/homebrew/bin/bash`).
:::

## Bash Versions

```bash
# Check your Bash version
echo $BASH_VERSION
bash --version

# macOS ships with Bash 3.2 (GPL v2) — install newer via Homebrew
brew install bash
# Then add to /etc/shells and chsh -s /opt/homebrew/bin/bash

# Common version thresholds:
# Bash 3.2 — macOS default (2006) — avoid associative arrays, coproc
# Bash 4.0 — associative arrays (declare -A), mapfile, improved globbing
# Bash 4.1 — printf %q, declare -l/-u, ${var@U}/@L for case conversion
# Bash 4.2 — lastpipe, negative array indices ${arr[-1]}
# Bash 4.3 — nameref (declare -n), improved coproc
# Bash 4.4 — ${var@Q} quoting, ${var@E} escape expansion
# Bash 5.0 — $EPOCHSECONDS, $EPOCHREALTIME, improved history, wait -n
# Bash 5.1 — ${!nameref}, loadable builtins improvements
# Bash 5.2 — assoc array improvements, ulimit improvements
```

## Interactive vs Non-Interactive Shell

```bash
# Check if the current shell is interactive
[[ $- == *i* ]] && echo "Interactive" || echo "Non-interactive"
# Or:
[[ -t 0 ]] && echo "Interactive (stdin is a terminal)"

# Interactive shell: started from a terminal (bash, bash -i)
# Non-interactive shell: runs a script (bash script.sh)
# Interactive shells read .bashrc; non-interactive ones typically don't
```

## Login vs Non-Login Shell

```bash
# Check if current shell is a login shell
shopt -q login_shell && echo "Login shell" || echo "Non-login shell"

# Login shell: bash invoked as first process after login (ssh, su -, new TTY)
# Non-login shell: bash started inside an existing session (terminal emulator, subshell)
```

## Shell Startup Files

| File | When Sourced | Purpose |
|------|-------------|---------|
| `/etc/profile` | Login shell | System-wide environment setup |
| `~/.bash_profile` | Login shell | User environment, PATH, exports |
| `~/.bash_login` | Login shell (if no `.bash_profile`) | Fallback login file |
| `~/.profile` | Login shell (if no `.bash_profile`/`.bash_login`) | sh-compatible login file |
| `~/.bashrc` | Interactive non-login shell | Aliases, functions, prompt, completions |
| `~/.bash_logout` | Login shell on exit | Cleanup on logout |
| `/etc/bash.bashrc` | Interactive shell (Debian/Ubuntu) | System-wide interactive settings |

```bash
# Standard pattern: .bash_profile sources .bashrc
# ~/.bash_profile:
if [[ -f ~/.bashrc ]]; then
  source ~/.bashrc
fi

# ~/.bashrc should guard against non-interactive sourcing:
[[ $- != *i* ]] && return  # Return early if not interactive
```

## Bash Strict Mode — set -euo pipefail

Every production script should start with:

```bash
#!/usr/bin/env bash
set -euo pipefail
```

| Flag | Meaning | Why it matters |
|------|---------|----------------|
| `-e` | Exit immediately if any command fails | Prevents silently continuing after errors |
| `-u` | Treat unset variables as errors | Catches typos in variable names |
| `-o pipefail` | Pipeline fails if any command fails | `false | true` would otherwise succeed |
| `-x` | Print each command before executing | Useful for debugging (add temporarily) |

```bash
#!/usr/bin/env bash
set -euo pipefail
# Optional: print commands for debugging
# set -x

# With -e: this exits immediately
false                    # Script stops here
echo "Never reached"    # Not executed

# With -u: this errors
echo "$UNDEFINED_VAR"   # Error: UNDEFINED_VAR: unbound variable
# Fix: use default value
echo "${UNDEFINED_VAR:-}"  # Empty string if unset (no error)

# With pipefail: this fails
cat nonexistent.txt | grep "pattern"  # Fails because cat failed
# Without pipefail: grep's exit code (0) masks cat's failure

# -e doesn't trigger in these contexts:
if false; then echo "ok"; fi  # 'false' in if condition is OK
! false                        # Negated command is OK
false || true                  # Right side of || is OK
```

:::caution[set -e Has Surprising Behavior]
`set -e` does not trigger when a failing command is part of a condition (`if`, `while`, `until`), follows `!`, or is part of `&&`/`||` chains (except the last command). This can lead to unexpected behavior. Test your scripts thoroughly and add explicit checks for critical operations.
:::

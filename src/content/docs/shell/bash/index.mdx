# Bash

**Bash** (Bourne Again Shell) is the default command interpreter on most Linux systems and was macOS's default shell until Catalina (2019). It is the lingua franca of Unix system automation.

## Bash vs sh vs zsh vs fish

| Shell  | Full Name                  | Notable Features                                                          | Default On                               |
| ------ | -------------------------- | ------------------------------------------------------------------------- | ---------------------------------------- |
| `sh`   | Bourne Shell               | POSIX standard, minimal, portable                                         | Scripting (POSIX systems)                |
| `bash` | Bourne Again Shell         | Superset of sh, arrays, `[[ ]]`, brace expansion                          | Most Linux distros, macOS (pre-Catalina) |
| `zsh`  | Z Shell                    | Bash-compatible + themes (Oh My Zsh), better completion, spell correction | macOS Catalina+                          |
| `fish` | Friendly Interactive Shell | Auto-suggestions, web-based config, NOT POSIX                             | Some Linux desktops                      |
| `dash` | Debian Almquist Shell      | Fast, minimal, POSIX-only                                                 | `/bin/sh` on Debian/Ubuntu               |
| `ksh`  | KornShell                  | Original associative arrays, `$(())` arithmetic                           | Solaris, AIX                             |

:::tip[Which to Use for Scripts]
Write scripts targeting `/bin/bash` for bash-specific features, or `/bin/sh` for POSIX portability. Never rely on `/bin/sh` being bash — on Debian/Ubuntu it's dash (much faster but lacks arrays, `[[ ]]`, and other bash features).
:::

## Checking Your Bash Version

```bash
echo $BASH_VERSION       # e.g., 5.2.15(1)-release
bash --version           # full version info
$BASH_VERSION            # variable, available inside scripts too
```

Bash 4.0+ added associative arrays and the `mapfile` builtin. macOS ships with Bash 3.2 (GPL v2). Install modern Bash via Homebrew: `brew install bash`.

## Interactive vs Non-Interactive Shell

- **Interactive shell**: reads commands from a terminal (you typing). Loads `.bashrc`, shows prompts.
- **Non-interactive shell**: runs a script. Does NOT load `.bashrc` (loads `.bash_profile` for login shells).
- **Login shell**: first shell after login. Loads `/etc/profile`, then `~/.bash_profile` (or `~/.profile`).
- **Non-login shell**: subshell or terminal emulator. Loads `~/.bashrc`.

```bash
# Test if interactive
[[ $- == *i* ]] && echo "Interactive" || echo "Non-interactive"

# Test if login shell
shopt -q login_shell && echo "Login shell" || echo "Non-login shell"
```

## Shell Startup Files

| File               | When Loaded                                  | Purpose                                |
| ------------------ | -------------------------------------------- | -------------------------------------- |
| `/etc/profile`     | Login shell (all users)                      | System-wide environment                |
| `~/.bash_profile`  | Login shell (user)                           | User PATH, exports — sources `.bashrc` |
| `~/.bashrc`        | Interactive non-login (user)                 | Aliases, functions, PS1                |
| `~/.bash_logout`   | Login shell exit                             | Cleanup on logout                      |
| `~/.profile`       | Login shell fallback (if no `.bash_profile`) | POSIX-compatible environment           |
| `/etc/bash.bashrc` | Interactive shell (system)                   | System-wide aliases/functions          |

```bash
# ~/.bash_profile — typical setup
# Load system profile
[ -f /etc/profile ] && source /etc/profile

# Load .bashrc for aliases and functions
[ -f ~/.bashrc ] && source ~/.bashrc

# User PATH
export PATH="$HOME/.local/bin:$HOME/bin:$PATH"

# Environment variables
export EDITOR=vim
export LANG=en_US.UTF-8
```

## Shebang

The shebang (`#!`) on line 1 tells the kernel which interpreter to use.

```bash
#!/bin/bash              # Use /bin/bash (may not exist on all systems)
#!/usr/bin/env bash      # Find bash in PATH (more portable, preferred)
#!/usr/bin/env -S bash --norc  # With options (env -S for GNU/Linux)
```

:::tip[Prefer #!/usr/bin/env bash]
`/bin/bash` may not exist on all systems (e.g., some BSDs, NixOS). `#!/usr/bin/env bash` finds bash in the user's PATH, which may be a newer Homebrew version on macOS.
:::

## set -euo pipefail

The most important line after the shebang:

```bash
#!/usr/bin/env bash
set -euo pipefail
```

| Flag          | Meaning                                             | Without it                               |
| ------------- | --------------------------------------------------- | ---------------------------------------- |
| `-e`          | Exit immediately on command failure (non-zero exit) | Script continues silently past errors    |
| `-u`          | Treat unset variables as errors                     | Typos silently expand to empty string    |
| `-o pipefail` | Pipeline fails if ANY command in pipe fails         | Only the last command's exit code counts |
| `-x`          | Print each command before executing (debug mode)    | Silent execution                         |

```bash
# Common combination
set -euo pipefail

# Enable trace mode for debugging (can toggle inside script)
set -x    # enable
set +x    # disable

# In subshells, set must be repeated
(set -euo pipefail; some_command)

# Combining flags
set -euxo pipefail   # everything + trace
```

:::caution[set -e Gotchas]
`set -e` has nuanced behavior:

- `if my_cmd; then ...` — set -e is suspended for the condition
- `my_cmd || true` — suppresses exit (safe pattern for allowed failures)
- `my_cmd && other_cmd` — set -e ignores failure of `my_cmd` in a compound command
- Arithmetic: `(( count++ ))` exits with code 1 when count was 0 (result is 0 = false). Use `(( count++ )) || true` or `count=$(( count + 1 ))`
  :::

# Bash

**Bash** (Bourne Again Shell) is the GNU Project's shell — the command interpreter for executing commands and writing scripts. It is the default login shell on most Linux distributions and macOS (up to Catalina). Understanding Bash is a foundational skill for any developer or system administrator working with Unix-like systems.

## What is Bash?

Bash is simultaneously:

- **A command interpreter** — reads commands interactively from the terminal.
- **A scripting language** — executes scripts stored in text files.
- **A job control system** — manages foreground/background processes.
- **A programming language** — supports variables, arrays, functions, conditionals, and loops.

## Bash vs sh vs zsh vs fish

| Feature | sh (POSIX) | Bash | zsh | fish |
|---|---|---|---|---|
| Standard | POSIX | GNU extension | — | — |
| Available by default | Most Unix | Most Linux | macOS 10.15+ | Requires install |
| Arrays | No | Yes (1D + assoc) | Yes | Yes |
| Associative arrays | No | Yes (4.0+) | Yes | Yes |
| `[[` test construct | No | Yes | Yes | No |
| Process substitution | No | Yes | Yes | No |
| History expansion | Limited | Yes | Yes | No |
| Syntax highlighting | No | No | Plugin | Yes (built-in) |
| Autosuggestions | No | No | Plugin | Yes (built-in) |
| Script portability | Max | Good | Good | Poor (different syntax) |
| POSIX compliant | Yes | Mostly | Mostly | No |

:::caution[sh vs bash]
Scripts with `#!/bin/sh` should only use POSIX-compatible syntax. Many systems link `/bin/sh` to `dash` (Debian) or `bash` in POSIX mode. If you need Bash features like arrays or `[[`, always use `#!/bin/bash` or `#!/usr/bin/env bash`.
:::

## Bash Versions

Bash has evolved significantly over the years:

| Version | Key Features |
|---|---|
| 2.0 (1996) | Programmable completion, `$'\n'` ANSI-C quoting |
| 3.0 (2004) | `=~` regex match in `[[`, associative array prototype |
| 4.0 (2009) | Associative arrays (`declare -A`), `mapfile`/`readarray`, `**` glob |
| 4.1 (2009) | `printf -v`, `read -i` |
| 4.2 (2011) | `declare -g` (global inside function), `printf %q` |
| 4.3 (2014) | `declare -n` nameref, `wait -n` |
| 4.4 (2016) | Coprocess improvements, `${param@operator}` transformations |
| 5.0 (2019) | `$EPOCHSECONDS`, `$EPOCHREALTIME`, `BASH_ARGV0` |
| 5.1 (2020) | `wait -p`, `${arr[-1]}` fixed |
| 5.2 (2022) | `GLOBSORT`, `read -d ''` improvements |

```bash
# Check your Bash version
echo $BASH_VERSION
bash --version
```

## Interactive vs Non-Interactive Shell

| Type | Description | Startup Files |
|---|---|---|
| **Interactive login** | `ssh user@host`, `su - user`, terminal on boot | `/etc/profile`, `~/.bash_profile` (or `~/.profile`) |
| **Interactive non-login** | Opening a new terminal tab | `~/.bashrc` |
| **Non-interactive** | Running a script (`bash script.sh`) | None (unless `BASH_ENV` is set) |
| **Non-interactive login** | `bash --login -c 'cmd'` | Same as login |

```bash
# Detect interactive shell
if [[ $- == *i* ]]; then
  echo "Interactive"
else
  echo "Non-interactive"
fi

# Detect login shell
if shopt -q login_shell; then
  echo "Login shell"
fi
```

## Shell Startup Files

```
/etc/profile         — System-wide, runs for all login shells
/etc/profile.d/*.sh  — System-wide snippets (sourced by /etc/profile)
~/.bash_profile      — User login shell (preferred over .profile if Bash)
~/.bash_login        — User login shell (only if .bash_profile doesn't exist)
~/.profile           — User login shell (POSIX, only if above don't exist)
~/.bashrc            — User interactive non-login shell
~/.bash_logout       — Runs on login shell exit
```

Best practice — put all customization in `~/.bashrc` and source it from `~/.bash_profile`:

```bash
# ~/.bash_profile
if [[ -f ~/.bashrc ]]; then
  source ~/.bashrc
fi
```

## Shebang Line

The shebang (`#!`) tells the OS which interpreter to use when the script is executed directly:

```bash
#!/bin/bash           # Absolute path — fast, but may fail if bash is elsewhere
#!/usr/bin/env bash   # Better: uses PATH to find bash — works across systems
#!/usr/bin/env -S bash --norc  # With options (GNU coreutils 8.30+)
```

:::tip[Prefer #!/usr/bin/env bash]
Use `#!/usr/bin/env bash` for scripts that need to run on macOS, Linux, and other Unix systems where bash may be installed at different paths (e.g., Homebrew installs to `/usr/local/bin/bash` on older macOS).
:::

## set -euo pipefail

This is the single most important thing to add to every Bash script:

```bash
#!/usr/bin/env bash
set -euo pipefail
```

| Option | Meaning |
|---|---|
| `-e` | Exit immediately if any command returns a non-zero status |
| `-u` | Treat unset variables as errors (prevents silent bugs from typos) |
| `-o pipefail` | Pipeline fails if any component fails (not just the last) |
| `-x` | Print each command before executing (useful for debugging) |

```bash
# -e catches errors:
set -e
cp nonexistent.txt /tmp/  # Script exits here on failure

# -u catches typos:
set -u
echo $DATABSE_URL  # Error: DATABSE_URL: unbound variable (caught a typo!)

# -o pipefail catches pipe failures:
set -o pipefail
grep 'pattern' huge_file.txt | head -1  # Fails if grep fails
# Without pipefail: only head's exit code is checked (which is 0)

# Common exception: allow a command to fail without exiting
set -e
command_that_might_fail || true   # Explicitly ignore failure
command_that_might_fail || echo "Failed, continuing anyway"

# Check exit code manually
if ! command_that_might_fail; then
  echo "Command failed"
fi
```

:::caution[set -e Gotchas]
`set -e` does NOT exit when:
- A command is part of an `if` condition, `while` condition, or `until` condition.
- A command is on the left side of `&&` or `||`.
- A command is in a negated form (`! command`).
- A command is a pipeline and the failure is not in the last command (use `pipefail` to fix this).
:::

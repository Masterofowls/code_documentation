Blueprint is Unreal Engine's visual scripting system that allows you to create game logic without writing C++ code. Blueprints use a node-based interface where you connect nodes to define behavior, making it accessible to designers and artists while remaining powerful enough for complex game systems.

## Why Blueprint?

- **Visual programming** — see data flow and logic without text code
- **Rapid prototyping** — iterate quickly without recompiling C++
- **Designer-friendly** — artists and designers can create game logic independently
- **Full UE5 integration** — access the entire Unreal API through Blueprint nodes
- **C++ interop** — call C++ functions from Blueprint and vice versa

## Blueprint Fundamentals

Blueprint scripts are composed of:

- **Nodes** — functions, events, variables, and operators
- **Pins** — execution pins (white) and data pins (colored by type)
- **Wires** — connections between pins that define data flow and execution order
- **Variables** — typed data stored in the Blueprint class

```
// Example Blueprint logic (pseudo-representation):
Event BeginPlay
  → Set Health = 100
  → Print String "Game Started!"

Event OnDamageTaken (DamageAmount: Float)
  → Health = Health - DamageAmount
  → Branch (Health <= 0)
    True  → Call Die()
    False → Call UpdateHealthBar()
```

:::tip[Blueprint vs C++]
Start with Blueprint for game logic and rapid prototyping. Move performance-critical code (physics, AI pathfinding, rendering) to C++ when needed. The two systems interoperate seamlessly in Unreal Engine 5.
:::

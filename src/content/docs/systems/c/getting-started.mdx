# Getting Started with C

This guide covers the practical foundations: how to compile C programs, structure projects with Make and CMake, and master data types, operators, and type conversion rules.

## Compiler Command Line

<Tabs>
  <TabItem label="GCC">
```bash
# Basic compile and link
gcc hello.c -o hello

# Recommended flags for development
gcc -std=c11 -Wall -Wextra -Wpedantic -g hello.c -o hello

# Optimized release build
gcc -std=c11 -O2 -DNDEBUG hello.c -o hello

# Compile to object file only (no linking)
gcc -std=c11 -c source.c -o source.o

# Link multiple object files
gcc main.o utils.o math.o -o program

# With math library and AddressSanitizer
gcc -std=c11 -Wall -g -fsanitize=address,undefined hello.c -o hello -lm
```
  </TabItem>
  <TabItem label="Clang">
```bash
# Basic compile
clang hello.c -o hello

# Development flags (Clang has slightly better diagnostics)
clang -std=c11 -Wall -Wextra -Wpedantic -g hello.c -o hello

# Emit LLVM IR for inspection
clang -std=c11 -S -emit-llvm hello.c -o hello.ll

# Static analysis
clang --analyze hello.c

# With sanitizers (recommended during development)
clang -std=c11 -g -fsanitize=address,undefined,memory hello.c -o hello
```
  </TabItem>
  <TabItem label="MSVC (cl.exe)">
```bash
# Basic compile (run in Developer Command Prompt)
cl hello.c /Fe:hello.exe

# With warnings and C11
cl /std:c11 /W4 /WX hello.c /Fe:hello.exe

# Debug build
cl /std:c11 /W4 /Zi /Od hello.c /Fe:hello_debug.exe

# Release build
cl /std:c11 /W4 /O2 /DNDEBUG hello.c /Fe:hello.exe
```

### Key Compiler Flags Reference

| Flag | Purpose |
|---|---|
| `-std=c11` | Enable C11 standard (also `c99`, `c17`, `c23`) |
| `-Wall` | Enable most common warnings |
| `-Wextra` | Enable additional warnings not in `-Wall` |
| `-Wpedantic` | Warn on non-standard extensions |
| `-Werror` | Treat all warnings as errors |
| `-g` | Include debug symbols (DWARF) |
| `-g3` | Maximum debug info including macros |
| `-O0` | No optimization (default, fastest compile) |
| `-O1` | Light optimization |
| `-O2` | Recommended release optimization |
| `-O3` | Aggressive (may increase code size) |
| `-Os` | Optimize for size |
| `-Og` | Optimize for debugging experience |
| `-c` | Compile to object file, skip linking |
| `-o <file>` | Output filename |
| `-I<dir>` | Add include search directory |
| `-L<dir>` | Add library search directory |
| `-l<name>` | Link against library (e.g., `-lm` for libm) |
| `-D<macro>` | Define preprocessor macro |
| `-fsanitize=address` | AddressSanitizer — detects memory errors |
| `-fsanitize=undefined` | UBSanitizer — detects undefined behavior |
| `-fstack-protector-strong` | Stack smashing protection |
| `-shared` | Build shared library (`.so`) |
| `-static` | Link statically |
| `-E` | Preprocess only |
| `-S` | Compile to assembly, stop |
| `-v` | Verbose — show all compilation steps |

## Makefile Basics

Make is the traditional build tool for C projects. A `Makefile` defines rules for building targets from sources.

```makefile
# Makefile for a simple C project

# Variables
CC      := gcc
CFLAGS  := -std=c11 -Wall -Wextra -Wpedantic -g
LDFLAGS :=
LDLIBS  := -lm

# Source and object files
SRCS    := main.c utils.c math_helpers.c
OBJS    := $(SRCS:.c=.o)
TARGET  := myprogram

# Default target (first target is the default)
.PHONY: all clean run debug release

all: $(TARGET)

# Linking rule
$(TARGET): $(OBJS)
	$(CC) $(LDFLAGS) $^ -o $@ $(LDLIBS)

# Pattern rule: compile .c to .o
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

# Convenience targets
run: all
	./$(TARGET)

debug: CFLAGS += -fsanitize=address,undefined
debug: all

release: CFLAGS := -std=c11 -O2 -DNDEBUG -Wall
release: all

clean:
	rm -f $(OBJS) $(TARGET)

# Auto-generated dependencies (advanced)
DEPS := $(OBJS:.o=.d)
-include $(DEPS)
%.o: %.c
	$(CC) $(CFLAGS) -MMD -MP -c $< -o $@
```

:::tip[Makefile Tabs]
Makefile recipes **must** be indented with a real `Tab` character, not spaces. Most editors have an option to show whitespace — use it to avoid mysterious make errors.
:::

### Common Make Variables

| Variable | Meaning |
|---|---|
| `$@` | The target filename |
| `$<` | The first prerequisite |
| `$^` | All prerequisites (deduplicated) |
| `$*` | The stem of the pattern match |
| `$(CC)` | C compiler (default: `cc`) |
| `$(CFLAGS)` | Compiler flags |
| `$(LDFLAGS)` | Linker flags |
| `$(LDLIBS)` | Libraries to link |

## CMake for C Projects

CMake is the de facto standard for cross-platform builds. For C projects:

```cmake
cmake_minimum_required(VERSION 3.20)
project(MyProject VERSION 1.0 LANGUAGES C)

# Require C11
set(CMAKE_C_STANDARD 11)
set(CMAKE_C_STANDARD_REQUIRED ON)
set(CMAKE_C_EXTENSIONS OFF)   # Disable GNU extensions, use strict C11

# Compiler warnings
add_compile_options(
  -Wall -Wextra -Wpedantic
  $<$<CONFIG:Debug>:-g -fsanitize=address,undefined>
  $<$<CONFIG:Release>:-O2 -DNDEBUG>
)

# Executable
add_executable(myprogram
  src/main.c
  src/utils.c
  src/math_helpers.c
)

# Include directories
target_include_directories(myprogram PRIVATE include/)

# Link libraries
target_link_libraries(myprogram PRIVATE m)  # -lm

# Install rules
install(TARGETS myprogram DESTINATION bin)
```

```bash
# Build with CMake
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Debug
cmake --build .

# Release
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . --parallel
```

## Data Types

### Fundamental Integer Types

```c
#include <stdio.h>
#include <limits.h>   // CHAR_MAX, INT_MAX, LLONG_MAX, etc.
#include <stdint.h>   // int8_t, uint32_t, int64_t, etc.

int main(void) {
    // Signed integer types (platform-dependent sizes)
    char   c  = 'A';           // 1 byte, may be signed or unsigned
    short  s  = -32000;        // at least 2 bytes
    int    i  = -2147483648;   // at least 2 bytes, usually 4
    long   l  = -2147483648L;  // at least 4 bytes
    long long ll = -9223372036854775807LL;  // at least 8 bytes

    // Unsigned variants
    unsigned char  uc  = 255;
    unsigned short us  = 65535;
    unsigned int   ui  = 4294967295U;
    unsigned long  ul  = 4294967295UL;
    unsigned long long ull = 18446744073709551615ULL;

    // Floating point
    float       f  = 3.14f;           // 32-bit IEEE 754
    double      d  = 3.14159265358979; // 64-bit IEEE 754
    long double ld = 3.14159265358979323846L; // 80 or 128 bits

    // sizeof — compile-time size in bytes
    printf("sizeof(char)      = %zu\n", sizeof(char));       // always 1
    printf("sizeof(short)     = %zu\n", sizeof(short));      // >= 2
    printf("sizeof(int)       = %zu\n", sizeof(int));        // >= 2, usually 4
    printf("sizeof(long)      = %zu\n", sizeof(long));       // >= 4
    printf("sizeof(long long) = %zu\n", sizeof(long long));  // >= 8
    printf("sizeof(float)     = %zu\n", sizeof(float));      // usually 4
    printf("sizeof(double)    = %zu\n", sizeof(double));     // usually 8

    // Limits from <limits.h>
    printf("INT_MAX  = %d\n", INT_MAX);
    printf("INT_MIN  = %d\n", INT_MIN);
    printf("UINT_MAX = %u\n", UINT_MAX);
    printf("LLONG_MAX = %lld\n", LLONG_MAX);
    return 0;
}
```

### Fixed-Width Types (`<stdint.h>`)

Prefer these when you need exact sizes — portable across all platforms:

```c
#include <stdint.h>
#include <inttypes.h>  // PRId32, PRIu64 format macros

int8_t   a = -128;              // exactly 8-bit signed
uint8_t  b = 255;               // exactly 8-bit unsigned
int16_t  c = -32768;            // exactly 16-bit signed
uint16_t d = 65535;             // exactly 16-bit unsigned
int32_t  e = -2147483648;       // exactly 32-bit signed
uint32_t f = 4294967295U;       // exactly 32-bit unsigned
int64_t  g = INT64_MIN;         // exactly 64-bit signed
uint64_t h = UINT64_MAX;        // exactly 64-bit unsigned

// Fast types (at least N bits, fastest for current platform)
int_fast8_t  fast8;
uint_fast32_t fast32;

// Least types (smallest type with at least N bits)
int_least16_t least16;

// Pointer-sized integer
intptr_t  ip;    // signed, can hold any pointer
uintptr_t up;    // unsigned, can hold any pointer
ptrdiff_t pd;    // signed difference between two pointers
size_t    sz;    // result of sizeof, array indexing

// Print with inttypes.h macros (portable format strings)
printf("%" PRId32 "\n", e);   // %d equivalent for int32_t
printf("%" PRIu64 "\n", h);   // %u equivalent for uint64_t
```

### Literals and Suffixes

```c
// Integer literals
int  dec  = 42;         // decimal
int  oct  = 052;        // octal (leading 0)
int  hex  = 0x2A;       // hexadecimal
int  bin  = 0b101010;   // binary (C23 / GCC extension)

// Integer suffixes
unsigned int   u  = 42U;
long           l  = 42L;
unsigned long  ul = 42UL;
long long      ll = 42LL;
unsigned long long ull = 42ULL;

// Floating point literals
float  f1  = 3.14f;      // float (suffix f or F)
float  f2  = 3.14F;
double d1  = 3.14;       // double (no suffix)
double d2  = 3.14e10;    // scientific notation
double d3  = 0x1.8p+1;  // hex float (C99)
long double ld = 3.14L;  // long double (suffix l or L)

// Character literals
char a = 'A';
char nl = '\n';   // newline
char tab = '\t';  // tab
char bs = '\\';   // backslash
char sq = '\'';   // single quote
char null = '\0'; // null terminator
char hex_char = '\x41'; // 'A' in hex
char oct_char = '\101'; // 'A' in octal
```

## printf / scanf Format Specifiers

```c
#include <stdio.h>
#include <stdint.h>

int main(void) {
    int   i = -42;
    unsigned int u = 42;
    float  f = 3.14f;
    double d = 3.14159265358979;
    char   c = 'Z';
    char  *s = "hello";
    void  *p = &i;
    size_t sz = 42;
    long long ll = -9999999999LL;

    // Integer formats
    printf("%d\n",   i);     // signed decimal int
    printf("%i\n",   i);     // same as %d
    printf("%u\n",   u);     // unsigned decimal
    printf("%o\n",   u);     // unsigned octal
    printf("%x\n",   u);     // unsigned hex lowercase (2a)
    printf("%X\n",   u);     // unsigned hex uppercase (2A)
    printf("%#x\n",  u);     // hex with 0x prefix
    printf("%lld\n", ll);    // long long signed
    printf("%llu\n", (unsigned long long)ll); // long long unsigned
    printf("%zu\n",  sz);    // size_t
    printf("%td\n",  (ptrdiff_t)42); // ptrdiff_t

    // Float formats
    printf("%f\n",   d);     // decimal notation: 3.141593
    printf("%e\n",   d);     // scientific: 3.141593e+00
    printf("%g\n",   d);     // shorter of %f or %e
    printf("%.2f\n", d);     // 2 decimal places: 3.14
    printf("%10.3f\n", d);   // width 10, 3 decimal places
    printf("%-10.3f|\n", d); // left-aligned
    printf("%+f\n",  d);     // always show sign: +3.141593

    // Other formats
    printf("%c\n",  c);      // character
    printf("%s\n",  s);      // string
    printf("%p\n",  p);      // pointer (address in hex)
    printf("%%\n");           // literal percent sign

    // Width and padding
    printf("%05d\n",  42);   // zero-padded to width 5: 00042
    printf("%-10s|\n", s);   // left-aligned string

    // scanf reads formatted input
    int n;
    scanf("%d", &n);         // Note: & address-of operator

    return 0;
}
```

## Operators

### Arithmetic Operators

```c
int a = 10, b = 3;
int add  = a + b;    // 13
int sub  = a - b;    // 7
int mul  = a * b;    // 30
int div  = a / b;    // 3 (integer division, truncates toward zero in C99+)
int mod  = a % b;    // 1 (remainder, sign follows dividend in C99+)
int neg  = -a;       // -10 (unary negation)

// Increment / decrement
int x = 5;
int pre_inc = ++x;   // increment first, then use: pre_inc=6, x=6
int post_inc = x++;  // use first, then increment: post_inc=6, x=7
int pre_dec  = --x;  // x=6
int post_dec = x--;  // 6, x=5
```

### Bitwise Operators

```c
unsigned int a = 0b1010;  // 10
unsigned int b = 0b1100;  // 12

unsigned int AND  = a & b;   // 0b1000 = 8  (both bits 1)
unsigned int OR   = a | b;   // 0b1110 = 14 (either bit 1)
unsigned int XOR  = a ^ b;   // 0b0110 = 6  (exactly one bit 1)
unsigned int NOT  = ~a;      // bitwise complement
unsigned int LSH  = a << 2;  // 0b101000 = 40 (left shift by 2)
unsigned int RSH  = a >> 1;  // 0b0101 = 5  (right shift by 1)

// Common bit manipulation patterns
unsigned int flags = 0;
flags |=  (1u << 3);    // set bit 3
flags &= ~(1u << 3);    // clear bit 3
flags ^=  (1u << 3);    // toggle bit 3
int is_set = (flags >> 3) & 1;  // test bit 3
```

### Logical and Comparison Operators

```c
int x = 5, y = 10;

// Comparison — return int (0 or 1)
int lt  = x < y;    // 1
int le  = x <= y;   // 1
int gt  = x > y;    // 0
int ge  = x >= y;   // 0
int eq  = x == y;   // 0 (equality, not assignment!)
int ne  = x != y;   // 1

// Logical — short-circuit evaluation
int both = x > 0 && y > 0;   // 1 — AND (stops at first false)
int either = x > 100 || y > 5; // 1 — OR (stops at first true)
int not_x  = !x;               // 0 — NOT (0 if nonzero, 1 if zero)

// Ternary operator: condition ? value_if_true : value_if_false
int max = x > y ? x : y;    // 10
int abs_val = x < 0 ? -x : x; // 5

// sizeof operator (compile-time, not a function)
size_t s1 = sizeof(int);       // 4
size_t s2 = sizeof(x);         // 4 (size of variable)
size_t s3 = sizeof(int[10]);   // 40

// Comma operator (rarely used deliberately)
int a = (1, 2, 3);   // evaluates all, result is 3
```

### Assignment Operators

```c
int n = 10;
n += 5;    // n = n + 5   → 15
n -= 3;    // n = n - 3   → 12
n *= 2;    // n = n * 2   → 24
n /= 4;    // n = n / 4   → 6
n %= 4;    // n = n % 4   → 2
n <<= 1;   // n = n << 1  → 4
n >>= 1;   // n = n >> 1  → 2
n &= 3;    // n = n & 3   → 2
n |= 8;    // n = n | 8   → 10
n ^= 5;    // n = n ^ 5   → 15
```

## Type Casting and Implicit Conversions

### Implicit Integer Promotions

```c
// Integer promotion: operands smaller than int are promoted to int
char  a = 10;
short b = 20;
int   c = a + b;   // a and b promoted to int before addition

// Usual arithmetic conversions (for binary operators):
// 1. If either is long double, convert other to long double
// 2. If either is double, convert other to double
// 3. If either is float, convert other to float
// 4. Perform integer promotions on both
// 5. If types still differ, convert smaller to larger
// 6. If signed vs unsigned same size, convert signed to unsigned

unsigned int  u = 10;
int           i = -1;
int result = (i < u) ? 1 : 0;  // WARNING: -1 converted to UINT_MAX > 10, result = 0!
```

### Explicit Casts

```c
double pi = 3.14159;
int    truncated = (int)pi;         // 3 — explicit cast, discard fractional part
float  f = (float)pi;              // narrowing: double → float
double d = (double)truncated;      // widening: int → double

// Pointer casts
int  x = 42;
void *vp = &x;                     // implicit: any pointer → void*
int  *ip = (int *)vp;             // explicit: void* → typed pointer (required)

// Type punning via union (the only defined way in C)
union {
    float  f;
    uint32_t u;
} pun;
pun.f = 3.14f;
printf("bits: %08X\n", pun.u);    // defined in C, UB in C++
```

:::caution[Signed/Unsigned Mixing]
Mixing signed and unsigned integers in comparisons is a frequent source of bugs. Enable `-Wsign-compare` (included in `-Wextra`) to catch these. Use explicit casts or `(size_t)` comparisons when iterating with a `size_t` loop variable.
:::

## Integer Overflow

```c
#include <limits.h>
#include <stdint.h>

// Signed integer overflow is UNDEFINED BEHAVIOR in C
int x = INT_MAX;
int overflow = x + 1;   // ❌ UB — compiler may assume this never happens

// Unsigned integer overflow is DEFINED: wraps modulo 2^N
unsigned int u = UINT_MAX;
unsigned int wrapped = u + 1;   // ✅ defined: 0

// Safe signed overflow check
int safe_add(int a, int b) {
    if ((b > 0 && a > INT_MAX - b) ||
        (b < 0 && a < INT_MIN - b)) {
        // would overflow
        return -1; // error
    }
    return a + b;
}

// C23: use _BitInt(N) for exact-width integer with defined overflow
// Or use compiler builtins:
int result;
if (__builtin_add_overflow(x, 1, &result)) {
    // overflow detected
}
```

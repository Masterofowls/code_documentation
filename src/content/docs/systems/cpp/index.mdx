# C++ Programming Language

C++ is a general-purpose programming language that extends C with object-oriented, generic, and functional programming features — all while maintaining zero-overhead abstraction. It is the language of choice for systems programming, game engines, high-frequency trading, compilers, and performance-critical applications worldwide.

## History and Philosophy

C++ was created by **Bjarne Stroustrup** at Bell Labs, beginning in 1979 as "C with Classes." It was renamed C++ in 1983. Stroustrup's guiding principle — which remains central today — is:

> *"What you don't use, you don't pay for. What you do use, you couldn't hand-code any better."*

This is the **zero-overhead abstraction** principle: abstractions like classes, templates, and RAII compile away to machine code as efficient as the equivalent hand-written C, with no hidden runtime cost.

## C++ Standards Timeline

| Standard | Year | Headline Features |
|----------|------|-------------------|
| **C++98** | 1998 | First ISO standard; templates; STL; exceptions; RTTI |
| **C++03** | 2003 | Bug fixes and clarifications to C++98 |
| **C++11** | 2011 | *Revolution*: move semantics; lambdas; `auto`; range-for; smart pointers; `nullptr`; variadic templates; `constexpr`; threads; `static_assert` |
| **C++14** | 2014 | Generic lambdas; `std::make_unique`; binary literals; relaxed `constexpr`; `[[deprecated]]` |
| **C++17** | 2017 | Structured bindings; `if constexpr`; `std::optional/variant/any`; `std::filesystem`; parallel algorithms; CTAD; fold expressions |
| **C++20** | 2020 | Concepts; Ranges; Coroutines; Modules; `std::format`; `std::span`; spaceship `&lt;=>` |
| **C++23** | 2023 | `std::expected`; `std::stacktrace`; `std::flat_map`; `std::print`; `if consteval` |

:::tip[Which Standard to Use]
Use **C++20** for new projects where compiler support allows. It offers the most significant quality-of-life improvements (concepts, ranges, format). Use **C++17** for teams that need broader compiler support. Avoid C++03/C++98 in new code.
:::

## C++ vs Other Languages

| Feature | C++ | C | Rust | Java |
|---------|-----|---|------|------|
| Memory management | Manual + RAII | Manual | Ownership/borrow | GC |
| Zero-cost abstractions | Yes | N/A | Yes | No |
| Undefined behavior | Yes | Yes | Minimal (safe) | No |
| Compile speed | Slow | Fast | Slow | Moderate |
| Binary size | Small | Tiny | Small | Large (JVM) |
| OOP | Yes | No | Trait-based | Yes |
| Generics | Templates | No | Generics + Traits | Generics |
| ABI stability | Fragile | Stable | Unstable | JVM stable |
| Embedded use | Yes | Yes | Yes (growing) | No |
| Game engines | Yes (dominant) | Partial | No | No |
| Learning curve | Very steep | Moderate | Very steep | Moderate |

## The Compilation Model

C++ shares C's compilation pipeline but adds important complexity:

### Name Mangling

Because C++ supports function overloading and namespaces, the linker needs unique names for each function. The compiler encodes the full signature into the symbol name:

```cpp
// C++ source
namespace mylib {
    int add(int a, int b);
    double add(double a, double b);
}

// Mangled names (GCC/Clang Itanium ABI):
// _ZN5mylib3addEii      — mylib::add(int, int)
// _ZN5mylib3addEdd      — mylib::add(double, double)

// extern "C" disables name mangling — for C interop
extern "C" {
    int c_compatible_function(int x);  // symbol: c_compatible_function
}
```

### Templates Must Be in Headers

```cpp
// WRONG — linker error if called from another .cpp
// math.h:   template<typename T> T square(T x);
// math.cpp: template<typename T> T square(T x) { return x * x; }

// CORRECT — full definition in header
// math.h:
template<typename T>
T square(T x) { return x * x; }

// Exception: explicit instantiation declarations
// math.h:   extern template int square<int>(int);
// math.cpp: template int square<int>(int);   // explicit instantiation
```

### ABI Compatibility

C++ does not have a stable ABI between compilers or even compiler versions. Mixing object files from GCC 12 and GCC 13 may cause crashes. The Itanium C++ ABI is the standard on Linux/macOS; MSVC has its own ABI.

```cpp
// ABI concerns:
// - std::string layout changed between GCC 4.x and 5.x (COW vs SSO)
// - Virtual table layout, RTTI, exception handling tables
// - Use C API (extern "C") at library boundaries for stability
// - Or use header-only libraries to avoid ABI issues entirely
```

## Undefined Behavior in C++

C++ inherits C's UB and adds more:

```cpp
#include <iostream>
#include <vector>

void ub_examples()
{
    // Signed integer overflow (same as C)
    int x = INT_MAX;
    // x + 1 is UB

    // Null pointer dereference
    int *p = nullptr;
    // *p = 5;   // UB

    // Out-of-bounds access
    std::vector<int> v = {1, 2, 3};
    // v[10] = 99;    // UB (use v.at(10) for bounds checking)
    // int arr[3]; arr[3] = 0;  // UB

    // Use after move
    std::vector<int> a = {1, 2};
    std::vector<int> b = std::move(a);
    // a.push_back(3);  // technically valid but a is in "valid but unspecified state"

    // Pure virtual function call during construction/destruction
    // Accessing virtual member via deleted/uninitialized vtable

    // Strict aliasing violation
    float f = 3.14f;
    // int *ip = reinterpret_cast<int *>(&f);
    // *ip = 0;    // UB (except via char/unsigned char)
}
```

:::caution[Enable Sanitizers Always]
Use `-fsanitize=address,undefined` in debug/test builds. UndefinedBehaviorSanitizer (UBSan) catches signed overflow, null dereference, misaligned access, and many other UB categories. AddressSanitizer (ASan) catches memory errors. These tools are essential.
:::

## Quick Start

```cpp
// hello.cpp — modern C++20 Hello World
#include <print>    // C++23: std::print
#include <string>
#include <vector>
#include <ranges>
#include <algorithm>

int main()
{
    // Range-based for, auto type deduction
    std::vector<int> nums = {5, 2, 8, 1, 9, 3, 7, 4, 6};

    // Sort with ranges (C++20)
    std::ranges::sort(nums);

    // Filter and print (C++20 ranges + C++23 print)
    auto evens = nums | std::views::filter([](int n){ return n % 2 == 0; });

    for (int n : evens) {
        std::print("{} ", n);   // C++23
    }
    std::println("");   // prints newline

    // Or with C++17 formatting
    for (int n : evens) {
        std::cout << n << ' ';
    }
    std::cout << '\n';

    return 0;
}
```

```bash
g++ -std=c++23 -Wall -Wextra -O2 hello.cpp -o hello
./hello
# 2 4 6 8
```

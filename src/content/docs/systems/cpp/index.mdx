# C++ Programming Language

C++ is a general-purpose, systems-level language that extends C with object-oriented programming, generic programming via templates, and a rich standard library. It is the language of choice for applications where performance, expressiveness, and control must all coexist.

## What Is C++?

C++ was created by **Bjarne Stroustrup** at Bell Labs starting in 1979 (originally called "C with Classes"), with the first commercial release in 1985. His guiding philosophy was:

> "C++ is designed to be a statically typed, general-purpose language that is as efficient and portable as C; it is also designed to be a better C, to support data abstraction, to support object-oriented programming, and to support generic programming."

C++ is:
- **Compiled** — translated directly to machine code, not interpreted
- **Statically typed** — types are checked at compile time
- **Multi-paradigm** — supports procedural, object-oriented, functional, and generic styles
- **Zero-overhead abstraction** — high-level features compile to code as efficient as hand-written low-level code
- **Backward compatible with C** — almost all C code compiles as C++ (with minor exceptions)

## Standards Timeline

| Standard | Year | Key Features |
|----------|------|-------------|
| C++98 | 1998 | First ISO standard: templates, STL, exceptions, namespaces, RTTI |
| C++03 | 2003 | Bug-fix release for C++98 |
| C++11 | 2011 | **The Revolution**: move semantics, lambdas, auto, nullptr, range-for, smart pointers, threads |
| C++14 | 2014 | Refinements: generic lambdas, constexpr relaxation, `make_unique` |
| C++17 | 2017 | Structured bindings, `if constexpr`, `std::optional`, `std::variant`, `std::filesystem`, parallel algorithms |
| C++20 | 2020 | Concepts, ranges, coroutines, modules, `std::format`, `std::span`, `<=>` |
| C++23 | 2023 | `std::expected`, `std::stacktrace`, `std::flat_map`, `std::print`, `std::mdspan` |
| C++26 | ~2026 | In progress: `std::execution`, reflection, contracts |

:::tip[Which Standard to Target?]
For new projects in 2024: target **C++20** (broad compiler support, concepts, ranges, `std::format`). For maximum compatibility on embedded systems or older toolchains, **C++17** is an excellent baseline. C++23 is gaining support rapidly.
:::

## Zero-Overhead Abstraction Principle

The defining philosophy of C++: **You don't pay for what you don't use. What you do use, you couldn't hand-code better.**

```cpp
#include <vector>
#include <algorithm>
#include <numeric>

// High-level C++: readable, safe, and just as fast as raw C
std::vector<int> v = {5, 2, 8, 1, 9, 3};
std::sort(v.begin(), v.end());
int sum = std::accumulate(v.begin(), v.end(), 0);

// The above compiles to the same or better machine code as:
int arr[] = {5, 2, 8, 1, 9, 3};
// hand-coded sort + sum...

// Virtual functions: only pay for dynamic dispatch when you use virtual
class Animal {
public:
    virtual void speak() const = 0;  // virtual table lookup (~1ns cost)
    ~Animal() = default;
};

// Non-virtual: zero overhead vs free function
class Vec3 {
public:
    float x, y, z;
    float length() const { return std::sqrt(x*x + y*y + z*z); }  // inlined
};
```

## Compilation Model

C++ uses the same compilation pipeline as C (preprocessing → compilation → assembly → linking) with important additions:

### Name Mangling

C++ encodes type information into symbol names to support function overloading. The mangled name encodes the function name, namespace, and parameter types.

```cpp
// Source code
void process(int x);
void process(double x);
void process(std::string x);

// Mangled names (GCC/Clang style — ABI-specific)
// _Z7processi
// _Z7processd
// _Z7processNSt7__cxx1112basic_stringIcSt11char_traitsIcESaIcEEE
```

```bash
# Demangle symbols with c++filt
nm program | c++filt
```

### Templates in Headers

Unlike regular functions, templates must have their full definition visible at the point of instantiation — typically meaning they go in header files.

```cpp
// ✓ Correct — template definition in header
template <typename T>
T max_val(T a, T b) { return a > b ? a : b; }

// ✗ Incorrect — template declaration in header, definition in .cpp
// Will compile but fail to link: "undefined reference to max_val<int>"
```

### ABI Compatibility

C++ has no stable ABI standard. Binary code compiled by different compilers (or even different versions of the same compiler) may be incompatible.

```cpp
// The extern "C" linkage specification disables name mangling
// Required for creating C-compatible APIs from C++ code
extern "C" {
    void my_function(int x);  // C ABI: no mangling, C-callable
    int  my_other_function(const char *s, int n);
}

// Use PIMPL to hide C++ from C API consumers
// (implementation detail — not in public header)
```

## Undefined Behavior in C++

C++ inherits all of C's undefined behavior plus adds its own:

```cpp
// C++ specific UB:
struct Base { virtual void f() {} };
struct Derived : Base {};
Base *b = new Derived;
delete b;          // UB if Base doesn't have virtual destructor!

// Accessing object after its lifetime
std::string *p = new std::string("hello");
delete p;
(*p)[0];           // UB: use after free

// Signed integer overflow
int x = INT_MAX;
++x;               // UB

// Strict aliasing violation
float f = 1.0f;
int   i = *(int *)&f;  // UB! (use std::bit_cast<int>(f) in C++20)

// Data race
std::atomic<int> not_protected = 0;
// reading and writing from two threads without synchronization = UB
```

:::caution[Enable Sanitizers During Development]
```bash
clang++ -fsanitize=address,undefined -g main.cpp -o program
```
UBSan catches many undefined behaviors at runtime. ASan catches memory errors. Run your full test suite with sanitizers before every release.
:::

## First C++ Class Example

```cpp
#include <iostream>
#include <string>
#include <utility>   // std::move

class Person {
public:
    // Constructor
    Person(std::string name, int age)
        : name_(std::move(name)), age_(age) {}

    // Const member function — does not modify the object
    const std::string& name() const { return name_; }
    int age() const { return age_; }

    // Mutating member function
    void birthday() { ++age_; }

    // Operator overload for output
    friend std::ostream& operator<<(std::ostream& os, const Person& p) {
        return os << p.name_ << " (age " << p.age_ << ")";
    }

private:
    std::string name_;
    int         age_;
};

int main() {
    Person alice("Alice", 30);
    Person bob("Bob", 25);

    std::cout << alice << '\n';   // Alice (age 30)
    alice.birthday();
    std::cout << alice << '\n';   // Alice (age 31)

    // Range-based for with vector
    std::vector<Person> people = {
        {"Charlie", 22}, {"Diana", 35}, {"Eve", 28}
    };
    for (const auto& p : people) {
        std::cout << p << '\n';
    }

    return 0;
}
```

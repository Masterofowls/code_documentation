# C++ Programming Language

C++ is a general-purpose, multi-paradigm programming language designed for performance-critical systems software. It extends C with object-oriented programming, generic programming via templates, and a rich standard library — while maintaining C-compatible low-level access.

## What is C++?

Bjarne Stroustrup began developing C++ at Bell Labs in 1979, initially calling it "C with Classes." The goal was to add Simula-67's class mechanisms to C without sacrificing C's efficiency. The first public release came in 1985.

The C++ philosophy, in Stroustrup's words: **"You don't pay for what you don't use."** This zero-overhead abstraction principle means that high-level features (classes, templates, RAII) should compile to code no worse than hand-written C.

## C++ Standards Timeline

| Standard  | Year | Key Additions                                                                                                                                               |
| --------- | ---- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **C++98** | 1998 | First ISO standard; templates, STL, exceptions, namespaces, RTTI                                                                                            |
| **C++03** | 2003 | Bug fix release; no new features                                                                                                                            |
| **C++11** | 2011 | **Revolutionary** — move semantics, rvalue refs, lambdas, `auto`, range-for, smart pointers, `nullptr`, `constexpr`, variadic templates, `thread`, `atomic` |
| **C++14** | 2014 | Generic lambdas, variable templates, return type deduction, `make_unique`                                                                                   |
| **C++17** | 2017 | Structured bindings, `if constexpr`, `std::optional/variant/any`, `string_view`, `filesystem`, class template argument deduction                            |
| **C++20** | 2020 | Concepts, ranges, coroutines, modules, `std::format`, `std::span`, `<=>` operator, `jthread`, `std::bit_cast`                                               |
| **C++23** | 2024 | `std::expected`, `std::stacktrace`, `std::flat_map`, `std::mdspan`, `std::print`, `if consteval`                                                            |

:::tip[Which Standard to Target?]
For new projects in 2024: **C++20** is the sweet spot — excellent compiler support in GCC 11+, Clang 12+, and MSVC 19.28+. C++17 is the safe minimum for broad compatibility. C++23 is becoming usable but not yet fully supported everywhere.
:::

## C++ vs C vs Rust vs Java

| Feature            | C                       | C++                        | Rust                    | Java                                   |
| ------------------ | ----------------------- | -------------------------- | ----------------------- | -------------------------------------- |
| Paradigm           | Procedural              | Multi-paradigm             | Multi-paradigm          | OOP                                    |
| Memory model       | Manual                  | Manual + RAII              | Ownership/borrowing     | GC                                     |
| Null safety        | No (nullptr deref = UB) | No                         | Yes (Option<T>)         | Partial (null exists)                  |
| Undefined behavior | Many                    | Many                       | Minimal                 | Almost none                            |
| Generics           | No                      | Templates                  | Generics/Traits         | Generics (erased)                      |
| Performance        | High                    | High                       | High                    | Medium (JIT)                           |
| Startup time       | Instant                 | Instant                    | Instant                 | Slow (JVM)                             |
| Binary size        | Small                   | Medium                     | Medium                  | Large (JVM)                            |
| Concurrency        | Manual (pthreads)       | `std::thread` + atomics    | Fearless (borrow check) | `synchronized`, `java.util.concurrent` |
| Error handling     | Return codes            | Exceptions or return codes | `Result<T,E>`           | Checked/unchecked exceptions           |
| Build system       | Make/CMake              | CMake/Premake              | Cargo                   | Maven/Gradle                           |
| Ecosystem          | Vast (C libraries)      | Very large                 | Growing                 | Enormous                               |
| Learning curve     | Moderate                | Very steep                 | Steep                   | Moderate                               |
| Industry use       | OS, embedded, crypto    | Games, HPC, engines        | Systems, WebAssembly    | Enterprise, Android                    |

## Zero-Overhead Abstraction

```cpp
// C-style manual loop — maximum control
int sum_c_style(const int *arr, int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) sum += arr[i];
    return sum;
}

// C++ idiomatic with range-for — same performance after optimization
int sum_modern(const std::vector<int>& vec) {
    int sum = 0;
    for (int x : vec) sum += x;
    return sum;
}

// C++ with algorithms — can auto-vectorize
#include <numeric>
int sum_algo(const std::vector<int>& vec) {
    return std::accumulate(vec.begin(), vec.end(), 0);
}

// C++17 parallel execution — use all CPU cores
#include <execution>
int sum_parallel(const std::vector<int>& vec) {
    return std::reduce(std::execution::par_unseq, vec.begin(), vec.end(), 0);
}
```

All four produce equivalent or near-equivalent machine code with `-O2`.

```bash
# GCC: check C++ standard support
g++ --version
g++ -std=c++20 -Wall -Wextra -o test test.cpp

# Clang: with libc++ (LLVM's stdlib) or libstdc++ (GCC's)
clang++ -std=c++20 -stdlib=libc++ -Wall -Wextra -o test test.cpp

# MSVC (Visual Studio Developer Command Prompt)
cl /std:c++20 /W4 /EHsc test.cpp

# Enable all C++23 features on GCC 14
g++ -std=c++23 -Wall -Wextra -Wpedantic test.cpp -o test
```

## Compilation Model

C++ uses the same four-stage pipeline as C (preprocess → compile → assemble → link) with additional complexity:

**Name mangling:** C++ encodes type information into symbol names to support function overloading. `void foo(int)` and `void foo(double)` become different symbols.

```bash
# See mangled names
nm myapp | c++filt   # Demangle to readable names
# Or: nm myapp | head -20 to see raw mangled names
```

**Templates in headers:** Function and class templates must be defined in header files (not just declared), because the compiler needs the full definition to instantiate them for each type used.

**Precompiled headers (PCH)** speed up builds by pre-compiling expensive headers:

```bash
# Precompile stdafx.h
g++ -std=c++20 stdafx.h -o stdafx.h.gch
# Now including stdafx.h uses the PCH
g++ -std=c++20 main.cpp -o main
```

## Undefined Behavior in C++

C++ has all of C's undefined behavior, plus additional sources:

```cpp
// Violating strict aliasing
float f = 3.14f;
int *p = reinterpret_cast<int*>(&f);  // UB: accessing float as int
// Use std::bit_cast (C++20) or memcpy instead

// Signed integer overflow
int x = INT_MAX;
x++;  // UB

// Accessing out of bounds (no safety with raw arrays)
int arr[10];
arr[10] = 5;  // UB

// Using moved-from objects incorrectly
std::string s = "hello";
std::string t = std::move(s);
std::cout << s;  // Not UB but s is in valid but unspecified state

// Pure virtual function called during construction/destruction
struct Base {
    Base() { init(); }  // UB if init() is pure virtual
    virtual void init() = 0;
};

// Use AddressSanitizer + UBSan to catch UB:
// g++ -fsanitize=address,undefined -g test.cpp -o test
```

## Hello World with Modern C++

```cpp
// hello.cpp — Modern C++20
#include <iostream>
#include <format>   // C++20
#include <string>
#include <vector>

// Concepts (C++20): constrain template types
template<typename T>
concept Printable = requires(T t) {
    { std::cout << t } -> std::same_as<std::ostream&>;
};

// Constexpr function — evaluated at compile time
constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}

int main() {
    // std::format (C++20) — type-safe, fast formatting
    std::string msg = std::format("Hello, {}! 5! = {}", "World", factorial(5));
    std::cout << msg << '\n';

    // Range-based for with structured binding
    std::vector<std::pair<std::string, int>> scores = {
        {"Alice", 95}, {"Bob", 87}, {"Charlie", 91}
    };

    for (const auto& [name, score] : scores) {
        std::cout << std::format("  {:10s}: {}\n", name, score);
    }

    // Lambda with capture
    int threshold = 90;
    auto high_scorers = std::count_if(scores.begin(), scores.end(),
        [threshold](const auto& p) { return p.second >= threshold; }
    );
    std::cout << std::format("{} high scorers\n", high_scorers);

    return 0;
}
```

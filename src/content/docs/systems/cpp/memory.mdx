# Memory Management in C++

Modern C++ manages memory primarily through RAII and smart pointers, making manual `new`/`delete` rare. Understanding ownership semantics is the key to writing correct, leak-free C++ programs.

## RAII Deep Dive

RAII (Resource Acquisition Is Initialization) ensures every acquired resource has an owner responsible for its release. The resource is released automatically when the owner's destructor runs.

```cpp
#include <mutex>
#include <fstream>

// RAII for any resource via template
template<typename Resource, typename Deleter>
class UniqueResource {
public:
    UniqueResource(Resource r, Deleter d) : res_(r), del_(d), valid_(true) {}
    ~UniqueResource() { if (valid_) del_(res_); }

    UniqueResource(UniqueResource&&) = default;
    UniqueResource(const UniqueResource&) = delete;

    void release() { valid_ = false; }
    Resource get() const { return res_; }
private:
    Resource res_;
    Deleter  del_;
    bool     valid_;
};

void raii_examples() {
    // std::lock_guard — RAII mutex lock
    std::mutex mtx;
    {
        std::lock_guard<std::mutex> lock(mtx);
        // critical section
    }  // mutex released automatically

    // std::fstream — RAII file handle
    {
        std::ofstream file("output.txt");
        file << "hello\n";
    }  // file.close() called automatically

    // Custom RAII for C library resource
    auto handle = UniqueResource(
        fopen("data.bin", "rb"),
        [](FILE* f) { if (f) fclose(f); }
    );
}
```

## Smart Pointers

### std::unique_ptr — Exclusive Ownership

```cpp
#include <memory>
#include <string>
#include <vector>

struct Texture {
    int id;
    std::string name;
    Texture(int i, std::string n) : id(i), name(std::move(n)) {}
    ~Texture() { /* release GPU texture */ }
};

void unique_ptr_examples() {
    // Create with make_unique (C++14) — preferred
    auto tex = std::make_unique<Texture>(1, "diffuse");
    std::cout << tex->name << '\n';

    // unique_ptr is move-only
    auto tex2 = std::move(tex);   // tex is now nullptr
    // tex2.reset();              // explicitly release

    // unique_ptr for arrays
    auto buf = std::make_unique<int[]>(1024);
    buf[0] = 42;
    // No need to delete[] — unique_ptr<T[]> calls delete[]

    // Custom deleter
    auto raw_handle = std::unique_ptr<FILE, decltype(&fclose)>(
        fopen("file.txt", "r"), fclose
    );

    // Transfer to function
    auto process = [](std::unique_ptr<Texture> t) {
        std::cout << "Processing " << t->name << '\n';
    };
    process(std::make_unique<Texture>(2, "normal"));

    // Return from function — implicit move
    auto factory = []() -> std::unique_ptr<Texture> {
        return std::make_unique<Texture>(3, "specular");
    };
    auto t3 = factory();
}
```

### std::shared_ptr — Shared Ownership

```cpp
#include <memory>

struct Node {
    int value;
    std::shared_ptr<Node> next;   // owning pointer — creates a cycle if circular!
    Node(int v) : value(v) {}
};

void shared_ptr_examples() {
    // Create with make_shared — one allocation for control block + object
    auto a = std::make_shared<int>(42);
    auto b = a;   // shared ownership — ref count = 2
    auto c = a;   // ref count = 3

    std::cout << a.use_count() << '\n';  // 3

    b.reset();    // ref count = 2
    c.reset();    // ref count = 1
    // When a goes out of scope, ref count = 0 -> object deleted

    // Thread safety of shared_ptr:
    // - The reference COUNT is thread-safe (atomic increment/decrement)
    // - The POINTED-TO object is NOT protected — need separate mutex
    // - Creating copies of shared_ptr from different threads — safe
    // - Modifying the shared_ptr itself from different threads — unsafe

    // Custom deleter
    auto custom = std::shared_ptr<int>(new int(5), [](int* p) {
        std::cout << "Custom delete\n";
        delete p;
    });
}
```

### std::weak_ptr — Non-Owning Observer

```cpp
#include <memory>

void weak_ptr_examples() {
    auto sp = std::make_shared<int>(100);
    std::weak_ptr<int> wp = sp;   // observe without owning

    // Check if object still alive
    if (auto locked = wp.lock()) {  // returns shared_ptr or empty
        std::cout << *locked << '\n';  // 100
    }

    sp.reset();   // destroy the object

    std::cout << wp.expired() << '\n';  // 1 — object is gone
    if (auto locked = wp.lock()) {
        // Never reached
    } else {
        std::cout << "Object was destroyed\n";
    }

    // Breaking circular references:
    struct Parent;
    struct Child {
        std::weak_ptr<Parent> parent;  // weak to break cycle
    };
    struct Parent {
        std::shared_ptr<Child> child;  // owns child
    };
}

// enable_shared_from_this — get shared_ptr to *this from within a class
class Widget : public std::enable_shared_from_this<Widget> {
public:
    std::shared_ptr<Widget> get_self() {
        return shared_from_this();  // safe: returns shared_ptr to this
    }
    // Do NOT: return std::shared_ptr<Widget>(this) — creates second ownership!
};
```

## Memory Orders and Atomics

```cpp
#include <atomic>
#include <thread>

std::atomic<int>  g_counter{0};
std::atomic<bool> g_ready{false};
int               g_data = 0;

// memory_order_relaxed — no ordering guarantees, just atomicity
void relaxed_increment() {
    g_counter.fetch_add(1, std::memory_order_relaxed);
}

// Release-acquire: establish happens-before relationship between threads
void producer() {
    g_data = 42;                                       // write data
    g_ready.store(true, std::memory_order_release);   // publish
}

void consumer() {
    while (!g_ready.load(std::memory_order_acquire)) {}  // wait for publish
    std::cout << g_data << '\n';  // guaranteed to see 42
}

// memory_order_seq_cst — sequential consistency (default, strongest, slowest)
// Use for compare_exchange and when you need global ordering

// compare_exchange — lock-free operations
bool push_to_stack(std::atomic<int*>& top, int* node, int* expected) {
    // Atomically: if (*top == expected) { *top = node; return true; }
    return top.compare_exchange_strong(
        expected, node,
        std::memory_order_release,
        std::memory_order_relaxed
    );
}
```

## Placement new and Allocator Control

```cpp
#include <new>
#include <memory>

void placement_new_example() {
    // Pre-allocate raw storage
    alignas(int) unsigned char storage[sizeof(int)];

    // Construct object in existing storage — no heap allocation
    int* p = new (storage) int(42);
    std::cout << *p << '\n';

    // Must call destructor explicitly (no delete — not heap allocated)
    p->~int();

    // nothrow new — returns nullptr instead of throwing
    int* q = new (std::nothrow) int[1000000000];
    if (!q) std::cout << "Allocation failed\n";
    delete[] q;  // safe even if q is nullptr
}

// Custom operator new/delete for a class
struct TrackedObject {
    static std::size_t allocation_count;

    void* operator new(std::size_t size) {
        ++allocation_count;
        return ::operator new(size);  // call global new
    }

    void operator delete(void* ptr) noexcept {
        --allocation_count;
        ::operator delete(ptr);
    }
};
std::size_t TrackedObject::allocation_count = 0;
```

## Polymorphic Memory Resources (C++17)

```cpp
#include <memory_resource>
#include <vector>
#include <array>

void pmr_example() {
    // Stack-backed buffer — no heap allocation!
    std::array<std::byte, 4096> buf;
    std::pmr::monotonic_buffer_resource pool(buf.data(), buf.size());

    // Containers using the pool
    std::pmr::vector<int> v(&pool);
    std::pmr::string s("hello", &pool);

    v.push_back(1); v.push_back(2); v.push_back(3);

    // Pool allocations are O(1) — just bump a pointer
    // All memory freed at once when pool is destroyed

    // Unsynchronized pool — efficient single-threaded pool
    std::pmr::unsynchronized_pool_resource upool;
    std::pmr::vector<std::pmr::string> vs(&upool);
    vs.emplace_back("fast allocation");

    // Chain fallback
    std::pmr::monotonic_buffer_resource small(256);
    // When small runs out, fall through to new_delete_resource()
}
```

## Exception Safety Guarantees

```cpp
#include <stdexcept>
#include <vector>

class SafeVector {
    std::vector<int> data_;
public:
    // NOTHROW guarantee: cannot throw (noexcept)
    std::size_t size() const noexcept { return data_.size(); }
    void clear() noexcept { data_.clear(); }

    // BASIC guarantee: if exception thrown, object is in valid (but unspecified) state
    void push_two(int a, int b) {
        data_.push_back(a);  // might throw std::bad_alloc
        data_.push_back(b);  // if this throws, a was added (valid but changed state)
    }

    // STRONG guarantee (commit-or-rollback): exception leaves object unchanged
    void push_two_strong(int a, int b) {
        // Modify a copy, then swap (never-fail swap)
        auto copy = data_;           // might throw — but data_ unchanged
        copy.push_back(a);           // might throw — but data_ unchanged
        copy.push_back(b);           // might throw — but data_ unchanged
        data_.swap(copy);            // noexcept — commit
    }

    // noexcept affects performance: move_if_noexcept uses move only if noexcept
    // std::vector uses this when reallocating
};

// Mark functions noexcept when they genuinely cannot throw
// Prefer noexcept on: destructors, move operations, swap, getters
// Never put noexcept on functions that might throw — std::terminate is called
```

:::tip[Smart Pointer Selection Guide]
- **`unique_ptr`**: 99% of cases. Single owner. Zero overhead vs raw pointer.
- **`shared_ptr`**: When true shared ownership is needed. Has reference count overhead.
- **`weak_ptr`**: Observer/cache that doesn't prevent deletion. Always paired with `shared_ptr`.
- **Raw pointer/reference**: Non-owning access to object whose lifetime is managed elsewhere.

Never use raw `new`/`delete` in application code. Use `make_unique` and `make_shared`.
:::

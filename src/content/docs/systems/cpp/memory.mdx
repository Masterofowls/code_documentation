# Memory Management in C++

C++ builds on C's manual memory model but adds **RAII** (Resource Acquisition Is Initialization) — a pattern that ties resource lifetime to object lifetime, making leaks and double-frees structurally impossible when used correctly.

## RAII — Resource Acquisition Is Initialization

```cpp
#include <fstream>
#include <mutex>
#include <memory>

// RAII principle: acquire in constructor, release in destructor
// Destructor is called automatically when object goes out of scope

// Example: File RAII wrapper
class FileGuard {
    FILE* fp_;
public:
    explicit FileGuard(const char* path, const char* mode)
        : fp_(fopen(path, mode))
    {
        if (!fp_) throw std::runtime_error(std::string("Cannot open: ") + path);
    }
    ~FileGuard() { if (fp_) fclose(fp_); }

    // Non-copyable (resource is unique)
    FileGuard(const FileGuard&) = delete;
    FileGuard& operator=(const FileGuard&) = delete;

    // Movable
    FileGuard(FileGuard&& o) noexcept : fp_(o.fp_) { o.fp_ = nullptr; }
    FileGuard& operator=(FileGuard&& o) noexcept {
        if (this != &o) { if (fp_) fclose(fp_); fp_ = o.fp_; o.fp_ = nullptr; }
        return *this;
    }

    FILE* get() const { return fp_; }
};

// RAII in action — resource always freed even if exception thrown
void process_file(const char* path)
{
    FileGuard fg(path, "r");   // opens file
    // ... use fg.get() ...
    // automatically closed when fg goes out of scope (even via exception)
}   // ~FileGuard() called here

// std::lock_guard is the canonical mutex RAII wrapper
void thread_safe_operation(std::mutex& mtx, int& data)
{
    std::lock_guard<std::mutex> lock(mtx);  // locks mutex
    data++;
    // mutex automatically unlocked when lock goes out of scope
}
```

## Smart Pointers

### unique_ptr — Exclusive Ownership

```cpp
#include <memory>
#include <vector>

struct Resource {
    int id;
    Resource(int i) : id(i) { printf("Resource %d created\n", id); }
    ~Resource() { printf("Resource %d destroyed\n", id); }
};

void unique_ptr_demo()
{
    // Creation — prefer make_unique (exception-safe, single allocation)
    auto p1 = std::make_unique<Resource>(1);
    std::unique_ptr<Resource> p2(new Resource(2));  // less preferred

    // Access
    p1->id = 99;
    (*p1).id = 100;

    // Cannot copy — ownership is exclusive
    // auto p3 = p1;            // ERROR: copy deleted
    auto p3 = std::move(p1);    // OK: transfer ownership; p1 is now null

    if (!p1) {
        std::cout << "p1 is null after move\n";
    }

    // Release and reset
    Resource* raw = p3.release();  // give up ownership (returns raw pointer)
    delete raw;                    // now you own it again — must delete manually

    auto p4 = std::make_unique<Resource>(4);
    p4.reset();            // destroy immediately
    p4.reset(new Resource(5));  // replace with new resource

    // Array variant
    auto arr = std::make_unique<int[]>(10);
    arr[0] = 42;   // no bounds check — same as raw array
    // arr is automatically deleted[] when out of scope

    // Custom deleter
    auto file_ptr = std::unique_ptr<FILE, decltype(&fclose)>(
        fopen("test.txt", "w"), &fclose);

    // Vector of unique_ptr (polymorphism + no copies)
    std::vector<std::unique_ptr<Resource>> resources;
    resources.push_back(std::make_unique<Resource>(10));
    resources.emplace_back(std::make_unique<Resource>(11));
}
```

### shared_ptr — Shared Ownership

```cpp
#include <memory>

void shared_ptr_demo()
{
    // make_shared allocates object + control block in ONE allocation (efficient)
    auto sp1 = std::make_shared<Resource>(1);
    std::cout << "use_count: " << sp1.use_count() << '\n';  // 1

    // Copying increments reference count — O(1), thread-safe
    auto sp2 = sp1;    // reference count = 2
    auto sp3 = sp2;    // reference count = 3
    std::cout << "use_count: " << sp1.use_count() << '\n';  // 3

    // When sp3 goes out of scope, count decrements
    sp3.reset();
    std::cout << "after reset: " << sp1.use_count() << '\n';  // 2

    // Object is destroyed when count reaches 0
}  // sp1, sp2destroyed → Resource 1 destroyed

// Control block: thread-safe reference count, weak count, deleter, allocator
// The ref count itself is atomic — ONLY the count, NOT the object
// Two threads calling sp.use_count() safely; two threads writing *sp is NOT safe

// enable_shared_from_this — get a shared_ptr to "this" from within a member
class Widget : public std::enable_shared_from_this<Widget> {
public:
    std::shared_ptr<Widget> self() {
        return shared_from_this();  // safe: returns shared_ptr sharing control block
        // return std::shared_ptr<Widget>(this);  // WRONG: creates second control block!
    }
    void schedule_callback() {
        auto self_ref = shared_from_this();  // keeps Widget alive during async op
        // post_async([self_ref](){ self_ref->do_something(); });
    }
};
```

### weak_ptr — Break Reference Cycles

```cpp
#include <memory>
#include <iostream>

struct Node {
    int value;
    std::shared_ptr<Node> next;     // strong reference
    std::weak_ptr<Node>   prev;     // weak reference — breaks cycle

    Node(int v) : value(v) {}
    ~Node() { std::cout << "Node " << value << " destroyed\n"; }
};

void weak_ptr_demo()
{
    auto n1 = std::make_shared<Node>(1);
    auto n2 = std::make_shared<Node>(2);
    auto n3 = std::make_shared<Node>(3);

    n1->next = n2;
    n2->prev = n1;  // weak_ptr — doesn't prevent n1 from being destroyed
    n2->next = n3;
    n3->prev = n2;

    // Using weak_ptr
    std::weak_ptr<Node> weak = n2;

    // Must "lock" before using (returns shared_ptr or null if expired)
    if (auto locked = weak.lock()) {
        std::cout << "n2 still alive: " << locked->value << '\n';
    }

    n1.reset();     // destroys n1 (count=0 — n2's weak_ptr to n1 doesn't count)
    n2.reset();     // destroys n2
    n3.reset();     // destroys n3

    if (weak.expired()) {
        std::cout << "n2 is gone\n";
    }
}

// Cache using weak_ptr
class Cache {
    std::unordered_map<int, std::weak_ptr<Resource>> cache_;
public:
    std::shared_ptr<Resource> get(int id) {
        auto& wp = cache_[id];
        if (auto sp = wp.lock()) {
            return sp;   // still alive
        }
        // Recreate and cache
        auto sp = std::make_shared<Resource>(id);
        wp = sp;
        return sp;
    }
};
```

## Memory Order and Atomics

```cpp
#include <atomic>
#include <thread>

// std::atomic<T> — lock-free operations on primitive types
std::atomic<int>  counter{0};
std::atomic<bool> ready{false};
std::atomic<int*> shared_ptr_atomic{nullptr};

// Memory orderings:
// relaxed  — only atomicity, no ordering constraints
// acquire  — no reads/writes AFTER this load can be reordered BEFORE it
// release  — no reads/writes BEFORE this store can be reordered AFTER it
// acq_rel  — acquire + release (for read-modify-write ops)
// seq_cst  — full sequential consistency (default, most expensive)

void producer_consumer_demo()
{
    int data = 0;

    std::thread producer([&]() {
        data = 42;  // write data
        // Release: ensures data write is visible before ready is set
        ready.store(true, std::memory_order_release);
    });

    std::thread consumer([&]() {
        // Acquire: ensures ready load happens before data read
        while (!ready.load(std::memory_order_acquire))
            ;  // spin-wait
        // data is guaranteed to be 42 here due to acquire/release pair
        std::cout << data << '\n';
    });

    producer.join();
    consumer.join();
}

// Common atomic operations
void atomic_ops_demo()
{
    std::atomic<int> x{0};

    // Load / store
    int val = x.load(std::memory_order_relaxed);
    x.store(42, std::memory_order_relaxed);

    // Read-modify-write operations
    int old_val = x.fetch_add(1);  // returns old value, increments
    x.fetch_sub(1);
    x.fetch_and(0xFF);
    x.fetch_or(0x01);
    x.fetch_xor(0x01);

    // Compare-and-swap (the foundation of lock-free algorithms)
    int expected = 42;
    bool swapped = x.compare_exchange_strong(
        expected, 100,          // if x == expected, set x = 100
        std::memory_order_acq_rel,
        std::memory_order_relaxed
    );
    if (!swapped) {
        // expected now contains the actual value of x
    }

    // compare_exchange_weak — may spuriously fail (use in loop)
    while (!x.compare_exchange_weak(expected, expected + 1,
                                     std::memory_order_relaxed)) {
        // expected updated to current value on failure — retry
    }

    // atomic_flag — guaranteed lock-free
    std::atomic_flag flag = ATOMIC_FLAG_INIT;
    bool was_set = flag.test_and_set();   // sets and returns old value
    flag.clear();
}
```

## Custom Allocators and PMR

```cpp
#include <memory_resource>
#include <vector>
#include <list>

void pmr_demo()
{
    // Polymorphic Memory Resources (C++17) — swap allocators at runtime

    // Stack buffer (avoids heap for small containers)
    alignas(std::max_align_t) char stack_buffer[4096];
    std::pmr::monotonic_buffer_resource pool{stack_buffer, sizeof(stack_buffer)};

    // Falls back to heap when stack buffer is exhausted
    std::pmr::monotonic_buffer_resource pool_with_fallback{
        stack_buffer, sizeof(stack_buffer),
        std::pmr::new_delete_resource()
    };

    // pmr containers use the pool allocator
    std::pmr::vector<int> v{&pool};
    v.reserve(100);  // all from stack_buffer — no heap allocation!
    for (int i = 0; i < 100; i++) v.push_back(i);

    std::pmr::string s{"hello world", &pool};
    std::pmr::map<int, std::pmr::string> m{&pool};

    // Synchronized pool (thread-safe)
    std::pmr::synchronized_pool_resource sync_pool;
    std::pmr::vector<double> thread_safe_vec{&sync_pool};

    // Unsynchronized pool (single-thread, faster)
    std::pmr::unsynchronized_pool_resource fast_pool;
    std::pmr::list<int> lst{&fast_pool};

    // Reset monotonic buffer — O(1) free everything
    pool.release();  // invalidates all allocations (be careful!)
}
```

## Exception Safety Guarantees

```cpp
#include <vector>
#include <stdexcept>

// Three levels of exception safety:

// NOTHROW GUARANTEE — function never throws (most reliable)
int safe_add(int a, int b) noexcept {  // noexcept = nothrow guarantee
    return a + b;
}

// STRONG GUARANTEE — if an exception is thrown, state is unchanged (commit/rollback)
class TransactionLog {
    std::vector<std::string> log_;
public:
    void add_entry(const std::string& entry)  // strong guarantee
    {
        // push_back might throw if allocation fails
        // But if it does, log_ is unchanged (vector::push_back is strongly safe)
        log_.push_back(entry);   // atomic: either succeeds fully or doesn't add
    }

    // Manual strong guarantee via copy-and-swap
    void replace_all(std::vector<std::string> new_log)
    {
        // Make a temporary copy — if an exception occurs, temp is destroyed
        // Only swap after all operations succeed
        std::swap(log_, new_log);  // swap is noexcept — safe
        // new_log (old data) destroyed here
    }
};

// BASIC GUARANTEE — if exception is thrown, no resources are leaked; state is valid
// (but may have changed — weakest useful guarantee)

// noexcept specifier — tells compiler/callers this won't throw
// Enables optimizations: move ops used instead of copy in some contexts
class BigData {
    std::vector<int> data_;
public:
    // Move constructor SHOULD be noexcept — enables vector growth optimization
    BigData(BigData&& other) noexcept
        : data_(std::move(other.data_))
    {}
    BigData& operator=(BigData&&) noexcept = default;
};

// noexcept on wrong function — program terminates if violation occurs (std::terminate)
// Do NOT put noexcept on functions that might throw!
```

:::tip[noexcept and move semantics]
Declare move constructors and move assignment operators `noexcept` whenever possible. `std::vector` uses `std::move_if_noexcept` during reallocation — if your move constructor is not `noexcept`, vector will copy instead of move (much slower). Use `static_assert(std::is_nothrow_move_constructible_v&lt;T&gt;)` to verify.
:::

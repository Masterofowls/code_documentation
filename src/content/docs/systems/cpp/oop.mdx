# Object-Oriented Programming in C++

C++ provides a full OOP model: classes with access control, constructors, destructors, operator overloading, inheritance, and virtual dispatch — all with zero-overhead abstraction.

## Classes In Depth

```cpp
#include <string>
#include <iostream>

class BankAccount {
public:  // accessible by everyone
    BankAccount(std::string owner, double initial_balance);

    void deposit(double amount);
    bool withdraw(double amount);
    double balance() const;       // const member: does not modify *this
    const std::string& owner() const;

    // mutable: can be modified even in a const member function
    mutable int access_count = 0;

protected: // accessible by this class and derived classes
    double balance_;

private:  // accessible only within this class (and friends)
    std::string owner_;

    // Friend declaration: grants access to private members
    friend class Auditor;
    friend std::ostream& operator<<(std::ostream&, const BankAccount&);
};

// Inline definition after class declaration
double BankAccount::balance() const {
    ++access_count;   // OK: mutable member
    return balance_;
}

bool BankAccount::withdraw(double amount) {
    if (amount > balance_) return false;
    balance_ -= amount;
    return true;
}

// The 'this' pointer: points to the current object
void BankAccount::deposit(double amount) {
    this->balance_ += amount;  // explicit this->; usually omitted
}
```

## Constructors

```cpp
#include <string>
#include <vector>
#include <utility>   // std::move

class Person {
public:
    // Default constructor
    Person() : name_("Unknown"), age_(0) {}

    // Parameterized constructor with member initializer list
    // Prefer initializer list over assignment in body — more efficient
    Person(std::string name, int age)
        : name_(std::move(name)), age_(age) {
        if (age_ < 0) throw std::invalid_argument("Age cannot be negative");
    }

    // Delegating constructor (C++11): call another constructor
    explicit Person(std::string name) : Person(std::move(name), 0) {}

    // explicit: prevents implicit conversion
    explicit Person(int age) : Person("Unknown", age) {}

    // Copy constructor
    Person(const Person& other)
        : name_(other.name_), age_(other.age_) {}

    // Move constructor (C++11)
    Person(Person&& other) noexcept
        : name_(std::move(other.name_)), age_(std::exchange(other.age_, 0)) {}

private:
    std::string name_;
    int         age_;
};

// Aggregate initialization (C++11/17): classes with no user-provided constructors
struct Point { double x, y, z; };
Point p = {1.0, 2.0, 3.0};          // positional
Point q = {.x = 1.0, .z = 3.0};    // C++20 designated initializers

// Brace initialization: works with any constructor, prevents narrowing
Person alice{"Alice", 30};   // same as Person("Alice", 30)
Person bob{42};              // calls explicit Person(int)
std::vector<int> v{1,2,3,4}; // initializer_list constructor
```

## Destructor and RAII

RAII (Resource Acquisition Is Initialization) ties resource lifetime to object lifetime.

```cpp
#include <cstdio>
#include <stdexcept>

// RAII wrapper for FILE*
class FileGuard {
public:
    explicit FileGuard(const char *path, const char *mode) {
        file_ = std::fopen(path, mode);
        if (!file_) throw std::runtime_error("Cannot open file");
    }

    // Destructor: automatically called when object goes out of scope
    ~FileGuard() {
        if (file_) std::fclose(file_);
    }

    FILE* get() const { return file_; }

    // Non-copyable, non-movable (simplest case)
    FileGuard(const FileGuard&)            = delete;
    FileGuard& operator=(const FileGuard&) = delete;

private:
    FILE* file_;
};

void process_file(const char *path) {
    FileGuard f(path, "r");    // file is opened
    // ... use f.get() ...
    // file is automatically closed when f goes out of scope,
    // even if an exception is thrown — no leaks!
}  // ~FileGuard() called here
```

## Rule of 0 / 3 / 5

<CardGrid>
  <Card title="Rule of Zero" icon="approve-check">
    If your class doesn't directly manage a raw resource (raw pointer, file descriptor), do nothing — let the compiler generate copy/move/destructor. Use smart pointers and RAII wrappers.
  </Card>
  <Card title="Rule of Three" icon="warning">
    If you define any of: destructor, copy constructor, copy assignment — define all three. (C++03 and earlier)
  </Card>
  <Card title="Rule of Five" icon="warning">
    If you define any of: destructor, copy constructor, copy assignment, move constructor, move assignment — define all five. (C++11+)
  </Card>
</CardGrid>

```cpp
// Rule of Five example: manually managed resource
class Buffer {
public:
    explicit Buffer(size_t size)
        : data_(new int[size]), size_(size) {}

    // 1. Destructor
    ~Buffer() { delete[] data_; }

    // 2. Copy constructor: deep copy
    Buffer(const Buffer& other)
        : data_(new int[other.size_]), size_(other.size_) {
        std::copy(other.data_, other.data_ + size_, data_);
    }

    // 3. Copy assignment: deep copy with self-assignment check
    Buffer& operator=(const Buffer& other) {
        if (this != &other) {
            int* new_data = new int[other.size_];
            std::copy(other.data_, other.data_ + other.size_, new_data);
            delete[] data_;
            data_ = new_data;
            size_ = other.size_;
        }
        return *this;
    }

    // 4. Move constructor: steal the resource
    Buffer(Buffer&& other) noexcept
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;  // leave moved-from in valid but unspecified state
        other.size_ = 0;
    }

    // 5. Move assignment
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data_;
            data_       = other.data_;
            size_       = other.size_;
            other.data_ = nullptr;
            other.size_ = 0;
        }
        return *this;
    }

    int*   data() { return data_; }
    size_t size() const { return size_; }

private:
    int*   data_;
    size_t size_;
};
```

## Copy and Move Semantics

```cpp
#include <utility>   // std::move, std::forward
#include <vector>
#include <string>

void copy_move_demo() {
    std::string s1 = "hello";

    // Copy: new string, both s1 and s2 are valid
    std::string s2 = s1;
    std::cout << s1 << " " << s2 << '\n';  // hello hello

    // Move: transfers ownership, s1 is left in valid but unspecified state
    std::string s3 = std::move(s1);
    // s1 may be empty now — don't use s1's value after move
    std::cout << s3 << '\n';               // hello

    // Move is used automatically for:
    // - Return values (NRVO/RVO: compiler may elide the copy/move entirely)
    // - Objects being inserted at end of container
    // - Temporaries (rvalue expressions)

    std::vector<std::string> v;
    v.push_back("world");          // copy from lvalue
    v.push_back(std::string("world")); // move from temporary
    v.emplace_back("world");       // construct in-place (best)

    // std::move does NOT move — it's a cast to rvalue reference
    // The actual move happens in the constructor/assignment operator
}

// Perfect forwarding: pass arguments to another function preserving value category
template <typename T, typename... Args>
T* make_object(Args&&... args) {
    return new T(std::forward<Args>(args)...);
    // forward: passes lvalue as lvalue, rvalue as rvalue
}
```

## Operator Overloading

```cpp
#include <iostream>
#include <compare>  // std::strong_ordering (C++20)
#include <cmath>

struct Vec2 {
    float x, y;

    // Arithmetic operators (non-member, symmetric)
    friend Vec2 operator+(Vec2 a, const Vec2& b) { return {a.x+b.x, a.y+b.y}; }
    friend Vec2 operator-(Vec2 a, const Vec2& b) { return {a.x-b.x, a.y-b.y}; }
    friend Vec2 operator*(Vec2 v, float s)        { return {v.x*s, v.y*s}; }
    friend Vec2 operator*(float s, Vec2 v)        { return {s*v.x, s*v.y}; }

    // Compound assignment (member)
    Vec2& operator+=(const Vec2& rhs) { x += rhs.x; y += rhs.y; return *this; }
    Vec2& operator-=(const Vec2& rhs) { x -= rhs.x; y -= rhs.y; return *this; }
    Vec2& operator*=(float s)         { x *= s; y *= s; return *this; }

    // Comparison — C++20 spaceship generates all comparison operators
    auto operator<=>(const Vec2&) const = default;  // lexicographic comparison

    // Subscript operator
    float& operator[](int i) {
        return i == 0 ? x : y;
    }
    const float& operator[](int i) const {
        return i == 0 ? x : y;
    }

    // Unary negation
    Vec2 operator-() const { return {-x, -y}; }

    // Stream output (friend non-member)
    friend std::ostream& operator<<(std::ostream& os, const Vec2& v) {
        return os << "(" << v.x << ", " << v.y << ")";
    }

    // Function call operator (functor)
    float operator()() const { return std::sqrt(x*x + y*y); }  // length

    // Explicit conversion operator
    explicit operator bool() const { return x != 0.0f || y != 0.0f; }
};

// Pre/post increment example
struct Counter {
    int value = 0;
    Counter& operator++()    { ++value; return *this; }   // pre-increment
    Counter  operator++(int) { Counter tmp = *this; ++value; return tmp; } // post-increment
};
```

## Inheritance

```cpp
#include <memory>
#include <vector>
#include <iostream>

// Abstract base class (interface)
class Shape {
public:
    // Pure virtual: subclasses MUST override
    virtual double area() const = 0;
    virtual double perimeter() const = 0;
    virtual void draw() const = 0;

    // Non-pure virtual: provides default, can be overridden
    virtual std::string name() const { return "Shape"; }

    // Virtual destructor: REQUIRED when using base class pointers
    virtual ~Shape() = default;

protected:
    static constexpr double PI = 3.14159265358979;
};

class Circle : public Shape {   // public inheritance (IS-A relationship)
public:
    explicit Circle(double r) : radius_(r) {}

    double area() const override { return PI * radius_ * radius_; }
    double perimeter() const override { return 2 * PI * radius_; }
    void draw() const override { std::cout << "○ Circle r=" << radius_ << '\n'; }
    std::string name() const override { return "Circle"; }

private:
    double radius_;
};

class Rectangle : public Shape {
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}

    double area() const override { return width_ * height_; }
    double perimeter() const override { return 2 * (width_ + height_); }
    void draw() const override { std::cout << "□ Rect " << width_ << "x" << height_ << '\n'; }

private:
    double width_, height_;
};

// Polymorphism via base class pointer
void print_shape_info(const Shape& s) {
    std::cout << s.name() << " area=" << s.area()
              << " perimeter=" << s.perimeter() << '\n';
    s.draw();
}

int main() {
    // Collection of heterogeneous shapes via smart pointers
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(5.0));
    shapes.push_back(std::make_unique<Rectangle>(4.0, 6.0));
    shapes.push_back(std::make_unique<Circle>(3.0));

    for (const auto& shape : shapes) {
        print_shape_info(*shape);  // virtual dispatch
    }
    return 0;
}
```

### Virtual Table (vtable)

Every class with virtual functions has a hidden vtable — an array of function pointers. Each object has a hidden vptr pointing to its class's vtable.

```cpp
// Memory layout of a derived class object (conceptual):
// [vptr] → vtable[area, perimeter, draw, name, ~Shape]
// [radius_]
//
// Virtual dispatch: obj.area() → *vptr[0](obj) — one extra indirection
```

## Multiple Inheritance and Diamond Problem

```cpp
class Animal {
public:
    virtual void breathe() { std::cout << "Animal breathes\n"; }
    virtual ~Animal() = default;
};

class FlyingAnimal : virtual public Animal {  // virtual inheritance
    virtual void fly() = 0;
};

class AquaticAnimal : virtual public Animal {  // virtual inheritance
    virtual void swim() = 0;
};

// Duck inherits from both: shared single Animal subobject (no diamond!)
class Duck : public FlyingAnimal, public AquaticAnimal {
public:
    void fly()   override { std::cout << "Duck flies\n"; }
    void swim()  override { std::cout << "Duck swims\n"; }
    void breathe() override { std::cout << "Duck breathes\n"; }
};
```

## Static Members

```cpp
class Counter {
public:
    Counter() { ++instance_count; }
    ~Counter() { --instance_count; }

    // Static member function: no 'this' pointer, can't access non-static members
    static int count() { return instance_count; }

    // Static constexpr: inline in header is fine
    static constexpr int kMax = 1000;

private:
    // Static member: shared across all instances
    static int instance_count;
};

// Static member must be defined outside the class (except constexpr in C++17+)
int Counter::instance_count = 0;

Counter::count();  // 0
Counter a, b, c;
Counter::count();  // 3
```

## CRTP (Curiously Recurring Template Pattern)

```cpp
// Static polymorphism: virtual dispatch at compile time with zero overhead
template <typename Derived>
class Printable {
public:
    void print() const {
        static_cast<const Derived*>(this)->print_impl();
    }

    void print_twice() const {
        print(); print();
    }
};

class Foo : public Printable<Foo> {
    friend class Printable<Foo>;
    void print_impl() const { std::cout << "Foo\n"; }
};

class Bar : public Printable<Bar> {
    friend class Printable<Bar>;
    void print_impl() const { std::cout << "Bar\n"; }
};

// No vtable, no virtual dispatch — all resolved at compile time
Foo f;
f.print_twice();   // Foo\nFoo\n (static dispatch, inlineable)
```

:::tip[CRTP vs Virtual Dispatch]
Use CRTP when the set of derived types is known at compile time and you need maximum performance (hot paths, math libraries). Use virtual dispatch when you need to store heterogeneous types in a container or when the set of subtypes isn't known at compile time.
:::

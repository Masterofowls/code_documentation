Testing asynchronous code requires special handling so Jest knows to wait for async operations before evaluating assertions. Jest supports multiple patterns: the classic `done` callback, returning Promises, and `async/await`.

## The Problem with Naive Async Tests

Without proper async handling, tests complete before async operations finish — causing false positives:

```typescript
// ❌ WRONG — test completes before fetchUser resolves
it('fetches a user', () => {
  fetchUser(1).then(user => {
    expect(user.name).toBe('Alice'); // never runs!
  });
  // Jest sees no assertion and passes the test
});
```

---

## Pattern 1: Callbacks with `done`

For legacy callback-style APIs, use the `done` parameter. Jest waits until `done()` is called (or fails if `done` is not called within `testTimeout`).

```typescript
function fetchUserCallback(id: number, callback: (err: Error | null, user?: User) => void) {
  setTimeout(() => {
    if (id <= 0) {
      callback(new Error('Invalid ID'));
    } else {
      callback(null, { id, name: 'Alice' });
    }
  }, 100);
}

it('returns a user via callback', done => {
  fetchUserCallback(1, (err, user) => {
    expect(err).toBeNull();
    expect(user?.name).toBe('Alice');
    done(); // signal completion
  });
});

it('passes an error for invalid IDs', done => {
  fetchUserCallback(-1, (err, user) => {
    expect(err).toBeInstanceOf(Error);
    expect(err?.message).toBe('Invalid ID');
    expect(user).toBeUndefined();
    done();
  });
});
```

:::caution[Always Call done in Error Paths]
If the callback path for an error takes a branch that never calls `done()`, the test will hang until it times out. Always wrap callback tests in try/catch and call `done(error)` on failure:

```typescript
it('handles errors', done => {
  fetchUserCallback(-1, (err) => {
    try {
      expect(err).toBeInstanceOf(Error);
      done();
    } catch (e) {
      done(e); // fail the test immediately with the assertion error
    }
  });
});
```
:::

## Pattern 2: Returning a Promise

Jest waits for a returned Promise to settle. If the promise rejects, the test fails.

```typescript
function fetchUser(id: number): Promise<User> {
  return fetch(`/api/users/${id}`).then(res => res.json());
}

// Return the promise — Jest will wait for it
it('fetches a user by ID', () => {
  return fetchUser(1).then(user => {
    expect(user.name).toBe('Alice');
  });
});

// Or use .resolves matcher
it('resolves with the user', () => {
  return expect(fetchUser(1)).resolves.toMatchObject({ name: 'Alice' });
});

// Or use .rejects matcher for error cases
it('rejects for a missing user', () => {
  return expect(fetchUser(999)).rejects.toThrow('Not found');
});
```

## Pattern 3: async/await (Recommended)

`async/await` is the cleanest way to write async tests. It reads like synchronous code and handles both success and error paths naturally.

```typescript
it('fetches a user successfully', async () => {
  const user = await fetchUser(1);
  expect(user.name).toBe('Alice');
  expect(user.id).toBe(1);
});

it('throws for an invalid user ID', async () => {
  await expect(fetchUser(-1)).rejects.toThrow('Invalid ID');
});

// Testing the error shape more precisely
it('throws a specific error type', async () => {
  try {
    await fetchUser(-1);
    // If we reach here, the test should fail
    expect(true).toBe(false); // or: fail('Expected an error to be thrown');
  } catch (error) {
    expect(error).toBeInstanceOf(ApiError);
    expect((error as ApiError).statusCode).toBe(400);
    expect((error as ApiError).message).toBe('Invalid ID');
  }
});
```

## `expect.assertions` — Guarding Async Error Paths

`expect.assertions(n)` verifies that exactly `n` assertions ran during the test. This is critical for async error paths — without it, a silently-swallowed exception could cause a test to pass without ever running its assertions.

```typescript
it('correctly handles a rejected promise', async () => {
  expect.assertions(2); // MUST run exactly 2 assertions

  try {
    await fetchUser(-1);
  } catch (error) {
    expect(error).toBeInstanceOf(Error);          // assertion 1
    expect((error as Error).message).toContain('Invalid'); // assertion 2
  }

  // If fetchUser(-1) doesn't throw, expect.assertions(2) fails the test
});
```

:::tip[Always Use expect.assertions for Error Paths]
Whenever your test has a `try/catch` to verify an error is thrown, always add `expect.assertions(N)` at the top. This prevents false positives when the error path is accidentally not taken.
:::

## `.resolves` and `.rejects` Matchers

These matchers let you apply any synchronous matcher to the resolved or rejected value of a Promise:

```typescript
const userPromise = fetchUser(1);

// .resolves — assert on the resolved value
await expect(userPromise).resolves.toBe('Alice');       // if resolves to string
await expect(userPromise).resolves.toMatchObject({ name: 'Alice' });
await expect(userPromise).resolves.toHaveProperty('id', 1);
await expect(userPromise).resolves.not.toBeNull();

// .rejects — assert on the rejection reason
await expect(fetchUser(-1)).rejects.toThrow('Invalid ID');
await expect(fetchUser(-1)).rejects.toBeInstanceOf(Error);
await expect(fetchUser(-1)).rejects.toHaveProperty('statusCode', 400);
```

:::caution[Always await .resolves and .rejects]
You must `await` (or `return`) expectations using `.resolves` and `.rejects`. Without `await`, Jest won't wait for the assertion and the test will pass even if it fails.

```typescript
// ❌ Missing await — always passes, never actually checks
expect(fetchUser(-1)).rejects.toThrow('Invalid ID');

// ✓ Correct
await expect(fetchUser(-1)).rejects.toThrow('Invalid ID');
```
:::

## Testing with Fake Timers

Fake timers let you test code that uses `setTimeout` or `setInterval` without actually waiting for real time to pass.

```typescript
function delay(ms: number): Promise<void> {
  return new Promise(resolve => setTimeout(resolve, ms));
}

async function retryWithBackoff<T>(
  fn: () => Promise<T>,
  retries = 3,
  baseDelay = 1000,
): Promise<T> {
  for (let attempt = 0; attempt < retries; attempt++) {
    try {
      return await fn();
    } catch (err) {
      if (attempt < retries - 1) {
        await delay(baseDelay * 2 ** attempt); // 1s, 2s, 4s
      } else {
        throw err;
      }
    }
  }
  throw new Error('Unreachable');
}
```

```typescript
describe('retryWithBackoff', () => {
  beforeEach(() => {
    jest.useFakeTimers();
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('retries on failure and eventually succeeds', async () => {
    const fn = jest
      .fn()
      .mockRejectedValueOnce(new Error('fail'))
      .mockRejectedValueOnce(new Error('fail'))
      .mockResolvedValueOnce('success');

    const promise = retryWithBackoff(fn);

    // Advance past first backoff (1000ms)
    await jest.advanceTimersByTimeAsync(1000);
    // Advance past second backoff (2000ms)
    await jest.advanceTimersByTimeAsync(2000);

    const result = await promise;
    expect(result).toBe('success');
    expect(fn).toHaveBeenCalledTimes(3);
  });

  it('throws after exhausting all retries', async () => {
    const fn = jest.fn().mockRejectedValue(new Error('always fails'));

    const promise = retryWithBackoff(fn, 2, 500);

    await jest.advanceTimersByTimeAsync(500);
    await jest.advanceTimersByTimeAsync(1000);

    await expect(promise).rejects.toThrow('always fails');
    expect(fn).toHaveBeenCalledTimes(2);
  });
});
```

## Testing `fetch`

Mock the global `fetch` function to test code that makes HTTP requests:

```typescript
// src/api/users.ts
export async function getUsers(): Promise<User[]> {
  const response = await fetch('/api/users');
  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }
  return response.json();
}
```

```typescript
// src/api/users.test.ts
global.fetch = jest.fn();

const mockFetch = global.fetch as jest.MockedFunction<typeof fetch>;

describe('getUsers', () => {
  afterEach(() => {
    mockFetch.mockClear();
  });

  it('returns an array of users', async () => {
    mockFetch.mockResolvedValueOnce({
      ok: true,
      status: 200,
      json: async () => [{ id: 1, name: 'Alice' }, { id: 2, name: 'Bob' }],
    } as Response);

    const users = await getUsers();

    expect(users).toHaveLength(2);
    expect(users[0]).toMatchObject({ name: 'Alice' });
    expect(mockFetch).toHaveBeenCalledWith('/api/users');
  });

  it('throws on a non-ok response', async () => {
    mockFetch.mockResolvedValueOnce({
      ok: false,
      status: 500,
      statusText: 'Internal Server Error',
    } as Response);

    await expect(getUsers()).rejects.toThrow('HTTP 500: Internal Server Error');
  });

  it('propagates network errors', async () => {
    mockFetch.mockRejectedValueOnce(new TypeError('Failed to fetch'));

    await expect(getUsers()).rejects.toThrow('Failed to fetch');
  });
});
```

## Testing Axios

```typescript
// src/api/products.ts
import axios from 'axios';

export async function getProduct(id: number) {
  const { data } = await axios.get(`/api/products/${id}`);
  return data;
}
```

```typescript
// src/api/products.test.ts
import axios from 'axios';
import { getProduct } from './products';

jest.mock('axios');
const mockedAxios = jest.mocked(axios);

it('fetches a product by ID', async () => {
  mockedAxios.get.mockResolvedValueOnce({
    data: { id: 1, name: 'Widget', price: 9.99 },
  });

  const product = await getProduct(1);

  expect(product).toMatchObject({ name: 'Widget', price: 9.99 });
  expect(mockedAxios.get).toHaveBeenCalledWith('/api/products/1');
});

it('handles Axios errors', async () => {
  mockedAxios.get.mockRejectedValueOnce({
    response: { status: 404, data: { message: 'Not found' } },
    isAxiosError: true,
  });

  await expect(getProduct(999)).rejects.toMatchObject({
    response: { status: 404 },
  });
});
```

## Testing Event Emitters

```typescript
import { EventEmitter } from 'events';

class DataPipeline extends EventEmitter {
  async process(input: string): Promise<void> {
    this.emit('start', { input });
    await delay(100);
    const output = input.toUpperCase();
    this.emit('complete', { input, output });
  }
}
```

```typescript
describe('DataPipeline', () => {
  it('emits start and complete events', async () => {
    const pipeline = new DataPipeline();
    const onStart = jest.fn();
    const onComplete = jest.fn();

    pipeline.on('start', onStart);
    pipeline.on('complete', onComplete);

    await pipeline.process('hello');

    expect(onStart).toHaveBeenCalledWith({ input: 'hello' });
    expect(onComplete).toHaveBeenCalledWith({ input: 'hello', output: 'HELLO' });
  });

  it('can await an event using a Promise', () => {
    const pipeline = new DataPipeline();

    const completePromise = new Promise(resolve => {
      pipeline.once('complete', resolve);
    });

    pipeline.process('world');

    return expect(completePromise).resolves.toMatchObject({ output: 'WORLD' });
  });
});
```

## Real-World Async Service Test

A complete example testing an async user registration service:

```typescript
// src/services/authService.ts
import { UserRepository } from './UserRepository';
import { EmailService } from './EmailService';
import { hashPassword } from '../utils/crypto';

export class AuthService {
  constructor(
    private readonly users: UserRepository,
    private readonly email: EmailService,
  ) {}

  async register(input: { name: string; email: string; password: string }) {
    const existing = await this.users.findByEmail(input.email);
    if (existing) {
      throw new Error('Email already registered');
    }

    const hashed = await hashPassword(input.password);
    const user = await this.users.create({
      name: input.name,
      email: input.email,
      passwordHash: hashed,
    });

    await this.email.sendWelcome({ to: user.email, name: user.name });

    return user;
  }
}
```

```typescript
// src/services/authService.test.ts
import { AuthService } from './authService';

const mockUsers = {
  findByEmail: jest.fn(),
  create: jest.fn(),
};

const mockEmail = {
  sendWelcome: jest.fn(),
};

describe('AuthService.register', () => {
  let service: AuthService;

  beforeEach(() => {
    service = new AuthService(mockUsers as any, mockEmail as any);
    jest.clearAllMocks();
  });

  it('creates a new user and sends a welcome email', async () => {
    expect.assertions(4);

    mockUsers.findByEmail.mockResolvedValueOnce(null);
    mockUsers.create.mockResolvedValueOnce({ id: 1, name: 'Alice', email: 'alice@example.com' });
    mockEmail.sendWelcome.mockResolvedValueOnce(undefined);

    const user = await service.register({
      name: 'Alice',
      email: 'alice@example.com',
      password: 'secret123',
    });

    expect(user.name).toBe('Alice');
    expect(mockUsers.create).toHaveBeenCalledWith(
      expect.objectContaining({ email: 'alice@example.com' }),
    );
    expect(mockEmail.sendWelcome).toHaveBeenCalledWith({
      to: 'alice@example.com',
      name: 'Alice',
    });
    expect(mockUsers.findByEmail).toHaveBeenCalledWith('alice@example.com');
  });

  it('throws if the email is already registered', async () => {
    expect.assertions(2);

    mockUsers.findByEmail.mockResolvedValueOnce({ id: 99, email: 'alice@example.com' });

    await expect(
      service.register({ name: 'Alice', email: 'alice@example.com', password: 'secret' }),
    ).rejects.toThrow('Email already registered');

    expect(mockUsers.create).not.toHaveBeenCalled();
  });
});
```

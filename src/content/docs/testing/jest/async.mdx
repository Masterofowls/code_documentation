Asynchronous code is everywhere in JavaScript — network requests, database queries, timers, event handlers. Jest provides multiple patterns for testing async code reliably and without introducing flakiness.

## Why Async Tests Need Special Care

The core risk with async testing is that a test can pass even when an assertion never executes:

```typescript
// ❌ DANGEROUS — this test always passes, even if the assertion fails
it('gets user data', () => {
  fetchUser(1).then(user => {
    expect(user.name).toBe('Alice'); // assertion may be skipped
  });
  // Jest sees no assertions, considers test passed
});
```

Jest provides `expect.assertions(n)` to guard against this:

```typescript
// ✅ Will fail if the callback never runs
it('gets user data', done => {
  expect.assertions(1); // exactly 1 assertion must execute
  fetchUser(1).then(user => {
    expect(user.name).toBe('Alice');
    done();
  });
});
```

## Callbacks with `done`

For legacy code or libraries that use Node.js-style callbacks, use the `done` parameter.

```typescript
import { readFileCallback } from './file-reader';

it('reads a file asynchronously', done => {
  readFileCallback('./fixture.txt', (error, data) => {
    expect(error).toBeNull();
    expect(data).toContain('expected content');
    done(); // signal test completion
  });
});

it('calls done with an error on failure', done => {
  readFileCallback('./nonexistent.txt', (error, data) => {
    expect(error).not.toBeNull();
    expect(error.code).toBe('ENOENT');
    done();
  });
});
```

If your callback might call `done` with an error argument, Jest will fail the test:

```typescript
it('handles callback errors via done(error)', done => {
  riskyOperation((error, result) => {
    if (error) {
      done(error); // this fails the test with the error
      return;
    }
    expect(result).toBeDefined();
    done();
  });
});
```

:::caution[Timeout with done]
If `done` is never called, Jest fails the test with a timeout error after 5 seconds (default `testTimeout`). Always ensure every code path calls `done()`, including error paths.
:::

## Testing Promises

You can test Promises by returning them from the test function. Jest waits for the returned Promise to resolve or reject.

```typescript
// ✅ Return the promise
it('resolves with user data', () => {
  return fetchUser(1).then(user => {
    expect(user.name).toBe('Alice');
    expect(user.id).toBe(1);
  });
});

// ✅ Return a rejection assertion
it('rejects when user not found', () => {
  return expect(fetchUser(9999)).rejects.toThrow('User not found');
});
```

### Using `.resolves` and `.rejects` Matchers

Jest provides special matchers for Promises that combine awaiting with assertions:

```typescript
// .resolves — unwraps the resolved value for further assertion
it('fetches user successfully', () => {
  return expect(fetchUser(1)).resolves.toEqual({
    id: 1,
    name: 'Alice',
    email: 'alice@example.com',
  });
});

it('resolves with the correct user name', async () => {
  await expect(fetchUser(1)).resolves.toMatchObject({ name: 'Alice' });
});

// .rejects — asserts the Promise rejects
it('rejects on invalid user id', async () => {
  await expect(fetchUser(-1)).rejects.toThrow('Invalid user ID');
});

it('rejects with a specific error type', async () => {
  await expect(fetchUser(-1)).rejects.toBeInstanceOf(ValidationError);
});

it('rejects with specific error properties', async () => {
  await expect(fetchUser(-1)).rejects.toMatchObject({
    message: 'Invalid user ID',
    statusCode: 400,
  });
});
```

:::tip[Always Await or Return]
When using `.resolves` or `.rejects`, you **must** either `await` the assertion or `return` it from the test. Without this, the assertion becomes a floating Promise that resolves after Jest has already marked the test as passed.
:::

## async/await

The cleanest way to write async tests — async/await makes async tests read like synchronous code:

```typescript
it('fetches and transforms user data', async () => {
  const user = await fetchUser(1);

  expect(user.id).toBe(1);
  expect(user.name).toBe('Alice');
  expect(user.displayName).toBe('Alice (Admin)');
  expect(user.createdAt).toBeInstanceOf(Date);
});
```

### Testing Async Error Paths with try/catch

```typescript
it('throws a specific error when the request fails', async () => {
  expect.assertions(2); // ensure the catch block runs

  try {
    await fetchUser(-1);
  } catch (error) {
    expect(error).toBeInstanceOf(ValidationError);
    expect((error as ValidationError).message).toBe('Invalid user ID');
  }
});
```

### Using expect.assertions()

`expect.assertions(n)` is critical for async error tests where you expect a code path to execute:

```typescript
// Without expect.assertions, this test could pass vacuously
// if the promise resolves instead of rejects
it('rejects invalid input', async () => {
  expect.assertions(1); // MUST run exactly one assertion

  try {
    await processInvalidData({ type: 'unknown' });
    // if we get here, the test should fail — but without
    // expect.assertions(1), it would pass silently
  } catch (error) {
    expect(error.message).toMatch(/unsupported type/i);
  }
});

// expect.hasAssertions() — at least one assertion must run
it('processes all items', async () => {
  expect.hasAssertions();

  for await (const item of asyncIterator()) {
    expect(item).toBeDefined();
  }
});
```

## Testing setTimeout and setInterval

Use Jest's fake timers for deterministic control over time-dependent code:

```typescript
import { debounce } from './utils';

describe('debounce', () => {
  beforeEach(() => jest.useFakeTimers());
  afterEach(() => jest.useRealTimers());

  it('only calls the function once after rapid successive calls', () => {
    const mockFn = jest.fn();
    const debounced = debounce(mockFn, 300);

    debounced();
    debounced();
    debounced();

    // Nothing called yet — still within debounce window
    expect(mockFn).not.toHaveBeenCalled();

    // Fast-forward time
    jest.advanceTimersByTime(300);

    // Now it should have been called exactly once
    expect(mockFn).toHaveBeenCalledTimes(1);
  });

  it('resets the timer on each call', () => {
    const mockFn = jest.fn();
    const debounced = debounce(mockFn, 300);

    debounced();
    jest.advanceTimersByTime(200); // 200ms in, still waiting
    debounced();                   // reset timer
    jest.advanceTimersByTime(200); // 400ms from first, 200ms from second — still waiting
    expect(mockFn).not.toHaveBeenCalled();

    jest.advanceTimersByTime(100); // 300ms after second call
    expect(mockFn).toHaveBeenCalledTimes(1);
  });
});
```

### Async Fake Timers

For code that uses `await` with timers, combine fake timers with async/await:

```typescript
import { retry } from './retry-utils';

describe('retry', () => {
  beforeEach(() => jest.useFakeTimers());
  afterEach(() => jest.useRealTimers());

  it('retries failed requests with exponential backoff', async () => {
    const mockFetch = jest.fn()
      .mockRejectedValueOnce(new Error('Network error'))
      .mockRejectedValueOnce(new Error('Network error'))
      .mockResolvedValue({ data: 'success' });

    const promise = retry(mockFetch, { retries: 3, delay: 1000 });

    // First attempt fails immediately
    await Promise.resolve();
    expect(mockFetch).toHaveBeenCalledTimes(1);

    // Wait for backoff delay
    jest.advanceTimersByTime(1000);
    await Promise.resolve();
    expect(mockFetch).toHaveBeenCalledTimes(2);

    // Wait for second backoff
    jest.advanceTimersByTime(2000);
    await Promise.resolve();
    expect(mockFetch).toHaveBeenCalledTimes(3);

    const result = await promise;
    expect(result.data).toBe('success');
  });
});
```

## Testing Fetch

```typescript
// Mock the global fetch
global.fetch = jest.fn();

describe('ApiClient', () => {
  afterEach(() => {
    jest.clearAllMocks();
  });

  it('fetches user data successfully', async () => {
    const mockUser = { id: 1, name: 'Alice', email: 'alice@example.com' };

    (global.fetch as jest.Mock).mockResolvedValue({
      ok: true,
      status: 200,
      json: async () => mockUser,
    });

    const user = await ApiClient.getUser(1);

    expect(global.fetch).toHaveBeenCalledWith(
      'https://api.example.com/users/1',
      expect.objectContaining({
        method: 'GET',
        headers: expect.objectContaining({
          'Content-Type': 'application/json',
        }),
      })
    );

    expect(user).toEqual(mockUser);
  });

  it('throws an ApiError when the server returns 404', async () => {
    (global.fetch as jest.Mock).mockResolvedValue({
      ok: false,
      status: 404,
      statusText: 'Not Found',
      json: async () => ({ message: 'User not found' }),
    });

    await expect(ApiClient.getUser(9999)).rejects.toThrow('User not found');
  });

  it('throws a NetworkError when fetch itself fails', async () => {
    (global.fetch as jest.Mock).mockRejectedValue(
      new TypeError('Failed to fetch')
    );

    await expect(ApiClient.getUser(1)).rejects.toBeInstanceOf(NetworkError);
  });
});
```

## Testing Axios

```typescript
import axios from 'axios';

jest.mock('axios');
const mockedAxios = axios as jest.Mocked<typeof axios>;

describe('UserService with Axios', () => {
  it('fetches user list', async () => {
    mockedAxios.get.mockResolvedValue({
      data: [{ id: 1, name: 'Alice' }, { id: 2, name: 'Bob' }],
      status: 200,
    });

    const users = await UserService.list();

    expect(mockedAxios.get).toHaveBeenCalledWith('/api/users');
    expect(users).toHaveLength(2);
    expect(users[0].name).toBe('Alice');
  });

  it('creates a new user', async () => {
    const newUser = { name: 'Carol', email: 'carol@example.com' };
    mockedAxios.post.mockResolvedValue({
      data: { id: 3, ...newUser },
      status: 201,
    });

    const created = await UserService.create(newUser);

    expect(mockedAxios.post).toHaveBeenCalledWith('/api/users', newUser);
    expect(created.id).toBe(3);
  });

  it('handles Axios error responses', async () => {
    const axiosError = {
      isAxiosError: true,
      response: { status: 422, data: { errors: ['Email is invalid'] } },
    };
    mockedAxios.post.mockRejectedValue(axiosError);

    await expect(UserService.create({ name: 'X', email: 'bad' }))
      .rejects.toMatchObject({ message: expect.stringContaining('invalid') });
  });
});
```

## Testing Event Emitters

```typescript
import { EventEmitter } from 'events';
import { FileWatcher } from './FileWatcher';

describe('FileWatcher', () => {
  it('emits a "change" event when a file is modified', done => {
    expect.assertions(2);

    const watcher = new FileWatcher('./test-fixture.txt');

    watcher.on('change', (filename, stats) => {
      expect(filename).toBe('test-fixture.txt');
      expect(stats.size).toBeGreaterThan(0);
      done();
    });

    watcher.triggerChange(); // simulate change
  });

  it('emits "change" events — promise-based', () => {
    const watcher = new FileWatcher('./test-fixture.txt');

    const changePromise = new Promise<string>(resolve => {
      watcher.once('change', filename => resolve(filename));
    });

    watcher.triggerChange();

    return expect(changePromise).resolves.toBe('test-fixture.txt');
  });
});
```

## Real-World Async Service Test

```typescript
// services/payment.service.ts
export class PaymentService {
  constructor(
    private stripe: StripeClient,
    private db: Database,
    private mailer: Mailer,
  ) {}

  async processPayment(userId: string, amount: number): Promise<PaymentResult> {
    const customer = await this.db.findUser(userId);
    if (!customer) throw new Error('Customer not found');

    const charge = await this.stripe.charge({
      amount,
      currency: 'usd',
      customer: customer.stripeId,
    });

    await this.db.createPaymentRecord({
      userId,
      chargeId: charge.id,
      amount,
      status: 'completed',
    });

    await this.mailer.send({
      to: customer.email,
      subject: 'Payment Confirmed',
      body: `Your payment of $${(amount / 100).toFixed(2)} was successful.`,
    });

    return { chargeId: charge.id, status: 'success' };
  }
}

// services/payment.service.test.ts
describe('PaymentService.processPayment', () => {
  const mockStripe = { charge: jest.fn() };
  const mockDb = { findUser: jest.fn(), createPaymentRecord: jest.fn() };
  const mockMailer = { send: jest.fn() };

  let service: PaymentService;

  beforeEach(() => {
    jest.clearAllMocks();
    service = new PaymentService(mockStripe, mockDb, mockMailer);

    mockDb.findUser.mockResolvedValue({
      id: 'user-1',
      email: 'alice@example.com',
      stripeId: 'cus_abc123',
    });
    mockStripe.charge.mockResolvedValue({ id: 'ch_xyz789' });
    mockDb.createPaymentRecord.mockResolvedValue(undefined);
    mockMailer.send.mockResolvedValue(undefined);
  });

  it('charges the correct amount and returns the charge id', async () => {
    const result = await service.processPayment('user-1', 4999);

    expect(mockStripe.charge).toHaveBeenCalledWith({
      amount: 4999,
      currency: 'usd',
      customer: 'cus_abc123',
    });
    expect(result).toEqual({ chargeId: 'ch_xyz789', status: 'success' });
  });

  it('saves the payment record to the database', async () => {
    await service.processPayment('user-1', 4999);

    expect(mockDb.createPaymentRecord).toHaveBeenCalledWith({
      userId: 'user-1',
      chargeId: 'ch_xyz789',
      amount: 4999,
      status: 'completed',
    });
  });

  it('sends a confirmation email with the correct amount', async () => {
    await service.processPayment('user-1', 4999);

    expect(mockMailer.send).toHaveBeenCalledWith({
      to: 'alice@example.com',
      subject: 'Payment Confirmed',
      body: expect.stringContaining('$49.99'),
    });
  });

  it('throws when the customer does not exist', async () => {
    mockDb.findUser.mockResolvedValue(null);

    await expect(service.processPayment('user-999', 4999))
      .rejects.toThrow('Customer not found');

    expect(mockStripe.charge).not.toHaveBeenCalled();
    expect(mockMailer.send).not.toHaveBeenCalled();
  });

  it('does not send an email if the Stripe charge fails', async () => {
    mockStripe.charge.mockRejectedValue(new Error('Card declined'));

    await expect(service.processPayment('user-1', 4999))
      .rejects.toThrow('Card declined');

    expect(mockDb.createPaymentRecord).not.toHaveBeenCalled();
    expect(mockMailer.send).not.toHaveBeenCalled();
  });
});
```

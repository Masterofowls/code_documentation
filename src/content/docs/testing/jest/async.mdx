Asynchronous code is everywhere in modern JavaScript — API calls, database queries, timers, file I/O. Jest provides multiple strategies for testing async code, and choosing the right one makes your tests more reliable and readable.

## The Problem with Async Tests

The most common async testing mistake is a test that finishes before the async operation completes:

```typescript
// ❌ This test will ALWAYS pass — even if the function rejects!
it('fetches user data', () => {
  fetchUser(1).then((user) => {
    expect(user.name).toBe('Alice'); // Never reached if test exits early
  });
});
```

Jest considers a test complete when the test function returns. For async code, you must either `return` the promise, use `async/await`, or use the `done` callback.

## Callbacks — The `done` Parameter

For legacy callback-based APIs, Jest provides the `done` parameter. The test does not complete until `done()` is called.

```typescript
it('reads a file successfully', (done) => {
  readFile('./data.txt', 'utf8', (err, content) => {
    if (err) {
      done(err); // Fail the test with the error
      return;
    }
    expect(content).toContain('Hello');
    done(); // Signal test completion
  });
});
```

:::caution[Always Pass Errors to done()]
If you call `done()` without an argument in an error branch, the test will pass even though an error occurred. Always call `done(err)` when there's an error — this fails the test with the error as the message.
:::

```typescript
// Pattern for ensuring done is called in all code paths
it('handles both success and error cases', (done) => {
  eventEmitter.on('data', (value) => {
    try {
      expect(value).toBe(42);
      done();
    } catch (error) {
      done(error); // Forward assertion errors to Jest
    }
  });

  eventEmitter.emit('data', 42);
});
```

## Promises

You can test promises by returning them from the test function. Jest waits for the returned promise to settle.

```typescript
// ✅ Return the promise
it('fetches a user by ID', () => {
  return fetchUser(1).then((user) => {
    expect(user).toEqual({ id: 1, name: 'Alice' });
  });
});

// ✅ Return the rejection assertion
it('rejects for unknown user IDs', () => {
  return expect(fetchUser(999)).rejects.toThrow('User not found');
});
```

### `.resolves` and `.rejects` Matchers

Jest provides `.resolves` and `.rejects` modifiers that unwrap promises and apply matchers to the resolved/rejected value:

```typescript
// .resolves — asserts the promise resolves with a matching value
it('resolves with user data', () => {
  return expect(fetchUser(1)).resolves.toEqual({
    id: 1,
    name: 'Alice',
    email: 'alice@example.com',
  });
});

it('resolves with a defined value', () => {
  return expect(createSession()).resolves.toBeDefined();
});

// .rejects — asserts the promise rejects
it('rejects when credentials are wrong', () => {
  return expect(login('alice', 'wrong-password')).rejects.toThrow(
    'Invalid credentials',
  );
});

it('rejects with the correct error class', () => {
  return expect(fetchUser(-1)).rejects.toThrow(ValidationError);
});

it('rejects with matching error properties', () => {
  return expect(fetchUser(-1)).rejects.toMatchObject({
    message: 'Invalid user ID',
    statusCode: 400,
  });
});
```

## async / await

`async/await` is the cleanest and most readable way to test asynchronous code in Jest. Mark the test function as `async` and `await` each async operation.

```typescript
it('creates and retrieves a user', async () => {
  const created = await userService.create({
    name: 'Alice',
    email: 'alice@example.com',
  });

  expect(created.id).toBeDefined();
  expect(created.name).toBe('Alice');

  const fetched = await userService.findById(created.id);
  expect(fetched).toEqual(created);
});
```

### Async Error Testing with try/catch

```typescript
it('throws when creating a user with a duplicate email', async () => {
  await userService.create({ email: 'alice@example.com' });

  try {
    await userService.create({ email: 'alice@example.com' });
    // If we reach here, the function didn't throw — fail the test explicitly
    throw new Error('Expected userService.create to throw, but it did not');
  } catch (error) {
    expect(error).toBeInstanceOf(ConflictError);
    expect((error as ConflictError).message).toContain('already exists');
  }
});
```

The cleaner approach using `.rejects`:

```typescript
it('throws ConflictError for duplicate email', async () => {
  await userService.create({ email: 'alice@example.com' });

  await expect(
    userService.create({ email: 'alice@example.com' }),
  ).rejects.toThrow(ConflictError);
});
```

## `expect.assertions(N)` — Guard Against Silent Passes

`expect.assertions(N)` declares that exactly N assertions must be called during the test. If the async code never executes, the test fails.

```typescript
it('handles the rejection path', async () => {
  // This test MUST execute exactly 2 assertions
  expect.assertions(2);

  try {
    await fetchUser(-1);
  } catch (error) {
    expect(error).toBeInstanceOf(ValidationError);       // assertion 1
    expect((error as ValidationError).statusCode).toBe(400); // assertion 2
  }
});
```

```typescript
// expect.hasAssertions() — at least 1 assertion must have run
it('calls the success callback', (done) => {
  expect.hasAssertions();

  service.process((result) => {
    expect(result).toBe('ok');
    done();
  });
});
```

:::tip[When to Use expect.assertions()]
Use `expect.assertions(N)` whenever you test error/rejection paths using try/catch. Without it, a bug that makes the code succeed silently will cause your catch block never to run, and the test will pass without executing any assertions.
:::

## Testing setTimeout and setInterval with Fake Timers

```typescript
describe('retry logic', () => {
  beforeEach(() => {
    jest.useFakeTimers();
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('retries 3 times with exponential backoff', async () => {
    const mockApi = jest.fn()
      .mockRejectedValueOnce(new Error('Timeout'))
      .mockRejectedValueOnce(new Error('Timeout'))
      .mockResolvedValueOnce({ data: 'success' });

    const promise = retryWithBackoff(mockApi, { maxRetries: 3 });

    // First retry fires after 1000ms
    await jest.advanceTimersByTimeAsync(1000);
    expect(mockApi).toHaveBeenCalledTimes(2);

    // Second retry fires after 2000ms
    await jest.advanceTimersByTimeAsync(2000);
    expect(mockApi).toHaveBeenCalledTimes(3);

    const result = await promise;
    expect(result).toEqual({ data: 'success' });
  });

  it('polls for status every 5 seconds', () => {
    const mockPoll = jest.fn().mockReturnValue('pending');
    startPolling(mockPoll, 5000);

    jest.advanceTimersByTime(5000);
    expect(mockPoll).toHaveBeenCalledTimes(1);

    jest.advanceTimersByTime(15000);
    expect(mockPoll).toHaveBeenCalledTimes(4);
  });
});
```

## Testing `fetch`

```typescript
// Global fetch mock using jest.fn()
global.fetch = jest.fn();

const mockFetch = global.fetch as jest.MockedFunction<typeof fetch>;

describe('apiClient', () => {
  beforeEach(() => {
    mockFetch.mockResolvedValue({
      ok: true,
      status: 200,
      json: jest.fn().mockResolvedValue({ id: 1, name: 'Alice' }),
    } as unknown as Response);
  });

  afterEach(() => {
    mockFetch.mockReset();
  });

  it('calls the correct endpoint with authorization header', async () => {
    await apiClient.getUser(1);

    expect(mockFetch).toHaveBeenCalledWith(
      '/api/users/1',
      expect.objectContaining({
        headers: expect.objectContaining({
          Authorization: 'Bearer test-token',
        }),
      }),
    );
  });

  it('returns parsed JSON on success', async () => {
    const user = await apiClient.getUser(1);
    expect(user).toEqual({ id: 1, name: 'Alice' });
  });

  it('throws ApiError when response is not ok', async () => {
    mockFetch.mockResolvedValueOnce({
      ok: false,
      status: 404,
      json: jest.fn().mockResolvedValue({ message: 'User not found' }),
    } as unknown as Response);

    await expect(apiClient.getUser(999)).rejects.toThrow(ApiError);
  });
```

## Testing Axios

```typescript
import axios from 'axios';

jest.mock('axios');

const mockedAxios = axios as jest.Mocked<typeof axios>;

describe('UserRepository', () => {
  it('fetches a user by ID', async () => {
    mockedAxios.get.mockResolvedValue({
      data: { id: 1, name: 'Alice' },
      status: 200,
    });

    const user = await userRepository.findById(1);

    expect(mockedAxios.get).toHaveBeenCalledWith('/api/users/1');
    expect(user).toEqual({ id: 1, name: 'Alice' });
  });

  it('handles Axios network errors', async () => {
    mockedAxios.get.mockRejectedValue({
      isAxiosError: true,
      response: { status: 503, data: { message: 'Service unavailable' } },
    });

    await expect(userRepository.findById(1)).rejects.toThrow('Service unavailable');
  });

  it('handles POST requests', async () => {
    const newUser = { name: 'Bob', email: 'bob@example.com' };

    mockedAxios.post.mockResolvedValue({
      data: { id: 2, ...newUser },
      status: 201,
    });

    const created = await userRepository.create(newUser);

    expect(mockedAxios.post).toHaveBeenCalledWith('/api/users', newUser);
    expect(created.id).toBe(2);
  });
});
```

## Testing Event Emitters

```typescript
import { EventEmitter } from 'events';

class DataStream extends EventEmitter {
  async start() {
    // Emits 'data' events as it processes
    for (const chunk of await fetchChunks()) {
      this.emit('data', chunk);
    }
    this.emit('end');
  }
}

describe('DataStream', () => {
  it('emits data events for each chunk', (done) => {
    const stream = new DataStream();
    const receivedData: string[] = [];

    stream.on('data', (chunk) => {
      receivedData.push(chunk);
    });

    stream.on('end', () => {
      try {
        expect(receivedData).toHaveLength(3);
        expect(receivedData).toEqual(['chunk1', 'chunk2', 'chunk3']);
        done();
      } catch (error) {
        done(error);
      }
    });

    stream.start();
  });

  it('resolves a promise when end event fires', () => {
    const stream = new DataStream();

    const endPromise = new Promise<void>((resolve) => {
      stream.on('end', resolve);
    });

    stream.start();

    return expect(endPromise).resolves.toBeUndefined();
  });
});
```

## Testing Polling Functions

```typescript
describe('pollUntilComplete', () => {
  beforeEach(() => jest.useFakeTimers());
  afterEach(() => jest.useRealTimers());

  it('resolves when the condition is met', async () => {
    let callCount = 0;
    const mockCheck = jest.fn().mockImplementation(async () => {
      callCount++;
      return callCount >= 3 ? 'complete' : 'pending';
    });

    const promise = pollUntilComplete(mockCheck, {
      interval: 1000,
      condition: (result) => result === 'complete',
    });

    await jest.advanceTimersByTimeAsync(2500);

    const result = await promise;
    expect(result).toBe('complete');
    expect(mockCheck).toHaveBeenCalledTimes(3);
  });

  it('rejects after the maximum number of attempts', async () => {
    const mockCheck = jest.fn().mockResolvedValue('pending');

    const promise = pollUntilComplete(mockCheck, {
      interval: 1000,
      maxAttempts: 5,
      condition: (result) => result === 'complete',
    });

    await jest.advanceTimersByTimeAsync(5000);

    await expect(promise).rejects.toThrow('Max attempts exceeded');
    expect(mockCheck).toHaveBeenCalledTimes(5);
  });
});
```

## Real-World Async Service Test

```typescript
// src/services/orderService.ts
export class OrderService {
  constructor(
    private readonly db: Database,
    private readonly paymentGateway: PaymentGateway,
    private readonly emailService: EmailService,
  ) {}

  async placeOrder(userId: number, items: CartItem[]): Promise<Order> {
    const total = items.reduce((sum, item) => sum + item.price * item.qty, 0);
    const payment = await this.paymentGateway.charge(userId, total);

    if (!payment.success) {
      throw new PaymentError(payment.declineReason);
    }

    const order = await this.db.orders.create({
      userId,
      items,
      total,
      paymentId: payment.id,
    });

    await this.emailService.send(userId, 'order_confirmation', { order });
    return order;
  }
}

// __tests__/orderService.test.ts
describe('OrderService.placeOrder', () => {
  const mockDb = { orders: { create: jest.fn() } };
  const mockPaymentGateway = { charge: jest.fn() };
  const mockEmailService = { send: jest.fn() };

  let service: OrderService;

  const cartItems: CartItem[] = [
    { productId: 1, name: 'Widget', price: 9.99, qty: 2 },
    { productId: 2, name: 'Gadget', price: 49.99, qty: 1 },
  ];

  beforeEach(() => {
    service = new OrderService(mockDb as any, mockPaymentGateway as any, mockEmailService as any);

    mockPaymentGateway.charge.mockResolvedValue({ success: true, id: 'pay_123' });
    mockDb.orders.create.mockResolvedValue({ id: 'order_456', items: cartItems, total: 69.97 });
    mockEmailService.send.mockResolvedValue(undefined);
  });

  it('charges the correct total', async () => {
    await service.placeOrder(1, cartItems);
    expect(mockPaymentGateway.charge).toHaveBeenCalledWith(1, 69.97);
  });

  it('creates an order record with the payment ID', async () => {
    await service.placeOrder(1, cartItems);
    expect(mockDb.orders.create).toHaveBeenCalledWith(
      expect.objectContaining({ paymentId: 'pay_123' }),
    );
  });

  it('sends a confirmation email after success', async () => {
    const order = await service.placeOrder(1, cartItems);
    expect(mockEmailService.send).toHaveBeenCalledWith(
      1,
      'order_confirmation',
      { order },
    );
  });

  it('throws PaymentError without creating an order when payment fails', async () => {
    expect.assertions(2);
    mockPaymentGateway.charge.mockResolvedValueOnce({
      success: false,
      declineReason: 'Insufficient funds',
    });

    await expect(service.placeOrder(1, cartItems)).rejects.toThrow(PaymentError);
    expect(mockDb.orders.create).not.toHaveBeenCalled();
  });
});
```

Jest's configuration lives in `jest.config.ts` (or `jest.config.js`, or the `jest` key in `package.json`). This reference covers every commonly-used option with practical examples.

## Full Configuration Template

```typescript
import type { Config } from 'jest';

const config: Config = {
  // --- Test Discovery ---
  roots: ['<rootDir>/src'],
  testMatch: ['**/__tests__/**/*.{ts,tsx}', '**/*.{spec,test}.{ts,tsx}'],
  testPathIgnorePatterns: ['/node_modules/', '/dist/', '/build/'],

  // --- Environment ---
  testEnvironment: 'jsdom',
  testEnvironmentOptions: {
    url: 'http://localhost',
  },

  // --- Transform ---
  preset: 'ts-jest',
  transform: {
    '^.+\\.tsx?$': ['ts-jest', { tsconfig: 'tsconfig.test.json' }],
  },

  // --- Module Resolution ---
  moduleNameMapper: {
    '^@/(.*)$': '<rootDir>/src/$1',
    '\\.css$': '<rootDir>/__mocks__/styleMock.ts',
    '\\.svg$': '<rootDir>/__mocks__/svgMock.ts',
    '\\.(jpg|jpeg|png|gif|webp)$': '<rootDir>/__mocks__/fileMock.ts',
  },
  moduleFileExtensions: ['ts', 'tsx', 'js', 'jsx', 'json', 'node'],
  modulePaths: ['<rootDir>/src'],

  // --- Setup ---
  globalSetup: '<rootDir>/jest.globalSetup.ts',
  globalTeardown: '<rootDir>/jest.globalTeardown.ts',
  setupFilesAfterFramework: ['<rootDir>/jest.setup.ts'],

  // --- Coverage ---
  collectCoverageFrom: ['src/**/*.{ts,tsx}', '!src/**/*.d.ts'],
  coverageReporters: ['text', 'lcov', 'html'],
  coverageProvider: 'v8',
  coverageThreshold: {
    global: { branches: 80, functions: 85, lines: 85, statements: 85 },
  },

  // --- Performance ---
  maxWorkers: '50%',
  testTimeout: 10000,
  cacheDirectory: '<rootDir>/.jest-cache',
  cache: true,

  // --- Output ---
  verbose: false,
  reporters: ['default'],

  // --- Behaviour ---
  clearMocks: true,
  restoreMocks: true,
  detectOpenHandles: true,
  forceExit: false,
  bail: 1,
};

export default config;
```

## `testEnvironment`

Controls the global environment in which tests run.

<Tabs>
  <TabItem label="node">

    The default environment. Uses Node.js globals — suitable for backend APIs, CLI tools, and pure utility functions.

    ```typescript
    testEnvironment: 'node',
    ```

    Available globals: `Buffer`, `process`, `__dirname`, `require`  
    Not available: `window`, `document`, `localStorage`

  </TabItem>
  <TabItem label="jsdom">

    Simulates a browser environment using jsdom. Required for React components and any code that uses DOM APIs.

    ```typescript
    testEnvironment: 'jsdom',
    testEnvironmentOptions: {
      url: 'http://localhost:3000',
      customExportConditions: [''],
    },
    ```

    Available globals: `window`, `document`, `navigator`, `localStorage`, `fetch` (via polyfill)

  </TabItem>
  <TabItem label="Per-file override">

    Override the environment for a single file using a docblock comment:

    ```typescript
    /**
     * @jest-environment jsdom
     */
    import { render } from '@testing-library/react';
    // ...
    ```

  </TabItem>
</Tabs>

## `transform`

Tells Jest how to transform files before running them. Each key is a regex matching file paths.

### ts-jest (TypeScript with type-checking)

```typescript
transform: {
  '^.+\\.tsx?$': ['ts-jest', {
    tsconfig: 'tsconfig.json',
    diagnostics: {
      warnOnly: false,       // fail tests on TypeScript errors
      ignoreCodes: [2322],   // ignore specific TS error codes
    },
    isolatedModules: true,   // faster: skip full type-checking
  }],
},
```

### babel-jest (fast TypeScript without type-checking)

```typescript
transform: {
  '^.+\\.tsx?$': ['babel-jest', {
    presets: [
      ['@babel/preset-env', { targets: { node: 'current' } }],
      ['@babel/preset-react', { runtime: 'automatic' }],
      '@babel/preset-typescript',
    ],
  }],
},
```

### Combining multiple transforms

```typescript
transform: {
  '^.+\\.tsx?$': ['ts-jest', {}],              // TypeScript
  '^.+\\.jsx?$': 'babel-jest',                  // JavaScript (node_modules)
  '^.+\\.graphql$': 'jest-transform-graphql',    // GraphQL
  '^.+\\.mdx?$': '@mdx-js/jest',               // MDX
},
transformIgnorePatterns: [
  '/node_modules/(?!(some-esm-package|another-esm-package))',
],
```

## `moduleNameMapper`

Maps module identifiers to mock files or alternative paths. Essential for TypeScript path aliases, CSS modules, and static assets.

### Path Aliases

```typescript
// tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"],
      "@components/*": ["./src/components/*"],
      "@hooks/*": ["./src/hooks/*"]
    }
  }
}
```

```typescript
// jest.config.ts — mirror the tsconfig paths
moduleNameMapper: {
  '^@/(.*)$': '<rootDir>/src/$1',
  '^@components/(.*)$': '<rootDir>/src/components/$1',
  '^@hooks/(.*)$': '<rootDir>/src/hooks/$1',
},
```

### CSS Modules and Static Assets

```typescript
// __mocks__/styleMock.ts
module.exports = new Proxy({}, { get: (_, prop) => prop });
// usage: styles.myClass → 'myClass'

// __mocks__/fileMock.ts
module.exports = 'test-file-stub';

// __mocks__/svgMock.tsx
const SvgMock = ({ title }: { title?: string }) => <svg>{title}</svg>;
export default SvgMock;
export const ReactComponent = SvgMock;
```

```typescript
moduleNameMapper: {
  '\\.module\\.(css|scss|sass)$': 'identity-obj-proxy',  // npm install --save-dev identity-obj-proxy
  '\\.css$': '<rootDir>/__mocks__/styleMock.ts',
  '\\.svg$': '<rootDir>/__mocks__/svgMock.tsx',
  '\\.(jpg|jpeg|png|gif|webp|avif|ico|bmp|tiff)$': '<rootDir>/__mocks__/fileMock.ts',
},
```

## `setupFilesAfterFramework`

An array of module paths that run **after** the test framework is installed in the environment. Use this to configure global matchers and test utilities.

```typescript
setupFilesAfterFramework: [
  '<rootDir>/jest.setup.ts',
],
```

```typescript
// jest.setup.ts
import '@testing-library/jest-dom';    // DOM matchers
import 'jest-extended';                // Extra matchers
import { server } from './src/mocks/server'; // MSW server

// Start mock service worker before all tests
beforeAll(() => server.listen({ onUnhandledRequest: 'error' }));
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

// Global mock for IntersectionObserver (not in jsdom)
global.IntersectionObserver = jest.fn().mockImplementation(() => ({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
}));

// Global mock for ResizeObserver
global.ResizeObserver = jest.fn().mockImplementation(() => ({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
}));
```

:::tip[setupFiles vs setupFilesAfterFramework]
`setupFiles` runs before the test framework is installed — use it for polyfills. `setupFilesAfterFramework` runs after — use it for custom matchers and framework configuration.
:::

## `globalSetup` and `globalTeardown`

Run once before/after all test suites, in a separate Node.js context (no access to the jsdom environment). Perfect for starting/stopping databases, servers, or other external services.

```typescript
// jest.globalSetup.ts
import { GenericContainer } from 'testcontainers';

let postgresContainer: StartedTestContainer;

export default async function globalSetup() {
  postgresContainer = await new GenericContainer('postgres:16')
    .withEnvironment({ POSTGRES_PASSWORD: 'test' })
    .withExposedPorts(5432)
    .start();

  process.env.DATABASE_URL =
    `postgresql://postgres:test@localhost:${postgresContainer.getMappedPort(5432)}/testdb`;
}
```

```typescript
// jest.globalTeardown.ts
export default async function globalTeardown() {
  await postgresContainer?.stop();
}
```

```typescript
// jest.config.ts
const config: Config = {
  globalSetup: '<rootDir>/jest.globalSetup.ts',
  globalTeardown: '<rootDir>/jest.globalTeardown.ts',
};
```

:::caution[No Shared State Between globalSetup and Tests]
`globalSetup` runs in a separate Node.js process. Variables set there are NOT automatically available in tests — use `process.env` to pass values, or write to a temp file.
:::

## `projects` — Monorepo and Multi-Environment Setup

Run multiple Jest configurations in one command — ideal for monorepos or projects that need both jsdom and node environments:

```typescript
// jest.config.ts (root)
const config: Config = {
  projects: [
    // Test frontend components with jsdom
    {
      displayName: 'frontend',
      testEnvironment: 'jsdom',
      testMatch: ['<rootDir>/packages/ui/**/*.test.tsx'],
      setupFilesAfterFramework: ['<rootDir>/jest.setup.ts'],
      moduleNameMapper: {
        '\\.css$': '<rootDir>/__mocks__/styleMock.ts',
      },
    },
    // Test backend API with node
    {
      displayName: 'backend',
      testEnvironment: 'node',
      testMatch: ['<rootDir>/packages/api/**/*.test.ts'],
    },
    // Test shared utilities
    {
      displayName: 'shared',
      testEnvironment: 'node',
      testMatch: ['<rootDir>/packages/shared/**/*.test.ts'],
    },
  ],
};
```

## Performance Options

```typescript
const config: Config = {
  // Limit workers to avoid exhausting CI resources
  maxWorkers: 2,         // fixed number
  maxWorkers: '50%',     // percentage of available CPUs

  // Bail after N test suite failures (0 = run all)
  bail: 1,

  // Increase timeout for slow integration tests (ms)
  testTimeout: 30_000,

  // Cache transformed modules between runs
  cache: true,
  cacheDirectory: '<rootDir>/.jest-cache',

  // Run failed tests first in watch mode
  // (enabled by default)
};
```

## Debugging and Safety Options

```typescript
const config: Config = {
  // Print individual test names (slower for large suites)
  verbose: false,

  // Warn about tests leaving handles open (database connections, timers)
  detectOpenHandles: true,

  // Force exit after all tests complete (use with caution — hides open handles)
  forceExit: false,

  // Randomise test order within each file
  randomize: false,

  // Show full diff for large objects (default: truncates at ~50 lines)
  expand: false,
};
```

:::tip[detectOpenHandles]
Enable `detectOpenHandles: true` during development to find tests that leave database connections, HTTP servers, or timers open after completing. These cause Jest to hang at the end of the test run. Fix the root cause rather than using `forceExit`.
:::

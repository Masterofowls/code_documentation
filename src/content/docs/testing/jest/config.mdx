`jest.config.ts` is the central configuration file for Jest. This reference covers every major option with explanations and examples for complex setups.

## Full Configuration Template

```typescript
import type { Config } from 'jest';

const config: Config = {
  // --- Environment ---
  testEnvironment: 'node',
  testEnvironmentOptions: {},

  // --- Discovery ---
  roots: ['<rootDir>/src'],
  testMatch: ['**/__tests__/**/*.{ts,tsx}', '**/*.{spec,test}.{ts,tsx}'],
  testPathIgnorePatterns: ['/node_modules/', '/dist/'],
  moduleFileExtensions: ['ts', 'tsx', 'js', 'jsx', 'json', 'node'],

  // --- Transform ---
  transform: {
    '^.+\\.(ts|tsx)$': ['ts-jest', { tsconfig: 'tsconfig.test.json' }],
  },
  transformIgnorePatterns: ['/node_modules/'],

  // --- Module Resolution ---
  moduleNameMapper: {
    '^@/(.*)$': '<rootDir>/src/$1',
    '\\.(css|less|scss|sass)$': '<rootDir>/src/__mocks__/styleMock.ts',
    '\\.(jpg|jpeg|png|gif|svg|webp)$': '<rootDir>/src/__mocks__/fileMock.ts',
  },
  modulePaths: ['<rootDir>/src'],

  // --- Setup ---
  globalSetup: '<rootDir>/src/test-utils/globalSetup.ts',
  globalTeardown: '<rootDir>/src/test-utils/globalTeardown.ts',
  setupFiles: ['<rootDir>/src/test-utils/envSetup.ts'],
  setupFilesAfterFramework: [
    '@testing-library/jest-dom',
    '<rootDir>/src/test-utils/customMatchers.ts',
  ],

  // --- Coverage ---
  collectCoverage: false,
  collectCoverageFrom: ['src/**/*.{ts,tsx}', '!src/**/*.d.ts'],
  coverageDirectory: 'coverage',
  coverageProvider: 'babel',
  coverageReporters: ['text', 'lcov', 'html'],
  coverageThreshold: {
    global: { statements: 80, branches: 75, functions: 85, lines: 80 },
  },

  // --- Performance ---
  maxWorkers: '50%',
  testTimeout: 10000,

  // --- Output ---
  verbose: false,
  reporters: ['default'],

  // --- Behavior ---
  clearMocks: true,
  restoreMocks: true,
  resetModules: false,
  detectOpenHandles: true,
  forceExit: false,

  // --- Snapshots ---
  snapshotSerializers: ['jest-serializer-html'],
  snapshotFormat: { escapeString: false, printBasicPrototype: false },
};

export default config;
```

## testEnvironment

Controls the JavaScript environment used to run your tests.

```typescript
// Node.js environment (default) — for server-side and CLI code
testEnvironment: 'node';

// jsdom environment — for React components and browser APIs
testEnvironment: 'jsdom';

// Custom environment — e.g., for Miniflare (Cloudflare Workers)
testEnvironment: '@miniflare/jest-environment-miniflare';
```

You can also set the environment per-file using a docblock comment:

```typescript
/**
 * @jest-environment jsdom
 */
import { render } from '@testing-library/react';
// ...
```

### testEnvironmentOptions

Pass options to the environment:

```typescript
testEnvironmentOptions: {
  // jsdom options
  url: 'http://localhost:3000',
  userAgent: 'Jest Test Runner',
  resources: 'usable',          // enables loading external resources
  runScripts: 'dangerously',    // enables <script> tag execution
  pretendToBeVisual: true,      // enables requestAnimationFrame
}
```

## transform

Specifies how to transform source files before running them. Jest uses `babel-jest` by default if Babel is available.

<Tabs>
  <TabItem label="ts-jest">
    ```typescript
    transform: {
      '^.+\\.tsx?$': ['ts-jest', {
        tsconfig: {
          jsx: 'react-jsx',
          strict: true,
        },
        diagnostics: {
          warnOnly: false,     // type errors fail tests
          ignoreCodes: ['TS151001'],
        },
      }],
    }
    ```
</TabItem>
  <TabItem label="babel-jest">
    ```typescript
    transform: {
      '^.+\\.(ts|tsx|js|jsx)$': ['babel-jest', {
        presets: [
          ['@babel/preset-env', { targets: { node: 'current' } }],
          ['@babel/preset-typescript'],
          ['@babel/preset-react', { runtime: 'automatic' }],
        ],
      }],
    }
    ```
</TabItem>
  <TabItem label="SWC (fastest)">
    ```typescript
    // npm install --save-dev @swc/jest @swc/core
    transform: {
      '^.+\\.(ts|tsx)$': ['@swc/jest', {
        jsc: {
          parser: { syntax: 'typescript', tsx: true },
          transform: { react: { runtime: 'automatic' } },
        },
      }],
    }
    ```
</TabItem>
</Tabs>

### transformIgnorePatterns

By default, Jest does not transform `node_modules`. If a module you use is ESM-only, add it to the ignore exceptions:

```typescript
// Transform node_modules that use ESM (e.g., uuid, nanoid, chalk v5+)
transformIgnorePatterns: ['/node_modules/(?!(uuid|nanoid|chalk|@acme)/)'];
```

## moduleNameMapper

Maps module paths to mock files or path aliases. Processed in order — the first match wins.

```typescript
moduleNameMapper: {
  // Path aliases (must match tsconfig.json paths)
  '^@/(.*)$': '<rootDir>/src/$1',
  '^@components/(.*)$': '<rootDir>/src/components/$1',
  '^@utils/(.*)$': '<rootDir>/src/utils/$1',

  // CSS Modules — return an object with className keys
  '\\.module\\.(css|scss|sass)$': 'identity-obj-proxy',

  // Plain CSS/SCSS — return empty string
  '\\.(css|less|scss|sass)$': '<rootDir>/src/__mocks__/styleMock.ts',

  // Static assets — return the filename
  '\\.(jpg|jpeg|png|gif|webp|avif|ico|svg)$': '<rootDir>/src/__mocks__/fileMock.ts',

  // SVG as React component (if using SVGR)
  '\\.svg$': '<rootDir>/src/__mocks__/svgMock.tsx',
}
```

```typescript
// src/__mocks__/styleMock.ts
export default {};
export const stylesheet = {};
```

```typescript
// src/__mocks__/fileMock.ts
const path = require('path');
module.exports = {
  default: (filepath: string) => path.basename(filepath),
};
```

```typescript
// src/__mocks__/svgMock.tsx
const SvgMock = ({ title }: { title?: string }) => <svg role="img" aria-label={title} />;
export default SvgMock;
export const ReactComponent = SvgMock;
```

## setupFilesAfterFramework

Runs after the test framework is installed in the environment. Use for:

- Installing custom matchers
- Setting up global mocks
- Configuring test utilities

```typescript
setupFilesAfterFramework: [
  '@testing-library/jest-dom', // adds toBeInTheDocument(), etc.
  'jest-extended', // adds extra matchers
  '<rootDir>/src/test-utils/setup.ts',
];
```

```typescript
// src/test-utils/setup.ts
import '@testing-library/jest-dom';
import { server } from './msw-server'; // Mock Service Worker

// Start MSW server before all tests
beforeAll(() => server.listen({ onUnhandledRequest: 'error' }));
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

// Suppress specific console errors in tests
const originalError = console.error;
beforeAll(() => {
  jest.spyOn(console, 'error').mockImplementation((...args) => {
    if (
      typeof args[0] === 'string' &&
      args[0].includes('Warning: ReactDOM.render')
    ) {
      return;
    }
    originalError(...args);
  });
});
```

## setupFiles

Runs before the test framework is installed. Use for polyfills and environment configuration:

```typescript
setupFiles: [
  '<rootDir>/src/test-utils/polyfills.ts',
  '<rootDir>/src/test-utils/env.ts',
];
```

```typescript
// src/test-utils/polyfills.ts
import 'whatwg-fetch'; // fetch polyfill for Node
import { TextEncoder, TextDecoder } from 'util';

Object.assign(global, { TextDecoder, TextEncoder });

// Mock IntersectionObserver (not in jsdom)
global.IntersectionObserver = jest.fn().mockImplementation(() => ({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
}));

// Mock ResizeObserver
global.ResizeObserver = jest.fn().mockImplementation(() => ({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
}));
```

## globalSetup and globalTeardown

Run once before/after the entire test suite (in the main Jest process, not in workers). Used for:

- Starting/stopping external services (databases, servers)
- Creating global test fixtures

```typescript
// src/test-utils/globalSetup.ts
import { execSync } from 'child_process';

export default async function globalSetup(): Promise<void> {
  // Start a test database
  process.env.DATABASE_URL = 'postgresql://test:test@localhost:5433/test_db';

  execSync(
    'docker run --rm -d --name jest-postgres \
    -e POSTGRES_USER=test -e POSTGRES_PASSWORD=test -e POSTGRES_DB=test_db \
    -p 5433:5432 postgres:15-alpine',
    { stdio: 'inherit' },
  );

  // Wait for database to be ready
  await waitForDatabase(process.env.DATABASE_URL, { timeout: 30000 });

  // Run migrations
  execSync('npx prisma migrate deploy', { stdio: 'inherit' });
}

// src/test-utils/globalTeardown.ts
export default async function globalTeardown(): Promise<void> {
  execSync('docker stop jest-postgres', { stdio: 'inherit' });
}
```

## projects — Monorepo Multi-Project Configuration

Run multiple Jest configurations from a single command:

```typescript
// jest.config.ts at repo root
const config: Config = {
  projects: [
    {
      displayName: 'unit',
      testMatch: ['<rootDir>/src/**/*.test.ts'],
      testEnvironment: 'node',
      preset: 'ts-jest',
    },
    {
      displayName: 'react',
      testMatch: ['<rootDir>/src/**/*.test.tsx'],
      testEnvironment: 'jsdom',
      preset: 'ts-jest',
      setupFilesAfterFramework: ['@testing-library/jest-dom'],
    },
    {
      displayName: 'integration',
      testMatch: ['<rootDir>/src/**/*.integration.test.ts'],
      testEnvironment: 'node',
      preset: 'ts-jest',
      testTimeout: 30000,
      maxWorkers: 1,
    },
  ],
};
```

Run a specific project:

```bash
npx jest --selectProjects unit
npx jest --selectProjects react integration
```

## maxWorkers and testTimeout

```typescript
// maxWorkers — controls parallelism
maxWorkers: '50%',   // use 50% of available CPU cores (default)
maxWorkers: 4,       // exactly 4 workers
maxWorkers: 1,       // sequential (useful for integration tests with shared state)

// testTimeout — default timeout per test in milliseconds
testTimeout: 5000,   // 5 seconds (default)
testTimeout: 30000,  // 30 seconds for integration/slow tests
```

Override timeout per test:

```typescript
it('slow integration test', async () => {
  // ...
}, 30000); // 30 second timeout for this test only
```

## detectOpenHandles and forceExit

```typescript
// detectOpenHandles — identify what keeps Jest from exiting cleanly
detectOpenHandles: true,
// When tests finish but Jest hangs, this prints what's keeping the process open
// (open database connections, HTTP servers, timers, etc.)

// forceExit — force Jest to exit after tests complete
// Use only as last resort — better to fix the leak with detectOpenHandles
forceExit: false,
```

:::caution[Avoid forceExit in Production]
`forceExit: true` hides resource leaks. Always use `detectOpenHandles: true` first to identify and fix open handles. Reserve `forceExit` for legacy codebases where fixing leaks is not feasible.
:::

## snapshotSerializers

Customize how values are serialized in snapshots:

```typescript
snapshotSerializers: [
  'jest-serializer-html', // prettier HTML snapshots
  '@emotion/jest/serializer', // Emotion CSS-in-JS
  'jest-snapshot-serializer-raw', // raw string snapshots
];
```

```typescript
snapshotFormat: {
  escapeString: false,          // don't escape unicode in snapshots
  printBasicPrototype: false,   // don't print Object {} wrapper
}
```

## verbose Mode

```typescript
verbose: true, // print each individual test name as it runs
```

```
PASS  src/utils/math.test.ts
  add
    ✓ returns the sum of two positive numbers (2ms)
    ✓ handles negative numbers (1ms)
    ✓ handles floating point numbers (1ms)
  divide
    ✓ divides two numbers (1ms)
    ✓ throws when dividing by zero (1ms)
```

Without `verbose: true`, Jest only prints failed tests and a summary line per file.

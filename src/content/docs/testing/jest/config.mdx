`jest.config.ts` is the central configuration file for Jest. This reference covers every significant option with examples, trade-offs, and practical guidance for real-world setups.

## Configuration File Formats

Jest accepts configuration in several formats:

```typescript
// jest.config.ts (recommended — TypeScript, type-safe)
import type { Config } from 'jest';

const config: Config = {
  testEnvironment: 'node',
};

export default config;
```

```javascript
// jest.config.js (CommonJS)
/** @type {import('jest').Config} */
module.exports = {
  testEnvironment: 'node',
};
```

```json
// package.json — suitable for simple configs
{
  "jest": {
    "testEnvironment": "node"
  }
}
```

:::tip[Use jest.config.ts]
The TypeScript config file gives you full type safety, IntelliSense for all options, and the ability to import and compose configuration. It's the preferred format for any non-trivial setup.
:::

## testEnvironment

Controls the global environment in which tests run:

```typescript
// jest.config.ts
export default {
  // 'node' (default): Node.js globals — process, Buffer, __dirname
  testEnvironment: 'node',

  // 'jsdom': Browser-like environment with window, document, localStorage
  testEnvironment: 'jsdom',

  // Custom environment
  testEnvironment: '@jest/globals', // Jest globals only, no Node or browser globals
  testEnvironment: './myCustomEnvironment.ts', // path to custom environment module
};
```

Set the environment per-file using a docblock:

```typescript
/**
 * @jest-environment jsdom
 */
describe('DOM-heavy tests', () => {
  it('accesses window', () => {
    expect(typeof window).toBe('object');
  });
});
```

Pass environment options:

```typescript
export default {
  testEnvironmentOptions: {
    url: 'https://example.com', // sets window.location
    customExportConditions: ['node', 'require', 'default'],
    resources: 'usable', // enable resource loading in jsdom
  },
};
```

## transform

Tells Jest how to transform files before executing them:

```typescript
// ts-jest (recommended for TypeScript)
export default {
  transform: {
    '^.+\\.tsx?$': ['ts-jest', {
      tsconfig: 'tsconfig.test.json', // custom tsconfig for tests
      diagnostics: {
        warnOnly: false, // fail tests on TypeScript errors
      },
    }],
  },
};
```

```typescript
// babel-jest (faster, no type checking)
export default {
  transform: {
    '^.+\\.[jt]sx?$': ['babel-jest', {
      presets: [
        ['@babel/preset-env', { targets: { node: 'current' } }],
        '@babel/preset-typescript',
        '@babel/preset-react',
      ],
    }],
    '^.+\\.svg$': '<rootDir>/svgTransform.js',
  },
};
```

```typescript
// SWC (fastest — Rust-based)
export default {
  transform: {
    '^.+\\.[jt]sx?$': ['@swc/jest', {
      jsc: {
        parser: { syntax: 'typescript', tsx: true },
        transform: { react: { runtime: 'automatic' } },
      },
    }],
  },
};
```

| Transformer | Speed | Type checking | Setup complexity |
|---|---|---|---|
| `ts-jest` | Medium | Yes | Low |
| `babel-jest` | Fast | No | Low |
| `@swc/jest` | Very fast | No | Medium |

## moduleNameMapper

Maps import paths to mocked modules or alternative paths. Essential for TypeScript path aliases and mocking static assets:

```typescript
export default {
  moduleNameMapper: {
    // TypeScript path aliases — must match tsconfig.json paths
    '^@/(.*)$': '<rootDir>/src/$1',
    '^@components/(.*)$': '<rootDir>/src/components/$1',
    '^@utils/(.*)$': '<rootDir>/src/utils/$1',
    '^@hooks/(.*)$': '<rootDir>/src/hooks/$1',

    // CSS modules — return an object proxy
    '\\.module\\.(css|scss|less)$': 'identity-obj-proxy',

    // Plain CSS files — return empty string
    '\\.(css|scss|less)$': '<rootDir>/__mocks__/styleMock.js',

    // Static file assets — return filename
    '\\.(jpg|jpeg|png|gif|webp|avif|svg|ico|bmp|tiff)$': '<rootDir>/__mocks__/fileMock.js',

    // Font files
    '\\.(woff|woff2|eot|ttf|otf)$': '<rootDir>/__mocks__/fileMock.js',

    // Environment-specific module
    '^@env$': '<rootDir>/src/env.test.ts',
  },
};
```

```javascript
// __mocks__/styleMock.js
module.exports = {};

// __mocks__/fileMock.js
module.exports = 'test-file-stub';
```

```bash
# Install identity-obj-proxy for CSS modules
npm install --save-dev identity-obj-proxy
```

## setupFiles and setupFilesAfterFramework

Run setup code before or after the test framework is installed:

```typescript
export default {
  // Runs before the test framework (jest-jasmine2 or jest-circus) is installed
  // Use for: polyfills, environment globals
  setupFiles: [
    '<rootDir>/jest.polyfills.ts',
    'fake-indexeddb/auto', // IndexedDB polyfill
  ],

  // Runs after the test framework is installed, before each test file
  // Use for: extending expect(), configuring testing-library
  setupFilesAfterFramework: [
    '@testing-library/jest-dom',
    'jest-extended',
    '<rootDir>/jest.setup.ts',
  ],
};
```

```typescript
// jest.setup.ts — runs before every test file
import '@testing-library/jest-dom';

// Configure global test timeout
jest.setTimeout(10000);

// Mock globals
global.ResizeObserver = jest.fn().mockImplementation(() => ({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
}));

global.IntersectionObserver = jest.fn().mockImplementation(() => ({
  observe: jest.fn(),
  unobserve: jest.fn(),
  disconnect: jest.fn(),
}));

// Mock matchMedia
Object.defineProperty(window, 'matchMedia', {
  writable: true,
  value: jest.fn().mockImplementation((query: string) => ({
    matches: false,
    media: query,
    onchange: null,
    addListener: jest.fn(),
    removeListener: jest.fn(),
    addEventListener: jest.fn(),
    removeEventListener: jest.fn(),
    dispatchEvent: jest.fn(),
  })),
});

// Suppress console.error for expected warnings in tests
const originalConsoleError = console.error;
beforeAll(() => {
  console.error = (...args: unknown[]) => {
    if (typeof args[0] === 'string' && args[0].includes('prop-types')) return;
    originalConsoleError(...args);
  };
});
afterAll(() => {
  console.error = originalConsoleError;
});
```

## globalSetup and globalTeardown

Run once before/after all test suites in the entire test run (not per file):

```typescript
export default {
  globalSetup: '<rootDir>/jest.globalSetup.ts',
  globalTeardown: '<rootDir>/jest.globalTeardown.ts',
};
```

```typescript
// jest.globalSetup.ts
import { GenericContainer } from 'testcontainers';

export default async function globalSetup() {
  // Start a real PostgreSQL container for integration tests
  const postgres = await new GenericContainer('postgres:15')
    .withEnvironment({
      POSTGRES_DB: 'testdb',
      POSTGRES_USER: 'test',
      POSTGRES_PASSWORD: 'test',
    })
    .withExposedPorts(5432)
    .start();

  process.env.TEST_DATABASE_URL = `postgresql://test:test@localhost:${postgres.getMappedPort(5432)}/testdb`;

  // Store reference for teardown
  (global as any).__POSTGRES_CONTAINER__ = postgres;
}

// jest.globalTeardown.ts
export default async function globalTeardown() {
  await (global as any).__POSTGRES_CONTAINER__?.stop();
}
```

:::caution[globalSetup Scope]
Code in `globalSetup` runs in a separate Node.js context from test files. You cannot use `jest.fn()` or access mock state from `globalSetup`. It's only suitable for starting/stopping external resources (databases, servers).
:::

## projects — Monorepo and Multi-Config

Run multiple Jest configurations in a single `jest` invocation:

```typescript
// Root jest.config.ts (monorepo)
import type { Config } from 'jest';

const config: Config = {
  projects: [
    // Unit tests with jsdom
    {
      displayName: 'unit',
      testMatch: ['<rootDir>/src/**/*.test.ts'],
      testEnvironment: 'jsdom',
      setupFilesAfterFramework: ['@testing-library/jest-dom'],
    },
    // Integration tests with Node
    {
      displayName: 'integration',
      testMatch: ['<rootDir>/src/**/*.integration.test.ts'],
      testEnvironment: 'node',
      testTimeout: 30000,
    },
    // Package-specific config
    '<rootDir>/packages/api/jest.config.ts',
    '<rootDir>/packages/ui/jest.config.ts',
    '<rootDir>/packages/shared/jest.config.ts',
  ],
};

export default config;
```

Run a specific project:

```bash
npx jest --selectProjects unit
npx jest --selectProjects integration
```

## Performance Options

```typescript
export default {
  // Number of worker processes
  // Default: number of CPUs - 1
  maxWorkers: '50%', // use 50% of available CPUs
  maxWorkers: 4,     // use exactly 4 workers
  maxWorkers: 1,     // run serially (--runInBand equivalent)

  // Run serially in the current process (no workers)
  // Good for debugging and memory-constrained CI environments
  // equivalent to maxWorkers: 1
  runInBand: false,

  // Cache transformed files
  cache: true,
  cacheDirectory: '/tmp/jest-cache',

  // Only run tests related to changed files (requires git)
  onlyChanged: false,

  // Run tests in a specific shard (for CI parallelism)
  // Set via CLI: --shard=1/4
};
```

## Timeout

```typescript
export default {
  // Default timeout for each test (ms)
  testTimeout: 5000, // 5 seconds (default)

  // For slow integration tests:
  testTimeout: 30000, // 30 seconds
};

// Override per test:
it('runs a slow operation', async () => {
  // ...
}, 15000); // 15 second timeout for this test only

// Override globally in a describe:
describe('slow tests', () => {
  jest.setTimeout(20000);

  it('takes a while', async () => { /* ... */ });
});
```

## Reporters

```typescript
export default {
  // Default reporter shows colored terminal output
  reporters: ['default'],

  // Multiple reporters
  reporters: [
    'default',                    // keep the standard output
    ['jest-junit', {              // JUnit XML for CI systems
      outputDirectory: 'reports',
      outputName: 'junit.xml',
      classNameTemplate: '{classname}',
      titleTemplate: '{title}',
      ancestorSeparator: ' › ',
      usePathForSuiteName: true,
    }],
    ['jest-html-reporter', {      // HTML report
      pageTitle: 'Test Report',
      outputPath: 'reports/test-report.html',
      includeFailureMsg: true,
      includeSuiteFailure: true,
    }],
  ],
};
```

## Handling Open Handles and Forced Exit

```typescript
export default {
  // Warn about async operations that prevent Jest from exiting cleanly
  detectOpenHandles: true,

  // Force Jest to exit after all tests complete
  // (use detectOpenHandles first to find the real cause)
  forceExit: true,
};
```

:::tip[detectOpenHandles Before forceExit]
Always run with `detectOpenHandles: true` first to identify what's keeping Jest alive (unclosed database connections, running servers, etc.). Fix the root cause instead of using `forceExit`, which can mask real cleanup bugs.
:::

export default {
  // Custom snapshot serializers
  snapshotSerializers: [
    'enzyme-to-json/serializer',      // Enzyme component trees

  // Snapshot format options

  // Where to store snapshot files
  // Default: __snapshots__ directory next to test file
  // Can be customized with a resolver
};

## Module Resolution

```typescript
export default {
  // Add directories to module resolution (like node_modules)
  modulePaths: ['<rootDir>/src'],

  // File extensions to resolve (in order)
  moduleFileExtensions: ['ts', 'tsx', 'js', 'jsx', 'json', 'node'],

  // Custom resolver
  resolver: '<rootDir>/jest.resolver.js',

  // Ignore certain paths from module resolution
  modulePathIgnorePatterns: ['<rootDir>/dist/', '<rootDir>/build/'],
};
```

## Complete Example Configuration

```typescript
// jest.config.ts — production-ready configuration
import type { Config } from 'jest';
import { pathsToModuleNameMapper } from 'ts-jest';
import { compilerOptions } from './tsconfig.json';

const config: Config = {
  displayName: 'MyApp',
  preset: 'ts-jest',
  testEnvironment: 'jsdom',
  rootDir: '.',

  testMatch: [
    '<rootDir>/src/**/*.test.{ts,tsx}',
    '<rootDir>/src/**/__tests__/**/*.{ts,tsx}',
  ],

  testPathIgnorePatterns: [
    '/node_modules/',
    '/dist/',
    '/coverage/',
    '\\.e2e\\.test\\.',
  ],

  transform: {
    '^.+\\.tsx?$': ['ts-jest', { diagnostics: { warnOnly: false } }],
  },

  moduleNameMapper: {
    ...pathsToModuleNameMapper(compilerOptions.paths, { prefix: '<rootDir>/' }),
    '\\.module\\.(css|scss)$': 'identity-obj-proxy',
    '\\.(css|scss)$': '<rootDir>/__mocks__/styleMock.js',
    '\\.(jpg|png|svg|gif)$': '<rootDir>/__mocks__/fileMock.js',
  },

  setupFiles: ['<rootDir>/jest.polyfills.ts'],
  setupFilesAfterFramework: [
    '@testing-library/jest-dom',
    '<rootDir>/jest.setup.ts',
  ],

  collectCoverage: false, // use --coverage flag
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/*.stories.{ts,tsx}',
    '!src/**/index.ts',
  ],
  coverageDirectory: 'coverage',
  coverageReporters: ['text', 'lcov', 'html'],
  coverageProvider: 'babel',
  coverageThreshold: {
    global: { branches: 80, functions: 80, lines: 80, statements: 80 },
  },

  clearMocks: true,
  restoreMocks: true,

  testTimeout: 10000,
  maxWorkers: '50%',

  reporters: [
    'default',
    process.env.CI ? ['jest-junit', { outputDirectory: 'reports' }] : null,
  ].filter(Boolean) as Config['reporters'],

  verbose: false,
  detectOpenHandles: true,
};

export default config;
```

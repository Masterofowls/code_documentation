`jest.config.ts` is the central configuration file for Jest. Understanding each option allows you to tailor Jest precisely to your project's needs — from simple TypeScript projects to complex monorepos.

## Complete Configuration Example

```typescript
import type { Config } from 'jest';

const config: Config = {
  // ── Environment ──────────────────────────────────────────────
  testEnvironment: 'node',

  // ── Discovery ─────────────────────────────────────────────────
  roots: ['<rootDir>/src'],
  testMatch: [
    '**/__tests__/**/*.{ts,tsx}',
    '**/*.{spec,test}.{ts,tsx}',
  ],
  testPathIgnorePatterns: [
    '/node_modules/',
    '/dist/',
    '/.next/',
  ],

  // ── Transform ─────────────────────────────────────────────────
  preset: 'ts-jest',
  transform: {
    '^.+\\.tsx?$': ['ts-jest', { tsconfig: './tsconfig.test.json' }],
  },

  // ── Module Resolution ─────────────────────────────────────────
  moduleNameMapper: {
    '^@/(.*)$': '<rootDir>/src/$1',
    '^@shared/(.*)$': '<rootDir>/packages/shared/src/$1',
    '\\.css$': '<rootDir>/src/__mocks__/styleMock.ts',
    '\\.(jpg|png|svg|gif|webp)$': '<rootDir>/src/__mocks__/fileMock.ts',
  },
  moduleFileExtensions: ['ts', 'tsx', 'js', 'jsx', 'json'],
  modulePaths: ['<rootDir>/src'],

  // ── Setup ─────────────────────────────────────────────────────
  setupFiles: ['<rootDir>/src/test-polyfills.ts'],
  setupFilesAfterFramework: ['<rootDir>/src/test-setup.ts'],
  globalSetup: '<rootDir>/src/global-setup.ts',
  globalTeardown: '<rootDir>/src/global-teardown.ts',

  // ── Coverage ──────────────────────────────────────────────────
  collectCoverage: false,
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/*.test.{ts,tsx}',
    '!src/**/*.stories.{ts,tsx}',
    '!src/test-utils/**',
  ],
  coverageDirectory: 'coverage',
  coverageReporters: ['text', 'lcov', 'html'],
  coverageProvider: 'babel',
  coverageThreshold: {
    global: { branches: 75, functions: 85, lines: 80, statements: 80 },
  },

  // ── Performance ───────────────────────────────────────────────
  maxWorkers: '50%',
  testTimeout: 10000,

  // ── Output ────────────────────────────────────────────────────
  verbose: false,
  silent: false,
  reporters: ['default'],

  // ── Behavior ──────────────────────────────────────────────────
  clearMocks: true,
  restoreMocks: false,
  resetMocks: false,
  detectOpenHandles: false,
  forceExit: false,

  // ── Snapshots ─────────────────────────────────────────────────
  snapshotSerializers: ['@emotion/jest/serializer'],
  updateSnapshot: 'new',
};

export default config;
```

## testEnvironment

The environment that tests run in:

```typescript
// 'node' — default, no DOM APIs
testEnvironment: 'node'

// 'jsdom' — simulated browser environment
testEnvironment: 'jest-environment-jsdom'

// Per-file override using docblock
// @jest-environment jsdom
```

| Environment | Use Case | DOM APIs |
|---|---|---|
| `node` | Backend services, utilities, CLI tools | ❌ Not available |
| `jest-environment-jsdom` | React components, DOM manipulation | ✅ Available |
| `@jest-environment-node` (explicit) | When you need to be explicit | ❌ |

For projects with mixed environments (e.g., API tests use `node`, component tests use `jsdom`), use the `projects` config or per-file docblock comments.

## transform

`transform` maps file extensions to transformer packages. Jest must transform non-JavaScript files before executing them.

<Tabs>
  <TabItem label="ts-jest">
    ```typescript
    transform: {
      '^.+\\.tsx?$': ['ts-jest', {
        tsconfig: {
          jsx: 'react-jsx',
          strict: true,
        },
        diagnostics: {
          warnOnly: false,    // fail tests on TypeScript errors
          ignoreCodes: [],
        },
      }],
    }
    ```
  </TabItem>
  <TabItem label="babel-jest">
    ```typescript
    transform: {
      '^.+\\.tsx?$': ['babel-jest', {
        presets: [
          ['@babel/preset-env', { targets: { node: 'current' } }],
          '@babel/preset-typescript',
          ['@babel/preset-react', { runtime: 'automatic' }],
        ],
      }],
    }
    ```
  </TabItem>
  <TabItem label="SWC (fastest)">
    ```typescript
    // npm install --save-dev @swc/jest @swc/core
    transform: {
      '^.+\\.(t|j)sx?$': ['@swc/jest', {
        jsc: {
          parser: { syntax: 'typescript', tsx: true },
          transform: { react: { runtime: 'automatic' } },
        },
      }],
    }
    ```
  </TabItem>
</Tabs>

:::tip[SWC for Large Projects]
`@swc/jest` (Rust-based transform) can be 10-20x faster than `ts-jest` for large codebases. It strips types without checking them — run `tsc --noEmit` separately in CI for type safety.
:::

## moduleNameMapper

Maps import paths to alternative paths or mock files. Essential for TypeScript path aliases and asset mocking.

```typescript
moduleNameMapper: {
  // TypeScript path aliases (match tsconfig.json paths)
  '^@/(.*)$': '<rootDir>/src/$1',
  '^@components/(.*)$': '<rootDir>/src/components/$1',
  '^@hooks/(.*)$': '<rootDir>/src/hooks/$1',
  '^@utils/(.*)$': '<rootDir>/src/utils/$1',

  // CSS Modules — return a proxy that returns the class name
  '\\.module\\.(css|scss|sass)$': 'identity-obj-proxy',

  // Global CSS — return empty object
  '\\.(css|scss|sass|less)$': '<rootDir>/src/__mocks__/styleMock.ts',

  // Image/media files — return the filename as a string
  '\\.(jpg|jpeg|png|gif|webp|svg|ico)$': '<rootDir>/src/__mocks__/fileMock.ts',

  // Font files
  '\\.(woff|woff2|eot|ttf|otf)$': '<rootDir>/src/__mocks__/fileMock.ts',
},
```

```typescript
// src/__mocks__/styleMock.ts
export default {};

// src/__mocks__/fileMock.ts
export default 'test-file-stub';

// For CSS Modules — returns the class name as the value
// npm install --save-dev identity-obj-proxy
```

## setupFiles and setupFilesAfterFramework

Two separate setup hooks with different timing:

| Option | When it Runs | Use For |
|---|---|---|
| `setupFiles` | Before Jest installs in the environment | Polyfills, global variable setup |
| `setupFilesAfterFramework` | After Jest is installed, before each test file | Custom matchers, mock globals, environment config |

```typescript
// src/test-polyfills.ts (setupFiles)
// Runs before the test framework is installed
import 'whatwg-fetch';  // fetch polyfill for Node.js < 18
global.TextEncoder = require('util').TextEncoder;
global.TextDecoder = require('util').TextDecoder;

// src/test-setup.ts (setupFilesAfterFramework)
// Runs after Jest is available — can use jest.fn(), expect, etc.
import '@testing-library/jest-dom';
import { server } from './mocks/server';

// Set up MSW mock service worker
beforeAll(() => server.listen({ onUnhandledRequest: 'error' }));
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

// Mock window.matchMedia (jsdom doesn't implement it)
Object.defineProperty(window, 'matchMedia', {
  writable: true,
  value: jest.fn().mockImplementation(query => ({
    matches: false,
    media: query,
    addEventListener: jest.fn(),
    removeEventListener: jest.fn(),
  })),
});
```

## globalSetup and globalTeardown

Run once per test run (not per file) in a separate Node.js process. Use for spinning up test databases, servers, or other expensive resources.

```typescript
// src/global-setup.ts
import { startTestDatabase } from './test-utils/database';

export default async function globalSetup() {
  console.log('Starting test database...');
  const db = await startTestDatabase();
  process.env.TEST_DATABASE_URL = db.connectionString;

  // Store cleanup reference — can't use closures across processes
  // so write to a temp file or environment variable
  process.env.TEST_DB_PID = String(db.pid);
}
```

```typescript
// src/global-teardown.ts
import { stopTestDatabase } from './test-utils/database';

export default async function globalTeardown() {
  const pid = Number(process.env.TEST_DB_PID);
  await stopTestDatabase(pid);
  console.log('Test database stopped.');
}
```

:::caution[globalSetup Runs in a Separate Process]
`globalSetup` and `globalTeardown` run in a separate Node.js context from your tests. Values set on `global` or module-level variables are **not** accessible in tests. Use `process.env` to share data.
:::

## projects — Monorepo Multi-Project

Run multiple Jest configurations in a single invocation. Ideal for monorepos or projects with both frontend and backend test suites.

```typescript
import type { Config } from 'jest';

const config: Config = {
  projects: [
    {
      displayName: 'api',
      testEnvironment: 'node',
      testMatch: ['<rootDir>/src/api/**/*.test.ts'],
      preset: 'ts-jest',
    },
    {
      displayName: 'ui',
      testEnvironment: 'jest-environment-jsdom',
      testMatch: ['<rootDir>/src/ui/**/*.test.tsx'],
      preset: 'ts-jest',
      setupFilesAfterFramework: ['<rootDir>/src/test-setup.ts'],
    },
    {
      displayName: 'shared',
      testEnvironment: 'node',
      testMatch: ['<rootDir>/packages/shared/**/*.test.ts'],
      preset: 'ts-jest',
    },
  ],
};

export default config;
```

Run a specific project:

```bash
npx jest --selectProjects api
npx jest --selectProjects ui shared
```

## maxWorkers and testTimeout

```typescript
// maxWorkers — how many worker processes to use
maxWorkers: 4,          // absolute number
maxWorkers: '50%',      // 50% of available CPUs (default)
maxWorkers: '100%',     // all CPUs (may overwhelm the system)

// testTimeout — default timeout per test in milliseconds
testTimeout: 5000,      // 5 seconds (default)
testTimeout: 30000,     // 30 seconds (for integration tests)
```

Override timeout for individual tests:

```typescript
it('waits for the server to start', async () => {
  await waitForServer();
}, 30000); // 30-second timeout for this test only
```

## Reporters

```typescript
reporters: [
  'default',                          // built-in terminal reporter

  // JUnit XML for CI systems (Jenkins, GitHub Actions test annotations)
  ['jest-junit', {
    outputDirectory: 'reports',
    outputName: 'junit.xml',
    classNameTemplate: '{classname}',
    titleTemplate: '{title}',
    ancestorSeparator: ' > ',
  }],

  // HTML report
  ['jest-html-reporter', {
    pageTitle: 'Test Report',
    outputPath: 'reports/test-report.html',
    includeFailureMsg: true,
    includeConsoleLog: false,
  }],
],
```

Install reporters:

```bash
npm install --save-dev jest-junit jest-html-reporter
```

## detectOpenHandles and forceExit

```typescript
// detectOpenHandles — warn about handles preventing Jest from exiting
// Useful for debugging "Jest did not exit" issues
detectOpenHandles: true,

// forceExit — forcefully exit after tests complete
// Use only as a last resort — fix the actual open handles instead
forceExit: false,
```

Common causes of open handles:
- Unclosed database connections
- Running HTTP servers not shut down in `afterAll`
- Pending `setInterval` calls
- Open WebSocket connections

## snapshotSerializers

Custom serializers control how values are displayed in `.snap` files:

```typescript
snapshotSerializers: [
  // @emotion/jest — prettier snapshots for Emotion CSS-in-JS
  '@emotion/jest/serializer',

  // jest-serializer-html — format HTML strings in snapshots
  'jest-serializer-html',

  // enzyme-to-json — if still using Enzyme
  'enzyme-to-json/serializer',
],
```

## verbose, silent, and bail

```typescript
// verbose — show each test name as it runs (default: false)
verbose: true,

// silent — suppress all console output from tests (default: false)
silent: false,

// bail — stop after N test failures (default: 0 = run all)
bail: 1,   // stop after the first failure
bail: 5,   // stop after 5 failures
```

## Complete Production-Ready Configuration

```typescript
import type { Config } from 'jest';
import { pathsToModuleNameMapper } from 'ts-jest';
import { compilerOptions } from './tsconfig.json';

const config: Config = {
  preset: 'ts-jest',
  testEnvironment: 'jest-environment-jsdom',
  roots: ['<rootDir>/src'],

  moduleNameMapper: {
    ...pathsToModuleNameMapper(compilerOptions.paths ?? {}, { prefix: '<rootDir>/' }),
    '\\.module\\.css$': 'identity-obj-proxy',
    '\\.(css|svg|png|jpg)$': '<rootDir>/src/__mocks__/fileMock.ts',
  },

  setupFilesAfterFramework: ['<rootDir>/src/test-setup.ts'],

  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/*.test.{ts,tsx}',
    '!src/**/*.stories.tsx',
  ],

  coverageThreshold: {
    global: { branches: 75, functions: 85, lines: 80, statements: 80 },
  },

  clearMocks: true,
  maxWorkers: '50%',
  testTimeout: 10000,
  bail: 0,
};

export default config;
```

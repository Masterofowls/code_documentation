This guide walks you through installing Jest in a TypeScript project, writing your first tests, and understanding the essential patterns you'll use every day.

## Installation

<Tabs>
  <TabItem label="ts-jest (recommended)">
    `ts-jest` compiles TypeScript directly using the TypeScript compiler, giving you accurate type checking during tests.

    ```bash
    npm install --save-dev jest ts-jest @types/jest typescript
    ```

    Create `jest.config.ts`:

    ```typescript
    import type { Config } from 'jest';

    const config: Config = {
      preset: 'ts-jest',
      testEnvironment: 'node',
      roots: ['<rootDir>/src'],
      testMatch: ['**/__tests__/**/*.ts', '**/*.test.ts', '**/*.spec.ts'],
      collectCoverageFrom: ['src/**/*.ts', '!src/**/*.d.ts'],
    };

    export default config;
    ```

</TabItem>
  <TabItem label="babel-jest">
    `babel-jest` transforms TypeScript via Babel — faster compilation but no type checking at test time.

    ```bash
    npm install --save-dev jest @types/jest babel-jest \
      @babel/core @babel/preset-env @babel/preset-typescript
    ```

    Create `babel.config.js`:

    ```javascript
    module.exports = {
      presets: [
        ['@babel/preset-env', { targets: { node: 'current' } }],
        '@babel/preset-typescript',
      ],
    };
    ```

    Create `jest.config.ts`:

    ```typescript
    import type { Config } from 'jest';

    const config: Config = {
      testEnvironment: 'node',
      transform: {
        '^.+\\.tsx?$': 'babel-jest',
      },
    };

    export default config;
    ```

</TabItem>
  <TabItem label="jsdom (React)">
    For React projects, add jsdom support and `@testing-library/react`:

    ```bash
    npm install --save-dev jest ts-jest @types/jest typescript \
      jest-environment-jsdom @testing-library/react @testing-library/jest-dom \
      @testing-library/user-event
    ```

    Create `jest.config.ts`:

    ```typescript
    import type { Config } from 'jest';

    const config: Config = {
      preset: 'ts-jest',
      testEnvironment: 'jest-environment-jsdom',
      setupFilesAfterFramework: ['<rootDir>/src/setupTests.ts'],
    };

    export default config;
    ```

    Create `src/setupTests.ts`:

    ```typescript
    import '@testing-library/jest-dom';
    ```

</TabItem>
</Tabs>

## jest.config.ts Setup

A complete `jest.config.ts` for a TypeScript project:

```typescript
import type { Config } from 'jest';
import { pathsToModuleNameMapper } from 'ts-jest';
import { compilerOptions } from './tsconfig.json';

const config: Config = {
  // Use ts-jest preset for TypeScript support
  preset: 'ts-jest',

  // Test environment: 'node' for backend, 'jsdom' for frontend
  testEnvironment: 'node',

  // Where to look for test files
  roots: ['<rootDir>/src'],

  // File patterns Jest recognises as tests
  testMatch: ['**/__tests__/**/*.[jt]s?(x)', '**/?(*.)+(spec|test).[jt]s?(x)'],

  // Map TypeScript path aliases to Jest module resolution
  moduleNameMapper: pathsToModuleNameMapper(compilerOptions.paths ?? {}, {
    prefix: '<rootDir>/',
  }),

  // Run this file after the test framework is installed in the environment
  setupFilesAfterFramework: ['<rootDir>/src/setupTests.ts'],

  // Collect coverage from these files
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/index.ts',
    '!src/**/*.stories.tsx',
  ],

  // Fail if coverage drops below these thresholds
  coverageThreshold: {
    global: {
      branches: 80,
      functions: 80,
      lines: 80,
      statements: 80,
    },
  },

  // Timeout per test in milliseconds (default: 5000)
  testTimeout: 10000,

  // Show individual test results with the test suite hierarchy
  verbose: true,
};

export default config;
```

## testMatch Patterns

Jest uses glob patterns to find test files. The default patterns are:

```typescript
testMatch: [
  '**/__tests__/**/*.[jt]s?(x)', // Files inside __tests__ directories
  '**/?(*.)+(spec|test).[jt]s?(x)', // Files ending in .spec.ts or .test.ts
];
```

Common custom patterns:

```typescript
// Only test TypeScript files (no JS)
testMatch: ['**/*.test.ts', '**/*.spec.ts'];

// Only files inside __tests__ directories
testMatch: ['**/__tests__/**/*.ts'];

// Include integration tests in a separate directory
testMatch: ['**/tests/unit/**/*.ts', '**/tests/integration/**/*.ts'];
```

You can also use `testRegex` as an alternative to `testMatch`:

```typescript
testRegex: '(/__tests__/.*|(\\.|/)(test|spec))\\.[jt]sx?$';
```

:::tip[Co-locate Tests with Source]
Place test files next to the source files they test (`math.ts` → `math.test.ts`). This makes the relationship explicit and keeps test coverage visible at a glance in the file tree.
:::

## npm Scripts

Add these scripts to `package.json`:

```json
{
  "scripts": {
    "test": "jest",
    "test:watch": "jest --watch",
    "test:coverage": "jest --coverage",
    "test:ci": "jest --ci --coverage --maxWorkers=2",
    "test:verbose": "jest --verbose",
    "test:bail": "jest --bail"
  }
}
```

| Script                  | Purpose                                   |
| ----------------------- | ----------------------------------------- |
| `npm test`              | Run all tests once                        |
| `npm run test:watch`    | Re-run tests on file change               |
| `npm run test:coverage` | Run tests and generate coverage report    |
| `npm run test:ci`       | CI-optimised run (no interactive prompts) |

## Running Tests

### Run all tests

```bash
npx jest
# or
npm test
```

### Run a single test file

```bash
npx jest src/utils/math.test.ts
npx jest math        # partial match — runs any file with "math" in the path
```

### Run tests matching a name pattern

The `--testNamePattern` (or `-t`) flag filters tests by their `describe` / `it` name:

```bash
npx jest -t "add function"
npx jest -t "handles negative"
npx jest src/math.test.ts -t "add"
```

### Watch mode

Watch mode re-runs affected tests whenever source files change:

```bash
npx jest --watch       # re-runs tests related to changed files (requires git)
npx jest --watchAll    # re-runs all tests on any change
```

Inside watch mode, press:

| Key | Action                      |
| --- | --------------------------- |
| `a` | Run all tests               |
| `f` | Run only failed tests       |
| `p` | Filter by file path pattern |
| `t` | Filter by test name pattern |
| `u` | Update failing snapshots    |
| `q` | Quit watch mode             |

:::caution[Watch Mode Requires Git]
`--watch` uses Git to determine which files have changed. Use `--watchAll` in repositories without Git, or when you want to run tests on every save regardless of what changed.
:::

## Your First Test File

```typescript
// src/utils/math.ts
export const add = (a: number, b: number): number => a + b;

export const divide = (a: number, b: number): number => {
  if (b === 0) throw new Error('Division by zero');
  return a / b;
};

export const clamp = (value: number, min: number, max: number): number =>
  Math.min(Math.max(value, min), max);
```

```typescript
// src/utils/math.test.ts
import { add, divide, clamp } from './math';

describe('add', () => {
  it('adds two positive numbers', () => {
    expect(add(2, 3)).toBe(5);
  });

  it('adds negative numbers', () => {
    expect(add(-1, -2)).toBe(-3);
  });

  it('adds zero', () => {
    expect(add(5, 0)).toBe(5);
  });
});

describe('divide', () => {
  it('divides two numbers', () => {
    expect(divide(10, 2)).toBe(5);
  });

  it('throws when dividing by zero', () => {
    expect(() => divide(10, 0)).toThrow('Division by zero');
  });
});

describe('clamp', () => {
  it('returns the value when within range', () => {
    expect(clamp(5, 0, 10)).toBe(5);
  });

  it('returns the minimum when below range', () => {
    expect(clamp(-5, 0, 10)).toBe(0);
  });

  it('returns the maximum when above range', () => {
    expect(clamp(15, 0, 10)).toBe(10);
  });
});
```

## Test File Naming Conventions

| Convention             | Example             | When to Use                                   |
| ---------------------- | ------------------- | --------------------------------------------- |
| `.test.ts` suffix      | `math.test.ts`      | Standard unit tests co-located with source    |
| `.spec.ts` suffix      | `math.spec.ts`      | BDD-style tests; common in Angular            |
| `__tests__/` directory | `__tests__/math.ts` | When you prefer to separate tests from source |
| `.int.test.ts`         | `api.int.test.ts`   | Integration tests (separate testMatch)        |
| `.e2e.test.ts`         | `auth.e2e.test.ts`  | End-to-end tests run separately               |

## Jest Globals and Lifecycle Hooks

Jest injects these global functions into every test file. You do not need to import them.

### Test definition

```typescript
test('a basic test', () => {
  /* ... */
});
it('an alias for test', () => {
  /* ... */
});
describe('a group of related tests', () => {
  /* ... */
});
```

### Lifecycle hooks

```typescript
describe('UserService', () => {
  let db: Database;
  let service: UserService;

  // Runs ONCE before all tests in this describe block
  beforeAll(async () => {
    db = await Database.connect('test://localhost/test_db');
  });

  // Runs ONCE after all tests in this describe block
  afterAll(async () => {
    await db.disconnect();
  });

  // Runs before EACH test in this describe block
  beforeEach(() => {
    service = new UserService(db);
  });

  // Runs after EACH test in this describe block
  afterEach(async () => {
    await db.collection('users').deleteMany({});
  });

  it('creates a user', async () => {
    const user = await service.create({
      name: 'Alice',
      email: 'alice@example.com',
    });
    expect(user.id).toBeDefined();
  });

  it('finds a user by email', async () => {
    await service.create({ name: 'Bob', email: 'bob@example.com' });
    const found = await service.findByEmail('bob@example.com');
    expect(found?.name).toBe('Bob');
  });
});
```

Lifecycle hooks nest correctly inside `describe` blocks. Outer hooks run before inner hooks:

```typescript
beforeAll(() => console.log('outer beforeAll')); // 1st
describe('inner', () => {
  beforeAll(() => console.log('inner beforeAll')); // 2nd
  beforeEach(() => console.log('inner beforeEach')); // before each test
  it('test one', () => {
    /* ... */
  });
  it('test two', () => {
    /* ... */
  });
  afterAll(() => console.log('inner afterAll')); // 2nd to last
});
afterAll(() => console.log('outer afterAll')); // last
```

### Skipping and focusing tests

```typescript
// Skip a test temporarily
it.skip('a test to skip', () => {
  /* ... */
});
xtest('another way to skip', () => {
  /* ... */
});
xit('yet another way', () => {
  /* ... */
});

// Run ONLY these tests (use for debugging; never commit)
it.only('run only this test', () => {
  /* ... */
});
fit('another way to focus', () => {
  /* ... */
});

// Skip an entire describe block
describe.skip('a skipped suite', () => {
  /* ... */
});

// Focus an entire describe block
describe.only('a focused suite', () => {
  /* ... */
});
```

:::caution[Never Commit .only]
`it.only` and `describe.only` cause all other tests to be skipped. If committed, other tests will silently not run in CI. Add a lint rule (`jest/no-focused-tests`) to catch this automatically.
:::

### Parameterised tests with test.each

```typescript
test.each([
  [1, 2, 3],
  [0, 0, 0],
  [-1, 1, 0],
  [100, -50, 50],
])('add(%i, %i) === %i', (a, b, expected) => {
  expect(add(a, b)).toBe(expected);
});

// Object syntax (cleaner for many parameters)
test.each([
  { a: 1, b: 2, expected: 3 },
  { a: -1, b: 1, expected: 0 },
])('add($a, $b) returns $expected', ({ a, b, expected }) => {
  expect(add(a, b)).toBe(expected);
});
```

### todo tests

Mark tests you plan to write later:

```typescript
test.todo('handles unicode characters in names');
test.todo('rate-limits failed login attempts');
```

These show up in the test output as reminders without failing the suite.

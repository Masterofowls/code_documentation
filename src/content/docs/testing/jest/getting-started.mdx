This guide walks you through installing Jest in a TypeScript project, configuring it for your needs, and writing your first test suite from scratch.

## Installation

<Tabs>
  <TabItem label="ts-jest (recommended)">

    ts-jest is a TypeScript preprocessor for Jest that lets Jest transpile `.ts` and `.tsx` files. It also supports type-checking during tests.

    ```bash
    npm install --save-dev jest ts-jest @types/jest
    npx ts-jest config:init
    ```

    This creates a `jest.config.ts` file pre-configured for TypeScript.

  </TabItem>
  <TabItem label="babel-jest">

    babel-jest uses Babel to strip types before running. Faster but no type-checking during tests.

    ```bash
    npm install --save-dev jest @types/jest babel-jest @babel/core @babel/preset-env @babel/preset-typescript
    ```

    Create `babel.config.js`:

    ```javascript
    module.exports = {
      presets: [
        ['@babel/preset-env', { targets: { node: 'current' } }],
        '@babel/preset-typescript',
      ],
    };
    ```

  </TabItem>
</Tabs>

:::tip[TypeScript Path Aliases]
If your project uses TypeScript path aliases (e.g., `@/components/...`), you will need to configure `moduleNameMapper` in `jest.config.ts` to mirror your `tsconfig.json` paths. See the [Configuration guide](/testing/jest/config) for details.
:::

## jest.config.ts Setup

Create a `jest.config.ts` at the root of your project:

```typescript
import type { Config } from 'jest';

const config: Config = {
  // Use ts-jest to handle TypeScript files
  preset: 'ts-jest',

  // Node environment for backend code; use 'jsdom' for frontend/React
  testEnvironment: 'node',

  // Where to find test files
  roots: ['<rootDir>/src'],

  // Patterns Jest uses to detect test files
  testMatch: [
    '**/__tests__/**/*.{ts,tsx}',
    '**/*.{spec,test}.{ts,tsx}',
  ],

  // Transform TypeScript files
  transform: {
    '^.+\\.tsx?$': ['ts-jest', {
      tsconfig: 'tsconfig.json',
    }],
  },

  // Module file extensions to resolve
  moduleFileExtensions: ['ts', 'tsx', 'js', 'jsx', 'json'],

  // Run this file after the test framework is installed
  setupFilesAfterFramework: ['<rootDir>/jest.setup.ts'],

  // Collect coverage from these files
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/index.ts',
  ],

  // Coverage thresholds
  coverageThreshold: {
    global: {
      branches: 80,
      functions: 80,
      lines: 80,
      statements: 80,
    },
  },
};

export default config;
```

:::caution[setupFilesAfterFramework Typo]
The correct key is `setupFilesAfterFramework` in older configs, but Jest 27+ uses `setupFilesAfterFramework`. Double-check the spelling matches your Jest version.  In all current versions it is **`setupFilesAfterFramework`**.
:::

## testMatch Patterns

Jest uses glob patterns to discover test files. The default patterns cover the most common conventions:

| Pattern | Matches |
|---------|---------|
| `**/__tests__/**/*.{js,ts}` | Any file inside a `__tests__` directory |
| `**/*.test.{js,ts}` | Files ending in `.test.js` or `.test.ts` |
| `**/*.spec.{js,ts}` | Files ending in `.spec.js` or `.spec.ts` |

```typescript
// jest.config.ts — custom testMatch example
const config: Config = {
  testMatch: [
    '<rootDir>/src/**/*.test.ts',   // only .test.ts files under src/
    '<rootDir>/tests/**/*.spec.ts', // spec files in a separate tests/ dir
  ],
};
```

You can also use `testPathIgnorePatterns` to exclude folders:

```typescript
testPathIgnorePatterns: [
  '/node_modules/',
  '/dist/',
  '/coverage/',
],
```

## npm Scripts

Add these scripts to your `package.json`:

```json
{
  "scripts": {
    "test": "jest",
    "test:watch": "jest --watch",
    "test:ci": "jest --ci --coverage --forceExit",
    "test:coverage": "jest --coverage",
    "test:verbose": "jest --verbose"
  }
}
```

| Script | Description |
|--------|-------------|
| `npm test` | Run all tests once |
| `npm run test:watch` | Re-run tests on file change |
| `npm run test:ci` | Optimised for CI — no watch mode, with coverage |
| `npm run test:coverage` | Generate coverage report |
| `npm run test:verbose` | Show individual test names |

## Running Specific Tests

### Run a Single Test File

```bash
npx jest src/utils/math.test.ts
# or with npm test (-- passes args to jest)
npm test -- src/utils/math.test.ts
```

### Run Tests Matching a Pattern

```bash
# Matches any test file path containing "math"
npx jest math

# Matches test files in the utils directory
npx jest src/utils
```

### Run Tests by Name

Use `--testNamePattern` (or `-t`) to run only tests whose `describe`/`it` name matches:

```bash
# Run only tests whose name contains "formatCurrency"
npx jest -t "formatCurrency"

# Run only tests in the "add function" describe block
npx jest -t "add function"
```

### Run a Single Test Interactively

In watch mode, press `p` to filter by file pattern, or `t` to filter by test name:

```bash
npx jest --watch
# Press 'p' → type file pattern
# Press 't' → type test name pattern
# Press 'a' → run all tests
# Press 'q' → quit watch mode
```

## Watch Mode

Watch mode re-runs tests automatically whenever related source files or test files change. It uses Jest's dependency graph to run only the tests affected by your changes.

```bash
npx jest --watch          # watch mode (requires git)
npx jest --watchAll       # watch mode (no git required)
```

:::tip[Watch Mode in CI]
Never use `--watch` or `--watchAll` in CI pipelines — they block indefinitely. Use `--ci` flag instead which disables interactive prompts and exits with a non-zero code on failure.
:::

## Your First Test File

Create `src/utils/math.ts`:

```typescript
export function add(a: number, b: number): number {
  return a + b;
}

export function divide(a: number, b: number): number {
  if (b === 0) {
    throw new Error('Cannot divide by zero');
  }
  return a / b;
}

export function clamp(value: number, min: number, max: number): number {
  return Math.min(Math.max(value, min), max);
}
```

Create `src/utils/math.test.ts`:

```typescript
import { add, divide, clamp } from './math';

describe('add', () => {
  it('adds two positive numbers', () => {
    expect(add(2, 3)).toBe(5);
  });

  it('adds a positive and a negative number', () => {
    expect(add(10, -4)).toBe(6);
  });

  it('returns 0 when both arguments are 0', () => {
    expect(add(0, 0)).toBe(0);
  });
});

describe('divide', () => {
  it('divides two numbers', () => {
    expect(divide(10, 2)).toBe(5);
  });

  it('returns a decimal for non-even division', () => {
    expect(divide(1, 3)).toBeCloseTo(0.333, 2);
  });

  it('throws an error when dividing by zero', () => {
    expect(() => divide(10, 0)).toThrow('Cannot divide by zero');
  });
});

describe('clamp', () => {
  it('returns the value when within range', () => {
    expect(clamp(5, 0, 10)).toBe(5);
  });

  it('returns the minimum when value is below range', () => {
    expect(clamp(-5, 0, 10)).toBe(0);
  });

  it('returns the maximum when value is above range', () => {
    expect(clamp(15, 0, 10)).toBe(10);
  });
});
```

Run the tests:

```bash
npx jest src/utils/math.test.ts --verbose
```

Expected output:

```

    ✓ adds a positive and a negative number
    ✓ returns 0 when both arguments are 0

  divide
    ✓ divides two numbers
    ✓ returns a decimal for non-even division
    ✓ throws an error when dividing by zero (1ms)

  clamp
    ✓ returns the value when within range
    ✓ returns the minimum when value is below range
    ✓ returns the maximum when value is above range

Test Suites: 1 passed, 1 total
Tests:       9 passed, 9 total
```

## Test File Naming Conventions

| Convention | Example | Notes |
|------------|---------|-------|
| Co-located `.test.ts` | `math.test.ts` next to `math.ts` | Most common; keeps tests close to source |
| Co-located `.spec.ts` | `math.spec.ts` next to `math.ts` | Common in Angular projects |
| `__tests__` directory | `__tests__/math.ts` | Groups all tests; preferred for larger projects |
| Integration test folder | `tests/integration/api.test.ts` | Separate folder for integration/E2E tests |

:::tip[Prefer Co-located Tests]
Co-locating test files next to their source modules makes it easier to find tests, understand coverage at a glance, and keep tests updated when source files move or are renamed.
:::

## Jest Globals and Lifecycle Hooks

Jest provides global functions available in every test file without importing them.

### Test Structure Globals

```typescript
describe('block name', () => { /* group related tests */ });
it('test name', () => { /* individual test; alias for test() */ });
test('test name', () => { /* same as it() */ });
```

### Lifecycle Hooks

```typescript
describe('UserService', () => {
  let db: Database;
  let service: UserService;

  // Runs once before all tests in this describe block
  beforeAll(async () => {
    db = await Database.connect('sqlite::memory:');
    await db.migrate();
  });

  // Runs once after all tests in this describe block
  afterAll(async () => {
    await db.close();
  });

  // Runs before each individual test
  beforeEach(async () => {
    service = new UserService(db);
    await db.seed({ users: [{ id: 1, name: 'Alice' }] });
  });

  // Runs after each individual test
  afterEach(async () => {
    await db.truncate('users');
    jest.clearAllMocks();
  });

  it('finds a user by ID', async () => {
    const user = await service.findById(1);
    expect(user?.name).toBe('Alice');
  });

  it('returns null for a missing user', async () => {
    const user = await service.findById(999);
    expect(user).toBeNull();
  });
});
```

### Skipping and Focusing Tests

```typescript
// Skip a test (shows as skipped in output, not failing)
it.skip('pending test', () => { /* ... */ });
xit('also skipped', () => { /* ... */ });

// Run ONLY this test (useful during development — never commit!)
it.only('focused test', () => { /* ... */ });
fit('also focused', () => { /* ... */ });

// Skip an entire describe block
describe.skip('WIP feature', () => { /* ... */ });

// Run only this describe block
describe.only('feature under focus', () => { /* ... */ });
```

:::caution[Never Commit .only]
Committing `it.only` or `describe.only` to source control will cause all other tests to be silently skipped in CI — breaking your coverage without any obvious error. Use a lint rule (`jest/no-focused-tests` from `eslint-plugin-jest`) to catch this automatically.
:::

## jest.setup.ts

Create a setup file to configure global test utilities:

```typescript
// jest.setup.ts
import '@testing-library/jest-dom';   // adds .toBeInTheDocument() etc.
import 'jest-extended';               // adds 70+ extra matchers

// Silence specific console output during tests
beforeEach(() => {
  jest.spyOn(console, 'error').mockImplementation(() => undefined);
  jest.spyOn(console, 'warn').mockImplementation(() => undefined);
});

afterEach(() => {
  jest.restoreAllMocks();
});
```

Reference it in `jest.config.ts`:

```typescript
setupFilesAfterFramework: ['<rootDir>/jest.setup.ts'],
```

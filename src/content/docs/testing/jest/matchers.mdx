Matchers are the heart of Jest assertions. Every assertion starts with `expect(value)` and chains a matcher that describes what the value should be. Jest ships over 40 built-in matchers and supports extending the set with `expect.extend()`.

## Equality Matchers

The three equality matchers behave differently and choosing the right one matters.

### `toBe` — Strict Reference Equality

`toBe` uses `Object.is()` — the same algorithm as `===` but handles `NaN` and `-0` correctly. Use it for primitives.

```typescript
expect(1 + 1).toBe(2);
expect('hello').toBe('hello');
expect(true).toBe(true);
expect(null).toBe(null);

// Object.is handles edge cases === does not
expect(NaN).toBe(NaN);           // passes (=== would fail)
expect(-0).not.toBe(0);          // passes

// ❌ Never use toBe for objects — it checks reference equality
const obj = { name: 'Alice' };
expect(obj).toBe(obj);           // passes (same reference)
expect({ name: 'Alice' }).not.toBe({ name: 'Alice' }); // different references
```

### `toEqual` — Deep Structural Equality

`toEqual` recursively compares every property. It ignores `undefined` properties in objects and array items.

```typescript
expect({ name: 'Alice', age: 30 }).toEqual({ name: 'Alice', age: 30 });
expect([1, 2, 3]).toEqual([1, 2, 3]);

// toEqual ignores undefined properties
expect({ a: 1, b: undefined }).toEqual({ a: 1 });  // passes!
expect([1, undefined, 3]).toEqual([1, , 3]);         // passes!

// Nested objects
expect({
  user: { profile: { name: 'Alice' } },
}).toEqual({
  user: { profile: { name: 'Alice' } },
});
```

### `toStrictEqual` — Strict Deep Equality

`toStrictEqual` is like `toEqual` but does **not** ignore `undefined` properties, checks array sparseness, and verifies object class/prototype.

```typescript
// undefined properties DO matter
expect({ a: 1, b: undefined }).toStrictEqual({ a: 1 }); // ❌ FAILS

// Object class matters
class Person { constructor(public name: string) {} }
expect(new Person('Alice')).not.toStrictEqual({ name: 'Alice' }); // different class

// Sparse arrays are distinguished
expect([1, , 3]).not.toStrictEqual([1, undefined, 3]); // ❌ FAILS with toEqual but PASSES with toStrictEqual
```

| Matcher | Checks class | Ignores undefined | Sparse arrays |
|---|---|---|---|
| `toBe` | — (reference equality) | — | — |
| `toEqual` | No | Yes (ignored) | Ignored |
| `toStrictEqual` | Yes | No | Distinguished |

:::tip[Which Equality Matcher to Use?]
- Use `toBe` for **primitives** (numbers, strings, booleans, null, undefined).
- Use `toEqual` for **plain objects and arrays** in most cases.
- Use `toStrictEqual` when testing **class instances** or when `undefined` properties carry meaning.
:::

## Truthiness Matchers

```typescript
// toBeTruthy — passes for any value that coerces to true
expect(1).toBeTruthy();
expect('hello').toBeTruthy();
expect([]).toBeTruthy();   // empty array is truthy in JS
expect({}).toBeTruthy();   // empty object is truthy

// toBeFalsy — passes for any value that coerces to false
expect(0).toBeFalsy();
expect('').toBeFalsy();
expect(null).toBeFalsy();
expect(undefined).toBeFalsy();
expect(false).toBeFalsy();
expect(NaN).toBeFalsy();

// Specific nullish matchers
expect(null).toBeNull();
expect(undefined).toBeUndefined();
expect('value').toBeDefined();   // anything that is not undefined

// When you care about the exact type
expect(null).not.toBeUndefined();
expect(undefined).not.toBeNull();
```

:::caution[Avoid Overusing Truthiness]
`toBeTruthy()` and `toBeFalsy()` are intentionally vague — `expect(0).toBeFalsy()` gives no information about *why* 0 is wrong. Prefer specific matchers like `toBe(0)` or `toBeNull()` to write tests that give useful failure messages.
:::

## Number Matchers

```typescript
const value = 15.7;

// Comparison matchers
expect(value).toBeGreaterThan(10);
expect(value).toBeGreaterThanOrEqual(15.7);
expect(value).toBeLessThan(20);
expect(value).toBeLessThanOrEqual(15.7);

// Floating point equality
// 0.1 + 0.2 === 0.30000000000000004 in JavaScript — toBeCloseTo handles this
expect(0.1 + 0.2).toBeCloseTo(0.3);          // default: 2 decimal places
expect(0.1 + 0.2).toBeCloseTo(0.3, 5);       // 5 decimal places

// NaN
expect(NaN).toBeNaN();
expect(42).not.toBeNaN();

// Infinity
expect(Infinity).toBeGreaterThan(Number.MAX_SAFE_INTEGER);
expect(-Infinity).toBeLessThan(Number.MIN_SAFE_INTEGER);

// Type checking alongside value
expect(42).toBe(42);
expect(typeof 42).toBe('number');
```

```typescript
// Real-world example: testing a discount calculator
describe('calculateDiscount', () => {
  it('applies 10% discount to prices above $100', () => {
    const result = calculateDiscount(150);
    expect(result).toBeCloseTo(135, 2);
    expect(result).toBeLessThan(150);
    expect(result).toBeGreaterThan(0);
  });
});
```

## String Matchers

```typescript
const message = 'Hello, World! Welcome to Jest.';

// Substring containment
expect(message).toContain('World');
expect(message).not.toContain('Vue');

// Regex matching
expect(message).toMatch(/Hello/);
expect(message).toMatch(/welcome/i);   // case-insensitive flag
expect(message).toMatch(/^Hello/);     // anchored at start

// Exact string match — same as toBe
expect('jest').toBe('jest');

// String length
expect(message).toHaveLength(30);
expect('').toHaveLength(0);

// Starts/ends with — use regex or toContain
expect(message).toMatch(/^Hello/);
expect(message).toMatch(/Jest\.$/);
```

```typescript
// Practical example: validating generated tokens
describe('generateToken', () => {
  it('returns a 32-character hex string', () => {
    const token = generateToken();
    expect(token).toHaveLength(32);
    expect(token).toMatch(/^[a-f0-9]{32}$/);
  });

  it('returns a unique token on each call', () => {
    expect(generateToken()).not.toBe(generateToken());
  });
});
```

## Array Matchers

```typescript
const fruits = ['apple', 'banana', 'cherry'];

// Item containment
expect(fruits).toContain('banana');
expect(fruits).not.toContain('grape');

// Length
expect(fruits).toHaveLength(3);
expect([]).toHaveLength(0);

// Exact match
expect(fruits).toEqual(['apple', 'banana', 'cherry']);

// Partial match — array contains at minimum these items (order-independent)
expect(fruits).toEqual(
  expect.arrayContaining(['cherry', 'apple'])
);

// Ensure array does NOT contain these items
expect(fruits).not.toEqual(
  expect.arrayContaining(['grape', 'mango'])
);

// Nested arrays
expect([[1, 2], [3, 4]]).toEqual([[1, 2], [3, 4]]);
```

```typescript
// Real-world: testing paginated results
it('returns users sorted by name', async () => {
  const users = await userService.list({ sort: 'name' });

  expect(users).toHaveLength(3);
  expect(users.map((u) => u.name)).toEqual(['Alice', 'Bob', 'Charlie']);

  // Each user should have required fields
  users.forEach((user) => {
    expect(user).toHaveProperty('id');
    expect(user).toHaveProperty('name');
    expect(user).toHaveProperty('email');
  });
});
```

## Object Matchers

### `toHaveProperty` — Check a Specific Property

```typescript
const user = {
  id: 1,
  name: 'Alice',
  address: {
    city: 'London',
    postcode: 'EC1A 1BB',
  },
  roles: ['admin', 'user'],
};

// Check property exists
expect(user).toHaveProperty('name');
expect(user).toHaveProperty('address.city');        // dot notation for nesting

// Check property has specific value
expect(user).toHaveProperty('name', 'Alice');
expect(user).toHaveProperty('address.city', 'London');

// Check array element by index
expect(user).toHaveProperty('roles[0]', 'admin');
expect(user).toHaveProperty('roles', ['admin', 'user']);
```

### `toMatchObject` — Partial Object Match

`toMatchObject` checks that the received object contains at least the expected properties. Extra properties in the received object are ignored.

```typescript
const response = {
  status: 200,
  data: { id: 42, name: 'Alice', createdAt: '2024-01-01T00:00:00Z' },
  headers: { 'content-type': 'application/json' },
};

// Only check the fields you care about
expect(response).toMatchObject({
  status: 200,
  data: { id: 42, name: 'Alice' },
  // createdAt and headers are not checked
});

// Useful for arrays of objects
expect([
  { id: 1, name: 'Alice', token: 'abc123' },
  { id: 2, name: 'Bob', token: 'def456' },
]).toMatchObject([
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },
]);
```

### `expect.objectContaining` — Inline Partial Matching

```typescript
// Use inside other matchers
expect(receivedUser).toEqual(
  expect.objectContaining({
    name: expect.any(String),
    id: expect.any(Number),
    email: expect.stringMatching(/@/),
  })
);

// Great for spyOn/mock call assertion
expect(mockSendEmail).toHaveBeenCalledWith(
  expect.objectContaining({
    to: 'alice@example.com',
    subject: expect.stringContaining('Welcome'),
  })
);
```

## Error Matchers

```typescript
// Function that throws synchronously
const parseAge = (input: string): number => {
  const age = parseInt(input, 10);
  if (isNaN(age)) throw new TypeError(`Invalid age: "${input}"`);
  if (age < 0) throw new RangeError('Age cannot be negative');
  return age;
};

// toThrow — checks that a function throws at all
expect(() => parseAge('abc')).toThrow();

// toThrow with string — checks error message contains the string
expect(() => parseAge('abc')).toThrow('Invalid age');

// toThrow with regex — checks error message matches the pattern
expect(() => parseAge('abc')).toThrow(/Invalid age: "abc"/);

// toThrow with error class — checks error is an instance of the class
expect(() => parseAge('abc')).toThrow(TypeError);

// Combine class and message — pass an error instance
expect(() => parseAge('abc')).toThrow(new TypeError('Invalid age: "abc"'));

// RangeError for negative input
expect(() => parseAge('-5')).toThrow(RangeError);
expect(() => parseAge('-5')).not.toThrow(TypeError);
```

:::caution[Wrap Throwing Calls in Arrow Functions]
The function passed to `expect()` must be a wrapper function: `expect(() => throws()).toThrow()`. If you write `expect(throws()).toThrow()`, the error is thrown before `expect` can catch it and your test will fail with an unhandled error.
:::

### Async Error Matchers

For async functions that reject, use `.rejects`:

```typescript
// async functions
await expect(fetchUser(-1)).rejects.toThrow('User not found');
await expect(fetchUser(-1)).rejects.toThrow(NotFoundError);
await expect(fetchUser(-1)).rejects.toMatchObject({
  message: 'User not found',
  statusCode: 404,
});
```

## Custom Matchers with `expect.extend()`

When built-in matchers don't express your domain clearly, create custom matchers:

```typescript
// jest.setup.ts or a matchers file

expect.extend({
  toBeValidEmail(received: string) {
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    const pass = emailRegex.test(received);
    return {
      pass,
      message: () =>
        pass
          ? `Expected "${received}" NOT to be a valid email`
          : `Expected "${received}" to be a valid email`,
    };
  },

  toBeWithinRange(received: number, floor: number, ceiling: number) {
    const pass = received >= floor && received <= ceiling;
    return {
      pass,
      message: () =>
        pass
          ? `Expected ${received} NOT to be within range [${floor}, ${ceiling}]`
          : `Expected ${received} to be within range [${floor}, ${ceiling}]`,
    };
  },
});

// TypeScript: extend the jest Matchers interface
declare module 'expect' {
  interface Matchers<R> {
    toBeValidEmail(): R;
    toBeWithinRange(floor: number, ceiling: number): R;
  }
}
```

Usage in tests:

```typescript
it('validates user email', () => {
  expect('alice@example.com').toBeValidEmail();
  expect('not-an-email').not.toBeValidEmail();
});

it('generates a score in range', () => {
  const score = calculateScore(player);
  expect(score).toBeWithinRange(0, 100);
});
```

### `expect.any()` and `expect.anything()`

These asymmetric matchers are useful inside `toEqual` and `toHaveBeenCalledWith`:

```typescript
expect(result).toEqual({
  id: expect.any(Number),          // any number
  name: expect.any(String),        // any string
  createdAt: expect.any(Date),     // any Date object
  token: expect.anything(),        // anything except null/undefined
});

// Check mock was called with the right shape
expect(mockCreate).toHaveBeenCalledWith(
  expect.objectContaining({
    userId: expect.any(Number),
    timestamp: expect.any(Date),
  })
);
```

### `expect.stringContaining()` and `expect.stringMatching()`

```typescript
expect('Hello, World!').toEqual(expect.stringContaining('World'));
expect('token_abc123').toEqual(expect.stringMatching(/^token_[a-z0-9]+$/));

// Inside a larger assertion
expect(logEntry).toMatchObject({
  level: 'error',
  message: expect.stringContaining('Connection refused'),
  timestamp: expect.any(String),
});
```

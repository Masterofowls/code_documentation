Mocking is the practice of replacing real dependencies with controlled substitutes during testing. Jest provides a comprehensive mocking system that can replace individual functions, entire modules, timers, and even the system clock — all without modifying production code.

## `jest.fn()` — Mock Functions

`jest.fn()` creates a mock function that tracks calls, arguments, and return values.

```typescript
const mockGreet = jest.fn();

mockGreet('Alice');
mockGreet('Bob');

expect(mockGreet).toHaveBeenCalledTimes(2);
expect(mockGreet).toHaveBeenCalledWith('Alice');
expect(mockGreet).toHaveBeenLastCalledWith('Bob');
```

### Controlling return values

```typescript
const mockAdd = jest.fn();

// Return the same value every time
mockAdd.mockReturnValue(42);
expect(mockAdd(1, 2)).toBe(42);
expect(mockAdd(10, 20)).toBe(42);

// Return different values per call
mockAdd.mockReturnValueOnce(1).mockReturnValueOnce(2).mockReturnValueOnce(3);

expect(mockAdd()).toBe(1);
expect(mockAdd()).toBe(2);
expect(mockAdd()).toBe(3);
expect(mockAdd()).toBeUndefined(); // exhausted — returns undefined
```

### Async return values

```typescript
const mockFetchUser = jest.fn();

// Resolve with a value
mockFetchUser.mockResolvedValue({ id: 1, name: 'Alice' });
const user = await mockFetchUser(1);
expect(user).toEqual({ id: 1, name: 'Alice' });

// Reject with an error
mockFetchUser.mockRejectedValue(new Error('Network error'));
await expect(mockFetchUser(1)).rejects.toThrow('Network error');

// Different outcomes per call
mockFetchUser
  .mockResolvedValueOnce({ id: 1, name: 'Alice' })
  .mockRejectedValueOnce(new Error('Rate limited'));
```

### Custom implementation

```typescript
const mockDivide = jest.fn((a: number, b: number) => {
  if (b === 0) throw new Error('Division by zero');
  return a / b;
});

expect(mockDivide(10, 2)).toBe(5);
expect(() => mockDivide(10, 0)).toThrow('Division by zero');

// Override the implementation later
mockDivide.mockImplementation((a, b) => a + b); // now it adds instead

// Override for just the next call
mockDivide.mockImplementationOnce(() => -1);
expect(mockDivide(10, 2)).toBe(-1); // uses mockImplementationOnce
expect(mockDivide(10, 2)).toBe(12); // back to mockImplementation
```

### Inspecting mock calls

```typescript
const mockSave = jest.fn().mockResolvedValue({ id: 1 });

await mockSave({ name: 'Alice' });
await mockSave({ name: 'Bob' });

// All calls
console.log(mockSave.mock.calls);
// [ [{ name: 'Alice' }], [{ name: 'Bob' }] ]

// All results
console.log(mockSave.mock.results);
// [ { type: 'return', value: Promise }, { type: 'return', value: Promise } ]

// The function instance (useful for checking `this`)
console.log(mockSave.mock.instances);

// Assertion helpers
expect(mockSave).toHaveBeenCalledTimes(2);
expect(mockSave).toHaveBeenCalledWith({ name: 'Alice' });
expect(mockSave).toHaveBeenNthCalledWith(2, { name: 'Bob' });
```

## `jest.spyOn()` — Spying Without Replacing

`jest.spyOn` wraps an existing method to track calls while keeping the original implementation. This is ideal when you want to verify a method was called without changing its behaviour.

```typescript
const calculator = {
  add: (a: number, b: number) => a + b,
  multiply: (a: number, b: number) => a * b,
};

const spy = jest.spyOn(calculator, 'add');

const result = calculator.add(2, 3);

// Original implementation still runs
expect(result).toBe(5);

// But we can assert on the call
expect(spy).toHaveBeenCalledWith(2, 3);
expect(spy).toHaveBeenCalledTimes(1);

// Restore the original implementation
spy.mockRestore();
```

### Spying and overriding the implementation

```typescript
// Spy on a method and replace it with a mock
const consoleSpy = jest.spyOn(console, 'error').mockImplementation(() => {});

// Now console.error won't print anything during the test
expect(() => {
  someCodeThatCallsConsoleError();
}).not.toThrow();

expect(consoleSpy).toHaveBeenCalled();

// Restore the original console.error
consoleSpy.mockRestore();
```

### Spying on module imports

```typescript
import * as fileUtils from './fileUtils';

describe('processFile', () => {
  it('calls readFile with the correct path', () => {
    const readSpy = jest
      .spyOn(fileUtils, 'readFile')
      .mockReturnValue('file content');

    processFile('./data.txt');

    expect(readSpy).toHaveBeenCalledWith('./data.txt');
    readSpy.mockRestore();
  });
});
```

## `jest.mock()` — Module Mocking

`jest.mock()` replaces an entire module with mock implementations. Jest hoists `jest.mock()` calls to the top of the file, above all imports.

### Auto-mock

Jest can automatically generate a mock where every exported function becomes a `jest.fn()`:

```typescript
jest.mock('./database'); // auto-mock: all exports become jest.fn()

import { findUser, saveUser } from './database';

test('creates a user', async () => {
  (findUser as jest.Mock).mockResolvedValue(null);
  (saveUser as jest.Mock).mockResolvedValue({ id: 1, name: 'Alice' });

  const user = await createUser({ name: 'Alice' });

  expect(saveUser).toHaveBeenCalledWith(
    expect.objectContaining({ name: 'Alice' }),
  );
  expect(user.id).toBe(1);
});
```

### Manual mock factory

Provide a factory function to control exactly what the mock returns:

```typescript
jest.mock('./emailService', () => ({
  sendEmail: jest.fn().mockResolvedValue({ messageId: 'abc123' }),
  validateEmail: jest.fn().mockReturnValue(true),
}));

import { sendEmail } from './emailService';

test('sends a welcome email on registration', async () => {
  await registerUser({ email: 'alice@example.com', name: 'Alice' });

  expect(sendEmail).toHaveBeenCalledWith({
    to: 'alice@example.com',
    subject: 'Welcome to the platform!',
    body: expect.stringContaining('Alice'),
  });
});
```

### Partial mock with `jest.requireActual`

Mock some exports while keeping others real:

```typescript
jest.mock('./config', () => ({
  ...jest.requireActual('./config'), // keep all real exports
  API_URL: 'http://localhost:3000', // override just this one
  MAX_RETRIES: 1,
}));
```

### Mocking default exports

```typescript
// The module uses a default export
jest.mock('./logger', () => ({
  __esModule: true, // Required for ES module default exports
  default: {
    info: jest.fn(),
    error: jest.fn(),
    warn: jest.fn(),
  },
}));

import logger from './logger';

test('logs an info message', () => {
  doSomething();
  expect(logger.info).toHaveBeenCalledWith('Something happened');
});
```

### Mocking named exports

```typescript
jest.mock('axios', () => ({
  get: jest.fn(),
  post: jest.fn(),
  create: jest.fn(() => ({
    get: jest.fn(),
    post: jest.fn(),
    interceptors: {
      request: { use: jest.fn() },
      response: { use: jest.fn() },
    },
  })),
}));
```

## The `__mocks__` Directory

For mocks you want to reuse across multiple test files, place them in a `__mocks__` directory:

```
src/
├── services/
│   ├── __mocks__/
│   │   └── emailService.ts   ← manual mock
│   └── emailService.ts       ← real implementation
└── __mocks__/
    ├── fs.ts                 ← mock for Node built-in 'fs'
    └── path.ts               ← mock for Node built-in 'path'
```

In your test files, call `jest.mock('./services/emailService')` without a factory — Jest will automatically use the file from `__mocks__/`:

```typescript
// src/services/__mocks__/emailService.ts
export const sendEmail = jest.fn().mockResolvedValue({ messageId: 'mock-id' });
export const validateEmail = jest.fn().mockReturnValue(true);
```

```typescript
// In your test file
jest.mock('./services/emailService'); // uses __mocks__/emailService.ts automatically

import { sendEmail } from './services/emailService';
```

:::tip[Node built-in mocks]
To mock Node built-in modules (`fs`, `path`, `crypto`), place the mock in the root `__mocks__/` directory and call `jest.mock('fs')`. The mock file name must match the module name exactly.
:::

## Mocking Timers

Jest can replace `setTimeout`, `setInterval`, `clearTimeout`, `clearInterval`, `Date`, and `process.nextTick` with fake implementations you control.

```typescript
describe('retry logic', () => {
  beforeEach(() => {
    jest.useFakeTimers();
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('retries after a delay', () => {
    const mockFn = jest
      .fn()
      .mockRejectedValueOnce(new Error('Failed'))
      .mockResolvedValue('Success');

    const retryPromise = retryWithDelay(mockFn, { delay: 1000, maxRetries: 3 });

    // Advance time by 1 second to trigger the retry
    jest.advanceTimersByTime(1000);

    await retryPromise;
    expect(mockFn).toHaveBeenCalledTimes(2);
  });
});
```

### Timer control methods

```typescript
jest.useFakeTimers();

// Run all pending timers to completion
jest.runAllTimers();

// Run only the next queued timer
jest.runOnlyPendingTimers();

// Advance time by N milliseconds (triggers any timers due in that window)
jest.advanceTimersByTime(5000);

// Returns the count of pending fake timers
jest.getTimerCount();

// Restore real timers
jest.useRealTimers();
```

### Mocking `Date`

```typescript
describe('timestamping', () => {
  beforeEach(() => {
    jest.useFakeTimers();
    jest.setSystemTime(new Date('2024-01-15T10:00:00Z'));
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('stamps the current time', () => {
    const event = createEvent('user_login');
    expect(event.timestamp).toBe('2024-01-15T10:00:00.000Z');
  });

  it('calculates time since last login', () => {
    const lastLogin = new Date('2024-01-14T10:00:00Z');
    const hoursSince = getHoursSinceLastLogin(lastLogin);
    expect(hoursSince).toBe(24);
  });
});
```

## Clearing, Resetting, and Restoring Mocks

Jest provides three levels of mock cleanup, each doing progressively more:

| Method          | Clears call history | Removes return values | Removes implementation | Restores original |
| --------------- | ------------------- | --------------------- | ---------------------- | ----------------- |
| `mockClear()`   | ✅                  | ❌                    | ❌                     | ❌                |
| `mockReset()`   | ✅                  | ✅                    | ✅                     | ❌                |
| `mockRestore()` | ✅                  | ✅                    | ✅                     | ✅ (spies only)   |

```typescript
const mockFn = jest.fn().mockReturnValue(42);

mockFn();

// mockClear: only clears call counts and recorded calls
mockFn.mockClear();
expect(mockFn).toHaveBeenCalledTimes(0); // reset
expect(mockFn()).toBe(42); // return value preserved

// mockReset: clears calls AND removes return value / implementation
mockFn.mockReset();
expect(mockFn()).toBeUndefined(); // return value gone

// mockRestore: only works on spies (jest.spyOn); restores original function
const spy = jest.spyOn(Math, 'random').mockReturnValue(0.5);
spy.mockRestore();
expect(Math.random()).not.toBe(0.5); // real Math.random is back
```

### Automatic cleanup in `jest.config.ts`

Configure Jest to automatically clear mocks between tests:

```typescript
const config: Config = {
  // Automatically call mockClear() before each test
  clearMocks: true,

  // Automatically call mockReset() before each test
  resetMocks: true,

  // Automatically call mockRestore() before each test
  restoreMocks: true,
};
```

:::caution[Don't rely on manual clearance in beforeEach]
If you manually call `mockFn.mockClear()` in `beforeEach` and then add `clearMocks: true` to config (or vice versa), you clear the mock twice. Choose one approach and stick with it. `clearMocks: true` in config is recommended for consistency.
:::

## Mocking ES Modules

ES module mocking requires `--experimental-vm-modules` in Node.js. An alternative is to use the `jest.unstable_mockModule` API:

```typescript
// Using the experimental ESM support
const { default: myModule } = await import('./myModule');
jest.unstable_mockModule('./myModule', () => ({
  default: jest.fn().mockReturnValue('mocked'),
}));
```

For projects using Babel or ts-jest (CommonJS output), standard `jest.mock()` works as usual because TypeScript/Babel compiles ES modules to CommonJS before Jest runs.

## Practical Example: Mocking an API Client

```typescript
// src/services/userService.ts
import { httpClient } from './httpClient';

export async function getUserById(id: number) {
  const response = await httpClient.get(`/users/${id}`);
  return response.data;
}

export async function updateUser(id: number, data: Partial<User>) {
  const response = await httpClient.patch(`/users/${id}`, data);
  return response.data;
}
```

```typescript
// src/services/userService.test.ts
jest.mock('./httpClient');

import { httpClient } from './httpClient';
import { getUserById, updateUser } from './userService';

const mockGet = httpClient.get as jest.Mock;
const mockPatch = httpClient.patch as jest.Mock;

describe('getUserById', () => {
  beforeEach(() => {
    jest.clearAllMocks();
  });

  it('fetches user data from the correct endpoint', async () => {
    mockGet.mockResolvedValue({ data: { id: 1, name: 'Alice' } });

    const user = await getUserById(1);

    expect(mockGet).toHaveBeenCalledWith('/users/1');
    expect(user).toEqual({ id: 1, name: 'Alice' });
  });

  it('propagates network errors', async () => {
    mockGet.mockRejectedValue(new Error('Network error'));

    await expect(getUserById(1)).rejects.toThrow('Network error');
  });
});

describe('updateUser', () => {
  it('patches the user with provided data', async () => {
    mockPatch.mockResolvedValue({ data: { id: 1, name: 'Bob' } });

    const result = await updateUser(1, { name: 'Bob' });

    expect(mockPatch).toHaveBeenCalledWith('/users/1', { name: 'Bob' });
    expect(result.name).toBe('Bob');
  });
});
```

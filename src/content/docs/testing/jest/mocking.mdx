Mocking is the practice of replacing real dependencies with controlled substitutes during testing. Jest provides a rich, integrated mocking system that covers functions, modules, timers, and the system clock.

## jest.fn() — Mock Functions

`jest.fn()` creates a mock function that records all calls, arguments, and return values. Use it to replace callbacks, event handlers, or injected dependencies.

```typescript
// Create a basic mock function
const mockCallback = jest.fn();

// Call it
mockCallback('hello', 42);
mockCallback('world');

// Inspect the call record
expect(mockCallback).toHaveBeenCalledTimes(2);
expect(mockCallback).toHaveBeenCalledWith('hello', 42);
expect(mockCallback).toHaveBeenLastCalledWith('world');
expect(mockCallback).toHaveBeenNthCalledWith(1, 'hello', 42);

// Access raw call data
console.log(mockCallback.mock.calls);
// [['hello', 42], ['world']]
console.log(mockCallback.mock.results);
// [{ type: 'return', value: undefined }, ...]
```

### Controlling Return Values

```typescript
const mockFn = jest.fn();

// Always return the same value
mockFn.mockReturnValue(42);
mockFn(); // → 42
mockFn(); // → 42

// Return a sequence of values, then fall back to default
mockFn
  .mockReturnValueOnce('first')
  .mockReturnValueOnce('second')
  .mockReturnValue('default');

mockFn(); // → 'first'
mockFn(); // → 'second'
mockFn(); // → 'default'
mockFn(); // → 'default'

// Return a resolved Promise (for async functions)
const mockAsync = jest.fn().mockResolvedValue({ data: [] });
await mockAsync(); // → { data: [] }

// Return a rejected Promise
const mockFailing = jest.fn().mockRejectedValue(new Error('Network error'));
await expect(mockFailing()).rejects.toThrow('Network error');

// Sequence with async values
mockAsync
  .mockResolvedValueOnce({ data: ['item1'] })
  .mockRejectedValueOnce(new Error('timeout'))
  .mockResolvedValue({ data: [] });
```

### Custom Implementations

```typescript
// mockImplementation — full custom logic
const mockMultiply = jest.fn().mockImplementation((a: number, b: number) => a * b);
mockMultiply(3, 4); // → 12

// mockImplementationOnce — custom logic for the next call only
const mockFn = jest.fn()
  .mockImplementationOnce(() => 'special first call')
  .mockImplementation(() => 'default');

mockFn(); // → 'special first call'
mockFn(); // → 'default'

// Pass through to original implementation
const original = (x: number) => x * 2;
const spy = jest.fn(original); // wraps the original
spy(5); // → 10, but call is still recorded
```

### Practical Example: Testing a Service

```typescript
// services/notification.service.ts
export interface NotificationService {
  send(userId: string, message: string): Promise<void>;
}

// services/order.service.ts
export class OrderService {
  constructor(private notifications: NotificationService) {}

  async placeOrder(userId: string, items: string[]) {
    // ... process order
    await this.notifications.send(userId, `Order confirmed: ${items.length} items`);
    return { orderId: 'ord-123', status: 'confirmed' };
  }
}

// services/order.service.test.ts
describe('OrderService', () => {
  it('sends a confirmation notification after placing an order', async () => {
    const mockNotify = jest.fn<() => Promise<void>>().mockResolvedValue();
    const service = new OrderService({ send: mockNotify });

    await service.placeOrder('user-1', ['shirt', 'pants']);

    expect(mockNotify).toHaveBeenCalledOnce();
    expect(mockNotify).toHaveBeenCalledWith(
      'user-1',
      'Order confirmed: 2 items'
    );
  });
});
```

## jest.spyOn() — Spying on Existing Methods

`jest.spyOn()` creates a mock function that wraps an existing method, recording calls while optionally changing the behavior. Unlike `jest.fn()`, spies can be easily restored.

```typescript
// Spy without replacing behavior
const warnSpy = jest.spyOn(console, 'warn');

someFunction(); // internally calls console.warn('...')

expect(warnSpy).toHaveBeenCalled();
expect(warnSpy).toHaveBeenCalledWith(expect.stringContaining('deprecated'));

// Restore original implementation
warnSpy.mockRestore();
```

```typescript
// Spy and replace behavior
const mathSpy = jest.spyOn(Math, 'random').mockReturnValue(0.5);

const result = generateRandomId(); // uses Math.random internally
expect(result).toBe('id-0.5'); // deterministic output

mathSpy.mockRestore(); // cleanup
```

```typescript
// Spy on a class method
import { UserRepository } from './UserRepository';

const findByIdSpy = jest
  .spyOn(UserRepository.prototype, 'findById')
  .mockResolvedValue({ id: 1, name: 'Alice' });

const user = await new UserRepository().findById(1);
expect(user.name).toBe('Alice');

findByIdSpy.mockRestore();
```

:::tip[jest.spyOn vs jest.fn()]
Use `jest.spyOn()` when you want to monitor calls to an existing method without fully replacing it, and when you need clean `mockRestore()` behavior. Use `jest.fn()` when creating a dependency from scratch (constructor injection, callbacks).
:::

## jest.mock() — Module Mocking

`jest.mock()` replaces an entire module with a mock version for the duration of the test file. It is hoisted to the top of the file by Babel/Jest transforms.

### Auto-Mock

```typescript
// Jest infers the shape of the module and replaces all exports with jest.fn()
jest.mock('./database');

import { findUser, createUser } from './database';

// findUser and createUser are now jest.fn() — return undefined by default
(findUser as jest.Mock).mockResolvedValue({ id: 1, name: 'Alice' });

const user = await findUser(1);
expect(user.name).toBe('Alice');
```

### Partial Mock with `jest.requireActual`

Keep the real implementation for most exports, override only what you need:

```typescript
jest.mock('./config', () => ({
  ...jest.requireActual('./config'),   // keep real exports
  DATABASE_URL: 'sqlite::memory:',    // override just this one
  LOG_LEVEL: 'silent',
}));
```

### Factory Function Mock

Provide a complete replacement implementation:

```typescript
jest.mock('./email-client', () => ({
  sendEmail: jest.fn().mockResolvedValue({ messageId: 'msg-123', success: true }),
  validateEmail: jest.fn().mockReturnValue(true),
  getDeliveryStatus: jest.fn().mockResolvedValue('delivered'),
}));

import { sendEmail } from './email-client';

it('sends a welcome email after registration', async () => {
  await registerUser({ email: 'alice@example.com', name: 'Alice' });

  expect(sendEmail).toHaveBeenCalledWith({
    to: 'alice@example.com',
    subject: 'Welcome!',
    body: expect.stringContaining('Alice'),
  });
});
```

### Mocking Default Exports

```typescript
// When the module exports a default class or function:
jest.mock('./Logger', () => {
  return jest.fn().mockImplementation(() => ({
    log: jest.fn(),
    error: jest.fn(),
    warn: jest.fn(),
  }));
});

import Logger from './Logger';
// Logger is a mock constructor
const instance = new Logger();
expect(instance.log).toBeDefined();
```

### Mocking Named and Default Exports Together

```typescript
jest.mock('./analytics', () => ({
  __esModule: true,              // Required for ES module default exports
  default: jest.fn(),            // Mock the default export
  trackEvent: jest.fn(),         // Mock a named export
  setUserId: jest.fn(),
}));
```

## `__mocks__` Directory — Manual Mocks

For modules that need complex mock implementations reused across multiple test files, create manual mocks in a `__mocks__` directory.

```
src/
├── services/
│   ├── email.service.ts
│   └── __mocks__/
│       └── email.service.ts    ← manual mock
├── utils/
│   └── logger.ts
└── __mocks__/
    └── fs.ts                   ← Node.js built-in mock
```

```typescript
// src/services/__mocks__/email.service.ts
const EmailService = jest.fn().mockImplementation(() => ({
  send: jest.fn().mockResolvedValue({ success: true }),
  verify: jest.fn().mockResolvedValue(true),
  getTemplate: jest.fn().mockReturnValue('<p>{{body}}</p>'),
}));

export default EmailService;
export const sendTransactionalEmail = jest.fn().mockResolvedValue({ id: 'msg-1' });
```

Activate the manual mock in tests:

```typescript
jest.mock('../services/email.service'); // auto-uses __mocks__/email.service.ts

import EmailService from '../services/email.service';
const instance = new EmailService();
// instance.send is already a jest.fn()
```

For Node.js built-in modules, place the mock in `<rootDir>/__mocks__/`:

```typescript
// __mocks__/fs.ts
const fs = jest.createMockFromModule('fs') as typeof import('fs');
fs.readFileSync = jest.fn().mockReturnValue('mocked file content');
module.exports = fs;
```

:::caution[Automatic vs Manual Mocks]
Manual mocks in `__mocks__/` are **not** activated automatically for user modules — you must still call `jest.mock('./path/to/module')`. For Node.js built-ins (`fs`, `path`, `crypto`), manual mocks in the root `__mocks__/` **are** activated automatically.
:::

## Mocking Timers

Jest can replace `setTimeout`, `setInterval`, `clearTimeout`, `clearInterval`, `Date`, `queueMicrotask`, and `setImmediate` with controllable fakes.

```typescript
describe('polling mechanism', () => {
  beforeEach(() => {
    jest.useFakeTimers();
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('retries 3 times before giving up', () => {
    const mockFn = jest.fn();

    startPolling(mockFn, { interval: 1000, maxRetries: 3 });

    // Immediately: no calls yet
    expect(mockFn).not.toHaveBeenCalled();

    // Advance by 1 second
    jest.advanceTimersByTime(1000);
    expect(mockFn).toHaveBeenCalledTimes(1);

    // Advance another 2 seconds
    jest.advanceTimersByTime(2000);
    expect(mockFn).toHaveBeenCalledTimes(3);
  });

  it('clears all pending timers', () => {
    const mockCallback = jest.fn();
    setTimeout(mockCallback, 5000);

    jest.runAllTimers(); // executes all pending timers
    expect(mockCallback).toHaveBeenCalledTimes(1);
  });

  it('runs only currently-pending timers (not those they schedule)', () => {
    const outer = jest.fn();
    const inner = jest.fn();

    setTimeout(() => {
      outer();
      setTimeout(inner, 1000); // scheduled by outer
    }, 500);

    jest.runOnlyPendingTimers(); // runs outer, but not inner
    expect(outer).toHaveBeenCalledTimes(1);
    expect(inner).not.toHaveBeenCalled();
  });
});
```

### Mocking the Date

```typescript
describe('session expiration', () => {
  beforeEach(() => {
    jest.useFakeTimers();
    jest.setSystemTime(new Date('2024-01-15T10:00:00.000Z'));
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('expires sessions after 30 minutes', () => {
    const session = createSession('user-1');
    expect(session.isExpired()).toBe(false);

    // Advance time by 31 minutes
    jest.advanceTimersByTime(31 * 60 * 1000);

    expect(session.isExpired()).toBe(true);
  });
});
```

## Clearing and Resetting Mocks

Jest provides four methods for managing mock state. Understanding the differences prevents state leaks between tests.

| Method | Clears calls/results | Resets implementation | Restores original |
|---|---|---|---|
| `mockFn.mockClear()` | ✅ | ❌ | ❌ |
| `mockFn.mockReset()` | ✅ | ✅ (returns undefined) | ❌ |
| `mockFn.mockRestore()` | ✅ | ✅ | ✅ (spyOn only) |
| `jest.clearAllMocks()` | ✅ (all mocks) | ❌ | ❌ |
| `jest.resetAllMocks()` | ✅ (all mocks) | ✅ (all mocks) | ❌ |
| `jest.restoreAllMocks()` | ✅ (all mocks) | ✅ (all mocks) | ✅ (all spies) |

```typescript
const mockFn = jest.fn().mockReturnValue(42);

mockFn(); // call it once

// mockClear — reset call history, keep implementation
mockFn.mockClear();
expect(mockFn).not.toHaveBeenCalled(); // call history cleared
mockFn(); // still returns 42

// mockReset — reset call history AND remove implementation
mockFn.mockReset();
expect(mockFn).not.toHaveBeenCalled();
mockFn(); // returns undefined — implementation removed

// mockRestore — only works on jest.spyOn() — also restores original method
const consoleSpy = jest.spyOn(console, 'log').mockImplementation(() => {});
consoleSpy.mockRestore();
// console.log is back to normal
```

Configure automatic cleanup in `jest.config.ts`:

```typescript
const config: Config = {
  clearMocks: true,     // equivalent to jest.clearAllMocks() in afterEach
  resetMocks: false,    // equivalent to jest.resetAllMocks() in afterEach
  restoreMocks: false,  // equivalent to jest.restoreAllMocks() in afterEach
};
```

:::tip[Recommended Config]
Enable `clearMocks: true` in your Jest config to automatically clear call counts between tests. Only enable `restoreMocks: true` if you use `jest.spyOn()` heavily and want automatic cleanup. Avoid `resetMocks: true` as it silently wipes mock implementations set during module loading.
:::

## Inspecting Mock Calls

The `.mock` property on every mock function contains the full call history:

```typescript
const mockFn = jest.fn();
mockFn('a', 1);
mockFn('b', 2);
mockFn('c', 3);

// All calls
console.log(mockFn.mock.calls);
// [['a', 1], ['b', 2], ['c', 3]]

// All return values
console.log(mockFn.mock.results);
// [
//   { type: 'return', value: undefined },
//   { type: 'return', value: undefined },
//   { type: 'return', value: undefined },
// ]

// All `this` contexts
console.log(mockFn.mock.contexts);

// Number of calls
console.log(mockFn.mock.calls.length); // 3

// Last call arguments
console.log(mockFn.mock.lastCall); // ['c', 3]
```

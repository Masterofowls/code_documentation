Mocking is the practice of replacing real dependencies with controlled substitutes during tests. Jest provides a complete, built-in mocking system that requires no external libraries.

## `jest.fn()` — Mock Functions

`jest.fn()` creates a standalone mock function that records every call made to it.

### Basic Usage

```typescript
const mockAdd = jest.fn();

mockAdd(1, 2);
mockAdd(3, 4);

expect(mockAdd).toHaveBeenCalledTimes(2);
expect(mockAdd).toHaveBeenCalledWith(1, 2);
expect(mockAdd).toHaveBeenLastCalledWith(3, 4);
expect(mockAdd).toHaveBeenNthCalledWith(1, 1, 2);
```

### Controlling Return Values

```typescript
const mockFetch = jest.fn();

// Return the same value on every call
mockFetch.mockReturnValue({ status: 200, data: 'ok' });

// Return different values on successive calls
mockFetch
  .mockReturnValueOnce({ status: 200 })
  .mockReturnValueOnce({ status: 404 })
  .mockReturnValue({ status: 500 }); // default for all remaining calls
```

### Controlling Resolved / Rejected Promises

```typescript
const mockGetUser = jest.fn();

// Resolve with a value
mockGetUser.mockResolvedValue({ id: 1, name: 'Alice' });

// Reject with an error
mockGetUser.mockRejectedValue(new Error('Network error'));

// Once variants
mockGetUser
  .mockResolvedValueOnce({ id: 1, name: 'Alice' })
  .mockRejectedValueOnce(new Error('Rate limited'));
```

### Custom Implementations

```typescript
const mockMultiply = jest.fn().mockImplementation((a: number, b: number) => a * b);
expect(mockMultiply(3, 4)).toBe(12);

// One-time implementation — reverts to default after one call
const mockRandom = jest.fn()
  .mockImplementationOnce(() => 0.5)
  .mockImplementation(() => Math.random());

expect(mockRandom()).toBe(0.5);    // first call
expect(mockRandom()).not.toBe(0.5); // subsequent calls use Math.random
```

### Inspecting Call History

```typescript
const mockSave = jest.fn();

mockSave({ id: 1 });
mockSave({ id: 2 });

// Access call arguments
console.log(mockSave.mock.calls);
// [[{ id: 1 }], [{ id: 2 }]]

// Access return values
console.log(mockSave.mock.results);
// [{ type: 'return', value: undefined }, ...]

// Access instances (when used as a constructor)
console.log(mockSave.mock.instances);
```

## `jest.spyOn` — Non-Replacing Spies

`jest.spyOn` creates a spy on an _existing_ method without replacing its implementation. It lets you observe calls while the real function continues to execute.

```typescript
const mathObj = {
  add: (a: number, b: number) => a + b,
};

const spy = jest.spyOn(mathObj, 'add');

const result = mathObj.add(2, 3);

expect(result).toBe(5);               // real implementation ran
expect(spy).toHaveBeenCalledWith(2, 3);
expect(spy).toHaveBeenCalledTimes(1);
```

### Overriding the Implementation

You can replace the spy's implementation temporarily:

```typescript
const consoleSpy = jest.spyOn(console, 'error').mockImplementation(() => undefined);

// Code under test calls console.error internally
runSomethingThatLogsErrors();

expect(consoleSpy).toHaveBeenCalledWith(expect.stringContaining('Warning'));

// Restore the real console.error
consoleSpy.mockRestore();
```

### Spying on Class Methods

```typescript
import { UserService } from './UserService';

const service = new UserService();
const spy = jest.spyOn(service, 'findById').mockResolvedValue({ id: 1, name: 'Alice' });

const user = await service.findById(1);

expect(user).toEqual({ id: 1, name: 'Alice' });
expect(spy).toHaveBeenCalledWith(1);

spy.mockRestore();
```

:::tip[spyOn vs fn()]
Use `jest.spyOn` when you want to observe calls to an existing method on a real object, optionally overriding its behaviour. Use `jest.fn()` for standalone mock functions that don't replace an existing implementation.
:::

## `jest.mock()` — Module Mocking

`jest.mock()` replaces an entire module with an automatically-generated mock. Jest hoists `jest.mock()` calls to the top of the file (before imports), so they work even with ES module syntax.

### Auto-Mock

Auto-mocking replaces every exported function with a `jest.fn()`:

```typescript
jest.mock('./emailService');

import { sendEmail } from './emailService';

it('calls sendEmail with correct arguments', async () => {
  await registerUser({ name: 'Alice', email: 'alice@example.com' });

  expect(sendEmail).toHaveBeenCalledWith({
    to: 'alice@example.com',
    subject: 'Welcome!',
    body: expect.stringContaining('Alice'),
  });
});
```

### Manual Mock with Factory Function

Provide a factory to control what the mock exports:

```typescript
jest.mock('./database', () => ({
  query: jest.fn(),
  connect: jest.fn().mockResolvedValue(true),
  disconnect: jest.fn().mockResolvedValue(true),
}));

import { query } from './database';

it('queries the database', async () => {
  (query as jest.Mock).mockResolvedValueOnce([{ id: 1, name: 'Alice' }]);

  const result = await getUsers();

  expect(result).toHaveLength(1);
  expect(query).toHaveBeenCalledWith('SELECT * FROM users');
});
```

### Partial Mock with `jest.requireActual`

Keep some real exports while mocking others:

```typescript
jest.mock('./config', () => ({
  ...jest.requireActual('./config'),   // keep all real exports
  API_URL: 'http://localhost:3000',    // override just this one
  DEBUG: false,
}));
```

### Mocking Default Exports

```typescript
// axios has a default export
jest.mock('axios', () => ({
  default: {
    get: jest.fn(),
    post: jest.fn(),
  },
  get: jest.fn(),
  post: jest.fn(),
}));

import axios from 'axios';

it('fetches users', async () => {
  (axios.get as jest.Mock).mockResolvedValueOnce({
    data: [{ id: 1, name: 'Alice' }],
  });

  const users = await fetchUsers();
  expect(users).toHaveLength(1);
});
```

### Mocking Named Exports

```typescript
// Module: ./validators.ts
// export function isEmail(s: string): boolean { ... }
// export function isPhone(s: string): boolean { ... }

jest.mock('./validators', () => ({
  isEmail: jest.fn().mockReturnValue(true),
  isPhone: jest.fn().mockReturnValue(false),
}));
```

## `__mocks__` Directory

Place manual mock implementations in a `__mocks__` folder adjacent to the module:

```
src/
├── services/
│   ├── __mocks__/
│   │   └── emailService.ts   ← manual mock for emailService.ts
│   └── emailService.ts
└── __mocks__/
    └── axios.ts              ← manual mock for node_modules/axios
```

```typescript
// src/services/__mocks__/emailService.ts
export const sendEmail = jest.fn().mockResolvedValue({ messageId: 'mock-id' });
export const sendBulkEmail = jest.fn().mockResolvedValue({ sent: 0, failed: 0 });
```

For node_modules mocks, place the file in the root-level `__mocks__` directory. It is used **automatically** (without calling `jest.mock('axios')`):

```typescript
// __mocks__/axios.ts
const axiosMock = {
  get: jest.fn(),
  post: jest.fn(),
  put: jest.fn(),
  delete: jest.fn(),
  create: jest.fn().mockReturnThis(),
  defaults: { headers: { common: {} } },
};

export default axiosMock;
```

:::caution[Automock vs Manual Mock]
For modules in `node_modules`, placing a file in `__mocks__` at the project root makes it the automatic mock. For local modules, you still need to call `jest.mock('./path/to/module')` — the `__mocks__` file is only used when that call is made.
:::

## Mocking ES Modules

When your project uses native ESM (`"type": "module"` in `package.json`), Jest requires additional setup:

```typescript
// jest.config.ts
export default {
  extensionsToTreatAsEsm: ['.ts'],
  transform: {
    '^.+\\.tsx?$': ['ts-jest', { useESM: true }],
  },
};
```

With `unstable_mockModule` for dynamic ES module mocking:

```typescript
const mockSendEmail = jest.fn();

// Must use unstable_mockModule for ES modules
jest.unstable_mockModule('./emailService.js', () => ({
  sendEmail: mockSendEmail,
}));

// Import AFTER mocking
const { registerUser } = await import('./userService.js');
```

## Mocking Timers

Jest can take control of `setTimeout`, `setInterval`, `clearTimeout`, `clearInterval`, `Date`, and `queueMicrotask`.

### Basic Timer Control

```typescript
jest.useFakeTimers();

it('calls callback after 1 second', () => {
  const callback = jest.fn();

  setTimeout(callback, 1000);

  expect(callback).not.toHaveBeenCalled();

  jest.advanceTimersByTime(1000); // advance clock by 1s

  expect(callback).toHaveBeenCalledTimes(1);
});

afterEach(() => {
  jest.useRealTimers(); // restore real timers
});
```

### Timer Helpers

```typescript
jest.useFakeTimers();

const callback = jest.fn();
setTimeout(callback, 100);
setTimeout(callback, 200);
setTimeout(callback, 300);

// Run all pending timers regardless of delay
jest.runAllTimers();
expect(callback).toHaveBeenCalledTimes(3);

// Run only currently-pending timers (not those added by callbacks)
jest.runOnlyPendingTimers();

// Advance by a specific number of milliseconds
jest.advanceTimersByTime(150);  // runs the 100ms timer only

// Advance until no more timers are pending
jest.runAllTimersAsync(); // async variant for Promises
```

### Mocking `Date`

```typescript
jest.useFakeTimers();
jest.setSystemTime(new Date('2024-01-15T10:00:00.000Z'));

const result = getCurrentTimestamp();
expect(result).toBe('2024-01-15T10:00:00.000Z');

jest.useRealTimers();
```

## Clearing, Resetting, and Restoring Mocks

Understanding the difference between these three operations is essential:

| Method | Effect |
|--------|--------|
| `mockFn.mockClear()` | Clears call history and instances; **keeps** implementation |
| `mockFn.mockReset()` | Clears everything + **removes** implementation (returns `undefined`) |
| `mockFn.mockRestore()` | Only works on `jest.spyOn` — restores the original function |
| `jest.clearAllMocks()` | `mockClear()` on every mock in the test file |
| `jest.resetAllMocks()` | `mockReset()` on every mock in the test file |
| `jest.restoreAllMocks()` | `mockRestore()` on every spy in the test file |

```typescript
const mockFn = jest.fn().mockReturnValue(42);

mockFn();
mockFn();

mockFn.mockClear();
expect(mockFn).not.toHaveBeenCalled();  // call history cleared
expect(mockFn()).toBe(42);               // implementation still works

mockFn.mockReset();
expect(mockFn()).toBeUndefined();        // implementation also cleared
```

### Automating Cleanup

Configure Jest to automatically clear/reset mocks after each test in `jest.config.ts`:

```typescript
const config: Config = {
  clearMocks: true,      // jest.clearAllMocks() after each test
  resetMocks: false,     // jest.resetAllMocks() after each test
  restoreMocks: true,    // jest.restoreAllMocks() after each test
};
```

:::tip[Recommended Setup]
Enable `clearMocks: true` and `restoreMocks: true` in your config. This ensures each test starts with a clean mock state while keeping mock implementations (so you don't have to re-declare them in every test).
:::

`@testing-library/react` is the standard library for testing React components with Jest. It encourages testing from the user's perspective — interacting with the component as a user would, using accessible roles and labels rather than implementation details.

## Installation and Setup

```bash
npm install --save-dev @testing-library/react @testing-library/user-event \
  @testing-library/jest-dom jest-environment-jsdom
```

Configure Jest for jsdom and custom matchers:

```typescript
// jest.config.ts
import type { Config } from 'jest';

const config: Config = {
  testEnvironment: 'jest-environment-jsdom',
  setupFilesAfterFramework: ['<rootDir>/src/test-setup.ts'],
};

export default config;
```

```typescript
// src/test-setup.ts
import '@testing-library/jest-dom';
```

`@testing-library/jest-dom` adds custom matchers like `toBeInTheDocument()`, `toHaveClass()`, `toHaveValue()`, and `toBeDisabled()`.

## render()

`render()` mounts a React component into a jsdom DOM node and returns query utilities:

```typescript
import { render, screen } from '@testing-library/react';
import { UserCard } from './UserCard';

it('displays the user name and role', () => {
  render(<UserCard name="Alice Johnson" role="Admin" avatarUrl="/alice.jpg" />);

  expect(screen.getByRole('heading', { name: 'Alice Johnson' })).toBeInTheDocument();
  expect(screen.getByText('Admin')).toBeInTheDocument();
  expect(screen.getByRole('img', { name: 'Alice Johnson' })).toHaveAttribute(
    'src',
    '/alice.jpg'
  );
});
```

`render()` returns several utilities:

```typescript
const {
  container,          // the DOM node wrapping the rendered component
  baseElement,        // document.body
  debug,              // prints the current DOM to console
  rerender,           // re-render with new props
  unmount,            // unmount the component
  asFragment,         // returns a DocumentFragment (useful for snapshots)
} = render(<MyComponent prop="value" />);

// Re-render with new props
rerender(<MyComponent prop="updated" />);

// Debug the current DOM
debug(); // prints to console
debug(screen.getByRole('button')); // prints just the button
```

## Screen Queries

`screen` provides query functions bound to `document.body`. Queries come in three variants:

| Variant | Returns | Throws if missing? | Use when |
|---|---|---|---|
| `getBy*` | Element | ✅ Yes | Element must be in DOM right now |
| `queryBy*` | Element or `null` | ❌ No | Asserting element is **not** present |
| `findBy*` | Promise\<Element\> | ✅ Yes (async) | Element appears asynchronously |
| `getAllBy*` | Element[] | ✅ if empty | Multiple elements expected |
| `queryAllBy*` | Element[] (empty OK) | ❌ No | Multiple elements, may be empty |
| `findAllBy*` | Promise\<Element[]\> | ✅ if empty | Multiple async elements |

### Query Priority (Accessibility-First)

The Testing Library recommends this priority order — prefer queries higher in the list:

| Priority | Query | Finds by |
|---|---|---|
| 1 | `getByRole` | ARIA role + accessible name |
| 2 | `getByLabelText` | `<label>` text |
| 3 | `getByPlaceholderText` | Input placeholder |
| 4 | `getByText` | Visible text content |
| 5 | `getByDisplayValue` | Current form field value |
| 6 | `getByAltText` | Image alt text |
| 7 | `getByTitle` | Title attribute |
| 8 | `getByTestId` | `data-testid` attribute (last resort) |

### getByRole

```typescript
// Buttons
screen.getByRole('button', { name: 'Submit' });
screen.getByRole('button', { name: /submit/i }); // regex

// Links
screen.getByRole('link', { name: 'Learn more' });

// Headings
screen.getByRole('heading', { name: 'Dashboard', level: 2 });

// Form inputs
screen.getByRole('textbox', { name: 'Email address' });
screen.getByRole('checkbox', { name: 'Remember me' });
screen.getByRole('combobox', { name: 'Country' });
screen.getByRole('spinbutton', { name: 'Quantity' }); // number input

// Lists
screen.getByRole('list');
screen.getAllByRole('listitem');

// Dialogs
screen.getByRole('dialog', { name: 'Confirm Delete' });

// Tables
screen.getByRole('table');
screen.getAllByRole('row');
screen.getAllByRole('cell');
```

### getByText, getByLabel, getByTestId

```typescript
// getByText
screen.getByText('Hello World');            // exact match
screen.getByText('Hello', { exact: false }); // substring match
screen.getByText(/hello/i);                 // regex (case-insensitive)

// getByLabelText — finds input associated with <label>
screen.getByLabelText('Email Address');
screen.getByLabelText(/email/i);
screen.getByLabelText('Password', { selector: 'input[type="password"]' });

// getByPlaceholderText
screen.getByPlaceholderText('Enter your email');

// getByTestId — only when no accessible query works
// Requires data-testid="your-id" on the element
screen.getByTestId('loading-spinner');
screen.getByTestId('dropdown-menu');
```

## userEvent vs fireEvent

Prefer `userEvent` over `fireEvent`. `userEvent` simulates realistic user interactions, dispatching the full sequence of events (pointerdown, mousedown, focus, keydown, keyup, input, change, click, etc.) that a real browser would fire.

| Action | `fireEvent` | `userEvent` |
|---|---|---|
| Click a button | `fireEvent.click(btn)` | `await userEvent.click(btn)` |
| Type in a field | `fireEvent.change(input, { target: { value: 'hi' } })` | `await userEvent.type(input, 'hi')` |
| Realistic events | Only dispatches one event | Dispatches all related events |
| Requires setup | No | Requires `userEvent.setup()` |

```typescript
import userEvent from '@testing-library/user-event';

describe('LoginForm', () => {
  it('submits credentials when form is filled and submitted', async () => {
    // Setup userEvent with fake timers support
    const user = userEvent.setup();

    const onSubmit = jest.fn();
    render(<LoginForm onSubmit={onSubmit} />);

    // Click — triggers full mouse event chain
    await user.click(screen.getByRole('textbox', { name: 'Email' }));

    // Type — triggers keydown, keypress, input, keyup for each character
    await user.type(screen.getByRole('textbox', { name: 'Email' }), 'alice@example.com');
    await user.type(screen.getByLabelText('Password'), 'secret123');

    // Clear a field before typing
    await user.clear(screen.getByRole('textbox', { name: 'Email' }));
    await user.type(screen.getByRole('textbox', { name: 'Email' }), 'new@email.com');

    // Click submit button
    await user.click(screen.getByRole('button', { name: 'Sign In' }));

    expect(onSubmit).toHaveBeenCalledWith({
      email: 'new@email.com',
      password: 'secret123',
    });
  });

  it('selects an option from a dropdown', async () => {
    const user = userEvent.setup();
    render(<CountrySelector />);

    await user.selectOptions(
      screen.getByRole('combobox', { name: 'Country' }),
      'United States'
    );

    expect(screen.getByRole('combobox', { name: 'Country' })).toHaveValue('US');
  });

  it('checks and unchecks a checkbox', async () => {
    const user = userEvent.setup();
    render(<NewsletterSignup />);

    const checkbox = screen.getByRole('checkbox', { name: 'Subscribe to newsletter' });

    expect(checkbox).not.toBeChecked();
    await user.click(checkbox);
    expect(checkbox).toBeChecked();
    await user.click(checkbox);
    expect(checkbox).not.toBeChecked();
  });

  it('uses keyboard shortcuts', async () => {
    const user = userEvent.setup();
    render(<Textarea />);

    await user.click(screen.getByRole('textbox'));
    await user.keyboard('{Control>}a{/Control}'); // Ctrl+A to select all
    await user.keyboard('{Delete}');              // Delete selected
    await user.type(screen.getByRole('textbox'), 'new content');
  });
});
```

## waitFor and Async Rendering

For components that update asynchronously (after a fetch, timeout, or state change), use `waitFor` or `findBy*` queries:

```typescript
import { render, screen, waitFor } from '@testing-library/react';
import userEvent from '@testing-library/user-event';

it('shows search results after query submission', async () => {
  const user = userEvent.setup();

  render(<SearchPage />);

  await user.type(screen.getByRole('textbox', { name: 'Search' }), 'React');
  await user.click(screen.getByRole('button', { name: 'Search' }));

  // findByText is equivalent to waitFor(() => screen.getByText(...))
  const firstResult = await screen.findByText('React Testing Library');
  expect(firstResult).toBeInTheDocument();

  // Alternatively, use waitFor for more complex assertions
  await waitFor(() => {
    expect(screen.getAllByRole('listitem')).toHaveLength(5);
  });
});

it('removes loading indicator after data loads', async () => {
  render(<DataGrid />);

  // Loading state
  expect(screen.getByTestId('loading-spinner')).toBeInTheDocument();

  // waitForElementToBeRemoved — cleaner than polling
  await waitForElementToBeRemoved(() => screen.queryByTestId('loading-spinner'));

  // Data is now loaded
  expect(screen.getByRole('table')).toBeInTheDocument();
  expect(screen.getAllByRole('row')).toHaveLength(11); // header + 10 data rows
});
```

## Testing Forms with Validation

```typescript
it('shows validation errors when form is submitted empty', async () => {
  const user = userEvent.setup();
  render(<RegistrationForm />);

  // Submit without filling in any fields
  await user.click(screen.getByRole('button', { name: 'Register' }));

  // Validation messages should appear
  expect(await screen.findByText('Name is required')).toBeInTheDocument();
  expect(screen.getByText('Email is required')).toBeInTheDocument();
  expect(screen.getByText('Password must be at least 8 characters')).toBeInTheDocument();

  // Form should not have submitted
  expect(screen.queryByText('Registration successful!')).not.toBeInTheDocument();
});

it('disables the submit button while submitting', async () => {
  const user = userEvent.setup();
  const onSubmit = jest.fn(() => new Promise(resolve => setTimeout(resolve, 1000)));
  render(<ContactForm onSubmit={onSubmit} />);

  await user.type(screen.getByLabelText('Name'), 'Alice');
  await user.type(screen.getByLabelText('Message'), 'Hello!');

  const submitBtn = screen.getByRole('button', { name: 'Send' });
  await user.click(submitBtn);

  // Button should be disabled during submission
  expect(submitBtn).toBeDisabled();

  // After submission completes
  await waitFor(() => expect(submitBtn).not.toBeDisabled());
});
```

## Testing Hooks with renderHook

```typescript
import { renderHook, act } from '@testing-library/react';
import { useCounter } from './useCounter';

describe('useCounter', () => {
  it('starts at the initial value', () => {
    const { result } = renderHook(() => useCounter(10));
    expect(result.current.count).toBe(10);
  });

  it('increments the count', () => {
    const { result } = renderHook(() => useCounter(0));

    act(() => {
      result.current.increment();
    });

    expect(result.current.count).toBe(1);
  });

  it('resets to the initial value', () => {
    const { result } = renderHook(() => useCounter(5));

    act(() => {
      result.current.increment();
      result.current.increment();
    });

    expect(result.current.count).toBe(7);

    act(() => {
      result.current.reset();
    });

    expect(result.current.count).toBe(5);
  });
});

// Testing a hook that fetches data
describe('useUsers', () => {
  it('loads users on mount', async () => {
    global.fetch = jest.fn().mockResolvedValue({
      ok: true,
      json: async () => [{ id: 1, name: 'Alice' }],
    });

    const { result } = renderHook(() => useUsers());

    expect(result.current.loading).toBe(true);

    await waitFor(() => expect(result.current.loading).toBe(false));

    expect(result.current.users).toHaveLength(1);
    expect(result.current.users[0].name).toBe('Alice');
  });
});
```

## Testing with Context

```typescript
// Wrap in context provider using the `wrapper` option
import { ThemeContext } from './ThemeContext';
import { ThemedButton } from './ThemedButton';

const ThemeWrapper = ({ children }: { children: React.ReactNode }) => (
  <ThemeContext.Provider value={{ theme: 'dark', toggle: jest.fn() }}>
    {children}
  </ThemeContext.Provider>
);

it('applies the dark theme class', () => {
  render(<ThemedButton label="Click me" />, { wrapper: ThemeWrapper });

  expect(screen.getByRole('button', { name: 'Click me' })).toHaveClass('btn-dark');
});

// Create a custom render function for reuse across tests
import { render as rtlRender } from '@testing-library/react';

function renderWithProviders(
  ui: React.ReactElement,
  { theme = 'light', ...options } = {}
) {
  const Wrapper = ({ children }: { children: React.ReactNode }) => (
    <ThemeContext.Provider value={{ theme, toggle: jest.fn() }}>
      <QueryClientProvider client={new QueryClient()}>
        {children}
      </QueryClientProvider>
    </ThemeContext.Provider>
  );
  return rtlRender(ui, { wrapper: Wrapper, ...options });
}
```

## Testing with React Router

```typescript
import { MemoryRouter, Route, Routes } from 'react-router-dom';

it('navigates to the profile page on link click', async () => {
  const user = userEvent.setup();

  render(
    <MemoryRouter initialEntries={['/dashboard']}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/profile" element={<Profile />} />
      </Routes>
    </MemoryRouter>
  );

  await user.click(screen.getByRole('link', { name: 'My Profile' }));

  expect(await screen.findByRole('heading', { name: 'My Profile' })).toBeInTheDocument();
});
```

## act() Wrapper

`act()` ensures all React state updates and effects are flushed before making assertions. Most Testing Library utilities wrap their operations in `act()` automatically — but you may need it directly when testing hooks or triggering updates from outside React:

```typescript
import { act } from '@testing-library/react';

it('handles external state changes', async () => {
  const { result } = renderHook(() => useWebSocket('ws://localhost:8080'));

  // Simulate receiving a message from outside React
  await act(async () => {
    result.current.simulateMessage({ type: 'update', data: { count: 5 } });
  });

  expect(result.current.messageCount).toBe(5);
});
```

:::tip[act() is Automatic in Most Cases]
When using `userEvent`, `fireEvent`, `waitFor`, `findBy*` queries, or `render`/`rerender`, Testing Library wraps everything in `act()` for you. Only reach for `act()` directly when you're manually triggering React state changes outside these utilities.
:::

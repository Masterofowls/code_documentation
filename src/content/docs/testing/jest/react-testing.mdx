`@testing-library/react` is the de-facto standard for testing React components. Its philosophy is to test components the way users interact with them — through the DOM and accessible semantics — rather than testing implementation details.

## Setup

```bash
npm install --save-dev @testing-library/react @testing-library/user-event @testing-library/jest-dom jest-environment-jsdom
```

Configure Jest to use jsdom and import jest-dom's custom matchers:

```typescript
// jest.config.ts
import type { Config } from 'jest';

const config: Config = {
  testEnvironment: 'jest-environment-jsdom',
  setupFilesAfterFramework: ['<rootDir>/src/setupTests.ts'],
};

export default config;
```

```typescript
// src/setupTests.ts
import '@testing-library/jest-dom';
```

Now every test file has access to custom matchers like `toBeInTheDocument()`, `toBeVisible()`, `toHaveTextContent()`, and 30+ more.

## `render()` — Mounting Components

```typescript
import { render } from '@testing-library/react';
import { UserCard } from './UserCard';

test('renders the user name', () => {
  const { getByText, getByRole, container } = render(
    <UserCard name="Alice" role="Admin" email="alice@example.com" />
  );

  expect(getByText('Alice')).toBeInTheDocument();
  expect(getByRole('heading', { name: 'Alice' })).toBeInTheDocument();
});
```

`render()` returns queries scoped to the rendered component. It also mounts the component into a `div` appended to `document.body`.

### The `screen` object

Prefer using the globally accessible `screen` object instead of destructuring from `render()`:

```typescript
import { render, screen } from '@testing-library/react';

test('renders a submit button', () => {
  render(<LoginForm />);

  // Using screen is recommended — easier to use in multi-render tests
  expect(screen.getByRole('button', { name: /submit/i })).toBeInTheDocument();
});
```

## Screen Queries Reference

| Query type      | 0 matches      | 1 match                 | 2+ matches            | Use case                     |
| --------------- | -------------- | ----------------------- | --------------------- | ---------------------------- |
| `getBy...`      | Throws         | Returns element         | Throws                | Element must exist           |
| `queryBy...`    | Returns `null` | Returns element         | Throws                | Element may not exist        |
| `findBy...`     | Throws (async) | Returns element (async) | Throws                | Element appears async        |
| `getAllBy...`   | Throws         | Returns array           | Returns array         | Multiple elements must exist |
| `queryAllBy...` | Returns `[]`   | Returns array           | Returns array         | Multiple elements (optional) |
| `findAllBy...`  | Throws (async) | Returns array (async)   | Returns array (async) | Multiple async elements      |

### Available query selectors

```typescript
// By ARIA role (most preferred — tests accessibility)
screen.getByRole('button', { name: /submit/i });
screen.getByRole('heading', { level: 1 });
screen.getByRole('textbox', { name: /email/i });
screen.getByRole('checkbox', { name: /agree/i });
screen.getByRole('combobox', { name: /country/i });
screen.getByRole('listitem');
screen.getByRole('dialog', { name: /confirm delete/i });

// By text content
screen.getByText('Submit');
screen.getByText(/submit/i); // case-insensitive regex
screen.getByText('Submit', { exact: false }); // substring match

// By label (form elements)
screen.getByLabelText('Email address');
screen.getByLabelText(/email/i);

// By placeholder
screen.getByPlaceholderText('Enter your email');

// By test ID (last resort — not tied to UI or accessibility)
screen.getByTestId('submit-button');

// By alt text (images)
screen.getByAltText('Company logo');

// By title attribute
screen.getByTitle('Close dialog');
```

:::tip[Accessibility-First Query Priority]
Use queries in this order of preference:

1. `getByRole` — most resilient, tests accessibility
2. `getByLabelText` — great for form fields
3. `getByPlaceholderText` — fallback for unlabelled inputs
4. `getByText` — for non-interactive content
5. `getByTestId` — last resort, add `data-testid` to the DOM

This order encourages building accessible UIs.
:::

## `userEvent` vs `fireEvent`

`@testing-library/user-event` simulates real user interactions, triggering all the browser events a real interaction would produce. `fireEvent` dispatches a single synthetic DOM event.

| Feature                       | `userEvent`                                                                                                         | `fireEvent`                   |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Simulates real browser events | Yes (full event sequence)                                                                                           | No (single event)             |
| Typing                        | Triggers `keydown`, `keypress`, `input`, `keyup` per character                                                      | Dispatches one `change` event |
| Clicking                      | `pointerover`, `mouseover`, `pointermove`, `mousemove`, `pointerdown`, `mousedown`, `pointerup`, `mouseup`, `click` | Single `click` event          |
| Keyboard shortcuts            | Yes                                                                                                                 | Manual                        |
| Async                         | Returns a Promise (v14+)                                                                                            | Synchronous                   |

**Always prefer `userEvent`** for tests that reflect real user interactions:

```typescript
import userEvent from '@testing-library/user-event';
import { render, screen } from '@testing-library/react';
import { LoginForm } from './LoginForm';

test('submits the form with email and password', async () => {
  const user = userEvent.setup();
  const mockSubmit = jest.fn();

  render(<LoginForm onSubmit={mockSubmit} />);

  await user.type(screen.getByLabelText(/email/i), 'alice@example.com');
  await user.type(screen.getByLabelText(/password/i), 'secret123');
  await user.click(screen.getByRole('button', { name: /sign in/i }));

  expect(mockSubmit).toHaveBeenCalledWith({
    email: 'alice@example.com',
    password: 'secret123',
  });
});
```

### Common `userEvent` interactions

```typescript
const user = userEvent.setup();

// Typing
await user.type(input, 'Hello World');
await user.clear(input); // clear existing value
await user.type(input, '{Enter}'); // special keys

// Clicking
await user.click(button);
await user.dblClick(element);
await user.rightClick(element);

// Select dropdown
await user.selectOptions(select, 'option-value');
await user.selectOptions(select, ['value1', 'value2']); // multi-select
await user.deselectOptions(select, 'option-value');

// Keyboard
await user.keyboard('{Tab}');
await user.keyboard('{Escape}');
await user.keyboard('{Control>}a{/Control}'); // Ctrl+A

// Hover
await user.hover(element);
await user.unhover(element);

// File upload
await user.upload(fileInput, new File(['content'], 'test.txt'));
```

## `waitFor` and `waitForElementToBeRemoved`

Use these utilities for content that appears or disappears asynchronously:

```typescript
import { render, screen, waitFor, waitForElementToBeRemoved } from '@testing-library/react';
import userEvent from '@testing-library/user-event';

test('shows user list after loading', async () => {
  render(<UserList />);

  // Wait for loading spinner to disappear
  await waitForElementToBeRemoved(screen.getByText(/loading/i));

  // Now assert on the loaded content
  expect(screen.getByText('Alice')).toBeInTheDocument();
  expect(screen.getByText('Bob')).toBeInTheDocument();
});

test('displays error message on failed submission', async () => {
  const user = userEvent.setup();
  render(<ContactForm />);

  await user.click(screen.getByRole('button', { name: /submit/i }));

  // Wait for async validation or API response
  await waitFor(() => {
    expect(screen.getByRole('alert')).toHaveTextContent(/email is required/i);
  });
});
```

`waitFor` polls the assertion until it passes or the timeout expires (default: 1000ms). Use `screen.findBy*` as a shorthand for `waitFor` + `getBy*`:

```typescript
// These are equivalent:
await waitFor(() => screen.getByText('Alice'));
await screen.findByText('Alice');
```

## Testing Forms

```typescript
// src/components/RegistrationForm.tsx
function RegistrationForm({ onSuccess }: { onSuccess: (user: User) => void }) {
  const [errors, setErrors] = useState<Record<string, string>>({});

  const handleSubmit = async (e: FormEvent) => {
    e.preventDefault();
    const formData = new FormData(e.target as HTMLFormElement);
    const data = Object.fromEntries(formData);

    const validation = validateRegistration(data);
    if (!validation.valid) {
      setErrors(validation.errors);
      return;
    }

    const user = await registerUser(data);
    onSuccess(user);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name
        <input name="name" aria-describedby={errors.name ? 'name-error' : undefined} />
      </label>
      {errors.name && <p id="name-error" role="alert">{errors.name}</p>}
      <label>
        Email
        <input name="email" type="email" />
      </label>
      {errors.email && <p role="alert">{errors.email}</p>}
      <button type="submit">Register</button>
    </form>
  );
}
```

```typescript
// src/components/RegistrationForm.test.tsx
import { render, screen, waitFor } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { RegistrationForm } from './RegistrationForm';

jest.mock('../services/userService', () => ({
  registerUser: jest.fn().mockResolvedValue({ id: 1, name: 'Alice' }),
}));

describe('RegistrationForm', () => {
  const user = userEvent.setup();

  it('calls onSuccess with user data on valid submission', async () => {
    const onSuccess = jest.fn();
    render(<RegistrationForm onSuccess={onSuccess} />);

    await user.type(screen.getByLabelText(/name/i), 'Alice');
    await user.type(screen.getByLabelText(/email/i), 'alice@example.com');
    await user.click(screen.getByRole('button', { name: /register/i }));

    await waitFor(() => {
      expect(onSuccess).toHaveBeenCalledWith({ id: 1, name: 'Alice' });
    });
  });

  it('shows validation errors for empty fields', async () => {
    render(<RegistrationForm onSuccess={jest.fn()} />);

    await user.click(screen.getByRole('button', { name: /register/i }));

    expect(await screen.findAllByRole('alert')).toHaveLength(2);
  });
});
```

## Testing Custom Hooks with `renderHook`

```typescript
// src/hooks/useCounter.ts
import { useState } from 'react';

export function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);
  const increment = () => setCount((c) => c + 1);
  const decrement = () => setCount((c) => c - 1);
  const reset = () => setCount(initialValue);
  return { count, increment, decrement, reset };
}
```

```typescript
// src/hooks/useCounter.test.ts
import { renderHook, act } from '@testing-library/react';
import { useCounter } from './useCounter';

describe('useCounter', () => {
  it('initialises with the provided value', () => {
    const { result } = renderHook(() => useCounter(10));
    expect(result.current.count).toBe(10);
  });

  it('increments the count', () => {
    const { result } = renderHook(() => useCounter());

    act(() => {
      result.current.increment();
    });

    expect(result.current.count).toBe(1);
  });

  it('resets to the initial value', () => {
    const { result } = renderHook(() => useCounter(5));

    act(() => {
      result.current.increment();
      result.current.increment();
      result.current.reset();
    });

    expect(result.current.count).toBe(5);
  });
});
```

## Testing Context

```typescript
// src/context/ThemeContext.tsx
export const ThemeContext = createContext<{ theme: string; toggle: () => void } | null>(null);

export function ThemeProvider({ children }: { children: ReactNode }) {
  const [theme, setTheme] = useState<'light' | 'dark'>('light');
  const toggle = () => setTheme(t => t === 'light' ? 'dark' : 'light');
  return (
    <ThemeContext.Provider value={{ theme, toggle }}>
      {children}
    </ThemeContext.Provider>
  );
}
```

```typescript
// src/components/ThemedButton.test.tsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { ThemeProvider } from '../context/ThemeContext';
import { ThemedButton } from './ThemedButton';

function renderWithTheme(ui: ReactElement) {
  return render(<ThemeProvider>{ui}</ThemeProvider>);
}

test('applies the current theme class', () => {
  renderWithTheme(<ThemedButton>Click me</ThemedButton>);
  expect(screen.getByRole('button')).toHaveClass('theme-light');
});

test('updates when theme is toggled', async () => {
  const user = userEvent.setup();
  renderWithTheme(
    <>
      <ThemeToggle />
      <ThemedButton>Click me</ThemedButton>
    </>
  );

  await user.click(screen.getByRole('button', { name: /toggle theme/i }));
  expect(screen.getByRole('button', { name: /click me/i })).toHaveClass('theme-dark');
});
```

## Testing with React Router

```typescript
import { render, screen } from '@testing-library/react';
import { MemoryRouter, Route, Routes } from 'react-router-dom';
import { UserProfile } from './UserProfile';

function renderWithRouter(ui: ReactElement, { route = '/' } = {}) {
  return render(
    <MemoryRouter initialEntries={[route]}>
      {ui}
    </MemoryRouter>
  );
}

test('renders user profile for the given ID', async () => {
  renderWithRouter(
    <Routes>
      <Route path="/users/:id" element={<UserProfile />} />
    </Routes>,
    { route: '/users/42' }
  );

  await screen.findByRole('heading', { name: /user profile/i });
  expect(screen.getByText('ID: 42')).toBeInTheDocument();
});
```

## The `act()` Wrapper

React Testing Library wraps most interactions in `act()` automatically. You need to call it manually only when triggering state updates outside of RTL's utilities:

```typescript
import { act } from '@testing-library/react';

test('updates count after external event', () => {
  const { result } = renderHook(() => useExternalCounter());

  act(() => {
    externalEventEmitter.emit('increment');
  });

  expect(result.current.count).toBe(1);
});
```

:::caution[act() warning in tests]
If you see "An update to Component inside a test was not wrapped in act()" in your test output, it means an async state update is happening outside of RTL's wrapped interactions. Wrap the triggering action in `act()` or use `waitFor()` to wait for the update to complete.
:::

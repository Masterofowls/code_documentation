`@testing-library/react` (RTL) provides utilities for rendering React components and querying them the way a real user would. It intentionally avoids exposing component internals — testing what the user sees and does rather than implementation details.

## Setup

<Steps>
1. Install the testing library packages:

    ```bash
    npm install --save-dev @testing-library/react @testing-library/user-event @testing-library/jest-dom
    ```

2. Configure Jest to use jsdom:

    ```typescript
    // jest.config.ts
    const config: Config = {
      testEnvironment: 'jsdom',
      setupFilesAfterFramework: ['<rootDir>/jest.setup.ts'],
    };
    ```

3. Create the setup file:

    ```typescript
    // jest.setup.ts
    import '@testing-library/jest-dom';
    // Now you can use: toBeInTheDocument(), toBeVisible(), toHaveValue(), etc.
    ```
</Steps>

## `render()`

`render()` mounts a React component into a simulated DOM. It returns a collection of query utilities bound to that component's output:

```typescript
import { render, screen } from '@testing-library/react';
import { UserProfile } from './UserProfile';

it('renders the user profile', () => {
  render(<UserProfile name="Alice" role="Admin" email="alice@example.com" />);

  expect(screen.getByRole('heading', { name: 'Alice' })).toBeInTheDocument();
  expect(screen.getByText('Admin')).toBeInTheDocument();
  expect(screen.getByText('alice@example.com')).toBeInTheDocument();
});
```

### Render Options

```typescript
render(<App />, {
  wrapper: ({ children }) => (
    <BrowserRouter>
      <ThemeProvider theme={darkTheme}>
        {children}
      </ThemeProvider>
    </BrowserRouter>
  ),
});
```

## Screen Queries

`screen` provides all query methods bound to the document body. Always prefer `screen` over destructuring from `render()` — it makes it easier to spot which queries belong to which render call.

### Query Types at a Glance

| Prefix | No match | One match | Multiple matches | When to use |
|--------|----------|-----------|-----------------|-------------|
| `getBy...` | **Throws** | Returns element | **Throws** | Element should exist right now |
| `queryBy...` | Returns `null` | Returns element | **Throws** | Element might not exist |
| `findBy...` | **Throws** (after wait) | Returns element | **Throws** | Element appears asynchronously |
| `getAllBy...` | **Throws** | Returns array | Returns array | Multiple elements expected |
| `queryAllBy...` | Returns `[]` | Returns array | Returns array | Multiple, might be absent |
| `findAllBy...` | **Throws** (after wait) | Returns array | Returns array | Multiple, appear async |

### `getByRole` (Preferred)

Query by [ARIA role](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles) — the most accessible and resilient query:

```typescript
// Buttons
screen.getByRole('button', { name: 'Submit' });
screen.getByRole('button', { name: /submit/i }); // regex match

// Links
screen.getByRole('link', { name: 'Read more' });

// Headings
screen.getByRole('heading', { name: 'Welcome', level: 1 });
screen.getByRole('heading', { level: 2 });

// Form elements
screen.getByRole('textbox', { name: 'Email address' });
screen.getByRole('checkbox', { name: 'Accept terms' });
screen.getByRole('combobox', { name: 'Country' }); // <select>
screen.getByRole('spinbutton', { name: 'Quantity' }); // number input

// Structure
screen.getByRole('dialog', { name: 'Confirm deletion' });
screen.getByRole('alert');
screen.getByRole('navigation');
screen.getByRole('list');
screen.getByRole('listitem');
screen.getByRole('img', { name: 'User avatar' }); // images with alt text
```

### `getByText`

```typescript
// Exact match
screen.getByText('Hello, World!');

// Case-insensitive regex
screen.getByText(/hello/i);

// Custom selector
screen.getByText('Submit', { selector: 'button' });

// Partial match
screen.getByText(/loading/i);
```

### `getByLabelText`

Best for form inputs — finds the input associated with a `<label>`:

```typescript
screen.getByLabelText('Email address');
screen.getByLabelText(/email/i);
```

### `getByPlaceholderText`

```typescript
screen.getByPlaceholderText('Search products...');
```

### `getByTestId`

Use as a **last resort** when no accessible query is possible:

```typescript
// In your component: <div data-testid="loading-spinner" />
screen.getByTestId('loading-spinner');
```

:::tip[Query Priority Order]
RTL recommends this priority order for queries, from most to least preferred:
1. `getByRole` — accessible to everyone
2. `getByLabelText` — form elements
3. `getByPlaceholderText` — if no label exists
4. `getByText` — non-interactive elements
5. `getByDisplayValue` — current form input value
6. `getByAltText` — images
7. `getByTitle` — title attribute
8. `getByTestId` — last resort only
:::

## `userEvent` vs `fireEvent`

`@testing-library/user-event` simulates how a real user interacts — it triggers all the intermediate events a browser would fire (focus, keydown, keypress, input, keyup, blur, etc.). `fireEvent` only fires a single event with no side effects.

**Always prefer `userEvent`** for realistic tests.

```typescript
import userEvent from '@testing-library/user-event';
```

### Setup

```typescript
// Create a user-event instance (recommended for RTL v14+)
const user = userEvent.setup();

it('fills in the login form', async () => {
  render(<LoginForm onSubmit={jest.fn()} />);
  const user = userEvent.setup();

  await user.type(screen.getByLabelText('Email'), 'alice@example.com');
  await user.type(screen.getByLabelText('Password'), 'secret123');
  await user.click(screen.getByRole('button', { name: 'Sign in' }));
});
```

### Common User Actions

```typescript
const user = userEvent.setup();

// Clicking
await user.click(screen.getByRole('button', { name: 'Submit' }));
await user.dblClick(screen.getByText('Item'));
await user.tripleClick(screen.getByRole('textbox')); // select all text

// Typing
await user.type(screen.getByRole('textbox'), 'Hello, World!');
await user.clear(screen.getByRole('textbox'));
await user.type(screen.getByRole('textbox'), '{selectall}{backspace}'); // clear + type

// Keyboard shortcuts
await user.keyboard('{Enter}');
await user.keyboard('{Tab}');
await user.keyboard('{Escape}');
await user.keyboard('{Control>}a{/Control}'); // Ctrl+A

// Dropdowns
await user.selectOptions(screen.getByRole('combobox'), 'United Kingdom');
await user.selectOptions(screen.getByRole('listbox'), ['option1', 'option2']); // multi-select
await user.deselectOptions(screen.getByRole('listbox'), 'option1');

// Hover
await user.hover(screen.getByText('Hover me'));
await user.unhover(screen.getByText('Hover me'));

// Uploading files
const file = new File(['content'], 'test.png', { type: 'image/png' });
await user.upload(screen.getByLabelText('Profile picture'), file);

// Tabs
await user.tab(); // move focus forward
await user.tab({ shift: true }); // move focus backward
```

## `waitFor` and `waitForElementToBeRemoved`

These utilities wait for async state changes:

```typescript
import { waitFor, waitForElementToBeRemoved } from '@testing-library/react';

it('shows the loaded data', async () => {
  render(<UserList />);

  // Wait for the loading spinner to disappear
  await waitForElementToBeRemoved(() => screen.queryByRole('progressbar'));

  // Then assert the data is visible
  expect(screen.getByText('Alice')).toBeInTheDocument();
  expect(screen.getByText('Bob')).toBeInTheDocument();
});

it('shows an error message on failure', async () => {
  server.use(
    rest.get('/api/users', (req, res, ctx) => res(ctx.status(500))),
  );

  render(<UserList />);

  await waitFor(() => {
    expect(screen.getByRole('alert')).toHaveTextContent('Failed to load users');
  });
});
```

## Testing Forms

```typescript
// src/components/RegisterForm.tsx
export function RegisterForm({ onSubmit }: { onSubmit: (data: FormData) => void }) {
  const [errors, setErrors] = useState<Record<string, string>>({});
  // ...form implementation
}
```

```typescript
describe('RegisterForm', () => {
  it('submits valid form data', async () => {
    const onSubmit = jest.fn();
    const user = userEvent.setup();
    render(<RegisterForm onSubmit={onSubmit} />);

    await user.type(screen.getByLabelText('Name'), 'Alice');
    await user.type(screen.getByLabelText('Email'), 'alice@example.com');
    await user.type(screen.getByLabelText('Password'), 'SecurePassword1!');
    await user.click(screen.getByRole('button', { name: 'Register' }));

    expect(onSubmit).toHaveBeenCalledWith({
      name: 'Alice',
      email: 'alice@example.com',
      password: 'SecurePassword1!',
    });
  });

  it('shows validation errors for empty required fields', async () => {
    const user = userEvent.setup();
    render(<RegisterForm onSubmit={jest.fn()} />);

    // Submit without filling in any fields
    await user.click(screen.getByRole('button', { name: 'Register' }));

    expect(screen.getByText('Name is required')).toBeInTheDocument();
    expect(screen.getByText('Email is required')).toBeInTheDocument();
    expect(screen.getByText('Password is required')).toBeInTheDocument();
  });

  it('shows an error for an invalid email address', async () => {
    const user = userEvent.setup();
    render(<RegisterForm onSubmit={jest.fn()} />);

    await user.type(screen.getByLabelText('Email'), 'not-an-email');
    await user.tab(); // trigger blur validation

    await waitFor(() => {
      expect(screen.getByText('Invalid email address')).toBeInTheDocument();
    });
  });
});
```

## Testing Hooks with `renderHook`

Test custom hooks in isolation without building a component around them:

```typescript
// src/hooks/useCounter.ts
export function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);
  const increment = () => setCount(c => c + 1);
  const decrement = () => setCount(c => c - 1);
  const reset = () => setCount(initialValue);
  return { count, increment, decrement, reset };
}
```

```typescript
import { renderHook, act } from '@testing-library/react';
import { useCounter } from './useCounter';

describe('useCounter', () => {
  it('initialises with the given value', () => {
    const { result } = renderHook(() => useCounter(10));
    expect(result.current.count).toBe(10);
  });

  it('increments the count', () => {
    const { result } = renderHook(() => useCounter());

    act(() => {
      result.current.increment();
    });

    expect(result.current.count).toBe(1);
  });

  it('resets to the initial value', () => {
    const { result } = renderHook(() => useCounter(5));

    act(() => {
      result.current.increment();
      result.current.increment();
      result.current.reset();
    });

    expect(result.current.count).toBe(5);
  });
});
```

## Testing Context

Wrap the component under test with the required Context provider:

```typescript
// src/contexts/ThemeContext.tsx
export const ThemeContext = createContext<'light' | 'dark'>('light');
export const ThemeProvider = ({ children, theme = 'light' }) => (
  <ThemeContext.Provider value={theme}>{children}</ThemeContext.Provider>
);
```

```typescript
function renderWithTheme(ui: React.ReactElement, theme: 'light' | 'dark' = 'light') {
  return render(<ThemeProvider theme={theme}>{ui}</ThemeProvider>);
}

it('renders correctly in dark theme', () => {
  renderWithTheme(<ThemedButton label="Click me" />, 'dark');
  const button = screen.getByRole('button', { name: 'Click me' });
  expect(button).toHaveClass('btn-dark');
});
```

## Testing with Router

```typescript
import { MemoryRouter, Route, Routes } from 'react-router-dom';

function renderWithRouter(ui: React.ReactElement, initialPath = '/') {
  return render(
    <MemoryRouter initialEntries={[initialPath]}>
      {ui}
    </MemoryRouter>,
  );
}

it('renders the correct page based on the route', () => {
  render(
    <MemoryRouter initialEntries={['/users/1']}>
      <Routes>
        <Route path="/users/:id" element={<UserPage />} />
      </Routes>
    </MemoryRouter>,
  );

  expect(screen.getByRole('heading', { name: /user 1/i })).toBeInTheDocument();
});
```

## `act()` Wrapper

React batches state updates into renders. When triggering state changes outside React's control (custom timers, manual DOM mutations, Node.js events), wrap the action in `act()`:

```typescript
import { act } from '@testing-library/react';

it('updates after a state change', () => {
  const { result } = renderHook(() => useCounter());

  // act() ensures all React updates flush before asserting
  act(() => {
    result.current.increment();
  });

  expect(result.current.count).toBe(1);
});

// RTL's userEvent wraps actions in act automatically
// fireEvent also wraps in act
// Manual act is needed for: setTimeout callbacks, EventEmitter events, etc.
it('updates after a timer fires', () => {
  jest.useFakeTimers();
  render(<CountdownTimer seconds={3} />);

  act(() => {
    jest.advanceTimersByTime(1000);
  });

  expect(screen.getByText('2')).toBeInTheDocument();
  jest.useRealTimers();
});
```

:::tip[Most RTL utilities wrap act() for you]
`userEvent`, `fireEvent`, and RTL's `waitFor` all call `act()` internally. You only need to call `act()` manually when triggering state changes outside of these utilities — for example, when advancing fake timers or manually firing Node.js events.
:::

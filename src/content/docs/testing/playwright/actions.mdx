Playwright actions interact with elements in a browser. Every action **auto-waits** for the element to be visible, stable, and actionable before executing — no explicit waits needed in most cases.

## click

The most common action — simulates a mouse click with full browser event dispatching:

```typescript
// Left click (default)
await page.getByRole('button', { name: 'Submit' }).click();

// Double click
await page.getByRole('button', { name: 'Edit' }).dblclick();

// Right click (context menu)
await page.getByRole('row', { name: 'file.txt' }).click({ button: 'right' });

// Click at specific position within the element
await page.getByRole('img', { name: 'Canvas' }).click({
  position: { x: 100, y: 50 },
});

// Click with modifier keys
await page.getByRole('row', { name: 'Item A' }).click({ modifiers: ['Shift'] });
await page
  .getByRole('link', { name: 'File' })
  .click({ modifiers: ['Control'] });

// Force click — bypasses actionability checks (use sparingly)
await page.getByRole('button').click({ force: true });

// Click and hold
await page.getByRole('button').click({ delay: 500 });
```

:::caution[Avoid force: true]
`force: true` skips Playwright's actionability checks (visibility, stability, enabled state). Only use it when the element is intentionally "hidden" but technically clickable — most of the time, if `force: true` is needed, your locator is wrong or the element is not ready.
:::

## fill vs type

Both actions enter text into input fields, but they behave differently:

|              | `fill`                     | `type`                                                |
| ------------ | -------------------------- | ----------------------------------------------------- |
| Speed        | Fast — sets value directly | Slow — one character at a time                        |
| Events fired | `input`, `change`          | `keydown`, `keypress`, `input`, `keyup` per character |
| Clears first | Yes, always                | No (appends)                                          |
| Use case     | Form inputs                | When keydown/keyup handlers matter                    |

```typescript
// fill — preferred for most inputs
await page.getByLabel('Email').fill('alice@example.com');
await page.getByLabel('Search').fill('playwright testing');

// type — use when character-by-character events are required
await page.getByLabel('Search').type('pl'); // triggers autocomplete suggestions
```

### Clearing an Input

```typescript
// fill clears before typing
await page.getByLabel('Name').fill('Alice'); // replaces any existing value

// Explicit clear
await page.getByLabel('Name').clear();

// Select all and replace (for inputs that don't respond to fill)
await page.getByLabel('Name').selectText();
await page.keyboard.type('New value');
```

## press — Keyboard Shortcuts

Send keyboard events using key names:

```typescript
// Single keys
await page.getByRole('textbox').press('Enter');
await page.getByRole('textbox').press('Tab');
await page.getByRole('textbox').press('Escape');
await page.getByRole('textbox').press('ArrowDown');
await page.getByRole('textbox').press('Backspace');
await page.getByRole('textbox').press('Delete');

// Modifier + key combinations
await page.getByRole('textbox').press('Control+a'); // select all
await page.getByRole('textbox').press('Control+c'); // copy
await page.getByRole('textbox').press('Control+v'); // paste
await page.getByRole('textbox').press('Meta+z'); // Cmd+Z on macOS

// Function keys
await page.keyboard.press('F5');
await page.keyboard.press('F12');
```

### keyboard.type and keyboard.press

For global keyboard events not tied to a specific element:

```typescript
// Type text globally (no element focus needed)
await page.keyboard.type('Hello, World!');
await page.keyboard.type('slow', { delay: 100 }); // 100ms between characters

// Press a key globally
await page.keyboard.press('Enter');
await page.keyboard.press('Escape');

// Hold modifier key
await page.keyboard.down('Shift');
await page.getByRole('link', { name: 'About' }).click();
await page.keyboard.up('Shift');
```

## check / uncheck — Checkboxes and Radio Buttons

```typescript
// Check a checkbox (idempotent — won't double-check)
await page.getByLabel('Accept terms and conditions').check();
await expect(page.getByLabel('Accept terms')).toBeChecked();

// Uncheck a checkbox
await page.getByLabel('Subscribe to newsletter').uncheck();
await expect(page.getByLabel('Subscribe')).not.toBeChecked();

// Radio buttons
await page.getByLabel('Monthly billing').check();
await page.getByLabel('Annual billing').check(); // unchecks Monthly

// setChecked — set to a specific state
await page.getByLabel('Dark mode').setChecked(true);
await page.getByLabel('Dark mode').setChecked(false);
```

## selectOption — Dropdowns

```typescript
const countrySelect = page.getByRole('combobox', { name: 'Country' });

// Select by value
await countrySelect.selectOption('US');

// Select by label (visible text)
await countrySelect.selectOption({ label: 'United States' });

// Select by index
await countrySelect.selectOption({ index: 2 });

// Multi-select (multiple options at once)
await page
  .getByLabel('Interests')
  .selectOption(['coding', 'design', 'testing']);

// Deselect all (multi-select)
await page.getByLabel('Tags').selectOption([]);
```

## hover, focus, blur

```typescript
// Hover over element (triggers :hover CSS styles, tooltips)
await page.getByRole('button', { name: 'Help' }).hover();
await expect(page.getByRole('tooltip')).toBeVisible();

// Hover at specific position
await page.getByRole('img').hover({ position: { x: 50, y: 50 } });

// Focus an element (without clicking)
await page.getByLabel('Name').focus();
await expect(page.getByLabel('Name')).toBeFocused();

// Blur (remove focus)
await page.getByLabel('Name').blur();
```

## dragTo — Drag and Drop

```typescript
// Drag from source to target
const source = page.getByTestId('draggable-item');
const target = page.getByTestId('drop-zone');

await source.dragTo(target);

// Drag with explicit steps (smoother for complex drag implementations)
await source.dragTo(target, { sourcePosition: { x: 10, y: 10 } });

// Manual drag using mouse
await page.mouse.move(100, 200);
await page.mouse.down();
await page.mouse.move(300, 200, { steps: 10 }); // move in 10 steps
await page.mouse.up();
```

## uploadFiles

```typescript
// Single file upload
await page.getByLabel('Profile photo').setInputFiles('path/to/photo.jpg');

// Multiple files
await page
  .getByLabel('Attachments')
  .setInputFiles(['path/to/file1.pdf', 'path/to/file2.docx']);

// Upload from buffer (no file on disk needed)
await page.getByLabel('Document').setInputFiles({
  name: 'test.txt',
  mimeType: 'text/plain',
  buffer: Buffer.from('Hello, World!'),
});

// Clear the file input
await page.getByLabel('Attachment').setInputFiles([]);

// Handle file chooser dialog
const fileChooserPromise = page.waitForEvent('filechooser');
await page.getByRole('button', { name: 'Upload' }).click();
const fileChooser = await fileChooserPromise;
await fileChooser.setFiles('path/to/file.pdf');
```

## tap — Touch / Mobile

Simulate touch events on mobile viewports:

```typescript
// Set mobile viewport in test
test.use({ viewport: { width: 375, height: 812 }, hasTouch: true });

test('mobile navigation', async ({ page }) => {
  await page.goto('/');
  await page.getByRole('button', { name: 'Menu' }).tap();
  await expect(page.getByRole('navigation')).toBeVisible();
});
```

## mouse — Low-Level Mouse Control

```typescript
// Move mouse to absolute coordinates
await page.mouse.move(500, 300);

// Move in steps (for drag/hover over paths)
await page.mouse.move(500, 300, { steps: 20 });

// Click at coordinates
await page.mouse.click(500, 300);
await page.mouse.dblclick(500, 300);

// Press and release individually (for drag)
await page.mouse.down();
await page.mouse.move(600, 300);
await page.mouse.up();

// Mouse wheel / scroll
await page.mouse.wheel(0, 300); // scroll down 300px
```

## page.evaluate — Execute JavaScript in the Browser

Run arbitrary JavaScript in the browser page context:

```typescript
// Read a DOM property
const title = await page.evaluate(() => document.title);

// Manipulate the DOM
await page.evaluate(() => {
  document.body.style.backgroundColor = 'red';
});

// Pass arguments to the browser context
const result = await page.evaluate(({ x, y }) => x + y, { x: 10, y: 20 });
expect(result).toBe(30);

// Read localStorage
const token = await page.evaluate(() => localStorage.getItem('auth_token'));

// Evaluate on a specific element
const text = await page.getByRole('heading').evaluate((el) => el.textContent);

// Complex DOM inspection
const tableData = await page.evaluate(() => {
  return Array.from(document.querySelectorAll('table tbody tr')).map((row) => ({
    cells: Array.from(row.querySelectorAll('td')).map((td) =>
      td.textContent?.trim(),
    ),
  }));
});
```

## page.exposeFunction

Expose a Node.js function to the browser context:

```typescript
// Expose a Node.js function to the browser
await page.exposeFunction('readFile', async (filePath: string) => {
  return fs.promises.readFile(filePath, 'utf-8');
});

// Call it from browser-side evaluate
const content = await page.evaluate(async () => {
  return (window as any).readFile('/tmp/data.json');
});
```

## scrollIntoViewIfNeeded

Scroll an element into the visible viewport:

```typescript
// Scroll element into view (called automatically by most actions)
await page.getByTestId('footer-section').scrollIntoViewIfNeeded();

// Manual scroll on the page
await page.evaluate(() => window.scrollTo(0, document.body.scrollHeight));

// Scroll inside a container
const container = page.getByTestId('scrollable-list');
await container.evaluate((el) => el.scrollTo(0, el.scrollHeight));
```

## waitForLoadState

Wait for the page to reach a specific load state:

```typescript
// After navigation, wait for network to be idle
await page.goto('/dashboard');
await page.waitForLoadState('networkidle'); // no network activity for 500ms

// Wait for DOM to be parsed (fastest)
await page.waitForLoadState('domcontentloaded');

// Wait for all resources (images, scripts, styles) to load
await page.waitForLoadState('load');

// After a click that triggers navigation
await page.getByRole('link', { name: 'Products' }).click();
await page.waitForLoadState('networkidle');
await expect(page.getByRole('heading', { name: 'Products' })).toBeVisible();
```

| Load state         | Condition                     | Use case                      |
| ------------------ | ----------------------------- | ----------------------------- |
| `domcontentloaded` | HTML parsed, DOM available    | Fast navigation check         |
| `load`             | All resources loaded          | Page with images/fonts        |
| `networkidle`      | No network activity for 500ms | SPAs with async data fetching |

:::tip[Prefer Assertions Over waitForLoadState]
Most of the time, waiting for a specific element to be visible is better than waiting for a load state — it's more specific about what you need:

```typescript
// ✅ More precise
await expect(page.getByRole('heading', { name: 'Dashboard' })).toBeVisible();

// Works but less precise
await page.waitForLoadState('networkidle');
```

:::

## Handling Downloads

```typescript
// Wait for a download to start when clicking
const downloadPromise = page.waitForEvent('download');
await page.getByRole('button', { name: 'Export CSV' }).click();
const download = await downloadPromise;

// Save the file
await download.saveAs('/tmp/export.csv');

// Get the suggested filename
console.log(download.suggestedFilename());

// Get the download URL
console.log(download.url());
```

## Handling Dialogs

```typescript
// Automatically dismiss all dialogs
page.on('dialog', (dialog) => dialog.dismiss());

// Accept all dialogs
page.on('dialog', (dialog) => dialog.accept());

// Handle a specific dialog with input
page.once('dialog', async (dialog) => {
  expect(dialog.type()).toBe('prompt');
  expect(dialog.message()).toContain('Enter your name');
  await dialog.accept('Alice');
});

await page.getByRole('button', { name: 'Prompt me' }).click();
```

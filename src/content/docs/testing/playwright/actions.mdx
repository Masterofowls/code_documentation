Actions are how Playwright interacts with a web page. Each action method auto-waits for the element to be visible, stable, and enabled before performing the interaction — eliminating the need for explicit `waitFor` calls in most scenarios.

## click

The most common action — performs a left mouse click:

```typescript
// Standard left click
await page.getByRole('button', { name: 'Submit' }).click();

// Right-click — opens context menu
await page.getByRole('img', { name: 'User avatar' }).click({ button: 'right' });

// Double-click — on an editable cell, for example
await page.getByRole('cell', { name: 'Alice' }).dblclick();

// Click at a specific offset from the element's top-left corner
await page.getByRole('canvas').click({ position: { x: 100, y: 200 } });

// Hold modifier keys while clicking
await page.getByRole('link', { name: 'Download' }).click({ modifiers: ['Alt'] });

// Ctrl+Click for multi-select
await page.getByRole('row').nth(2).click({ modifiers: ['ControlOrMeta'] });
await page.getByRole('row').nth(4).click({ modifiers: ['Shift'] });

// Click without waiting for navigation
await page.getByRole('button', { name: 'Delete' }).click({ noWaitAfter: true });

// Click with a delay (holds down the mouse button)
await page.getByRole('button').click({ delay: 100 });

// Force click (bypass actionability checks)
await page.locator('.overlay-covered-button').click({ force: true });
```

:::caution[force: true]
`force: true` bypasses visibility, stability, and enabled checks — use only as a last resort when you are certain the element is in the state you expect. In most cases, if Playwright won't click an element, your test is exposing a real UI problem.
:::

## fill vs type

Both methods enter text into an input, but they work differently:

| Method | Speed | Events Fired | Use For |
|---|---|---|---|
| `fill()` | Fast — sets value directly | `input`, `change` | Standard inputs, forms |
| `type()` | Slow — one character at a time | `keydown`, `keypress`, `input`, `keyup` for each char | Inputs with per-keystroke logic (autocomplete, validation) |
| `pressSequentially()` | Slow | Full keyboard events per char | Same as `type()` (Playwright v1.38+) |

```typescript
// fill — clears existing value then sets the new one
await page.getByLabel('Search').fill('playwright testing');
await page.getByLabel('Email').fill('alice@example.com');

// fill with empty string — clears the field
await page.getByLabel('Search').fill('');

// type — triggers key events for each character (useful for autocomplete)
await page.getByLabel('City').type('Port');
// wait for autocomplete dropdown to appear
await page.getByRole('option', { name: 'Portland' }).click();

// pressSequentially (v1.38+ alias for type)
await page.getByLabel('Code').pressSequentially('123456');
```

## press

Simulate keyboard key presses:

```typescript
// Single keys
await page.getByLabel('Search').press('Enter');
await page.getByRole('dialog').press('Escape');
await page.keyboard.press('F5');         // on the page itself
await page.keyboard.press('Tab');

// Key combinations
await page.keyboard.press('Control+a');  // Select all
await page.keyboard.press('Control+c');  // Copy
await page.keyboard.press('Control+v');  // Paste
await page.keyboard.press('Control+z');  // Undo
await page.keyboard.press('Meta+Shift+z'); // Redo (macOS)

// Arrow navigation
await page.keyboard.press('ArrowDown');
await page.keyboard.press('ArrowUp');
await page.keyboard.press('PageDown');
await page.keyboard.press('Home');
await page.keyboard.press('End');

// Function keys
await page.keyboard.press('F12');

// Key sequences
await page.getByRole('textbox').press('Control+A');  // Select all text
await page.getByRole('textbox').press('Backspace');  // Delete selection
```

Common key names: `Enter`, `Tab`, `Escape`, `Space`, `Backspace`, `Delete`, `ArrowUp`, `ArrowDown`, `ArrowLeft`, `ArrowRight`, `Home`, `End`, `PageUp`, `PageDown`, `F1`-`F12`.

## check and uncheck

For checkboxes and radio buttons:

```typescript
// Check a checkbox
await page.getByRole('checkbox', { name: 'Accept terms' }).check();

// Uncheck a checkbox
await page.getByRole('checkbox', { name: 'Subscribe' }).uncheck();

// Verify state
await expect(page.getByRole('checkbox', { name: 'Remember me' })).toBeChecked();

// setChecked — idempotent (sets to target state regardless of current)
await page.getByRole('checkbox', { name: 'Newsletter' }).setChecked(true);
await page.getByRole('checkbox', { name: 'Newsletter' }).setChecked(false);

// Radio button selection
await page.getByRole('radio', { name: 'Credit card' }).check();
await expect(page.getByRole('radio', { name: 'Credit card' })).toBeChecked();
```

## selectOption

For `<select>` dropdowns:

```typescript
// Select by value
await page.getByRole('combobox', { name: 'Country' }).selectOption('US');

// Select by display label
await page.getByRole('combobox', { name: 'Country' }).selectOption({ label: 'United States' });

// Select by index (0-based)
await page.getByRole('combobox', { name: 'Country' }).selectOption({ index: 0 });

// Multi-select — pass an array
await page.getByLabel('Languages').selectOption(['javascript', 'typescript', 'python']);

// Select by label (multi)
await page.getByLabel('Skills').selectOption([
  { label: 'JavaScript' },
  { label: 'TypeScript' },
]);
```

## hover

Move the mouse cursor over an element to trigger hover states:

```typescript
// Basic hover — triggers :hover CSS, mouseover events
await page.getByRole('button', { name: 'More options' }).hover();

// Hover over a specific position within an element
await page.getByRole('image', { name: 'Chart' }).hover({ position: { x: 150, y: 75 } });

// Hover to reveal a tooltip, then assert on it
await page.getByRole('button', { name: 'Info' }).hover();
await expect(page.getByRole('tooltip')).toBeVisible();
await expect(page.getByRole('tooltip')).toContainText('Click for more details');
```

## focus and blur

```typescript
// Focus an element (triggers :focus styles)
await page.getByLabel('Email').focus();
await expect(page.getByLabel('Email')).toBeFocused();

// Blur (remove focus) — triggers validation messages
await page.getByLabel('Email').blur();
await expect(page.getByText('Email is required')).toBeVisible();

// Tab through form fields
await page.getByLabel('First name').focus();
await page.keyboard.press('Tab');
// focus moves to next field
await expect(page.getByLabel('Last name')).toBeFocused();
```

## dragTo

Drag an element to a target location:

```typescript
// Drag one element onto another
await page.getByTestId('draggable-item').dragTo(page.getByTestId('drop-zone'));

// Drag to specific coordinates
await page.getByTestId('handle').dragTo(page.locator('#target'), {
  sourcePosition: { x: 10, y: 10 },   // offset from source
  targetPosition: { x: 50, y: 50 },   // offset from target
});

// Manual drag using mouse
await page.getByTestId('slider-handle').hover();
await page.mouse.down();
await page.mouse.move(300, 0, { steps: 10 }); // 10 intermediate steps
await page.mouse.up();
```

## uploadFiles

```typescript
// Single file upload
await page.getByLabel('Profile picture').setInputFiles('path/to/photo.jpg');

// Multiple files
await page.getByLabel('Attachments').setInputFiles([
  'path/to/document.pdf',
  'path/to/image.png',
]);

// Clear a file input
await page.getByLabel('Profile picture').setInputFiles([]);

// Upload from a buffer (in-memory file)
await page.getByLabel('Logo').setInputFiles({
  name: 'logo.png',
  mimeType: 'image/png',
  buffer: Buffer.from('fake image data'),
});

// Handle a file chooser dialog
const fileChooserPromise = page.waitForEvent('filechooser');
await page.getByRole('button', { name: 'Upload' }).click();
const fileChooser = await fileChooserPromise;
await fileChooser.setFiles('path/to/file.pdf');
```

## tap — Mobile Touch Actions

```typescript
// Tap (touch event — use for mobile testing)
await page.getByRole('button', { name: 'Submit' }).tap();

// Multi-touch gesture
await page.touchscreen.tap(100, 200);
```

Enable touch via `devices` configuration:

```typescript
// playwright.config.ts
projects: [{
  name: 'Mobile Safari',
  use: { ...devices['iPhone 12'] },
}],
```

## Mouse Control

For precise mouse movements:

```typescript
// Move the mouse to a location
await page.mouse.move(100, 200);
await page.mouse.move(300, 400, { steps: 10 }); // smooth movement

// Press and hold / release buttons
await page.mouse.down();
await page.mouse.move(200, 200);
await page.mouse.up();

// Click at coordinates
await page.mouse.click(100, 200);
await page.mouse.dblclick(100, 200);
```

## Keyboard Control

For complex keyboard interactions not covered by `press()`:

```typescript
// Hold a key while clicking
await page.keyboard.down('Shift');
await page.getByRole('row').nth(4).click();
await page.keyboard.up('Shift');

// Type text (all characters at once)
await page.keyboard.type('Hello, World!');

// Insert text (no key events — just inserts the string)
await page.keyboard.insertText('Pasted text 🎉');
```

## page.evaluate — Running JavaScript in the Browser

Execute JavaScript in the browser context:

```typescript
// Get a value from the page
const title = await page.evaluate(() => document.title);

// Manipulate the DOM
await page.evaluate(() => {
  document.getElementById('hidden-element')!.style.display = 'block';
});

// Pass arguments to the script
const elementText = await page.evaluate(
  (selector) => document.querySelector(selector)?.textContent ?? '',
  '.my-element'
);

// Get the bounding box of an element
const box = await page.getByRole('main').boundingBox();
console.log(`Main content: ${box?.width}x${box?.height} at (${box?.x}, ${box?.y})`);

// Evaluate on a locator (runs JS with the element as argument)
const value = await page.getByRole('textbox').evaluate(
  (el: HTMLInputElement) => el.value
);

// Scroll to a position
await page.evaluate(() => window.scrollTo(0, 500));
```

## page.exposeFunction

Expose a Node.js function to be callable from browser scripts:

```typescript
// Make a function available on window in the browser
await page.exposeFunction('sha256', async (text: string) => {
  const crypto = require('crypto');
  return crypto.createHash('sha256').update(text).digest('hex');
});

// Now callable from page.evaluate or page scripts
const hash = await page.evaluate(async () => {
  return await (window as any).sha256('hello world');
});
```

## scrollIntoViewIfNeeded

Scroll an element into the visible viewport:

```typescript
// Scroll element into view before interacting
await page.getByRole('button', { name: 'Load more' }).scrollIntoViewIfNeeded();
await page.getByRole('button', { name: 'Load more' }).click();

// Scroll the page itself
await page.evaluate(() => window.scrollTo(0, document.body.scrollHeight));

// Scroll to bottom of a scrollable container
await page.locator('.scroll-container').evaluate(el => {
  el.scrollTop = el.scrollHeight;
});
```

## waitForLoadState

Control when to consider navigation complete:

```typescript
// 'load' — window.load event fired (all resources loaded)
await page.goto('/dashboard', { waitUntil: 'load' });

// 'domcontentloaded' — HTML parsed, no waiting for images/fonts
await page.goto('/dashboard', { waitUntil: 'domcontentloaded' });

// 'networkidle' — no network requests for 500ms (fragile for SPAs)
await page.goto('/dashboard', { waitUntil: 'networkidle' });

// Wait after navigation
await page.getByRole('link', { name: 'Profile' }).click();
await page.waitForLoadState('load');

// Wait for a specific URL
await page.waitForURL('**/dashboard');
await page.waitForURL(/\/dashboard/);
await page.waitForURL('https://app.example.com/dashboard');
```

:::tip[Prefer waitForURL over networkidle]
`waitUntil: 'networkidle'` can cause flaky tests — SPAs often have background network activity. Use `page.waitForURL()` to wait for a specific navigation, or web-first assertions to wait for expected UI state.
:::

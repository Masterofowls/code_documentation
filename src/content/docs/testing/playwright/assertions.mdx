Playwright's assertion library is built around the concept of **web-first assertions** — assertions that automatically retry until they pass or the timeout expires. This eliminates timing issues and makes tests resilient to animation delays, data loading, and dynamic rendering.

## Web-First Assertions

Standard assertion libraries (like Jest's `expect`) evaluate the value once. If a React component is still loading when the assertion runs, it fails immediately.

Playwright's `expect()` is different:

```typescript
// ✗ This will fail if the element loads asynchronously
const text = await page.getByText('Welcome').textContent();
expect(text).toBe('Welcome'); // evaluated once, right now

// ✓ This retries until the assertion passes or times out
await expect(page.getByText('Welcome')).toBeVisible();
await expect(page.getByText('Welcome')).toHaveText('Welcome, Alice!');
```

The retry interval is 100ms by default, with a configurable timeout (default: 5 seconds from `expect.timeout` in config):

```typescript
// Custom timeout for a single assertion
await expect(page.getByRole('heading', { name: 'Done' })).toBeVisible({ timeout: 10_000 });
```

## Visibility Assertions

```typescript
// toBeVisible — element is in the DOM and visible
await expect(page.getByRole('dialog')).toBeVisible();
await expect(page.getByText('Loading...')).toBeVisible();

// toBeHidden — element is not in the DOM, or has display:none/visibility:hidden
await expect(page.getByRole('dialog')).toBeHidden();
await expect(page.getByTestId('error-banner')).toBeHidden();

// toBeAttached — element is in the DOM (regardless of visibility)
await expect(page.locator('#hidden-field')).toBeAttached();

// not modifier
await expect(page.getByRole('progressbar')).not.toBeVisible();
```

## State Assertions

```typescript
// toBeEnabled / toBeDisabled
await expect(page.getByRole('button', { name: 'Submit' })).toBeEnabled();
await expect(page.getByRole('button', { name: 'Submit' })).toBeDisabled();

// Verify form can be submitted
await expect(page.getByRole('button', { name: 'Register' })).not.toBeDisabled();

// toBeChecked — for checkboxes and radio buttons
await expect(page.getByRole('checkbox', { name: 'Remember me' })).toBeChecked();
await expect(page.getByRole('checkbox', { name: 'Newsletter' })).not.toBeChecked();
await expect(page.getByRole('checkbox')).toBeChecked({ checked: false });

// toBeEditable — input is not readonly and not disabled
await expect(page.getByLabel('Username')).toBeEditable();

// toBeEmpty — input has no value, element has no text content
await expect(page.getByLabel('Notes')).toBeEmpty();

// toBeFocused — element has keyboard focus
await expect(page.getByLabel('Email')).toBeFocused();

// toBeInViewport — element is visible in the viewport
await expect(page.getByRole('button', { name: 'Load more' })).toBeInViewport();
```

## Text Assertions

```typescript
// toHaveText — check text content
await expect(page.getByRole('heading')).toHaveText('Welcome, Alice!');
await expect(page.getByRole('heading')).toHaveText(/welcome, alice/i); // regex
await expect(page.getByRole('heading')).toHaveText('Welcome', { exact: false }); // substring

// toContainText — alias for substring matching
await expect(page.getByRole('status')).toContainText('saved');
await expect(page.locator('.error')).toContainText('required field');

// Match against all elements in a locator (array)
await expect(page.getByRole('listitem')).toHaveText([
  'First item',
  'Second item',
  'Third item',
]);

// Regex array
await expect(page.getByRole('listitem')).toHaveText([/first/i, /second/i, /third/i]);

// toHaveValue — for inputs, textareas, selects
await expect(page.getByLabel('Email')).toHaveValue('alice@example.com');
await expect(page.getByLabel('Country')).toHaveValue('GB');
await expect(page.getByLabel('Search')).toHaveValue(/react/);
```

## Attribute and Class Assertions

```typescript
// toHaveAttribute
await expect(page.getByRole('link', { name: 'Home' })).toHaveAttribute('href', '/');
await expect(page.getByRole('img', { name: 'Logo' })).toHaveAttribute('src', /logo/);
await expect(page.getByRole('textbox')).toHaveAttribute('placeholder', 'Search...');
await expect(page.getByRole('button', { name: 'Like' })).toHaveAttribute('aria-pressed', 'true');

// Check attribute exists (regardless of value)
await expect(page.getByRole('link')).toHaveAttribute('rel');

// Check attribute does NOT exist
await expect(page.getByRole('textbox')).not.toHaveAttribute('readonly');

// toHaveClass — check CSS classes
await expect(page.getByRole('button', { name: 'Submit' })).toHaveClass('btn-primary');
await expect(page.getByRole('button')).toHaveClass(/btn/);
await expect(page.getByRole('button')).toHaveClass('btn btn-primary active'); // exact full class string

// toHaveId — check element id
await expect(page.getByRole('main')).toHaveId('main-content');
```

## Count Assertions

```typescript
// toHaveCount — check number of matching elements
await expect(page.getByRole('listitem')).toHaveCount(5);
await expect(page.getByRole('row')).toHaveCount(6); // 1 header + 5 data rows
await expect(page.getByTestId('notification')).toHaveCount(0); // none visible
```

## URL and Title Assertions

```typescript
// toHaveURL — check current page URL
await expect(page).toHaveURL('/dashboard');
await expect(page).toHaveURL('https://example.com/dashboard');
await expect(page).toHaveURL(/\/dashboard$/);
await expect(page).toHaveURL(new URL('/dashboard', 'https://example.com'));

// toHaveTitle — check page title
await expect(page).toHaveTitle('Dashboard — My App');
await expect(page).toHaveTitle(/Dashboard/);

// Combining URL and title after navigation
await page.getByRole('link', { name: 'Settings' }).click();
await expect(page).toHaveURL('/settings');
await expect(page).toHaveTitle(/Settings/);
```

## Visual Regression — toHaveScreenshot()

Playwright includes built-in screenshot comparison for visual regression testing:

```typescript
// Full page screenshot comparison
await expect(page).toHaveScreenshot('homepage.png');

// Element screenshot comparison
await expect(page.getByTestId('product-card')).toHaveScreenshot('product-card.png');

// With options
await expect(page).toHaveScreenshot('dashboard.png', {
  maxDiffPixels: 100,           // allow up to 100 different pixels
  maxDiffPixelRatio: 0.01,      // allow up to 1% pixel difference
  threshold: 0.2,               // per-pixel threshold (0.0–1.0)
  clip: { x: 0, y: 0, width: 800, height: 600 }, // screenshot region
  mask: [page.getByTestId('timestamp')], // mask dynamic content
  animations: 'disabled',       // disable CSS animations
  caret: 'hide',                // hide text cursors
});
```

### Managing Screenshots

```bash
# First run — creates reference screenshots
npx playwright test

# Update screenshots after intentional UI changes
npx playwright test --update-snapshots

# Screenshots are stored in:
# tests/homepage.spec.ts-snapshots/homepage-chromium-linux.png
```

:::tip[Mask Dynamic Content]
Always mask elements with dynamic content (timestamps, profile photos, random IDs) to prevent false failures. Use the `mask` option to pass an array of locators to black-out before comparison.
:::

:::caution[Screenshot Tests Are Platform-Specific]
Screenshots differ between operating systems and browser versions because of font rendering, sub-pixel antialiasing, and platform-specific UI elements. Generate reference screenshots on the same platform that runs your CI — typically Linux. Do not mix macOS-generated and Linux-compared screenshots.
:::

## API Response Assertions

```typescript
import { expect } from '@playwright/test';

// Test API responses using the request fixture
test('API returns correct user data', async ({ request }) => {
  const response = await request.get('/api/users/1');

  // toBeOK — status code is 200-299
  await expect(response).toBeOK();

  // Check specific status
  expect(response.status()).toBe(200);

  // Assert response body
  const body = await response.json();
  expect(body).toMatchObject({ id: 1, name: 'Alice' });
});

test('API rejects unauthorized access', async ({ request }) => {
  const response = await request.get('/api/admin/users');
  expect(response.status()).toBe(401);
});
```

## Soft Assertions

Soft assertions do not stop the test on failure — they collect all failures and report them together at the end. This is useful for validating multiple independent properties in a single test:

```typescript
test('product page has all required information', async ({ page }) => {
  await page.goto('/products/widget-pro');

  // These won't stop the test if they fail
  await expect.soft(page.getByRole('heading')).toHaveText('Widget Pro');
  await expect.soft(page.getByTestId('price')).toHaveText('$49.99');
  await expect.soft(page.getByTestId('stock-status')).toContainText('In stock');
  await expect.soft(page.getByRole('button', { name: 'Add to Cart' })).toBeEnabled();
  await expect.soft(page.getByRole('img', { name: 'Widget Pro' })).toBeVisible();

  // This hard assertion will stop the test
  // All soft failures are reported alongside it
  await expect(page).toHaveURL('/products/widget-pro');
});
```

If two soft assertions fail, the test reports both failures instead of stopping at the first.

## Custom Assertion Messages

Add descriptive messages to make failures easier to diagnose:

```typescript
await expect(
  page.getByRole('button', { name: 'Submit' }),
  'Submit button should be enabled after all required fields are filled'
).toBeEnabled();

await expect(
  page.getByRole('alert'),
  'Error alert should appear after submitting invalid form data'
).toContainText('Please fix the following errors');

await expect(
  page,
  'Should redirect to dashboard after successful login'
).toHaveURL('/dashboard');
```

## Polling Assertion

For conditions that are not directly tied to DOM state:

```typescript
// expect.poll — retry a function until it returns a truthy value
await expect.poll(async () => {
  const response = await page.request.get('/api/job-status/123');
  const data = await response.json();
  return data.status;
}, { timeout: 30_000, intervals: [1000, 2000, 5000] }).toBe('complete');

// Useful for background jobs, async processing, webhooks
await expect.poll(
  () => emailService.getLastEmail(),
  { message: 'Confirmation email should arrive within 30 seconds', timeout: 30_000 }
).toMatchObject({ subject: /confirm/i });
```

Playwright's assertions are **web-first** — they automatically retry until the expected condition is true or the timeout expires. This eliminates the most common source of flaky tests: assertions evaluated before the UI has finished updating.

## Web-First Assertions

Traditional assertion libraries (Jest's `expect`, Node's `assert`) evaluate immediately and fail if the condition isn't true at that instant. Playwright's `expect` from `@playwright/test` retries automatically:

```typescript
// ❌ Brittle — fails if the element isn't visible at this exact millisecond
const isVisible = await page.getByText('Success').isVisible();
expect(isVisible).toBe(true);  // standard expect — no retry

// ✅ Resilient — retries until visible or timeout expires
await expect(page.getByText('Success')).toBeVisible(); // Playwright expect
```

The default retry timeout for assertions is `5000ms` (5 seconds). Configure globally:

```typescript
// playwright.config.ts
use: {
  actionTimeout: 15000,  // action methods (click, fill, etc.)
},
expect: {
  timeout: 8000,         // assertion retries
},
```

Override per assertion:

```typescript
await expect(page.getByText('Loaded')).toBeVisible({ timeout: 15000 });
```

## Visibility Assertions

```typescript
// toBeVisible — element exists in DOM AND is visible
await expect(page.getByRole('dialog')).toBeVisible();

// toBeHidden — element is either not in DOM or hidden (display:none, visibility:hidden)
await expect(page.getByRole('dialog')).toBeHidden();

// Practical patterns
await expect(page.getByTestId('loading-spinner')).toBeVisible();
await page.getByRole('button', { name: 'Submit' }).click();
await expect(page.getByTestId('loading-spinner')).toBeHidden();
await expect(page.getByText('Submitted successfully!')).toBeVisible();

// toBeAttached — in DOM but possibly not visible
await expect(page.getByRole('tooltip')).toBeAttached();

// toBeDetached — not present in DOM at all
await expect(page.locator('#deleted-element')).toBeDetached();
```

## State Assertions

```typescript
// toBeEnabled / toBeDisabled
await expect(page.getByRole('button', { name: 'Submit' })).toBeEnabled();
await expect(page.getByRole('button', { name: 'Submit' })).toBeDisabled();

// toBeChecked / not.toBeChecked
await expect(page.getByRole('checkbox', { name: 'Accept terms' })).toBeChecked();
await expect(page.getByRole('checkbox', { name: 'Subscribe' })).not.toBeChecked();

// toBeChecked with indeterminate state
await expect(page.getByRole('checkbox')).toBeChecked({ checked: false });

// toBeFocused — element has keyboard focus
await page.getByLabel('Email').click();
await expect(page.getByLabel('Email')).toBeFocused();

// toBeEmpty — input value or text content is empty
await expect(page.getByLabel('Search')).toBeEmpty();
await expect(page.getByRole('list')).toBeEmpty(); // list has no items

// toBeInViewport — element is visible within the viewport
await expect(page.getByRole('main')).toBeInViewport();
```

## Text Assertions

```typescript
// toHaveText — element's text content
await expect(page.getByRole('heading')).toHaveText('Dashboard');
await expect(page.getByRole('heading')).toHaveText(/dashboard/i);  // regex
await expect(page.getByRole('heading')).toHaveText('Dashboard', { exact: false });

// Multiple elements — pass an array
const items = page.getByRole('listitem');
await expect(items).toHaveText(['First item', 'Second item', 'Third item']);
await expect(items).toHaveText([/first/i, /second/i, /third/i]);

// toContainText — substring match (looser than toHaveText)
await expect(page.getByRole('paragraph')).toContainText('Welcome');
await expect(page.getByRole('paragraph')).toContainText(/welcome/i);

// toContainText with array
const rows = page.getByRole('row');
await expect(rows).toContainText(['Alice', 'Bob']); // each row contains the text

// toHaveValue — current value of form inputs
await expect(page.getByLabel('Email')).toHaveValue('alice@example.com');
await expect(page.getByLabel('Age')).toHaveValue('30');
await expect(page.getByLabel('Bio')).toHaveValue(/some text/);

// toHaveValues — for multi-select
await expect(page.getByLabel('Languages')).toHaveValues(['js', 'ts']);
```

## Attribute and Class Assertions

```typescript
// toHaveAttribute — check an HTML attribute value
await expect(page.getByRole('link', { name: 'Home' })).toHaveAttribute('href', '/');
await expect(page.getByRole('img', { name: 'Logo' })).toHaveAttribute('src', /logo/);
await expect(page.getByRole('textbox')).toHaveAttribute('disabled');    // attribute present
await expect(page.getByRole('textbox')).not.toHaveAttribute('readonly');

// toHaveClass — element's class list
await expect(page.getByRole('button', { name: 'Primary' })).toHaveClass('btn-primary');
await expect(page.getByRole('button')).toHaveClass(/active/);  // regex
// Exact class string match (all classes, exact order)
await expect(page.locator('.card')).toHaveClass('card card--featured is-visible');

// toHaveCSS — computed CSS property value
await expect(page.getByRole('main')).toHaveCSS('display', 'flex');
await expect(page.locator('.error-text')).toHaveCSS('color', 'rgb(255, 0, 0)');

// toHaveId — element's id attribute
await expect(page.getByRole('main')).toHaveId('main-content');
```

## Count and Structure Assertions

```typescript
// toHaveCount — number of matching elements
await expect(page.getByRole('listitem')).toHaveCount(5);
await expect(page.getByRole('row')).toHaveCount(11); // 1 header + 10 data rows

// After adding an item:
await page.getByRole('button', { name: 'Add' }).click();
await expect(page.getByRole('listitem')).toHaveCount(6);
```

## Page-Level Assertions

```typescript
// toHaveURL — current page URL
await expect(page).toHaveURL('https://app.example.com/dashboard');
await expect(page).toHaveURL(/\/dashboard/);
await expect(page).toHaveURL('**/dashboard');  // glob pattern

// toHaveTitle — page document title
await expect(page).toHaveTitle('Dashboard — My App');
await expect(page).toHaveTitle(/My App/);
```

## Visual Regression — toHaveScreenshot

Playwright can capture and compare screenshots to detect visual regressions:

```typescript
// Full page screenshot
await expect(page).toHaveScreenshot('homepage.png');

// Element screenshot
await expect(page.getByRole('main')).toHaveScreenshot('main-content.png');

// With comparison options
await expect(page).toHaveScreenshot('dashboard.png', {
  threshold: 0.1,       // allow up to 10% per-pixel difference
  maxDiffPixels: 100,   // or allow up to 100 differing pixels
  maxDiffPixelRatio: 0.02,  // or up to 2% of all pixels
});

// Clip to a specific area
await expect(page).toHaveScreenshot('header.png', {
  clip: { x: 0, y: 0, width: 1280, height: 80 },
});

// Mask dynamic areas (timestamps, avatars, etc.)
await expect(page).toHaveScreenshot('profile.png', {
  mask: [
    page.getByTestId('timestamp'),
    page.getByRole('img', { name: 'User avatar' }),
  ],
});

// Animations — wait for them to complete before comparing
await page.waitForTimeout(500); // let animations settle
await expect(page).toHaveScreenshot('animated-element.png', {
  animations: 'disabled',  // or 'allow'
});
```

On first run, Playwright generates baseline screenshots in a `__screenshots__` directory. On subsequent runs, it compares against the baseline. Update baselines with:

```bash
npx playwright test --update-snapshots
```

:::caution[Screenshot Snapshots and CI]
Screenshots may differ between operating systems (macOS, Linux, Windows) due to font rendering, anti-aliasing, and pixel density. Store baseline screenshots generated on the same OS as your CI environment. Consider using the Playwright Docker image for consistent rendering.
:::

## API Response Assertions

```typescript
// toBeOK — response status is 2xx
const response = await page.request.get('/api/users');
await expect(response).toBeOK();

// Check status code explicitly
expect(response.status()).toBe(200);
expect(response.status()).toBe(201);

// Check response body
const body = await response.json();
expect(body).toMatchObject({ users: expect.arrayContaining([]) });
```

## Soft Assertions

Soft assertions continue test execution even when they fail, collecting all failures and reporting them at the end:

```typescript
test('profile page has all required elements', async ({ page }) => {
  await page.goto('/profile');

  // These won't stop the test if they fail
  await expect.soft(page.getByRole('heading', { name: 'My Profile' })).toBeVisible();
  await expect.soft(page.getByLabel('Display Name')).toHaveValue('Alice Johnson');
  await expect.soft(page.getByLabel('Email')).toHaveValue('alice@example.com');
  await expect.soft(page.getByRole('img', { name: 'Profile photo' })).toBeVisible();
  await expect.soft(page.getByRole('button', { name: 'Edit Profile' })).toBeEnabled();

  // Hard assertion at the end — fails if any soft assertion failed
  expect(test.info().errors).toHaveLength(0);
});
```

:::tip[When to Use Soft Assertions]
Use soft assertions when validating multiple independent properties of a page (e.g., a smoke test that checks many elements). Use hard assertions when a failure makes subsequent steps meaningless (e.g., checking the modal is visible before asserting its content).
:::

## Custom Assertion Messages

Add context to assertion failures:

```typescript
// Custom message as the last argument
await expect(
  page.getByRole('alert'),
  'Error alert should not be visible after successful submission'
).not.toBeVisible();

await expect(
  page.getByRole('heading'),
  `Expected to be on the dashboard, current URL: ${page.url()}`
).toHaveText('Dashboard');

// With object syntax
await expect(
  page.getByRole('button', { name: 'Save' }),
  { message: 'Save button should be enabled after filling all required fields' }
).toBeEnabled();
```

## Negation with not

Every assertion can be negated with `.not`:

```typescript
await expect(page.getByRole('dialog')).not.toBeVisible();
await expect(page.getByRole('button', { name: 'Delete' })).not.toBeDisabled();
await expect(page.getByText('Error')).not.toBeAttached();
await expect(page.getByLabel('Email')).not.toBeEmpty();
await expect(page.getByRole('checkbox')).not.toBeChecked();
```

## Complete Assertions Reference

| Assertion | Description |
|---|---|
| `toBeVisible()` | Element is in DOM and visible |
| `toBeHidden()` | Element is hidden or not in DOM |
| `toBeEnabled()` | Form element is enabled |
| `toBeDisabled()` | Form element is disabled |
| `toBeChecked()` | Checkbox/radio is checked |
| `toBeFocused()` | Element has keyboard focus |
| `toBeEmpty()` | Input value or text content is empty |
| `toBeAttached()` | Element is in the DOM |
| `toBeDetached()` | Element is not in the DOM |
| `toBeInViewport()` | Element is in visible viewport |
| `toHaveText(str)` | Element text matches |
| `toContainText(str)` | Element text contains substring |
| `toHaveValue(str)` | Input's current value |
| `toHaveValues(arr)` | Multi-select values |
| `toHaveAttribute(name, val)` | HTML attribute value |
| `toHaveClass(str)` | CSS class name |
| `toHaveCSS(prop, val)` | Computed CSS value |
| `toHaveId(str)` | Element id attribute |
| `toHaveCount(n)` | Number of matching elements |
| `toHaveURL(url)` | Current page URL |
| `toHaveTitle(str)` | Page document title |
| `toHaveScreenshot(name)` | Visual regression snapshot |
| `toBeOK()` | Response status is 2xx |

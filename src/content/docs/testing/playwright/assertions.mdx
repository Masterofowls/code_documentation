Playwright's assertions are **web-first**: instead of checking the DOM once and failing immediately, they retry the assertion repeatedly until it passes or the timeout is exceeded. This eliminates the need for manual `waitFor` calls in most situations.

## Web-First Assertion Philosophy

```typescript
// ❌ Old approach — fragile, race conditions
await page.waitForSelector('.success-toast');
const text = await page.textContent('.success-toast');
expect(text).toBe('Saved successfully');

// ✅ Playwright web-first — auto-retries, no race condition
await expect(page.getByRole('status')).toHaveText('Saved successfully');
```

Playwright retries the assertion until:
- The assertion passes (test continues)
- The assertion timeout expires (test fails with a descriptive error)

The default assertion timeout is **5 seconds** (configurable via `expect.timeout` in `playwright.config.ts`).

## Visibility Assertions

```typescript
// Element is visible (attached, not hidden by CSS, has non-zero size)
await expect(page.getByRole('dialog')).toBeVisible();
await expect(page.getByText('Welcome back!')).toBeVisible();

// Element is hidden (visibility: hidden, display: none, opacity: 0, or detached)
await expect(page.getByRole('dialog')).toBeHidden();
await expect(page.getByTestId('error-banner')).toBeHidden();

// Element is attached to DOM (may still be invisible)
await expect(page.getByTestId('template')).toBeAttached();
await expect(page.getByTestId('removed-element')).not.toBeAttached();

// Element is in the viewport
await expect(page.getByRole('button', { name: 'Scroll target' })).toBeInViewport();
await expect(page.getByRole('button', { name: 'Scroll target' })).toBeInViewport({
  ratio: 0.5, // at least 50% of the element must be visible
});
```

## State Assertions

```typescript
// Enabled / disabled form elements
await expect(page.getByRole('button', { name: 'Submit' })).toBeEnabled();
await expect(page.getByRole('button', { name: 'Submit' })).toBeDisabled();

await expect(page.getByLabel('Email')).toBeEnabled();
await expect(page.getByLabel('Email')).toBeEditable();

// Checked / unchecked checkboxes and radio buttons
await expect(page.getByRole('checkbox', { name: 'Terms' })).toBeChecked();
await expect(page.getByRole('checkbox', { name: 'Newsletter' })).not.toBeChecked();
await expect(page.getByRole('radio', { name: 'Monthly' })).toBeChecked();

// Input is empty
await expect(page.getByLabel('Search')).toBeEmpty();

// Element has focus
await expect(page.getByLabel('Email')).toBeFocused();
```

## Text Assertions

```typescript
// Exact text
await expect(page.getByRole('heading', { level: 1 })).toHaveText('User Dashboard');

// Partial text match
await expect(page.getByRole('alert')).toHaveText(/error occurred/i);

// Substring match
await expect(page.getByTestId('breadcrumb')).toContainText('Settings');

// Array of texts — matches all listitem elements in order
await expect(page.getByRole('listitem')).toHaveText([
  'First item',
  'Second item',
  'Third item',
]);

// Case-insensitive regex for each item
await expect(page.getByRole('listitem')).toHaveText([
  /first/i,
  /second/i,
  /third/i,
]);

// Inner text (doesn't include hidden text)
await expect(page.getByRole('status')).toHaveText('Processing...');
```

:::tip[toHaveText vs toContainText]
`toHaveText` matches the full text content of the element. `toContainText` matches if the element contains the given text as a substring. Use `toContainText` when you only care about part of the text.
:::

## Value Assertions

```typescript
// Input / textarea current value
await expect(page.getByLabel('Email')).toHaveValue('alice@example.com');
await expect(page.getByLabel('Description')).toHaveValue(/lorem ipsum/i);
await expect(page.getByRole('spinbutton', { name: 'Quantity' })).toHaveValue('3');

// Select element current value
await expect(page.getByRole('combobox', { name: 'Country' })).toHaveValue('CA');
```

## Attribute and Class Assertions

```typescript
// Check for a specific attribute value
await expect(page.getByRole('link', { name: 'Docs' })).toHaveAttribute('href', '/docs');
await expect(page.getByRole('link', { name: 'Docs' })).toHaveAttribute('href', /\/docs/);
await expect(page.getByRole('img')).toHaveAttribute('alt', expect.stringContaining('logo'));

// Check that an attribute exists (any value)
await expect(page.getByRole('textbox')).toHaveAttribute('placeholder');

// Check CSS class
await expect(page.getByRole('button', { name: 'Active' })).toHaveClass('btn-active');
await expect(page.getByRole('button')).toHaveClass(/btn-(primary|secondary)/);

// Check that an element has ALL of these classes
await expect(page.getByTestId('card')).toHaveClass('card shadow-md rounded-lg');

// CSS property value
await expect(page.getByTestId('progress-bar')).toHaveCSS('width', '75%');
await expect(page.getByTestId('sidebar')).toHaveCSS('display', 'flex');
```

## Count Assertion

```typescript
// Exact count
await expect(page.getByRole('listitem')).toHaveCount(5);

// Zero elements
await expect(page.getByRole('alert')).toHaveCount(0);

// At least / at most (use custom assertion or count() method)
const count = await page.getByRole('row').count();
expect(count).toBeGreaterThan(0);
```

## URL and Title Assertions

```typescript
// Full URL (string or regex)
await expect(page).toHaveURL('https://app.example.com/dashboard');
await expect(page).toHaveURL('/dashboard');             // relative to baseURL
await expect(page).toHaveURL(/\/users\/\d+/);           // regex

// URL containing a fragment
await expect(page).toHaveURL(/\#section-2/);

// Page title
await expect(page).toHaveTitle('Dashboard — My App');
await expect(page).toHaveTitle(/Dashboard/);
```

## Screenshot Assertions — Visual Regression

Playwright can take pixel-perfect screenshots and compare them to stored baselines:

```typescript
// Full page screenshot
await expect(page).toHaveScreenshot('dashboard.png');

// Element screenshot
await expect(page.getByTestId('chart')).toHaveScreenshot('revenue-chart.png');

// With tolerance options
await expect(page).toHaveScreenshot('homepage.png', {
  maxDiffPixels: 100,         // allow up to 100 different pixels
  threshold: 0.1,             // allow 10% color difference per pixel
  maxDiffPixelRatio: 0.01,    // allow 1% of pixels to differ
});

// Clip to a specific area
await expect(page).toHaveScreenshot('header.png', {
  clip: { x: 0, y: 0, width: 1280, height: 80 },
});

// Mask dynamic content (timestamps, avatars, ads)
await expect(page).toHaveScreenshot('profile.png', {
  mask: [
    page.getByTestId('last-seen'),
    page.getByTestId('user-avatar'),
    page.getByTestId('ad-banner'),
  ],
});

// Full page (scrolled)
await expect(page).toHaveScreenshot('full-page.png', { fullPage: true });
```

<Steps>
1. **First run** — Playwright generates baseline `.png` files in `tests/__snapshots__/`
2. **Subsequent runs** — Playwright compares current screenshots to baselines, failing if they differ
3. **Updating baselines** — Run `npx playwright test --update-snapshots` after intentional UI changes
4. **Commit baselines** — Baseline screenshots should be committed to version control
</Steps>

:::tip[Snapshot Directory]
Configure where screenshot baselines are stored:

```typescript
// playwright.config.ts
expect: {
  snapshotDir: './tests/screenshots',
  snapshotPathTemplate: '{snapshotDir}/{testFilePath}/{arg}{ext}',
},
```
:::

:::caution[Visual Tests are Environment-Sensitive]
Screenshot tests can fail due to font rendering differences between operating systems, anti-aliasing variations, or animation timing. Run visual tests in a consistent Docker environment in CI and use `maxDiffPixels` tolerance for minor rendering differences.
:::

## API Response Assertions

```typescript
// Check that a response has a 2xx status code
const response = await page.request.get('/api/health');
await expect(response).toBeOK();

// Check exact status
expect(response.status()).toBe(200);

// Check response body
const data = await response.json();
expect(data).toMatchObject({ status: 'healthy' });

// In network interception
page.on('response', async (response) => {
  if (response.url().includes('/api/metrics')) {
    await expect(response).toBeOK();
  }
});
```

## Soft Assertions

By default, a failed assertion stops the test immediately. Soft assertions collect all failures and report them together at the end:

```typescript
test('verifies all product page elements', async ({ page }) => {
  await page.goto('/products/wireless-headphones');

  // Collect all failures instead of stopping at the first one
  await expect.soft(page.getByRole('heading', { level: 1 })).toHaveText('Wireless Headphones Pro');
  await expect.soft(page.getByTestId('price')).toHaveText('$149.99');
  await expect.soft(page.getByTestId('sku')).toHaveText('WHP-001');
  await expect.soft(page.getByRole('button', { name: 'Add to cart' })).toBeEnabled();
  await expect.soft(page.getByRole('img', { name: 'Product image' })).toBeVisible();
  await expect.soft(page.getByRole('tab', { name: 'Reviews' })).toBeVisible();

  // All soft assertion failures are reported together
  // Hard assertions still stop immediately:
  await expect(page).toHaveURL('/products/wireless-headphones');
});
```

## Custom Assertion Timeout

Override the default timeout for specific assertions:

```typescript
// Default timeout (from playwright.config.ts)
await expect(page.getByRole('button')).toBeVisible();

// Short timeout for elements that should appear immediately
await expect(page.getByRole('tooltip')).toBeVisible({ timeout: 1000 });

// Long timeout for slow operations
await expect(page.getByRole('status')).toHaveText('Import complete', { timeout: 60000 });
```

Configure the global assertion timeout:

```typescript
// playwright.config.ts
expect: {
  timeout: 10000, // 10 seconds (default is 5000)
},
```

## Custom Expect Messages

Add context to assertion failures with a custom message:

```typescript
await expect(
  page.getByRole('heading', { level: 1 }),
  'Page should show the user name in the H1 after login',
).toHaveText('Alice Johnson');

// Using test.step for structured reporting
await test.step('Verify order confirmation page', async () => {
  await expect(page, 'Should redirect to order confirmation').toHaveURL(/\/orders\/\d+/);
  await expect(
    page.getByRole('heading', { name: 'Order Confirmed' }),
    'Confirmation heading should be visible',
  ).toBeVisible();
  await expect(
    page.getByTestId('order-number'),
    'Order number should be displayed',
  ).toBeVisible();
});
```

## Full Assertion Reference

| Assertion | Description |
|---|---|
| `toBeVisible()` | Element is visible |
| `toBeHidden()` | Element is hidden or detached |
| `toBeAttached()` | Element is in the DOM |
| `toBeEnabled()` | Form element is not disabled |
| `toBeDisabled()` | Form element is disabled |
| `toBeEditable()` | Input can be edited |
| `toBeChecked()` | Checkbox/radio is checked |
| `toBeEmpty()` | Input has no value, element has no children |
| `toBeFocused()` | Element has focus |
| `toBeInViewport()` | Element is in the viewport |
| `toHaveText(str)` | Element text matches |
| `toContainText(str)` | Element text contains substring |
| `toHaveValue(str)` | Input/select has value |
| `toHaveValues(arr)` | Multi-select has values |
| `toHaveAttribute(name, val)` | Element has attribute |
| `toHaveClass(str)` | Element has CSS class |
| `toHaveCSS(prop, val)` | Element has CSS property value |
| `toHaveCount(n)` | Locator matches N elements |
| `toHaveId(str)` | Element has id attribute |
| `toHaveURL(str)` | Page URL matches |
| `toHaveTitle(str)` | Page title matches |
| `toHaveScreenshot(name)` | Visual regression match |
| `toBeOK()` | API response is 2xx |

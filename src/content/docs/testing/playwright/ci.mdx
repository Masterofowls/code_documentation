Running Playwright in CI requires attention to browser installation, parallelism, retry strategies, artifact collection, and environment-specific configuration. This guide covers everything needed to make Playwright tests reliable, fast, and observable in CI pipelines.

## GitHub Actions — Complete Workflow

```yaml
# .github/workflows/playwright.yml
name: Playwright Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  test:
    name: 'Playwright Tests — Shard ${{ matrix.shardIndex }}/${{ matrix.shardTotal }}'
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        shardIndex: [1, 2, 3, 4]
        shardTotal: [4]

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      # Cache browser binaries to avoid re-downloading on every run
      - name: Cache Playwright browsers
        uses: actions/cache@v4
        id: playwright-cache
        with:
          path: ~/.cache/ms-playwright
          key: playwright-browsers-${{ runner.os }}-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            playwright-browsers-${{ runner.os }}-

      - name: Install Playwright browsers
        if: steps.playwright-cache.outputs.cache-hit != 'true'
        run: npx playwright install --with-deps

      - name: Install system dependencies only (when browsers are cached)
        if: steps.playwright-cache.outputs.cache-hit == 'true'
        run: npx playwright install-deps

      - name: Run Playwright tests
        run: npx playwright test --shard=${{ matrix.shardIndex }}/${{ matrix.shardTotal }}
        env:
          BASE_URL: ${{ vars.STAGING_URL || 'http://localhost:3000' }}
          TEST_USER_EMAIL: ${{ secrets.TEST_USER_EMAIL }}
          TEST_USER_PASSWORD: ${{ secrets.TEST_USER_PASSWORD }}
          TEST_ADMIN_EMAIL: ${{ secrets.TEST_ADMIN_EMAIL }}
          TEST_ADMIN_PASSWORD: ${{ secrets.TEST_ADMIN_PASSWORD }}
          CI: true

      # Upload blob reports for later merging
      - name: Upload blob report
        if: ${{ !cancelled() }}
        uses: actions/upload-artifact@v4
        with:
          name: blob-report-${{ matrix.shardIndex }}
          path: blob-report
          retention-days: 1

  # Merge shard reports into one HTML report
  merge-reports:
    name: Merge and Publish Reports
    if: ${{ !cancelled() }}
    needs: test
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci

      - name: Download blob reports from all shards
        uses: actions/download-artifact@v4
        with:
          path: all-blob-reports
          pattern: blob-report-*
          merge-multiple: true

      - name: Merge reports into HTML
        run: npx playwright merge-reports --reporter html ./all-blob-reports

      - name: Upload merged HTML report
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 30
```

## Configuring playwright.config.ts for CI

```typescript
// playwright.config.ts
import { defineConfig, devices } from '@playwright/test';
import os from 'os';

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,

  // Never allow .only() in CI — prevents shipping focused tests
  forbidOnly: !!process.env.CI,

  // Retry failed tests in CI to handle flakiness
  retries: process.env.CI ? 2 : 0,

  // Workers: in CI use 1 worker per shard; locally use half the CPUs
  workers: process.env.CI ? 1 : Math.floor(os.cpus().length / 2),

  // Reporter configuration
  reporter: process.env.CI
    ? [
        ['blob'],           // for shard merging
        ['github'],         // GitHub Actions annotations
        ['junit', { outputFile: 'test-results/junit.xml' }],
      ]
    : [
        ['html', { open: 'on-failure' }],
        ['list'],
      ],

  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
    video: 'retain-on-failure',
    actionTimeout: 15000,
    navigationTimeout: 30000,
  },

  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    // Only run Firefox and WebKit on main branch or when explicitly requested
    ...(process.env.CI && process.env.GITHUB_REF !== 'refs/heads/main'
      ? []
      : [
          { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
          { name: 'webkit', use: { ...devices['Desktop Safari'] } },
        ]),
  ],
});
```

## Retries

```typescript
// playwright.config.ts
export default defineConfig({
  // Retry all failed tests up to 2 times in CI
  retries: process.env.CI ? 2 : 0,
});
```

```typescript
// Override retries for specific tests
test('flaky integration test', async ({ page }) => {
  test.slow(); // triples the test timeout
  // ...
});

// Annotate known flaky tests
test('known flaky test', async ({ page }) => {
  test.fixme(!!process.env.CI, 'Flaky in CI — tracked in GH-456');
  // ...
});
```

Tests that pass on a retry are marked as **flaky** in the HTML report — this helps identify tests that need investigation.

:::tip[Retries Are Not a Fix]
Retries reduce flaky failures in CI output but don't fix the underlying cause. A test that retries twice before passing adds significant time to your CI pipeline. Track flaky tests and fix their root causes.
:::

## Sharding

Sharding splits your test suite across multiple CI machines, reducing total wall-clock time:

```bash
# Run shard 1 of 4 (each machine runs 25% of tests)
npx playwright test --shard=1/4

# In GitHub Actions matrix strategy:
# matrix.shardIndex: [1, 2, 3, 4]
# Run: npx playwright test --shard=${{ matrix.shardIndex }}/${{ matrix.shardTotal }}
```

Each shard produces a `blob-report/` directory with partial results. Merge them after all shards complete:

```bash
# Merge all blob reports into one HTML report
npx playwright merge-reports --reporter html ./all-blob-reports

# Or merge into multiple formats
npx playwright merge-reports --reporter html,json ./all-blob-reports
```

## Parallelism

Within a single machine, Playwright runs test files in parallel using worker processes:

```typescript
// playwright.config.ts
export default defineConfig({
  // Each file gets its own worker (default)
  fullyParallel: false,

  // Each test in every file runs in parallel (faster but requires isolation)
  fullyParallel: true,

  // Explicit worker count
  workers: 4,           // absolute number
  workers: '50%',       // percentage of CPUs
});
```

Force serial execution for tests that share state:

```typescript
// Mark a test file as serial (tests run in order, one at a time)
test.describe.configure({ mode: 'serial' });

test.describe('Checkout flow', () => {
  test.describe.configure({ mode: 'serial' });

  test('step 1: add to cart', async ({ page }) => { /* ... */ });
  test('step 2: enter shipping', async ({ page }) => { /* ... */ });
  test('step 3: confirm payment', async ({ page }) => { /* ... */ });
});
```

## Reporters

```typescript
// playwright.config.ts
reporter: [
  // Terminal output
  ['list'],       // one line per test
  ['dot'],        // minimal dots
  ['line'],       // single line with progress

  // HTML report
  ['html', {
    outputFolder: 'playwright-report',
    open: 'on-failure', // auto-open on failure (local only)
  }],

  // JUnit XML (Azure DevOps, Jenkins, GitLab)
  ['junit', {
    outputFile: 'test-results/results.xml',
    suiteName: 'E2E Tests',
  }],

  // GitHub Actions annotations
  ['github'],

  // JSON output for custom tooling
  ['json', { outputFile: 'test-results/results.json' }],

  // Blob for shard merging
  ['blob', { outputDir: 'blob-report' }],

  // Allure reporter
  ['allure-playwright', {
    detail: true,
    outputFolder: 'allure-results',
    suiteTitle: false,
  }],
],
```

```bash
# Install Allure reporter
npm install --save-dev allure-playwright

# Generate and open Allure report
npx allure generate allure-results --clean -o allure-report
npx allure open allure-report
```

## Docker

Use Microsoft's official Playwright Docker image for consistent browser versions:

```dockerfile
# Dockerfile.ci
FROM mcr.microsoft.com/playwright:v1.48.0-jammy

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .

# Run tests
CMD ["npx", "playwright", "test"]
```

```yaml
# docker-compose.test.yml
services:
  playwright:
    build:
      context: .
      dockerfile: Dockerfile.ci
    environment:
      - BASE_URL=http://app:3000
      - CI=true
    depends_on:
      app:
        condition: service_healthy

  app:
    build: .
    ports:
      - "3000:3000"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 5s
      timeout: 5s
      retries: 10
```

```yaml
# GitHub Actions with Docker
- name: Run Playwright tests in Docker
  run: docker compose -f docker-compose.test.yml run playwright
```

:::tip[Consistent Visual Tests with Docker]
Always run screenshot comparison tests in the same Docker image to ensure identical font rendering, anti-aliasing, and color profiles. Visual test failures caused by OS differences are one of the most common CI headaches.
:::

## Environment Variables

```typescript
// playwright.config.ts — use environment variables for all environment-specific values
export default defineConfig({
  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:3000',
  },
  webServer: process.env.CI
    ? undefined  // In CI, the app is already running
    : {
        command: 'npm run dev',
        url: 'http://localhost:3000',
        reuseExistingServer: true,
      },
});
```

```bash
# .env.test (committed — no secrets)
BASE_URL=http://localhost:3000
PLAYWRIGHT_BROWSERS_PATH=0

# .env.ci (injected by CI as secrets)
BASE_URL=https://staging.example.com
TEST_USER_EMAIL=test@example.com
TEST_USER_PASSWORD=testpass123
```

Accessing secrets in tests:

```typescript
test('logs in with credentials', async ({ page }) => {
  await page.goto('/login');
  await page.getByLabel('Email').fill(process.env.TEST_USER_EMAIL!);
  await page.getByLabel('Password').fill(process.env.TEST_USER_PASSWORD!);

});
```

## Caching Browser Binaries

```yaml
# Cache based on Playwright version (via package-lock.json hash)
- name: Cache Playwright binaries
  uses: actions/cache@v4
  id: playwright-cache
  with:
    path: |
      ~/.cache/ms-playwright
      %LOCALAPPDATA%\ms-playwright  # Windows
    key: playwright-${{ runner.os }}-${{ hashFiles('**/package-lock.json') }}

- name: Install Playwright (full install if cache miss)
  if: steps.playwright-cache.outputs.cache-hit != 'true'
  run: npx playwright install --with-deps

- name: Install only system deps (cache hit — browsers already cached)
  if: steps.playwright-cache.outputs.cache-hit == 'true'
  run: npx playwright install-deps
```

## Flaky Test Detection

```bash
# Run each test multiple times to detect flakiness
npx playwright test --repeat-each=3

# Run with retries and see which tests are flaky
npx playwright test --retries=3

# Run a specific test many times to reproduce intermittent failure
npx playwright test tests/checkout.spec.ts --repeat-each=10 --reporter=list
```

```yaml
# Scheduled job to detect flaky tests
on:
  schedule:
    - cron: '0 2 * * *'  # Run at 2 AM daily

jobs:
  flaky-detection:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npx playwright install --with-deps
      - run: npx playwright test --repeat-each=5 --reporter=json
      - name: Parse and report flaky tests
        run: node scripts/report-flaky-tests.js test-results/results.json
```

## Artifact Upload on Failure

```yaml
- name: Upload test results on failure
  if: failure()
  uses: actions/upload-artifact@v4
  with:
    name: playwright-failures-${{ github.run_id }}
    path: |
      test-results/
      playwright-report/
    retention-days: 14

- name: Upload traces on failure
  if: failure()
  uses: actions/upload-artifact@v4
  with:
    name: playwright-traces-${{ github.run_id }}
    path: test-results/**/*.zip
    retention-days: 7
```

:::caution[Artifact Storage Costs]
Test artifacts (videos, traces, screenshots) can consume significant storage in GitHub Actions. Set appropriate retention days and only collect artifacts on failure (`if: failure()`) rather than on every run.
:::

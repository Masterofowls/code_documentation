This guide covers everything needed to go from zero to a running Playwright test suite — from installation through your first test and CI-ready configuration.

## Installation

The official initializer scaffolds a complete project structure:

```bash
npm init playwright@latest
```

The interactive wizard asks:

- **Language**: TypeScript or JavaScript
- **Test directory**: `tests/` (default)
- **GitHub Actions**: Yes/No (adds a workflow file)
- **Install browsers**: Yes (installs Chromium, Firefox, WebKit)

### Manual Installation

```bash
npm install --save-dev @playwright/test

# Install browser binaries
npx playwright install

# Install browsers AND system dependencies (for CI)
npx playwright install --with-deps

# Install only specific browsers
npx playwright install chromium firefox
```

## Project Structure

After initialization, your project looks like this:

```
project/
├── tests/
│   ├── example.spec.ts         ← sample test generated by init
│   └── auth.setup.ts           ← authentication setup (if configured)
├── tests-examples/
│   └── demo-todo-app.spec.ts   ← more examples
├── playwright.config.ts        ← main configuration file
├── package.json
└── .github/
    └── workflows/
        └── playwright.yml      ← CI workflow (if selected)
```

## playwright.config.ts

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  // Directory where tests are located
  testDir: './tests',

  // Run test files in parallel
  fullyParallel: true,

  // Fail the build on CI if you accidentally left test.only in the source code
  forbidOnly: !!process.env.CI,

  // Retry failed tests in CI
  retries: process.env.CI ? 2 : 0,

  // Opt out of parallel tests on CI (use sharding instead)
  workers: process.env.CI ? 1 : undefined,

  // Reporter to use
  reporter: 'html',

  // Shared settings for all projects
  use: {
    // Base URL — allows relative paths in page.goto('/')
    baseURL: process.env.BASE_URL ?? 'http://localhost:3000',

    // Collect trace on first retry of failed tests
    trace: 'on-first-retry',

    // Capture screenshot on failure
    screenshot: 'only-on-failure',

    // Record video on first retry
    video: 'on-first-retry',

    // Maximum time for actions (click, fill, etc.)
    actionTimeout: 15000,

    // Maximum time for page navigations
    navigationTimeout: 30000,
  },

  // Configure browser projects
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
    },
    // Mobile viewports
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 5'] },
    },
    {
      name: 'Mobile Safari',
      use: { ...devices['iPhone 12'] },
    },
  ],

  // Start a local dev server before running tests
  webServer: {
    command: 'npm run dev',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    stdout: 'pipe',
    stderr: 'pipe',
  },
});
```

## Installing Browsers

Playwright manages its own browser binaries:

```bash
# Install all browsers (Chromium, Firefox, WebKit)
npx playwright install

# Install specific browsers
npx playwright install chromium
npx playwright install webkit

# Install browsers WITH system dependencies (Linux CI)
npx playwright install --with-deps

# Install only system dependencies (not browsers)
npx playwright install-deps

# Show installed browsers
npx playwright --version
```

:::caution[Browser Binaries are not Chrome/Firefox]
Playwright installs its own patched versions of browser binaries, separate from any installed browsers. The binaries are stored in `~/.cache/ms-playwright/`. In CI, always run `npx playwright install --with-deps` to install system libraries (libglib, libx11, etc.).
:::

## Running Tests

```bash
# Run all tests
npx playwright test

# Run a specific test file
npx playwright test tests/login.spec.ts

# Run tests matching a name pattern
npx playwright test --grep "user can log in"
npx playwright test --grep-invert "slow"

# Run with a specific browser project
npx playwright test --project=chromium
npx playwright test --project=webkit --project=firefox

# Run in headed mode (see the browser)
npx playwright test --headed

# Run with UI mode (interactive test runner)
npx playwright test --ui

# Run and keep the browser open after tests
npx playwright test --headed --timeout=0

# Run in debug mode (pauses at first action)
npx playwright test --debug
npx PWDEBUG=1 npx playwright test
```

## Headed Mode

By default, Playwright runs tests headlessly. In headed mode, you see the browser:

```bash
npx playwright test --headed
npx playwright test --headed --slowMo=500  # slow down actions by 500ms
```

Headed mode is useful for:
- Visually debugging why a test fails
- Understanding the timing of animations
- Verifying that a locator correctly identifies the right element

## UI Mode

UI mode is Playwright's interactive test runner — one of its most powerful features:

```bash
npx playwright test --ui
```

UI mode provides:
- Real-time test execution with visual feedback
- Filter tests by name, file, tag, or project
- Re-run individual tests with one click
- Built-in Trace Viewer for any test
- Timeline of all actions with DOM snapshots
- Network traffic panel

:::tip[Use UI Mode During Development]
UI mode dramatically speeds up the test-writing workflow. Open it with `npx playwright test --ui`, write your test, save the file, and UI mode automatically re-runs it — showing you exactly what happened.
:::

## Trace Viewer

The Trace Viewer shows a complete recording of test execution:

```bash
# Record traces on failure
npx playwright test --trace on-first-retry

# Always record
npx playwright test --trace on

# Open trace from a file
npx playwright show-trace test-results/my-test/trace.zip

# View the HTML report (includes traces for failed tests)
npx playwright show-report
```

The trace contains:
- A timeline of every action
- DOM snapshots before and after each action
- Network requests and responses
- Console messages
- Screenshots and optionally video

## HTML Report

Jest generates an HTML report after each test run:

```bash
# Open the report
npx playwright show-report

# Configure where to save the report
# playwright.config.ts
reporter: [['html', { outputFolder: 'playwright-report', open: 'on-failure' }]],
```

The HTML report shows:
- Pass/fail/flaky status per test
- Grouped by test file and browser
- Attached traces, screenshots, and videos
- Error messages and stack traces

## Your First Test

```typescript
// tests/home.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Homepage', () => {
  test('shows the welcome heading', async ({ page }) => {
    await page.goto('/');

    // Assert the page title
    await expect(page).toHaveTitle(/My App/);

    // Assert a heading is visible
    await expect(page.getByRole('heading', { name: 'Welcome to My App' }))
      .toBeVisible();
  });

  test('navigation links work correctly', async ({ page }) => {
    await page.goto('/');

    // Click the About link
    await page.getByRole('link', { name: 'About' }).click();

    // Assert we navigated to the right page
    await expect(page).toHaveURL('/about');
    await expect(page.getByRole('heading', { name: 'About Us' })).toBeVisible();
  });

  test('contact form submits successfully', async ({ page }) => {
    await page.goto('/contact');

    await page.getByLabel('Name').fill('Alice Johnson');
    await page.getByLabel('Email').fill('alice@example.com');
    await page.getByLabel('Message').fill('Hello from Playwright!');
    await page.getByRole('button', { name: 'Send Message' }).click();

    // Assert success message appears
    await expect(page.getByText('Message sent successfully')).toBeVisible();
  });
});
```

## TypeScript Configuration

Playwright has first-class TypeScript support — no additional setup needed. The `@playwright/test` package ships with TypeScript declarations.

For IDE support and custom fixtures, create a `tsconfig.json` that includes the test directory:

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "outDir": "./dist"
  },
  "include": ["tests/**/*.ts", "playwright.config.ts"]
}
```

## Common Configuration Options Reference

| Option | Default | Description |
|---|---|---|
| `testDir` | `./tests` | Directory to search for test files |
| `timeout` | `30000` | Max time per test (ms) |
| `expect.timeout` | `5000` | Max time for web-first assertions |
| `retries` | `0` | Auto-retry failed tests N times |
| `workers` | `50% of CPUs` | Number of parallel workers |
| `fullyParallel` | `false` | Run all tests in parallel |
| `use.baseURL` | `undefined` | Base URL for `page.goto('/')` |
| `use.headless` | `true` | Run without visible browser |
| `use.viewport` | `1280x720` | Default browser viewport |
| `use.locale` | system | Browser locale (`en-US`, `fr-FR`) |
| `use.timezoneId` | system | Timezone (`America/New_York`) |
| `use.geolocation` | `undefined` | Mock geolocation |
| `use.permissions` | `[]` | Browser permissions to grant |
| `use.storageState` | `undefined` | Pre-loaded auth state |

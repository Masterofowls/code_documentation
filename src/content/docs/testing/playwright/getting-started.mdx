This guide walks through setting up Playwright from scratch — creating a project, installing browsers, running tests in different modes, and configuring `playwright.config.ts` for real-world use.

## Initialize a New Project

The quickest way to set up Playwright is with the official initializer:

```bash
npm init playwright@latest
```

The wizard will prompt you to:
1. Choose TypeScript or JavaScript
2. Name the tests directory (default: `tests/` or `e2e/`)
3. Optionally add a GitHub Actions workflow
4. Install Playwright browsers

After completion, your project will have:

```
project/
├── tests/
│   └── example.spec.ts      ← sample test file
├── tests-examples/
│   └── demo-todo-app.spec.ts ← more complex example
├── playwright.config.ts      ← configuration file
├── package.json
└── .github/
    └── workflows/
        └── playwright.yml    ← CI workflow (if selected)
```

## Installing Browsers

Playwright downloads its own browser binaries — separate from any browsers installed on your system:

```bash
# Install all browsers (Chromium, Firefox, WebKit)
npx playwright install

# Install only specific browsers
npx playwright install chromium
npx playwright install chromium firefox

# Install browsers with system dependencies (required in CI)
npx playwright install --with-deps

# Install a specific browser version
npx playwright install chromium@1097
```

:::tip[Browser Binary Location]
Playwright stores browser binaries in `~/.cache/ms-playwright/` on Linux/macOS and `%USERPROFILE%\AppData\Local\ms-playwright\` on Windows. These binaries are version-pinned to the installed Playwright version.
:::

## Running Tests

```bash
# Run all tests (headless by default)
npx playwright test

# Run a specific file
npx playwright test tests/login.spec.ts

# Run tests matching a title pattern
npx playwright test --grep "dashboard"
npx playwright test --grep-invert "slow"  # exclude matching tests

# Run in specific browser
npx playwright test --project=chromium
npx playwright test --project=firefox
npx playwright test --project=webkit

# Run multiple browsers simultaneously
npx playwright test --project=chromium --project=firefox
```

### Headed Mode

```bash
# Run with browser window visible
npx playwright test --headed

# Run headed in a specific browser
npx playwright test --headed --project=chromium

# Slow down execution for visual debugging
npx playwright test --headed --slowmo=500  # 500ms delay between actions
```

### UI Mode

UI Mode provides an interactive, visual test runner — the best way to develop and debug tests:

```bash
npx playwright test --ui
```

Features of UI Mode:
- Visual test list with pass/fail status
- Live browser preview as tests run
- Time-travel debugging through recorded actions
- Watch mode — re-runs tests on file save
- Network request inspection
- Console log viewer

### Trace Viewer

The Trace Viewer records detailed diagnostic information for debugging failures:

```bash
# Record trace for all tests
npx playwright test --trace on

# Record trace only on failure (recommended for CI)
npx playwright test --trace retain-on-failure

# Open a recorded trace
npx playwright show-trace test-results/login-chromium/trace.zip
```

Trace viewer shows:
- A timeline of all actions
- Before/after screenshots for each action
- DOM snapshots at each step
- Network requests and responses
- Console logs and errors

### HTML Report

After running tests, view the HTML report:

```bash
npx playwright show-report
```

Or open `playwright-report/index.html` directly.

## playwright.config.ts

The configuration file controls every aspect of test execution:

```typescript
// playwright.config.ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  // Directory where test files are located
  testDir: './tests',

  // Match test files
  testMatch: '**/*.spec.ts',

  // Global timeout for each test
  timeout: 30_000,

  // Timeout for expect() assertions
  expect: { timeout: 5_000 },

  // Number of retries on failure (0 in dev, 2 in CI)
  retries: process.env.CI ? 2 : 0,

  // Number of parallel workers
  workers: process.env.CI ? 1 : undefined, // undefined = auto-detect

  // Fail the build if any test.only is committed
  forbidOnly: !!process.env.CI,

  // Reporter configuration
  reporter: [
    ['html', { outputFolder: 'playwright-report', open: 'never' }],
    ['list'],
  ],

  // Shared settings for all projects
  use: {
    // Base URL — use relative paths in tests
    baseURL: process.env.BASE_URL ?? 'http://localhost:3000',

    // Run tests headless (true) or with browser visible (false)
    headless: true,

    // Record a trace on first retry
    trace: 'on-first-retry',

    // Record video on first retry
    video: 'retain-on-failure',

    // Screenshot on failure
    screenshot: 'only-on-failure',

    // Viewport
    viewport: { width: 1280, height: 720 },

    // Bypass CSP for easier testing
    bypassCSP: false,

    // Authentication state (see Page Objects guide)
    // storageState: 'playwright/.auth/user.json',
  },

  // Browser projects
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
    },
    // Mobile viewports
    {
      name: 'mobile-chrome',
      use: { ...devices['Pixel 5'] },
    },
    {
      name: 'mobile-safari',
      use: { ...devices['iPhone 13'] },
    },
  ],

  // Start a local dev server before running tests
  webServer: {
    command: 'npm run dev',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    timeout: 120_000,
  },
});
```

### Key Config Options Reference

| Option | Type | Description |
|---|---|---|
| `testDir` | `string` | Directory to search for test files |
| `timeout` | `number` | Max ms for a single test (default: 30000) |
| `retries` | `number` | Times to retry a failing test |
| `workers` | `number` | Parallel worker processes |
| `forbidOnly` | `boolean` | Fail if `test.only` is found |
| `use.baseURL` | `string` | Base URL for `page.goto('/')` |
| `use.headless` | `boolean` | Run without browser UI |
| `use.trace` | `string` | When to record trace |
| `use.video` | `string` | When to record video |
| `use.screenshot` | `string` | When to take screenshot |
| `use.storageState` | `string` | Pre-load auth cookies/localStorage |
| `webServer` | `object` | Start a local server before tests |

## Your First Test

```typescript
// tests/homepage.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Homepage', () => {
  test('displays the page title', async ({ page }) => {
    await page.goto('/');
    await expect(page).toHaveTitle(/My App/);
  });

  test('has a visible navigation bar', async ({ page }) => {
    await page.goto('/');
    await expect(page.getByRole('navigation')).toBeVisible();
  });

  test('navigates to the about page', async ({ page }) => {
    await page.goto('/');
    await page.getByRole('link', { name: 'About' }).click();
    await expect(page).toHaveURL('/about');
    await expect(page.getByRole('heading', { name: 'About Us', level: 1 })).toBeVisible();
  });
});
```

Run it:

```bash
npx playwright test tests/homepage.spec.ts --project=chromium
```

## TypeScript Support

Playwright has first-class TypeScript support with no configuration required. Types are included in the `@playwright/test` package:

```typescript
import { test, expect, type Page, type BrowserContext } from '@playwright/test';

// Typed fixtures
test('typed test', async ({ page, context, browser, request }) => {
  // page: Page
  // context: BrowserContext
  // browser: Browser
  // request: APIRequestContext
});
```

For strict TypeScript, add these to your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "strict": true,
    "module": "commonjs",
    "target": "ES2020",
    "lib": ["ES2020"],
    "types": ["@playwright/test"]
  },
  "include": ["tests/**/*.ts", "playwright.config.ts"]
}
```

:::tip[Separate tsconfig for Tests]
Keep a separate `tsconfig.json` in your tests directory to avoid Playwright types conflicting with Jest types in projects that use both.
:::

## webServer: Starting Your App Automatically

```typescript
// playwright.config.ts
webServer: {
  command: 'npm run build && npm run preview',
  url: 'http://localhost:4173',
  reuseExistingServer: !process.env.CI,
  stdout: 'pipe',
  stderr: 'pipe',
  timeout: 120_000,
},
```

Playwright will start your server, wait until `url` responds with a 200, then run all tests. It tears down the server when tests finish.

This guide walks you through installing Playwright from scratch, understanding the generated project structure, and running your first tests with the full suite of developer tools.

## Installation

The fastest way to start is with the official initializer:

```bash
npm init playwright@latest
```

The initializer prompts you to choose:
- JavaScript or TypeScript (choose TypeScript)
- Where to put tests (default: `tests/`)
- Whether to add a GitHub Actions workflow
- Whether to install browsers now

Or install manually:

```bash
npm install --save-dev @playwright/test
```

## Installing Browsers

Playwright bundles its own browser binaries. Install them after package installation:

```bash
# Install all browsers (Chromium, Firefox, WebKit)
npx playwright install

# Install specific browser only
npx playwright install chromium
npx playwright install firefox
npx playwright install webkit

# Install with system dependencies (required in CI / Docker)
npx playwright install --with-deps

# Install specific version
npx playwright install chromium@1091
```

Browsers are stored in `~/.cache/ms-playwright` (or `%LOCALAPPDATA%\ms-playwright` on Windows).

:::tip[Pinning Browser Versions]
Playwright pins browser versions to each package release. Run `npx playwright install` after every `@playwright/test` version upgrade to keep browsers in sync.
:::

## Project Structure

After running `npm init playwright@latest`:

```
my-project/
├── tests/
│   └── example.spec.ts          ← sample test file
├── tests-examples/
│   └── demo-todo-app.spec.ts    ← more complete examples
├── playwright.config.ts          ← main configuration file
├── package.json
└── .github/
    └── workflows/
        └── playwright.yml        ← GitHub Actions workflow (if selected)
```

For larger projects, a recommended structure:

```
my-project/
├── tests/
│   ├── auth/
│   │   ├── login.spec.ts
│   │   └── registration.spec.ts
│   ├── dashboard/
│   │   └── overview.spec.ts
│   └── global-setup.ts          ← global auth setup
├── pages/                        ← Page Object Models
│   ├── BasePage.ts
│   ├── LoginPage.ts
│   └── DashboardPage.ts
├── fixtures/                     ← shared test fixtures
│   └── index.ts
├── playwright.config.ts
└── .auth/                        ← stored auth state (gitignored)
    └── user.json
```

## playwright.config.ts

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  // Directory containing test files
  testDir: './tests',

  // Run test files in parallel
  fullyParallel: true,

  // Fail fast: stop after N failures in CI
  forbidOnly: !!process.env.CI,

  // Retry failed tests in CI
  retries: process.env.CI ? 2 : 0,

  // Number of parallel workers
  workers: process.env.CI ? 1 : undefined,

  // Reporter configuration
  reporter: [
    ['html'],
    process.env.CI ? ['github'] : ['list'],
  ],

  // Shared settings for all projects
  use: {
    // Base URL for page.goto('/')
    baseURL: process.env.BASE_URL || 'http://localhost:3000',

    // Collect trace on first retry only (saves disk space)
    trace: 'on-first-retry',

    // Screenshot on failure
    screenshot: 'only-on-failure',

    // Video recording
    video: 'retain-on-failure',

    // Global timeout for each action
    actionTimeout: 10000,

    // Navigation timeout
    navigationTimeout: 30000,
  },

  // Browser projects
  projects: [
    // Setup project (runs global-setup.ts before tests)
    {
      name: 'setup',
      testMatch: /global-setup\.ts/,
    },

    // Test projects
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
      dependencies: ['setup'],
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
      dependencies: ['setup'],
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
      dependencies: ['setup'],
    },

    // Mobile viewports
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 5'] },
      dependencies: ['setup'],
    },
  ],

  // Start a development server before tests
  webServer: {
    command: 'npm run start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    timeout: 120000,
  },

  // Global timeout per test
  timeout: 30000,

  // Maximum number of failures before aborting
  maxFailures: process.env.CI ? 10 : undefined,
});
```

## Your First Test

```typescript
// tests/homepage.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Homepage', () => {
  test('has the correct title', async ({ page }) => {
    await page.goto('/');

    await expect(page).toHaveTitle(/My Application/);
  });

  test('shows the main navigation', async ({ page }) => {
    await page.goto('/');

    await expect(page.getByRole('navigation')).toBeVisible();
    await expect(page.getByRole('link', { name: 'Home' })).toBeVisible();
    await expect(page.getByRole('link', { name: 'About' })).toBeVisible();
  });

  test('navigates to about page', async ({ page }) => {
    await page.goto('/');

    await page.getByRole('link', { name: 'About' }).click();

    await expect(page).toHaveURL('/about');
    await expect(page.getByRole('heading', { name: 'About Us', level: 1 })).toBeVisible();
  });
});
```

## Running Tests

```bash
# Run all tests
npx playwright test

# Run specific file
npx playwright test tests/homepage.spec.ts

# Run tests matching a name
npx playwright test -g "shows the main navigation"

# Run in a specific browser
npx playwright test --project=chromium
npx playwright test --project=firefox

# Run in headed mode (see the browser)
npx playwright test --headed

# Run with UI mode (visual test runner)
npx playwright test --ui

# Show the HTML report
npx playwright show-report

# Run in debug mode (pauses before each action)
npx playwright test --debug

# Run with trace recording
npx playwright test --trace on
```

## Headed Mode

By default, Playwright runs in **headless** mode (no visible browser window). Use `--headed` to see the browser during development:

```bash
npx playwright test --headed
npx playwright test --headed --slowMo=500  # slow down by 500ms per action
```

Or configure in `playwright.config.ts`:

```typescript
use: {
  headless: false,
  slowMo: 100, // ms delay between actions
}
```

## UI Mode

UI Mode is Playwright's interactive test runner. It shows:
- File tree of all tests with pass/fail status
- Real-time test execution in a browser panel
- Time-travel through trace timeline
- Watch mode — re-run tests as you save files
- Filter tests by name, tag, or status

```bash
npx playwright test --ui
```

## Trace Viewer

The trace file captures a complete recording of a test run, allowing you to replay and inspect it after the fact:

```bash
# Record traces for all tests
npx playwright test --trace on

# Record only on failure (recommended for CI)
npx playwright test --trace on-first-retry

# View a trace file
npx playwright show-trace test-results/homepage-has-title/trace.zip
```

The trace viewer shows:
- Timeline of all actions
- Before/after screenshots for each step
- Network requests (URL, headers, body, status)
- Console messages
- DOM snapshot (you can inspect elements)
- Source code with highlighted current line

## HTML Report

After a test run, view the HTML report:

```bash
# Open the HTML report in a browser
npx playwright show-report

# Specify a custom report directory
npx playwright show-report my-report
```

The report shows:
- Test summary (passed, failed, skipped, flaky)
- Per-test details with trace links
- Screenshots and video attachments on failure
- Test duration and retry history

Configure the report output:

```typescript
reporter: [
  ['html', { outputFolder: 'playwright-report', open: 'on-failure' }],
],
```

## TypeScript Support

Playwright has first-class TypeScript support — no configuration needed. Tests are executed directly using `esbuild` (no separate `tsc` step required):

```typescript
// All Playwright types are automatically available
import { test, expect, Page, BrowserContext, Locator } from '@playwright/test';

// Type your page objects and fixtures
interface UserFixtures {
  loginPage: LoginPage;
  authenticatedPage: Page;
}

const test = base.extend<UserFixtures>({
  loginPage: async ({ page }, use) => {
    await use(new LoginPage(page));
  },
});
```

Create a `tsconfig.json` for your tests:

```json
{
  "extends": "../tsconfig.json",
  "compilerOptions": {
    "target": "ES2022",
    "module": "CommonJS",
    "lib": ["ES2022"],
    "strict": true,
    "noUncheckedIndexedAccess": true
  },
  "include": ["tests/**/*.ts", "pages/**/*.ts", "fixtures/**/*.ts"]
}
```

:::tip[No Type Checking at Runtime]
Playwright uses esbuild for fast execution and does not perform type checking at test runtime. Run `tsc --noEmit` separately in CI to catch type errors in your test files.
:::

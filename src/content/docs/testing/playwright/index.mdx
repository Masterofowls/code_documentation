Playwright is a modern end-to-end testing framework developed by Microsoft that enables reliable, cross-browser automation across Chromium, Firefox, and WebKit using a single API. It was built by the same team that created Puppeteer and addresses the pain points of older browser automation tools.

## What is Playwright?

Playwright executes tests in real browsers — not simulated DOM environments. This means your tests interact with the exact same rendering engine, JavaScript runtime, and network stack that your users encounter.

```typescript
// A complete Playwright test
import { test, expect } from '@playwright/test';

test('user can log in and see their dashboard', async ({ page }) => {
  await page.goto('/login');

  await page.getByLabel('Email').fill('alice@example.com');
  await page.getByLabel('Password').fill('SecurePass123!');
  await page.getByRole('button', { name: 'Sign In' }).click();

  await expect(page).toHaveURL('/dashboard');
  await expect(page.getByRole('heading', { name: 'Welcome, Alice' })).toBeVisible();
});
```

## Cross-Browser Testing

Playwright supports three browser engines through a single, unified API:

| Engine | Browser | Use case |
|---|---|---|
| **Chromium** | Chrome, Edge, Brave | Default — covers ~65% of global browser usage |
| **Firefox** | Firefox | Gecko engine testing — catches Firefox-specific bugs |
| **WebKit** | Safari, iOS Safari | Critical for mobile web and Apple ecosystem |

```typescript
// playwright.config.ts — configure multiple browsers
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox',  use: { ...devices['Desktop Firefox'] } },
    { name: 'webkit',   use: { ...devices['Desktop Safari'] } },
    { name: 'mobile-chrome', use: { ...devices['Pixel 5'] } },
    { name: 'mobile-safari', use: { ...devices['iPhone 13'] } },
  ],
});
```

## Playwright vs Cypress vs Selenium

| Feature | **Playwright** | **Cypress** | **Selenium** |
|---|---|---|---|
| Languages | TypeScript, JavaScript, Python, Java, C# | JavaScript / TypeScript only | All major languages |
| Browsers | Chromium, Firefox, WebKit | Chromium, Firefox (no WebKit) | All browsers (via drivers) |
| Architecture | Out-of-process (CDP + WebSocket) | In-process (same JS context) | WebDriver protocol |
| Auto-waiting | Built-in, web-first assertions | Built-in | Manual waits required |
| Parallel tests | Per-test worker isolation | Limited (paid feature) | External grid required |
| Speed | Very fast | Fast | Slower |
| Debugging | Trace Viewer, UI Mode, codegen | Time Travel Debugger | Browser DevTools |
| Visual testing | Built-in screenshots | Plugin required | Plugin required |
| Network intercept | Full (mock, modify, abort) | Full | Limited |
| iframes | Full support | Limited | Full support |
| Multiple tabs | Full support | No | Full support |
| Mobile testing | Device emulation | Limited | Via Appium |
| File downloads | Built-in | Built-in | Complex |
| API testing | Built-in `request` fixture | Separate `cy.request` | External library |
| Component testing | Yes (experimental) | Yes (mature) | No |
| Pricing | Free, open source | Free core + paid features | Free, open source |

:::tip[When to Choose Playwright]
Choose Playwright for: cross-browser coverage including Safari/WebKit, parallel test execution with worker isolation, rich debugging tools (Trace Viewer), full network interception, and multi-tab or multi-frame scenarios. It is the most capable option for teams needing comprehensive E2E coverage.
:::

:::caution[When Cypress Still Wins]
Cypress has a more mature component testing story and a larger plugin ecosystem. If your team is heavily invested in Cypress and you only need Chromium + Firefox, switching has limited benefit. Playwright shines most for cross-browser requirements and large parallel test suites.
:::

## Playwright Architecture

Understanding Playwright's architecture helps you write better tests and debug failures:

```
Test Runner (Node.js)
    │
    ├── Worker 1 ─── BrowserContext ─── Page ─── CDP/WebSocket ─── Chromium
    ├── Worker 2 ─── BrowserContext ─── Page ─── CDP/WebSocket ─── Firefox
    └── Worker 3 ─── BrowserContext ─── Page ─── CDP/WebSocket ─── WebKit
```

### Browser, Context, and Page

| Concept | Analogy | Isolation level |
|---|---|---|
| **Browser** | The browser application | Shared process |
| **BrowserContext** | An incognito window | Isolated cookies, storage, auth |
| **Page** | A browser tab | Isolated navigation state |

Each test gets its own `BrowserContext` by default, providing complete isolation of cookies, localStorage, sessionStorage, and authentication state between tests.

```typescript
test('isolated contexts', async ({ browser }) => {
  // Create two isolated contexts (like two different users)
  const aliceContext = await browser.newContext();
  const bobContext = await browser.newContext();

  const alicePage = await aliceContext.newPage();
  const bobPage = await bobContext.newPage();

  await alicePage.goto('/dashboard'); // Alice's session
  await bobPage.goto('/dashboard');   // Bob's session — completely isolated
});
```

## Auto-Waiting Philosophy

Playwright's most important feature is **auto-waiting**. Every action and assertion automatically waits for the target element to reach the expected state before proceeding — no manual `waitFor`, `sleep`, or polling required.

```typescript
// ✓ Playwright waits automatically for each of these:
await page.getByRole('button', { name: 'Submit' }).click();
// Waits for: element to exist, be visible, be enabled, and be stable

await expect(page.getByText('Success')).toBeVisible();
// Retries assertion until element is visible or timeout expires
```

This eliminates the most common source of flaky tests: timing issues from fixed delays or insufficient waits.

## Developer Tools

### Trace Viewer

The Trace Viewer is a powerful post-mortem debugging tool. It records every action, network request, console log, and screenshot during a test run.

```bash
# Record traces
npx playwright test --trace on

# Open trace for a specific test
npx playwright show-trace test-results/trace.zip
```

### UI Mode

UI Mode provides a visual, interactive test runner with live browser preview:

```bash
npx playwright test --ui
```

### Code Generator (codegen)

Automatically generate test code by recording your browser interactions:

```bash
npx playwright codegen https://example.com
npx playwright codegen --device="iPhone 13" https://example.com
```

### Visual Comparisons

Playwright includes built-in screenshot comparison:

```typescript
await expect(page).toHaveScreenshot('dashboard.png');
await expect(page.getByTestId('chart')).toHaveScreenshot('chart.png', {
  maxDiffPixels: 100,
});
```

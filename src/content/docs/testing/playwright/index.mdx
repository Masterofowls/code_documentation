Playwright is a modern end-to-end testing framework developed by Microsoft. It enables fast, reliable, cross-browser automation for Chromium (Chrome/Edge), Firefox, and WebKit (Safari) from a single API.

## What is Playwright?

Playwright was built by the same team that created Puppeteer. It solves the key pain points of browser automation at scale:

- **Auto-waiting**: Playwright automatically waits for elements to be actionable before interacting — eliminating most flaky `sleep()` calls
- **Cross-browser**: A single test suite runs on Chromium, Firefox, and WebKit
- **Isolated contexts**: Each test gets a fresh browser context (like an incognito window) with no shared cookies, localStorage, or cache
- **Powerful APIs**: Network interception, file uploads, clipboard, geolocation, permissions, mobile emulation, and more

```typescript
import { test, expect } from '@playwright/test';

test('user can log in and see the dashboard', async ({ page }) => {
  await page.goto('https://app.example.com/login');

  await page.getByLabel('Email').fill('alice@example.com');
  await page.getByLabel('Password').fill('secret123');
  await page.getByRole('button', { name: 'Sign in' }).click();

  await expect(page).toHaveURL(/\/dashboard/);
  await expect(page.getByRole('heading', { name: 'Welcome, Alice' })).toBeVisible();
});
```

## Playwright vs Cypress vs Selenium

| Feature | **Playwright** | **Cypress** | **Selenium** |
|---------|----------------|-------------|--------------|
| Browsers | Chromium, Firefox, WebKit | Chromium, Firefox, Edge | All browsers via WebDriver |
| Language | JS, TS, Python, Java, .NET | JS / TS only | JS, Python, Java, C#, Ruby |
| Architecture | Out-of-process (CDP + WebSocket) | In-process (same JS context) | WebDriver protocol |
| Auto-waiting | Yes (built-in, smart) | Yes (limited assertions) | No (manual waits required) |
| Parallel tests | Workers (OS processes) | Requires paid plan or plugin | Grid required |
| Test isolation | Excellent (browser contexts) | Tab-level only | Session-level |
| Network mocking | Yes (request/response) | Yes (cy.intercept) | Limited (extension-based) |
| Component tests | Yes (experimental) | Yes (mature) | No |
| iframes | Yes (frameLocator) | Limited | Yes |
| Shadow DOM | Yes | Limited | Via JavaScriptExecutor |
| Mobile emulation | Yes | Limited | Limited |
| Debugging | Trace viewer, UI mode | Time-travel debugger | Limited |
| CI performance | Very fast | Good | Slow (driver overhead) |
| License | Apache 2.0 (open source) | MIT (core OSS) | Apache 2.0 |
| Pricing | Free | Free (runner) + paid dashboard | Free |

:::tip[When to Choose Playwright]
Choose Playwright when you need true cross-browser coverage including Safari (WebKit), test isolation at scale, native parallelism without a paid plan, or Python/Java/.NET test suites alongside your JavaScript tests.
:::

:::caution[Playwright and Component Testing]
Playwright's component testing (`@playwright/experimental-ct-*`) is still experimental. For React/Vue/Angular component tests, consider `@testing-library/react` with Jest or Vitest for unit-level component tests, and use Playwright for full integration/E2E flows.
:::

## Playwright Architecture

### Browser, Context, and Page

Playwright's three-level hierarchy isolates tests at the right granularity:

```
Browser (one per worker)
└── BrowserContext (one per test — isolated incognito)
    └── Page (one or more tabs)
        └── Frame (iframes, main frame)
```

```typescript
// Playwright creates and manages this hierarchy automatically
test('example', async ({ browser, context, page }) => {
  // browser — the Chromium/Firefox/WebKit process
  // context — fresh isolated session (cookies, storage, permissions)
  // page    — a single tab within the context
  await page.goto('/');
});
```

### Out-of-Process Communication

Playwright communicates with browsers via the **Chrome DevTools Protocol (CDP)** for Chromium and custom protocols for Firefox/WebKit. Unlike Cypress (which runs inside the browser), Playwright runs in a separate Node.js process — giving it access to the file system, full Node.js APIs, and no origin restrictions.

## Auto-Waiting Philosophy

Playwright's most important feature is automatic waiting. Before performing any action, Playwright checks that the target element:

1. **Exists** in the DOM
2. **Is visible** (not hidden, not covered, not 0 opacity)
3. **Is stable** (not animating)
4. **Is enabled** (not disabled)
5. **Is actionable** (can receive events)

```typescript
// No explicit waits needed — Playwright handles all of these automatically:
await page.getByRole('button', { name: 'Submit' }).click();
// ↑ Waits until the button exists, is visible, is stable, and is enabled

await expect(page.getByText('Success')).toBeVisible();
// ↑ Retries the assertion until it passes (up to the configured timeout)
```

## Key Developer Tools

### Trace Viewer

The trace viewer records and replays every test step — including DOM snapshots, network requests, console logs, and screenshots:

```bash
npx playwright test --trace on
npx playwright show-trace test-results/example/trace.zip
```

### Codegen

Automatically generates test code by recording your interactions in the browser:

```bash
npx playwright codegen https://app.example.com
# Opens a browser with a recording toolbar
# Click, type, and navigate — code appears in the panel on the right
```

### UI Mode

An interactive test runner with time-travel debugging:

```bash
npx playwright test --ui
```

## Visual Comparisons (Screenshot Testing)

```typescript
test('home page matches snapshot', async ({ page }) => {
  await page.goto('/');
  await expect(page).toHaveScreenshot('homepage.png', {
    maxDiffPixels: 100,          // allow up to 100 different pixels
    threshold: 0.1,              // 10% pixel difference tolerance
    mask: [page.locator('.ads')], // mask dynamic content
  });
});
```

## API Testing

Playwright can make HTTP requests directly — without a browser — using the `request` fixture:

```typescript
test('POST /api/users creates a new user', async ({ request }) => {
  const response = await request.post('/api/users', {
    data: { name: 'Alice', email: 'alice@example.com' },
  });

  expect(response.ok()).toBeTruthy();
  const body = await response.json();
  expect(body).toMatchObject({ name: 'Alice' });
});
```

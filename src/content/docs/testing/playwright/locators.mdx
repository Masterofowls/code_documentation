Playwright's locator API is the foundation of reliable tests. Locators are **lazy** (they don't immediately query the DOM) and **auto-waiting** (every action on a locator waits for the element to be ready). A locator describes how to find an element; the actual DOM query happens at the moment of interaction.

## Locator Philosophy

Playwright encourages **resilient locators** that reflect what users see and interact with, not implementation details like CSS classes or XPath positions.

```typescript
// ❌ Fragile — breaks when CSS is refactored
page.locator('.btn.btn-primary.submit-btn');

// ❌ Fragile — breaks when DOM structure changes
page.locator('div > form > div:nth-child(3) > button');

// ✅ Resilient — matches the visible accessible name
page.getByRole('button', { name: 'Submit order' });

// ✅ Resilient — matches form label
page.getByLabel('Email address');
```

Locators from semantic queries survive refactoring of CSS classes, DOM structure, and component internals.

## getByRole — Preferred Locator

`getByRole` queries elements by their ARIA role — the same way screen readers and accessibility tools navigate the page. This is the most accessible and most recommended query.

```typescript
// Buttons
page.getByRole('button', { name: 'Submit' });
page.getByRole('button', { name: /submit/i }); // regex — case-insensitive
page.getByRole('button', { name: 'Save', exact: true });

// Links
page.getByRole('link', { name: 'Home' });
page.getByRole('link', { name: /learn more/i });

// Headings
page.getByRole('heading', { name: 'Dashboard' });
page.getByRole('heading', { name: 'Sign up', level: 1 }); // h1 specifically

// Form elements
page.getByRole('textbox', { name: 'Email' });
page.getByRole('textbox', { name: 'Search' });
page.getByRole('checkbox', { name: 'Remember me' });
page.getByRole('radio', { name: 'Monthly' });
page.getByRole('combobox', { name: 'Country' }); // <select>
page.getByRole('spinbutton', { name: 'Quantity' }); // <input type="number">
page.getByRole('slider', { name: 'Volume' });

// Navigation / structure
page.getByRole('navigation', { name: 'Main navigation' });
page.getByRole('main');
page.getByRole('banner');
page.getByRole('contentinfo'); // <footer>

// Lists
page.getByRole('list', { name: 'Search results' });
page.getByRole('listitem');

// Dialogs
page.getByRole('dialog', { name: 'Confirm deletion' });
page.getByRole('alertdialog');

// Tables
page.getByRole('table', { name: 'Order history' });
page.getByRole('row', { name: /Order #1042/ });
page.getByRole('cell', { name: '$42.00' });
page.getByRole('columnheader', { name: 'Amount' });

// Images
page.getByRole('img', { name: 'Profile photo' });

// Tabs
page.getByRole('tab', { name: 'Settings' });
page.getByRole('tabpanel', { name: 'Settings' });
```

### Common ARIA Roles Reference

| Role          | HTML elements                                 | Notes                        |
| ------------- | --------------------------------------------- | ---------------------------- |
| `button`      | `<button>`, `[role="button"]`                 | Also `<input type="submit">` |
| `link`        | `<a href>`                                    | Must have `href` attribute   |
| `heading`     | `<h1>`–`<h6>`                                 | Use `level` to specify h1–h6 |
| `textbox`     | `<input type="text/email/...">`, `<textarea>` | Use for text input           |
| `checkbox`    | `<input type="checkbox">`                     |                              |
| `radio`       | `<input type="radio">`                        |                              |
| `combobox`    | `<select>`, custom dropdowns                  |                              |
| `listbox`     | Multi-select custom dropdowns                 |                              |
| `option`      | `<option>` inside select                      |                              |
| `img`         | `<img>` with `alt`                            | Must have non-empty `alt`    |
| `dialog`      | `<dialog>`, `[role="dialog"]`                 |                              |
| `alert`       | `[role="alert"]`                              | For error messages           |
| `status`      | `[role="status"]`                             | For success messages         |
| `progressbar` | `[role="progressbar"]`                        | Loading indicators           |
| `menu`        | `[role="menu"]`                               | Dropdown menus               |
| `menuitem`    | `[role="menuitem"]`                           | Menu items                   |
| `tab`         | `[role="tab"]`                                | Tab navigation               |
| `tabpanel`    | `[role="tabpanel"]`                           | Tab content                  |

## getByText

Finds elements by their visible text content:

```typescript
page.getByText('Welcome back!'); // exact string
page.getByText(/welcome/i); // regex (case-insensitive)
page.getByText('Price: $42', { exact: true }); // exact match (no substring)
page.getByText('$42', { exact: false }); // substring (default)
```

:::caution[getByText Matches All Elements]
`getByText` matches any element containing that text — including parent elements. If multiple elements match, Playwright may throw. Use `getByRole` when possible for more precise matching.
:::

## getByLabel

Finds form inputs by their associated `<label>` text (also matches `aria-label` and `aria-labelledby`):

```typescript
page.getByLabel('Email address');
page.getByLabel(/password/i);
page.getByLabel('Date of birth', { exact: false });
```

Works with:

- `<label for="id">` + `<input id="id">`
- `<label><input /></label>` (implicit label)
- `aria-label="Email address"`
- `aria-labelledby="heading-id"`

## getByPlaceholder

Finds inputs by their placeholder text:

```typescript
page.getByPlaceholder('Enter your email');
page.getByPlaceholder('Search...');
page.getByPlaceholder(/yyyy-mm-dd/i);
```

## getByAltText

Finds images by their alt text:

```typescript
page.getByAltText('Company logo');
page.getByAltText(/user avatar/i);
page.getByAltText('', { exact: true }); // decorative image with empty alt
```

## getByTitle

Finds elements by their `title` attribute:

```typescript
page.getByTitle('Close dialog');
page.getByTitle(/delete/i);
```

## getByTestId

Finds elements by their `data-testid` attribute (or your configured attribute). Use as a **last resort** when no semantic locator is possible:

```typescript
page.getByTestId('user-avatar');
page.getByTestId('price-display');
```

Configure the attribute name in `playwright.config.ts`:

```typescript
use: {
  testIdAttribute: 'data-pw',  // use data-pw instead of data-testid
}
```

```html
<!-- In your component -->
<div data-pw="user-avatar">...</div>
```

:::tip[testId Usage]
`data-testid` attributes are invisible to users and can be added to any element. They're especially useful for elements that lack an ARIA role — like custom UI components that don't expose semantic roles.
:::

## locator() — CSS and XPath (Last Resort)

Use `page.locator()` with CSS selectors or XPath as a last resort when no role-based or semantic locator is available:

```typescript
// CSS selectors
page.locator('.user-card');
page.locator('#main-content');
page.locator('[data-status="active"]');
page.locator('input[type="file"]');

// XPath
page.locator('xpath=//button[contains(@class, "primary")]');
page.locator('//table//tr[3]//td[2]');

// Text selector (pseudo-class)
page.locator('text=Submit');

// CSS + text combination
page.locator('button:has-text("Submit")');
page.locator('li:has-text("Option A")');
```

## Chaining Locators

Locators can be chained to narrow the search scope:

```typescript
// Find a button inside a specific card
const card = page.getByRole('article', { name: 'Premium Plan' });
await card.getByRole('button', { name: 'Select plan' }).click();

// Find a form field inside a dialog
const dialog = page.getByRole('dialog', { name: 'Edit profile' });
await dialog.getByLabel('Display name').fill('Alice');
await dialog.getByRole('button', { name: 'Save changes' }).click();

// Deeply nested
const sidebar = page.locator('[data-testid="sidebar"]');
const nav = sidebar.getByRole('navigation');
await nav.getByRole('link', { name: 'Settings' }).click();
```

## Filtering Locators

Use `.filter()` to narrow a broad locator to a specific element:

```typescript
// Get all list items, then filter to those containing "Alice"
const aliceItem = page.getByRole('listitem').filter({ hasText: 'Alice' });

// Filter by a child locator
const activeRow = page.getByRole('row').filter({
  has: page.getByRole('cell', { name: 'Active' }),
});

// Combine filters
const adminCard = page
  .getByRole('article')
  .filter({ hasText: 'Alice' })
  .filter({ has: page.getByRole('img', { name: 'Admin badge' }) });
```

## nth(), first(), last()

When a locator matches multiple elements, use index selectors:

```typescript
const rows = page.getByRole('row');

await rows.first().click(); // first matching element
await rows.last().click(); // last matching element
await rows.nth(2).click(); // 0-indexed: third row
await rows.nth(-1).click(); // last row (negative index)

// Count matching elements
const count = await rows.count();
expect(count).toBe(10);
```

:::caution[nth() and Test Stability]
`nth(3)` is fragile if the number of elements changes. Prefer `.filter({ hasText: 'specific content' })` to identify the specific element you mean, rather than relying on position.
:::

## Frames — frameLocator

To interact with elements inside an `<iframe>`, use `frameLocator`:

```typescript
// Target content inside an iframe
const paymentFrame = page.frameLocator('iframe[title="Payment form"]');

await paymentFrame.getByLabel('Card number').fill('4242424242424242');
await paymentFrame.getByLabel('Expiry date').fill('12/26');
await paymentFrame.getByLabel('CVV').fill('123');
await paymentFrame.getByRole('button', { name: 'Pay now' }).click();

// Nested iframes
const nestedFrame = page
  .frameLocator('#outer-frame')
  .frameLocator('#inner-frame');

await nestedFrame.getByRole('button', { name: 'Confirm' }).click();
```

## locator.all() — Lists

`locator.all()` resolves a locator to an array of all matching elements at the current moment. Use for iterating over lists:

```typescript
// Get all product cards and check their prices
const productCards = page.getByRole('article', { name: /product/i });
const cards = await productCards.all();

expect(cards.length).toBeGreaterThan(0);

for (const card of cards) {
  await expect(card.getByRole('heading')).toBeVisible();
  await expect(card.getByText(/\$\d+\.\d{2}/)).toBeVisible();
}

// Extract all text content
const links = await page.getByRole('link').all();
const texts = await Promise.all(links.map((link) => link.textContent()));
console.log(texts.filter(Boolean)); // ['Home', 'About', 'Contact']
```

:::caution[locator.all() is a Snapshot]
`locator.all()` resolves the locator **once** — it doesn't auto-wait. If the list is populated asynchronously, wait for at least one item to be visible first:

```typescript
await expect(page.getByRole('listitem').first()).toBeVisible();
const items = await page.getByRole('listitem').all();
```

:::

## Inspecting the Accessibility Tree

Use Playwright's `accessibility.snapshot()` to understand what ARIA roles and names are available on a page:

```bash
# Open the page inspector to explore accessibility tree
npx playwright open https://example.com
```

In a test:

```typescript
test('accessibility snapshot', async ({ page }) => {
  await page.goto('/dashboard');

  const snapshot = await page.accessibility.snapshot();
  console.log(JSON.stringify(snapshot, null, 2));
  // Shows the full ARIA tree — use to find the correct role/name for locators
});
```

You can also use the Playwright Inspector (launched with `--debug`):

```bash
npx playwright test --debug
```

The inspector shows a live highlight of locators as you type them, making it easy to verify your locator before committing to it.

Locators are Playwright's way of finding elements on the page. Unlike Selenium's `findElement`, Playwright locators are **lazy** (they don't search the DOM until an action is performed) and have **auto-waiting** built in — they retry automatically until the element exists, is visible, is stable, and is actionable.

## The Locator Philosophy

A good locator should:
1. **Survive refactors** — changing CSS classes or internal structure shouldn't break tests
2. **Reflect user experience** — if a user can't find the button, the test shouldn't either
3. **Be accessible** — tests that use ARIA roles validate accessibility as a side effect

Priority order (most preferred to least):

| Priority | Locator | Resilience |
|---|---|---|
| 1 | `getByRole` | ⭐⭐⭐⭐⭐ — ARIA roles are semantic |
| 2 | `getByLabel` | ⭐⭐⭐⭐⭐ — label association is semantic |
| 3 | `getByPlaceholder` | ⭐⭐⭐⭐ — placeholder text is visible |
| 4 | `getByText` | ⭐⭐⭐⭐ — visible text is user-facing |
| 5 | `getByAltText` | ⭐⭐⭐⭐ — alt text is semantic |
| 6 | `getByTitle` | ⭐⭐⭐ — title attribute may change |
| 7 | `getByTestId` | ⭐⭐ — implementation detail, invisible to users |
| 8 | `locator('css')` | ⭐ — last resort, breaks on refactoring |
| 9 | `locator('xpath')` | ⭐ — last resort, very brittle |

## getByRole — ARIA Roles

`getByRole` is the most powerful and preferred locator. It mirrors how screen readers navigate the page:

```typescript
// Buttons
page.getByRole('button', { name: 'Submit' })
page.getByRole('button', { name: /save/i })         // case-insensitive regex
page.getByRole('button', { name: 'Delete', exact: true }) // exact match only

// Links
page.getByRole('link', { name: 'Home' })
page.getByRole('link', { name: /dashboard/i })

// Headings
page.getByRole('heading', { name: 'User Profile' })
page.getByRole('heading', { level: 1 })             // h1 only
page.getByRole('heading', { name: 'Settings', level: 2 }) // h2 named "Settings"

// Form inputs
page.getByRole('textbox', { name: 'Email' })        // <input type="text"> or type="email"
page.getByRole('textbox', { name: /password/i })    // password field
page.getByRole('checkbox', { name: 'Remember me' })
page.getByRole('radio', { name: 'Monthly billing' })
page.getByRole('combobox', { name: 'Country' })     // <select>
page.getByRole('spinbutton', { name: 'Quantity' })  // <input type="number">
page.getByRole('searchbox')                         // <input type="search">

// Status / feedback
page.getByRole('alert')                             // error messages, warnings
page.getByRole('status')                            // success messages
page.getByRole('progressbar')                       // loading indicators
page.getByRole('dialog')                            // modal dialogs
page.getByRole('dialog', { name: 'Confirm Delete' })

// Navigation / landmarks
page.getByRole('navigation')
page.getByRole('navigation', { name: 'Main navigation' })
page.getByRole('main')
page.getByRole('banner')                            // <header>
page.getByRole('contentinfo')                       // <footer>

// Tables
page.getByRole('table')
page.getByRole('row')
page.getByRole('columnheader', { name: 'Price' })
page.getByRole('cell', { name: '$9.99' })

// Lists
page.getByRole('list')
page.getByRole('listitem')

// Tabs and tab panels
page.getByRole('tab', { name: 'Details' })
page.getByRole('tabpanel', { name: 'Details' })

// Menus
page.getByRole('menu')
page.getByRole('menuitem', { name: 'Edit' })
page.getByRole('menuitemcheckbox', { name: 'Show sidebar' })
```

### Filtering by State

```typescript
// Find checked/unchecked checkboxes
page.getByRole('checkbox', { name: 'Accept terms', checked: true })
page.getByRole('checkbox', { checked: false }) // any unchecked checkbox

// Find disabled/enabled elements
page.getByRole('button', { name: 'Submit', disabled: true })
page.getByRole('textbox', { disabled: false })

// Find selected option in listbox
page.getByRole('option', { name: 'Canada', selected: true })

// Find expanded/collapsed elements
page.getByRole('button', { name: 'Menu', expanded: true })
```

:::tip[Finding the Right Role]
Run `npx playwright codegen` and click through your app — Playwright codegen automatically generates `getByRole` locators. You can also use the Playwright Inspector (`--debug` flag) to inspect the accessibility tree.
:::

## getByText

`getByText` finds elements by their visible text content:

```typescript
// Exact string match (default)
page.getByText('Welcome back!')

// Partial match
page.getByText('Welcome')  // matches "Welcome back!" too

// Exact match (override partial behavior)
page.getByText('Welcome', { exact: true })  // only "Welcome" exactly

// Case-insensitive regex
page.getByText(/welcome back/i)

// Regex with wildcards
page.getByText(/order #\d+/)
```

:::caution[getByText Matches All Elements]
`getByText` searches all elements including `div`, `span`, `p`, etc. If multiple elements contain the same text, the locator will match multiple elements. Use `getByRole` with a name when possible for more specific targeting.
:::

## getByLabel

Finds form elements by their associated `<label>` text. Works with `for` attributes, `aria-label`, and wrapped labels:

```typescript
// Associated via for/id
page.getByLabel('Email address')
page.getByLabel('Email address', { exact: false }) // partial match

// Works with aria-label too
page.getByLabel('Search the site')

// Case-insensitive
page.getByLabel(/email/i)
```

```html
<!-- All of these are found by getByLabel('Email') -->
<label for="email">Email</label><input id="email" type="email">

<label>Email <input type="email"></label>

<input type="email" aria-label="Email">

<input type="email" aria-labelledby="email-label">
<span id="email-label">Email</span>
```

## getByPlaceholder

```typescript
page.getByPlaceholder('Search...')
page.getByPlaceholder(/enter your email/i)
page.getByPlaceholder('mm/dd/yyyy', { exact: true })
```

## getByAltText

Find images and inputs by their `alt` attribute:

```typescript
page.getByAltText('Company logo')
page.getByAltText(/product image/i)

// Also works for <input type="image" alt="...">
page.getByAltText('Submit button')
```

## getByTitle

Find elements by their `title` attribute (used for tooltips):

```typescript
page.getByTitle('Close dialog')
page.getByTitle(/help/i)
```

## getByTestId

Use only when no semantic locator is available:

```typescript
// Requires data-testid attribute on the element
page.getByTestId('user-avatar')
page.getByTestId('nav-menu-toggle')
```

Configure a custom attribute name:

```typescript
// playwright.config.ts
use: {
  testIdAttribute: 'data-cy',  // use data-cy instead of data-testid
}
```

:::tip[Strategic Use of data-testid]
Reserve `data-testid` for elements that are genuinely impossible to identify with accessible queries — custom SVG icon buttons, canvas elements, or third-party component internals. Never add `data-testid` as a first resort.
:::

## CSS and XPath (Last Resort)

```typescript
// CSS selector
page.locator('div.user-card > h2.username')
page.locator('#submit-btn')
page.locator('[data-state="open"]')

// XPath
page.locator('xpath=//button[@type="submit"]')
page.locator('//div[@class="container"]//span')

// CSS with :has, :not, :nth-child
page.locator('ul > li:nth-child(2)')
page.locator('form:has(input[name="email"])')
```

:::caution[CSS Selectors Break on Refactoring]
CSS selectors that reference class names (`div.btn-primary`) or DOM structure (`form > div > input`) will break whenever the implementation changes. They also provide no accessibility validation. Use them only when you have no alternative.
:::

## Chaining Locators

Scope locators to a parent element:

```typescript
// Find the "Edit" button within the second user card
const userCards = page.getByRole('article');
const secondCard = userCards.nth(1);
await secondCard.getByRole('button', { name: 'Edit' }).click();

// Find elements within a specific section
const sidebar = page.getByRole('complementary');
await sidebar.getByRole('link', { name: 'Settings' }).click();

// Chain multiple locators
const dialog = page.getByRole('dialog', { name: 'Edit User' });
await dialog.getByLabel('Full Name').fill('Alice Johnson');
await dialog.getByRole('button', { name: 'Save' }).click();
```

## Filtering Locators

```typescript
// Filter by text content
page.getByRole('listitem').filter({ hasText: 'Alice' })
page.getByRole('listitem').filter({ hasText: /alice/i })

// Filter by contained element
page.getByRole('listitem').filter({
  has: page.getByRole('img', { name: 'avatar' }),
})

// Filter by NOT containing text
page.getByRole('listitem').filter({ hasNotText: 'Inactive' })

// Combine filters
page.getByRole('row')
  .filter({ has: page.getByRole('cell', { name: 'Alice' }) })
  .filter({ hasNot: page.getByText('Deactivated') })
```

## nth(), first(), last()

```typescript
// Get the first matching element
page.getByRole('listitem').first()

// Get the last matching element
page.getByRole('listitem').last()

// Get by zero-based index
page.getByRole('listitem').nth(0)  // same as first()
page.getByRole('listitem').nth(2)  // third item
page.getByRole('listitem').nth(-1) // same as last()
```

## locator.all() — Working with Lists

`locator.all()` returns an array of all matching locators (bypasses auto-waiting — all elements must already exist):

```typescript
// Iterate over all matching elements
const items = await page.getByRole('listitem').all();
for (const item of items) {
  console.log(await item.textContent());
}

// Collect all text values
const names = await page.getByRole('cell', { name: /^[A-Z]/ }).allTextContents();
expect(names).toEqual(['Alice', 'Bob', 'Carol']);

// Collect all inner texts
const prices = await page.getByTestId('price').allInnerTexts();

// Count elements
const count = await page.getByRole('listitem').count();
expect(count).toBeGreaterThan(0);
```

## frameLocator — Working with iFrames

```typescript
// Locate the frame first, then query within it
const frame = page.frameLocator('iframe[name="payment-frame"]');
await frame.getByLabel('Card Number').fill('4111111111111111');
await frame.getByLabel('Expiry').fill('12/26');
await frame.getByLabel('CVV').fill('123');

// Frame by URL pattern
const adFrame = page.frameLocator('iframe[src*="ads.example.com"]');

// Nested frames
const innerFrame = page
  .frameLocator('#outer-frame')
  .frameLocator('#inner-frame');
```

## Reusing Locators

Store locators in variables to avoid repeating selectors:

```typescript
test('checks out successfully', async ({ page }) => {
  await page.goto('/cart');

  // Store frequently used locators
  const cartItems = page.getByRole('row').filter({ hasNot: page.getByRole('columnheader') });
  const total = page.getByTestId('order-total');
  const checkoutBtn = page.getByRole('button', { name: 'Proceed to checkout' });

  // Use them multiple times
  await expect(cartItems).toHaveCount(2);
  await expect(total).toHaveText('$49.98');

  await checkoutBtn.click();
  await expect(page).toHaveURL('/checkout');
});
```

:::tip[Locators Are Lazy]
Locator definitions don't query the DOM. The DOM is only searched when you perform an action or assertion. This means you can define locators before navigating to the page.
:::

## Accessibility Tree Inspection

Use the Playwright Inspector to explore the accessibility tree:

```bash
# Launch in debug mode
npx playwright test --debug
```

In the Inspector, click "Explore" and hover over elements to see:
- Their computed ARIA role
- Their accessible name
- The recommended `getByRole` locator

You can also use the `accessibility` API in tests:

```typescript
// Dump the full accessibility tree
const snapshot = await page.accessibility.snapshot();
console.log(JSON.stringify(snapshot, null, 2));

// Find a node in the tree
const loginHeading = snapshot?.children?.find(
  (node) => node.role === 'heading' && node.name === 'Log In',
);
expect(loginHeading).toBeDefined();
```

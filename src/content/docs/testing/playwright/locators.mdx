Locators are Playwright's mechanism for finding page elements. Unlike raw CSS selectors or XPath, Playwright locators are **auto-waiting** (they retry until the element is found or the timeout expires), **strict** (fail if more than one element matches), and **resilient** to page structure changes.

## Locator Philosophy

The central principle of Playwright's locator API is: **prefer queries that match how users perceive the page**.

A user doesn't know that the "Submit" button has a class of `.btn-primary.mt-4`. They know it's a button labelled "Submit". Tests that use the accessible label are far more resilient to styling and layout changes.

**Priority order** (most resilient to least):

1. `getByRole` — ARIA role + accessible name
2. `getByLabel` — form label association
3. `getByPlaceholder` — input placeholder text
4. `getByText` — visible text content
5. `getByAltText` — image alt text
6. `getByTitle` — title attribute
7. `getByTestId` — `data-testid` attribute
8. `locator('css selector')` — only when nothing else works

## getByRole

`getByRole` is the most resilient and preferred locator. It matches elements by their ARIA role and accessible name — the same way screen readers navigate the page.

```typescript
// Buttons
page.getByRole('button', { name: 'Submit' });
page.getByRole('button', { name: /submit/i });    // case-insensitive regex
page.getByRole('button', { name: 'Submit', exact: true });

// Links
page.getByRole('link', { name: 'Learn more' });
page.getByRole('link', { name: /privacy/i });

// Headings — use 'level' to specify h1-h6
page.getByRole('heading', { name: 'Dashboard' });
page.getByRole('heading', { name: 'Settings', level: 2 });

// Form inputs
page.getByRole('textbox', { name: 'Email address' });
page.getByRole('textbox');                    // any text input
page.getByRole('searchbox');                  // input[type=search]
page.getByRole('spinbutton', { name: 'Age' }); // number input
page.getByRole('slider', { name: 'Volume' });

// Selects / comboboxes
page.getByRole('combobox', { name: 'Country' });

// Checkboxes and radios
page.getByRole('checkbox', { name: 'Remember me' });
page.getByRole('radio', { name: 'Credit card' });

// Switches (ARIA role="switch")
page.getByRole('switch', { name: 'Dark mode' });

// Lists
page.getByRole('list');
page.getByRole('listitem').first();

// Table
page.getByRole('table');
page.getByRole('row').nth(2);           // third row (0-indexed)
page.getByRole('cell', { name: 'Alice' });
page.getByRole('columnheader', { name: 'Name' });

// Navigation / landmarks
page.getByRole('navigation', { name: 'Main navigation' });
page.getByRole('main');
page.getByRole('complementary');        // aside element
page.getByRole('dialog', { name: 'Confirm delete' });
page.getByRole('alert');                // live region
page.getByRole('status');               // polite live region

// Tabs
page.getByRole('tab', { name: 'Overview' });
page.getByRole('tabpanel');

// Menus
page.getByRole('menu');
page.getByRole('menuitem', { name: 'Settings' });
```

### Common ARIA Roles Reference

| HTML Element | Default ARIA Role |
|---|---|
| `<button>` | `button` |
| `<a href>` | `link` |
| `<input type="text">` | `textbox` |
| `<input type="checkbox">` | `checkbox` |
| `<input type="radio">` | `radio` |
| `<input type="number">` | `spinbutton` |
| `<input type="search">` | `searchbox` |
| `<select>` | `combobox` |
| `<h1>-<h6>` | `heading` |
| `<ul>, <ol>` | `list` |
| `<li>` | `listitem` |
| `<table>` | `table` |
| `<tr>` | `row` |
| `<td>` | `cell` |
| `<th>` | `columnheader` or `rowheader` |
| `<nav>` | `navigation` |
| `<main>` | `main` |
| `<aside>` | `complementary` |
| `<img alt="text">` | `img` |
| `<dialog>` | `dialog` |

:::tip[Use Playwright's accessibility inspector]
Run `npx playwright codegen --target=playwright-test your-url` to interactively generate locators. The generated code uses `getByRole` wherever possible.
:::

## getByText

Find elements by their visible text content:

```typescript
// Exact string match (default)
page.getByText('Hello World');

// Substring match (exact: false)
page.getByText('Hello', { exact: false });

// Regex match
page.getByText(/hello world/i);
page.getByText(/^\d{4}-\d{2}-\d{2}$/); // date pattern

// Works on any element with text content
await expect(page.getByText('Successfully submitted')).toBeVisible();
```

:::caution[getByText Matches Many Elements]
`getByText` can match `<p>`, `<span>`, `<div>`, `<label>`, and other elements. If multiple elements contain the same text, use `getByRole` or narrow it with `filter()`.
:::

## getByLabel

Find form inputs associated with a `<label>` element:

```typescript
// Matches: <label>Email</label><input ...>
page.getByLabel('Email');
page.getByLabel('Email address');
page.getByLabel(/email/i);

// Also matches aria-label and aria-labelledby
// <input aria-label="Search the site">
page.getByLabel('Search the site');

// Restrict to a specific element type
page.getByLabel('Country', { selector: 'select' });
page.getByLabel('Bio', { selector: 'textarea' });
```

## getByPlaceholder, getByAltText, getByTitle

```typescript
// getByPlaceholder — input placeholder text
page.getByPlaceholder('Enter your email address');
page.getByPlaceholder(/email/i);

// getByAltText — image alt text
page.getByAltText('Company logo');
page.getByAltText(/avatar/i);

// getByTitle — title attribute tooltip text
page.getByTitle('Close dialog');
page.getByTitle(/settings/i);
```

## getByTestId

Use `data-testid` as a last resort when no accessible query works:

```typescript
// <div data-testid="loading-spinner">
page.getByTestId('loading-spinner');

// Custom testid attribute (configure in playwright.config.ts)
// use: { testIdAttribute: 'data-cy' }
page.getByTestId('submit-form'); // matches data-cy="submit-form"
```

Configure the test ID attribute:

```typescript
// playwright.config.ts
use: {
  testIdAttribute: 'data-testid', // default
  // or use a custom attribute:
  // testIdAttribute: 'data-cy',
}
```

:::tip[Avoid data-testid Overuse]
Every `data-testid` you add to production markup is maintenance debt. Before adding one, ask: "Could a user find this element by role, label, or text?" If yes, use the accessible query instead.
:::

## locator() with CSS and XPath

Use CSS selectors or XPath only when no semantic locator is available:

```typescript
// CSS selector
page.locator('.submit-button');
page.locator('#user-form input[type="email"]');
page.locator('[data-component="DatePicker"]');

// XPath — use sparingly, highly brittle
page.locator('//button[contains(@class, "primary")]');
page.locator('xpath=//div[@role="dialog"]//button[1]');

// :has-text pseudo-class — CSS with text filtering
page.locator('div.card:has-text("Alice")');

// :has pseudo-class — element containing another
page.locator('li:has(a[href="/dashboard"])');
```

:::caution[CSS and XPath are Brittle]
CSS selectors and XPath break silently when developers rename classes, change element nesting, or refactor markup. Prefer role-based and semantic locators — they test behavior, not implementation.
:::

## Chaining Locators

Narrow down locators by chaining `.locator()` calls:

```typescript
// Find a button inside a specific section
const sidebar = page.locator('aside[aria-label="Sidebar"]');
await sidebar.getByRole('link', { name: 'Settings' }).click();

// Find a form within a modal
const modal = page.getByRole('dialog', { name: 'Edit Profile' });
await modal.getByLabel('Display Name').fill('Alice Johnson');
await modal.getByRole('button', { name: 'Save' }).click();

// Find table rows within a specific table
const dataTable = page.getByRole('table', { name: 'Users' });
const rows = dataTable.getByRole('row');
await expect(rows).toHaveCount(11); // 1 header + 10 data rows
```

## Filtering Locators

The `.filter()` method narrows an existing locator:

```typescript
// Filter by text
const listItems = page.getByRole('listitem');
const aliceItem = listItems.filter({ hasText: 'Alice' });

// Filter by nested locator
const userCards = page.locator('.user-card');
const adminCards = userCards.filter({
  has: page.getByRole('badge', { name: 'Admin' }),
});

// Combine filters
const activeAdmins = page.locator('.user-card')
  .filter({ hasText: 'Active' })
  .filter({ has: page.getByRole('badge', { name: 'Admin' }) });

await expect(activeAdmins).toHaveCount(3);
```

## nth(), first(), and last()

When a locator matches multiple elements, select a specific one:

```typescript
const buttons = page.getByRole('button');

// By index (0-based)
await buttons.nth(0).click();   // first button
await buttons.nth(2).click();   // third button

// Convenience methods
await buttons.first().click();
await buttons.last().click();

// Chained
await page.getByRole('listitem').last().getByRole('link').click();
```

:::caution[Avoid Position-Based Selection]
`nth()`, `first()`, and `last()` are fragile when new elements are added or the order changes. Prefer `filter({ hasText: '...' })` or `filter({ has: ... })` to select specific elements by identity, not position.
:::

## locator.all() — Iterating Over Lists

```typescript
// Get all matching elements as an array
const items = await page.getByRole('listitem').all();

for (const item of items) {
  const text = await item.textContent();
  console.log(text);
}

// Assert properties of each item
const checkboxes = await page.getByRole('checkbox').all();
for (const checkbox of checkboxes) {
  await expect(checkbox).not.toBeChecked();
}

// Count
const count = await page.getByRole('listitem').count();
expect(count).toBe(5);
```

## frameLocator — Working with iframes

Content inside `<iframe>` elements is accessed via `frameLocator`:

```typescript
// Target content inside an iframe
const iframe = page.frameLocator('iframe[name="payment-widget"]');

await iframe.getByLabel('Card Number').fill('4111111111111111');
await iframe.getByLabel('Expiry').fill('12/26');
await iframe.getByLabel('CVC').fill('123');

// Nested iframes
const outerFrame = page.frameLocator('#outer-frame');
const innerFrame = outerFrame.frameLocator('#inner-frame');
await innerFrame.getByRole('button', { name: 'Submit' }).click();
```

## Inspecting the Accessibility Tree

Use the accessibility snapshot to understand what Playwright sees:

```typescript
// Print the accessibility tree of the page
const snapshot = await page.accessibility.snapshot();
console.log(JSON.stringify(snapshot, null, 2));

// Check a specific element's accessible name
const button = page.getByRole('button').first();
const name = await button.getAttribute('aria-label')
  ?? await button.textContent();
console.log('Button accessible name:', name);
```

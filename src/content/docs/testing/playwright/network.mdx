Playwright provides comprehensive network control — mock API responses, intercept and modify requests, simulate network conditions, and test API endpoints directly without a browser. This enables fully isolated E2E tests that don't depend on backend availability.

## page.route() — Request Interception

`page.route(pattern, handler)` intercepts requests matching the pattern:

```typescript
// Intercept all API calls
await page.route('/api/**', async (route) => {
  // Decide what to do with the request
});

// Intercept by URL regex
await page.route(/\/api\/users/, async (route) => { /* ... */ });

// Intercept specific resource types
await page.route('**/*', async (route) => {
  if (route.request().resourceType() === 'image') {
    await route.abort();
  } else {
    await route.continue();
  }
});
```

## route.fulfill() — Mock Responses

Return a fake response without making a real network request:

```typescript
test('displays users from API', async ({ page }) => {
  // Mock the users endpoint
  await page.route('/api/users', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify([
        { id: 1, name: 'Alice', email: 'alice@example.com', role: 'admin' },
        { id: 2, name: 'Bob', email: 'bob@example.com', role: 'user' },
      ]),
    });
  });

  await page.goto('/users');

  await expect(page.getByRole('row', { name: /alice/i })).toBeVisible();
  await expect(page.getByRole('row', { name: /bob/i })).toBeVisible();
});
```

### fulfill() Options

| Option | Type | Description |
|---|---|---|
| `status` | `number` | HTTP status code (default: 200) |
| `headers` | `Record<string, string>` | Response headers |
| `contentType` | `string` | Shorthand for `Content-Type` header |
| `body` | `string \| Buffer` | Response body |
| `json` | `any` | JSON body (sets Content-Type automatically) |
| `path` | `string` | File path to read body from |

```typescript
// Using the json shorthand (Playwright 1.23+)
await page.route('/api/products', async (route) => {
  await route.fulfill({
    json: {
      products: [
        { id: 'p1', name: 'Widget', price: 9.99 },
        { id: 'p2', name: 'Gadget', price: 24.99 },
      ],
      total: 2,
    },
  });
});

// Serve a fixture file
await page.route('/api/large-dataset', async (route) => {
  await route.fulfill({
    status: 200,
    contentType: 'application/json',
    path: 'tests/fixtures/large-dataset.json',
  });
});

// Simulate error responses
await page.route('/api/users/999', async (route) => {
  await route.fulfill({
    status: 404,
    json: { error: 'User not found', code: 'USER_NOT_FOUND' },
  });
});

// Simulate server errors
await page.route('/api/orders', async (route) => {
  await route.fulfill({
    status: 500,
    json: { error: 'Internal server error' },
  });
});
```

## route.abort() — Block Requests

Prevent requests from being made at all:

```typescript
// Block all analytics and tracking requests
await page.route(/google-analytics|googletagmanager|hotjar|segment/, async (route) => {
  await route.abort();
});

// Block image downloads to speed up tests
await page.route(/\.(jpg|jpeg|png|gif|webp|svg)$/, async (route) => {
  await route.abort();
});

// Block a specific failing service
await page.route('/api/recommendations', async (route) => {
  await route.abort('failed'); // simulates a network failure
});
```

### Abort reasons

| Reason | Description |
|---|---|
| `'aborted'` | Operation was aborted |
| `'accessdenied'` | Permission to access a resource was denied |
| `'addressunreachable'` | IP address is unreachable |
| `'blockedbyclient'` | Connection was blocked by the client |
| `'connectionaborted'` | Connection was aborted |
| `'connectionclosed'` | Connection was closed |
| `'connectionfailed'` | Connection attempt failed |
| `'connectionrefused'` | Connection was refused |
| `'connectionreset'` | Connection was reset |
| `'failed'` | Generic failure |
| `'timedout'` | Request timed out |

## route.continue() — Modify and Forward Requests

Pass the request through with optional modifications:

```typescript
// Add auth header to all API requests
await page.route('/api/**', async (route) => {
  await route.continue({
    headers: {
      ...route.request().headers(),
      'Authorization': `Bearer ${testToken}`,
      'X-Test-Mode': 'true',
    },
  });
});

// Modify request body
await page.route('/api/orders', async (route) => {
  if (route.request().method() === 'POST') {
    const body = route.request().postDataJSON();
    await route.continue({
      postData: JSON.stringify({ ...body, source: 'test-suite' }),
    });
  } else {
    await route.continue();
  }
});

// Redirect to a different URL
await page.route('/api/legacy/*', async (route) => {
  const newUrl = route.request().url().replace('/api/legacy/', '/api/v2/');
  await route.continue({ url: newUrl });
});
```

## Waiting for Requests and Responses

```typescript
// Wait for a specific request to be made
const searchRequest = page.waitForRequest('/api/search*');
await page.getByRole('searchbox').fill('playwright');
await page.keyboard.press('Enter');
const request = await searchRequest;
expect(request.url()).toContain('q=playwright');

// Wait for a specific response
const usersResponse = page.waitForResponse('/api/users');
await page.reload();
const response = await usersResponse;
expect(response.status()).toBe(200);

// Wait for response with condition
const submitResponse = page.waitForResponse(
  (response) => response.url().includes('/api/orders') && response.status() === 201
);
await page.getByRole('button', { name: 'Place Order' }).click();
const orderResponse = await submitResponse;
const order = await orderResponse.json();
expect(order.id).toBeDefined();
```

## API Testing with request Fixture

Playwright's `request` fixture lets you make HTTP requests directly, without a browser page:

```typescript
import { test, expect } from '@playwright/test';

test.describe('Users API', () => {
  test('GET /api/users returns a list of users', async ({ request }) => {
    const response = await request.get('/api/users');

    await expect(response).toBeOK();
    const users = await response.json();
    expect(users).toBeInstanceOf(Array);
    expect(users.length).toBeGreaterThan(0);
    expect(users[0]).toMatchObject({ id: expect.any(Number), name: expect.any(String) });
  });

  test('POST /api/users creates a new user', async ({ request }) => {
    const response = await request.post('/api/users', {
      data: { name: 'Test User', email: `test+${Date.now()}@example.com` },
    });

    expect(response.status()).toBe(201);
    const user = await response.json();
    expect(user.id).toBeDefined();
    expect(user.name).toBe('Test User');
  });

  test('PUT /api/users/:id updates a user', async ({ request }) => {
    const response = await request.put('/api/users/1', {
      data: { name: 'Updated Name' },
    });

    await expect(response).toBeOK();
    const user = await response.json();
    expect(user.name).toBe('Updated Name');
  });

  test('DELETE /api/users/:id returns 204', async ({ request }) => {
    const response = await request.delete('/api/users/999');
    expect(response.status()).toBe(204);
  });
});
```

### Authenticating API Requests

```typescript
// Create an authenticated API context
const apiContext = await request.newContext({
  baseURL: 'https://api.example.com',
  extraHTTPHeaders: {
    'Authorization': `Bearer ${process.env.API_TOKEN}`,

  },

const response = await apiContext.get('/protected/resource');
await expect(response).toBeOK();
```

## HAR Recording and Playback

HAR (HTTP Archive) recording captures all network activity and enables playback — perfect for testing against real API responses without hitting the server repeatedly:

```typescript
// Record HAR during a test run
test('record HAR for checkout flow', async ({ page }) => {
  // Start recording
  await page.routeFromHAR('./tests/fixtures/checkout.har', { update: true });
  await page.goto('/checkout');
  // All real network requests are made AND recorded to checkout.har
});

// Play back from HAR (no real network requests)
test('checkout flow from HAR', async ({ page }) => {
  await page.routeFromHAR('./tests/fixtures/checkout.har');
  await page.goto('/checkout');
  // All requests are served from the HAR file
});

// Fallback for unmatched requests
await page.routeFromHAR('./tests/fixtures/app.har', {
  fallback: 'continue', // make real request for unmatched URLs
  notFound: 'abort',    // abort requests not in HAR
});
```

## Offline Mode

```typescript
test('shows offline message when network is unavailable', async ({ page, context }) => {
  await page.goto('/dashboard');

  // Simulate going offline
  await context.setOffline(true);

  await page.getByRole('button', { name: 'Sync data' }).click();

  await expect(page.getByRole('alert')).toContainText('No internet connection');

  // Restore connection
  await context.setOffline(false);

  await page.getByRole('button', { name: 'Retry' }).click();
  await expect(page.getByRole('alert')).not.toBeVisible();
});
```

## Monitoring All Network Requests

```typescript
// Collect all requests during a test
test('tracks API calls during checkout', async ({ page }) => {
  const apiRequests: string[] = [];

  page.on('request', (request) => {
    if (request.url().includes('/api/')) {
      apiRequests.push(`${request.method()} ${request.url()}`);
    }
  });

  page.on('response', (response) => {
    if (!response.ok() && response.url().includes('/api/')) {
      console.warn(`Failed: ${response.status()} ${response.url()}`);
    }
  });

  await page.goto('/checkout');
  await completePurchase(page);

  expect(apiRequests).toContain('POST /api/orders');
  expect(apiRequests).toContain('POST /api/payments');
});

// Wait for all network requests to settle
await page.waitForLoadState('networkidle');
```

:::tip[Use Network Mocking for Speed and Reliability]
Mocking API responses with `route.fulfill()` makes tests 3–10x faster (no real HTTP calls), eliminates flakiness from backend instability, and lets you easily test error states and edge cases that are hard to reproduce against a real API.
:::

:::caution[Balance Mocking and Real API Tests]
Heavy mocking can give false confidence — your mocked API may not reflect real API behavior. Maintain a set of "contract tests" that run against the real API to verify your mocks remain accurate. Mock at the E2E layer for speed, but validate contracts separately.
:::

CSS animations are a powerful way to add life to your interfaces. Done right, they improve user experience. Done wrong, they cause jank, accessibility issues, and performance problems.

## CSS Transitions

Transitions animate a property from one value to another when a state change occurs (hover, focus, class change):

```css
.button {
  background: #0066cc;
  color: white;
  transform: translateY(0);
  box-shadow: 0 2px 4px rgb(0 0 0 / 0.1);

  /* Shorthand: property duration timing-function delay */
  transition:
    background 0.2s ease,
    transform 0.2s ease,
    box-shadow 0.2s ease;

  /* All properties */
  transition: all 0.2s ease;

  /* Multiple independent transitions */
  transition:
    background-color 0.2s ease,
    transform 0.15s ease 0.05s,
    /* 50ms delay */ box-shadow 0.2s ease;
}

.button:hover {
  background: #004499;
  transform: translateY(-2px);
  box-shadow: 0 6px 12px rgb(0 0 0 / 0.2);
}

.button:active {
  transform: translateY(0);
  box-shadow: 0 2px 4px rgb(0 0 0 / 0.1);
}
```

### Transition Properties

| Property                     | Description                   | Common values                                                            |
| ---------------------------- | ----------------------------- | ------------------------------------------------------------------------ |
| `transition-property`        | Which CSS property to animate | `all`, `background`, `transform`                                         |
| `transition-duration`        | How long the transition takes | `0.2s`, `300ms`                                                          |
| `transition-timing-function` | The pace curve                | `ease`, `linear`, `ease-in`, `ease-out`, `ease-in-out`, `cubic-bezier()` |
| `transition-delay`           | Wait before starting          | `0s`, `0.1s`, `200ms`                                                    |

### `cubic-bezier()` Timing Functions

```css
.animated {
  /* Predefined values */
  transition-timing-function: ease; /* cubic-bezier(0.25, 0.1, 0.25, 1) */
  transition-timing-function: linear; /* cubic-bezier(0, 0, 1, 1) */
  transition-timing-function: ease-in; /* cubic-bezier(0.42, 0, 1, 1) */
  transition-timing-function: ease-out; /* cubic-bezier(0, 0, 0.58, 1) */
  transition-timing-function: ease-in-out; /* cubic-bezier(0.42, 0, 0.58, 1) */

  /* Custom cubic bezier: control points P1(x1, y1) and P2(x2, y2) */
  /* Use https://cubic-bezier.com to visualise */
  transition-timing-function: cubic-bezier(
    0.34,
    1.56,
    0.64,
    1
  ); /* Spring bounce */
  transition-timing-function: cubic-bezier(
    0.87,
    0,
    0.13,
    1
  ); /* Strong ease-in-out */

  /* Step functions (no interpolation — jumps between values) */
  transition-timing-function: steps(4, end); /* 4 equal steps */
  transition-timing-function: step-start;
  transition-timing-function: step-end;
}
```

## CSS `@keyframes` Animations

Unlike transitions, `@keyframes` animations run automatically without a state trigger and give you full control over every step:

```css
/* Define the animation */
@keyframes slide-in-from-left {
  from {
    transform: translateX(-100%);
    opacity: 0;
  }
  to {
    transform: translateX(0);
    opacity: 1;
  }
}

@keyframes pulse {
  0% {
    transform: scale(1);
  }
  50% {
    transform: scale(1.05);
  }
  100% {
    transform: scale(1);
  }
}

@keyframes shimmer {
  0% {
    background-position: -200% 0;
  }
  100% {
    background-position: 200% 0;
  }
}

/* Apply the animation */
.notification {
  animation-name: slide-in-from-left;
  animation-duration: 0.4s;
  animation-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1);
  animation-delay: 0.1s;
  animation-iteration-count: 1; /* 1, 2, infinite */
  animation-direction: normal; /* normal, reverse, alternate, alternate-reverse */
  animation-fill-mode: both; /* none, forwards, backwards, both */
  animation-play-state: running; /* running, paused */

  /* Shorthand: name duration timing-function delay iteration direction fill-mode */
  animation: slide-in-from-left 0.4s cubic-bezier(0.34, 1.56, 0.64, 1) 0.1s 1
    normal both;
}

/* Multiple animations */
.loading-card {
  animation:
    shimmer 1.5s linear infinite,
    fade-in 0.3s ease forwards;
}
```

### `animation-fill-mode`

| Value       | Effect                                                   |
| ----------- | -------------------------------------------------------- |
| `none`      | Animation has no effect before/after playing (default)   |
| `forwards`  | Keeps the final keyframe state after animation ends      |
| `backwards` | Applies the first keyframe state during the delay period |
| `both`      | Combines forwards + backwards (most commonly useful)     |

## GPU-Accelerated Transforms

The browser renders most CSS changes by triggering **reflow** (geometry recalculation) and **repaint** (pixel painting). Some properties skip this entirely and go straight to the **compositor** for GPU acceleration.

**Properties that are GPU-accelerated (no reflow/repaint):**

- `transform` (translate, scale, rotate, skew, perspective)
- `opacity`
- `filter`
- `clip-path` (partially)

```css
/* ❌ Triggers layout reflow — slow for animation */
.bad-animation {
  transition:
    width 0.3s,
    height 0.3s,
    top 0.3s,
    left 0.3s;
}

/* ✅ Uses transform — GPU composited, no layout recalculation */
.good-animation {
  transition:
    transform 0.3s,
    opacity 0.3s;
}

/* Translate instead of top/left for moving elements */
.card:hover {
  transform: translateY(-4px); /* ✅ GPU */
  /* top: -4px;                     ❌ triggers layout */
}

/* Scale instead of width/height */
.btn:active {
  transform: scale(0.97); /* ✅ GPU */
  /* width: 97%;                    ❌ triggers layout */
}
```

### `will-change`

Hints to the browser that an element will be animated, allowing it to promote the element to its own GPU layer _before_ the animation starts:

```css
/* Use sparingly — each promoted layer uses GPU memory */
.animated-element {
  will-change: transform, opacity;
}

/* Best practice: add will-change just before animation starts */
.card {
  transition: transform 0.3s;
}

.card:hover {
  will-change: transform; /* Promote just when about to animate */
  transform: translateY(-4px);
}
```

:::caution[Don't Overuse will-change]
Applying `will-change` to many elements simultaneously consumes significant GPU memory and can _degrade_ performance. Apply it dynamically via JavaScript just before an animation starts, and remove it after the animation ends.
:::

## Scroll-Driven Animations

Modern CSS can link animation progress to scroll position — no JavaScript needed:

```css
@keyframes reveal-on-scroll {
  from {
    opacity: 0;
    transform: translateY(2rem);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.scroll-animated {
  animation: reveal-on-scroll linear both;
  animation-timeline: view(); /* Links to element's visibility in viewport */
  animation-range: entry 0% entry 30%; /* Animate during first 30% of entering viewport */
}

/* Progress bar tied to page scroll */
.reading-progress {
  position: fixed;
  top: 0;
  left: 0;
  height: 3px;
  background: #0066cc;
  transform-origin: left;
  animation: grow-progress linear;
  animation-timeline: scroll(); /* Links to root scroll container */
}

@keyframes grow-progress {
  from {
    transform: scaleX(0);
  }
  to {
    transform: scaleX(1);
  }
}
```

## View Transitions API

Animate between page states or DOM updates with the View Transitions API:

```css
/* Style the transition animation */
::view-transition-old(root) {
  animation: fade-out 0.25s ease forwards;
}

::view-transition-new(root) {
  animation: fade-in 0.25s ease forwards;
}

/* Name specific elements for individual transitions */
.hero-image {
  view-transition-name: hero;
}

::view-transition-old(hero) {
  animation: scale-out 0.4s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}

::view-transition-new(hero) {
  animation: scale-in 0.4s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}
```

```js
// Trigger a view transition
async function navigateTo(url) {
  if (!document.startViewTransition) {
    // Fallback for browsers without support
    window.location.href = url;
    return;
  }

  const transition = document.startViewTransition(async () => {
    // Update the DOM — this happens between old and new screenshots
    const response = await fetch(url);
    const html = await response.text();
    document.querySelector('main').innerHTML = html;
  });

  await transition.finished;
}
```

## `prefers-reduced-motion` — Always Include

:::caution[Always Respect User Motion Preferences]
Many users have vestibular disorders, epilepsy, or other conditions that make motion on screen physically uncomfortable or dangerous. Always check `prefers-reduced-motion` and reduce or eliminate animations for these users.
:::

```css
/* Base animations */
.card {
  transition:
    transform 0.3s ease,
    box-shadow 0.3s ease;
}

.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 25px rgb(0 0 0 / 0.15);
}

@keyframes spin {
  to {
    transform: rotate(360deg);
  }
}

.loading-spinner {
  animation: spin 0.8s linear infinite;
}

/* Reduce all animations for users who prefer it */
@media (prefers-reduced-motion: reduce) {
  /* Option 1: Disable all transitions and animations */
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }

  /* Option 2: Keep subtle alternatives */
  .card:hover {
    transform: none;
    box-shadow: 0 0 0 2px #0066cc; /* Non-motion highlight instead */
  }

  .loading-spinner {
    animation: none;
    border: 3px solid #0066cc; /* Static indicator */
  }
}
```

## The FLIP Technique

**FLIP** (First, Last, Invert, Play) is a technique for animating layout changes using transforms (GPU-friendly) instead of layout-triggering properties:

```js
function animateLayoutChange(element, changeCallback) {
  // 1. FIRST: record current position
  const first = element.getBoundingClientRect();

  // 2. LAST: make the change
  changeCallback();

  // 3. LAST: record new position
  const last = element.getBoundingClientRect();

  // 4. INVERT: calculate the difference
  const deltaX = first.left - last.left;
  const deltaY = first.top - last.top;
  const deltaW = first.width / last.width;
  const deltaH = first.height / last.height;

  // Apply the inverted transform to make it look like the old position
  element.style.transform = `translate(${deltaX}px, ${deltaY}px) scale(${deltaW}, ${deltaH})`;
  element.style.transformOrigin = 'top left';

  // 5. PLAY: animate to the natural (new) position
  requestAnimationFrame(() => {
    element.style.transition =
      'transform 0.4s cubic-bezier(0.34, 1.56, 0.64, 1)';
    element.style.transform = '';
  });

  element.addEventListener(
    'transitionend',
    () => {
      element.style.transition = '';
      element.style.transformOrigin = '';
    },
    { once: true },
  );
}
```

CSS animations bring interfaces to life. When used thoughtfully, they improve usability by communicating state changes. When overused, they create cognitive overload and accessibility problems.

## Transitions

Transitions smoothly interpolate between two CSS states.

```css
/* transition: property duration timing-function delay */

.button {
  background-color: #6366f1;
  color: white;
  transform: translateY(0);
  box-shadow: 0 2px 4px rgb(0 0 0 / 0.15);

  /* Transition multiple properties */
  transition:
    background-color 0.2s ease,
    transform 0.15s ease,
    box-shadow 0.2s ease;
}

.button:hover {
  background-color: #4f46e5;
  transform: translateY(-2px);
  box-shadow: 0 8px 16px rgb(0 0 0 / 0.2);
}

.button:active {
  transform: translateY(0);
  box-shadow: 0 1px 2px rgb(0 0 0 / 0.15);
}
```

| Property | Values | Description |
|---|---|---|
| `transition-property` | `all`, property name, `none` | Which properties to animate |
| `transition-duration` | `0.3s`, `300ms` | How long the transition takes |
| `transition-timing-function` | `ease`, `linear`, `ease-in`, `ease-out`, `ease-in-out`, `cubic-bezier()`, `steps()` | Speed curve |
| `transition-delay` | `0s`, `0.1s` | Wait before starting |

```css
/* Transition shorthand examples */

/* All properties — convenient but impacts performance */
.card { transition: all 0.3s ease; }

/* Specific properties — preferred for performance */
.card {
  transition:
    opacity 0.3s ease,
    transform 0.3s ease;
}

/* Delayed transition (e.g., tooltip that appears after a pause) */
.tooltip {
  opacity: 0;
  transition: opacity 0.2s ease 0.5s; /* 0.5s delay */
}
.tooltip-trigger:hover .tooltip {
  opacity: 1;
  transition-delay: 0s; /* No delay when showing, instant hiding */
}
```

## cubic-bezier Timing Functions

The timing function controls the speed curve of the transition.

```css
/* Built-in easing functions */
.ease        { transition-timing-function: ease; }          /* cubic-bezier(0.25,0.1,0.25,1) */
.linear      { transition-timing-function: linear; }
.ease-in     { transition-timing-function: ease-in; }       /* Starts slow, ends fast */
.ease-out    { transition-timing-function: ease-out; }      /* Starts fast, ends slow */
.ease-in-out { transition-timing-function: ease-in-out; }   /* Slow at both ends */

/* Custom cubic-bezier: cubic-bezier(x1, y1, x2, y2) */
/* x1,y1 = handle for start, x2,y2 = handle for end */

.bounce {
  transition-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1);
  /* Overshoot (spring effect) */
}

.snappy {
  transition-timing-function: cubic-bezier(0.16, 1, 0.3, 1);
  /* Fast start, slow end — feels snappy and responsive */
}

.anticipation {
  transition-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55);
  /* Slight pullback before the main motion */
}

/* steps() for sprite animations and discrete jumps */
.sprite {
  animation-timing-function: steps(8, end);
  /* Jump in 8 discrete steps */
}
```

:::tip[Use easings.net]
Visit [easings.net](https://easings.net/) and [cubic-bezier.com](https://cubic-bezier.com/) to visually preview easing curves and copy the CSS values.
:::

## CSS Animations with @keyframes

While transitions animate between two states, `@keyframes` animations can define multiple intermediate states and run automatically.

```css
/* ─── Define keyframes ───────────────────────────────────────────────────── */

@keyframes fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}

@keyframes slide-up {
  from {
    opacity: 0;
    transform: translateY(24px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

@keyframes bounce {
  0%   { transform: translateY(0); animation-timing-function: ease-in; }
  40%  { transform: translateY(-20px); animation-timing-function: ease-out; }
  65%  { transform: translateY(-10px); animation-timing-function: ease-in; }
  82%  { transform: translateY(-5px); animation-timing-function: ease-out; }
  93%  { transform: translateY(-2px); animation-timing-function: ease-in; }
  100% { transform: translateY(0); }
}

/* ─── Apply animations ───────────────────────────────────────────────────── */

.hero-text {
  animation: slide-up 0.6s ease-out both;
  /* name duration timing-function fill-mode */
}

.loading-spinner {
  animation: spin 1s linear infinite;
  /* name duration timing-function iteration-count */
}

/* Staggered entrance animations */
.stagger > *:nth-child(1) { animation: slide-up 0.4s ease-out 0.0s both; }
.stagger > *:nth-child(2) { animation: slide-up 0.4s ease-out 0.1s both; }
.stagger > *:nth-child(3) { animation: slide-up 0.4s ease-out 0.2s both; }
.stagger > *:nth-child(4) { animation: slide-up 0.4s ease-out 0.3s both; }
```

### Animation Properties Reference

```css
.animated-element {
  animation-name: slide-up;
  animation-duration: 0.6s;
  animation-timing-function: ease-out;
  animation-delay: 0.2s;
  animation-iteration-count: 1;        /* or: infinite, 3 */
  animation-direction: normal;         /* normal | reverse | alternate | alternate-reverse */
  animation-fill-mode: both;           /* none | forwards | backwards | both */
  animation-play-state: running;       /* running | paused */

  /* Shorthand: name duration timing-function delay iteration direction fill-mode */
  animation: slide-up 0.6s ease-out 0.2s 1 normal both;

  /* Multiple animations */
  animation:
    fade-in 0.4s ease-out both,
    slide-up 0.6s ease-out both;
}
```

| `animation-fill-mode` | Before | During | After |
|---|---|---|---|
| `none` | Pre-animation state | Animating | Pre-animation state |
| `forwards` | Pre-animation state | Animating | Stays at final keyframe |
| `backwards` | Jumps to first keyframe | Animating | Pre-animation state |
| `both` | Jumps to first keyframe | Animating | Stays at final keyframe |

## GPU-Accelerated Transforms

For smooth, performant animations, animate only **`transform`** and **`opacity`**. These are handled by the GPU compositor and do not trigger layout or paint.

```css
/* ✅ GPU-accelerated: no layout or paint needed */
.card:hover {
  transform: translateY(-4px) scale(1.02);
  opacity: 0.95;
}

/* ❌ Triggers layout reflow (expensive) */
.bad-animation:hover {
  width: 120%;     /* Triggers layout */
  height: 200px;   /* Triggers layout */
  top: -10px;      /* Triggers layout (on positioned element) */
}

/* ❌ Triggers paint (less expensive than layout but still slow) */
.also-bad:hover {
  background-color: red;  /* Triggers paint */
  border-radius: 50%;     /* May trigger paint */
  box-shadow: 0 10px 30px; /* Triggers paint */
}
```

### Transform Functions

```css
.transforms {
  /* Translation (movement) */
  transform: translateX(20px);        /* Move right */
  transform: translateY(-20px);       /* Move up */
  transform: translate(20px, -20px);  /* Move right and up */
  transform: translateZ(0);           /* GPU promotion hint (use sparingly) */
  transform: translate3d(0, 0, 0);    /* Alternative GPU hint */

  /* Scaling */
  transform: scale(1.1);              /* Uniform scale */
  transform: scaleX(1.5);             /* Scale horizontally */
  transform: scaleY(0.8);             /* Scale vertically */

  /* Rotation */
  transform: rotate(45deg);           /* 2D rotation */
  transform: rotateX(45deg);          /* Rotate around X axis (3D) */
  transform: rotateY(45deg);          /* Rotate around Y axis (3D) */

  /* Skewing */
  transform: skewX(10deg);
  transform: skewY(-5deg);

  /* Combined (order matters! Applied right to left) */
  transform: translateY(-50%) rotate(45deg) scale(1.1);
}
```

## will-change

```css
/* Hint to the browser that this element will be animated */
/* Creates a new GPU compositing layer */

.animated-card {
  will-change: transform, opacity;
  /* Use only when you KNOW the element will animate soon */
}

/* Apply/remove dynamically for temporary animations */
.will-animate {
  /* Add this class just before the animation starts */
  will-change: transform;
}
```

<Aside type="caution" title="Use will-change sparingly">
`will-change` causes the browser to allocate GPU memory for every element it's applied to. Over-using it wastes GPU RAM and can actually hurt performance. Apply it just before animation starts and remove it after: add/remove a class via JavaScript at the right moment.
</Aside>

## Scroll-Driven Animations

Animate elements based on scroll position with no JavaScript.

```css
/* Link an animation to the scroll progress of the page */
@keyframes grow-bar {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}

.reading-progress-bar {
  position: fixed;
  top: 0;
  left: 0;
  height: 4px;
  width: 100%;
  background: #6366f1;
  transform-origin: left;

  /* animation-timeline: scroll() — ties animation to page scroll */
  animation: grow-bar linear;
  animation-timeline: scroll();
}

/* Animate elements as they enter the viewport */
@keyframes reveal {
  from {
    opacity: 0;
    transform: translateY(40px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.scroll-reveal {
  animation: reveal linear both;
  animation-timeline: view();
  /* view() = tied to element's position within the viewport */
  animation-range: entry 0% entry 40%;
  /* Start animating when element enters, finish when 40% is visible */
}
```

## View Transitions API

```css
/* Enable smooth page transitions in a Single Page App */

/* Step 1: Set names on elements you want to animate between pages */
.hero-image {
  view-transition-name: hero;
}
.page-title {
  view-transition-name: page-title;
}

/* Step 2: Style the transition (optional — browser provides defaults) */
::view-transition-old(hero),
::view-transition-new(hero) {
  animation-duration: 0.4s;
  animation-timing-function: ease-in-out;
}
```

```js
// Step 3: Wrap navigation in startViewTransition()
document.querySelectorAll('a').forEach((link) => {
  link.addEventListener('click', async (e) => {
    e.preventDefault();
    const href = link.href;

    if (!document.startViewTransition) {
      window.location.href = href;
      return;
    }

    document.startViewTransition(async () => {
      const html = await fetch(href).then((r) => r.text());
      document.body.innerHTML = new DOMParser()
        .parseFromString(html, 'text/html')
        .body.innerHTML;
      history.pushState({}, '', href);
    });
  });
});
```

## prefers-reduced-motion

Always respect users who have requested reduced motion in their OS accessibility settings.

```css
/* ─── The essential rule — include in EVERY project ─────────────────────── */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

/* ─── Better approach: opt IN to motion rather than opting out ───────────── */
/* Default: no motion */
.card {
  transition: none;
}

/* Only add motion for users who haven't opted out */
@media (prefers-reduced-motion: no-preference) {
  .card {
    transition: transform 0.3s ease, box-shadow 0.3s ease;
  }
  .card:hover {
    transform: translateY(-4px);
    box-shadow: 0 12px 24px rgb(0 0 0 / 0.15);
  }
}
```

:::caution[Motion accessibility]
Vestibular disorders, epilepsy, and other conditions make large or rapid animations painful or dangerous. Always include a `prefers-reduced-motion` check. WCAG 2.1 Success Criterion 2.3.3 (AAA) requires a way to disable non-essential motion. Target AA minimum by never autoplay animations for more than 5 seconds without a pause mechanism.
:::

## FLIP Animation Technique

FLIP (First, Last, Invert, Play) enables smooth animations between two arbitrary DOM states.

```js
// FLIP: animate DOM reordering smoothly

// 1. FIRST — capture initial positions
const cards = document.querySelectorAll('.card');
const firstPositions = new Map();
cards.forEach((card) => {
  firstPositions.set(card, card.getBoundingClientRect());
});

// 2. LAST — perform the DOM change (sorting, filtering, reordering)
sortCards(); // Whatever changes the DOM layout

// 3. INVERT — calculate the difference and apply as transform
cards.forEach((card) => {
  const first = firstPositions.get(card);
  const last = card.getBoundingClientRect();

  const dx = first.left - last.left;
  const dy = first.top - last.top;

  // Snap the card back to where it USED to be
  card.style.transform = `translate(${dx}px, ${dy}px)`;
});

// 4. PLAY — remove the inverted transform (with a transition)
requestAnimationFrame(() => {
  cards.forEach((card) => {
    card.style.transition = 'transform 0.4s cubic-bezier(0.4, 0, 0.2, 1)';
    card.style.transform = '';
  });
});
```

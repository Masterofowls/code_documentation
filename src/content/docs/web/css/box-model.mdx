Every element in CSS is represented as a rectangular box. The **box model** defines how the content, padding, border, and margin of that box relate to each other and to the surrounding layout.

## The Box Model

```
┌─────────────────────────────────────────┐
│              margin                     │
│  ┌───────────────────────────────────┐  │
│  │            border                 │  │
│  │  ┌─────────────────────────────┐  │  │
│  │  │          padding            │  │  │
│  │  │  ┌───────────────────────┐  │  │  │
│  │  │  │       content         │  │  │  │
│  │  │  │  width × height       │  │  │  │
│  │  │  └───────────────────────┘  │  │  │
│  │  └─────────────────────────────┘  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

```css
.box {
  /* Content area */
  width: 300px;
  height: 200px;

  /* Padding — space inside the border */
  padding: 1.5rem; /* all sides */
  padding: 1rem 2rem; /* vertical horizontal */
  padding: 1rem 1.5rem 2rem; /* top h-sides bottom */
  padding: 0.5rem 1rem 1.5rem 2rem; /* top right bottom left */

  /* Individual sides */
  padding-top: 1rem;
  padding-right: 1.5rem;
  padding-bottom: 1rem;
  padding-left: 1.5rem;

  /* Border */
  border: 2px solid #e2e8f0;
  border-top: 3px solid #0066cc;
  border-radius: 0.5rem;

  /* Margin — space outside the border */
  margin: 2rem;
  margin: 1rem auto; /* auto centers block elements */
  margin: 0 auto 2rem;

  /* Individual margin sides */
  margin-top: 1.5rem;
  margin-inline: auto; /* logical property: left + right */
}
```

## `box-sizing`: Why `border-box` is Better

The default `box-sizing: content-box` makes `width` and `height` apply to the **content area only** — padding and border are added on top. This makes layout maths awkward:

```css
/* content-box (default) — confusing arithmetic */
.box {
  box-sizing: content-box;
  width: 300px;
  padding: 20px;
  border: 2px solid;
  /* Total rendered width: 300 + 20 + 20 + 2 + 2 = 344px */
}

/* border-box — intuitive: width INCLUDES padding and border */
.box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid;
  /* Total rendered width: exactly 300px */
  /* Content area: 300 - 20 - 20 - 2 - 2 = 256px */
}
```

### Apply border-box Globally (Best Practice)

```css
/*
  The modern CSS reset: apply border-box to everything.
  The * selector inherits box-sizing rather than setting it directly,
  so component libraries can opt back into content-box if needed.
*/
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

:::tip[Always Use border-box]
`border-box` is far more intuitive. When you set `width: 50%`, you get exactly 50% — padding and border don't cause overflow. Almost every CSS framework and reset uses it as the default.
:::

## Margin Collapse

Vertical margins between adjacent block elements **collapse** — they merge into the larger of the two values rather than adding together.

```css
/* These two paragraphs have margin-bottom: 2rem and margin-top: 1rem */
/* The gap between them is 2rem (the larger value), not 3rem */
p {
  margin-top: 1rem;
  margin-bottom: 2rem;
}
```

### Three Cases Where Margins Collapse

```html
<!-- 1. Adjacent siblings: margin collapses between them -->
<div style="margin-bottom: 2rem">First</div>
<div style="margin-top: 1rem">Second</div>
<!-- Gap = max(2rem, 1rem) = 2rem -->

<!-- 2. Parent and first/last child (if no border/padding between them) -->
<div class="parent" style="margin-top: 3rem">
  <div class="child" style="margin-top: 1rem">Child</div>
</div>
<!-- Parent's margin-top = max(3rem, 1rem) = 3rem -->
<!-- The child's margin "escapes" the parent -->

<!-- 3. Empty blocks: own top and bottom margins collapse -->
<div style="margin-top: 1rem; margin-bottom: 2rem; height: 0">
  <!-- This empty block has effective margin of 2rem, not 3rem -->
</div>
```

### Preventing Margin Collapse

```css
/* Add any of these to the parent to prevent parent-child collapse: */
.parent {
  padding-top: 1px; /* Any padding prevents collapse */
  border-top: 1px solid transparent; /* Any border prevents collapse */
  overflow: hidden; /* Creates a block formatting context */
  display: flex; /* Flexbox does not collapse margins */
  display: grid; /* Grid does not collapse margins */
}
```

:::caution[Margin Collapse Only Applies Vertically]
Horizontal margins never collapse. Only top/bottom (block-axis) margins collapse, and only for **block-level elements in normal flow** — not for flex/grid items, positioned elements, or floated elements.
:::

## Negative Margins

Negative margins pull elements toward each other or outside their container:

```css
/* Pull element upward / overlap with previous element */
.overlap-card {
  margin-top: -2rem;
}

/* Extend element to fill container padding */
.full-bleed-image {
  margin-inline: -1.5rem; /* negate the parent's padding */
}

/* Create optical alignment */
.outdented-heading {
  margin-left: -0.1em; /* compensate for font's optical margin */
}
```

## Overflow

```css
.container {
  overflow: visible; /* Default — content escapes the box */
  overflow: hidden; /* Clip content and hide scrollbar */
  overflow: scroll; /* Always show scrollbars (even if content fits) */
  overflow: auto; /* Show scrollbars only when content overflows */
  overflow: clip; /* Like hidden but no scroll programming — also clips in margin area */

  /* Control axes separately */
  overflow-x: hidden;
  overflow-y: auto;

  /* Text overflow (requires white-space: nowrap and overflow: hidden) */
  overflow: hidden;
  white-space: nowrap;
  text-overflow: ellipsis; /* Show "…" when text overflows */

  /* Multi-line text truncation (WebKit — widely supported) */
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

## `outline` vs `border`

| Property         | `border`                          | `outline`                                        |
| ---------------- | --------------------------------- | ------------------------------------------------ |
| Box model        | Part of box — takes up space      | Drawn outside the border — no space taken        |
| Rounded corners  | `border-radius` applies           | `outline` follows box shape (in modern browsers) |
| Per-side control | `border-top`, `border-left`, etc. | No per-side control                              |
| Typical use      | Design element                    | Focus indicator                                  |

```css
/* Border: part of the element's visual design */
.card {
  border: 2px solid #e2e8f0;
  border-radius: 0.75rem;
}

/* Outline: focus indicator — never remove without a replacement! */
:focus-visible {
  outline: 3px solid #0066cc;
  outline-offset: 3px;
  border-radius: 4px; /* Modern browsers honour this for outline */
}

/* Remove outline only when you have a visible replacement */
button:focus:not(:focus-visible) {
  outline: none;
}
```

## `box-shadow`

```css
.card {
  /* Single shadow: offset-x offset-y blur spread colour */
  box-shadow: 0 1px 3px 0 rgb(0 0 0 / 0.1);

  /* Multiple shadows (comma-separated, front to back) */
  box-shadow:
    0 1px 2px 0 rgb(0 0 0 / 0.05),
    0 4px 6px -1px rgb(0 0 0 / 0.1);

  /* Inset shadow (inside the element) */
  box-shadow: inset 0 2px 4px 0 rgb(0 0 0 / 0.05);

  /* Coloured shadow */
  box-shadow: 0 4px 20px rgb(0 102 204 / 0.3);
}
```

## `border-radius`

```css
.box {
  /* All corners equal */
  border-radius: 0.5rem;

  /* Two values: top-left+bottom-right / top-right+bottom-left */
  border-radius: 0.5rem 1rem;

  /* Four values: TL TR BR BL */
  border-radius: 0.25rem 0.5rem 0.75rem 1rem;

  /* Individual corners */
  border-top-left-radius: 0.5rem;
  border-top-right-radius: 0;
  border-bottom-right-radius: 0.5rem;
  border-bottom-left-radius: 0;

  /* Perfect circle */
  border-radius: 50%;

  /* Pill shape */
  border-radius: 9999px;

  /* Elliptical corners: horizontal-radius / vertical-radius */
  border-radius: 50% / 30%;
}
```

## `object-fit` and `object-position`

Control how replaced content (images, video) fills its container:

```css
.cover-image {
  width: 400px;
  height: 300px;
  object-fit: cover; /* Crop to fill — keeps aspect ratio */
  object-position: center top; /* Focus on the top */
}

.contain-image {
  object-fit: contain; /* Letterbox — fits entirely, may have blank space */
}

.fill-image {
  object-fit: fill; /* Stretch to fill — may distort */
}

.none-image {
  object-fit: none; /* Natural size, may overflow or be cropped */
}

.scale-down-image {
  object-fit: scale-down; /* none or contain — whichever is smaller */
}
```

## Visibility vs `display: none` vs `opacity: 0`

| Property                              | Visible | Takes up space | Accessible          | Animatable |
| ------------------------------------- | ------- | -------------- | ------------------- | ---------- |
| `visibility: visible`                 | ✅ Yes  | ✅ Yes         | ✅ Yes              | Partial    |
| `visibility: hidden`                  | ❌ No   | ✅ Yes         | ❌ Hidden from AT   | Partial    |
| `display: none`                       | ❌ No   | ❌ No          | ❌ Hidden from AT   | ❌ No      |
| `opacity: 0`                          | ❌ No   | ✅ Yes         | ✅ Still accessible | ✅ Yes     |
| `opacity: 0` + `pointer-events: none` | ❌ No   | ✅ Yes         | ✅ Still accessible | ✅ Yes     |

```css
/* Fade in/out animation — use opacity, not display */
.modal {
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.3s ease;
}

.modal.is-open {
  opacity: 1;
  pointer-events: auto;
}

/* For accessibility: hide visually but not from screen readers */
.sr-only {
  visibility: hidden; /* Takes up space — useful when you need the layout preserved */
}

/* Complete removal from layout and AT */
[hidden] {
  display: none !important;
}
```

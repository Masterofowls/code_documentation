CSS custom properties (also called CSS variables) bring the power of variables to native CSS — no preprocessor required. They are inheritable, can be changed at runtime with JavaScript, and enable powerful theming patterns.

## Declaring and Using Custom Properties

```css
/* ─── Declaration: --property-name: value ────────────────────────────────── */
/* Custom properties MUST start with two dashes */

:root {
  /* Declared on :root = globally available throughout the document */
  --color-primary: #6366f1;
  --color-primary-dark: #4f46e5;
  --color-text: #1e293b;
  --color-background: #ffffff;

  --font-family-base: 'Inter', system-ui, sans-serif;
  --font-size-base: 1rem;

  --spacing-xs: 0.25rem;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 1.5rem;
  --spacing-xl: 2rem;

  --radius-sm: 0.25rem;
  --radius-md: 0.5rem;
  --radius-lg: 1rem;
  --radius-full: 9999px;

  --shadow-sm: 0 1px 3px rgb(0 0 0 / 0.1);
  --shadow-md: 0 4px 6px -1px rgb(0 0 0 / 0.1), 0 2px 4px -2px rgb(0 0 0 / 0.1);
  --shadow-lg: 0 10px 15px -3px rgb(0 0 0 / 0.1), 0 4px 6px -4px rgb(0 0 0 / 0.1);
}

/* ─── Usage: var(--property-name) ────────────────────────────────────────── */

.button {
  background-color: var(--color-primary);
  color: white;
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius-md);
  font-family: var(--font-family-base);
  box-shadow: var(--shadow-sm);
}

.button:hover {
  background-color: var(--color-primary-dark);
  box-shadow: var(--shadow-md);
}

/* ─── Fallback value ──────────────────────────────────────────────────────── */
/* var(--name, fallback) */

.card {
  /* If --card-padding is not defined, fall back to 1.5rem */
  padding: var(--card-padding, 1.5rem);
  border-radius: var(--card-radius, var(--radius-md)); /* Fallback to another variable */
  color: var(--card-text-color, var(--color-text, #1e293b)); /* Chained fallbacks */
}
```

<Aside type="tip" title="Custom properties are case-sensitive">
`--Color-Primary` and `--color-primary` are different variables. Use a consistent naming convention. **Kebab-case with a namespace prefix** (e.g., `--brand-color-primary`, `--card-bg`) is most common.
</Aside>

## Scope: :root vs Local

```css
/* ─── Global scope (:root) ──────────────────────────────────────────────── */
:root {
  --brand-color: #6366f1;
  /* Available to every element in the document */
}

/* ─── Component scope (local) ──────────────────────────────────────────── */
.alert {
  /* These variables only apply inside .alert and its descendants */
  --alert-bg: #fef9c3;
  --alert-border: #fde047;
  --alert-text: #713f12;

  background-color: var(--alert-bg);
  border: 1px solid var(--alert-border);
  color: var(--alert-text);
  padding: var(--spacing-md);
  border-radius: var(--radius-md);
}

.alert--error {
  /* Override the local variables for the error variant */
  --alert-bg: #fef2f2;
  --alert-border: #fca5a5;
  --alert-text: #7f1d1d;
}

.alert--success {
  --alert-bg: #f0fdf4;
  --alert-border: #86efac;
  --alert-text: #14532d;
}
```

    const toggle = document.getElementById('theme-toggle');
    const root = document.documentElement;

    // Load saved preference

        (!saved && window.matchMedia('(prefers-color-scheme: dark)').matches)) {

## Updating Custom Properties with JavaScript

```js
// Read a custom property
const style = getComputedStyle(document.documentElement);
const primaryColor = style.getPropertyValue('--color-primary').trim();
// Returns: '#6366f1'

// Set a custom property on :root (affects the whole page)
document.documentElement.style.setProperty('--color-primary', '#10b981');

// Set on a specific element (scoped to that element and descendants)
const card = document.getElementById('featured-card');
card.style.setProperty('--card-highlight-color', '#fbbf24');

// Remove a custom property (reverts to inherited/default value)
document.documentElement.style.removeProperty('--color-primary');

// ─── Dynamic progress bar ─────────────────────────────────────────────────
const progressBar = document.getElementById('progress');
function updateProgress(percent) {
  progressBar.style.setProperty('--progress', `${percent}%`);
}
```

```css
/* CSS side of the progress bar */
.progress-bar {
  --progress: 0%;
  width: 100%;
  height: 8px;
  background-color: var(--color-border);
  border-radius: var(--radius-full);
  overflow: hidden;
}

.progress-bar::after {
  content: '';
  display: block;
  height: 100%;
  width: var(--progress); /* Uses the JS-set variable */
  background-color: var(--color-primary);
  border-radius: inherit;
  transition: width 0.3s ease;
}
```

## @property — Typed Custom Properties

`@property` defines the type, initial value, and inheritance behaviour of a custom property. This enables CSS to animate custom properties and provides better validation.

```css
/* Without @property, custom properties cannot be interpolated/animated */

/* ─── Define a typed custom property ─────────────────────────────────────── */
@property --hue {
  syntax: '<number>';       /* Type: number, color, length, percentage, etc. */
  inherits: false;          /* Whether children inherit this property */
  initial-value: 264;       /* Default value (must match the syntax) */
}

@property --accent-color {
  syntax: '<color>';
  inherits: true;
  initial-value: #6366f1;
}

@property --card-rotation {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

/* ─── Animate the typed custom property ─────────────────────────────────── */
@keyframes hue-rotate {
  to { --hue: 364; } /* Animate the number */
}

.rainbow-text {
  color: oklch(0.6 0.2 var(--hue));
  animation: hue-rotate 4s linear infinite;
  /* Without @property, this would NOT animate smoothly — it would jump */
}

/* ─── Animate a gradient (impossible without @property!) ─────────────────── */
@property --gradient-stop {
  syntax: '<percentage>';
  inherits: false;
  initial-value: 0%;
}

.gradient-animation {
  background: linear-gradient(135deg, #6366f1 var(--gradient-stop), #8b5cf6 100%);
  transition: --gradient-stop 0.4s ease;
}

.gradient-animation:hover {
  --gradient-stop: 50%;
}
```

## Design Token Patterns

```css
/* ─── Scale-based design tokens ─────────────────────────────────────────── */
:root {
  /* Spacing scale (multiples of 4px) */
  --space-1: 0.25rem;  /* 4px */
  --space-2: 0.5rem;   /* 8px */
  --space-3: 0.75rem;  /* 12px */
  --space-4: 1rem;     /* 16px */
  --space-5: 1.25rem;  /* 20px */
  --space-6: 1.5rem;   /* 24px */
  --space-8: 2rem;     /* 32px */
  --space-10: 2.5rem;  /* 40px */
  --space-12: 3rem;    /* 48px */
  --space-16: 4rem;    /* 64px */
  --space-20: 5rem;    /* 80px */

  /* Typography scale */
  --text-xs:   0.75rem;   /* 12px */
  --text-sm:   0.875rem;  /* 14px */
  --text-base: 1rem;      /* 16px */
  --text-lg:   1.125rem;  /* 18px */
  --text-xl:   1.25rem;   /* 20px */
  --text-2xl:  1.5rem;    /* 24px */
  --text-3xl:  1.875rem;  /* 30px */
  --text-4xl:  2.25rem;   /* 36px */

  /* Semantic tokens — reference primitive tokens */
  --heading-font-size: var(--text-4xl);
  --body-font-size: var(--text-base);
  --small-font-size: var(--text-sm);

  --content-gap: var(--space-6);
  --section-spacing: var(--space-16);
  --component-padding: var(--space-4);
}

/* ─── Component variant pattern ─────────────────────────────────────────── */
.badge {
  /* Default values */
  --badge-bg: var(--color-surface);
  --badge-text: var(--color-text);
  --badge-border: var(--color-border);

  display: inline-flex;
  align-items: center;
  padding: var(--space-1) var(--space-3);
  background-color: var(--badge-bg);
  color: var(--badge-text);
  border: 1px solid var(--badge-border);
  border-radius: var(--radius-full);
  font-size: var(--text-sm);
  font-weight: 500;
}

/* Variants just override the variables */
.badge--primary   { --badge-bg: #ede9fe; --badge-text: #4c1d95; --badge-border: #c4b5fd; }
.badge--success   { --badge-bg: #dcfce7; --badge-text: #14532d; --badge-border: #86efac; }
.badge--warning   { --badge-bg: #fef9c3; --badge-text: #713f12; --badge-border: #fde047; }
.badge--error     { --badge-bg: #fef2f2; --badge-text: #7f1d1d; --badge-border: #fca5a5; }
```

:::tip[Semantic token naming]
Use two layers of tokens:
1. **Primitive tokens**: `--color-blue-500: #6366f1` — the raw scale
2. **Semantic tokens**: `--color-primary: var(--color-blue-500)` — the meaning

This lets you retheme an entire design system by only changing semantic tokens, while primitives remain stable.
:::

## Custom Property Inheritance

Custom properties participate in the CSS inheritance model like any other inherited property.

```css
/* Parent defines the variable */
.theme-purple {
  --theme-color: #7c3aed;
}

/* Child inherits it automatically */
.theme-purple .button {
  background-color: var(--theme-color); /* Inherits #7c3aed from parent */
}

/* Override at any level in the tree */
.theme-purple .special-section {
  --theme-color: #db2777; /* Override for this subtree */
}

/* .special-section's children now get #db2777 */
.theme-purple .special-section .button {
  background-color: var(--theme-color); /* Gets #db2777 */
}
```

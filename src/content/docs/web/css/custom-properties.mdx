CSS Custom Properties — commonly called CSS variables — let you define reusable values in CSS. They are far more powerful than preprocessor variables because they cascade, inherit, can be updated at runtime, and respond to media queries.

## Declaring and Using Custom Properties

```css
/* ─── Declaration ────────────────────────────────────────── */
/* Must start with two dashes (--). Case-sensitive. */
:root {
  --color-primary: #3b82f6;
  --color-primary-hover: #2563eb;
  --spacing-base: 1rem;
  --border-radius: 0.5rem;
  --font-heading: 'Inter', sans-serif;
}

/* ─── Usage with var() ───────────────────────────────────── */
.button {
  background: var(--color-primary);
  padding: calc(var(--spacing-base) * 0.5) var(--spacing-base);
  border-radius: var(--border-radius);
  font-family: var(--font-heading);
}

.button:hover {
  background: var(--color-primary-hover);
}
```

## Fallback Values

`var()` accepts a fallback value as its second argument:

```css
.element {
  /* If --color-accent is not defined, use #f59e0b */
  color: var(--color-accent, #f59e0b);

  /* Fallback can itself use var() */
  color: var(--color-accent, var(--color-primary, #3b82f6));

  /* Fallback can be any valid value for the property */
  font-size: var(--font-size-hero, clamp(2rem, 5vw, 4rem));
  gap: var(--grid-gap, 1.5rem);
}
```

:::tip[Fallbacks Are for Undefined Variables, Not Invalid Ones]
The fallback in `var(--undefined, red)` is used when the variable is not defined. However, if a variable is defined but has an **invalid** value for the property (e.g. `--size: hello`), the fallback is **not** used — the property is set to its inherited or initial value. This is a common gotcha.
:::

## Scope: `:root` vs Local

Custom properties are **scoped to the element they're declared on** and inherited by descendants:

```css
/* Global scope — available everywhere */
:root {
  --font-size-base: 1rem;
  --color-text: #374151;
}

/* Component scope — only available inside .card */
.card {
  --card-padding: 1.5rem;
  --card-border: 1px solid #e5e7eb;

  padding: var(--card-padding);
  border: var(--card-border);
}

/* Override for a specific variant */
.card--featured {
  --card-padding: 2rem;
  --card-border: 2px solid #3b82f6;
}

/* Deep descendant inherits from ancestor scope */
.card .card-title {
  /* Can use --card-padding even though it's declared on .card */
  margin-bottom: calc(var(--card-padding) / 2);
}
```

## Updating with JavaScript

CSS custom properties can be read and updated at runtime with JavaScript — no classList manipulation needed for dynamic values:

```js
const root = document.documentElement; // :root element

// Read a custom property
const primaryColor = getComputedStyle(root).getPropertyValue('--color-primary').trim();
console.log(primaryColor); // '#3b82f6'

// Set a custom property globally
root.style.setProperty('--color-primary', '#8b5cf6');

// Set on a specific element
const card = document.querySelector('.card');
card.style.setProperty('--card-accent', '#f59e0b');

// Remove an override (reverts to the :root value)
card.style.removeProperty('--card-accent');

// Dynamic theme switcher
function setTheme(theme) {
  const themes = {
    blue:   { '--color-primary': '#3b82f6', '--color-primary-hover': '#2563eb' },
    purple: { '--color-primary': '#8b5cf6', '--color-primary-hover': '#7c3aed' },
    green:  { '--color-primary': '#22c55e', '--color-primary-hover': '#16a34a' },
  };

  Object.entries(themes[theme]).forEach(([property, value]) => {
    root.style.setProperty(property, value);
  });
}

// Responsive font size scaling
function updateSpacing() {
  const viewport = window.innerWidth;
  const scale = Math.max(0.75, Math.min(1, viewport / 1440));
  root.style.setProperty('--spacing-scale', scale);
}

window.addEventListener('resize', updateSpacing);
updateSpacing();
```

## `@property` — Typed Custom Properties

`@property` lets you declare the **type**, **initial value**, and **inheritance** of a custom property. This unlocks animation of custom properties:

```css
/* Declare a typed custom property */
@property --hue {
  syntax: '<number>';
  inherits: false;
  initial-value: 220;
}

@property --color-primary {
  syntax: '<color>';
  inherits: true;
  initial-value: #3b82f6;
}

@property --progress {
  syntax: '<percentage>';
  inherits: false;
  initial-value: 0%;
}

/* Now we can ANIMATE a custom property! */
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.animated-gradient {
  background: linear-gradient(var(--gradient-angle), #3b82f6, #8b5cf6);
  animation: rotate-gradient 3s linear infinite;
}

@keyframes rotate-gradient {
  to { --gradient-angle: 360deg; }
}

/* Animate a hue-shift */
@property --hue {
  syntax: '<integer>';
  inherits: false;
  initial-value: 0;
}

.hue-shift {
  background: hsl(var(--hue), 80%, 60%);
  transition: --hue 500ms ease;
}

.hue-shift:hover {
  --hue: 120; /* Smoothly animates from 0 to 120 */
}
```

## Design Token Patterns

### Token Architecture

```css
/* ─── 1. Primitive tokens — raw values ────────────────── */
:root {
  /* Colour palette */
  --blue-50: #eff6ff;
  --blue-100: #dbeafe;
  --blue-500: #3b82f6;
  --blue-600: #2563eb;
  --blue-900: #1e3a8a;

  /* Spacing scale */
  --space-1: 0.25rem;
  --space-2: 0.5rem;
  --space-3: 0.75rem;
  --space-4: 1rem;
  --space-6: 1.5rem;
  --space-8: 2rem;
  --space-12: 3rem;
  --space-16: 4rem;

  /* Typography scale */
  --text-xs: 0.75rem;
  --text-sm: 0.875rem;
  --text-base: 1rem;
  --text-lg: 1.125rem;
  --text-xl: 1.25rem;
  --text-2xl: 1.5rem;
  --text-4xl: 2.25rem;
}

/* ─── 2. Semantic tokens — reference primitive tokens ──── */
:root {
  /* Interactive */
  --color-action:       var(--blue-500);
  --color-action-hover: var(--blue-600);
  --color-action-muted: var(--blue-50);

  /* Surface */
  --color-surface:        #ffffff;
  --color-surface-raised: var(--blue-50);

  /* Text */
  --color-text-primary:   #111827;
  --color-text-secondary: #6b7280;
  --color-text-disabled:  #9ca3af;
}

/* ─── 3. Component tokens — use semantic tokens ─────────── */
.button {
  --button-bg:     var(--color-action);
  --button-bg-hover: var(--color-action-hover);
  --button-color:  #ffffff;
  --button-padding: var(--space-2) var(--space-4);
  --button-radius: var(--space-2);

  background: var(--button-bg);
  color: var(--button-color);
  padding: var(--button-padding);
  border-radius: var(--button-radius);
}

.button:hover {
  background: var(--button-bg-hover);
}
```

### Responsive Spacing Scale

```css
:root {
  --spacing-scale: 1;
}

@media (min-width: 768px) {
  :root { --spacing-scale: 1.125; }
}

@media (min-width: 1280px) {
  :root { --spacing-scale: 1.25; }
}

.section {
  padding-block: calc(var(--space-16) * var(--spacing-scale));
}
```

### Component Variant Pattern

```css
/* Base button with default values */
.btn {
  --btn-bg:       #3b82f6;
  --btn-color:    #ffffff;
  --btn-border:   transparent;
  --btn-size:     var(--text-sm);
  --btn-padding:  var(--space-2) var(--space-4);
  --btn-radius:   0.375rem;

  background: var(--btn-bg);
  color: var(--btn-color);
  border: 1px solid var(--btn-border);
  font-size: var(--btn-size);
  padding: var(--btn-padding);
  border-radius: var(--btn-radius);
}

/* Override tokens for variants — single selector per variant */
.btn--secondary {
  --btn-bg:     transparent;
  --btn-color:  #3b82f6;
  --btn-border: #3b82f6;
}

.btn--danger {
  --btn-bg:    #ef4444;
}

.btn--lg {
  --btn-size:    var(--text-base);
  --btn-padding: var(--space-3) var(--space-6);
}
```

## CSS Houdini

CSS Houdini exposes the browser's CSS engine internals to JavaScript, enabling custom layout algorithms, custom paint effects, and typed custom properties:

```js
// Worklet registration (served over HTTPS)
CSS.paintWorklet.addModule('/worklets/polka-dot.js');
```

```css
/* Use the custom paint worklet */
.polka-background {
  background: paint(polka-dot);
  --dot-spacing: 20px;
  --dot-radius: 4px;
  --dot-color: #3b82f6;
}
```

:::tip[`@property` is Part of Houdini]
`@property` (Typed OM — CSS Properties and Values API Level 1) is a Houdini API that is now broadly supported without flags. It is the safest and most practical Houdini feature to use in production today.
:::

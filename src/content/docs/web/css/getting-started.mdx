This guide covers the fundamental concepts you need to write CSS confidently — from linking a stylesheet to understanding why one rule beats another.

## Linking CSS to HTML

There are three ways to apply CSS to an HTML document:

<Tabs>
  <TabItem label="External Stylesheet (Recommended)">
    ```html
    <head>
      <link rel="stylesheet" href="/styles/main.css" />
    </head>
    ```

    The `<link>` tag in `<head>` is the standard approach. The stylesheet is:
    - Cached by browsers across pages
    - Separated from HTML (maintainable)
    - Loaded in parallel with other resources

    Multiple stylesheets are fine — they cascade together:
    ```html
    <link rel="stylesheet" href="/styles/reset.css" />
    <link rel="stylesheet" href="/styles/tokens.css" />
    <link rel="stylesheet" href="/styles/main.css" />
    <link rel="stylesheet" href="/styles/components.css" />
    ```
  </TabItem>
  <TabItem label="Internal Style Block">
    ```html
    <head>
      <style>
        body {
          font-family: sans-serif;
          margin: 0;
        }

        h1 {
          color: #3b82f6;
        }
      </style>
    </head>
    ```

    Use for styles specific to a single page or for critical above-the-fold CSS that you want inline to avoid a render-blocking stylesheet request.
  </TabItem>
  <TabItem label="Inline Styles">
    ```html
    <!-- Applied directly to an element via the style attribute -->
    <p style="color: red; font-weight: bold;">Inline styled text</p>
    ```

    Inline styles have the highest specificity (short of `!important`) and are hard to maintain. Reserve them for:
    - Dynamic values set by JavaScript
    - CSS custom property values set by JavaScript
    - Email HTML (limited CSS support)
  </TabItem>
</Tabs>

## The Cascade Order

When multiple rules target the same element and property, the browser must choose one. The cascade resolves conflicts in this priority order (highest wins):

```
1. Transition declarations
2. Important user-agent declarations
3. Important user declarations
4. Important author (@layer, non-layered, inline)
5. Animations
6. Normal author declarations (non-layered, then @layer in reverse order)
7. Normal user declarations
8. Normal user-agent declarations
```

For practical day-to-day CSS, remember: **specificity beats source order** within author styles of the same importance level.

## Specificity Calculation

Specificity is a four-part score: **`(inline, ID, class/attr/pseudo-class, element/pseudo-element)`**

```css
/* Specificity: (0, 0, 0, 0) — no specificity */
* { color: red; }

/* Specificity: (0, 0, 0, 1) */
p { color: red; }
::before { content: ''; }

/* Specificity: (0, 0, 1, 0) */
.warning { color: orange; }
[type="text"] { border: 1px solid; }
:hover { opacity: 0.8; }
:nth-child(2) { font-style: italic; }

/* Specificity: (0, 1, 0, 0) */
#header { background: navy; }

/* Specificity: (0, 0, 1, 1) — class + element */
p.intro { font-size: 1.1rem; }

/* Specificity: (0, 1, 1, 0) — ID + class */
#nav .link { text-decoration: none; }

/* Specificity: (0, 0, 2, 1) — two classes + element */
ul.list li.active { color: green; }

/* Specificity: (1, 0, 0, 0) — inline style */
/* <p style="color: blue;"> — always beats any selector */
```

### Specificity Gotchas

```css
/* :is(), :not(), :has() take the specificity of their most specific argument */
:is(#id, .class) { } /* specificity of #id = (0, 1, 0, 0) */
:not(.class) { }     /* specificity of .class = (0, 0, 1, 0) */

/* :where() always has zero specificity — great for resets */
:where(h1, h2, h3) { margin-block: 0; } /* (0, 0, 0, 0) — easily overridden */
```

## `!important`

`!important` escalates a declaration to override any normal declaration:

```css
/* Avoid this — creates specificity arms races */
.button {
  color: blue !important;
}

/* When !important is acceptable */
.sr-only {
  /* Screen-reader-only utility must NEVER be visible regardless of component styles */
  position: absolute !important;
  width: 1px !important;
  height: 1px !important;
  overflow: hidden !important;
  clip: rect(0, 0, 0, 0) !important;
}
```

:::caution[`!important` is a Maintenance Trap]
Once you use `!important` you create a problem that can only be solved by another `!important`. Instead, fix specificity — use lower-specificity selectors or restructure your CSS architecture.
:::

## Inheritance Keywords

These special values work on any CSS property:

```css
.child {
  /* inherit: use the computed value from the parent element */
  color: inherit;

  /* initial: reset to the property's CSS specification default */
  /* (note: this is NOT the browser stylesheet default — it's the spec initial) */
  font-size: initial;  /* -> 'medium' */
  display: initial;    /* -> 'inline' */

  /* unset: inherit if the property is inheritable, otherwise initial */
  border: unset;       /* -> initial (border doesn't inherit) */
  color: unset;        /* -> inherit (color does inherit) */

  /* revert: rolls back to the browser's stylesheet (user-agent) default */
  /* This is usually what you want when "resetting" */
  display: revert;     /* -> 'block' for div, 'inline' for span, etc. */

  /* revert-layer: rolls back to the previous @layer */
  color: revert-layer;

  /* all: applies the chosen value to every property at once */
  all: revert;         /* Nuclear option: reset everything to browser defaults */
}
```

## Colour Values

CSS supports many colour formats:

```css
.element {
  /* Named colours (148 keywords) */
  color: red;
  color: rebeccapurple;
  color: transparent;

  /* Hex — 3, 6, or 8 characters (#RRGGBBAA) */
  color: #fff;           /* short for #ffffff */
  color: #3b82f6;
  color: #3b82f680;      /* with 50% alpha */

  /* RGB */
  color: rgb(59, 130, 246);
  color: rgb(59 130 246);        /* modern space-separated syntax */
  color: rgb(59 130 246 / 0.5);  /* with alpha */

  /* HSL — Hue (0–360°), Saturation (%), Lightness (%) */
  color: hsl(217, 91%, 60%);
  color: hsl(217 91% 60%);
  color: hsl(217 91% 60% / 0.5); /* with alpha */

  /* oklch — perceptually uniform (recommended for modern CSS) */
  /* Lightness (0–1), Chroma (0–0.4), Hue (0–360°) */
  color: oklch(0.6 0.15 250);
  color: oklch(0.6 0.15 250 / 0.5);

  /* currentColor — inherits the current text colour */
  border-color: currentColor;
  fill: currentColor;

  /* CSS custom property (variable) */
  color: var(--color-primary);
}
```

### When to Use Each Format

| Format | Best For |
|---|---|
| Named | Quick prototyping, `transparent`, `currentColor` |
| Hex | Design system tokens, copy-paste from design tools |
| `hsl()` | Manual colour adjustments — easy to reason about lightness/saturation |
| `oklch()` | Modern designs — perceptually uniform, supports wide-gamut displays |
| `rgb()` | Programmatic manipulation, interop with JS |

## Length Units

### Absolute Units

```css
.element {
  /* px — device-independent pixels (96px = 1 inch on screen) */
  width: 300px;
  font-size: 16px;

  /* Rarely used in screen CSS: pt, pc, in, cm, mm */
  font-size: 12pt; /* Print stylesheets */
}
```

### Relative Units

```css
.parent {
  font-size: 20px;
}

.child {
  /* em — relative to the element's own font-size */
  /* Use for padding/margin that should scale with text */
  padding: 0.5em;   /* 0.5 × 20px = 10px */

  /* rem — relative to the root element (<html>) font-size */
  /* Use for consistent sizing across components */
  font-size: 1.5rem; /* 1.5 × 16px (root default) = 24px */

  /* ch — width of the '0' character in the current font */
  max-width: 65ch;   /* Ideal line length for readability */

  /* ex — height of the 'x' character */
  line-height: 2ex;
}

.viewport {
  /* vh / vw — 1% of viewport height / width */
  height: 100vh;   /* Full viewport height */
  width: 50vw;     /* Half viewport width */

  /* vmin / vmax — 1% of the smaller / larger viewport dimension */
  font-size: 4vmin;

  /* svh/dvh/lvh — small/dynamic/large viewport height (mobile-aware) */
  min-height: 100svh; /* Safe for mobile (excludes browser chrome) */
  min-height: 100dvh; /* Dynamic — adjusts as browser chrome shows/hides */

  /* cqi / cqb — container query inline/block (see Responsive page) */
}
```

## Mathematical Functions

### `calc()`

Perform calculations mixing different units:

```css
.element {
  /* Mix units — impossible without calc() */
  width: calc(100% - 2rem);
  height: calc(100vh - 64px);

  /* Nested calc() */
  font-size: calc(1rem + 0.5 * (100vw - 320px) / (1280 - 320));

  /* CSS custom properties in calc() */
  padding: calc(var(--spacing-base) * 2);
}
```

### `clamp()`

Constrain a value between a minimum and maximum, with a preferred fluid value in between:

```css
/* clamp(minimum, preferred, maximum) */
.element {
  /* Font size between 16px and 28px, fluid in between */
  font-size: clamp(1rem, 2.5vw, 1.75rem);

  /* Padding between 1rem and 3rem */
  padding: clamp(1rem, 5%, 3rem);

  /* Container width that's at least 320px, max 1200px, ideal 90% */
  width: clamp(320px, 90%, 1200px);
}
```

### `min()` and `max()`

```css
.element {
  /* min(): use whichever is smallest */
  /* "100% OR 600px, whichever is smaller" */
  width: min(100%, 600px);

  /* max(): use whichever is largest */
  /* "At least 2rem, or 4vw if that's bigger" */
  font-size: max(2rem, 4vw);

  /* Both work with any number of arguments */
  width: min(50%, 400px, 100vw - 2rem);
}
```

:::tip[`clamp()` = `max(min, min(preferred, max))`]
`clamp(a, b, c)` is mathematically equivalent to `max(a, min(b, c))`. The `clamp()` syntax is just more readable. All three — `calc()`, `min()`, and `max()` — can be nested inside each other.
:::

CSS (Cascading Style Sheets) brings visual design to HTML. This guide covers how to apply CSS to your pages, how the cascade resolves conflicts, and the full range of colour values and units available.

## Linking CSS to HTML

There are three ways to add CSS to a page:

<Tabs>
<TabItem label="External Stylesheet (Preferred)">

```html
<!-- In <head> — the preferred approach -->
<link rel="stylesheet" href="styles.css" />

<!-- Preload critical CSS for performance -->
<link rel="preload" href="critical.css" as="style" onload="this.onload=null;this.rel='stylesheet'" />
```

```css
/* styles.css */
body {
  margin: 0;
  font-family: system-ui, sans-serif;
}
```

✅ Cached by browsers — only downloaded once across multiple pages  
✅ Separation of concerns — styles separate from markup  
✅ Easy to maintain and share across pages  

</TabItem>
<TabItem label="Internal Style Block">

```html
<!-- In <head> — useful for page-specific critical CSS -->
<style>
  /* Scoped to this page only */
  .hero {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    min-height: 80vh;
  }
</style>
```

✅ No extra HTTP request  
❌ Not cached separately  
❌ Increases HTML document size  

</TabItem>
<TabItem label="Inline Styles">

```html
<!-- Applied directly to an element -->
<p style="color: #333; font-size: 1.125rem; margin-bottom: 1rem;">
  This paragraph has inline styles.
</p>
```

❌ Highest specificity — hard to override  
❌ Unmaintainable at scale  
✅ Only acceptable for dynamically calculated values set via JavaScript  

</TabItem>
</Tabs>

## Cascade Order

When multiple rules target the same element and property, the cascade determines the winner. Rules are evaluated in this priority order (higher wins):

1. **User agent stylesheets** — browser defaults (lowest)
2. **User stylesheets** — operating system or browser user preferences
3. **Author stylesheets** — your CSS
4. **Author `!important`** declarations
5. **User `!important`** declarations
6. **User agent `!important`** declarations (highest)

Within the same origin, specificity determines the winner. When specificity is equal, the **later rule wins**.

```css
/* In your stylesheet — source order matters when specificity is equal */
.btn { color: blue; }   /* defined first */
.btn { color: red; }    /* defined second — this wins */
```

## Specificity Calculation

Specificity is a four-number score: `(A, B, C, D)` where:
- **A** = Inline styles (`style=""`)
- **B** = ID selectors (`#id`)
- **C** = Class (`.class`), attribute (`[type]`), pseudo-class (`:hover`)
- **D** = Type (`p`, `div`), pseudo-element (`::before`)

```css
/*                              A  B  C  D */
*                           /* 0, 0, 0, 0 */
p                           /* 0, 0, 0, 1 */
.intro                      /* 0, 0, 1, 0 */
p.intro                     /* 0, 0, 1, 1 */
#sidebar                    /* 0, 1, 0, 0 */
#sidebar .widget            /* 0, 1, 1, 0 */
#sidebar .widget p          /* 0, 1, 1, 1 */
style="color: red"          /* 1, 0, 0, 0 */
```

Compare scores left-to-right — first difference determines the winner:

```css
/* Who wins? */
#nav .link         /* 0,1,1,0 */
.nav a.link        /* 0,0,2,1 */
/* Answer: #nav .link wins — 1 > 0 in the ID column */
```

:::tip[Keep Specificity Low]
High-specificity selectors are hard to override without escalating to even higher specificity or `!important`. Prefer class selectors (0-0-1-0) over ID selectors and avoid nesting more than 3 levels deep.
:::

## `!important`

`!important` overrides the normal specificity calculation:

```css
.button {
  color: white !important; /* Overrides even inline styles */
}
```

:::caution[Avoid !important Except as a Last Resort]
`!important` breaks the natural cascade, making debugging extremely difficult. Valid uses are limited to:
- Utility classes that must always apply (e.g., `.hidden { display: none !important }`)
- Overriding third-party stylesheet styles you can't change
- User stylesheets for accessibility overrides
:::

## Inheritance Control

```css
.parent {
  color: #333;
  font-size: 1rem;
  border: 1px solid #ccc; /* NOT inherited */
}

.child {
  /* Explicitly inherit any property, even non-inherited ones */
  color: inherit;

  /* Reset to the browser's default value for this property */
  font-size: initial;

  /* If inherited property: acts like inherit. If not: acts like initial. */
  font-weight: unset;

  /* Rolls back to the browser/user stylesheet value (CSS Cascading Level 4) */
  color: revert;

  /* Apply all inheritance behaviours at once */
  all: inherit;   /* Inherit everything from parent */
  all: initial;   /* Reset everything to browser defaults */
  all: unset;     /* Inherit inherited props, reset non-inherited */
}
```

## Colour Values

CSS supports many ways to express colour:

<Tabs>
<TabItem label="Named & Hex">

```css
.box {
  /* Named colours (140 standard names) */
  color: tomato;
  background: cornflowerblue;

  /* 6-digit hex */
  color: #e74c3c;

  /* 3-digit shorthand (equivalent to #ff6633) */
  color: #f63;

  /* 8-digit hex with alpha channel */
  color: #e74c3ccc; /* cc = 80% opacity */

  /* 4-digit hex with alpha */
  color: #f63c; /* 80% opacity */
}
```

</TabItem>
<TabItem label="RGB & HSL">

```css
.box {
  /* RGB (0-255 each channel) */
  color: rgb(231, 76, 60);

  /* RGB with alpha (0-1) */
  color: rgba(231, 76, 60, 0.8);

  /* Modern space-separated syntax (no comma) */
  color: rgb(231 76 60);
  color: rgb(231 76 60 / 80%);

  /* HSL: Hue(0-360), Saturation(%), Lightness(%) */
  color: hsl(5, 78%, 57%);
  color: hsla(5, 78%, 57%, 0.8);

  /* Modern HSL */
  color: hsl(5 78% 57%);
  color: hsl(5 78% 57% / 80%);
}
```

</TabItem>
<TabItem label="oklch (Modern)">

```css
.box {
  /*
    oklch(lightness chroma hue)
    - Perceptually uniform: same lightness = visually same brightness
    - Can express P3 wide-gamut colours
    - Great for generating accessible colour scales
  */

  /* L: 0-1, C: 0-0.4, H: 0-360 */
  color: oklch(0.65 0.22 27);           /* A warm red */
  color: oklch(0.70 0.15 250);          /* A cool blue */
  color: oklch(0.92 0.05 100);          /* A light yellow */

  /* With alpha */
  color: oklch(0.65 0.22 27 / 0.8);

  /* Relative colour syntax (CSS Color Level 5) */
  /* Derive a lighter version from a custom property */
  background: oklch(from var(--primary-color) calc(l + 0.2) c h);
}
```

</TabItem>
</Tabs>

## CSS Units

Choosing the right unit is crucial for responsive, accessible design.

### Absolute Units

| Unit | Equivalent | Use case |
|---|---|---|
| `px` | 1/96 of 1 inch | Borders, shadows, fine details |
| `pt` | 1/72 of 1 inch | Print stylesheets only |
| `cm`, `mm`, `in` | Physical units | Print stylesheets only |

### Relative Units

```css
.example {
  /* em: relative to the element's own font-size */
  padding: 0.75em;         /* 12px if font-size is 16px */
  font-size: 1.25em;       /* 20px if parent is 16px */

  /* rem: relative to the root <html> font-size */
  margin: 1.5rem;          /* 24px if root is 16px */
  font-size: 1.125rem;     /* 18px — unaffected by parent nesting */

  /* Viewport units */
  width: 50vw;             /* 50% of viewport width */
  height: 100vh;           /* 100% of viewport height */
  font-size: 4vmin;        /* 4% of the smaller viewport dimension */
  font-size: 4vmax;        /* 4% of the larger viewport dimension */

  /* Character units */
  max-width: 65ch;         /* 65 character-widths — optimal reading width */
  height: 2ex;             /* 2× the height of the 'x' character */

  /* Dynamic/Small/Large viewport units (modern) */
  min-height: 100dvh;      /* Dynamic viewport height (excludes mobile browser chrome) */
  min-height: 100svh;      /* Small viewport (minimum size — mobile browser chrome visible) */
  min-height: 100lvh;      /* Large viewport (maximum size — no browser chrome) */
}
```

:::tip[Use rem for Typography and Spacing]
`rem` units respect user font-size preferences set in the browser. If a user increases their default font size from 16px to 20px, all `rem`-based sizes scale accordingly — this is essential for accessibility. Use `rem` for font sizes, spacing, and most layout values. Reserve `px` for values that should never scale (borders, shadows).
:::

## CSS Math Functions

### `calc()`

```css
.sidebar {
  /* Mix units freely */
  width: calc(100% - 2rem);
  margin-top: calc(4rem + 10px);
  font-size: calc(1rem + 0.5vw);

  /* Nested calc (parentheses clarify intent) */
  padding: calc((100vw - 1200px) / 2);

  /* Use with custom properties */
  --sidebar-width: 280px;
  width: calc(100% - var(--sidebar-width) - 2rem);
}
```

### `clamp()`

Clamps a value between a minimum and maximum, with a preferred middle value:

```css
.prose {
  /*
    clamp(min, preferred, max)
    The preferred value grows with the viewport but stays within min/max
  */

  /* Font size: min 1rem, fluid, max 1.5rem */
  font-size: clamp(1rem, 2.5vw, 1.5rem);

  /* Container width: min 320px, 90% of viewport, max 1200px */
  max-width: clamp(320px, 90%, 1200px);

  /* Spacing that scales with viewport */
  padding: clamp(1rem, 5vw, 4rem);
}
```

### `min()` and `max()`

```css
.responsive {
  /* min(): uses the smallest value — like a maximum constraint */
  width: min(100%, 1200px);            /* Full width, but never wider than 1200px */
  font-size: min(5vw, 2rem);           /* Fluid, but caps at 2rem */

  /* max(): uses the largest value — like a minimum constraint */
  padding: max(1rem, 3vw);             /* At least 1rem padding */
  height: max(200px, 30vh);            /* At least 200px tall */

  /* Combine for complex constraints */
  font-size: clamp(1rem, max(2vw, 1.25rem), 2rem);
}
```

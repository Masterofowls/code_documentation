**CSS (Cascading Style Sheets)** is the language that controls the visual presentation of web pages. It works alongside HTML (structure) and JavaScript (behaviour) to deliver the complete web experience.

## What CSS Does

CSS controls every visual aspect of a page:

- **Layout** — how elements are positioned and sized (Flexbox, Grid, position)
- **Typography** — fonts, sizes, line heights, spacing
- **Colour and backgrounds** — colours, gradients, images
- **Animation** — transitions and keyframe animations
- **Responsive design** — adapting layouts to different screen sizes
- **Theming** — consistent design tokens, dark mode

```css
/* A simple example showing what CSS can do */
.card {
  /* Layout */
  display: flex;
  flex-direction: column;
  gap: 1rem;
  padding: 1.5rem;

  /* Visual */
  background: #fff;
  border: 1px solid #e2e8f0;
  border-radius: 0.75rem;
  box-shadow: 0 4px 6px -1px rgb(0 0 0 / 0.1);

  /* Typography */
  font-family: 'Inter', sans-serif;
  color: #1e293b;

  /* Animation */
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.card:hover {
  transform: translateY(-2px);
  box-shadow: 0 10px 15px -3px rgb(0 0 0 / 0.1);
}
```

## The Three Pillars: Cascade, Specificity, Inheritance

These three concepts govern how CSS rules are applied and are the source of most CSS confusion.

### The Cascade

The **cascade** is the algorithm that determines which CSS rule wins when multiple rules target the same element and property. It evaluates in this order:

1. **Origin and importance** — user agent styles → user styles → author styles → `!important` declarations (reversed)
2. **Specificity** — how specific the selector is
3. **Order of appearance** — later rules win over earlier ones (when specificity is equal)

```css
/* These all target the same element — which wins? */
.button { color: blue; }        /* specificity: 0-1-0 */
.nav .button { color: red; }    /* specificity: 0-2-0 — wins! */
button { color: green; }        /* specificity: 0-0-1 — loses */
```

### Specificity

Specificity is calculated as a four-part score: `(inline, ID, class, element)`

| Selector type | Score |
|---|---|
| Inline style (`style="..."`) | `1-0-0-0` |
| ID (`#id`) | `0-1-0-0` |
| Class (`.class`), attribute (`[attr]`), pseudo-class (`:hover`) | `0-0-1-0` |
| Element (`p`, `div`), pseudo-element (`::before`) | `0-0-0-1` |
| Universal selector (`*`), combinators, `:where()` | `0-0-0-0` |

```css
p                   /* 0-0-0-1 */
.intro              /* 0-0-1-0 */
p.intro             /* 0-0-1-1 */
#hero               /* 0-1-0-0 */
#hero p.intro       /* 0-1-1-1 */
style=""            /* 1-0-0-0 */
```

### Inheritance

Some CSS properties are **inherited** by default — children receive the parent's value unless overridden. Others are **not inherited** — each element gets the initial value.

```css
/* Inherited by default: */
/* color, font-*, line-height, text-*, list-style, cursor, visibility */

/* Not inherited by default: */
/* background, border, margin, padding, width, height, display, position */

.parent {
  color: #333;       /* children inherit this */
  border: 1px solid; /* children do NOT inherit this */
}

.child {
  color: inherit;    /* explicitly inherit any property */
  color: initial;    /* reset to browser default */
  color: unset;      /* inherit if inherited property, else initial */
  color: revert;     /* reset to browser/user stylesheet value */
}
```

## CSS3 Features Highlights

CSS3 introduced a massive leap in capabilities:

| Feature | Description |
|---|---|
| **Flexbox** | One-dimensional layout model |
| **Grid** | Two-dimensional layout model |
| **Custom Properties** | CSS variables with the cascade |
| **Animations & Transitions** | Native CSS animations |
| **Media Queries** | Responsive design breakpoints |
| **Transforms** | 2D/3D element transformations |
| **Gradients** | Linear, radial, conic gradients |
| **Shadows** | `box-shadow`, `text-shadow` |
| **Border Radius** | Rounded corners |
| **Multi-column layout** | Magazine-style column flow |
| **Calc & Functions** | `calc()`, `clamp()`, `min()`, `max()` |

## The Modern CSS Landscape

### Preprocessors

CSS preprocessors extend CSS with variables, nesting, mixins, and functions. They compile to standard CSS.

| Preprocessor | Extension | Key features |
|---|---|---|
| **Sass/SCSS** | `.scss` / `.sass` | Variables, nesting, mixins, functions, partials |
| **Less** | `.less` | Similar to Sass, JS-based |
| **Stylus** | `.styl` | Very flexible syntax |

```scss
// SCSS example
$primary: #0066cc;
$radius: 0.5rem;

@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

.card {
  border-radius: $radius;
  padding: 1.5rem;

  &:hover {
    background: darken($primary, 10%);
  }

  .card-title {
    color: $primary;
    font-size: 1.25rem;
  }
}
```

:::tip[Native CSS Has Caught Up]
CSS custom properties (variables), nesting (now natively supported), and container queries have reduced the need for preprocessors significantly. For new projects, evaluate whether you actually need a preprocessor.
:::

### CSS-in-JS

CSS-in-JS libraries write styles directly in JavaScript/TypeScript, with styles scoped to components:

```js
// styled-components (React)
import styled from 'styled-components';

const Button = styled.button`
  background: ${(props) => props.primary ? '#0066cc' : '#fff'};
  color: ${(props) => props.primary ? '#fff' : '#0066cc'};
  padding: 0.5rem 1.5rem;
  border: 2px solid #0066cc;
  border-radius: 0.375rem;
  cursor: pointer;
  transition: all 0.2s;

  &:hover {
    transform: translateY(-1px);
  }
`;

// Usage
<Button primary>Primary Action</Button>
<Button>Secondary Action</Button>
```

Popular CSS-in-JS libraries: **styled-components**, **Emotion**, **vanilla-extract**, **Stitches**, **Panda CSS**.

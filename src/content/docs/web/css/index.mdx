CSS (**Cascading Style Sheets**) is the language used to describe how HTML elements are presented on screen — their layout, typography, colours, animations, and more. CSS works alongside HTML (structure) and JavaScript (behaviour) as one of the three core technologies of the web.

## What CSS Does

Every visual aspect of a webpage is controlled by CSS:

- **Layout** — Flexbox, CSS Grid, positioning, multi-column
- **Typography** — fonts, sizes, line-height, spacing, direction
- **Colour & backgrounds** — fills, gradients, images
- **Spacing** — margins, padding, gaps
- **Animation** — transitions, keyframe animations, scroll-driven effects
- **Responsive design** — media queries, container queries, fluid sizing
- **Visual effects** — shadows, borders, transforms, filters, blend modes

## The Cascade

The "Cascading" in CSS describes how the browser resolves conflicts when multiple rules target the same element. The cascade considers three things, in order of priority:

<Steps>
1. **Origin and importance**

   Styles from different sources are weighted in this order (lowest to highest):
   - User-agent (browser default) styles
   - User styles (accessibility settings)
   - Author styles (your CSS)
   - Author `!important` styles
   - User `!important` styles
   - Browser `!important` styles

2. **Specificity**

   When origin and importance are equal, the rule with higher specificity wins. See the [Getting Started](/web/css/getting-started) page for the full specificity algorithm.

3. **Source order**

   When specificity is also equal, the **last declared rule** wins.
</Steps>

```css
/* Source order: the second rule wins */
p { color: red; }
p { color: blue; } /* This one is used */

/* Specificity: the more specific rule wins regardless of order */
p { color: red; }          /* specificity 0-0-0-1 */
.intro { color: blue; }    /* specificity 0-0-1-0 — this wins */
```

## Specificity

Specificity is calculated as a four-part score: `(inline, id, class/attr/pseudo-class, element/pseudo-element)`.

| Selector | Score |
|---|---|
| `style="..."` (inline) | `1-0-0-0` |
| `#header` | `0-1-0-0` |
| `.nav-link` | `0-0-1-0` |
| `[type="text"]` | `0-0-1-0` |
| `:hover` | `0-0-1-0` |
| `div` | `0-0-0-1` |
| `::before` | `0-0-0-1` |
| `*` | `0-0-0-0` |
| `#nav .link:hover` | `0-1-2-0` |
| `ul li.active a` | `0-0-1-3` |

## Inheritance

Some CSS properties are inherited by child elements; others are not. Understanding which properties inherit saves you a lot of repetitive declarations.

```css
/* Inherited properties — set once on parent, children inherit automatically */
body {
  font-family: 'Inter', sans-serif; /* all text inherits this */
  font-size: 1rem;
  line-height: 1.6;
  color: #374151;
}

/* Non-inherited properties — must set explicitly on each element */
.box {
  border: 1px solid red;    /* children do NOT get this border */
  background: yellow;       /* children do NOT get this background */
  padding: 1rem;            /* children do NOT get this padding */
  width: 300px;             /* children do NOT get this width */
}
```

**Inherited:** `color`, `font-*`, `line-height`, `text-align`, `text-transform`, `letter-spacing`, `word-spacing`, `cursor`, `visibility`, `list-style-*`, `direction`

**Not inherited:** `background`, `border`, `padding`, `margin`, `width`, `height`, `display`, `position`, `overflow`, `box-shadow`

### Inheritance Keywords

```css
.child {
  color: inherit;  /* Explicitly inherit from parent */
  border: initial; /* Reset to browser default */
  font-size: unset; /* Inherit if inheritable, otherwise initial */
  display: revert; /* Reset to browser stylesheet value */
  all: revert;     /* Reset ALL properties to browser defaults */
}
```

## CSS3 Features

CSS3 is not a single release but a collection of modules. Here are the most impactful additions:

| Module | Key Features |
|---|---|
| Selectors Level 3 | `:not()`, `~`, attribute selectors |
| Selectors Level 4 | `:is()`, `:where()`, `:has()` |
| Flexbox | One-dimensional layout |
| CSS Grid | Two-dimensional layout |
| Custom Properties | `--variable` / `var()` |
| Animations | `@keyframes`, `animation` |
| Transitions | `transition` |
| Transforms | `translate`, `rotate`, `scale` |
| Media Queries | `@media` responsive breakpoints |
| Container Queries | `@container` component-level responsiveness |
| `calc()` | Math in property values |
| `clamp()` | Fluid sizing between min and max |
| Colour Spaces | `oklch()`, `lab()`, `display-p3` |
| Cascade Layers | `@layer` — explicit cascade management |

## Preprocessors

CSS preprocessors extend CSS with programming features and compile down to standard CSS:

<Tabs>
  <TabItem label="Sass / SCSS">
    ```css
    /* Variables */
    $primary: #3b82f6;
    $spacing-base: 1rem;

    /* Nesting */
    .card {
      padding: $spacing-base;

      &:hover {
        transform: translateY(-2px);
      }

      &__title {
        font-size: 1.25rem;
      }
    }

    /* Mixins */
    @mixin flex-center {
      display: flex;
      align-items: center;
      justify-content: center;
    }

    .hero {
      @include flex-center;
    }
    ```
  </TabItem>
  <TabItem label="PostCSS">
    PostCSS is a tool that transforms CSS with JavaScript plugins. Popular plugins:

    - **Autoprefixer** — adds vendor prefixes automatically
    - **cssnano** — minifies CSS
    - **postcss-preset-env** — use future CSS syntax today
    - **postcss-import** — inline `@import` rules

    ```css
    /* Input (with postcss-preset-env) */
    .element {
      background: oklch(70% 0.15 200);
      container-type: inline-size;
    }

    /* Output (after PostCSS with Autoprefixer) */
    .element {
      background: rgb(50 150 200); /* fallback */
      background: oklch(70% 0.15 200);
      container-type: inline-size;
    }
    ```
  </TabItem>
</Tabs>

## CSS-in-JS

CSS-in-JS libraries let you write CSS inside JavaScript files, scoped to components:

```js
// Styled Components (React)
import styled from 'styled-components';

const Button = styled.button`
  background: ${props => props.primary ? '#3b82f6' : 'transparent'};
  color: ${props => props.primary ? 'white' : '#3b82f6'};
  padding: 0.5rem 1rem;
  border-radius: 0.25rem;
  border: 2px solid #3b82f6;

  &:hover {
    opacity: 0.9;
  }
`;

// Usage
<Button primary>Primary</Button>
<Button>Secondary</Button>
```

Popular CSS-in-JS solutions: **Styled Components**, **Emotion**, **vanilla-extract**, **Linaria**, **CSS Modules** (semi-CSS-in-JS).

:::tip[Modern Native CSS Often Replaces Preprocessors]
CSS Custom Properties (`var()`), nesting (now native in CSS), `calc()`, `clamp()`, and container queries solve many problems that used to require Sass or PostCSS. Consider whether you truly need a preprocessor in new projects.
:::

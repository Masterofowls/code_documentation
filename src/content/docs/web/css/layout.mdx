Layout is the process of determining the size and position of every element on the page. CSS offers multiple layout models, each suited to different design requirements.

## Normal Flow

In **normal flow** (the default), block elements stack vertically and inline elements flow horizontally within their containers.

```css
/* Block elements in normal flow */
div, p, h1, ul, section {
  display: block; /* Default for these elements */
  /* - Stacks vertically */
  /* - Width fills the parent */
  /* - Height determined by content */
}

/* Inline elements in normal flow */
span, a, strong, em {
  display: inline;
  /* - Flows horizontally with text */
  /* - Width and height are ignored */
}

/* inline-block: inline flow, but respects width/height/margin */
.badge {
  display: inline-block;
  padding: 0.25rem 0.5rem;
  border-radius: 9999px;
}
```

## Flexbox

Flexbox is a **one-dimensional** layout model — it arranges items either in a row or a column.

<Tabs>
  <TabItem label="Container Properties">
    ```css
    .flex-container {
      display: flex;             /* or inline-flex */

      /* Direction of the main axis */
      flex-direction: row;       /* → row (default) */
      flex-direction: row-reverse; /* ← reverse row */
      flex-direction: column;    /* ↓ column */
      flex-direction: column-reverse; /* ↑ reverse column */

      /* Wrapping */
      flex-wrap: nowrap;         /* All items on one line (default) */
      flex-wrap: wrap;           /* Items wrap to next line */
      flex-wrap: wrap-reverse;   /* Items wrap upward */

      /* Shorthand for flex-direction + flex-wrap */
      flex-flow: row wrap;

      /* ─── Alignment ──────────────────────────────────────────────── */

      /* justify-content: alignment along the MAIN axis */
      justify-content: flex-start;    /* Pack items at start (default) */
      justify-content: flex-end;      /* Pack items at end */
      justify-content: center;        /* Center items */
      justify-content: space-between; /* First at start, last at end, space between */
      justify-content: space-around;  /* Equal space around each item */
      justify-content: space-evenly;  /* Equal space between all gaps */

      /* align-items: alignment along the CROSS axis (single line) */
      align-items: stretch;           /* Fill cross-axis size (default) */
      align-items: flex-start;        /* Align to start of cross axis */
      align-items: flex-end;          /* Align to end of cross axis */
      align-items: center;            /* Center on cross axis */
      align-items: baseline;          /* Align by text baseline */

      /* align-content: alignment of WRAPPED lines (multi-line only) */
      align-content: flex-start;
      align-content: center;
      align-content: space-between;
      align-content: stretch;

      /* gap: space between flex items */
      gap: 1rem;           /* Same gap in both directions */
      gap: 1rem 2rem;      /* row-gap column-gap */
      row-gap: 1rem;
      column-gap: 2rem;
    }
    ```
  </TabItem>
  <TabItem label="Item Properties">
    ```css
    .flex-item {
      /* flex-grow: how much the item GROWS relative to siblings */
      /* 0 = don't grow (default), 1 = grow equally, 2 = grow twice as fast */
      flex-grow: 0;
      flex-grow: 1;

      /* flex-shrink: how much the item SHRINKS when space is tight */
      /* 1 = shrink equally (default), 0 = never shrink */
      flex-shrink: 1;
      flex-shrink: 0; /* Use with min-width to prevent squishing */

      /* flex-basis: the item's ideal size BEFORE grow/shrink apply */
      flex-basis: auto;    /* Size from width/height (default) */
      flex-basis: 0;       /* Start from 0, only grow */
      flex-basis: 200px;   /* Fixed starting size */
      flex-basis: 33.33%;  /* Percentage of container */

      /* flex shorthand: grow | shrink | basis */
      flex: 0 1 auto;      /* Default: don't grow, do shrink, auto basis */
      flex: 1;             /* = flex: 1 1 0 — grows and shrinks equally */
      flex: 1 1 200px;     /* Grows and shrinks from a 200px base */
      flex: none;          /* = flex: 0 0 auto — rigid, no grow or shrink */

      /* align-self: override container's align-items for this item */
      align-self: auto;         /* Use container's align-items */
      align-self: flex-start;
      align-self: center;
      align-self: stretch;

      /* order: change visual order without changing DOM order */
      /* Default is 0; lower numbers appear first */
      order: -1;   /* Move to the start */
      order: 1;    /* Move toward the end */
    }
    ```
  </TabItem>
  <TabItem label="Common Patterns">
    ```css
    /* ─── Perfect centering ──────────────────────────────────────────────── */
    .center {
      display: flex;
      align-items: center;
      justify-content: center;
    }

    /* ─── Sticky footer ───────────────────────────────────────────────────── */
    body {
      display: flex;
      flex-direction: column;
      min-height: 100vh;
    }
    main { flex: 1; } /* Main content grows to fill available space */

    /* ─── Holy grail layout (header, [aside, main, aside], footer) ───────── */
    .holy-grail {
      display: flex;
      flex-direction: column;
      min-height: 100vh;
    }
    .holy-grail__content {
      display: flex;
      flex: 1;
    }
    .holy-grail__nav   { flex: 0 0 200px; }
    .holy-grail__main  { flex: 1; }
    .holy-grail__aside { flex: 0 0 160px; }

    /* ─── Equal-height cards ─────────────────────────────────────────────── */
    .card-grid {
      display: flex;
      flex-wrap: wrap;
      gap: 1.5rem;
    }
    .card-grid > * {
      flex: 1 1 280px; /* Grows, shrinks, minimum 280px width */
    }
    ```
  </TabItem>
</Tabs>

## CSS Grid

CSS Grid is a **two-dimensional** layout model — it arranges items in both rows and columns simultaneously.

<Tabs>
  <TabItem label="Defining Grids">
    ```css
    .grid-container {
      display: grid;

      /* ─── Columns and rows ─────────────────────────────────────────── */
      grid-template-columns: 200px 1fr 1fr;     /* Fixed + flexible */
      grid-template-columns: repeat(3, 1fr);    /* 3 equal columns */
      grid-template-columns: repeat(12, 1fr);   /* 12-column grid */

      /* minmax() — set minimum and maximum size for a track */
      grid-template-columns: minmax(200px, 1fr) 2fr;

      /* Fluid columns without media queries */
      /* auto-fill: creates as many columns as fit */
      grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));

      /* auto-fit: same but collapses empty tracks */
      grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));

      grid-template-rows: auto 1fr auto; /* header, main, footer */

      /* Named areas — visual representation of the layout */
      grid-template-areas:
        "header  header  header"
        "sidebar main    main"
        "footer  footer  footer";

      /* ─── Gaps ──────────────────────────────────────────────────────── */
      gap: 1.5rem;
      row-gap: 1rem;
      column-gap: 2rem;

      /* ─── Alignment ─────────────────────────────────────────────────── */
      justify-items: stretch;  /* Horizontal alignment of items in cells */
      align-items: stretch;    /* Vertical alignment of items in cells */
      place-items: center;     /* Shorthand: align-items justify-items */

      justify-content: start;  /* Alignment of grid tracks in container */
      align-content: start;
      place-content: space-between;
    }
    ```
  </TabItem>
  <TabItem label="Placing Items">
    ```css
    /* ─── Line-based placement ────────────────────────────────────────────── */
    .item {
      /* grid-column: start-line / end-line */
      grid-column: 1 / 3;        /* Spans from column line 1 to 3 */
      grid-column: 1 / -1;       /* From start to end (full width) */
      grid-column: 2 / span 2;   /* Start at line 2, span 2 columns */
      grid-column: span 3;       /* Span 3 columns from auto placement */

      grid-row: 1 / 3;           /* From row line 1 to 3 */
      grid-row: 2 / span 2;      /* Start at row 2, span 2 rows */

      /* Shorthand: row-start / column-start / row-end / column-end */
      grid-area: 1 / 1 / 3 / 4;
    }

    /* ─── Named area placement ────────────────────────────────────────────── */
    .header  { grid-area: header; }
    .sidebar { grid-area: sidebar; }
    .main    { grid-area: main; }
    .footer  { grid-area: footer; }

    /* ─── Per-item alignment ──────────────────────────────────────────────── */
    .centered-item {
      justify-self: center;  /* Horizontal */
      align-self: center;    /* Vertical */
      place-self: center;    /* Shorthand */
    }
    ```
  </TabItem>
  <TabItem label="auto-fill vs auto-fit">
    ```css
    /* auto-fill: always creates as many columns as fit, even if empty */
    .auto-fill-grid {
      display: grid;
      grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
      /* On a 700px container: creates 3 columns (200px each = 600px)
         The 3 columns stretch to fill 700px.
         A 4th empty column would be created if the container were wider. */
    }

    /* auto-fit: empty columns collapse to 0, existing columns expand */
    .auto-fit-grid {
      display: grid;
      grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
      /* On a 700px container with 2 items: both items fill 50% each.
         Empty tracks collapse. Items expand to fill all available space. */
    }
    ```
  </TabItem>
  <TabItem label="Subgrid">
    ```css
    /* subgrid — inherit parent grid's tracks in nested grid */
    .card-grid {
      display: grid;
      grid-template-columns: repeat(3, 1fr);
      grid-template-rows: auto;
      gap: 1.5rem;
    }

    .card {
      display: grid;
      grid-row: span 3;  /* Card spans 3 rows */

      /* Subgrid — card rows align to the parent grid's row tracks */
      grid-template-rows: subgrid;
    }

    /* Now all card titles, bodies, and footers align horizontally */
    .card__title  { grid-row: 1; }
    .card__body   { grid-row: 2; }
    .card__footer { grid-row: 3; }
    ```
  </TabItem>
</Tabs>

## Positioning

```css
/* static: default — element in normal flow, top/right/bottom/left have no effect */
.default { position: static; }

/* relative: offset FROM the element's normal flow position */
/* Still takes up space in normal flow */
.relative {
  position: relative;
  top: 10px;    /* Move 10px DOWN from its natural position */
  left: -20px;  /* Move 20px LEFT from its natural position */
  /* Other elements are not affected */
}

/* absolute: removed from normal flow, positioned relative to nearest
   ancestor that has position: relative/absolute/fixed/sticky */
.absolute {
  position: absolute;
  top: 0;
  right: 0;
  /* If no positioned ancestor: positioned relative to the initial containing block */
}

/* fixed: removed from flow, positioned relative to the VIEWPORT */
/* Stays in place when scrolling */
.fixed {
  position: fixed;
  bottom: 1.5rem;
  right: 1.5rem;
  z-index: 100;
}

/* sticky: hybrid of relative and fixed */
/* Acts like relative until it reaches a threshold, then acts like fixed */
.sticky-header {
  position: sticky;
  top: 0;        /* Sticks to top of its scroll container */
  z-index: 50;
}
```

## z-index and Stacking Contexts

```css
/* z-index only works on positioned elements (position != static)
   and flex/grid items */

/* Elements with higher z-index appear on top */
.overlay    { position: fixed; z-index: 500; }
.modal      { position: fixed; z-index: 600; }
.tooltip    { position: absolute; z-index: 700; }

/* ─── Stacking Contexts ────────────────────────────────────────────────────
   A stacking context is an isolated z-index space.
   Elements INSIDE a stacking context cannot overlap elements OUTSIDE it.
   Properties that create a new stacking context: */

.creates-stacking-context {
  position: relative; z-index: 1;  /* Any z-index other than auto */
  opacity: 0.99;                    /* Any opacity less than 1 */
  transform: translateZ(0);         /* Any transform */
  filter: blur(0);                  /* Any filter */
  isolation: isolate;               /* Explicit — recommended way */
  will-change: transform;           /* will-change with certain values */
}

/* ─── isolation: isolate — cleanly create a stacking context ────────────── */
.card {
  isolation: isolate; /* Create a new stacking context for this component */
}
/* ::before and ::after inside .card won't bleed through other elements */
```

## Multi-Column Layout

```css
/* Newspaper-style multi-column text flow */
.newspaper-columns {
  columns: 3;          /* 3 equal columns */
  columns: 200px;      /* As many ~200px columns as fit */
  columns: 3 200px;    /* 3 columns, minimum 200px each */

  column-gap: 2rem;
  column-rule: 1px solid #e5e7eb; /* Decorative line between columns */
}

/* Prevent element from being split across columns */
.no-break {
  break-inside: avoid;
  page-break-inside: avoid; /* Legacy */
}

/* Force a column break */
.column-break-before { break-before: column; }

/* Span all columns (like a heading that covers all columns) */
.spanning-heading {
  column-span: all;
  font-size: 1.5rem;
  margin-bottom: 1rem;
}
```

## Float (Legacy and Modern Clearfix)

Floats were the pre-Flexbox way to create layouts. Today, use them only for their original purpose: wrapping text around images.

```css
/* Original use case — text flows around the floated image */
.article-image {
  float: left;
  margin: 0 1.5rem 1rem 0;
  width: 40%;
}

/* Clear float — prevent following elements from wrapping */
.clear { clear: both; }

/* Modern clearfix — contain floats within their parent */
.clearfix::after {
  content: '';
  display: table;
  clear: both;
}

/* Better: use flow-root (modern, no hacks) */
.float-container {
  display: flow-root;
  /* Creates a BFC that automatically contains floated children */
}
```

<Aside type="tip" title="Use Flexbox and Grid instead of floats">
For anything beyond floating text around an image, use Flexbox (one-dimensional) or Grid (two-dimensional). They are more predictable, powerful, and easier to reason about.
</Aside>

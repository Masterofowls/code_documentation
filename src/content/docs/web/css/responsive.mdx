Responsive design ensures your site works beautifully on every device — from a 320px phone to a 4K monitor. Modern CSS tools make this more powerful and less hacky than ever.

## Mobile-First Approach

Mobile-first means writing **base styles for the smallest screen**, then adding complexity for larger screens with `min-width` media queries. This is both a development philosophy and a performance technique.

```css
/* ✅ Mobile-first: base styles work on all screens */
.card-grid {
  display: grid;
  grid-template-columns: 1fr; /* 1 column on mobile */
  gap: 1rem;
}

/* Add columns as the viewport grows */
@media (min-width: 640px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1024px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* ❌ Desktop-first: remove things on mobile — error-prone */
.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* Has to be overridden */
}

@media (max-width: 1023px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (max-width: 639px) {
  .card-grid {
    grid-template-columns: 1fr;
  }
}
```

:::tip[Mobile-First = Smaller Default CSS]
Mobile-first code sends less CSS to mobile devices (which often have slower connections). Overrides only run on larger screens, where bandwidth is typically better.
:::

## Media Queries

### Width Breakpoints

```css
/* Common breakpoint system (Tailwind-inspired) */
/* xs: < 480px     (phones — no query, base styles) */
@media (min-width: 480px) {
  /* sm: small phones landscape */
}
@media (min-width: 640px) {
  /* sm: large phones, small tablets */
}
@media (min-width: 768px) {
  /* md: tablets */
}
@media (min-width: 1024px) {
  /* lg: laptops */
}
@media (min-width: 1280px) {
  /* xl: desktops */
}
@media (min-width: 1536px) {
  /* 2xl: wide monitors */
}

/* Range query (modern syntax — Chrome 104+) */
@media (480px <= width < 768px) {
  /* Tablets only — no overlap with phone or desktop styles */
}

/* Legacy range with max-width */
@media (max-width: 767px) {
  /* Anything smaller than 768px — only use when you need an upper limit */
}
```

### Media Features Beyond Width

```css
/* Orientation */
@media (orientation: landscape) {
  .hero {
    min-height: 60vh;
  } /* Shorter hero in landscape */
}

@media (orientation: portrait) {
  .sidebar {
    display: none;
  } /* Hide sidebar on portrait mobile */
}

/* Dark mode system preference */
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0f172a;
    --text: #f1f5f9;
  }
}

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

/* Input device: hover capability */
@media (hover: hover) and (pointer: fine) {
  /* Mouse/trackpad users — safe to use hover-only states */
  .card:hover {
    transform: translateY(-4px);
  }
  .btn:hover {
    background: var(--primary-hover);
  }
}

@media (hover: none) and (pointer: coarse) {
  /* Touch screen users — larger tap targets */
  .btn {
    min-height: 44px;
    min-width: 44px;
  }
}

/* Print styles */
@media print {
  .no-print {
    display: none;
  }
  nav,
  footer,
  .ads {
    display: none;
  }
  body {
    font-size: 12pt;
    color: black;
    background: white;
  }
  a::after {
    content: ' (' attr(href) ')';
  }
}

/* High contrast mode */
@media (forced-colors: active) {
  .custom-checkbox {
    border: 2px solid ButtonText;
    forced-color-adjust: none;
  }
}
```

## Container Queries

Container queries are a **paradigm shift** — instead of responding to the viewport size, components respond to the size of their _container_. A sidebar card and a main content card can have different layouts at the same viewport width.

```css
/* 1. Mark a container */
.card-wrapper {
  container-type: inline-size; /* Respond to width changes */
  container-name: card; /* Optional name for targeting */

  /* Shorthand */
  container: card / inline-size;
}

/* 2. Query the container from inside it */
.card {
  /* Default: stacked layout */
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* If the card's *container* is >= 400px wide */
@container (min-width: 400px) {
  .card {
    grid-template-columns: auto 1fr;
  }

  .card-image {
    width: 140px;
    aspect-ratio: 1;
  }
}

/* Named container query */
@container card (min-width: 600px) {
  .card-title {
    font-size: 1.5rem;
  }
}
```

### Container Query Units

```css
/* cqi — 1% of the container's inline size (width for horizontal writing) */
/* cqb — 1% of the container's block size (height for horizontal writing) */
/* cqw / cqh — container's width / height */
/* cqmin / cqmax — smaller / larger of cqw or cqh */

.card-title {
  font-size: clamp(1rem, 4cqi, 1.5rem); /* Fluid relative to container width */
}
```

:::tip[Container Queries vs Media Queries]
Use **media queries** for macro layout (page structure, navigation patterns). Use **container queries** for component-level responsiveness (cards, widgets, UI components that are reused in different contexts). Container queries produce truly encapsulated, composable components.
:::

## Fluid Typography

Fluid typography scales smoothly with the viewport — no abrupt jumps at breakpoints.

```css
/* Using clamp() — min size, fluid middle, max size */
:root {
  /* Base font size */
  font-size: clamp(1rem, 0.9rem + 0.5vw, 1.25rem);
}

/* Heading scale — each level has its own fluid range */
h1 {
  font-size: clamp(2rem, 5vw + 1rem, 4rem);
  /* 32px → grows → 64px */
}

h2 {
  font-size: clamp(1.5rem, 3vw + 0.75rem, 2.5rem);
}

h3 {
  font-size: clamp(1.25rem, 2vw + 0.5rem, 1.75rem);
}

/* Fluid line height */
p {
  font-size: clamp(1rem, 0.9rem + 0.5vw, 1.125rem);
  line-height: clamp(1.5, 1.4 + 0.5vw, 1.75);
}
```

## Fluid Spacing

Apply the same clamp technique to spacing for a smooth, proportional feel:

```css
:root {
  /* Fluid spacing scale */
  --space-xs: clamp(0.25rem, 0.5vw, 0.5rem);
  --space-sm: clamp(0.5rem, 1vw, 1rem);
  --space-md: clamp(1rem, 2vw, 1.5rem);
  --space-lg: clamp(1.5rem, 4vw, 3rem);
  --space-xl: clamp(2rem, 6vw, 5rem);
  --space-2xl: clamp(3rem, 8vw, 8rem);
}

/* Fluid section padding */
.section {
  padding-block: var(--space-2xl);
}

/* Fluid grid gap */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: var(--space-md);
}
```

## Responsive Images in CSS

```css
/* Cover images — fill container, maintain ratio */
.hero-image {
  width: 100%;
  aspect-ratio: 16 / 9;
  object-fit: cover;
  object-position: center;
}

/* Background images for different densities */
.banner {
  background-image: url('banner-1x.jpg');
  background-size: cover;
  background-position: center;
}

@media (min-resolution: 2dppx), (-webkit-min-device-pixel-ratio: 2) {
  .banner {
    background-image: url('banner-2x.jpg');
  }
}
```

## CSS Logical Properties

Logical properties use **block** and **inline** axes rather than physical top/right/bottom/left — making styles work correctly for RTL (right-to-left) and vertical writing modes automatically.

```css
/* Physical → Logical equivalents */

/* Margins */
margin-top:    → margin-block-start
margin-bottom: → margin-block-end
margin-left:   → margin-inline-start
margin-right:  → margin-inline-end
margin-top + margin-bottom: → margin-block
margin-left + margin-right: → margin-inline

/* Padding */
padding-top:    → padding-block-start
padding-left:   → padding-inline-start
padding-block:  → padding-top + padding-bottom
padding-inline: → padding-left + padding-right

/* Sizing */
width:      → inline-size
height:     → block-size
max-width:  → max-inline-size
max-height: → max-block-size
min-width:  → min-inline-size

/* Borders */
border-top:    → border-block-start
border-left:   → border-inline-start
border-radius: top-left → border-start-start-radius

/* Position */
top:    → inset-block-start
bottom: → inset-block-end
left:   → inset-inline-start
right:  → inset-inline-end
inset:  → all four sides at once

/* Practical example */
.card {
  /* Works for LTR, RTL, and vertical text */
  padding-block: 1.5rem;
  padding-inline: 2rem;
  border-inline-start: 4px solid var(--primary);
  margin-block-end: 2rem;
  max-inline-size: 600px;
}
```

## Dynamic Viewport Units

The classic `100vh` on mobile includes the browser's address bar, causing unexpected overflow. Use the newer dynamic viewport units:

```css
/* The three viewport variants */
.hero {
  /* svh — Small Viewport Height: assumes UI chrome is visible */
  /* Safest for minimum heights */
  min-height: 100svh;

  /* lvh — Large Viewport Height: assumes UI chrome is hidden */
  /* Use when you need the absolute maximum height */
  max-height: 100lvh;

  /* dvh — Dynamic Viewport Height: updates as chrome shows/hides */
  /* Best for fullscreen experiences that respond to scroll */
  height: 100dvh;
}

/* Inline equivalents */
.full-width {
  width: 100svw; /* Small viewport width */
  width: 100dvw; /* Dynamic viewport width */
}
```

## CSS Grid for Responsive Layouts (No Media Queries)

CSS Grid with `auto-fit` and `minmax()` can create responsive layouts that need **zero media queries**:

```css
/* Responsive card grid — reflows automatically */
.card-grid {
  display: grid;
  /* "As many columns as will fit, each at least 280px wide" */
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 1.5rem;
}

/* Responsive image gallery */
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 0.75rem;
}

/* Responsive form layout */
.form-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 1rem;

  /* Full-width fields */
  .field--full {
    grid-column: 1 / -1;
  }
}

/* RAM pattern: Repeat, Auto, Minmax */
.ram-grid {
  --min-col-width: 240px;
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(var(--min-col-width), 1fr));
  gap: var(--space-md);
}
```

:::tip[When to Use Grid Auto vs Media Queries]
Use `auto-fit`/`auto-fill` for **content-driven** layouts where the number of columns should adapt to available space. Use media queries when you need **specific** control over the layout at defined breakpoints — for example, when a 2-column layout at exactly 768px+ is a design requirement.
:::

Responsive web design ensures your site works beautifully on every device — from a 320px phone to a 4K monitor. Modern CSS provides powerful tools to make layouts fluid, components adaptive, and typography scalable — often without a single media query.

## Mobile-First Approach

Mobile-first means writing base styles for small screens and progressively enhancing for larger screens using `min-width` media queries.

<Tabs>
  <TabItem label="Mobile-First (✅ Recommended)">
    ```css
    /* Base styles — mobile (320px+) */
    .card-grid {
      display: grid;
      grid-template-columns: 1fr;
      gap: 1rem;
    }

    /* Tablet — enhance for medium screens */
    @media (min-width: 640px) {
      .card-grid {
        grid-template-columns: repeat(2, 1fr);
        gap: 1.5rem;
      }
    }

    /* Desktop — enhance for large screens */
    @media (min-width: 1024px) {
      .card-grid {
        grid-template-columns: repeat(3, 1fr);
        gap: 2rem;
      }
    }
    ```
  </TabItem>
  <TabItem label="Desktop-First (❌ Avoid)">
    ```css
    /* Base styles — desktop (everything) */
    .card-grid {
      display: grid;
      grid-template-columns: repeat(3, 1fr);
      gap: 2rem;
    }

    /* Degrade for medium screens */
    @media (max-width: 1024px) {
      .card-grid {
        grid-template-columns: repeat(2, 1fr);
        gap: 1.5rem;
      }
    }

    /* Degrade for small screens */
    @media (max-width: 640px) {
      .card-grid {
        grid-template-columns: 1fr;
        gap: 1rem;
      }
    }
    ```
  </TabItem>
</Tabs>

:::tip[Why Mobile-First?]
Mobile devices often have slower CPUs. Mobile-first means they parse less CSS (the overrides for larger screens are skipped). It also forces you to prioritise essential content for constrained viewports before adding complexity.
:::

## Media Queries

### Syntax and Common Breakpoints

```css
/* Width-based breakpoints (most common) */
@media (min-width: 480px)  { /* Phones, landscape */ }
@media (min-width: 640px)  { /* Small tablets (sm) */ }
@media (min-width: 768px)  { /* Tablets (md) */ }
@media (min-width: 1024px) { /* Laptops (lg) */ }
@media (min-width: 1280px) { /* Desktops (xl) */ }
@media (min-width: 1536px) { /* Wide desktops (2xl) */ }

/* Orientation */
@media (orientation: landscape) { }
@media (orientation: portrait)  { }

/* System colour scheme */
@media (prefers-color-scheme: dark)  { }
@media (prefers-color-scheme: light) { }

/* Motion preference */
@media (prefers-reduced-motion: reduce) { }
@media (prefers-reduced-motion: no-preference) { }

/* Pointer precision */
@media (hover: hover) and (pointer: fine) {
  /* Desktop with mouse — can use hover effects */
  .button:hover { background: #2563eb; }
}

@media (hover: none) and (pointer: coarse) {
  /* Touch screen — larger tap targets */
  .button { min-height: 44px; }
}

/* Print */
@media print {
  nav, .ads, .sidebar { display: none; }
  a[href]::after { content: ' (' attr(href) ')'; }
}
```

### Logical Operators in Media Queries

```css
/* AND — both conditions must be true */
@media (min-width: 640px) and (max-width: 1024px) {
  /* Only tablet-sized screens */
}

/* Comma = OR — either condition can be true */
@media (max-width: 480px), (orientation: portrait) {
  /* Narrow screens OR portrait orientation */
}

/* NOT */
@media not print {
  /* Everything except print */
}

/* Range syntax (Level 4 — widely supported) */
@media (640px <= width <= 1024px) {
  /* Between 640px and 1024px (inclusive) */
}

@media (width >= 768px) {
  /* Same as (min-width: 768px) */
}
```

## Container Queries

Container queries let a component adapt based on the size of its **container** — not the viewport. This is a game-changer for reusable components.

```css
/* ─── Step 1: Define the container ─────────────────────── */
.card-wrapper {
  container-type: inline-size; /* Respond to width changes */
  container-name: card;        /* Optional — for specific targeting */
}

/* Shorthand */
.card-wrapper {
  container: card / inline-size;
}

/* ─── Step 2: Query the container ──────────────────────── */
/* "If this component's container is at least 400px wide..." */
@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
    gap: 1rem;
  }
}

/* Named container query */
@container card (min-width: 400px) {
  .card-title {
    font-size: 1.5rem;
  }
}
```

### Container Query Units

```css
/* cqi — 1% of the container's inline size (usually width) */
/* cqb — 1% of the container's block size (usually height) */
/* cqw — 1% of the container's width */
/* cqh — 1% of the container's height */
/* cqmin — smaller of cqw and cqh */
/* cqmax — larger of cqw and cqh */

.card-title {
  font-size: clamp(1rem, 4cqi, 2rem);
}

.card-image {
  width: 50cqi; /* Half the container's inline size */
}
```

## Fluid Typography

Use `clamp()` to create font sizes that scale smoothly between breakpoints — no media queries needed:

```css
/* clamp(minimum, preferred, maximum) */

:root {
  /* Body text: 16px → 18px between 320px and 1280px viewports */
  --text-base: clamp(1rem, 0.9375rem + 0.3125vw, 1.125rem);

  /* Large heading: 28px → 60px */
  --text-hero: clamp(1.75rem, 1.25rem + 4vw, 3.75rem);

  /* Medium heading: 20px → 36px */
  --text-h2: clamp(1.25rem, 0.875rem + 3vw, 2.25rem);

  /* Small heading: 18px → 24px */
  --text-h3: clamp(1.125rem, 0.9375rem + 1.5vw, 1.5rem);
}

body        { font-size: var(--text-base); }
h1          { font-size: var(--text-hero); }
h2          { font-size: var(--text-h2); }
h3          { font-size: var(--text-h3); }
```

### Calculating Fluid Values

The formula for a linear interpolation between two sizes:

```
preferred = calc(minSize + (maxSize - minSize) * (100vw - minVP) / (maxVP - minVP))
```

```css
/* Font-size: 16px at 320px viewport → 24px at 1280px viewport */
font-size: clamp(
  1rem,
  calc(1rem + (24 - 16) * ((100vw - 320px) / (1280 - 320))),
  1.5rem
);
```

## Fluid Spacing

Apply the same fluid approach to spacing:

```css
:root {
  /* Section padding: 2rem → 6rem */
  --section-spacing: clamp(2rem, 5vw, 6rem);

  /* Card padding: 1rem → 2rem */
  --card-padding: clamp(1rem, 3vw, 2rem);

  /* Content max-width with auto side padding */
  --content-width: min(90%, 1200px);
  --content-padding: max(1rem, (100% - 1200px) / 2);
}

.section {
  padding-block: var(--section-spacing);
}

.container {
  max-width: var(--content-width);
  margin-inline: auto;
}
```

## Responsive Images

```html
<!-- srcset with size descriptors — browser chooses optimal image -->
<img
  src="/images/hero-800.jpg"
  srcset="
    /images/hero-480.jpg  480w,
    /images/hero-800.jpg  800w,
    /images/hero-1200.jpg 1200w,
    /images/hero-1600.jpg 1600w
  "
  sizes="
    (max-width: 480px)  100vw,
    (max-width: 1024px) 90vw,
    1200px
  "
  alt="Hero image"
  width="1200"
  height="630"
/>
```

```css
/* CSS for truly responsive images */
img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* Responsive background images */
.hero {
  background-image: url('/images/hero-mobile.jpg');
  background-size: cover;
  background-position: center;
}

@media (min-width: 768px) {
  .hero {
    background-image: url('/images/hero-desktop.jpg');
  }
}
```

## Logical Properties

Logical properties adapt to the writing direction and text orientation — essential for international sites supporting RTL languages (Arabic, Hebrew) or vertical text (Japanese):

```css
/* ─── Physical → Logical mapping ────────────────────────── */

/* Margins */
margin-top    → margin-block-start
margin-bottom → margin-block-end
margin-left   → margin-inline-start
margin-right  → margin-inline-end

/* Shorthands */
margin-block: 1rem;         /* top + bottom */
margin-inline: auto;        /* left + right */
padding-block: 0.5rem 1rem; /* top bottom */
padding-inline: 1rem 2rem;  /* left right */

/* Borders */
border-top    → border-block-start
border-left   → border-inline-start

/* Sizing */
width  → inline-size
height → block-size
min-width → min-inline-size
max-height → max-block-size

/* Positioning */
top   → inset-block-start
left  → inset-inline-start
inset: 0;              /* top right bottom left = 0 */
inset-inline: 0 auto;  /* left: 0; right: auto */

/* Text alignment */
text-align: start; /* Left in LTR, Right in RTL */
text-align: end;
```

```css
/* Example: modal that works in LTR and RTL */
.modal {
  position: fixed;
  inset: 0;                  /* top: 0; right: 0; bottom: 0; left: 0 */
  margin: auto;
  inline-size: min(90%, 600px);
  padding-block: 2rem;
  padding-inline: 1.5rem;
  border-inline-start: 4px solid var(--color-primary);
}
```

## Viewport Units for Mobile

```css
/* The problem: 100vh includes browser chrome on mobile, causing overflow */
.hero-old {
  height: 100vh; /* Too tall on mobile — browser toolbar pushes content off screen */
}

/* The solution: use environment-aware viewport units */
.hero {
  /* svh — Small Viewport Height (excludes browser chrome completely) */
  min-height: 100svh;

  /* dvh — Dynamic Viewport Height (updates as chrome shows/hides) */
  min-height: 100dvh;

  /* lvh — Large Viewport Height (as if no chrome) */
  min-height: 100lvh;

  /* Recommendation: use svh for safe layout, dvh for full-bleed sections */
}
```

## CSS Grid: Responsive Without Media Queries

The most elegant responsive pattern requires zero media queries:

```css
/* Cards fill the grid, each at least 280px wide */
/* When there's not enough room, they stack — automatically */
.responsive-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, 280px), 1fr));
  gap: 1.5rem;
}

/* Same technique for a navigation */
.nav-links {
  display: flex;
  flex-wrap: wrap;
  gap: 0.5rem;
}
```

## Common Responsive Patterns

```css
/* ─── Sidebar layout that stacks on mobile ──────────────── */
.page-with-sidebar {
  display: grid;
  grid-template-columns: 1fr;
  gap: 2rem;
}

@media (min-width: 768px) {
  .page-with-sidebar {
    grid-template-columns: 280px 1fr;
  }
}

/* ─── Typography scale ───────────────────────────────────── */
:root {
  font-size: 100%; /* 16px default */
}

@media (min-width: 1280px) {
  :root {
    font-size: 112.5%; /* 18px on large screens */
  }
}

/* ─── Hide/show elements ─────────────────────────────────── */
.mobile-only  { display: block; }
.desktop-only { display: none; }

@media (min-width: 768px) {
  .mobile-only  { display: none; }
  .desktop-only { display: block; }
}

/* ─── Container for page content ────────────────────────── */
.container {
  width: 100%;
  max-width: 1200px;
  margin-inline: auto;
  padding-inline: clamp(1rem, 5vw, 3rem);
}
```

:::caution[Breakpoints Should Come from Your Content]
Resist using fixed breakpoints like 768px and 1024px just because they match popular device sizes. Instead, add a breakpoint when your content looks broken or awkward. Content-derived breakpoints make your design more resilient to future device sizes.
:::

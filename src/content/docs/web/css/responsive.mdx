Responsive design ensures your site looks and works well on every screen size — from a 320px phone to a 4K monitor. Modern CSS makes this more achievable than ever, with fewer media queries required.

## Mobile-First Approach

The **mobile-first** approach means writing base styles for small screens, then progressively enhancing for larger screens using `min-width` media queries.

```css
/* ─── Mobile-first example ───────────────────────────────────────────────── */

/* Base styles: small screens (< 640px) */
.card-grid {
  display: grid;
  grid-template-columns: 1fr; /* Single column on mobile */
  gap: 1rem;
}

/* Tablet: 2 columns (≥ 640px) */
@media (min-width: 40rem) { /* 640px */
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 1.5rem;
  }
}

/* Desktop: 3 columns (≥ 1024px) */
@media (min-width: 64rem) { /* 1024px */
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 2rem;
  }
}
```

:::tip[Why mobile-first?]
Mobile-first means older browsers (which often run on older phones) get the simpler base styles. CSS is progressive — you add complexity for larger screens. This also forces you to prioritise content: if it's not important enough for mobile, is it important at all?
:::

## Media Queries

```css
/* ─── Viewport width ─────────────────────────────────────────────────────── */
@media (min-width: 768px) { /* Styles for viewports 768px and wider */ }
@media (max-width: 767px) { /* Styles for viewports 767px and narrower */ }

/* ─── Range queries (modern syntax) ─────────────────────────────────────── */
@media (768px <= width <= 1024px) { /* Between 768px and 1024px */ }
@media (width >= 1280px) { /* 1280px and wider */ }

/* ─── Common breakpoints (rem-based) ────────────────────────────────────── */
/* Using rem instead of px respects user browser font size preferences */
@media (min-width: 20rem)  { /* 320px — small phones */ }
@media (min-width: 30rem)  { /* 480px — large phones */ }
@media (min-width: 40rem)  { /* 640px — large phones / small tablets */ }
@media (min-width: 48rem)  { /* 768px — tablets */ }
@media (min-width: 64rem)  { /* 1024px — small desktops / large tablets */ }
@media (min-width: 80rem)  { /* 1280px — desktops */ }
@media (min-width: 96rem)  { /* 1536px — large desktops */ }

/* ─── Orientation ────────────────────────────────────────────────────────── */
@media (orientation: landscape) {
  .landscape-layout { flex-direction: row; }
}
@media (orientation: portrait) {
  .portrait-layout { flex-direction: column; }
}

/* ─── User preference queries ────────────────────────────────────────────── */
@media (prefers-color-scheme: dark)  { /* Dark mode styles */ }
@media (prefers-color-scheme: light) { /* Light mode styles */ }

@media (prefers-reduced-motion: reduce) {
  /* Remove animations for users who prefer no motion */
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

@media (prefers-contrast: high) {
  /* Enhance contrast for accessibility */
  :root {
    --color-text: #000000;
    --color-bg: #ffffff;
    --color-border: #000000;
  }
}

/* ─── Device capability queries ──────────────────────────────────────────── */
/* hover: does the device support hover? */
@media (hover: hover) {
  /* Only apply hover effects on devices that support hover (mice) */
  .button:hover { background-color: #4f46e5; }
}

/* pointer: what kind of pointing device? */
@media (pointer: coarse) {
  /* Touch devices — make touch targets larger */
  button, a { min-height: 44px; min-width: 44px; }
}
@media (pointer: fine) {
  /* Precision pointer (mouse) — can use smaller targets */
  .tooltip-trigger { cursor: help; }
}

/* ─── Print ───────────────────────────────────────────────────────────────── */
@media print {
  nav, aside, footer, .no-print { display: none; }
  body { font-size: 12pt; color: black; background: white; }
  a[href]::after { content: ' (' attr(href) ')'; }
  .page-break { break-before: page; }
}
```

## Container Queries

Container queries allow elements to respond to the **size of their container** rather than the viewport. This enables truly reusable components.

```css
/* ─── Define a containment context ─────────────────────────────────────── */
.card-wrapper {
  container-type: inline-size; /* Enable container queries based on width */
  container-name: card;        /* Optional name for targeting */
}

/* Shorthand */
.card-wrapper {
  container: card / inline-size;
}

/* ─── Container query ────────────────────────────────────────────────────── */
/* .card styles apply when .card-wrapper is ≥ 480px wide */
@container (min-width: 480px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
  }
}

/* Target a named container */
@container card (min-width: 600px) {
  .card__title { font-size: 1.5rem; }
}

/* ─── Container query units ──────────────────────────────────────────────── */
/* cqi = 1% of the container's inline size (usually width) */
/* cqb = 1% of the container's block size (usually height) */
/* cqw = 1% of the container's width */
/* cqh = 1% of the container's height */
/* cqmin = smaller of cqi and cqb */
/* cqmax = larger of cqi and cqb */

.card__title {
  font-size: clamp(1rem, 5cqi, 2rem);
  /* Scales with container width, not viewport width */
}
```

<Aside type="tip" title="Container queries vs media queries">
Use **media queries** for page-level layout changes (switching from 1 to 3 columns). Use **container queries** for component-level changes (card layout adapting to its grid cell width). This separation makes components reusable across different contexts.
</Aside>

## Fluid Typography

```css
/* ─── clamp() fluid typography ───────────────────────────────────────────── */
/* clamp(minimum, preferred, maximum) */

/* The preferred value grows linearly between min and max viewport widths */
/* Formula: clamp(min, calc(min + (max - min) * (100vw - minVp) / (maxVp - minVp)), max) */

:root {
  /* Fluid type scale — grows from 320px to 1440px viewport */
  --text-sm:   clamp(0.8rem,  0.75rem + 0.25vw,  0.875rem);
  --text-base: clamp(1rem,    0.9rem  + 0.5vw,   1.125rem);
  --text-lg:   clamp(1.125rem, 1rem   + 0.625vw, 1.375rem);
  --text-xl:   clamp(1.25rem, 1.1rem  + 0.75vw,  1.625rem);
  --text-2xl:  clamp(1.5rem,  1.25rem + 1.25vw,  2rem);
  --text-3xl:  clamp(1.875rem, 1.5rem + 1.875vw, 2.5rem);
  --text-4xl:  clamp(2.25rem, 1.75rem + 2.5vw,   3.5rem);
}

h1 { font-size: var(--text-4xl); }
h2 { font-size: var(--text-3xl); }
h3 { font-size: var(--text-2xl); }
p  { font-size: var(--text-base); }
```

## Fluid Spacing

```css
/* ─── Fluid spacing scale ────────────────────────────────────────────────── */
:root {
  --space-sm:  clamp(0.75rem,  2vw,   1rem);
  --space-md:  clamp(1rem,     3vw,   1.5rem);
  --space-lg:  clamp(1.5rem,   4vw,   2.5rem);
  --space-xl:  clamp(2rem,     6vw,   4rem);
  --space-2xl: clamp(3rem,     8vw,   6rem);
}

/* Fluid section padding */
section {
  padding-block: var(--space-2xl);
  padding-inline: clamp(1rem, 5%, 2rem);
}
```

## Responsive Images in CSS

```css
/* Responsive background images */
.hero {
  background-image: url('/images/hero-small.jpg');
  background-size: cover;
  background-position: center;

  @media (min-width: 768px) {
    background-image: url('/images/hero-medium.jpg');
  }

  @media (min-width: 1280px) {
    background-image: url('/images/hero-large.jpg');
  }
}

/* High-DPI / Retina displays */
@media (-webkit-min-device-pixel-ratio: 2),
       (resolution >= 2dppx) {
  .logo {
    background-image: url('/images/logo@2x.png');
    background-size: 200px 60px; /* Explicit size to display at 1x */
  }
}

/* Prefer WebP if supported */
.hero {
  background-image: url('/images/hero.jpg'); /* Fallback */
}
/* Use image-set() for responsive, format-aware backgrounds */
.hero {
  background-image: image-set(
    url('/images/hero.webp') type('image/webp'),
    url('/images/hero.jpg') type('image/jpeg')
  );
}
```

## Logical Properties

Logical properties use writing-mode-relative terms (`inline`, `block`, `start`, `end`) instead of physical directions (`top`, `right`, `bottom`, `left`). They enable RTL language support automatically.

```css
/* ─── Physical vs Logical ───────────────────────────────────────────────── */

/* Physical (LTR-only) */
.physical {
  margin-left: 1rem;
  margin-right: 1rem;
  padding-top: 1rem;
  padding-bottom: 1rem;
  border-left: 2px solid;
  text-align: left;
}

/* Logical (LTR and RTL) */
.logical {
  margin-inline: 1rem;    /* margin-left + margin-right */
  padding-block: 1rem;    /* padding-top + padding-bottom */
  border-inline-start: 2px solid; /* left in LTR, right in RTL */
  text-align: start;      /* left in LTR, right in RTL */
}

/* ─── Full logical property reference ───────────────────────────────────── */
.full-logical {
  /* Block axis = vertical (in horizontal writing modes) */
  margin-block: 1rem;         /* shorthand */
  margin-block-start: 1rem;   /* margin-top in LTR */
  margin-block-end: 1rem;     /* margin-bottom in LTR */
  padding-block: 1rem;
  padding-block-start: 0.5rem;
  padding-block-end: 1.5rem;
  border-block: 1px solid;
  inset-block: 0;             /* top and bottom */

  /* Inline axis = horizontal (in horizontal writing modes) */
  margin-inline: auto;        /* shorthand — centers horizontally */
  margin-inline-start: 0;     /* margin-left in LTR */
  margin-inline-end: 1rem;    /* margin-right in LTR */
  padding-inline: 1.5rem;
  border-inline: 1px solid;
  inset-inline: 0;            /* left and right */

  /* Sizing */
  inline-size: 100%;          /* width in horizontal writing mode */
  block-size: auto;           /* height in horizontal writing mode */
  min-inline-size: 280px;
  max-inline-size: 800px;
  max-block-size: 400px;

  /* Position */
  inset: 0;                   /* shorthand for top/right/bottom/left */
  inset-block-start: 1rem;    /* top in LTR */
  inset-inline-start: 1rem;   /* left in LTR */
}
```

## Viewport Units

```css
/* ─── Standard viewport units ───────────────────────────────────────────── */
.standard {
  width: 100vw;   /* 100% of viewport width */
  height: 100vh;  /* 100% of viewport height */
  font-size: 5vw; /* 5% of viewport width */
  padding: 3vmin; /* 3% of the smaller viewport dimension */
}

/* ─── The problem with 100vh on mobile ───────────────────────────────────── */
/* Mobile browsers have a dynamic UI (address bar shows/hides as user scrolls).
   100vh is the FULL height including hidden UI elements.
   This causes content to be obscured behind the browser chrome. */

/* ─── New viewport units (2022+) ────────────────────────────────────────── */
.modern-viewport {
  /* svh = small viewport height (UI fully visible — the MINIMUM height) */
  min-height: 100svh; /* Safe: never hidden behind browser chrome */

  /* lvh = large viewport height (UI fully hidden — the MAXIMUM height) */
  height: 100lvh;     /* Full height when UI is scrolled away */

  /* dvh = dynamic viewport height (updates as UI appears/disappears) */
  height: 100dvh;     /* Most accurate, but can trigger layout shifts */

  /* Similarly: svw, lvw, dvw, svmin, lvmin, dvmin, svmax, lvmax, dvmax */
}

/* Recommended approach for full-screen layouts */
.full-screen-layout {
  min-height: 100svh; /* Guaranteed to fit, even with browser UI visible */
}
.app-shell {
  height: 100dvh;     /* Dynamic — adapts as chrome shows/hides */
  display: flex;
  flex-direction: column;
  overflow: hidden;
}
```

## Responsive Layouts Without Media Queries

Modern CSS can create responsive layouts that adapt automatically without any `@media` queries.

```css
/* ─── Auto-responsive grid (fluid columns) ──────────────────────────────── */
.auto-grid {
  display: grid;
  /* Cards are minimum 280px; create as many as fit; stretch to fill */
  grid-template-columns: repeat(auto-fill, minmax(min(280px, 100%), 1fr));
  gap: 1.5rem;
}

/* ─── Flexible sidebar + content ─────────────────────────────────────────── */
.sidebar-layout {
  display: flex;
  flex-wrap: wrap;
  gap: 2rem;
}
.sidebar-layout__sidebar {
  flex: 1 1 280px; /* Grows and shrinks, minimum 280px */
  max-width: 320px;
}
.sidebar-layout__content {
  flex: 1 1 400px; /* Minimum 400px — if less than 400px available, wraps */
}

/* ─── Stack/row switcher ──────────────────────────────────────────────────── */
/* Stacks vertically when container is narrow, rows when wide */
.auto-switch {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.auto-switch > * {
  flex: 1 1 200px; /* Each child wraps when container is too narrow */
}

/* ─── Intrinsically responsive typography ───────────────────────────────── */
.fluid-text {
  font-size: clamp(1rem, 2.5vw, 1.5rem);
  line-height: clamp(1.4, 1.2 + 0.5vw, 1.7);
}
```

:::tip[Start without media queries]
Before reaching for `@media`, try to solve the problem with intrinsic layout techniques: `auto-fill`/`auto-fit` grids, `flex-wrap`, `clamp()`, and `min-content`/`max-content` sizing. Many layouts that previously required 3 breakpoints can now be solved with a single `grid-template-columns` declaration.
:::

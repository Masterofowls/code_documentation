CSS selectors determine which HTML elements a rule applies to. Understanding the full selector toolkit lets you write precise, efficient styles without resorting to extra classes or `!important`.

## Basic Selectors

```css
/* Universal selector — every element (specificity: 0-0-0-0) */
* {
  box-sizing: border-box;
}

/* Type selector — all elements of this type */
h1 { font-size: 2.25rem; }
p  { line-height: 1.7; }
a  { color: #0066cc; }

/* Class selector — elements with this class */
.card       { border-radius: 0.5rem; }
.is-active  { font-weight: 600; }

/* ID selector — the one element with this id (specificity: 0-1-0-0) */
#hero       { min-height: 80vh; }
#main-nav   { position: sticky; top: 0; }

/* Multiple selectors — same rule applies to all */
h1, h2, h3, h4, h5, h6 {
  font-weight: 700;
  line-height: 1.2;
}
```

## Attribute Selectors

Attribute selectors target elements based on the presence or value of HTML attributes:

```css
/* [attr] — has the attribute (any value) */
[required]           { border-color: #f59e0b; }
[disabled]           { opacity: 0.5; cursor: not-allowed; }
[data-theme]         { transition: background 0.3s; }

/* [attr=value] — exact match */
[type="text"]        { border: 1px solid #ccc; }
[type="submit"]      { background: #0066cc; color: white; }
[rel="nofollow"]     { color: #666; }

/* [attr~=value] — value is in a space-separated list */
[class~="card"]      { border-radius: 0.5rem; }     /* matches "card active" */

/* [attr|=value] — equals "val" or starts with "val-" (language codes) */
[lang|="en"]         { font-family: 'Georgia', serif; } /* matches "en", "en-US", "en-GB" */
[hreflang|="fr"]     { quotes: "«\00A0" "\00A0»"; }

/* [attr^=value] — starts with value */
[href^="https://"]   { /* secure links */ }
[href^="mailto:"]    { cursor: pointer; }
[class^="icon-"]     { display: inline-flex; align-items: center; }

/* [attr$=value] — ends with value */
[href$=".pdf"]       { padding-right: 1.25em; background: url('pdf-icon.svg') right center no-repeat; }
[href$=".zip"]       { padding-right: 1.25em; background: url('zip-icon.svg') right center no-repeat; }
[src$=".webp"]       { image-rendering: crisp-edges; }

/* [attr*=value] — contains value anywhere */
[class*="is-"]       { /* any utility class like is-active, is-hidden */ }
[href*="youtube.com"] { /* YouTube links */ }
[data-testid*="btn"] { /* testing hook selectors */ }

/* Case-insensitive matching */
[type="TEXT" i]      { /* matches "text", "TEXT", "Text" */ }
```

:::tip[Attribute Selectors and Performance]
Attribute selectors with wildcards (`*=`, `$=`, `^=`) are slightly slower than class or ID selectors because the browser needs to parse attribute values. For critical hot paths (large lists, animations), prefer class selectors.
:::

## Combinators

Combinators express relationships between elements:

```css
/* Descendant combinator (space) — any level of nesting */
.nav a            { color: inherit; text-decoration: none; }
article p         { margin-block: 1rem; }

/* Child combinator (>) — direct children only */
.menu > li        { display: flex; align-items: center; }
ul > li > a       { padding: 0.5rem 1rem; }

/* Adjacent sibling combinator (+) — immediately following sibling */
h2 + p            { margin-top: 0.5rem; font-size: 1.125rem; } /* Lead paragraph */
input + label     { margin-left: 0.5rem; }

/* General sibling combinator (~) — all following siblings */
h2 ~ p            { color: #555; }
.is-open ~ .panel { display: block; }

/* Combining combinators */
.sidebar > .widget + .widget { margin-top: 1.5rem; } /* All except first widget */
```

## Pseudo-Classes

Pseudo-classes target elements based on state or structural position.

### User Interaction States

```css
a:hover           { text-decoration: underline; color: #004499; }
a:active          { color: #003088; }
a:visited         { color: #6b21a8; }

button:hover      { background: #004499; transform: translateY(-1px); }
button:active     { transform: translateY(0); }

/* :focus — any focus method */
input:focus       { outline: 2px solid #0066cc; outline-offset: 2px; }

/* :focus-visible — keyboard/programmatic focus only (not mouse click) */
/* Better for mouse users who find focus rings distracting */
button:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}
button:focus:not(:focus-visible) {
  outline: none; /* Hide outline when focused by mouse */
}

/* :focus-within — element or any descendant is focused */
.form-field:focus-within label {
  color: #0066cc;
  font-weight: 600;
}
```

### Structural Pseudo-Classes

```css
/* :first-child, :last-child, :only-child */
li:first-child    { border-top: none; }
li:last-child     { border-bottom: none; }
p:only-child      { font-size: 1.25rem; } /* Solo paragraph gets larger */

/* :nth-child(An + B) */
tr:nth-child(even)        { background: #f8fafc; }    /* Zebra striping */
tr:nth-child(odd)         { background: #fff; }
li:nth-child(3)           { font-weight: bold; }       /* Third item only */
li:nth-child(3n)          { color: red; }              /* Every 3rd */
li:nth-child(n + 4)       { display: none; }           /* From 4th onwards */
li:nth-child(-n + 3)      { font-weight: bold; }       /* First 3 only */

/* :nth-of-type — count only elements of the same type */
p:nth-of-type(1)          { font-size: 1.2rem; }       /* First <p> in parent */
img:nth-of-type(odd)      { float: left; }
img:nth-of-type(even)     { float: right; }

/* :first-of-type, :last-of-type, :only-of-type */
h2:first-of-type          { margin-top: 0; }
p:last-of-type            { margin-bottom: 0; }
```

### Negation and Matching

```css
/* :not() — exclude elements */
a:not(.btn)               { color: #0066cc; }           /* Links that aren't buttons */
input:not([type="submit"]):not([type="reset"]) { border: 1px solid #ccc; }
li:not(:last-child)       { border-bottom: 1px solid #eee; }

/* :is() — match any selector in the list (uses highest specificity of list) */
:is(h1, h2, h3, h4) { font-weight: 700; line-height: 1.2; }

/* Equivalent to: */
/* h1, h2, h3, h4 { ... } — but :is() is more concise in complex cases */
:is(article, section, aside) :is(h2, h3) {
  color: #1e293b;
}

/* :where() — same as :is() but ZERO specificity */
:where(h1, h2, h3, h4, h5, h6) {
  font-family: 'Georgia', serif;
}
/* Can be overridden by a simple class selector */

/* :has() — parent selector — target elements based on their children */
/* "Card that contains an image" */
.card:has(img)            { padding: 0; }
.card:has(img) .card-body { padding: 1.5rem; }

/* "Form that has been submitted" */
form:has(input:valid):not(:has(input:invalid)) .submit-btn {
  background: #16a34a;
}

/* Label whose following input is checked */
label:has(+ input:checked) { color: #0066cc; font-weight: 600; }

/* "Section with no headings" */
section:not(:has(h2, h3)) { border-top: 2px solid #f59e0b; }
```

### Form State Pseudo-Classes

```css
input:required        { border-left: 3px solid #f59e0b; }
input:optional        { border-left: 3px solid #e2e8f0; }
input:valid           { border-color: #16a34a; }
input:invalid         { border-color: #dc2626; }
input:disabled        { opacity: 0.5; cursor: not-allowed; background: #f8fafc; }
input:enabled         { cursor: text; }
input:read-only       { background: #f8fafc; }
input:read-write      { background: #fff; }
input:placeholder-shown { border-color: #cbd5e1; }
input:checked + label { font-weight: 600; }
input:indeterminate   { accent-color: #f59e0b; } /* for partial checkbox states */
```

## :where() vs :is() — The Specificity Difference

This is one of the most practically useful distinctions in modern CSS:

```css
/* :is() — specificity = highest in the list */
:is(#page, .article) h2 {
  color: red;
  /* Specificity: 0-1-0-1 (ID from #page contributes) */
}

/* :where() — specificity = ALWAYS 0 */
:where(#page, .article) h2 {
  color: blue;
  /* Specificity: 0-0-0-1 (the ID is ignored for specificity) */
}

/* Result: red wins, even though :where() comes later, because :is() */
/* has higher specificity via the #page selector */

/* Practical use: reset/base styles that should be easy to override */
:where(ul, ol) {
  margin: 0;
  padding: 0;
  list-style: none;
}
/* A simple .nav { padding: 1rem } will override this — no specificity battle */
```

## Pseudo-Elements

Pseudo-elements target specific parts of an element rather than the whole element:

```css
/* ::before and ::after — generated content */
.btn::before {
  content: '';
  display: block;
  position: absolute;
  inset: 0;
  border-radius: inherit;
  background: rgb(255 255 255 / 0);
  transition: background 0.15s;
}

.btn:hover::before {
  background: rgb(255 255 255 / 0.1);
}

/* Quote marks using ::before / ::after */
blockquote::before { content: open-quote; }
blockquote::after  { content: close-quote; }

/* ::placeholder — style input placeholder text */
input::placeholder {
  color: #94a3b8;
  font-style: italic;
  opacity: 1; /* Firefox reduces opacity by default */
}

/* ::selection — text selected by user */
::selection {
  background: #bfdbfe;
  color: #1e3a5f;
}

/* ::marker — the list bullet or number */
ul li::marker { color: #0066cc; }
ol li::marker { font-weight: 700; color: #0066cc; }

/* ::first-line — only the first rendered line */
p::first-line {
  font-variant: small-caps;
  font-weight: 600;
}

/* ::first-letter — drop cap */
.article-body > p:first-child::first-letter {
  font-size: 3.5em;
  font-weight: 700;
  float: left;
  line-height: 0.85;
  margin-right: 0.1em;
  color: #0066cc;
}
```

:::caution[Pseudo-Elements Require content Property for ::before / ::after]
`::before` and `::after` will not render unless they have a `content` property — even `content: ''` (empty string) is required.
:::

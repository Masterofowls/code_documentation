CSS selectors are patterns that match elements in the DOM. Choosing the right selector makes your CSS more readable, maintainable, and efficient.

## Basic Selectors

```css
/* Universal selector — matches every element */
* {
  box-sizing: border-box;
}

/* Type (element) selector */
p { line-height: 1.6; }
h1 { font-size: 2rem; }
button { cursor: pointer; }

/* Class selector */
.card { border-radius: 0.5rem; }
.text-large { font-size: 1.25rem; }

/* ID selector — use sparingly (high specificity, must be unique) */
#main-nav { position: sticky; top: 0; }

/* Multiple selectors — apply same styles to multiple targets */
h1,
h2,
h3 {
  font-weight: 700;
  line-height: 1.2;
}
```

## Attribute Selectors

```css
/* [attr] — has the attribute (any value) */
input[required] { border-left: 3px solid #ef4444; }
a[target] { text-decoration: underline dotted; }

/* [attr="val"] — exact match */
input[type="email"] { padding-left: 2rem; }
a[rel="noopener"] { color: inherit; }

/* [attr~="val"] — value is in a space-separated list */
/* Matches: class="card featured" or class="featured" */
[class~="featured"] { border: 2px solid gold; }

/* [attr|="val"] — value is exactly "val" OR starts with "val-" */
/* Matches: lang="en" and lang="en-US" but not lang="english" */
[lang|="en"] { font-family: 'Inter', sans-serif; }

/* [attr^="val"] — value starts with "val" */
a[href^="https://"] { background-image: url('/icons/external.svg'); }
a[href^="mailto:"] { background-image: url('/icons/email.svg'); }

/* [attr$="val"] — value ends with "val" */
a[href$=".pdf"]  { background-image: url('/icons/pdf.svg'); }
a[href$=".docx"] { background-image: url('/icons/word.svg'); }

/* [attr*="val"] — value contains "val" anywhere */
a[href*="example.com"] { font-weight: bold; }
img[src*="/thumbnails/"] { border-radius: 4px; }

/* Case-insensitive modifier */
a[href$=".PDF" i] { background-image: url('/icons/pdf.svg'); }
```

## Combinators

Combinators describe the relationship between two selectors:

```css
/* Descendant (space) — any depth */
/* Matches all <a> inside .nav, no matter how deeply nested */
.nav a { color: white; }

/* Child (>) — direct children only */
/* Matches only <li> that are direct children of <ul> */
ul > li { list-style: disc; }

/* Adjacent sibling (+) — immediately following sibling */
/* Matches <p> that comes directly after an <h2> */
h2 + p { font-size: 1.1rem; font-weight: 500; }

/* General sibling (~) — all following siblings */
/* Matches all <p> that come after an <h2> at the same level */
h2 ~ p { margin-top: 0; }
```

## Pseudo-classes

### User Interaction

```css
a:link    { color: #3b82f6; }  /* Unvisited link */
a:visited { color: #7c3aed; }  /* Visited link */
a:hover   { text-decoration: underline; }
a:active  { color: #1d4ed8; }  /* Being clicked */

/* :focus — keyboard focus (any method) */
button:focus { outline: 2px solid #3b82f6; }

/* :focus-visible — keyboard focus only (not mouse click) */
/* Preferred for modern focus styles */
button:focus-visible {
  outline: 2px solid #3b82f6;
  outline-offset: 2px;
}

/* :focus-within — any descendant has focus */
.form-group:focus-within label {
  color: #3b82f6;
}
```

### Structural Pseudo-classes

```css
/* :first-child / :last-child */
li:first-child { border-top: none; }
li:last-child  { border-bottom: none; }

/* :only-child */
li:only-child { text-align: center; }

/* :nth-child(An+B) */
li:nth-child(2)    { background: #f3f4f6; } /* 2nd item */
li:nth-child(even) { background: #f9fafb; } /* 2, 4, 6, ... */
li:nth-child(odd)  { background: white; }   /* 1, 3, 5, ... */
li:nth-child(3n)   { color: red; }          /* 3, 6, 9, ... */
li:nth-child(3n+1) { font-weight: bold; }   /* 1, 4, 7, ... */
li:nth-child(-n+3) { border-top: 2px solid blue; } /* first 3 */

/* :nth-last-child() — count from the end */
li:nth-last-child(2) { color: orange; } /* Second from last */

/* :nth-of-type() — like nth-child but counts only matching type */
p:nth-of-type(2) { font-style: italic; }

/* :first-of-type / :last-of-type */
img:first-of-type { float: left; }

/* :empty — no children (not even whitespace) */
td:empty { background: #f3f4f6; }

/* :root — the <html> element (higher specificity than html selector) */
:root {
  --color-primary: #3b82f6;
}
```

### Logical Pseudo-classes

```css
/* :not() — exclude matching elements */
a:not([href]) { color: gray; cursor: default; }
input:not([type="submit"]):not([type="reset"]) { border: 1px solid #d1d5db; }
li:not(:last-child) { border-bottom: 1px solid #e5e7eb; }

/* :is() — match any selector in the list (takes highest specificity of its args) */
/* Equivalent to writing all three separately */
:is(h1, h2, h3) { font-weight: 700; }
:is(article, section) > p { margin-bottom: 1rem; }

/* :where() — same as :is() but ALWAYS specificity 0 */
/* Perfect for resets and base styles you want overridable */
:where(h1, h2, h3, h4, h5, h6) {
  margin-block: 0;
  line-height: 1.2;
}

/* :has() — "parent selector" and relational selector */
/* Container with an image: add padding */
.card:has(img) { padding-top: 0; }

/* Form field that is currently invalid and has been interacted with */
.form-group:has(input:not(:placeholder-shown):invalid) label {
  color: #ef4444;
}

/* Figure that doesn't have a figcaption */
figure:not(:has(figcaption)) img {
  border-radius: 0.5rem;
}

/* Navigation with more than 5 links */
nav:has(> ul > li:nth-child(6)) {
  font-size: 0.875rem;
}
```

:::tip[`:where()` vs `:is()` Specificity]
Both `:is()` and `:where()` accept a selector list and match any element that matches any selector in the list. The **only difference** is specificity:
- `:is(h1, .title, #heading)` has specificity `(0, 1, 0, 0)` — the highest of its arguments
- `:where(h1, .title, #heading)` always has specificity `(0, 0, 0, 0)`

Use `:where()` in CSS resets and base styles so they're easily overridden. Use `:is()` when you want the specificity to match what you'd get writing the selectors separately.
:::

### Form Pseudo-classes

```css
input:enabled  { background: white; }
input:disabled { background: #f3f4f6; cursor: not-allowed; }

input:checked + label { font-weight: bold; }

input:required { border-left: 3px solid #ef4444; }
input:optional { border-left: 3px solid #d1d5db; }

input:valid   { border-color: #22c55e; }
input:invalid { border-color: #ef4444; }

/* Only show validation styles after the user has interacted */
input:not(:placeholder-shown):invalid {
  border-color: #ef4444;
  background: #fef2f2;
}

input:in-range     { border-color: #22c55e; }
input:out-of-range { border-color: #ef4444; }

input:read-only  { background: #f9fafb; }
input:read-write { background: white; }

/* :placeholder-shown — input currently showing placeholder text */
input:placeholder-shown { font-style: italic; }
```

### Other Pseudo-classes

```css
/* :target — element whose ID matches the URL fragment (#section-1) */
:target {
  background: #fef9c3;
  outline: 2px solid #eab308;
}

/* :lang() — element with a matching language */
:lang(ar) { direction: rtl; }
:lang(ja) { font-family: 'Noto Sans JP', sans-serif; }

/* :any-link — matches both :link and :visited */
:any-link { color: #3b82f6; }

/* :local-link — same-origin links (Level 4 — limited support) */
```

## Pseudo-elements

Pseudo-elements create virtual elements in the document:

```css
/* ::before / ::after — inject content before/after element content */
.badge::before {
  content: '🆕 ';
}

.external-link::after {
  content: ' ↗';
  font-size: 0.75em;
}

/* Empty content — still useful for visual effects */
.divider::after {
  content: '';
  display: block;
  height: 1px;
  background: #e5e7eb;
  margin-block: 2rem;
}

/* ::placeholder — style input placeholder text */
input::placeholder {
  color: #9ca3af;
  font-style: italic;
}

/* ::selection — style text selected by the user */
::selection {
  background: #3b82f6;
  color: white;
}

/* ::marker — style list item bullet or number */
li::marker {
  color: #3b82f6;
  font-weight: bold;
}

ul li::marker { content: '→ '; }
ol li::marker { color: #6b7280; }

/* ::first-line — style the first rendered line */
p::first-line {
  font-variant: small-caps;
  font-weight: bold;
}

/* ::first-letter — style the first letter (drop cap effect) */
p:first-of-type::first-letter {
  font-size: 3.5rem;
  font-weight: 700;
  float: left;
  line-height: 1;
  margin-right: 0.1em;
  color: #3b82f6;
}

/* ::backdrop — behind fullscreen elements and <dialog> */
dialog::backdrop {
  background: rgba(0, 0, 0, 0.7);
  backdrop-filter: blur(4px);
}

/* ::cue — style WebVTT captions in <video> */
::cue {
  font-size: 1rem;
  background: rgba(0, 0, 0, 0.8);
  color: white;
}

/* ::slotted() — style slotted content in Shadow DOM */
::slotted(p) { margin: 0; }

/* ::part() — style Shadow DOM parts exposed via the part attribute */
custom-button::part(button) {
  background: #3b82f6;
}
```

## Selector Performance

Browsers evaluate selectors right-to-left. Keep the **key selector** (rightmost) as specific as possible:

```css
/* Slow: browser finds all <a>, checks if inside .nav, then inside #site-header */
#site-header .nav a { }

/* Fast: browser finds all .nav-link elements, checks ancestry */
.nav-link { }

/* Avoid deeply nested selectors */
/* Bad */
.page > .content > .sidebar > .widget > .widget-title { }

/* Good */
.widget-title { }
```

:::caution[Don't Over-optimise Selectors]
CSS selector performance is rarely a bottleneck in real applications. Modern browsers are highly optimised. Write selectors for **readability and maintainability** first — only investigate selector performance if you have a measured performance problem.
:::

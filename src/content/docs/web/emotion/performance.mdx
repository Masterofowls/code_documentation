Emotion is designed to be fast, but a few key patterns make the difference between good and excellent performance — especially for server-rendered applications.

## How Emotion Renders Styles

Understanding Emotion's rendering pipeline helps you optimize it:

1. **Component renders** — a styled component or `css` prop is evaluated
2. **Style serialization** — Emotion converts the CSS to a string and hashes it to a unique class name
3. **Cache lookup** — if the class name already exists in the cache, skip to step 5
4. **Style injection** — the CSS rule is injected into a `<style>` tag in the document head
5. **Class name applied** — the generated class name is applied to the element

**Key insight:** Identical style objects produce the same class name. Emotion deduplicates styles automatically.

```tsx
// These two elements get the same class name — only one CSS rule is injected
<div css={{ color: 'red', fontSize: 14 }}>A</div>
<div css={{ color: 'red', fontSize: 14 }}>B</div>
```

## SSR with `extractCritical`

Without SSR extraction, users see a flash of unstyled content (FOUC) on first load. Emotion provides two server-side APIs.

### `extractCritical` (simpler, legacy)

```tsx
// pages/_document.tsx (Next.js Pages Router)
import Document, {
  Html,
  Head,
  Main,
  NextScript,
  DocumentContext,
} from 'next/document';
import { extractCritical } from '@emotion/server';

export default class MyDocument extends Document {
  static async getInitialProps(ctx: DocumentContext) {
    const originalRenderPage = ctx.renderPage;

    ctx.renderPage = () =>
      originalRenderPage({
        enhanceApp: (App) =>
          function EnhancedApp(props) {
            return <App {...props} />;
          },
      });

    const initialProps = await Document.getInitialProps(ctx);
    const critical = extractCritical(initialProps.html);

    return {
      ...initialProps,
      styles: (
        <>
          {initialProps.styles}
          <style
            data-emotion-css={critical.ids.join(' ')}
            dangerouslySetInnerHTML={{ __html: critical.css }}
          />
        </>
      ),
    };
  }

  render() {
    return (
      <Html lang="en">
        <Head />
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    );
  }
}
```

### `extractCriticalToChunks` + `CacheProvider` (recommended)

This approach is more correct — it handles SSR and hydration without class name mismatches:

<Steps>
1. **Install the server package**

```bash
npm install @emotion/server @emotion/cache
```

2. **Create a shared cache factory**

   ```ts
   // lib/emotion-cache.ts
   import createCache from '@emotion/cache';
   import type { EmotionCache } from '@emotion/cache';

   export function createEmotionCache(): EmotionCache {
     return createCache({
       key: 'css',
       prepend: true, // Insert Emotion styles before other styles in <head>
     });
   }
   ```

3. **Update `_document.tsx`**

   ```tsx
   // pages/_document.tsx
   import Document, {
     Html,
     Head,
     Main,
     NextScript,
     type DocumentContext,
     type DocumentInitialProps,
   } from 'next/document';
   import createEmotionServer from '@emotion/server/create-instance';
   import { createEmotionCache } from '../lib/emotion-cache';

   export default class MyDocument extends Document {
     static async getInitialProps(
       ctx: DocumentContext,
     ): Promise<DocumentInitialProps> {
       const cache = createEmotionCache();
       const { extractCriticalToChunks, constructStyleTagsFromChunks } =
         createEmotionServer(cache);

       const originalRenderPage = ctx.renderPage;
       ctx.renderPage = () =>
         originalRenderPage({
           enhanceApp: (
             App: React.ComponentType<React.ComponentProps<typeof App>>,
           ) =>
             function EnhancedApp(props) {
               return (
                 <CacheProvider value={cache}>
                   <App {...props} />
                 </CacheProvider>
               );
             },
         });

       const initialProps = await Document.getInitialProps(ctx);
       const emotionStyles = extractCriticalToChunks(initialProps.html);
       const emotionStyleTags = constructStyleTagsFromChunks(emotionStyles);

       return {
         ...initialProps,
         styles: (
           <>
             {initialProps.styles}
             {/* Inject extracted Emotion styles as <style> tags */}
             <div dangerouslySetInnerHTML={{ __html: emotionStyleTags }} />
           </>
         ),
       };
     }

     render() {
       return (
         <Html lang="en">
           <Head />
           <body>
             <Main />
             <NextScript />
           </body>
         </Html>
       );
     }
   }
   ```

4. **Wrap `_app.tsx` with CacheProvider**

   ```tsx
   // pages/_app.tsx
   import type { AppProps } from 'next/app';
   import { CacheProvider } from '@emotion/react';
   import { createEmotionCache } from '../lib/emotion-cache';
   import { ThemeProvider } from '@emotion/react';
   import { theme } from '../lib/theme';

   // Create a single cache instance for the client — reused across renders
   const clientSideEmotionCache = createEmotionCache();

   export default function App({
     Component,
     emotionCache = clientSideEmotionCache,
     pageProps,
   }: AppProps & { emotionCache?: import('@emotion/cache').EmotionCache }) {
     return (
       <CacheProvider value={emotionCache}>
         <ThemeProvider theme={theme}>
           <Component {...pageProps} />
         </ThemeProvider>
       </CacheProvider>
     );
   }
   ```

</Steps>

## Next.js App Router + Emotion

The App Router uses React Server Components, which are incompatible with Emotion at the root. Move Emotion to a client boundary:

```tsx
// app/EmotionRegistry.tsx
'use client';

import { useState } from 'react';
import { useServerInsertedHTML } from 'next/navigation';
import { CacheProvider } from '@emotion/react';
import createCache from '@emotion/cache';

export function EmotionRegistry({ children }: { children: React.ReactNode }) {
  const [cache] = useState(() => {
    const c = createCache({ key: 'css' });
    c.compat = true;
    return c;
  });

  useServerInsertedHTML(() => {
    const names = Object.keys(cache.inserted);
    if (!names.length) return null;

    let styles = '';
    const dataEmotion = names.join(' ');

    for (const name of names) {
      if (cache.inserted[name] !== true) {
        styles += cache.inserted[name];
      }
    }

    return (
      <style
        key="emotion"
        data-emotion={`${cache.key} ${dataEmotion}`}
        // eslint-disable-next-line react/no-danger
        dangerouslySetInnerHTML={{ __html: styles }}
      />
    );
  });

  return <CacheProvider value={cache}>{children}</CacheProvider>;
}
```

```tsx
// app/layout.tsx
import { EmotionRegistry } from './EmotionRegistry';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <EmotionRegistry>{children}</EmotionRegistry>
      </body>
    </html>
  );
}
```

:::caution[Emotion + Server Components]
Emotion requires a client-side JS runtime. Components that use `css` prop or `styled` must be marked `'use client'` or be rendered inside a client boundary. Consider CSS Modules for Server Components.
:::

## Avoiding Style Recalculation

### Define Styles Outside Components

The most common performance mistake is defining styles inside the component body — this creates a new style object on every render:

```tsx
// ❌ New object on every render — Emotion re-serializes each time
function BadComponent() {
  return (
    <div css={{ padding: 16, borderRadius: 8, background: 'white' }}>
      Content
    </div>
  );
}

// ✅ Defined once at module level — Emotion serializes once, reuses class name
const containerStyle = css`
  padding: 16px;
  border-radius: 8px;
  background: white;
`;

function GoodComponent() {
  return <div css={containerStyle}>Content</div>;
}
```

### Memoizing Dynamic Styles

When styles depend on props or state, memoize them to avoid recomputation:

```tsx
import { useMemo } from 'react';
import { css } from '@emotion/react';

type ColorProps = { hue: number; saturation: number };

function ColorSwatch({ hue, saturation }: ColorProps) {
  // Memoize — only recomputes when hue or saturation changes
  const style = useMemo(
    () => css`
      width: 3rem;
      height: 3rem;
      border-radius: 0.5rem;
      background-color: hsl(${hue}, ${saturation}%, 50%);
    `,
    [hue, saturation],
  );

  return <div css={style} />;
}
```

## Memoizing Styled Components

Styled components defined inside other components are recreated on each render, causing every child to remount:

```tsx
import { memo, useMemo } from 'react';
import styled from '@emotion/styled';

// ❌ Created on every parent render — all children remount
function ParentBad({ color }: { color: string }) {
  const Box = styled.div`
    background: ${color};
    padding: 1rem;
  `;

  return <Box>Content</Box>;
}

// ✅ Move static styled components to module scope
const Box = styled.div<{ color: string }>`
  background: ${({ color }) => color};
  padding: 1rem;
`;

function ParentGood({ color }: { color: string }) {
  return <Box color={color}>Content</Box>;
}
```

## babel-plugin-emotion Options

The Babel plugin provides build-time optimizations:

```json
{
  "plugins": [
    [
      "@emotion/babel-plugin",
      {
        "sourceMap": true,
        "autoLabel": "dev-only",
        "labelFormat": "[dirname]--[filename]--[local]",
        "cssPropOptimization": true,
        "importMap": {
          "@ui-library": {
            "styled": {
              "canonicalImport": ["@emotion/styled", "default"]
            },
            "css": {
              "canonicalImport": ["@emotion/react", "css"]
            }
          }
        }
      }
    ]
  ]
}
```

| Option                | Dev | Prod | Description                          |
| --------------------- | --- | ---- | ------------------------------------ |
| `sourceMap`           | ✅  | ❌   | Source maps for CSS debugging        |
| `autoLabel`           | ✅  | ❌   | Adds readable labels to class names  |
| `labelFormat`         | ✅  | —    | Format: `[local]` uses variable name |
| `cssPropOptimization` | ✅  | ✅   | Optimizes `css` prop at compile time |
| `hoist`               | ✅  | ✅   | Hoists static styles to module scope |

### autoLabel in Action

```tsx
// Input code
const styles = css`
  color: red;
`;
const Button = styled.button`
  background: blue;
`;

// With autoLabel + labelFormat: "[local]"
```

## Production vs Development Mode

| Feature         | Development       | Production               |
| --------------- | ----------------- | ------------------------ |
| Source maps     | Yes (with plugin) | No                       |
| Auto labels     | Yes               | No                       |
| Style injection | Runtime           | Runtime + SSR extraction |
| Error messages  | Verbose           | Minimal                  |
| Bundle size     | Larger            | ~7 KB gzip               |

## Best Practices Summary

```tsx
// ✅ 1. Define static styles at module level
const cardStyle = css`
  background: white;
  border-radius: 1rem;
`;

// ✅ 2. Use styled for reusable components with props
const Button = styled.button<{ $primary?: boolean }>`
  background: ${({ $primary }) => ($primary ? '#7c3aed' : '#f3f4f6')};
`;

// ✅ 3. Use $ prefix for transient props (not forwarded to DOM)
const Box = styled.div<{ $rounded?: boolean }>`
  border-radius: ${({ $rounded }) => ($rounded ? '0.5rem' : '0')};
`;

// ✅ 4. Memoize dynamic styles
function Dynamic({ size }: { size: number }) {
  const style = useMemo(
    () => css`
      width: ${size}px;
      height: ${size}px;
    `,
    [size],
  );
  return <div css={style} />;
}

// ✅ 5. Use CacheProvider for SSR
// ✅ 6. Keep ThemeProvider close to root to minimize re-renders
// ✅ 7. Split themes into light/dark objects — avoid computing at runtime

// ❌ Don't do this — inline objects on every render
function Bad() {
  return <div css={{ marginTop: 8 }}>Text</div>; // New object = re-serialization
}

// ✅ Do this instead
const marginStyle = css`
  margin-top: 8px;
`;
function Good() {
  return <div css={marginStyle}>Text</div>;
}
```

:::tip[When to reach for CSS Modules instead]
If you're building a Next.js 13+ App Router project and need maximum performance with React Server Components, CSS Modules or Tailwind CSS are better choices. Emotion requires client-side JavaScript and cannot be used in RSCs without a `'use client'` boundary.
:::

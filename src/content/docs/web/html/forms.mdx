HTML forms are the primary mechanism for collecting user input and sending it to a server. Native HTML form elements come with built-in accessibility, keyboard support, and validation — take advantage of them.

## The `<form>` Element

```html
<form
  action="/api/contact"
  method="post"
  enctype="application/x-www-form-urlencoded"
  novalidate
>
  <!-- form fields -->
  <button type="submit">Send</button>
</form>
```

### Key Attributes

| Attribute | Values | Description |
|---|---|---|
| `action` | URL | Where to send the form data. Defaults to current URL. |
| `method` | `get`, `post` | HTTP method. Use `post` for sensitive data and file uploads. |
| `enctype` | `application/x-www-form-urlencoded` (default), `multipart/form-data`, `text/plain` | How data is encoded. Use `multipart/form-data` for file uploads. |
| `novalidate` | boolean | Disables native browser validation (useful when using custom validation). |
| `autocomplete` | `on`, `off` | Controls browser autocomplete for the entire form. |

:::tip[GET vs POST]
Use `GET` for search forms and filters — the data appears in the URL and can be bookmarked. Use `POST` for login, registration, file uploads, and anything that modifies state on the server.
:::

## Labels

Every input must have a programmatically associated `<label>`. This is both a legal accessibility requirement (WCAG 1.3.1) and a usability improvement — clicking the label focuses its input.

```html
<!-- Preferred: for/id association -->
<div class="field">
  <label for="email">Email address</label>
  <input type="email" id="email" name="email" required />
</div>

<!-- Alternative: wrapping (no need for for/id) -->
<label>
  Password
  <input type="password" name="password" required />
</label>

<!-- Hidden label (last resort — prefer visible labels) -->
<label for="search" class="sr-only">Search</label>
<input type="search" id="search" name="q" placeholder="Search…" />
```

:::caution[Never Use placeholder as a Label Replacement]
Placeholder text disappears when the user starts typing, leaving them without context. It also has poor contrast by default. Always use a visible `<label>`.
:::

## `<fieldset>` and `<legend>`

Group related fields inside `<fieldset>` and label the group with `<legend>`. Essential for radio groups and checkbox groups.

```html
<form action="/register" method="post">
  <fieldset>
    <legend>Account Details</legend>

    <div class="field">
      <label for="reg-email">Email</label>
      <input type="email" id="reg-email" name="email" required autocomplete="email" />
    </div>

    <div class="field">
      <label for="reg-password">Password</label>
      <input type="password" id="reg-password" name="password" required
             autocomplete="new-password" minlength="8" />
    </div>
  </fieldset>

  <fieldset>
    <legend>Personal Details</legend>

    <div class="field">
      <label for="first-name">First name</label>
      <input type="text" id="first-name" name="first_name" required autocomplete="given-name" />
    </div>

    <div class="field">
      <label for="last-name">Last name</label>
      <input type="text" id="last-name" name="last_name" required autocomplete="family-name" />
    </div>
  </fieldset>

  <button type="submit">Create Account</button>
</form>
```

```html
  type="text"
  id="browser"
  name="browser"

  <option value="Edge" />
  <option value="Opera" />

```

## The `<output>` Element

Displays the result of a calculation:

```html
<form oninput="result.value = parseFloat(a.value) * parseFloat(b.value)">
  <label for="operand-a">Value A</label>
  <input type="number" id="operand-a" name="a" value="10" step="0.1" />

  ×

  <label for="operand-b">Value B</label>
  <input type="number" id="operand-b" name="b" value="5" step="0.1" />

  =

  <output name="result" for="operand-a operand-b">50</output>
</form>
```

## Form Validation

### Native Constraint Validation Attributes

```html
<form action="/signup" method="post" novalidate id="signup-form">

  <label for="username">Username</label>
  <input
    type="text"
    id="username"
    name="username"
    required
    minlength="3"
    maxlength="20"
    pattern="[a-zA-Z0-9_]+"
    title="Letters, numbers, and underscores only"
  />

  <label for="age">Age</label>
  <input type="number" id="age" name="age" required min="18" max="120" />

  <label for="site-url">Website</label>
  <input type="url" id="site-url" name="website" />

  <button type="submit">Sign Up</button>
</form>
```

| Attribute | Elements | Description |
|---|---|---|
| `required` | All inputs | Field must not be empty |
| `minlength` / `maxlength` | text, email, etc. | Character count limits |
| `min` / `max` | number, date, range | Numeric/date range limits |
| `step` | number, range, date | Valid increments |
| `pattern` | text, email, etc. | RegEx the value must match |
| `type` | All | Validates format (email, url, number) |

### CSS `:valid` and `:invalid`

```css
input:invalid {
  border-color: #dc2626;
  box-shadow: 0 0 0 3px rgba(220, 38, 38, 0.2);
}

input:valid {
  border-color: #16a34a;
}

/* Only show invalid styles after the user has interacted */
input:not(:placeholder-shown):invalid {
  border-color: #dc2626;
}
```

### Custom Validity with `setCustomValidity()`

For validation logic that can't be expressed with HTML attributes alone:

```js
const form = document.getElementById('signup-form');
const usernameInput = document.getElementById('username');
const takenUsernames = ['admin', 'root', 'user'];

usernameInput.addEventListener('input', async () => {
  const value = usernameInput.value.trim();

  if (takenUsernames.includes(value)) {
    usernameInput.setCustomValidity('That username is already taken. Please choose another.');
  } else {
    // Clear custom error — allow native validation to proceed
    usernameInput.setCustomValidity('');
  }

  // Trigger native validation UI update
  usernameInput.reportValidity();
});

form.addEventListener('submit', (e) => {
  if (!form.checkValidity()) {
    e.preventDefault();
    // Show errors on all invalid fields
    form.querySelectorAll(':invalid').forEach((field) => {
      field.reportValidity();
    });
  }
});
```

:::tip[Pair novalidate with Custom Validation]
Use `novalidate` on the form to suppress the browser's default validation UI (which is unstyled and inconsistent across browsers), then implement your own using the Constraint Validation API (`checkValidity()`, `setCustomValidity()`, `validity`).
:::

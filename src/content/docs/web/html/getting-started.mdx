This guide walks through the essential HTML elements you need to build any web page. Master these fundamentals and every other HTML topic becomes much easier to understand.

## Basic Document Structure

An HTML document is a tree of nested elements. At the top level sit three mandatory elements:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
  </body>
</html>
```

| Element | Role |
|---|---|
| `<html>` | Root element — wraps everything. Always include `lang` attribute. |
| `<head>` | Document metadata — charset, title, CSS links, meta tags. Nothing here is rendered directly. |
| `<body>` | All visible page content lives here. |

:::tip[Always Set the `lang` Attribute]
`<html lang="en">` tells browsers and assistive technologies what language the page is written in. Use a valid [BCP 47 language tag](https://www.iana.org/assignments/language-subtag-registry/language-subtag-registry): `en`, `en-US`, `fr`, `de`, `ja`, etc.
:::

## Block vs Inline Elements

HTML elements fall into two fundamental display categories:

<Tabs>
  <TabItem label="Block Elements">
    Block elements start on a new line and stretch to fill the full width of their container. They stack vertically.

    ```html
    <div>I am block</div>
    <p>I am also block</p>
    <h2>So am I</h2>
    <ul>
      <li>And me</li>
    </ul>
    ```

    Common block elements: `<div>`, `<p>`, `<h1>`–`<h6>`, `<ul>`, `<ol>`, `<li>`, `<table>`, `<form>`, `<header>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<blockquote>`, `<pre>`, `<hr>`.
  </TabItem>
  <TabItem label="Inline Elements">
    Inline elements flow within a line of text. They only take up as much width as their content.

    ```html
    <p>
      This is <strong>bold</strong>, this is <em>italic</em>,
      and this is a <a href="/">link</a> inside a paragraph.
    </p>
    ```

    Common inline elements: `<a>`, `<span>`, `<strong>`, `<em>`, `<code>`, `<img>`, `<button>`, `<input>`, `<label>`, `<abbr>`, `<cite>`, `<mark>`, `<time>`.
  </TabItem>
</Tabs>

:::caution[Block Inside Inline is Invalid]
Never nest a block element inside an inline element. `<a><div>...</div></a>` is invalid HTML. In HTML5 `<a>` is special — it can wrap block content as long as it contains no interactive descendants.
:::

## Headings

HTML provides six heading levels, `<h1>` through `<h6>`:

```html
<h1>Page Title (one per page)</h1>
<h2>Major Section</h2>
<h3>Subsection</h3>
<h4>Sub-subsection</h4>
<h5>Rarely needed</h5>
<h6>Almost never used</h6>
```

**The one `<h1>` per page rule:** Each page should have exactly one `<h1>` — the main topic of the page. This benefits SEO and assistive technology users who navigate by headings. Never skip heading levels (don't jump from `<h2>` to `<h4>`).

:::tip[Headings Are Structure, Not Style]
Never choose a heading level for its visual size. If you want bigger text, use CSS. Choose headings based on the document outline they create.
:::

## Paragraphs

```html
<p>
  This is a paragraph. Browsers automatically add spacing above and below.
  Line breaks in your source code are collapsed into a single space.
</p>

<p>
  This is the second paragraph. Even if you put
  lots of spaces           here,
  the browser renders just one space between words.
</p>
```

Paragraphs are the bread and butter of body text. Never use empty `<p>` tags or `<br>` chains to create visual spacing — use CSS margins instead.

## Links

The anchor element `<a>` creates hyperlinks:

```html
<!-- External link — always use rel="noopener" -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  Visit Example
</a>

<!-- Internal link (relative path) -->
<a href="/about">About Us</a>

<!-- Same-page anchor link -->
<a href="#contact">Jump to Contact Section</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Us</a>

<!-- Phone link (clickable on mobile) -->
<a href="tel:+15551234567">Call Us</a>

<!-- Download link -->
<a href="/report.pdf" download="annual-report-2025.pdf">Download Report</a>
```

### Key Link Attributes

| Attribute | Purpose |
|---|---|
| `href` | The URL the link points to. Required. |
| `target="_blank"` | Open in a new tab. |
| `rel="noopener noreferrer"` | Security: prevents the opened page from accessing `window.opener`. Always pair with `target="_blank"`. |
| `rel="nofollow"` | Tells search engines not to follow this link. |
| `download` | Prompts file download instead of navigation. |

## Images

```html
<!-- Basic image -->
<img
  src="/images/hero.jpg"
  alt="A mountain landscape at sunrise"
  width="800"
  height="450"
  loading="lazy"
/>

<!-- Decorative image (empty alt) -->
<img src="/icons/divider.svg" alt="" width="100" height="4" />

<!-- Image with caption (semantic) -->
<figure>
  <img
    src="/images/diagram.png"
    alt="Flowchart showing the authentication process"
    width="600"
    height="400"
  />
  <figcaption>Figure 1: Authentication flow diagram.</figcaption>
</figure>
```

### Image Attributes Explained

| Attribute | Purpose |
|---|---|
| `src` | Path or URL to the image file. Required. |
| `alt` | Alternative text for screen readers and when the image fails to load. Always include. Leave empty (`alt=""`) for decorative images. |
| `width` / `height` | Intrinsic dimensions in pixels. **Always include both** to prevent layout shift (CLS). |
| `loading="lazy"` | Defers loading until the image is near the viewport. Use for below-the-fold images. |
| `decoding="async"` | Decodes the image off the main thread to reduce jank. |

:::caution[Never Omit `alt` on Informative Images]
An image without `alt` is inaccessible to screen reader users. If the image conveys meaning, describe that meaning. If it is purely decorative, use `alt=""` explicitly.
:::

## Lists

HTML provides three types of lists:

<Tabs>
  <TabItem label="Unordered List">
    ```html
    <!-- Bullet points — order doesn't matter -->
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
    ```
  </TabItem>
  <TabItem label="Ordered List">
    ```html
    <!-- Numbered — sequence matters -->
    <ol>
      <li>Boil the water</li>
      <li>Add pasta</li>
      <li>Cook for 10 minutes</li>
    </ol>

    <!-- Start at a different number -->
    <ol start="5">
      <li>Step five</li>
      <li>Step six</li>
    </ol>

    <!-- Reversed numbering -->
    <ol reversed>
      <li>First (shown as 3)</li>
      <li>Second (shown as 2)</li>
      <li>Third (shown as 1)</li>
    </ol>
    ```
  </TabItem>
  <TabItem label="Description List">
    ```html
    <!-- Term/definition pairs — great for glossaries and metadata -->
    <dl>
      <dt>HTML</dt>
      <dd>HyperText Markup Language — the structure of the web.</dd>

      <dt>CSS</dt>
      <dd>Cascading Style Sheets — the styling of the web.</dd>

      <dt>JavaScript</dt>
      <dd>A scripting language for interactive behaviour on the web.</dd>
    </dl>
    ```
  </TabItem>
</Tabs>

Lists can be nested by placing a new list inside an `<li>` element:

```html
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>Backend
    <ul>
      <li>Node.js</li>
      <li>Python</li>
    </ul>
  </li>
</ul>
```

## Tables

Tables are for **tabular data** — never for layout.

```html
<table>
  <!-- Caption describes the table for all users -->
  <caption>Monthly Sales Figures — Q1 2025</caption>

  <thead>
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Revenue</th>
      <th scope="col">Units Sold</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">January</th>
      <td>$42,000</td>
      <td>210</td>
    </tr>
    <tr>
      <th scope="row">February</th>
      <td>$38,500</td>
      <td>192</td>
    </tr>
    <tr>
      <th scope="row">March</th>
      <td>$51,200</td>
      <td>256</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td>$131,700</td>
      <td>658</td>
    </tr>
  </tfoot>
</table>
```

### Table Accessibility

- `<caption>` gives the table a title visible to all users including screen readers.
- `<th scope="col">` identifies column headers.
- `<th scope="row">` identifies row headers.
- `scope` allows screen readers to announce which header each data cell belongs to.
- For complex tables with merged cells, use `id` and `headers` attributes.

### Spanning Cells

```html
<tr>
  <!-- Spans 2 columns -->
  <td colspan="2">Full Name</td>
  <!-- Spans 3 rows -->
  <td rowspan="3">Combined Cell</td>
</tr>
```

## Line Breaks and Horizontal Rules

```html
<!-- Force a line break within a paragraph (use sparingly) -->
<p>
  123 Main Street<br />
  Springfield, IL 62701
</p>

<!-- Thematic break / horizontal rule -->
<section>
  <h2>Chapter 1</h2>
  <p>Content...</p>
</section>
<hr />
<section>
  <h2>Chapter 2</h2>
  <p>More content...</p>
</section>
```

`<br>` should only be used where line breaks are semantically meaningful (poetry, addresses). Never use it to add vertical spacing — that's CSS's job.

`<hr>` represents a **thematic break** between paragraphs or sections. Style it with CSS.

## Character Entities

Some characters have special meaning in HTML and must be escaped:

| Character | Entity Name | Entity Number | Use Case |
|---|---|---|---|
| `&` | `&amp;` | `&#38;` | Ampersand in text |
| `<` | `&lt;` | `&#60;` | Less-than sign |
| `>` | `&gt;` | `&#62;` | Greater-than sign |
| `"` | `&quot;` | `&#34;` | Quote inside attribute values |
| `'` | `&apos;` | `&#39;` | Apostrophe inside attribute values |
| Non-breaking space | `&nbsp;` | `&#160;` | Prevent line break between words |
| © | `&copy;` | `&#169;` | Copyright symbol |
| ® | `&reg;` | `&#174;` | Registered trademark |
| ™ | `&trade;` | `&#8482;` | Trademark symbol |
| — | `&mdash;` | `&#8212;` | Em dash |
| – | `&ndash;` | `&#8211;` | En dash |
| … | `&hellip;` | `&#8230;` | Ellipsis |

```html
<p>AT&amp;T announced revenue of &gt;$30 billion last quarter.</p>
<p>&copy; 2025 My Company. All rights reserved.</p>
<p>Price: $9&nbsp;99 (non-breaking space between dollars and cents)</p>
```

:::tip[Modern HTML Uses UTF-8]
Because we declare `<meta charset="UTF-8">`, you can write most special characters directly in your HTML source — `©`, `—`, `€`, `→`. You don't need entities for those. Entities are mainly needed for `<`, `>`, `&`, and `"` inside attribute values.
:::

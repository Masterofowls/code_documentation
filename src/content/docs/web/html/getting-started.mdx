This guide covers the essential HTML elements you will use on every project. Once you understand these building blocks, you can construct any page structure.

## Basic Document Structure

Every HTML page needs the same skeleton before you add content:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My First Page</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to my page.</p>
  </body>
</html>
```

The `<head>` element contains **metadata** — information about the document that is not directly displayed. The `<body>` contains everything the user sees and interacts with.

## Block vs Inline Elements

HTML elements are either **block-level** or **inline** by default. This affects how they flow in the document.

| Property              | Block elements                 | Inline elements                     |
| --------------------- | ------------------------------ | ----------------------------------- |
| Line behaviour        | Starts on a new line           | Flows within text                   |
| Width                 | Fills parent width             | As wide as content                  |
| Height/margin/padding | All sides respected            | Top/bottom margins not applied      |
| Examples              | `<div>`, `<p>`, `<h1>`, `<ul>` | `<span>`, `<a>`, `<strong>`, `<em>` |

:::tip[CSS Overrides Display]
The block/inline distinction is the default rendering behaviour. You can always override it with CSS: `display: inline`, `display: block`, `display: inline-block`, etc.
:::

## Headings

HTML provides six heading levels, `<h1>` through `<h6>`. They create a **document outline** that is used by search engines and screen readers.

```html
<h1>Page Title (only one per page)</h1>
<h2>Major Section</h2>
<h3>Subsection</h3>
<h4>Sub-subsection</h4>
<h5>Rarely needed</h5>
<h6>Deepest level</h6>
```

### The One `<h1>` Rule

:::caution[One h1 Per Page]
Each page should have exactly **one `<h1>`** that describes the primary topic. It is both an SEO signal and an accessibility landmark. Screen reader users navigate by heading, so a clear hierarchy is essential.
:::

Keep headings in order — never skip from `<h2>` to `<h4>`. The visual size can always be adjusted with CSS.

## Paragraphs

The `<p>` element groups a block of text into a paragraph. Browsers add margin above and below each paragraph by default.

```html
<p>
  This is a paragraph. It can contain text, inline elements like
  <strong>bold</strong>, <em>italic</em>, <a href="/about">links</a>, and
  <code>code snippets</code>.
</p>

<p>
  A new <code>&lt;p&gt;</code> tag starts a new paragraph. Do not use empty
  paragraphs for spacing — use CSS margins instead.
</p>
```

## Links

The `<a>` (anchor) element creates hyperlinks. The `href` attribute specifies the destination.

```html
<!-- Absolute URL -->
<a href="https://example.com">Visit Example</a>

<!-- Relative URL -->
<a href="/about">About Us</a>

<!-- Anchor within the same page -->
<a href="#section-2">Jump to Section 2</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Us</a>

<!-- Telephone link -->
<a href="tel:+15551234567">Call Us</a>

<!-- Open in new tab — always add rel="noopener noreferrer" -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  External Link
</a>
```

### Link Attributes

| Attribute          | Purpose                                                    |
| ------------------ | ---------------------------------------------------------- |
| `href`             | Destination URL or anchor                                  |
| `target="_blank"`  | Open in a new tab/window                                   |
| `rel="noopener"`   | Prevents new tab accessing `window.opener` (security)      |
| `rel="noreferrer"` | Suppresses the Referer header (privacy + implies noopener) |
| `rel="nofollow"`   | Tells search engines not to follow this link               |
| `download`         | Triggers file download instead of navigation               |

:::caution[Always Use rel="noopener noreferrer" with target="_blank"]
Without `rel="noopener"`, a page opened in a new tab can access the opener's `window` object and redirect the original tab — a phishing vector known as "tab-napping". Always include it.
:::

## Images

```html
<!-- Basic image -->
<img src="photo.jpg" alt="A golden retriever sitting in autumn leaves" />

<!-- With explicit dimensions (prevents layout shift) -->
<img
  src="hero.webp"
  alt="Team working together in a bright office"
  width="1200"
  height="630"
/>

<!-- Lazy load below-the-fold images -->
<img
  src="gallery-item.jpg"
  alt="Abstract painting in shades of blue"
  width="800"
  height="600"
  loading="lazy"
/>
```

### Image Best Practices

- **Always include `alt`** — describes the image for screen readers and when the image fails to load. Use `alt=""` for purely decorative images.
- **Set `width` and `height`** — the browser reserves space before the image loads, preventing **Cumulative Layout Shift (CLS)**.
- **Use `loading="lazy"`** for images below the fold — defers loading until the image is near the viewport.
- **Use modern formats** — WebP is 25–35% smaller than JPEG at equivalent quality.

## Tables

Use tables for **tabular data** — data that has a natural rows-and-columns relationship. Never use tables for page layout.

```html
<table>
  <caption>
    Q1 2024 Sales by Region
  </caption>

  <thead>
    <tr>
      <th scope="col">Region</th>
      <th scope="col">Units Sold</th>
      <th scope="col">Revenue</th>
      <th scope="col">Change YoY</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">North</th>
      <td>1,240</td>
      <td>$62,000</td>
      <td>+12%</td>
    </tr>
    <tr>
      <th scope="row">South</th>
      <td>980</td>
      <td>$49,000</td>
      <td>−3%</td>
    </tr>
    <tr>
      <th scope="row">East</th>
      <td>1,540</td>
      <td>$77,000</td>
      <td>+28%</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td>3,760</td>
      <td>$188,000</td>
      <td>+13%</td>
    </tr>
  </tfoot>
</table>
```

### Accessible Table Attributes

| Attribute/Element                 | Purpose                                           |
| --------------------------------- | ------------------------------------------------- |
| `<caption>`                       | Title for the table — announced by screen readers |
| `<thead>` / `<tbody>` / `<tfoot>` | Groups rows for semantics and styling             |
| `<th>`                            | Header cell — bold/centred by default             |
| `scope="col"`                     | Header applies to the column below                |
| `scope="row"`                     | Header applies to the row to the right            |
| `colspan` / `rowspan`             | Merge cells across columns/rows                   |

## Line Breaks and Horizontal Rules

```html
<!-- br: line break within a block (use sparingly) -->
<address>
  123 Main Street<br />
  London<br />
  SW1A 1AA
</address>

<!-- hr: thematic break between content sections -->
<section>
  <h2>Chapter One</h2>
  <p>...</p>
</section>

<hr />

<section>
  <h2>Chapter Two</h2>
  <p>...</p>
</section>
```

:::tip[Don't Use br for Spacing]
Use `<br>` only where a line break is meaningful (addresses, poetry). For visual spacing between paragraphs or sections, use CSS `margin`.
:::

## HTML Entities

Some characters have special meaning in HTML and must be escaped. Use **named entities** or **numeric character references**:

| Character          | Named entity | Numeric   | Use case                         |
| ------------------ | ------------ | --------- | -------------------------------- |
| `&`                | `&amp;`      | `&#38;`   | Ampersand in text                |
| `&lt;`             | `&lt;`       | `&#60;`   | Less-than sign                   |
| `>`                | `&gt;`       | `&#62;`   | Greater-than sign                |
| `"`                | `&quot;`     | `&#34;`   | Quote inside attributes          |
| `'`                | `&apos;`     | `&#39;`   | Apostrophe (HTML5 only)          |
| non-breaking space | `&nbsp;`     | `&#160;`  | Prevent line break between words |
| ©                  | `&copy;`     | `&#169;`  | Copyright symbol                 |
| ®                  | `&reg;`      | `&#174;`  | Registered trademark             |
| ™                  | `&trade;`    | `&#8482;` | Trademark symbol                 |
| →                  | `&rarr;`     | `&#8594;` | Right arrow                      |
| €                  | `&euro;`     | `&#8364;` | Euro sign                        |

```html
<!-- Correct usage in HTML content -->
<p>Tom &amp; Jerry are classic characters. &copy; 2024 Warner Bros.</p>

<!-- In code examples, always escape angle brackets -->
<p>Use the <code>&lt;p&gt;</code> tag for paragraphs.</p>

<!-- Non-breaking space keeps "No. 1" on one line -->
<p>Ranked&nbsp;No.&nbsp;1 in the country</p>
```

:::tip[UTF-8 Reduces Entity Need]
With `<meta charset="UTF-8">`, you can type most characters directly — `©`, `€`, `→` — without entity escaping. Only escape characters that have syntactic meaning in HTML (`&`, `&lt;`, `>`).
:::

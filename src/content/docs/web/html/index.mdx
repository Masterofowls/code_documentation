HTML (**HyperText Markup Language**) is the standard markup language for creating web pages. It describes the structure of a web page semantically, and was originally designed to define the structure of documents such as headings, paragraphs, lists, links, and images.

Every webpage you visit is built on HTML. It works alongside CSS (for styling) and JavaScript (for behaviour) to form the three core technologies of the web.

## HTML5 vs Older HTML

HTML has evolved significantly over the decades. Understanding what changed with HTML5 helps you write modern, standards-compliant markup.

### Key HTML5 Improvements

| Feature | HTML4 / XHTML | HTML5 |
|---|---|---|
| Doctype | Long, complex string | `<!DOCTYPE html>` |
| Semantic elements | None (`<div>` everywhere) | `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>` |
| Multimedia | Required Flash/plugins | Native `<video>`, `<audio>` |
| Forms | Limited input types | `email`, `date`, `range`, `color`, and more |
| APIs | Minimal browser APIs | Canvas, Web Storage, Web Workers, Geolocation |
| Character encoding | Verbose `<meta http-equiv>` | `<meta charset="UTF-8">` |
| Void element closing | Required `/` in XHTML | Optional in HTML5 |
| Error handling | Strict in XHTML | Forgiving parser |

## The Browser Rendering Pipeline

Understanding how browsers turn HTML text into pixels helps you write faster, more efficient pages.

<Steps>
1. **Bytes → Characters**

   The browser receives raw bytes over the network. Using the declared character encoding (UTF-8), it converts those bytes into individual characters.

2. **Characters → Tokens**

   The HTML parser tokenises the character stream, identifying tags, attributes, and text nodes according to the HTML5 parsing spec.

3. **Tokens → DOM**

   Tokens are assembled into the **Document Object Model (DOM)** — a tree of objects representing every element, attribute, and text node on the page.

4. **CSS → CSSOM**

   Simultaneously, any linked or embedded CSS is parsed into the **CSS Object Model (CSSOM)** — a similar tree representing all styles.

5. **DOM + CSSOM → Render Tree**

   The browser combines the DOM and CSSOM into the **Render Tree**, which includes only visible nodes with their computed styles. `display: none` elements are excluded.

6. **Layout (Reflow)**

   The browser calculates the exact size and position of every render tree node on screen.

7. **Paint**

   Pixels are drawn to layers — text, colours, images, borders, and shadows.

8. **Composite**

   Individual layers are combined (composited) by the GPU and displayed on screen.
</Steps>

:::tip[Minimise Render-Blocking Resources]
`<link rel="stylesheet">` and `<script>` tags without `async` or `defer` block HTML parsing. Place stylesheets in `<head>` and scripts at the end of `<body>` (or use `defer`) to keep the pipeline moving.
:::

## HTML Document Structure

Every valid HTML5 document shares a common skeleton:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Document metadata — not visible to users -->
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Page Title — shown in browser tab</title>
    <meta name="description" content="Page description for search engines." />
    <link rel="stylesheet" href="/styles.css" />
  </head>
  <body>
    <!-- Visible page content goes here -->
    <header>
      <nav aria-label="Main navigation">
        <a href="/">Home</a>
        <a href="/about">About</a>
      </nav>
    </header>

    <main>
      <h1>Main Heading</h1>
      <p>Content begins here.</p>
    </main>

    <footer>
      <p>&copy; 2025 My Company. All rights reserved.</p>
    </footer>

    <!-- Scripts at end of body or use defer -->
    <script src="/app.js" defer></script>
  </body>
</html>
```

## The Doctype Declaration

```html
<!DOCTYPE html>
```

The doctype **must** be the very first line of every HTML document — before even the `<html>` tag. It is not an HTML tag itself; it is an instruction to the browser about which version of HTML the document uses.

- In HTML5, the doctype is deliberately short and simple.
- Without it, browsers enter **Quirks Mode**, emulating bugs from old browsers. Always include it.
- It is case-insensitive (`<!doctype html>` is equally valid).

## Character Encoding

```html
<meta charset="UTF-8" />
```

Always declare your character encoding as the **first element inside `<head>`** — before the `<title>` tag. This prevents the browser from misinterpreting characters before it knows the encoding.

**UTF-8** is the universally recommended encoding for HTML documents. It supports every character in every language, including emoji, mathematical symbols, and special characters.

:::caution[Encoding Mismatch Bugs]
If your server sends `Content-Type: text/html; charset=UTF-8` but your file is saved as Latin-1, users will see garbled characters (mojibake). Ensure your editor, file encoding, server headers, and `<meta charset>` all agree on UTF-8.
:::

## HTML vs XHTML

| Aspect | HTML5 | XHTML 1.x |
|---|---|---|
| Syntax strictness | Forgiving parser | Must be well-formed XML |
| Void elements | `<br>` or `<br />` both valid | Must self-close: `<br />` |
| Attribute values | Quotes optional in some cases | Always required |
| Case sensitivity | Tags case-insensitive | Must be lowercase |
| Error handling | Parser recovers from errors | Invalid XML = parse failure |
| MIME type | `text/html` | `application/xhtml+xml` |
| Practical use today | Universal | Largely obsolete |

XHTML was an attempt to bring XML strictness to HTML but proved too brittle for the open web. HTML5's forgiving parser proved more practical. XHTML is rarely used in new projects today.

## HTML in the Modern Stack

HTML rarely stands alone. In modern web development it is generated or templated by frameworks:

- **React / Next.js** — JSX compiles to `React.createElement()` calls that produce HTML DOM nodes.
- **Vue / Nuxt** — Single File Components (`*.vue`) with `<template>` blocks.
- **Svelte / SvelteKit** — HTML-first `.svelte` files compiled to vanilla JS.
- **Astro** — HTML-centric `.astro` components with zero-JS by default.
- **Traditional templating** — Jinja2 (Python), Blade (Laravel), ERB (Rails), Handlebars.

Regardless of the framework, the output is HTML. Understanding HTML deeply makes you a better developer in any of these ecosystems.

:::tip[Validate Your HTML]
Use the [W3C Markup Validation Service](https://validator.w3.org/) to check any page for errors. Valid HTML is more consistently rendered across browsers, easier to maintain, and more accessible.
:::

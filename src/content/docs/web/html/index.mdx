HTML (**HyperText Markup Language**) is the standard language for creating web pages. It describes the structure and meaning of web content using a system of **elements** represented by tags. Every website you visit is built on HTML — CSS styles it, JavaScript powers its interactivity, but HTML is the foundation.

## HTML5 vs Older HTML

HTML has evolved significantly since its creation by Tim Berners-Lee in 1991. The current living standard is **HTML5**, maintained by the WHATWG.

<Tabs>
<TabItem label="HTML5 (Modern)">

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Modern HTML5 Page</title>
  </head>
  <body>
    <header>
      <nav>...</nav>
    </header>
    <main>
      <article>...</article>
    </main>
    <footer>...</footer>
  </body>
</html>
```

</TabItem>
<TabItem label="HTML 4.01 (Legacy)">

<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />

  </head>

### Key improvements in HTML5

| Feature           | HTML 4 / XHTML               | HTML5                                      |
| ----------------- | ---------------------------- | ------------------------------------------ |
| Doctype           | Long, complex string         | `<!DOCTYPE html>`                          |
| Semantic elements | `<div>` for everything       | `<header>`, `<main>`, `<article>`, etc.    |
| Native media      | Requires Flash/plugins       | `<video>`, `<audio>`                       |
| Form inputs       | `text`, `checkbox`, `radio`… | `email`, `date`, `range`, `color`…         |
| APIs              | Very limited                 | Canvas, Web Workers, Storage, Geolocation… |
| Error handling    | Strict (XHTML) or loose      | Forgiving, well-defined parsing rules      |

:::tip[HTML5 is a Living Standard]
Unlike previous versions, HTML5 is continuously updated by the WHATWG. There is no "HTML6" — features are added incrementally. Always check [MDN](https://developer.mozilla.org) or [caniuse.com](https://caniuse.com) for current support.
:::

## HTML vs XHTML

**XHTML** (Extensible HyperText Markup Language) was a stricter XML-based reformulation of HTML. It required:

- All elements closed (`<br />` instead of `<br>`)
- All attributes quoted
- All tag names in lowercase
- Documents served as `application/xhtml+xml`

Modern HTML5 dropped most of these strict requirements while still encouraging clean markup. XHTML is largely obsolete for new projects.

:::caution[Avoid XHTML for New Projects]
XHTML 1.0 and 1.1 are deprecated. Stick with HTML5. If you need XML-based output (e.g., for RSS or data exchange), use proper XML tools — not XHTML for web pages.
:::

## The Browser Rendering Pipeline

Understanding how browsers turn raw HTML into pixels helps you write better, faster code.

<Steps>
1. **Navigation** — Browser requests the HTML document over HTTP/HTTPS.

2. **Parsing (Tokenization)** — The HTML parser reads bytes, converts them to characters, then tokenizes them into start tags, end tags, and text nodes.

3. **DOM Construction** — Tokens are assembled into the **Document Object Model (DOM)** — a tree of nodes representing the page structure.

4. **CSSOM Construction** — CSS files are fetched and parsed into the **CSS Object Model (CSSOM)**.

5. **Render Tree** — DOM + CSSOM are combined into a render tree that contains only visible nodes with their computed styles.

6. **Layout (Reflow)** — The browser calculates the exact position and size of every element on screen.

7. **Paint** — Pixels are drawn for each visible element (backgrounds, text, borders, images).

8. **Compositing** — Painted layers are combined and displayed on screen using the GPU.
   </Steps>

:::tip[Minimize Reflow]
Operations that change element geometry (size, position) trigger **reflow**, which is expensive. Batch DOM reads and writes together, use CSS transforms instead of changing `top`/`left`, and prefer `will-change` sparingly for animation-heavy elements.
:::

## HTML Document Structure

Every valid HTML document follows this structure:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Document metadata — not displayed -->
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Page Title — shown in browser tab</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <!-- Visible content goes here -->
    <h1>Hello, World!</h1>
    <p>This is a paragraph.</p>

    <!-- Scripts at end of body for performance -->
    <script src="app.js" defer></script>
  </body>
</html>
```

### The `<!DOCTYPE html>` Declaration

The doctype is **not an HTML tag** — it is an instruction to the browser about which version of HTML the document uses. For HTML5, it is simply:

```html
<!DOCTYPE html>
```

Without a doctype, browsers enter **quirks mode** — a backwards-compatibility mode that emulates old, inconsistent browser behaviour. Always include the doctype as the very first line.

### Character Encoding

Always declare `UTF-8` encoding. It supports every character in Unicode, including emoji, non-Latin scripts, and special symbols.

```html
<meta charset="UTF-8" />
```

Place this as the **first element inside `<head>`**, before `<title>`, so the browser knows the encoding before it parses any text content.

### The `lang` Attribute

The `lang` attribute on the root `<html>` element tells browsers, search engines, and assistive technologies what language the page is in:

```html
<html lang="en">
  <!-- English -->
  <html lang="fr">
    <!-- French -->
    <html lang="zh-Hans">
      <!-- Simplified Chinese -->
      <html lang="ar">
        <!-- Arabic (RTL) -->
      </html>
    </html>
  </html>
</html>
```

This is critical for screen readers (to select the correct voice/accent), translation tools, and SEO.

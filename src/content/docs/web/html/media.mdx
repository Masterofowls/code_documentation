HTML provides native elements for embedding images, video, audio, and other external content. Used correctly, they deliver great performance, accessibility, and a responsive user experience.

## Responsive Images

### `srcset` and `sizes`

The `srcset` attribute lets the browser choose the best image based on screen resolution and viewport size:

```html
<!-- srcset with pixel density descriptors (for icons, logos) -->
<img
  src="logo.png"
  srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x"
  alt="Acme Corp"
  width="200"
  height="60"
/>

<!-- srcset with width descriptors + sizes (for content images) -->
<img
  src="hero-800.jpg"
  srcset="
    hero-400.jpg   400w,
    hero-800.jpg   800w,
    hero-1200.jpg 1200w,
    hero-1600.jpg 1600w
  "
  sizes="
    (max-width: 480px)  100vw,
    (max-width: 768px)  100vw,
    (max-width: 1280px)  50vw,
    33vw
  "
  alt="Team photo from our annual summit"
  width="1600"
  height="900"
  loading="lazy"
/>
```

The browser uses `sizes` to understand how large the image will be displayed, then picks the smallest image from `srcset` that is still large enough for that display size, accounting for device pixel ratio.

### The `<picture>` Element

`<picture>` gives you explicit control over which image file is served. Use it for:

- **Art direction** — different crops for different screen sizes
- **Format selection** — serve WebP to browsers that support it, JPEG as fallback

```html
<!-- Art direction: different crops -->
<picture>
  <!-- Mobile: portrait crop -->
  <source
    media="(max-width: 767px)"
    srcset="hero-portrait-400.jpg 400w, hero-portrait-800.jpg 800w"
    sizes="100vw"
  />
  <!-- Tablet: square crop -->
  <source
    media="(max-width: 1023px)"
    srcset="hero-square-600.jpg 600w, hero-square-1200.jpg 1200w"
    sizes="100vw"
  />
  <!-- Desktop: landscape crop (fallback img) -->
  <img
    src="hero-landscape-1600.jpg"
    srcset="hero-landscape-800.jpg 800w, hero-landscape-1600.jpg 1600w"
    sizes="100vw"
    alt="Our headquarters building at sunset"
    width="1600"
    height="900"
  />
</picture>

<!-- Format selection: WebP with JPEG fallback -->
<picture>
  <source
    type="image/avif"
    srcset="photo-400.avif 400w, photo-800.avif 800w, photo-1200.avif 1200w"
    sizes="(max-width: 768px) 100vw, 50vw"
  />
  <source
    type="image/webp"
    srcset="photo-400.webp 400w, photo-800.webp 800w, photo-1200.webp 1200w"
    sizes="(max-width: 768px) 100vw, 50vw"
  />
  <img
    src="photo-800.jpg"
    srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
    sizes="(max-width: 768px) 100vw, 50vw"
    alt="Mountain landscape at dusk"
    width="1200"
    height="800"
    loading="lazy"
  />
</picture>
```

:::tip[Browser Source Order Matters]
The browser uses the **first `<source>` whose conditions match**. Always put your most specific or most preferred format first (AVIF → WebP → JPEG). The `<img>` fallback is used when no `<source>` matches.
:::

## Video

```html
<video
  controls
  width="1280"
  height="720"
  poster="video-thumbnail.jpg"
  preload="metadata"
>
  <!-- Serve WebM to browsers that support it; MP4 as universal fallback -->
  <source src="video.webm" type="video/webm" />
  <source src="video.mp4" type="video/mp4" />

  <!-- Captions / subtitles track -->
  <track
    kind="subtitles"
    src="captions-en.vtt"
    srclang="en"
    label="English"
    default
  />
  <track
    kind="subtitles"
    src="captions-fr.vtt"
    srclang="fr"
    label="Français"
  />

  <!-- Fallback message for ancient browsers -->
  <p>Your browser doesn't support HTML video. <a href="video.mp4">Download it</a>.</p>
</video>

<!-- Background / decorative video (no controls) -->
<video
  autoplay
  muted
  loop
  playsinline
  poster="bg-poster.jpg"
  aria-hidden="true"
>
  <source src="bg-video.webm" type="video/webm" />
  <source src="bg-video.mp4" type="video/mp4" />
</video>
```

### Video Attributes Reference

| Attribute | Description |
|---|---|
| `controls` | Show browser-native playback controls |
| `autoplay` | Start playing immediately (requires `muted` in most browsers) |
| `muted` | Mute audio by default |
| `loop` | Restart when it reaches the end |
| `playsinline` | Play inline on iOS (not fullscreen) |
| `poster` | Thumbnail image shown before play |
| `preload` | `none` / `metadata` / `auto` — hint on how much to buffer |
| `width` / `height` | Reserve space and prevent layout shift |

:::caution[Autoplaying Video with Sound is Blocked]
Browsers block autoplay with audio unless the user has interacted with the page. Use `autoplay muted` for background videos. If you need audio, require a user click to start playback.
:::

## Audio

```html
<audio controls preload="metadata">
  <source src="podcast-episode.opus" type="audio/ogg; codecs=opus" />
  <source src="podcast-episode.mp3" type="audio/mpeg" />
  <p>Your browser doesn't support audio. <a href="podcast-episode.mp3">Download MP3</a>.</p>
</audio>

<!-- Programmatic audio control -->
<button type="button" id="play-btn" aria-label="Play track">▶ Play</button>
<audio id="player" src="track.mp3" preload="none"></audio>

<script>
  const player = document.getElementById('player');
  const btn = document.getElementById('play-btn');

  btn.addEventListener('click', () => {
    if (player.paused) {
      player.play();
      btn.textContent = '⏸ Pause';
      btn.setAttribute('aria-label', 'Pause track');
    } else {
      player.pause();
      btn.textContent = '▶ Play';
      btn.setAttribute('aria-label', 'Play track');
    }
  });
</script>
```

## `<iframe>`

Use iframes for embedding third-party content: maps, video embeds, payment widgets.

```html
<!-- YouTube embed -->
<iframe
  src="https://www.youtube.com/embed/dQw4w9WgXcQ"
  title="Never Gonna Give You Up by Rick Astley"
  width="560"
  height="315"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  allowfullscreen
  loading="lazy"
  referrerpolicy="strict-origin-when-cross-origin"
></iframe>

<!-- Sandboxed iframe for untrusted content -->
<iframe
  src="user-content.html"
  sandbox="allow-scripts allow-same-origin"
  title="User-generated content preview"
  width="800"
  height="600"
></iframe>

<!-- Responsive iframe wrapper (16:9 aspect ratio) -->
<div style="position: relative; padding-bottom: 56.25%; height: 0; overflow: hidden;">
  <iframe
    src="https://www.youtube.com/embed/VIDEO_ID"
    title="Video title"
    allow="autoplay; fullscreen"
    loading="lazy"
    style="position: absolute; top: 0; left: 0; width: 100%; height: 100%; border: 0;"
  ></iframe>
</div>
```

### `sandbox` Attribute

The `sandbox` attribute restricts what the iframe content can do:

| Token | Allows |
|---|---|
| *(empty)* | Everything disabled |
| `allow-scripts` | JavaScript execution |
| `allow-same-origin` | Treat as same origin (needed for local storage access) |
| `allow-forms` | Form submission |
| `allow-popups` | Opening new windows |
| `allow-top-navigation` | Navigating the top-level frame |

:::tip[Always Add a title to iframes]
Screen readers announce the iframe's `title` to users. Without it, they will hear "iframe" — not helpful. Use a descriptive title that explains what is embedded.
:::

## Canvas

The `<canvas>` element provides a bitmap drawing surface controlled via JavaScript:

```html
<canvas
  id="game-canvas"
  width="800"
  height="600"
  aria-label="Game viewport"
  role="img"
>
  Your browser doesn't support canvas. Please upgrade.
</canvas>

<script>
  const canvas = document.getElementById('game-canvas');
  const ctx = canvas.getContext('2d');

  // Draw a filled rectangle
  ctx.fillStyle = '#0066cc';
  ctx.fillRect(50, 50, 200, 100);

  // Draw text
  ctx.fillStyle = '#ffffff';
  ctx.font = 'bold 20px sans-serif';
  ctx.fillText('Hello Canvas!', 80, 110);

  // Draw a circle
  ctx.beginPath();
  ctx.arc(400, 300, 80, 0, Math.PI * 2);
  ctx.strokeStyle = '#ff6600';
  ctx.lineWidth = 4;
  ctx.stroke();
</script>
```

:::caution[Canvas Has No Built-In Accessibility]
Content drawn on canvas is not accessible to screen readers. For important information, provide a text alternative via `aria-label`, `role="img"`, or a visible text description nearby.
:::

## Lazy Loading

### Native `loading="lazy"`

```html
<!-- Defer loading until the image is near the viewport -->
<img
  src="article-photo.jpg"
  alt="Team building event in the park"
  width="800"
  height="533"
  loading="lazy"
/>

<!-- iframe lazy loading -->
<iframe
  src="https://maps.google.com/..."
  title="Office location on Google Maps"
  loading="lazy"
  width="600"
  height="450"
></iframe>
```

:::tip[Don't Lazy-Load Above-the-Fold Images]
Never add `loading="lazy"` to the hero image or any image visible on initial page load — it will delay the **Largest Contentful Paint (LCP)**. Lazy load only images that start below the visible viewport.
:::

### Intersection Observer (Advanced Lazy Loading)

For more control — or for loading other resources besides images:

```js
const lazyImages = document.querySelectorAll('img[data-src]');

const observer = new IntersectionObserver(
  (entries, obs) => {
    entries.forEach((entry) => {
      if (!entry.isIntersecting) return;

      const img = entry.target;
      img.src = img.dataset.src;

      if (img.dataset.srcset) {
        img.srcset = img.dataset.srcset;
      }

      img.removeAttribute('data-src');
      img.removeAttribute('data-srcset');
      obs.unobserve(img);
    });
  },
  {
    // Load image when it's 200px from the viewport edge
    rootMargin: '200px 0px',
  }
);

lazyImages.forEach((img) => observer.observe(img));
```

```html
<!-- HTML for Intersection Observer lazy loading -->
<img
  data-src="large-photo.jpg"
  data-srcset="large-photo-800.jpg 800w, large-photo-1600.jpg 1600w"
  src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7"
  alt="Conference hall with attendees"
  width="1600"
  height="900"
/>
```

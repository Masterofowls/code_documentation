Modern web pages are rich with media — images, video, audio, and embedded content. This guide covers how to embed each type correctly, responsively, and accessibly.

## Responsive Images

### `srcset` and `sizes`

`srcset` provides multiple image versions; `sizes` tells the browser how wide the image will be rendered at each breakpoint:

```html
<img
  src="/images/hero-800.jpg"
  srcset="
    /images/hero-400.jpg  400w,
    /images/hero-800.jpg  800w,
    /images/hero-1200.jpg 1200w,
    /images/hero-1600.jpg 1600w
  "
  sizes="
    (max-width: 480px)  100vw,
    (max-width: 768px)  100vw,
    (max-width: 1200px) 50vw,
    800px
  "
  alt="Mountain landscape at sunrise"
  width="800"
  height="450"
  loading="lazy"
/>
```

The browser combines `srcset` widths with `sizes` slot widths and the device pixel ratio to choose the optimal image — no JavaScript required.

### The `<picture>` Element

`<picture>` lets you direct the browser to use different image formats or art-directed crops at different sizes:

```html
<!-- Format selection: WebP with JPEG fallback -->
<picture>
  <source type="image/avif" srcset="/images/hero.avif" />
  <source type="image/webp" srcset="/images/hero.webp" />
  <!-- Fallback for browsers that don't support <picture> or AVIF/WebP -->
  <img
    src="/images/hero.jpg"
    alt="Mountain landscape at sunrise"
    width="1200"
    height="675"
    loading="eager"
    fetchpriority="high"
  />
</picture>

<!-- Art direction: different crops at different sizes -->
<picture>
  <!-- Portrait crop for narrow screens -->
  <source
    media="(max-width: 600px)"
    srcset="/images/hero-portrait-400.webp 400w, /images/hero-portrait-800.webp 800w"
    sizes="100vw"
    type="image/webp"
  />
  <!-- Wide landscape crop for larger screens -->
  <source
    media="(min-width: 601px)"
    srcset="/images/hero-landscape-800.webp 800w, /images/hero-landscape-1600.webp 1600w"
    sizes="100vw"
    type="image/webp"
  />
  <img
    src="/images/hero-landscape-1200.jpg"
    alt="Mountain landscape at sunrise"
    width="1200"
    height="675"
  />
</picture>
```

:::tip[AVIF > WebP > JPEG]
AVIF offers 50% smaller file sizes than JPEG at the same quality, with WebP at roughly 30% smaller. Always list AVIF first, WebP second, and JPEG as the fallback in your `<picture>` sources.
:::

### Image Formats Guide

| Format | Best For | Notes |
|---|---|---|
| AVIF | Photographs, illustrations | Best compression, broad modern browser support |
| WebP | Photographs, illustrations | Good compression, very broad support |
| JPEG / JPG | Photographs | Universal support, lossy |
| PNG | Graphics with transparency, screenshots | Lossless, larger files |
| SVG | Icons, logos, illustrations | Vector, scalable, tiny for simple shapes |
| GIF | Simple animations | Replaced by CSS animations or `<video>` |

## Video

```html
<video
  controls
  width="800"
  height="450"
  poster="/images/video-poster.jpg"
  preload="metadata"
  muted
  playsinline
>
  <!-- Serve most efficient format first -->
  <source src="/videos/demo.webm" type="video/webm" />
  <source src="/videos/demo.mp4" type="video/mp4" />

  <!-- Captions / subtitles (required for accessibility) -->
  <track
    kind="subtitles"
    src="/captions/demo-en.vtt"
    srclang="en"
    label="English"
    default
  />
  <track
    kind="subtitles"
    src="/captions/demo-fr.vtt"
    srclang="fr"
    label="Français"
  />

  <!-- Fallback for very old browsers -->
  <p>
    Your browser does not support HTML video.
    <a href="/videos/demo.mp4">Download the video</a>.
  </p>
</video>
```

### Video Attributes Reference

| Attribute | Purpose |
|---|---|
| `controls` | Show native play/pause/volume controls |
| `autoplay` | Start playing immediately — **requires `muted`** to work in most browsers |
| `muted` | Start muted. Required for autoplay. |
| `loop` | Restart playback when it ends |
| `poster` | Image shown before playback starts |
| `preload` | `none` (don't load), `metadata` (load duration/dimensions), `auto` (load full video) |
| `playsinline` | Play inline on iOS (not fullscreen by default) |
| `width` / `height` | Prevent layout shift — always include |

### `<track>` Kinds

| Kind | Purpose |
|---|---|
| `subtitles` | Translation of dialogue for users who don't understand the language |
| `captions` | Subtitles + sound effects for deaf/hard-of-hearing users |
| `descriptions` | Audio descriptions for blind users |
| `chapters` | Navigation chapters for long videos |
| `metadata` | Data for scripts — not shown to users |

:::caution[Autoplay and Accessibility]
Never autoplay video with sound — it disorients screen reader users and is blocked by most browsers anyway. If you autoplay (e.g., for a background video), always use `muted` and provide a pause button.
:::

## Audio

```html
<audio controls preload="metadata">
  <source src="/audio/podcast-ep-42.opus" type="audio/ogg; codecs=opus" />
  <source src="/audio/podcast-ep-42.mp3" type="audio/mpeg" />
  <p>
    Your browser doesn't support HTML audio.
    <a href="/audio/podcast-ep-42.mp3">Download the episode</a>.
  </p>
</audio>
```

Audio elements support the same `controls`, `autoplay`, `muted`, `loop`, and `preload` attributes as video. Always provide a transcript for audio content — essential for deaf users and great for SEO.

## Iframes

```html
<!-- Embedded map -->
<iframe
  src="https://www.openstreetmap.org/export/embed.html?bbox=-0.13,51.49,-0.11,51.52"
  width="600"
  height="450"
  loading="lazy"
  title="OpenStreetMap showing central London"
  allowfullscreen
></iframe>

<!-- Embedded YouTube video -->
<iframe
  src="https://www.youtube-nocookie.com/embed/dQw4w9WgXcQ"
  width="560"
  height="315"
  title="YouTube video player"
  loading="lazy"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  allowfullscreen
  sandbox="allow-scripts allow-same-origin allow-presentation"
></iframe>
```

### `sandbox` Attribute

The `sandbox` attribute applies restrictions to iframe content:

| Value | Allows |
|---|---|
| (empty) | Maximum restriction — almost nothing works |
| `allow-scripts` | JavaScript execution |
| `allow-same-origin` | Treats content as from same origin |
| `allow-forms` | Form submission |
| `allow-popups` | New windows via `window.open()` |
| `allow-presentation` | Fullscreen and Presentation API |
| `allow-top-navigation` | Navigating the top-level frame |

:::tip[Use `youtube-nocookie.com` for GDPR]
Embedding YouTube with `youtube-nocookie.com` prevents YouTube from setting cookies until the user actually clicks play.
:::

## Canvas

`<canvas>` provides a 2D (and optionally 3D via WebGL) drawing surface controlled entirely by JavaScript:

```html
<canvas id="my-canvas" width="400" height="300">
  <!-- Fallback content for users without canvas support -->
  <p>Your browser does not support the canvas element.</p>
</canvas>
```

```js
const canvas = document.getElementById('my-canvas');
const ctx = canvas.getContext('2d');

// Draw a filled rectangle
ctx.fillStyle = '#3b82f6';
ctx.fillRect(20, 20, 150, 100);

// Draw text
ctx.font = 'bold 20px sans-serif';
ctx.fillStyle = '#ffffff';
ctx.fillText('Hello Canvas', 35, 80);

// Draw a circle
ctx.beginPath();
ctx.arc(300, 150, 60, 0, Math.PI * 2);
ctx.fillStyle = '#f59e0b';
ctx.fill();
ctx.strokeStyle = '#d97706';
ctx.lineWidth = 3;
ctx.stroke();
```

## SVG

SVG (Scalable Vector Graphics) can be used in four ways:

<Tabs>
  <TabItem label="Inline SVG">
    ```html
    <!-- Best for icons that need CSS styling or animation -->
    <svg
      xmlns="http://www.w3.org/2000/svg"
      viewBox="0 0 24 24"
      width="24"
      height="24"
      aria-hidden="true"
      focusable="false"
    >
      <path d="M12 2L2 7l10 5 10-5-10-5zM2 17l10 5 10-5M2 12l10 5 10-5" />
    </svg>
    ```
    Inline SVG is part of the DOM — you can target its elements with CSS and JavaScript. Best for interactive icons.
  </TabItem>
  <TabItem label="SVG as `<img>`">
    ```html
    <!-- Simple static SVG — can't be styled with CSS -->
    <img
      src="/icons/logo.svg"
      alt="My Company logo"
      width="120"
      height="40"
    />
    ```
    Treated like any other image. Cached by the browser. Cannot access internal SVG elements via CSS/JS.
  </TabItem>
  <TabItem label="SVG as CSS Background">
    ```css
    .icon-home {
      width: 24px;
      height: 24px;
      background-image: url('/icons/home.svg');
      background-size: contain;
      background-repeat: no-repeat;
    }
    ```
    Good for decorative icons. Cannot be coloured with `currentColor`. Hide from screen readers with `aria-hidden`.
  </TabItem>
  <TabItem label="SVG Sprite">
    ```html
    <!-- Define all icons once as hidden sprites -->
    <svg xmlns="http://www.w3.org/2000/svg" style="display:none" aria-hidden="true">
      <symbol id="icon-home" viewBox="0 0 24 24">
        <path d="M3 9l9-7 9 7v11a2 2 0 01-2 2H5a2 2 0 01-2-2z" />
        <polyline points="9,22 9,12 15,12 15,22" />
      </symbol>
      <symbol id="icon-search" viewBox="0 0 24 24">
        <circle cx="11" cy="11" r="8" />
        <line x1="21" y1="21" x2="16.65" y2="16.65" />
      </symbol>
    </svg>

    <!-- Use them anywhere in the document -->
    <svg aria-hidden="true" focusable="false" width="24" height="24">
      <use href="#icon-home" />
    </svg>

    <button>
      <svg aria-hidden="true" focusable="false" width="16" height="16">
        <use href="#icon-search" />
      </svg>
      Search
    </button>
    ```
  </TabItem>
</Tabs>

## Lazy Loading

### Native Lazy Loading

```html
<!-- Images and iframes below the fold -->
<img src="/images/feature.jpg" alt="Feature image" loading="lazy" width="600" height="400" />
<iframe src="https://maps.example.com/embed" loading="lazy" title="Location map"></iframe>

<!-- Hero images should load eagerly (the default) -->
<img src="/images/hero.jpg" alt="Hero image" loading="eager" fetchpriority="high" width="1200" height="675" />
```

### Intersection Observer API

For more control — fade-in animations, loading tracking, analytics:

```html
<img
  data-src="/images/lazy.jpg"
  data-srcset="/images/lazy-400.jpg 400w, /images/lazy-800.jpg 800w"
  alt="Lazy loaded image"
  width="800"
  height="450"
  class="lazy-image"
/>
```

```js
// Create an observer
const imageObserver = new IntersectionObserver((entries, observer) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;

      // Load the image
      img.src = img.dataset.src;
      if (img.dataset.srcset) {
        img.srcset = img.dataset.srcset;
      }

      img.classList.add('loaded');

      // Stop observing once loaded
      observer.unobserve(img);
    }
  });
}, {
  // Start loading 200px before the image enters the viewport
  rootMargin: '200px 0px',
  threshold: 0.01,
});

// Observe all lazy images
document.querySelectorAll('.lazy-image').forEach(img => {
  imageObserver.observe(img);
});
```

```css
.lazy-image {
  opacity: 0;
  transition: opacity 0.4s ease;
}

.lazy-image.loaded {
  opacity: 1;
}
```

:::tip[Prefer Native `loading="lazy"` First]
Native lazy loading is supported by all modern browsers and requires zero JavaScript. Only use Intersection Observer when you need extra control like animations, analytics, or support for placeholder blurring effects.
:::

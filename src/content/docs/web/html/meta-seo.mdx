The `<head>` element is invisible to users but critical for search engines, social platforms, browsers, and performance. Done right, it can significantly improve your search rankings, click-through rates, and page load speed.

## Essential Meta Tags

```html
<head>
  <!-- Character encoding — must be first in <head> -->
  <meta charset="UTF-8" />

  <!-- Responsive viewport — critical for mobile -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- Page title — shown in tabs, search results, bookmarks -->
  <!-- Ideal length: 50–60 characters -->
  <title>Complete HTML Meta & SEO Guide | DevDocs</title>

  <!-- Meta description — shown in search result snippets -->
  <!-- Ideal length: 150–160 characters -->
  <meta
    name="description"
    content="Learn how to use HTML meta tags, Open Graph, JSON-LD structured data, and resource hints to optimise your pages for search engines and social sharing."
  />

  <!-- Canonical URL — prevents duplicate content issues -->
  <link rel="canonical" href="https://devdocs.example.com/html/meta-seo" />

  <!-- Robots directive -->
  <meta name="robots" content="index, follow" />

  <!-- Language -->
  <html lang="en" />
</head>
```

### Robots Meta Directives

| Value | Effect |
|---|---|
| `index` | Allow indexing (default) |
| `noindex` | Do not include in search results |
| `follow` | Follow links on this page (default) |
| `nofollow` | Do not follow links |
| `noarchive` | Do not cache the page |
| `nosnippet` | Do not show a text snippet in results |
| `noimageindex` | Do not index images on this page |

```html
<!-- Noindex staging pages, thank-you pages, duplicate content -->
<meta name="robots" content="noindex, nofollow" />

<!-- Search engine specific -->
<meta name="googlebot" content="max-snippet:160, max-image-preview:large" />
```

## Open Graph

Open Graph tags control how your page appears when shared on Facebook, LinkedIn, Slack, Discord, and other platforms:

```html
<!-- Open Graph (OG) tags -->
<meta property="og:type" content="article" />
<meta property="og:title" content="Complete HTML Meta & SEO Guide" />
<meta
  property="og:description"
  content="Learn how to use HTML meta tags, Open Graph, JSON-LD structured data, and resource hints to optimise your pages."
/>
<meta property="og:url" content="https://devdocs.example.com/html/meta-seo" />
<meta
  property="og:image"
  content="https://devdocs.example.com/og/meta-seo.png"
/>
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:image:alt" content="Text 'HTML Meta & SEO Guide' on a dark background with code snippets" />
<meta property="og:site_name" content="DevDocs" />
<meta property="og:locale" content="en_US" />

<!-- Article-specific OG -->
<meta property="article:published_time" content="2024-03-15T10:00:00Z" />
<meta property="article:modified_time" content="2024-03-20T14:30:00Z" />
<meta property="article:author" content="https://devdocs.example.com/authors/jane-smith" />
<meta property="article:tag" content="HTML" />
<meta property="article:tag" content="SEO" />
```

:::tip[OG Image Best Practices]
- Recommended size: **1200×630px** (1.91:1 ratio)
- Minimum: 600×315px
- Maximum: 8MB
- Use absolute URLs (not relative paths)
- Keep text in the central 1000×520px "safe zone"
:::

## Twitter Card Tags

Twitter/X uses its own card tags (though it also reads Open Graph as a fallback):

```html
<!-- Summary card with large image -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:site" content="@DevDocs" />
<meta name="twitter:creator" content="@janesmith" />
<meta name="twitter:title" content="Complete HTML Meta & SEO Guide" />
<meta
  name="twitter:description"
  content="Learn how to use HTML meta tags, Open Graph, JSON-LD structured data, and resource hints."
/>
<meta
  name="twitter:image"
  content="https://devdocs.example.com/og/meta-seo.png"
/>
<meta name="twitter:image:alt" content="HTML Meta & SEO Guide cover image" />
```

### Twitter Card Types

| Card type | Use case |
|---|---|
| `summary` | Small thumbnail, default card |
| `summary_large_image` | Full-width image, most common for articles |
| `app` | Mobile app cards |
| `player` | Video/audio player cards |

## Favicons

```html
<!-- Classic .ico (all browsers) -->
<link rel="icon" href="/favicon.ico" sizes="any" />

<!-- SVG favicon (modern browsers — scalable, supports dark mode) -->
<link rel="icon" href="/favicon.svg" type="image/svg+xml" />

<!-- PNG fallback for browsers without SVG favicon support -->
<link rel="icon" href="/favicon-32x32.png" type="image/png" sizes="32x32" />
<link rel="icon" href="/favicon-16x16.png" type="image/png" sizes="16x16" />

<!-- Apple touch icon (home screen icon on iOS) -->
<link rel="apple-touch-icon" href="/apple-touch-icon.png" sizes="180x180" />

<!-- Web App Manifest (PWA: defines icons, theme colour, display mode) -->
<link rel="manifest" href="/site.webmanifest" />

<!-- Theme colour (browser chrome colour on mobile) -->
<meta name="theme-color" content="#0066cc" />
<!-- Different theme for dark mode -->
<meta name="theme-color" content="#1a1a2e" media="(prefers-color-scheme: dark)" />
```

### SVG Favicon with Dark Mode Support

```html
<!-- favicon.svg -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <style>
    rect { fill: #0066cc; }
    @media (prefers-color-scheme: dark) {
      rect { fill: #60a5fa; }
    }
  </style>
  <rect width="32" height="32" rx="6" />
  <text x="50%" y="50%" dominant-baseline="central" text-anchor="middle"
        fill="white" font-size="20" font-family="sans-serif" font-weight="bold">D</text>
</svg>
```

## International SEO with `hreflang`

```html
<!-- Tell search engines about language/region variants of the same page -->
<link rel="alternate" hreflang="en" href="https://example.com/en/page" />
<link rel="alternate" hreflang="en-US" href="https://example.com/en-us/page" />
<link rel="alternate" hreflang="en-GB" href="https://example.com/en-gb/page" />
<link rel="alternate" hreflang="fr" href="https://example.com/fr/page" />
<link rel="alternate" hreflang="de" href="https://example.com/de/page" />
<!-- x-default: fallback for users who don't match any variant -->
<link rel="alternate" hreflang="x-default" href="https://example.com/page" />
```

## Resource Hints

Resource hints tell the browser about resources it will need, allowing it to start downloading them early:

```html
<!-- preconnect: establish early connection (DNS + TCP + TLS) to important origins -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />

<!-- dns-prefetch: DNS lookup only (lower priority, wider browser support) -->
<link rel="dns-prefetch" href="https://www.google-analytics.com" />
<link rel="dns-prefetch" href="https://cdn.example.com" />

<!-- preload: fetch current page resources with high priority -->
<!-- Use for: hero image, critical fonts, above-the-fold CSS -->
<link
  rel="preload"
  href="/fonts/inter-var.woff2"
  as="font"
  type="font/woff2"
  crossorigin
/>
<link
  rel="preload"
  href="/hero-image.webp"
  as="image"
  fetchpriority="high"
/>
<link rel="preload" href="/critical.css" as="style" />

<!-- prefetch: fetch resources for the *next* navigation (low priority) -->
<!-- Ideal for pages the user is likely to visit next -->
<link rel="prefetch" href="/next-page.html" />
<link rel="prefetch" href="/images/next-hero.webp" as="image" />

<!-- modulepreload: preload ES module scripts -->
<link rel="modulepreload" href="/app.js" />
```

### Resource Hints Decision Guide

| Hint | When to use |
|---|---|
| `preconnect` | CDNs, font APIs, analytics — origins used early in page load |
| `dns-prefetch` | Third-party origins used later — cheaper than preconnect |
| `preload` | Current page critical resources (LCP image, fonts, key scripts) |
| `prefetch` | Resources needed for the next navigation |
| `modulepreload` | ES module entry points and their dependencies |

:::caution[Don't Over-Preload]
Preloading too many resources competes with critical resources. Each preloaded resource that isn't used within a few seconds generates a browser console warning. Be selective — focus on your LCP image and critical fonts.
:::

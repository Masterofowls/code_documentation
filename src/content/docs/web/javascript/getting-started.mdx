This guide walks you from zero to writing real JavaScript — in the browser console, as a standalone script, and with Node.js.

## Running JavaScript in the Browser

Every modern browser ships a JavaScript console you can use immediately — no install required.

<Steps>
1. Open any browser (Chrome, Firefox, Safari, Edge).
2. Press **F12** (or `Cmd+Option+I` on Mac) to open DevTools.
3. Click the **Console** tab.
4. Type a JavaScript expression and press **Enter**.
</Steps>

```js
// Try these in your browser console right now:
console.log('Hello, world!');          // output: Hello, world!
console.log(2 + 2);                    // output: 4
console.log(typeof 'JavaScript');      // output: string

// The console object has more useful methods:
console.warn('This is a warning');
console.error('This is an error');
console.table([{ name: 'Alice', age: 30 }, { name: 'Bob', age: 25 }]);
console.time('loop');
for (let i = 0; i < 100000; i++) {}
console.timeEnd('loop');               // output: loop: 1.23ms (approximately)
```

:::tip[Multi-line in Console]
Press **Shift+Enter** to insert a new line in the browser console without executing. This lets you write multi-line code before running it.
:::

### The Node.js REPL

Running `node` with no arguments opens the interactive REPL (Read-Eval-Print Loop):

```bash
node
```

```js
// In the REPL:
> 1 + 1
// output: 2
> const greet = name => `Hello, ${name}!`
// output: undefined
> greet('World')
// output: 'Hello, World!'
> .help    // show REPL commands
> .exit    // quit the REPL (or Ctrl+D)
```

### Your First Script File

```bash
# Create a file
echo "console.log('Hello from Node.js!');" > hello.js
node hello.js
# output: Hello from Node.js!
```

For a more complete first script:

```js
// hello.js
const args = process.argv.slice(2); // command-line arguments after 'node script.js'
const name = args[0] ?? 'World';

console.log(`Hello, ${name}!`);
console.log(`Node.js version: ${process.version}`);
console.log(`Platform: ${process.platform}`);
```

```bash
node hello.js Alice
# output: Hello, Alice!
# output: Node.js version: v20.11.0
# output: Platform: linux
```

### `defer` vs `async` vs Neither

| Attribute | Download | Execution | Use Case |
|-----------|----------|-----------|----------|
| *(none)* | Blocks HTML parsing | Immediately, inline | Avoid for external scripts |
| `async` | Parallel | As soon as downloaded (may be before DOM ready) | Independent scripts (analytics) |
| `defer` | Parallel | After HTML is fully parsed, in order | Most scripts — **recommended** |
| `type="module"` | Parallel | Deferred by default, in order | ES module scripts |

```html
<!-- Good: loads in parallel, executes after DOM is ready -->
<script src="app.js" defer></script>

<!-- For analytics that don't need the DOM -->
<script src="analytics.js" async></script>

<!-- ES Module — deferred by default -->
<script type="module" src="app.mjs"></script>
```

## Strict Mode

Strict mode (`"use strict"`) enables a stricter variant of JavaScript that catches common mistakes:

```js
'use strict';

// 1. Variables must be declared
undeclaredVar = 5; // ReferenceError: undeclaredVar is not defined

// 2. Deleting undeletable properties throws
delete Object.prototype; // TypeError

// 3. Duplicate parameter names are not allowed
function bad(a, a) {} // SyntaxError

// 4. `this` in functions without a context is undefined (not window)
function showThis() {
  console.log(this); // output: undefined (in strict mode)
}
showThis();

// 5. `with` statement is forbidden
with (obj) {} // SyntaxError
```

:::tip[ES Modules are Always Strict]
Code inside `<script type="module">` and `.mjs` files is automatically in strict mode. You don't need to add `"use strict"` manually.
:::

## Variables: `var`, `let`, and `const`

```js
// var — function-scoped, hoisted, avoid in modern JS
var x = 1;

// let — block-scoped, not hoisted (TDZ), reassignable
let count = 0;
count = 1; // OK

// const — block-scoped, not hoisted (TDZ), not reassignable
const MAX = 100;
MAX = 200; // TypeError: Assignment to constant variable

// const with objects/arrays — the binding is constant, not the value
const user = { name: 'Alice' };
user.name = 'Bob'; // OK — mutating the object is allowed
user = {};         // TypeError — can't rebind the variable
```

<Tabs>

function example() {
  if (true) {
    var varVariable = 'I am var';    // function-scoped
    let letVariable = 'I am let';    // block-scoped
    const constVar = 'I am const';   // block-scoped
  }

  console.log(varVariable);   // output: I am var
  console.log(letVariable);   // ReferenceError: letVariable is not defined
}
```
</TabItem>

// var in loops — a classic bug
for (var i = 0; i < 3; i++) {

// output: 3  (all closures share the same `i`)

```

## Hoisting

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before execution.

```js
// Function declarations are fully hoisted — callable before declaration
sayHello(); // output: Hello!
function sayHello() {
  console.log('Hello!');
}

// var is hoisted but not initialized
console.log(name);  // output: undefined (not ReferenceError)
var name = 'Alice';
console.log(name);  // output: Alice

// let and const are hoisted but NOT initialized — Temporal Dead Zone (TDZ)
console.log(city);  // ReferenceError: Cannot access 'city' before initialization
let city = 'London';
```

### Temporal Dead Zone (TDZ)

The TDZ is the region from the start of the block until the `let`/`const` declaration is reached. Accessing the variable in the TDZ throws a `ReferenceError`.

```js
{
  // TDZ for `value` starts here
  console.log(typeof value); // ReferenceError (not "undefined" like with var!)
  let value = 42;            // TDZ ends here
  console.log(value);        // output: 42
}
```

:::caution[typeof is Not Safe with let/const]
With `var`, `typeof undeclaredVar` returns `"undefined"`. With `let`/`const`, it throws a `ReferenceError` if accessed before initialization.
:::

## Type Coercion

JavaScript automatically converts values between types in certain situations — this is **implicit coercion**.

```js
// Loose equality (==) coerces types before comparing
console.log(1 == '1');    // output: true  (string '1' coerced to number)
console.log(0 == false);  // output: true  (false coerced to 0)
console.log(null == undefined); // output: true  (special case)
console.log(null == 0);   // output: false (null only equals undefined with ==)

// Strict equality (===) never coerces — always prefer this
console.log(1 === '1');   // output: false
console.log(0 === false); // output: false

// The + operator: if either operand is a string, it concatenates
console.log(1 + '2');     // output: '12'
console.log('3' - 1);     // output: 2   (- always coerces to number)
console.log('6' / '2');   // output: 3
console.log(true + 1);    // output: 2   (true coerced to 1)
console.log(false + 1);   // output: 1   (false coerced to 0)
console.log(null + 1);    // output: 1   (null coerced to 0)
console.log(undefined + 1); // output: NaN (undefined coerces to NaN)
```

:::caution[Always Use ===]
Implicit coercion from `==` causes some of the most subtle bugs in JavaScript. Always use `===` (strict equality) and `!==` (strict inequality) in your code.
:::

## The `typeof` Operator

```js
console.log(typeof 42);          // output: number
console.log(typeof 'hello');      // output: string
console.log(typeof true);         // output: boolean
console.log(typeof undefined);    // output: undefined
console.log(typeof null);         // output: object  ← historical bug, null is NOT an object
console.log(typeof {});           // output: object
console.log(typeof []);           // output: object  ← arrays are objects
console.log(typeof function(){}); // output: function
console.log(typeof Symbol());     // output: symbol
console.log(typeof 42n);          // output: bigint

// Better array check:
Array.isArray([]);  // output: true
// Better null check:
null === null;      // output: true
```

## `null`, `undefined`, `NaN`, and `Infinity`

```js
// undefined — variable declared but not assigned, or missing property/argument
let x;
console.log(x);              // output: undefined
console.log(x === undefined); // output: true

function greet(name) {
  console.log(name);         // output: undefined (called without argument)
}
greet();

// null — intentional absence of a value (must be set explicitly)
let user = null;
console.log(user === null);  // output: true

// NaN — "Not a Number" result of invalid numeric operations
console.log(0 / 0);          // output: NaN
console.log(parseInt('abc')); // output: NaN
console.log(Math.sqrt(-1));   // output: NaN
console.log(NaN === NaN);     // output: false — NaN is not equal to itself!
console.log(Number.isNaN(NaN)); // output: true — correct way to check

// Infinity
console.log(1 / 0);           // output: Infinity
console.log(-1 / 0);          // output: -Infinity
console.log(isFinite(Infinity)); // output: false
console.log(Number.isFinite(42)); // output: true
```

## Template Literals

Template literals (backtick strings) support embedded expressions and multi-line text:

```js
const name = 'Alice';
const age = 30;

// Basic interpolation
const greeting = `Hello, ${name}! You are ${age} years old.`;
console.log(greeting); // output: Hello, Alice! You are 30 years old.

// Expressions inside ${}
console.log(`2 + 2 = ${2 + 2}`);            // output: 2 + 2 = 4
console.log(`Is adult: ${age >= 18}`);       // output: Is adult: true
console.log(`Name length: ${name.length}`);  // output: Name length: 5

// Multi-line strings
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>Age: ${age}</p>
  </div>
`;

// Nested template literals
const items = ['apple', 'banana', 'cherry'];
const list = `Items: ${items.map(i => `<li>${i}</li>`).join('')}`;
console.log(list);
// output: Items: <li>apple</li><li>banana</li><li>cherry</li>
```

## Destructuring — First Look

Destructuring extracts values from arrays or objects into variables:

```js
// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];
console.log(first);  // output: 1
console.log(second); // output: 2
console.log(rest);   // output: [3, 4, 5]

// Skip elements
const [, , third] = [10, 20, 30];
console.log(third);  // output: 30

// Object destructuring
const person = { name: 'Alice', age: 30, city: 'London' };
const { name, age, city } = person;
console.log(name, age, city); // output: Alice 30 London

// Rename while destructuring
const { name: personName, age: personAge } = person;
console.log(personName); // output: Alice

// Default values
const { name: n, role = 'user' } = person;
console.log(role); // output: user (person.role was undefined)

// Nested destructuring
const { address: { street, zip } = {} } = { address: { street: '123 Main St', zip: '10001' } };
console.log(street); // output: 123 Main St

// Function parameter destructuring
function display({ name, age, role = 'guest' }) {
  console.log(`${name} (${age}) — ${role}`);
}
display({ name: 'Bob', age: 25 }); // output: Bob (25) — guest
```

## Spread and Rest — First Look

```js
// Spread: expand an iterable into individual elements
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const combined = [...arr1, ...arr2];
console.log(combined); // output: [1, 2, 3, 4, 5, 6]

// Spread to clone an array (shallow copy)
const original = [1, 2, 3];
const clone = [...original];
clone.push(4);
console.log(original); // output: [1, 2, 3]  — not affected

// Spread with objects (shallow merge)
const defaults = { color: 'blue', size: 'medium', weight: 10 };
const custom = { color: 'red', size: 'large' };
const merged = { ...defaults, ...custom };
console.log(merged);
// output: { color: 'red', size: 'large', weight: 10 }
// Later spread properties override earlier ones

// Rest parameters: collect remaining arguments into an array
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3, 4, 5)); // output: 15
console.log(sum(10, 20));         // output: 30

// Rest in destructuring
const [head, ...tail] = [1, 2, 3, 4, 5];
console.log(head); // output: 1
console.log(tail); // output: [2, 3, 4, 5]
```

:::tip[Spread vs Rest]
**Spread** expands an iterable *into* a context (function call arguments, array/object literal). **Rest** collects multiple values *from* a context into an array (function parameter, destructuring). Same `...` syntax, opposite directions.
:::

## Optional Chaining and Nullish Coalescing

Two ES2020 features that dramatically reduce defensive code:

```js
const user = {
  name: 'Alice',
  address: {
    city: 'London',
  },
};

// Optional chaining — short-circuits to undefined if any part is null/undefined
const zip = user?.address?.zipCode;
console.log(zip); // output: undefined  (no error thrown)

const firstFriend = user?.friends?.[0]?.name;
console.log(firstFriend); // output: undefined

// Without optional chaining (old approach):
const zipOld = user && user.address && user.address.zipCode;

// Nullish coalescing (??) — returns right side only when left is null or undefined
const city = user?.address?.city ?? 'Unknown city';
console.log(city); // output: London

const missing = null ?? 'default';
console.log(missing); // output: default

const zero = 0 ?? 'default';
console.log(zero); // output: 0  (0 is not null/undefined, unlike || which treats 0 as falsy)

const empty = '' ?? 'default';
console.log(empty); // output: ''  (empty string is not null/undefined)
```

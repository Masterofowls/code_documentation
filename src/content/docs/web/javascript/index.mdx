JavaScript is the world's most widely used programming language. Originally built to add interactivity to web pages, it now powers servers, mobile apps, desktop applications, and IoT devices through runtimes like Node.js, Deno, and Bun.

## A Brief History

JavaScript was created by **Brendan Eich** in just 10 days in May 1995 at Netscape Communications. It was originally called Mocha, renamed to LiveScript, and finally renamed JavaScript — a marketing decision capitalizing on Java's popularity at the time. Despite the similar name, the two languages are architecturally unrelated.

The language is standardized as **ECMAScript** by ECMA International (Technical Committee 39, or TC39). Browser vendors implement the specification, and TC39 publishes a new edition every year via a stage-based proposal process (Stage 0 → Stage 4).

:::tip[The TC39 Proposal Process]
New language features progress through five stages before becoming part of the spec:
- **Stage 0** — Strawperson: an initial idea
- **Stage 1** — Proposal: problem statement and rough API defined
- **Stage 2** — Draft: full spec text being written
- **Stage 3** — Candidate: spec complete, browsers begin implementations
- **Stage 4** — Finished: shipped in two engines, ready to be merged into the spec

Track active proposals at [tc39.es/proposals](https://tc39.es/proposals/).
:::

## JavaScript Engines

A JavaScript engine parses, compiles, and executes JavaScript code. All modern engines use **Just-In-Time (JIT) compilation** — they interpret code immediately but simultaneously compile frequently-executed ("hot") paths to native machine code.

| Engine | Maintained By | Used In |
|--------|--------------|---------|
| **V8** | Google | Chrome, Edge, Node.js, Deno, Bun |
| **SpiderMonkey** | Mozilla | Firefox, Firefox for Android |
| **JavaScriptCore (Nitro/JSC)** | Apple | Safari, WebKit |
| **Hermes** | Meta | React Native (iOS, Android) |

### How V8 Compiles JavaScript

```
JavaScript Source Code
        ↓
    Parser → Abstract Syntax Tree (AST)
        ↓
    Ignition (Interpreter) → Bytecode        ← fast startup, low memory
        ↓ (hot function detected by profiler)
    TurboFan (Optimizing Compiler) → Native Machine Code   ← peak performance
        ↓ (type assumption violated at runtime)
    Deoptimization → back to Ignition bytecode
```

V8 tracks which types flow into each function. If a function always receives numbers, TurboFan emits tight numeric machine code. If the type changes later, V8 deoptimizes. Writing **consistently-typed code** helps V8 stay optimized.

## Runtime Environments

JavaScript runs in many environments beyond the browser:

<CardGrid>
  <Card title="Browser" icon="laptop">
    The original home of JavaScript. Provides the DOM, CSSOM, Web APIs (`fetch`, `localStorage`, `IndexedDB`, `WebSockets`, `Canvas`, `WebGL`), and the `window`/`document` globals. Chrome (V8), Firefox (SpiderMonkey), and Safari (JSC) differ in which experimental APIs they support.
</Card>
  <Card title="Node.js" icon="rocket">
    Server-side runtime built on V8. Provides file system (`fs`), networking (`http`, `net`, `dgram`), streams, child processes, worker threads, and npm — the world's largest package registry with over 2 million packages.
</Card>
  <Card title="Deno" icon="star">
    Secure-by-default runtime built on V8, created by Node.js's original author. Requires explicit `--allow-read`, `--allow-net`, etc. permission flags. Supports TypeScript natively, uses URL-based imports, and ships a standard library.
</Card>
  <Card title="Bun" icon="seti:bun">
    Fast all-in-one runtime built on JavaScriptCore. Ships a bundler, test runner, and package manager in one binary. Nearly fully compatible with Node.js APIs and significantly faster in I/O benchmarks.
</Card>
</CardGrid>

:::caution[Environment-Specific APIs]
`document`, `window`, and `navigator` are browser-only. `fs`, `process`, and `path` are Node.js-only. Code that must run in both environments (isomorphic/universal JS) needs polyfills or conditional checks:
```js
if (typeof window !== 'undefined') {
  // Browser-only code
}
if (typeof process !== 'undefined' && process.versions?.node) {
  // Node.js-only code
}
```
:::

## JavaScript vs TypeScript

TypeScript is a **typed superset of JavaScript** developed by Microsoft. Every valid `.js` file is valid TypeScript. TypeScript adds a type system that is erased at compile time — the output is plain JavaScript.

<Tabs>
  <TabItem label="JavaScript">
```js
// No type safety — bugs appear at runtime
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price * item.quantity, 0);
}

calculateTotal([{ price: 10, quantity: 2 }, { price: '5', quantity: 3 }]);
// output: '20undefined' — silent type bug!
```
</TabItem>
  <TabItem label="TypeScript">
```js
// Type errors caught at compile time, before running
interface CartItem {
  price: number;
  quantity: number;
}

function calculateTotal(items: CartItem[]): number {
  return items.reduce((sum, item) => sum + item.price * item.quantity, 0);
}

calculateTotal([{ price: 10, quantity: 2 }, { price: '5', quantity: 3 }]);
// Compile error: Type 'string' is not assignable to type 'number'
```
</TabItem>
</Tabs>

TypeScript benefits:
- **Static typing** — catch entire classes of bugs before running the code
- **Better IDE support** — autocomplete, inline docs, safe refactoring across files
- **Interfaces and generics** — describe complex, reusable data shapes
- **Stricter code quality** — `noImplicitAny`, `strictNullChecks`, `noUnusedLocals`

:::tip[Incremental Adoption]
You can enable TypeScript's type checker on plain `.js` files using JSDoc comments and a `jsconfig.json`. No migration required. See [JSDoc TypeScript support](https://www.typescriptlang.org/docs/handbook/jsdoc-supported-types.html).
:::

## How JavaScript Executes: The Event Loop

JavaScript is **single-threaded** — only one function runs at a time. The event loop allows asynchronous work (network requests, timers, user input) without blocking the thread.

```
┌──────────────────────────────┐
│         Call Stack           │  ← currently running functions (LIFO)
├──────────────────────────────┤
│   Web APIs / Node.js APIs    │  ← setTimeout, fetch, fs.readFile (offloaded here)
├──────────────────────────────┤
│       Microtask Queue        │  ← Promise .then/.catch, queueMicrotask (HIGH priority)
├──────────────────────────────┤
│        Task Queue            │  ← setTimeout/setInterval callbacks, I/O (LOWER priority)
└──────────────────────────────┘
             ↑
          Event Loop
  (when call stack is empty:
   1. drain the entire microtask queue
   2. take ONE task from the task queue
   3. repeat)
```

```js
console.log('1 — synchronous');

setTimeout(() => console.log('4 — macrotask (task queue)'), 0);

Promise.resolve()
  .then(() => console.log('2 — microtask'))
  .then(() => console.log('3 — another microtask'));

// output: 1 — synchronous
// output: 2 — microtask
// output: 3 — another microtask
// output: 4 — macrotask (task queue)
```

## A Taste of Modern JavaScript

```js
// Destructuring + default values + optional chaining + nullish coalescing
const { name = 'Anonymous', address: { city = 'Unknown' } = {} } = user ?? {};
const street = user?.address?.street ?? 'No address on file';

// Array pipeline — filter, sort, take top N, format
const topEngineers = employees
  .filter(e => e.department === 'Engineering' && e.active)
  .sort((a, b) => b.salary - a.salary)
  .slice(0, 5)
  .map(({ name, salary, level }) => ({
    name,
    level,
    salary: new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(salary),
  }));

// async/await with structured error handling
async function loadUserWithOrders(userId) {
  const [userResult, ordersResult] = await Promise.allSettled([
    fetch(`/api/users/${userId}`).then(r => r.ok ? r.json() : Promise.reject(new Error(`HTTP ${r.status}`))),
    fetch(`/api/users/${userId}/orders`).then(r => r.json()),
  ]);

  return {
    user: userResult.status === 'fulfilled' ? userResult.value : null,
    orders: ordersResult.status === 'fulfilled' ? ordersResult.value : [],
    errors: [userResult, ordersResult]
      .filter(r => r.status === 'rejected')
      .map(r => r.reason.message),
  };
}

// Classes with private fields and static factory methods
class Vector2D {
  #x; #y;

  constructor(x, y) { this.#x = x; this.#y = y; }

  static fromAngle(angle, length = 1) {
    return new Vector2D(Math.cos(angle) * length, Math.sin(angle) * length);
  }

  add(other) { return new Vector2D(this.#x + other.#x, this.#y + other.#y); }
  scale(factor) { return new Vector2D(this.#x * factor, this.#y * factor); }
  get magnitude() { return Math.hypot(this.#x, this.#y); }

  toString() { return `Vector2D(${this.#x.toFixed(2)}, ${this.#y.toFixed(2)})`; }
}

const v1 = new Vector2D(3, 4);
console.log(v1.magnitude);          // output: 5
console.log(v1.scale(2).toString()); // output: Vector2D(6.00, 8.00)

// Generator-based lazy sequence processing
function* map(iterable, fn) {
  for (const item of iterable) yield fn(item);
}
function* filter(iterable, pred) {
  for (const item of iterable) if (pred(item)) yield item;
}
function* take(iterable, n) {
  let count = 0;
  for (const item of iterable) {
    if (count++ >= n) break;
    yield item;
  }
}
function* naturals() {
  let n = 1;
  while (true) yield n++;
}

// Lazy: only computes values as needed — no intermediate arrays
const firstFiveEvenSquares = [
  ...take(
    filter(map(naturals(), n => n * n), n => n % 2 === 0),
    5
  )
];
console.log(firstFiveEvenSquares); // output: [4, 16, 36, 64, 100]
```

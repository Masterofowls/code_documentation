Tailwind's default theme is a carefully designed set of values, but real projects almost always require customization. This guide covers every approach — from simple color additions to complex plugin-level changes.

## The `tailwind.config.js` File

The config file is the heart of your Tailwind setup:

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  // Files to scan for class names
  content: ['./src/**/*.{html,js,ts,jsx,tsx,mdx}'],

  // Dark mode strategy
  darkMode: 'class',

  theme: {
    // REPLACE defaults entirely (use with caution)
    fontFamily: {
      sans: ['Inter', 'system-ui', 'sans-serif'],
    },

    // EXTEND defaults (preferred for most customizations)
    extend: {
      colors: {
        brand: {
          50: '#f5f3ff',
          500: '#8b5cf6',
          900: '#4c1d95',
        },
      },
    },
  },

  plugins: [],
};
```

:::caution[extend vs overriding]
Values placed directly under `theme` _replace_ the defaults entirely. Values under `theme.extend` are _merged_ with the defaults. Almost always use `theme.extend` unless you intentionally want to remove all defaults.
:::

## Extending Colors

### Adding Brand Colors

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        // Single value
        'off-white': '#f8f7f4',

        // Full palette with shades
        brand: {
          50: '#f0f9ff',
          100: '#e0f2fe',
          200: '#bae6fd',
          300: '#7dd3fc',
          400: '#38bdf8',
          500: '#0ea5e9',
          600: '#0284c7',
          700: '#0369a1',
          800: '#075985',
          900: '#0c4a6e',
          950: '#082f49',
        },

        // Semantic names (map to existing palette or custom values)
        success: '#22c55e', // green-500
        warning: '#f59e0b', // amber-500
        error: '#ef4444', // red-500
        info: '#3b82f6', // blue-500
      },
    },
  },
};
```

### Using CSS Variables as Color Values

This pattern lets you swap palettes at runtime (great for themes):

```css
/* globals.css */
@layer base {
  :root {
    --color-primary-500: 139 92 246; /* violet-500 in RGB */
    --color-primary-600: 124 58 237; /* violet-600 */
  }

  [data-theme='green'] {
    --color-primary-500: 34 197 94; /* green-500 */
    --color-primary-600: 22 163 74; /* green-600 */
  }
}
```

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: {
          // The <alpha-value> placeholder enables opacity modifiers: bg-primary-500/50
          500: 'rgb(var(--color-primary-500) / <alpha-value>)',
          600: 'rgb(var(--color-primary-600) / <alpha-value>)',
        },
      },
    },
  },
};
```

```html
<!-- Full opacity -->
<button class="bg-primary-500 hover:bg-primary-600 text-white">Button</button>

<!-- 50% opacity modifier works because of <alpha-value> -->
<div class="bg-primary-500/50">Semi-transparent</div>
```

## Extending Typography

### Custom Font Families with Google Fonts

```html
<!-- index.html or _document.tsx — load the fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Cal+Sans&family=JetBrains+Mono&display=swap"
  rel="stylesheet"
/>
```

```js
// tailwind.config.js
const defaultTheme = require('tailwindcss/defaultTheme');

module.exports = {
  theme: {
    extend: {
      fontFamily: {
        // Prepend to the default sans stack
        sans: ['Inter', ...defaultTheme.fontFamily.sans],
        // Separate stacks for headings and code
        display: ['Cal Sans', 'Inter', ...defaultTheme.fontFamily.sans],
        mono: ['JetBrains Mono', ...defaultTheme.fontFamily.mono],
      },
    },
  },
};
```

```html
<h1 class="font-display text-4xl font-bold">Display Heading</h1>
<p class="font-sans text-base">Body text</p>
<code class="font-mono text-sm">const x = 42;</code>
```

## Custom Animations

```js
module.exports = {
  theme: {
    extend: {
      keyframes: {
        // Slide in from above
        'slide-in-from-top': {
          '0%': { transform: 'translateY(-10px)', opacity: '0' },
          '100%': { transform: 'translateY(0)', opacity: '1' },
        },
        // Fade and scale
        'fade-in-scale': {
          '0%': { transform: 'scale(0.95)', opacity: '0' },
          '100%': { transform: 'scale(1)', opacity: '1' },
        },
        // Shimmer for loading skeletons
        shimmer: {
          '0%': { backgroundPosition: '-200% 0' },
          '100%': { backgroundPosition: '200% 0' },
        },
        // Bounce dot for typing indicators
        'bounce-dot': {
          '0%, 100%': { transform: 'scale(0.8)', opacity: '0.5' },
          '50%': { transform: 'scale(1)', opacity: '1' },
        },
      },
      animation: {
        'slide-in-from-top': 'slide-in-from-top 0.2s ease-out',
        'fade-in-scale': 'fade-in-scale 0.15s ease-out',
        shimmer: 'shimmer 1.5s infinite linear',
        'bounce-dot': 'bounce-dot 1s infinite',
        'bounce-dot-delay': 'bounce-dot 1s 0.15s infinite',
        'bounce-dot-delay-2': 'bounce-dot 1s 0.3s infinite',
      },
    },
  },
};
```

```html

  <span class="w-2 h-2 rounded-full bg-gray-400 animate-bounce-dot"></span>

  <span
    class="w-2 h-2 rounded-full bg-gray-400 animate-bounce-dot-delay-2"
  ></span>
</div>
```

## The `@layer` Directive

`@layer` lets you add custom CSS within Tailwind's ordering system, ensuring correct specificity:

```css
/* globals.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Base layer — global resets, element defaults */
@layer base {
  h1 {
    @apply text-4xl font-bold tracking-tight;
  }
  h2 {
    @apply text-3xl font-semibold tracking-tight;
  }
  h3 {
    @apply text-2xl font-semibold;
  }

  a {
    @apply text-violet-600 hover:text-violet-800 underline-offset-2;
  }

  ::selection {
    @apply bg-violet-200 text-violet-900;
  }

  :focus-visible {
    @apply outline-2 outline-offset-2 outline-violet-500;
  }
}

/* Components layer — reusable UI patterns */
@layer components {
  .btn {
    @apply inline-flex items-center justify-center gap-2
           font-medium rounded-lg px-4 py-2 text-sm
           transition-colors duration-150
           focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2;
  }

  .btn-primary {
    @apply btn bg-violet-600 text-white hover:bg-violet-700
           focus-visible:ring-violet-500;
  }

  .btn-outline {
    @apply btn border border-gray-300 text-gray-700 hover:bg-gray-50
           focus-visible:ring-gray-400;
  }

  .btn-ghost {
    @apply btn text-gray-600 hover:bg-gray-100
           focus-visible:ring-gray-400;
  }

  .card {
    @apply bg-white rounded-2xl border border-gray-100
           shadow-sm hover:shadow-md transition-shadow;
  }

  .input {
    @apply w-full rounded-lg border border-gray-300 bg-white px-3 py-2 text-sm
           text-gray-900 placeholder:text-gray-400
           focus:outline-none focus:ring-2 focus:ring-violet-500 focus:border-transparent
           dark:bg-gray-800 dark:border-gray-600 dark:text-gray-100;
  }

  .badge {
    @apply inline-flex items-center rounded-full px-2.5 py-0.5
           text-xs font-medium;
  }
}

/* Utilities layer — custom utility classes */
@layer utilities {
  .text-balance {
    text-wrap: balance;
  }

  .scrollbar-hide {
    -ms-overflow-style: none;
    scrollbar-width: none;
  }
  .scrollbar-hide::-webkit-scrollbar {
    display: none;
  }

  .gradient-text {
    @apply bg-clip-text text-transparent;
  }
}
```

```html
<!-- Using custom component classes -->
<button class="btn-primary">Save changes</button>
<button class="btn-outline">Cancel</button>
<div class="card p-6">
  <input class="input" placeholder="Email address" />
</div>
<span class="badge bg-green-100 text-green-700">Active</span>
```

## Arbitrary Values

JIT allows you to use any value without adding it to the config:

```html
<!-- Width and height -->
<div class="w-[317px] h-[42px]">Precise sizing</div>

<!-- Colors -->
<div class="bg-[#bada55] text-[rgb(30,20,60)]">Custom colors</div>
<div class="bg-[hsl(200,100%,50%)]">HSL color</div>

<!-- Grid -->
<div class="grid grid-cols-[1fr_2fr_1fr]">Asymmetric grid</div>
<div class="grid grid-rows-[auto_1fr_auto]">Header, main, footer</div>

<!-- Arbitrary properties (escape hatch) -->
<div class="[background-image:url('/hero.jpg')]">Direct CSS property</div>
<div class="[--custom-var:42px]">Set CSS custom property</div>
<div class="[mask-image:linear-gradient(to_bottom,black,transparent)]">
  Mask
</div>

<!-- Breakpoints with arbitrary values -->
<div class="min-[900px]:flex-row">Custom breakpoint</div>
```

## The `theme()` Function in CSS

Use `theme()` to reference Tailwind config values in your custom CSS:

```css
@layer components {
  .hero-gradient {
    background: linear-gradient(
      135deg,
      theme('colors.violet.600'),
      theme('colors.indigo.600')
    );
  }

  .prose-custom {
    max-width: theme('spacing.prose');
    font-family: theme('fontFamily.serif');
    color: theme('colors.gray.800');
    line-height: theme('lineHeight.relaxed');
  }

  @media (min-width: theme('screens.lg')) {
    .hero-gradient {
      padding: theme('spacing.24') theme('spacing.16');
    }
  }
}
```

## Custom Screens (Breakpoints)

```js
module.exports = {
  theme: {
    extend: {
      screens: {
        xs: '475px',
        '3xl': '1920px',
        '4xl': '2560px',
        // Named breakpoints
        print: { raw: 'print' },
        // Height-based breakpoints
        short: { raw: '(max-height: 600px)' },
        tall: { raw: '(min-height: 900px)' },
      },
    },
  },
};
```

```html
<div class="hidden xs:block">Show on 475px+</div>
<div class="print:hidden">Hidden when printing</div>
<div class="short:py-2 tall:py-8">Responsive to height</div>
```

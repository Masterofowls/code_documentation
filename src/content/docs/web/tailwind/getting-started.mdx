This guide walks you through a complete Tailwind CSS setup — from installation to your first production-ready component.

## Prerequisites

- Node.js 18 or later
- A project with a package.json (npm, yarn, or pnpm)
- A bundler or build tool (Vite, webpack, Next.js, Astro, etc.)

## Installation

<Steps>
1. **Install Tailwind and its peer dependencies**

   ```bash
   npm install -D tailwindcss postcss autoprefixer
   ```

2. **Generate the configuration files**

   ```bash
   npx tailwindcss init -p
   ```

   This creates two files:
   - `tailwind.config.js` — your Tailwind configuration
   - `postcss.config.js` — your PostCSS pipeline

3. **Configure content paths**

   Open `tailwind.config.js` and add all file paths that contain Tailwind class names:

   ```js
   /** @type {import('tailwindcss').Config} */
   module.exports = {
     content: [
       './src/**/*.{html,js,ts,jsx,tsx,vue,svelte,astro}',
       './index.html',
     ],
     theme: {
       extend: {},
     },
     plugins: [],
   };
   ```

4. **Add the Tailwind directives to your CSS**

   Create (or open) your main CSS entry file — typically `src/index.css` or `src/globals.css`:

   ```css
   /* src/index.css */
   @tailwind base;
   @tailwind components;
   @tailwind utilities;
   ```

5. **Import the CSS file in your entry point**

   ```js
   // src/main.js or src/main.tsx
   import './index.css';
   ```

6. **Start your dev server**

   ```bash
   npm run dev
   ```

   Tailwind classes should now work in your project.

</Steps>

:::caution[Content paths matter]
If your content paths are wrong, Tailwind will generate an empty stylesheet in production. Always include every file type that uses Tailwind classes.
:::

## The Three Directives

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

| Directive              | What it injects                                                |
| ---------------------- | -------------------------------------------------------------- |
| `@tailwind base`       | Preflight — an opinionated CSS reset based on modern-normalize |
| `@tailwind components` | Any classes registered in `@layer components` or via plugins   |
| `@tailwind utilities`  | All utility classes, including JIT-generated arbitrary values  |

:::tip[Directive order matters]
Always keep the directives in this order. Utilities must come last so they can override component and base styles without needing `!important`.
:::

## PostCSS Configuration

```js
// postcss.config.js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

For production, add `cssnano` to minify the output:

```js
// postcss.config.js (production-aware)
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
    ...(process.env.NODE_ENV === 'production'
      ? {
          cssnano: {
            preset: ['default', { discardComments: { removeAll: true } }],
          },
        }
      : {}),
  },
};
```

## Editor Tooling

### VS Code IntelliSense

Install the official **Tailwind CSS IntelliSense** extension:

1. Open VS Code
2. Press `Ctrl+P` (Windows/Linux) or `Cmd+P` (macOS)
3. Run: `ext install bradlc.vscode-tailwindcss`

Features provided:

- Autocomplete for class names in HTML, JSX, TSX, CSS
- Hover previews showing the generated CSS
- Lint warnings for conflicting utilities
- Color swatches inline in the editor

Add this to `.vscode/settings.json` for the best experience:

```json
{
  "editor.quickSuggestions": {
    "strings": "on"
  },
  "tailwindCSS.includeLanguages": {
    "typescript": "javascript",
    "typescriptreact": "javascript"
  },
  "tailwindCSS.experimental.classRegex": [
    ["clsx\\(([^)]*)\\)", "(?:'|\"|`)([^']*)(?:'|\"|`)"],
    ["cn\\(([^)]*)\\)", "(?:'|\"|`)([^']*)(?:'|\"|`)"]
  ]
}
```

### Prettier Plugin for Class Sorting

The official Prettier plugin automatically sorts Tailwind class names in a consistent order:

```bash
npm install -D prettier prettier-plugin-tailwindcss
```

```js
// prettier.config.js
/** @type {import('prettier').Config} */
module.exports = {
  plugins: ['prettier-plugin-tailwindcss'],
  // Your other Prettier options
  singleQuote: true,
  trailingComma: 'all',
  printWidth: 100,
};
```

Before sorting:

```html
<div
  class="pt-2 text-sm flex container mx-auto bg-white rounded-lg p-4 shadow"
></div>
```

After sorting:

```html
<div
  class="container mx-auto flex rounded-lg bg-white p-4 pt-2 text-sm shadow"
></div>
```

:::tip[Why sort classes?]
Consistent class order makes code reviews easier, reduces merge conflicts, and makes it faster to scan which utilities are applied.
:::

```tsx
interface ProfileCardProps {

export function ProfileCard({

            className="w-24 h-24 rounded-full border-4 border-white object-cover shadow-md"
          />
        </div>

        <h2 className="text-xl font-bold text-gray-900">{name}</h2>
        <p className="text-sm text-gray-500 mb-3">{handle}</p>
        <p className="text-gray-600 text-sm leading-relaxed mb-4">{bio}</p>

        <div className="flex gap-6 mb-4 text-sm">
          <Stat label="Following" value={following} />
          <Stat label="Followers" value={followers} />
        </div>

        <button className="w-full bg-violet-600 hover:bg-violet-700 text-white font-semibold py-2 px-4 rounded-xl transition-colors duration-150">
          Follow
        </button>
      </div>
    </div>
  );
}

function Stat({ label, value }: { label: string; value: number }) {
  return (
    <div>
      <span className="font-semibold text-gray-900">
        {value.toLocaleString()}
      </span>
      <span className="text-gray-500 ml-1">{label}</span>
    </div>
  );
}
```

## Purging Unused Styles in Production

Tailwind's JIT engine only generates CSS for class names it finds in your content files. In production:

```bash
NODE_ENV=production npm run build
```

Typical output sizes:

- **Development**: All classes generated (can be large)
- **Production**: Only used classes (typically 5–15 KB gzipped)

:::caution[Dynamic class names are not detected]
Tailwind scans for complete class name strings. Dynamic construction breaks detection:

```js
// ❌ Will NOT work — class is never in the source as a complete string
const color = 'red';
<div className={`text-${color}-500`}>

// ✅ Works — complete class names are present in source
const classes = isError ? 'text-red-500' : 'text-green-500';
<div className={classes}>
```

:::

## Class Name Conventions

Use a utility like `clsx` or `cn` (clsx + tailwind-merge) for conditional class names:

```bash
npm install clsx tailwind-merge
```

```ts
// lib/utils.ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

```tsx
// Usage
import { cn } from '@/lib/utils';

function Button({ variant = 'primary', disabled, className, ...props }) {
  return (
    <button
      className={cn(
        // Base styles
        'inline-flex items-center justify-center rounded-lg font-medium transition-colors',
        // Variants
        variant === 'primary' && 'bg-violet-600 text-white hover:bg-violet-700',
        variant === 'outline' &&
          'border border-gray-300 text-gray-700 hover:bg-gray-50',
        variant === 'ghost' && 'text-gray-600 hover:bg-gray-100',
        // States
        disabled && 'opacity-50 cursor-not-allowed pointer-events-none',
        // Overrides from consumer
        className,
      )}
      disabled={disabled}
      {...props}
    />
  );
}
```

`tailwind-merge` intelligently resolves conflicts between Tailwind classes, so passing `className="bg-red-500"` to a component that already has `bg-violet-600` will correctly result in only `bg-red-500` being applied.

Tailwind's plugin API lets you extend the framework with reusable utilities, components, variants, and base styles. The official plugins and a rich ecosystem of community packages cover most common needs.

## Official Plugins

### @tailwindcss/typography

The typography plugin provides a `prose` class that applies beautiful typographic defaults to HTML content — ideal for markdown-rendered articles, blog posts, and documentation.

```bash
npm install -D @tailwindcss/typography
```

```js
// tailwind.config.js
module.exports = {
  plugins: [
    require('@tailwindcss/typography'),
  ],
};
```

```html
<!-- Apply prose to any HTML content block -->
<article class="prose prose-lg prose-slate max-w-none">
  <h1>Article Title</h1>
  <p>Paragraph text with <strong>bold</strong> and <a href="#">links</a>.</p>
  <pre><code>const hello = 'world';</code></pre>
  <ul>
    <li>List item one</li>
    <li>List item two</li>
  </ul>
</article>
```

#### Prose Size Modifiers

```html
<div class="prose prose-sm">Small text (14px base)</div>
<div class="prose">Default (16px base)</div>
<div class="prose prose-lg">Large (18px base)</div>
<div class="prose prose-xl">XL (20px base)</div>
<div class="prose prose-2xl">2XL (24px base)</div>
```

#### Prose Color Themes

```html
<article class="prose prose-slate">Blue-gray tones</article>
<article class="prose prose-gray">Neutral gray</article>
<article class="prose prose-zinc">Warm zinc</article>
<article class="prose prose-stone">Stone tones</article>
```

#### Dark Mode Prose

```html
<article class="prose dark:prose-invert">
  Adapts to dark mode — inverts the default colors
</article>
```

#### Customizing Prose

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      typography: (theme) => ({
        DEFAULT: {
          css: {
            color: theme('colors.gray.800'),
            a: {
              color: theme('colors.violet.600'),
              textDecoration: 'none',
              '&:hover': {
                textDecoration: 'underline',
              },
            },
            'code::before': { content: '""' },   // Remove backtick wrapping
            'code::after': { content: '""' },
            code: {
              backgroundColor: theme('colors.gray.100'),
              borderRadius: theme('borderRadius.md'),
              paddingInline: theme('spacing.1'),
            },
            pre: {
              backgroundColor: theme('colors.gray.900'),
            },
          },
        },
      }),
    },
  },
  plugins: [require('@tailwindcss/typography')],
};
```

### @tailwindcss/forms

Resets form element styles to a consistent, easily customizable baseline across browsers.

```bash
npm install -D @tailwindcss/forms
```

```js
// tailwind.config.js
module.exports = {
  plugins: [
    require('@tailwindcss/forms'),
    // Or with strategy option:
    require('@tailwindcss/forms')({ strategy: 'class' }),
  ],
};
```

**Strategies:**
- `'base'` (default) — global CSS reset applied to all form elements
- `'class'` — opt-in via `form-input`, `form-select`, `form-textarea`, `form-checkbox` classes

```html
<!-- With strategy: 'class' -->
<input type="text" class="form-input rounded-lg border-gray-300 text-sm" />
<select class="form-select rounded-lg border-gray-300 text-sm"></select>
<textarea class="form-textarea rounded-lg border-gray-300 text-sm"></textarea>
<input type="checkbox" class="form-checkbox rounded text-violet-600" />
<input type="radio" class="form-radio text-violet-600" />
```

### @tailwindcss/aspect-ratio

Provides the `aspect-*` utility classes (now also built into Tailwind v3.2+ natively):

```bash
npm install -D @tailwindcss/aspect-ratio
```

```html
<div class="aspect-w-16 aspect-h-9">
  <iframe src="https://www.youtube.com/embed/..." allowfullscreen></iframe>
</div>

<div class="aspect-w-4 aspect-h-3">
  <img src="..." class="object-cover" />
</div>
```

:::tip[Native aspect-ratio]
In Tailwind v3.2+, use `aspect-video` (16/9), `aspect-square` (1/1), or `aspect-[4/3]` (arbitrary) directly without the plugin.
:::

### @tailwindcss/container-queries

Enables styling based on container size rather than viewport size:

```bash
npm install -D @tailwindcss/container-queries
```

```js
module.exports = {
  plugins: [require('@tailwindcss/container-queries')],
};
```

```html
<!-- Define a container context -->
<div class="@container">
  <!-- Responds to the container's width, not the viewport -->
  <div class="flex flex-col @lg:flex-row gap-4">
    <img class="w-full @lg:w-48 @lg:h-auto aspect-square object-cover rounded-lg" />
    <div>
      <h2 class="text-sm @lg:text-lg font-semibold">Card Title</h2>
      <p class="text-xs @lg:text-sm text-gray-500">Description</p>
    </div>
  </div>
</div>
```

Container query breakpoints:
| Prefix | Min width |
|--------|-----------|
| `@xs:` | 20rem (320px) |
| `@sm:` | 24rem (384px) |
| `@md:` | 28rem (448px) |
| `@lg:` | 32rem (512px) |
| `@xl:` | 36rem (576px) |
| `@2xl:` | 42rem (672px) |

## Writing a Custom Plugin

```js
// tailwind.config.js
const plugin = require('tailwindcss/plugin');

module.exports = {
  plugins: [
    plugin(function ({ addUtilities, addComponents, addBase, addVariant, matchUtilities, theme }) {
      // See below for each API...
    }),
  ],
};
```

### `addUtilities` — Static Utilities

```js
plugin(function ({ addUtilities }) {
  addUtilities({
    // Gradient text
    '.gradient-text': {
      backgroundClip: 'text',
      '-webkit-background-clip': 'text',
      color: 'transparent',
    },
    // Scrollbar hide
    '.scrollbar-hide': {
      '-ms-overflow-style': 'none',
      'scrollbar-width': 'none',
      '&::-webkit-scrollbar': { display: 'none' },
    },
    // Full-page overlay
    '.overlay': {
      position: 'fixed',
      inset: '0',
      zIndex: '50',
      backgroundColor: 'rgb(0 0 0 / 0.5)',
    },
    // Visually hidden (accessibility)
    '.sr-only-focusable:not(:focus)': {
      position: 'absolute',
      width: '1px',
      height: '1px',
      padding: '0',
      margin: '-1px',
      overflow: 'hidden',
      clip: 'rect(0, 0, 0, 0)',
      whiteSpace: 'nowrap',
      borderWidth: '0',
    },
  });
}),
```

### `matchUtilities` — Dynamic Utilities with Values

`matchUtilities` creates utilities that accept values from the theme or arbitrary values:

```js
plugin(function ({ matchUtilities, theme }) {
  // Creates size-2, size-4, size-8, size-[42px], etc.
  matchUtilities(
    {
      size: (value) => ({
        width: value,
        height: value,
      }),
    },
    { values: theme('spacing') }
  );

  // Mask gradient utility
  matchUtilities(
    {
      'mask-fade': (value) => ({
        maskImage: `linear-gradient(to bottom, black ${value}, transparent)`,
        '-webkit-mask-image': `linear-gradient(to bottom, black ${value}, transparent)`,
      }),
    },
    { values: { sm: '60%', md: '75%', lg: '85%' } }
  );
}),
```

```html
<div class="size-12">48x48 square</div>
<div class="size-[100px]">100x100 arbitrary</div>
<div class="mask-fade-md">Fades at 75%</div>
```

### `addComponents` — Component Classes

```js
plugin(function ({ addComponents, theme }) {
  addComponents({
    '.btn': {
      display: 'inline-flex',
      alignItems: 'center',
      justifyContent: 'center',
      gap: theme('spacing.2'),
      borderRadius: theme('borderRadius.lg'),
      padding: `${theme('spacing.2')} ${theme('spacing.4')}`,
      fontSize: theme('fontSize.sm')[0],
      fontWeight: theme('fontWeight.medium'),
      lineHeight: theme('lineHeight.tight'),
      transitionProperty: 'color, background-color, border-color',
      transitionDuration: '150ms',
      '&:focus-visible': {
        outline: `2px solid ${theme('colors.violet.500')}`,
        outlineOffset: '2px',
      },
    },
    '.btn-primary': {
      backgroundColor: theme('colors.violet.600'),
      color: theme('colors.white'),
      '&:hover': { backgroundColor: theme('colors.violet.700') },
    },
    '.btn-secondary': {
      backgroundColor: theme('colors.gray.100'),
      color: theme('colors.gray.900'),
      '&:hover': { backgroundColor: theme('colors.gray.200') },
    },
  });
}),
```

### `addBase` — Base Element Styles

```js
plugin(function ({ addBase, theme }) {
  addBase({
    'h1': { fontSize: theme('fontSize.4xl'), fontWeight: theme('fontWeight.bold') },
    'h2': { fontSize: theme('fontSize.3xl'), fontWeight: theme('fontWeight.semibold') },
    'h3': { fontSize: theme('fontSize.2xl'), fontWeight: theme('fontWeight.semibold') },
    'a': {
      color: theme('colors.violet.600'),
      textDecoration: 'underline',
      textUnderlineOffset: '2px',
    },
    '*': { boxSizing: 'border-box' },
  });
}),
```

### `addVariant` — Custom Variants

```js
plugin(function ({ addVariant }) {
  // Style when the parent has the 'loading' attribute
  addVariant('loading', '[loading] &');
  addVariant('loading-self', '&[loading]');

  // Style based on data attributes
  addVariant('active', '&[data-state="active"]');
  addVariant('inactive', '&[data-state="inactive"]');
  addVariant('open', '&[data-state="open"]');
  addVariant('closed', '&[data-state="closed"]');

  // Print styles
  addVariant('print', '@media print');

  // High contrast mode
  addVariant('high-contrast', '@media (forced-colors: active)');
}),
```

```html
<div data-state="active" class="active:bg-blue-100 inactive:bg-gray-50">
  State-driven styles
</div>

<button loading class="loading-self:opacity-50 loading-self:cursor-wait">
  Submit
</button>
```

### Plugin with Options

```js
// plugins/skeleton.js
const plugin = require('tailwindcss/plugin');

module.exports = plugin.withOptions(
  function (options = {}) {
    const { color = 'gray', duration = '1.5s' } = options;

    return function ({ addComponents, theme }) {
      addComponents({
        '.skeleton': {
          backgroundColor: theme(`colors.${color}.200`),
          backgroundImage: `linear-gradient(90deg,
            ${theme(`colors.${color}.200`)} 25%,
            ${theme(`colors.${color}.100`)} 50%,
            ${theme(`colors.${color}.200`)} 75%)`,
          backgroundSize: '400% 100%',
          animation: `shimmer ${duration} ease-in-out infinite`,
          borderRadius: theme('borderRadius.md'),
        },
      });
    };
  },
  function (options = {}) {
    return {
      theme: {
        extend: {
          keyframes: {
            shimmer: {
              '0%': { backgroundPosition: '100% 50%' },
              '100%': { backgroundPosition: '0% 50%' },
            },
          },
        },
      },
    };
  }
);
```

```js
// tailwind.config.js
module.exports = {
  plugins: [
    require('./plugins/skeleton')({ color: 'slate', duration: '2s' }),
  ],
};
```

## Third-Party Plugins

### daisyUI

A comprehensive component library built on Tailwind that provides semantic class names:

```bash
npm install -D daisyui
```

```js
module.exports = {
  plugins: [require('daisyui')],
  daisyui: {
    themes: ['light', 'dark', 'cupcake', 'cyberpunk'],
    darkTheme: 'dark',
  },
};
```

```html
<button class="btn btn-primary">DaisyUI Button</button>
<div class="card w-96 bg-base-100 shadow-xl">
  <div class="card-body">
    <h2 class="card-title">Card Title</h2>
    <p>Card content</p>
  </div>
</div>
```

### shadcn/ui

shadcn/ui is not a plugin per se, but a collection of copy-paste components built with Tailwind and Radix UI primitives. It's installed component-by-component:

```bash
npx shadcn-ui@latest init
npx shadcn-ui@latest add button
npx shadcn-ui@latest add dialog
npx shadcn-ui@latest add input
```

shadcn/ui uses CSS variables for theming, making it easy to customize your entire app's look by editing a single CSS file.

### Headless UI Integration

Headless UI (from the Tailwind Labs team) provides unstyled, accessible UI primitives that you style entirely with Tailwind:

```bash
npm install @headlessui/react
```

```tsx
import { Dialog, Transition } from '@headlessui/react';
import { Fragment } from 'react';

function Modal({ isOpen, onClose, children }) {
  return (
    <Transition appear show={isOpen} as={Fragment}>
      <Dialog as="div" className="relative z-50" onClose={onClose}>
        {/* Backdrop */}
        <Transition.Child
          as={Fragment}
          enter="ease-out duration-200"
          enterFrom="opacity-0"
          enterTo="opacity-100"
          leave="ease-in duration-150"
          leaveFrom="opacity-100"
          leaveTo="opacity-0"
        >
          <div className="fixed inset-0 bg-black/50 backdrop-blur-sm" />
        </Transition.Child>

        {/* Modal panel */}
        <div className="fixed inset-0 overflow-y-auto">
          <div className="flex min-h-full items-center justify-center p-4">
            <Transition.Child
              as={Fragment}
              enter="ease-out duration-200"
              enterFrom="opacity-0 scale-95"
              enterTo="opacity-100 scale-100"
              leave="ease-in duration-150"
              leaveFrom="opacity-100 scale-100"
              leaveTo="opacity-0 scale-95"
            >
              <Dialog.Panel className="w-full max-w-md bg-white dark:bg-gray-900
                                       rounded-2xl p-6 shadow-xl">
                {children}
              </Dialog.Panel>
            </Transition.Child>
          </div>
        </div>
      </Dialog>
    </Transition>
  );
}
```

:::tip[Headless UI + Tailwind is a powerful combination]
Headless UI handles all the accessibility and keyboard behavior. Tailwind handles all the visual styling. This separation of concerns results in highly accessible, fully custom UIs.
:::

### Flowbite

Flowbite is a component library with interactive elements built on Tailwind:

```bash
npm install flowbite flowbite-react
npm install -D flowbite
```

```js
// tailwind.config.js
module.exports = {
  content: [
    // Add Flowbite source files so JIT sees its classes
    './node_modules/flowbite/**/*.js',
  ],
  plugins: [
    require('flowbite/plugin'),
  ],
};
```

```tsx
import { Button, Modal } from 'flowbite-react';

function App() {
  return (
    <Button color="purple" size="lg">
      Flowbite Button
    </Button>
  );
}
```

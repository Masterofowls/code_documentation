Tailwind's responsive design system is built around a **mobile-first** breakpoint strategy. Every utility can be prefixed with a breakpoint modifier to apply it only at that screen size and above.

## The Breakpoint System

| Prefix   | Min width | Typical target               |
| -------- | --------- | ---------------------------- |
| _(none)_ | 0px       | Mobile (default)             |
| `sm:`    | 640px     | Large phones / small tablets |
| `md:`    | 768px     | Tablets                      |
| `lg:`    | 1024px    | Laptops                      |
| `xl:`    | 1280px    | Desktops                     |
| `2xl:`   | 1536px    | Large monitors               |

```html
<!-- This div is full-width on mobile, half-width on md+, one-third on lg+ -->
<div class="w-full md:w-1/2 lg:w-1/3">Responsive width</div>
```

## Mobile-First Approach

Tailwind uses **min-width breakpoints**, which means:

- Unprefixed classes apply at _all_ screen sizes
- Prefixed classes apply at that size _and above_
- Styles at larger breakpoints override smaller ones

```html
<!-- ✅ Mobile-first: build up from mobile -->
<div class="text-sm md:text-base lg:text-lg">
  Grows in size as viewport widens
</div>

<!-- ❌ Not mobile-first: overriding large styles at small sizes -->
<div class="text-lg sm:text-base">This approach fights the system</div>
```

:::tip[Think small first]
Design your layout for the smallest screen first, then progressively enhance it for larger screens. This forces you to prioritize content and produces cleaner, more maintainable responsive code.
:::

## Responsive Typography

```html
<!-- Fluid heading that scales up -->
<h1
  class="text-2xl font-bold
           sm:text-3xl
           md:text-4xl
           lg:text-5xl
           xl:text-6xl"
>
  Hero Heading
</h1>

<!-- Body text that gets more comfortable to read on larger screens -->
<p
  class="text-sm leading-relaxed
          md:text-base md:leading-loose
          lg:max-w-prose"
>
  Article body copy
</p>

<!-- Navigation — hide text on mobile, show on desktop -->
<nav class="flex items-center gap-6">
  <a
    class="flex items-center gap-2 font-medium text-gray-700 hover:text-gray-900"
  >
    <svg class="w-5 h-5" ... />
    <span class="hidden md:inline">Dashboard</span>
  </a>
</nav>
```

## Responsive Grid

The most common responsive pattern: a single column on mobile, expanding to a multi-column grid on larger screens.

```html
<!-- 1 → 2 → 3 column grid -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6">
  <div class="bg-white rounded-xl p-6 shadow-sm">Card 1</div>
  <div class="bg-white rounded-xl p-6 shadow-sm">Card 2</div>
  <div class="bg-white rounded-xl p-6 shadow-sm">Card 3</div>
  <div class="bg-white rounded-xl p-6 shadow-sm">Card 4</div>
  <div class="bg-white rounded-xl p-6 shadow-sm">Card 5</div>
  <div class="bg-white rounded-xl p-6 shadow-sm">Card 6</div>
</div>

<!-- Magazine-style layout with spanning -->
<div class="grid grid-cols-1 md:grid-cols-3 lg:grid-cols-4 gap-4">
  <!-- Featured article spans full width on mobile, 2 cols on md, 3 on lg -->
  <article class="md:col-span-2 lg:col-span-3">
    <img
      class="w-full aspect-video object-cover rounded-xl"
      src="..."
      alt="..."
    />
    <h2 class="mt-4 text-xl font-bold">Featured Article</h2>
  </article>

  <!-- Sidebar: stacks below on mobile, side panel on md+ -->
  <aside class="space-y-4">
    <div class="bg-gray-50 rounded-xl p-4">Recent Posts</div>
  </aside>
</div>
```

## Responsive Flexbox

```html
<!-- Stack on mobile, row on sm+ -->
<div class="flex flex-col sm:flex-row gap-4">
  <div class="flex-1">Main content</div>
  <div class="sm:w-64">Sidebar</div>
</div>

<!-- Center on mobile, spread on larger screens -->

```

## The Container Class

The `container` class sets `max-width` relative to the current breakpoint:

| Breakpoint  | Max width |
| ----------- | --------- |
| _(default)_ | 100%      |
| `sm`        | 640px     |
| `md`        | 768px     |
| `lg`        | 1024px    |
| `xl`        | 1280px    |
| `2xl`       | 1536px    |

```html
<!-- Basic container — doesn't auto-center -->
<div class="container">...</div>

<!-- Container with auto centering and padding -->
<div class="container mx-auto px-4 sm:px-6 lg:px-8">...</div>
```

### Customizing Container

```js
// tailwind.config.js
module.exports = {
  theme: {
    container: {
      center: true, // auto mx-auto
      padding: {
        DEFAULT: '1rem',
        sm: '1.5rem',
        lg: '2rem',
      },
    },
  },
};
```

```html
<!-- With customized container, center and padding are built in -->
<div class="container">...</div>
```

## Custom Breakpoints

```js
// tailwind.config.js
module.exports = {
  theme: {
    screens: {
      // Override defaults entirely
      mobile: '480px',
      tablet: '768px',
      laptop: '1024px',
      desktop: '1280px',
    },
  },
};

// Or extend existing breakpoints
module.exports = {
  theme: {
    extend: {
      screens: {
        xs: '475px', // Add extra-small
        '3xl': '1920px', // Add extra-large
      },
    },
  },
};
```

```html
<!-- Using custom breakpoints -->
<div class="hidden xs:block">Visible from 475px</div>
<div class="text-base laptop:text-lg desktop:text-xl">
  Custom breakpoint text
</div>
```

## Show/Hide Per Breakpoint

```html
<!-- Mobile-only: visible on mobile, hidden on md+ -->
<div class="block md:hidden">Mobile menu button</div>

<!-- Desktop-only: hidden on mobile, visible on md+ -->
<div class="hidden md:block">Full desktop navigation</div>

<!-- Visible only in a range: hide below lg AND hide at 2xl+ -->
<div class="hidden lg:block 2xl:hidden">Tablet/laptop sidebar</div>
```

## Responsive Images

```html
<!-- Fluid responsive image -->
<img class="w-full h-auto rounded-xl" src="/hero.jpg" alt="Hero image" />

<!-- Image with responsive aspect ratio -->
<div
  class="aspect-video md:aspect-[4/3] lg:aspect-video overflow-hidden rounded-xl"
>
  <img class="w-full h-full object-cover" src="..." alt="..." />
</div>

<!-- Object-position responsive change -->
<img
  class="w-full h-48 object-cover object-center
         md:h-64 md:object-top
         lg:h-80 lg:object-center"
  src="..."
  alt="..."
/>

<!-- Art direction with picture + source sets -->
<!-- Note: Tailwind classes go on the img element, not source -->
<picture>
  <source media="(min-width: 1024px)" srcset="/hero-wide.jpg" />
  <source media="(min-width: 640px)" srcset="/hero-medium.jpg" />
  <img class="w-full h-auto" src="/hero-mobile.jpg" alt="Hero" />
</picture>
```

## Responsive Navigation Pattern

A common pattern: hamburger menu on mobile, full nav on desktop.

```html
<!-- Mobile header with hamburger -->
<header class="sticky top-0 z-40 bg-white border-b border-gray-100">
  <div class="container mx-auto px-4 h-16 flex items-center justify-between">
    <!-- Logo -->
    <a href="/" class="text-xl font-bold text-gray-900">Brand</a>

    <!-- Desktop nav — hidden on mobile -->
    <nav
      class="hidden md:flex items-center gap-8 text-sm font-medium text-gray-600"
    >
      <a class="hover:text-gray-900 transition-colors" href="/features"
        >Features</a
      >
      <a class="hover:text-gray-900 transition-colors" href="/pricing"
        >Pricing</a
      >
      <a class="hover:text-gray-900 transition-colors" href="/docs">Docs</a>
    </nav>

    <!-- CTA — different sizes per breakpoint -->
    <div class="flex items-center gap-3">
      <a
        href="/login"
        class="hidden sm:inline text-sm text-gray-600 hover:text-gray-900"
      >
        Sign in
      </a>
      <a
        href="/signup"
        class="bg-gray-900 text-white text-sm font-medium px-4 py-2 rounded-lg
                hover:bg-gray-700 transition-colors"
      >
        Get started
      </a>

      <!-- Hamburger — visible on mobile only -->
      <button
        class="md:hidden p-2 text-gray-600 hover:text-gray-900"
        aria-label="Open menu"
      >
        <svg
          class="w-5 h-5"
          fill="none"
          stroke="currentColor"
          viewBox="0 0 24 24"
        >
          <path
            stroke-linecap="round"
            stroke-linejoin="round"
            stroke-width="2"
            d="M4 6h16M4 12h16M4 18h16"
          />
        </svg>
      </button>
    </div>
  </div>
</header>
```

## Arbitrary Breakpoints

For one-off cases where the default breakpoints don't quite fit, use `min-[]` and `max-[]`:

```html
<!-- Apply only between specific pixel ranges -->
<div class="min-[900px]:grid-cols-2 min-[1200px]:grid-cols-3">
  Custom breakpoint grid
</div>

<!-- Max-width breakpoints (like desktop-first) -->
<div class="text-lg max-[600px]:text-sm">Smaller text only below 600px</div>

<!-- Combining with regular breakpoints -->
<div class="grid grid-cols-1 min-[500px]:grid-cols-2 lg:grid-cols-4 gap-4">
  Product grid with micro-breakpoint
</div>
```

:::caution[Arbitrary breakpoints and consistency]
Arbitrary breakpoints are useful for edge cases, but overusing them leads to inconsistency. Prefer the standard breakpoint tokens for the bulk of your responsive work.
:::

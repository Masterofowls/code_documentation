Three.js is an open-source JavaScript library that abstracts the low-level WebGL API into a friendly, scene-graph-based framework. Instead of writing hundreds of lines of raw shader code, you describe a scene with cameras, lights, meshes, and materials — and Three.js compiles that into GPU draw calls for you.

```js
import * as THREE from 'three';

const scene    = new THREE.Scene();
const camera   = new THREE.PerspectiveCamera(75, innerWidth / innerHeight, 0.1, 1000);
const renderer = new THREE.WebGLRenderer({ antialias: true });

renderer.setSize(innerWidth, innerHeight);
document.body.appendChild(renderer.domElement);

const cube = new THREE.Mesh(
  new THREE.BoxGeometry(1, 1, 1),
  new THREE.MeshStandardMaterial({ color: '#4f46e5' }),
);
scene.add(cube);
camera.position.z = 3;

const light = new THREE.DirectionalLight('#ffffff', 2);
light.position.set(5, 5, 5);
scene.add(light);

function animate() {
  requestAnimationFrame(animate);
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;
  renderer.render(scene, camera);
}
animate();
```

## WebGL Abstraction

WebGL is a JavaScript API that exposes OpenGL ES 2.0 / 3.0 to the browser. It is extraordinarily powerful but extremely verbose:

```js
// Raw WebGL — just to draw one triangle
const gl = canvas.getContext('webgl2');
const vsSource = `#version 300 es
  in vec4 aPosition;
  void main() { gl_Position = aPosition; }`;
const fsSource = `#version 300 es
  precision highp float;
  out vec4 fragColor;
  void main() { fragColor = vec4(0.31, 0.27, 0.90, 1.0); }`;
// ... 60+ more lines of boilerplate to compile, link, buffer, and draw
```

Three.js wraps all of this behind a clean API:

```js
// Three.js — same triangle
const geo = new THREE.BufferGeometry();
geo.setFromPoints([
  new THREE.Vector3(-0.5, -0.5, 0),
  new THREE.Vector3(0.5, -0.5, 0),
  new THREE.Vector3(0, 0.5, 0),
]);
scene.add(new THREE.Mesh(geo, new THREE.MeshBasicMaterial({ color: '#4f46e5' })));
```

## React Three Fiber (R3F)

React Three Fiber is a React renderer for Three.js. It lets you build 3D scenes declaratively using JSX:

```js
import { Canvas } from '@react-three/fiber';
import { OrbitControls } from '@react-three/drei';

function Box() {
  return (
    <mesh rotation={[0.5, 0.5, 0]}>
      <boxGeometry args={[1, 1, 1]} />
      <meshStandardMaterial color="#4f46e5" roughness={0.3} metalness={0.2} />
    </mesh>
  );
}

export default function App() {
  return (
    <Canvas camera={{ position: [0, 0, 3] }}>
      <ambientLight intensity={0.5} />
      <directionalLight position={[5, 5, 5]} intensity={1.5} />
      <Box />
      <OrbitControls />
    </Canvas>
  );
}
```

## Three.js vs Babylon.js vs PlayCanvas

| Feature | Three.js | Babylon.js | PlayCanvas |
|---|---|---|---|
| **License** | MIT | Apache 2.0 | MIT |
| **Bundle size** | ~600 KB | ~2 MB | ~1 MB |
| **API style** | Imperative / R3F declarative | Imperative + GUI tools | Imperative + cloud editor |
| **TypeScript** | ✅ Full types | ✅ Written in TS | ✅ Full types |
| **Physics** | Via Cannon/Rapier/Ammo | Cannon/Ammo/Havok built-in | Ammo/PhysX built-in |
| **Audio** | Via Web Audio API | Built-in audio engine | Built-in audio engine |
| **GUI editor** | ❌ (use lil-gui/R3F) | ✅ Babylon Inspector | ✅ Cloud-based editor |
| **VR / AR** | ✅ WebXR via examples | ✅ Built-in XR | ✅ Built-in XR |
| **Learning curve** | Gentle (great docs) | Moderate | Moderate |
| **Community** | Largest | Large | Medium |
| **npm downloads/wk** | ~1.5M | ~200K | ~80K |
| **Best for** | Creative web, visualisation, R3F | Games, enterprise apps | Team-based game dev |

:::tip[Three.js or Babylon.js?]
Choose **Three.js** when you want the most community support, the largest ecosystem of add-ons, or you are using React (via R3F). Choose **Babylon.js** when you need a built-in physics engine, a visual inspector, or are building a full game with audio.
:::

## Use Cases

### 3D Product Viewers

E-commerce sites use Three.js to display products in 3D with texture zooming, rotation, and configurable colours:

```js
// Load a GLTF product model
import { GLTFLoader } from 'three/addons/loaders/GLTFLoader.js';

const loader = new GLTFLoader();
loader.load('/models/shoe.glb', ({ scene: model }) => {
  scene.add(model);
  // Change colour by updating material
  model.traverse(child => {
    if (child.isMesh && child.name === 'Sole') {
      child.material.color.set('#ff6b35');
    }
  });
});
```

### Data Visualisation

Three.js is excellent for 3D charts, globe visualisations, and network graphs:

```js
// Plot 3D scatter data
const geometry  = new THREE.BufferGeometry();
const positions = new Float32Array(data.flatMap(d => [d.x, d.y, d.z]));
geometry.setAttribute('position', new THREE.BufferAttribute(positions, 3));
const points = new THREE.Points(geometry, new THREE.PointsMaterial({
  size: 0.05, color: '#4f46e5',
}));
scene.add(points);
```

## Browser and Device Support

Three.js requires WebGL 1.0 as a minimum. WebGL 2.0 (available in all modern browsers since 2021) unlocks additional features:

| Feature | WebGL 1 | WebGL 2 |
|---|---|---|
| Float textures | Extension | Native |
| Multiple render targets | Extension | Native |
| Instanced rendering | Extension | Native |
| VAOs | Extension | Native |
| 3D textures | ❌ | ✅ |
| Transform feedback | ❌ | ✅ |

Three.js automatically uses WebGL 2 when available and falls back to WebGL 1 gracefully.

```js
// Check WebGL 2 availability
if (renderer.capabilities.isWebGL2) {
  console.log('WebGL 2 active — all features available');
}
```

## Performance Baseline

Before optimising, understand what your hardware can handle:

| Device | Target FPS | Max Draw Calls | Max Triangles |
|---|---|---|---|
| High-end desktop GPU | 144 | 1000+ | 10M+ |
| Mid-range desktop GPU | 60 | 500 | 3M |
| Laptop (integrated) | 60 | 200 | 1M |
| Mobile flagship | 60 | 100 | 500K |
| Mobile mid-range | 30 | 50 | 200K |

Monitor performance with the built-in `stats.js` panel and Chrome DevTools GPU profiling. Key metrics to watch:

- **Draw calls** — each `Mesh` with a unique material is a draw call; merge meshes to reduce
- **Triangle count** — shown in the Three.js renderer info panel
- **Memory** — check `renderer.info.memory` for geometry and texture counts
- **Frame time** — target < 16.7ms for 60 FPS, < 6.9ms for 144 FPS

```js
// Print renderer stats each second
setInterval(() => {
  const { render, memory } = renderer.info;
  console.log(
    `Calls: ${render.calls} | Tris: ${render.triangles} | ` +
    `Geometries: ${memory.geometries} | Textures: ${memory.textures}`
  );
}, 1000);
```

## Quick-Start Checklist

<Steps>
1. Install Three.js with Vite: `npm create vite@latest my-app -- --template vanilla && npm i three`
2. Create a `Scene`, `PerspectiveCamera`, and `WebGLRenderer`
3. Add at least one `AmbientLight` + `DirectionalLight`
4. Create a `Mesh` with a geometry and `MeshStandardMaterial`
5. Start the `requestAnimationFrame` loop
6. Add `OrbitControls` for camera interaction
7. Handle `window.resize` to keep aspect ratio correct
8. Enable `renderer.shadowMap.enabled = true` and set `castShadow` / `receiveShadow`
</Steps>

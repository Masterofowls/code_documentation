XPath and XSLT are two complementary W3C standards that together form a powerful XML processing pipeline. **XPath** is a query language for selecting nodes from an XML tree. **XSLT** is a transformation language that uses XPath to match and rewrite XML into any other text format.

## XPath Basics

An XPath **location path** selects a set of nodes from the document tree. It is composed of **steps** separated by `/`.

```xml
/bookstore/book/title          (: absolute path from root :)
bookstore/book/title           (: relative path :)
//title                        (: all <title> elements anywhere :)
/bookstore/book[1]/title       (: first book's title :)
/bookstore/book/@category      (: category attributes of all books :)
```

---

## XPath Axes

An axis specifies the **direction of traversal** from the context node. The axis name is separated from the node test by `::`.

| Axis | Selects |
|---|---|
| `self::` | The context node itself |
| `child::` | Direct children |
| `parent::` | The parent node |
| `ancestor::` | All ancestors (parent, grandparent…) |
| `ancestor-or-self::` | Self plus all ancestors |
| `descendant::` | All descendants (children, grandchildren…) |
| `descendant-or-self::` | Self plus all descendants (`//` is shorthand) |
| `following-sibling::` | Siblings after the context node |
| `preceding-sibling::` | Siblings before the context node |
| `following::` | All nodes after context node in document order |
| `preceding::` | All nodes before context node in document order |
| `attribute::` | Attributes of the context node (`@` is shorthand) |
| `namespace::` | Namespace nodes of the context node |

```xml
/bookstore/child::book           (: same as /bookstore/book :)
//book/attribute::category       (: same as //book/@category :)
//title/parent::book             (: book elements that have a title child :)
//book[2]/following-sibling::book (: books after the second :)
//book[2]/preceding-sibling::book (: books before the second :)
//author/ancestor::bookstore     (: the bookstore root :)
```

---

## Abbreviated Syntax

XPath provides shortcuts for the most common axis/node combinations:

| Abbreviated | Full Form | Meaning |
|---|---|---|
| `node` | `child::node` | Direct child named `node` |
| `//` | `/descendant-or-self::node()/` | Any depth |
| `.` | `self::node()` | Current node |
| `..` | `parent::node()` | Parent node |
| `@attr` | `attribute::attr` | Attribute named `attr` |

## Predicates

Predicates filter a node-set using `[expression]` syntax:

```xml
/bookstore/book[1]                    (: first book (1-indexed) :)
/bookstore/book[last()]               (: last book :)
/bookstore/book[position() <= 2]      (: first two books :)
/bookstore/book[@category='fiction']  (: books with category=fiction :)
/bookstore/book[price > 20]           (: books costing more than 20 :)
/bookstore/book[author and price]     (: books that have both author and price :)
//book[count(author) > 1]             (: books with multiple authors :)
//title[@lang='fr']                   (: French titles :)
/bookstore/book[@id='3']/author[2]   (: second author of book id=3 :)
```

### Chaining Predicates

Multiple predicates are applied left to right:

```xml
/bookstore/book[@category='technical'][price > 30]
(: technical books costing more than 30 :)
```

## XPath Functions

### String Functions

```xml
string(//book[1]/title)              (: get text value as string :)
string-length(//book[1]/title)       (: character count :)
concat(//author, ' — ', //year)      (: string concatenation :)
contains(//title, 'Code')            (: boolean — true if substring found :)
starts-with(//title, 'The')          (: boolean :)
ends-with(//title, 'Code')           (: XPath 2.0+ :)
substring(//title, 1, 4)             (: first 4 characters :)
substring-after(//title, 'The ')     (: text after "The " :)
substring-before(//title, ' ')       (: text before first space :)
translate(//title, 'abc', 'ABC')     (: character replacement :)
normalize-space(//title)             (: collapse whitespace :)
upper-case(//title)                  (: XPath 2.0+ :)
lower-case(//title)                  (: XPath 2.0+ :)
matches(//title, '^The', 'i')        (: XPath 2.0+ regex :)
```

### Number Functions

```xml
number(//book[1]/price)              (: convert to number :)
sum(//price)                         (: sum all price values :)
floor(//price)                       (: round down :)
ceiling(//price)                     (: round up :)
round(//price)                       (: round to nearest :)
```

### Node-set Functions

```xml
count(//book)                        (: number of book nodes = 4 :)
count(//author)                      (: total author nodes = 5 :)
position()                           (: position in current node-set :)
last()                               (: size of current node-set :)
name(//book[1])                      (: qualified name = "book" :)
local-name(//book[1])                (: local name without prefix :)
namespace-uri(//book[1])             (: namespace URI :)
id('1')                              (: element with ID attribute = "1" :)
```

### Boolean Functions

```xml
not(@category)                       (: books without a category attribute :)
boolean(//book)                      (: true if node-set is non-empty :)
true()                               (: literal true :)
false()                              (: literal false :)
```

## XPath 2.0 and 3.1

XPath 2.0 (used in XSLT 2.0 and XQuery 1.0) introduced major improvements:

- **Sequences** — every value is an ordered sequence (replaces node-sets)
- **Types** — full XML Schema type system integration
- **For expressions** — `for $book in //book return $book/title`
- **If-then-else** — `if (//book) then 'has books' else 'empty'`
- **Regular expressions** — `matches()`, `replace()`, `tokenize()`
- **Range expressions** — `(1 to 10)`
- **Distinct values** — `distinct-values(//category)`

XPath 3.1 added maps, arrays, and arrow operators (`?`, `=>`) for modern functional-style expressions.

## XSLT Stylesheet Structure

An XSLT stylesheet is itself a valid XML document. It contains template rules that match XML nodes and produce output.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0"
  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <!-- Output declaration -->
  <xsl:output method="html" version="5.0"
              encoding="UTF-8" indent="yes"/>

  <!-- Root template -->
  <xsl:template match="/">
    <!-- content goes here -->
  </xsl:template>

</xsl:stylesheet>
```

## Core XSLT Instructions

| Instruction | Purpose |
|---|---|
| `xsl:template` | Define a template rule |
| `xsl:apply-templates` | Process child nodes recursively |
| `xsl:call-template` | Call a named template |
| `xsl:value-of` | Output the string value of an expression |
| `xsl:copy-of` | Copy a node-set verbatim |
| `xsl:copy` | Shallow copy current node |
| `xsl:for-each` | Iterate over a node-set |
| `xsl:if` | Conditional output |
| `xsl:choose` | Multi-branch conditional |
| `xsl:sort` | Sort nodes during iteration |
| `xsl:variable` | Declare a variable |
| `xsl:param` | Declare a parameter |
| `xsl:with-param` | Pass a parameter |
| `xsl:element` | Create an element dynamically |
| `xsl:attribute` | Create an attribute dynamically |
| `xsl:text` | Output literal text |
| `xsl:comment` | Output an XML comment |
| `xsl:message` | Diagnostic output (like `console.log`) |
| `xsl:import` | Import another stylesheet |
| `xsl:include` | Include another stylesheet |

## Full Transform Example — XML to HTML

This XSLT converts our bookstore XML into an HTML page:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0"
  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html" encoding="UTF-8" indent="yes"
              doctype-system="about:legacy-compat"/>

  <!-- Parameter for filtering by category -->
  <xsl:param name="filterCategory" select="''"/>

  <!-- Root template — builds the page shell -->
  <xsl:template match="/">
    <html lang="en">
      <head>
        <meta charset="UTF-8"/>
        <title>Bookstore Catalogue</title>
        <style>
          body { font-family: sans-serif; max-width: 900px; margin: 2rem auto; }
          table { border-collapse: collapse; width: 100%; }
          th, td { border: 1px solid #ccc; padding: 0.5rem 1rem; text-align: left; }
          th { background: #4f46e5; color: white; }
          tr:nth-child(even) { background: #f9fafb; }
          .badge { padding: 2px 8px; border-radius: 4px; font-size: 0.8em; }
          .fiction { background: #fef3c7; color: #92400e; }
          .technical { background: #dbeafe; color: #1e40af; }
        </style>
      </head>
      <body>
        <h1>Bookstore Catalogue</h1>
        <p>Total books: <xsl:value-of select="count(//book)"/></p>
        <p>Total value: $<xsl:value-of select="format-number(sum(//price), '0.00')"/></p>
        <xsl:apply-templates select="bookstore"/>
      </body>
    </html>
  </xsl:template>

  <!-- Template for the bookstore element -->
  <xsl:template match="bookstore">
    <table>
      <thead>
        <tr>
          <th>ID</th>
          <th>Title</th>
          <th>Author(s)</th>
          <th>Year</th>
          <th>Price</th>
          <th>Category</th>
        </tr>
      </thead>
      <tbody>
        <!-- Sort books by price descending -->
        <xsl:apply-templates select="book">
          <xsl:sort select="price" data-type="number" order="descending"/>
        </xsl:apply-templates>
      </tbody>
    </table>
  </xsl:template>

  <!-- Template for each book -->
  <xsl:template match="book">
    <tr>
      <td><xsl:value-of select="@id"/></td>
      <td>
        <xsl:value-of select="title"/>
        <xsl:if test="title/@lang != 'en'">
          <em> (<xsl:value-of select="title/@lang"/>)</em>
        </xsl:if>
      </td>
      <td>
        <!-- Handle multiple authors -->
        <xsl:for-each select="author">
          <xsl:value-of select="."/>
          <xsl:if test="position() != last()"><br/></xsl:if>
        </xsl:for-each>
      </td>
      <td><xsl:value-of select="year"/></td>
      <td>$<xsl:value-of select="format-number(price, '0.00')"/></td>
      <td>
        <xsl:call-template name="category-badge">
          <xsl:with-param name="cat" select="@category"/>
        </xsl:call-template>
      </td>
    </tr>
  </xsl:template>

  <!-- Named template for category badge -->
  <xsl:template name="category-badge">
    <xsl:param name="cat"/>
    <xsl:choose>
      <xsl:when test="$cat = 'fiction'">
        <span class="badge fiction">Fiction</span>
      </xsl:when>
      <xsl:when test="$cat = 'technical'">
        <span class="badge technical">Technical</span>
      </xsl:when>
      <xsl:otherwise>
        <span class="badge"><xsl:value-of select="$cat"/></span>
      </xsl:otherwise>
    </xsl:choose>
  </xsl:template>

</xsl:stylesheet>
```

## xsl:variable and xsl:param

Variables are immutable once set (no reassignment):

```xml
<!-- Global variable -->
<xsl:variable name="taxRate" select="0.20"/>

<!-- Computed variable -->
<xsl:variable name="expensiveBooks"
  select="//book[price &gt; 30]"/>

<!-- RTF (Result Tree Fragment) variable -->
<xsl:variable name="disclaimer">
  <p>Prices subject to change without notice.</p>
</xsl:variable>

<!-- Use the variable -->
<xsl:value-of select="$taxRate"/>
<xsl:copy-of select="$disclaimer"/>
```

Parameters are like variables but can be set from outside the stylesheet:

```xml
<!-- Stylesheet-level parameter with default -->
<xsl:param name="language" select="'en'"/>

<!-- Usage -->
<xsl:apply-templates select="//title[@lang = $language]"/>
```

## XQuery Basics

XQuery is to XML what SQL is to relational databases. It uses the **FLWOR** expression (`for`, `let`, `where`, `order by`, `return`):

```xml
(: XQuery — find all technical books over $30 sorted by price :)
for $book in /bookstore/book
where $book/@category = 'technical' and $book/price > 30
order by $book/price descending
return
  <result>
    <title>{ $book/title/text() }</title>
    <price>{ $book/price/text() }</price>
  </result>
```

XQuery is supported natively by databases like BaseX, eXist-db, MarkLogic, and Oracle XML DB.

:::tip[XSLT 2.0+ and XQuery Share XPath 2.0]
Once you learn XPath 2.0 expressions, you can use them in both XSLT 2.0+ stylesheets and XQuery 1.0+ queries. The expression language is identical — only the surrounding framework differs.
:::
